## 二、设计哲学

### 2.1 三大设计原则（宾尼小康）

**原则 1 · 主色调铁律**
> 整个 App 严格使用「淡天蓝」单一主色调，禁止引入红/橙/绿等对撞色作为视觉主体；
> 强调辅助色（暖橙/红/绿/黄）严格控量，仅用于 PRD 第七章规定的少数语义场景。

**原则 2 · 病历卡感**
> AI 卡片统一采用「白底 + 左侧 3px 天蓝竖线」的"病历卡"形态，强化「医疗专业」品牌锚点。

**原则 3 · 中老年友好**
> 顶栏使用最舒适的 A1 淡天蓝（`#f0f9ff → #dbeafe`），避免高饱和长时间停留刺眼；
> 关键正文字号不低于 14px，关键指标数字不低于 22px。

### 2.2 风格基准
- **参考标杆**：晓医（京东健康/叮当 AI 医生）、平安好医生、春雨医生
- **气质关键词**：清新、专业、可信、亲和、低刺激
- **拒绝关键词**：花哨、彩色拼贴、霓虹、深色暗黑

## 六、交互规范

### 6.1 响应分级（统一手势体感）
| 级别 | 时长 | 用途 |
|---|---|---|
| Instant | 100ms | 按下反馈 / Hover |
| Fast | 180ms | 弹出菜单 / Toast |
| Base | 240ms | 抽屉滑入 / 页面切换 |
| Slow | 360ms | 大型转场 / 全屏弹层 |

### 6.2 流式输出规范
- 字符吐字间隔：30~50ms / 字
- 闪烁光标：1px 宽 × 字号高 / 颜色 `#0284c7` / 500ms 闪烁
- 中断按钮：常驻输入栏右侧 / 出现于"AI 正在回答"状态
- 中断后：保留已输出内容，末尾出现"⏸ 已中断生成 · 继续 / 重答"

### 6.3 手势规范
- 抽屉：右侧右滑进入 / 左滑关闭（速度 > 0.3px/ms 触发）
- 长按：500ms 触发（家人 chip 长按出管理菜单）
- 双击：禁用（避免与放大冲突）
- 上拉刷新：禁用（避免与滚动冲突）

### 6.4 动效规范
- 缓动函数统一 `cubic-bezier(0.4, 0, 0.2, 1)`（标准）
- 出现：decelerate / 消失：accelerate
- 雷达图：进场 600ms 由中心展开
- 思考态：3 圆点交替 200ms / 整体循环 600ms

### 6.5 无障碍
- 文字与背景对比度 ≥ 4.5:1（AA 级）
- 主操作按钮可点击区域 ≥ 44×44pt
- 关键指标支持 VoiceOver 朗读（如"心率 75 次/分钟，正常"）
- 字体大小支持系统跟随（Settings → 字体大小，4 档）

## 七、辅助色控量（红线规范）

### 7.1 严禁滥用清单

| 辅助色 | **仅允许场景** | **禁止场景** |
|---|---|---|
| 暖橙 `#fbbf24` | 会员等级 / 星级评价 / 倒计时高亮 | 普通按钮 / 普通图标 / 普通分类 |
| 红 `#ef4444` | 紧急徽标（铃铛红点）/ 错误重试 / 危险操作 | 普通强调 / 普通分类 / 角标计数 |
| 绿 `#22c55e` | 在线状态 / 达标徽章 / 成功 Toast | 普通按钮 / 普通图标 |
| 黄 `#eab308` | "偏高/临界"指标徽章 | 普通强调 |

### 7.2 红线原则
> **任何"既可以用主色也可以用辅助色"的场景，一律使用主色。**
> 设计评审时若发现辅助色用于上述"禁止场景"，必须打回。
