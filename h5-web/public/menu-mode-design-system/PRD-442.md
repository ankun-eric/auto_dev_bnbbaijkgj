## 1. 项目背景与目标

### 1.1 背景
- 菜单模式是老用户的主路径，当前视觉散乱、卡片密度高、缺统一规范。
- AI 模式（PRD-441）已确认采用「宾尼小康」风格，菜单模式不同步会造成强割裂。
- 三端目前各画各的，缺统一 Design Tokens。

### 1.2 目标
1. **视觉统一**：菜单模式 6 个核心屏严格落地宾尼小康风格。
2. **三端一致**：沉淀 Design Tokens + H5 组件库，三端复用，差异 ≤ 5%。
3. **平稳过渡**：灰度发布 + 舆情监控，老用户切换流失率 < 1%。
4. **可延展**：组件库支撑后续 12~18 个二级页自由延展。

### 1.3 非目标（本期不做）
- ❌ 暗黑模式
- ❌ 注册/家人管理/订单详情/会员/通知中心 等二级页不单独出图
- ❌ 不重构现有业务逻辑，仅做视觉与组件层改造

## 3. Design Tokens（三端共用）

### 3.1 颜色 Tokens（11 级天蓝色阶）
```
--sky-50:  #f0f9ff
--sky-100: #e0f2fe
--sky-200: #bae6fd
--sky-300: #7dd3fc
--sky-400: #38bdf8   ← 主色（病历卡竖线、强调）
--sky-500: #0ea5e9   ← 渐变中段
--sky-600: #0284c7
--sky-700: #0369a1
--sky-800: #075985   ← Hero 深蓝
--sky-900: #0c4a6e
--sky-950: #082f49
```

### 3.2 语义辅助色（极少用）
```
--success: #22c55e    在线/成功
--danger:  #ef4444    红点/告警
--member:  #f59e0b    会员金徽
--text-1:  #0f172a    一级正文
--text-2:  #475569    二级辅助
--text-3:  #94a3b8    三级提示
--bg-base: #ffffff
--bg-soft: #f8fafc
```

### 3.3 阴影 Tokens（4 级，禁止纯黑）
```
--shadow-1: 0 1px 2px rgba(56,189,248,.08)
--shadow-2: 0 4px 12px rgba(56,189,248,.10)
--shadow-3: 0 8px 24px rgba(56,189,248,.12)
--shadow-4: 0 16px 40px rgba(56,189,248,.16)
```

### 3.4 圆角 Tokens（5 级）
```
--radius-xs:  4px
--radius-sm:  8px
--radius-md:  12px
--radius-lg:  16px
--radius-xl:  24px   ← 主卡片/登录卡
```

### 3.5 字号 Tokens（8 级，中老年友好）
```
--fs-12: 12px   --fs-14: 14px (正文最小)   --fs-16: 16px
--fs-18: 18px   --fs-20: 20px               --fs-24: 24px
--fs-30: 30px   --fs-36: 36px (关键指标)
```

### 3.6 间距 Tokens（8 级）
```
--sp-1: 4px   --sp-2: 8px    --sp-3: 12px   --sp-4: 16px
--sp-5: 20px  --sp-6: 24px   --sp-8: 32px   --sp-10: 40px
```

### 3.7 关键铁律
- 病历卡感：白底 + 左侧 3px `--sky-400` 竖线 + `--shadow-2`
- 渐变三层级：A1 顶栏（最浅）→ 主按钮（中段）→ Hero（最深）
- 按钮高度：48px，圆角 `--radius-md`
- 字距：正文 `letter-spacing: 0.02em`
- 禁止：纯黑阴影、强饱和度色（除语义辅助色）

## 6. 灰度发布与切换策略

### 6.1 灰度节奏
| 阶段 | 流量 | 时长 | 监控 |
|---|---|---|---|
| 内测 | 0.1%（员工 + 内测群） | 2 天 | 崩溃率/严重 bug |
| 灰度 1 | 10% | 3 天 | 留存、舆情、客诉量 |
| 灰度 2 | 30% | 4 天 | 同上 + 关键路径转化率 |
| 全量 | 100% | — | 持续监控 1 周 |

### 6.2 回滚预案
- H5：保留旧版静态资源，开关 30 秒切回。
- 小程序：保留上一审核版本，紧急回滚走小程序后台。
- Flutter：服务端下发 UI 配置开关，或下版本回滚。

### 6.3 舆情监控
- 客服关键词：「界面」「看不懂」「难用」「找不到」
- 客诉量阈值：超过日均 1.5 倍 → 暂停灰度扩量

## 7. 验收标准

### 7.1 视觉验收
- 6 屏与设计稿像素级对齐，差异 < 3%
- 三端同屏对比，UI 差异 < 5%
- 所有页面通过 Design Tokens 检查（无 hard-code 颜色/字号/圆角）

### 7.2 功能验收
- 主路径功能 0 回归 bug
- 灰度阶段 P0/P1 bug = 0

### 7.3 性能验收
- H5 首屏 LCP < 2.5s（4G）
- 小程序首屏 < 2s
- Flutter 冷启动 < 1.8s

### 7.4 体验验收
- 中老年用户测试：5 位 50+ 用户走核心路径
- 字号/对比度通过 WCAG AA
