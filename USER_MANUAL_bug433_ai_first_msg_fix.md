# BUG-433 修复 · AI 对话首条消息丢失 · 用户体验手册

> 修复编号：BUG-433-AI-HOME-VOICE-FIRST-MSG-LOST
> 适用产品：bini-health
> 修复范围：H5 主入口（AI 首页）+ 后端 FastAPI
> 部署日期：2026-05-10

## 1. 修复内容简介

修复了 AI 首页（"AI 对话"主入口）的「**首条消息丢失**」问题，覆盖三种发送方式：

- **文字输入**：手动键入并点发送 / 回车
- **语音输入**：长按语音录制 → 松手自动 ASR → 自动发送
- **预设快捷按钮**：点击"今天我感觉..."等预设问题

修复后，**任意方式发送的第一条用户消息**都会：

1. 立即出现在对话气泡中（乐观渲染）
2. 强制持久化到数据库（即使 AI 流式中断或网络断开）
3. AI 回复消息通过 `parent_id` 关联到对应用户消息
4. 数据库新增 `source` 字段记录发送来源（`text` / `voice` / `preset`）

## 3. 复现回归测试步骤

### 场景 A：预设按钮发送

1. 打开 AI 首页（首次进入或会话超过 30 分钟空闲）
2. 直接点击任意预设快捷按钮，例如「感冒了吃什么药比较好？」
3. **预期**：用户气泡立即出现 → AI 流式回复

### 场景 B：语音输入发送

1. 打开 AI 首页（同上前置条件）
2. 长按底部语音按钮 → 说话 → 松手
3. **预期**：ASR 识别文字回填 → 用户气泡立即出现 → AI 流式回复

### 场景 C：文字输入发送

1. 打开 AI 首页
2. 输入框键入"你好" → 点击发送
3. **预期**：用户气泡立即出现 → AI 流式回复

### 场景 D：空闲 30 分钟边界回归

1. 完成一次对话后等待 ≥30 分钟（或调小 `idleTimeout` 调试）
2. 用任一方式发送下一条消息
3. **预期**：自动开新会话；新会话中**首条用户消息不丢失**（核心修复点）

## 4. 数据库验证（可选，验证强约束）

部署服务器 MySQL 中执行：

```sql
SELECT id, session_id, role, source, parent_id, LEFT(content,30) AS preview, created_at
FROM chat_messages
ORDER BY id DESC LIMIT 10;
```

**预期**：
- 每个 `assistant` 消息都有 `parent_id` 指向同会话的 `user` 消息
- `source` 字段值为 `text` / `voice` / `preset` 之一

## 5. 已知限制

- 历史孤立 AI 消息（修复前产生）**未做 backfill**，仅做了向前兼容（`parent_id` 允许 NULL）
- 小程序 / Flutter App 因不存在该入口（聊天详情页都是带会话进入的），未做客户端代码变更，但后端 `source` 字段对它们也是兼容的（默认 `text`）
