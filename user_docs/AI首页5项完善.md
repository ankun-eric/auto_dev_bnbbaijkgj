## 使用说明

### 一、查看 AI 首页顶栏改造（Bug #1 + #2 + #3a）

1. 用手机浏览器（或电脑模拟手机视图）打开 [AI 健康管家首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home/) 并登录
2. 观察 **顶栏左上角的汉堡按钮（☰）**：
   - 三条横线**整体左对齐**
   - 第 1、2 条等长（约占图标宽度的 76%）
   - 第 3 条更短（约占图标宽度的 52%），向右收回明显距离
3. 观察 **顶栏中央**：
   - "小康"两字（如已设置自定义签名则显示该签名）**水平居中**于整个顶栏，与窗口中线对齐
   - 三横线与"小康"之间留出明显空白
4. 观察 **顶栏右上角**：
   - 原来的"⋯"三点改成了 **"圆圈 + 号"图标**（微信主界面右上角的"+"按钮样式）
   - 点击图标依然弹出原来的更多菜单（扫一扫 / 字体大小 / 立即分享 / 会员中心），功能与跳转**与改造前完全一致**

### 二、查看右上"+"菜单 · 会员中心角标已删（Bug #5）

1. 点击 AI 首页右上角的 **"+"圆圈按钮**
2. 在弹出菜单中找到 **"会员中心"** 这一项：
   - 文字仍为**金色加粗**，左侧仍为 👑 皇冠图标（未改）
   - 文字右侧的 **"新"字小角标已被移除**（这就是本次 Bug #5 的修复）
3. 点击"会员中心"可进入会员中心页（跳转逻辑不变）

### 三、查看侧边抽屉"会员中心"副标题（Bug #4）

1. 在 AI 首页点击 **顶栏左上角的汉堡按钮（☰）** 打开左侧抽屉
2. 在抽屉中找到 **会员中心** 入口（带金色 👑 图标）
3. "会员中心"标题下方副标题文字：
   - 原："权益管理与升级"
   - 新：**"权益升级"** ← 本次修复后的文案
   - 字号 12px、颜色 `#8B6914`（金棕色）与之前保持一致

### 四、铃铛抽屉与"用药提醒-全部"页数据一致性验证（Bug #3b）

1. **准备**：登录后确保账号下今天有用药计划（如无，可在【档案管理 → 健康数据 → 用药提醒】中先添加一条今日用药计划）
2. **打开铃铛抽屉**：
   - 在 AI 首页右侧浮动区域找到 **🔔 铃铛图标** 并点击
   - 抽屉中"💊 用药提醒"区块列出今天**待打卡**的所有用药记录
3. **对照"用药提醒-全部"页**：
   - 在铃铛抽屉中点击 **"用药提醒"区块右上角的 `查看全部 →`**
   - 跳到 [用药提醒-全部 页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home/medication-reminder/)
   - 对比 **"待服用"统计数字** 与 **铃铛抽屉"用药提醒 (N)" 的 N**：两者应**完全相等**
   - 对比每条用药的**状态文案 / 颜色 / 时间**：两端应**完全一致**（待服用 / 即将服用 / 已服用 / 已超时）
4. **打卡同步验证**：
   - 回到铃铛抽屉，对某条用药点击右侧的 **"✓ 已服用"** 按钮
   - 该条用药立即变为已打卡（按钮变灰），铃铛红点数 -1
   - 切到"用药提醒-全部"页（或先关闭抽屉再进入），刷新后可以看到**同一条**用药状态变为 **"已服用 HH:MM"**，状态颜色变绿
   - 数据完全打通，不存在两端打卡后状态不同步的旧 Bug

> 💡 **技术变更说明**：本次 Bug #3b 通过 **前端兼容层 adapter** 把铃铛抽屉的数据源切换到 `/api/medication-plans/today`，与"用药提醒-全部"页使用同一接口；打卡接口也统一到 `/api/medication-check-in`。后端无需改动，铃铛 UI 与交互保持原样。

## 注意事项

1. **必须强制刷新浏览器以获取最新版**：浏览器可能缓存了旧版 HTML/JS。请使用：
   - Windows：`Ctrl + Shift + R` 或 `Ctrl + F5`
   - macOS：`Cmd + Shift + R`
   - 微信内置浏览器：在右上角"⋯"中选择"刷新"，或清除微信缓存后重试
2. **顶栏标题不一定是"小康"**：如果用户在「AI 咨询配置 → 签名」中设置了自定义 AI 名称，顶栏中央显示的是自定义签名（最长 8 个汉字，超出加 `…` 省略号）。本次修复保证**无论显示什么文案都严格水平居中**
3. **铃铛红点数 = "用药提醒-全部"页"待服用"数**：从此两边永久一致，不再会出现"铃铛红点显示 N 条，点开抽屉是 0 条"或"两端打卡状态不同步"的旧问题
4. **本次完善不影响**：
   - 任何 API 接口的请求/响应字段
   - 后端业务逻辑
   - 数据库 schema
   - 移动端 APP / 微信小程序 / 管理后台
5. **服务端非 UI 自动化测试**：本次部署到服务器后，已自动验证：
   - 容器内 `.next` 编译产物中 `ai-home-more-icon-plus-circle`（新+号图标）出现 ≥ 1 次
   - 编译产物中 `权益升级` 出现 ≥ 1 次、`权益管理与升级` = 0 次
   - 编译产物中 `ai-home-more-menu-item-新角标` = 0 次（已删）
   - 编译产物中 `medication-plans/today` 出现 ≥ 1 次（铃铛已切换到新接口）
   - 外部 HTTPS GET 首页 / AI 首页 / 用药提醒页 / 登录页 / 后端健康检查全部 HTTP 200
6. **本次没有数据迁移、没有破坏性变更**，可随时回滚到前一版本
