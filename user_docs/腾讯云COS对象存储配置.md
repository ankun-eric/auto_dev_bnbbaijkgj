## 功能简介

本次更新为 bini-health 健康管理系统接入了**腾讯云COS对象存储**功能，主要包括：

1. **存储配置管理**：在管理后台配置COS连接信息、CDN加速域名、上传限制等
2. **全系统上传改造**：所有文件上传统一走COS通道，支持本地存储自动回退
3. **上传进度条**：H5、小程序、Flutter三端上传时显示进度条
4. **上传大小限制**：按业务模块配置文件上传大小限制，前端校验+后端校验
5. **数据迁移工具**：一键将本地存储的文件批量迁移到COS
6. **文件删除联动**：业务数据删除时同步清理COS上的文件
7. **CDN加速支持**：可配置CDN域名加速文件访问

## 使用说明

### 一、存储配置（仅超级管理员可见）

#### 1. 进入存储配置页面

1. 使用超级管理员账号登录 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 在左侧菜单中找到 **系统管理** → **存储配置**
3. 点击进入存储配置页面

#### 2. 配置COS基础信息

在「COS基础配置」区块中填写以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| SecretId | 腾讯云API密钥ID | AKIDxxxxxxxxxxxxxxxx |
| SecretKey | 腾讯云API密钥Key（脱敏显示） | 输入后显示为 **** |
| Bucket | COS存储桶名称 | my-bucket-1250000000 |
| Region | COS地域 | 下拉选择，如 ap-guangzhou |
| 统一路径前缀 | 文件存储路径前缀（可选） | prod/ |

#### 3. 配置CDN加速（可选）

在「CDN加速配置」区块中填写：

| 配置项 | 说明 |
|--------|------|
| CDN域名 | 绑定的CDN加速域名，如 cdn.example.com |
| CDN协议 | 选择 https 或 http |

> 配置CDN后，文件访问URL会自动使用CDN域名加速

#### 4. 配置上传限制

在「上传限制配置」区块中，可以按业务模块分别设置文件上传大小限制：

| 模块 | 默认限制 |
|------|----------|
| 用户头像 | 2 MB |
| 健康记录图片 | 10 MB |
| 健康记录视频 | 100 MB |
| 品牌LOGO | 2 MB |
| 轮播图/广告图 | 5 MB |
| 体检报告图片 | 10 MB |
| 其他文件 | 50 MB |

#### 5. 启用COS存储

启用COS存储需要按照以下三步流程操作：

1. **填写配置**：完成COS基础配置的填写
2. **测试连接**：点击「测试连接」按钮，系统会验证COS配置是否正确
3. **开启存储**：测试通过后，「启用COS」开关变为可操作状态，开启后新上传的文件将走COS通道

> ⚠️ 测试连接通过前，启用开关不可操作（置灰状态）

### 二、数据迁移

如果系统中已有文件存储在本地，可以使用数据迁移功能将文件批量迁移到COS：

#### 迁移步骤

1. **扫描文件**：在存储配置页面底部「数据迁移」区块，点击「扫描待迁移文件」
2. **查看汇总**：系统会按模块分组展示待迁移文件（文件数、总大小）
3. **选择模块**：勾选需要迁移的模块（默认全选）
4. **开始迁移**：确认后点击「开始迁移」
5. **查看进度**：页面实时显示迁移进度条、成功/失败数、当前文件
6. **处理失败**：如有失败文件，查看失败原因后点击「重试失败文件」

> 迁移过程中可以离开页面，回来后仍能看到进度

### 三、文件管理统计

在「文件管理」TAB页中，可以查看系统中的文件总数统计。该统计基于数据库计算，不消耗COS API流量。

### 四、上传体验

COS启用后，所有端（管理后台、H5、小程序、Flutter App）的文件上传行为如下：

- **上传前**：自动校验文件大小是否超过限制，超限时提示「文件大小超过限制（最大 X MB）」
- **上传中**：显示上传进度条（实时更新）
- **上传后**：文件存储在COS上，URL自动使用CDN域名（如已配置）
- **COS故障时**：自动回退到本地存储，不影响正常使用

## 注意事项

1. **权限要求**：存储配置功能仅对超级管理员可见，普通管理员无法看到该菜单
2. **COS配置获取**：需要先在 [腾讯云控制台](https://console.cloud.tencent.com/cos) 创建存储桶并获取API密钥
3. **CDN配置**：CDN域名需要在腾讯云控制台单独配置并绑定到COS存储桶
4. **多环境隔离**：建议不同环境（开发/测试/正式）使用不同的COS Bucket或路径前缀
5. **数据迁移注意**：
   - 迁移过程中系统正常运行，不影响用户使用
   - 同一时间只能运行一个迁移任务
   - 迁移完成后，旧的本地文件URL会自动更新为COS/CDN地址
6. **费用提醒**：COS存储和CDN加速会产生腾讯云费用，请关注用量
7. **COS未启用时**：系统正常使用本地存储，功能不受任何影响
