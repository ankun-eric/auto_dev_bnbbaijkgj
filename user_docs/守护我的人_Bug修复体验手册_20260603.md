## 功能简介

本次针对「健康档案 → 守护我的人」修复了 3 个体验 Bug：

1. **新建邀请流程改为「分两步」**：先填写「关系 + 名字」，点「确认」生成邀请记录后才展示二维码，符合"先填资料 → 再出二维码"的直觉预期。
2. **再次邀请不会再覆盖原有邀请**：现在每点一次「邀请别人守护我」，都会**新增一条**邀请记录，原来的待确认邀请不会被自动取消。在配额（默认 3 个）允许范围内，可以同时挂多条待对方扫码的邀请。
3. **列表卡片样式更清晰**：邀请态卡片头像显示**对方名字的首字**（不再是单调的"邀"字）；底部「取消邀请 / 查看邀请码」两个按钮独立成行，不再和头像、信息挤在一行。

## 使用说明

### 一、查看「守护我的人」列表

1. 打开 H5 项目并登录
2. 进入 **健康档案 → 守护我的人**
3. 列表顶部会显示一个**名额条**：
   - `已使用名额 0 / 3` 表示当前还没有人在守护你，最多可以有 3 位守护者（含邀请中）
   - 当名额满了（如 `3 / 3`），数字会变成**红色**，底部「邀请别人守护我」按钮**置灰不可点**，按钮下方提示「已达上限 3/3」
   - 名额上限来自运营后台的会员配置，升级会员后会自动提升

### 二、邀请新的守护者（两步流程）

1. 在列表页底部点击「**邀请别人守护我**」
2. **第一步：填写信息**
   - 选择对方与你的关系（爸爸/妈妈/爷爷/奶奶/儿子/女儿/叔叔/阿姨…，也可选「其他」自定义）
   - 输入对方的名字（必填）
   - 点击「**确认**」按钮
     - 名字为空 或 关系未选 时，按钮置灰不可点
     - 后端会立即创建一条邀请记录
3. **第二步：展示二维码**
   - 页面只显示二维码 + 「邀请对象：张三（叔叔）」+ 「24 小时内有效」+ 三个操作按钮（保存到本地 / 复制链接 / 转发微信好友）
   - 顶部左上角的「←」 和 底部「完成 / 返回列表」按钮，**都直接回到列表页**，不会回到第一步
4. 此时列表中会**新增一条邀请中的卡片**，对方扫码即可成为守护者

> 💡 提示：在不超出名额（默认 3）的前提下，你可以反复执行上述流程，**多条待确认邀请会同时挂在列表里**，相互独立。

### 三、查看已生成的二维码（不必重走流程）

1. 列表中找到对应的「邀请中」卡片
2. 点击卡片底部的「**查看邀请码**」按钮
3. 页面**弹出抽屉**展示二维码（不会切换页面）
4. 抽屉中包含：二维码 + 「邀请对象：xxx（关系）」+ 「24 小时内有效」+ 「关闭」按钮
5. 点击「关闭」或点击抽屉外的灰色遮罩即可回到列表

### 四、取消邀请

1. 列表中找到对应的「邀请中」卡片
2. 点击卡片底部的「**取消邀请**」按钮
3. 弹出二次确认弹窗：「确定取消对张三的邀请吗？取消后对方扫码将失效。」
4. 确认后，该邀请立即失效，**名额自动 -1**

### 五、达到上限时

- 已使用名额数（X）= 已激活守护者数 + 邀请中数
- 当 X ≥ Y（默认 3）时：
  - 名额条数字变红
  - 底部「邀请别人守护我」按钮置灰，下方常驻「已达上限 3/3」小字
  - 即使点击按钮，也会弹出「您的守护者已达上限」提示框，可选「升级会员」跳转到会员中心，或选「取消」关闭

## 注意事项

1. **24 小时有效期**：每条邀请二维码生成后 24 小时内有效，过期后系统会自动将其标记为过期，**不再占用名额**，列表中也不会再展示。
2. **多条 pending 共存**：本次修复后，旧的「邀请中」记录不会再被新邀请自动取消。请合理使用名额，可随时通过「取消邀请」释放名额。
3. **取消邀请会立即失效**：如果对方还没扫码，且你又点了取消，对方再扫码会提示已失效。
4. **小程序端不受影响**：本次仅修复 H5，小程序的相关流程保持原样，不需要重新下载。
5. **会员配额可调整**：默认每个用户最多 3 位守护者，运营在后台调整后立即生效，前端无需更新。
