## 功能简介

本次更新对支付宝「应用私钥」的格式校验进行**收窄**：

- **改造前**：后台同时兼容多种形态（带 PKCS#1 头尾、带 PKCS#8 头尾、纯 Base64、含脏字符等），导致报错文案歧义、用户反复踩坑。
- **改造后**：**仅接受**支付宝官方密钥工具实际产出的「中间 Base64 部分」（无 BEGIN/END 头尾）。校验更聚焦、报错更精准。

变更核心：

1. 后端：删除 PKCS#1 头尾兜底逻辑，新增 BEGIN/END 头尾检测短路报错；保留不可见字符清洗（BOM / 零宽空格 / 全角空格 / Tab / 换行 / 普通空格）。
2. 前端：「应用私钥」输入框 placeholder 与下方提示更新为直白指令型短句，明确告知「仅粘贴中间部分，不要包含 BEGIN/END 头尾」。
3. 测试：新增「含头尾必须报错」用例，覆盖 PRD §六 全部 8 个回归用例（T1~T8），服务器内 19/19 通过。

## 使用说明

### 入口路径

1. 打开 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/) 并使用管理员账号登录
2. 左侧菜单进入 **「支付配置」** 页面
3. 切换到 **「支付宝H5支付」** 或 **「支付宝APP支付」** Tab
4. 点击右上角 **「编辑」** 按钮，打开配置抽屉

### 「应用私钥」字段操作步骤（关键变化）

#### 步骤 1：从支付宝密钥工具获取「中间 Base64」内容

支付宝官方密钥工具产出的「应用私钥」文件实际只包含一长串 Base64 字符（A-Z、a-z、0-9、+、/、= 这 65 个字符），不含 `-----BEGIN PRIVATE KEY-----` 和 `-----END PRIVATE KEY-----` 两行。

#### 步骤 2：粘贴到「应用私钥（PEM）」输入框

- 输入框 placeholder 提示：**`粘贴密钥工具生成的中间 Base64 内容，不要包含 BEGIN/END 头尾`**
- 输入框下方提示：**`仅粘贴中间部分。粘了 BEGIN/END 头尾会被判为格式错误。`**
- 直接整段粘贴密钥工具产出的内容即可。粘贴时**不要**手动加 `-----BEGIN PRIVATE KEY-----` / `-----END PRIVATE KEY-----` 两行。
- 系统会自动清洗 BOM / 零宽空格 / 全角空格 / Tab / 换行 / 普通空格等不可见字符，无需用户手动整理。

#### 步骤 3：保存或保存并测试连接

- 点击「仅保存」：仅持久化配置（后端会先做 Base64 解码 + ASN.1 解析，校验通过才落库；落库时自动包装为标准 PKCS#8 PEM）。
- 点击「保存并测试连接」：先校验、再以保存后的私钥发起一次支付宝沙盒/正式连通性测试。

### 报错场景与文案对照

| 输入形态 | 校验结果 | 错误文案（后端响应） |
|----------|----------|---------------------|
| 纯中间 Base64（密钥工具产出） | ✅ 通过 | — |
| 中间 Base64 + 多余空格/换行 | ✅ 通过（自动清洗） | — |
| 中间 Base64 + BOM / 零宽 / 全角空格 | ✅ 通过（自动清洗） | — |
| 含 `-----BEGIN PRIVATE KEY-----` 完整 PKCS#8 PEM | ❌ 报错 | `请只粘贴中间 Base64 部分（去掉 BEGIN/END 两行）。` |
| 含 `-----BEGIN RSA PRIVATE KEY-----` 完整 PKCS#1 PEM | ❌ 报错 | `请只粘贴中间 Base64 部分（去掉 BEGIN/END 两行）。` |
| 仅 BEGIN/END 两行（中间空） | ❌ 报错 | `请只粘贴中间 Base64 部分（去掉 BEGIN/END 两行）。` |
| 公钥（BEGIN PUBLIC KEY） | ❌ 报错 | `请只粘贴中间 Base64 部分（去掉 BEGIN/END 两行）。` |
| 乱码字符串 / 空字符串 | ❌ 报错 | `应用私钥格式不被支持。请粘贴密钥工具生成的中间 Base64 内容。` |

## 注意事项

1. **存量用户的兼容性提示**：若您此前已配置过「应用私钥」（数据库中已是标准 PKCS#8 PEM 格式），**不会**因本次改动而失效。本次收窄仅在「保存」时生效；**已落库的私钥不会被重新校验**，签名/验签业务逻辑完全不变。
2. **重新粘贴时务必只粘中间部分**：如果您出于任何原因需要重新配置，**必须**改为粘贴密钥工具产出的「中间部分」（不带 BEGIN/END 两行）。粘贴含头尾的内容将被直接拒绝。
3. **支付宝平台公钥配置流程不变**：本次改动仅影响「应用私钥」字段，平台公钥/公钥证书等其他字段的填写规则不变。
4. **入库存储格式不变**：后端落库时仍会自动把校验通过的「中间 Base64」标准化为完整 PKCS#8 PEM（含头尾）。业务侧无感知。
