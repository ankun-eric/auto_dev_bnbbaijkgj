## 功能简介

本次更新解决了商品/卡项详情页的两个体验断层问题：

1. **时段「已约满」置灰预览**：以前用户在详情页看到的时段都显示为可点状态，要等到下单页才会被拦截。现在详情页就会**直接置灰**已约满或已过期的时段，并显示"已约满 / 已结束"文案，避免误操作。
2. **可用门店按距离排序 + 抽屉切换**：以前一个卡项绑定多家门店时，详情页只显示第一家。现在会按用户当前位置**计算距离并按近到远排序**，顶部展示距离最近的主门店卡片，点击右侧"切换 ▼"会弹出底部抽屉，可在多家门店之间快速切换。

支持端：H5、微信小程序、安卓 App、iOS App。后台 Admin 无 UI 改动。

## 使用说明

### 场景 1：在详情页查看时段是否约满

1. 任选一个**预约方式为「时段」**的商品（或卡项），打开详情页。
2. 在详情页中部找到**「可预约时段（今日）」**区域，会展示该商品今天的所有时段标签：
   - **正常可约**：白底黑字，圆角胶囊样式。
   - **已约满**：灰底灰字，标签后追加红色 "已约满" 字样，不可点击。
   - **已结束**（指今日已过期的时段）：灰底灰字，标签后追加红色 "已结束" 字样。
3. 用户对哪一段满了一目了然，避免点进下单页才发现时段不可约。

### 场景 2：在卡项详情页选择门店

1. 打开一个**适用门店有多家**的卡项详情页。
2. 系统在进入页面时会请求**定位授权**：
   - **授权成功**：自动以您所在位置为参考点，按距离从近到远排序门店。
   - **授权失败 / 拒绝**：自动按门店名升序兜底排序，并在主卡片下方提示"暂未获取位置，已按门店名排序"。
3. 顶部**主门店卡片**显示当前选中门店的：门店名、地址、距离 (如 "距您 1.32 km")。
4. 当卡项有 2 家及以上门店时，主卡片右侧会出现 **"切换 ▼"** 按钮：
   - 点击后，从屏幕底部弹出**门店选择抽屉**。
   - 抽屉中按距离升序列出全部可用门店；当前选中门店打勾标记。
   - 点击任意门店即可切换，主卡片实时同步更新；点击遮罩或抽屉外部区域可关闭抽屉。
5. 切换后的门店将作为下一步下单时的默认门店。

### 场景 3：管理员配置门店经纬度（Admin 端）

为了让"按距离排序"生效，管理员需要在后台为每家门店补录经纬度：

1. 登录 Admin 后台 → **门店管理**。
2. 编辑或新建门店，确保 `lat`（纬度）/ `lng`（经度）已填写。
3. 后端首次启动会自动通过 `schema_sync` 兜底为 `merchant_stores` 表补上 `lat / lng` 两列（无需手动迁移）。
4. 已绑定卡项的门店若**缺失经纬度**，会被排到列表末尾、不显示距离；若**所有门店都缺失或用户未授权定位**，整体降级为按门店名升序。

## 注意事项

1. **定位权限不是必选项**：不授权也能浏览门店，只是改为按门店名排序。
2. **门店经纬度数据**：如果线上的 `merchant_stores` 表中存量门店缺失经纬度，会被排到列表末尾；建议管理员尽快补录。
3. **小程序授权说明**：本次已在 `app.json` 中补充 `permission.scope.userLocation` 与 `requiredPrivateInfos: ["getLocation"]`，符合最新基础库合规要求。
4. **iOS 未签名 IPA 的限制**：使用 AltStore 自签的 App 有 7 天有效期，过期后需重新签名（如续 Apple Developer 账号则可延长）。如有正式 Apple Developer 账号，可在 GitHub Actions Workflow 中接入签名密钥，重新构建一次即可分发签名版本。
5. **时段预览只看今日**：详情页的时段预览固定查询当天的可用情况，下单时仍可在下单页选择未来日期。

## 常见问题排查

| 现象 | 可能原因 | 处理建议 |
|------|---------|---------|
| 详情页未显示时段区 | 该商品 `appointment_mode === 'none'` 或未配置 `time_slots` | 在 Admin 端将商品预约方式改为「时段」并填入时段配置 |
| 时段都没置灰 | 该商品未启用容量限制 / 暂无人下单 | 正常：当前还未约满 |
| 主门店卡片提示"暂未获取位置" | 用户拒绝了定位 / 浏览器 / 系统不支持 | 在系统设置开启定位权限后重新进入页面，或继续使用按门店名排序的兜底体验 |
| 距离不准 | 门店经纬度配置错误 | Admin 端编辑门店并修正 lat/lng |
| 抽屉打不开 | 该卡项只绑定了 1 家门店 | 主卡片右侧的"切换 ▼"按钮在 ≥2 家门店时才出现 |
