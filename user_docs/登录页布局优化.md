## 功能简介

本次更新对**全平台登录页**进行了视觉与交互的统一优化：

### 视觉重塑（贯穿 H5 / 小程序 / 安卓 / iOS 四端）

1. **顶部沉浸式品牌区**：屏幕上方 42% 高度采用绿色渐变（`#2fb56a → #5cd692`）的沉浸式品牌区，代替原先的纯色 + 大量留白布局
2. **92×92 白色圆形 LOGO 托盘**：项目 LOGO 居中嵌入白色圆形托盘内，托盘带柔和阴影；LOGO 图片仍由后台动态接口下发，不影响管理后台「品牌设置」的修改逻辑
3. **品牌文案聚焦**：
   - 主标题：**宾尼小康**（白色加粗，居中）
   - 副标题：**AI 健康管家 · 您的私人健康助手**
4. **表单卡片上浮**：登录表单卡片采用 `-28px` 上浮设计，圆角 24px，覆盖在渐变下沿；视觉重心提升、不再贴近底部导航条避免误触

### 交互优化

5. **登录按钮始终高亮可点**：不再依赖协议勾选/手机号长度做禁用，按钮永远点得动；点击后由系统校验弹出对应提示
6. **协议二次确认弹窗**：未勾选「我已阅读并同意《用户服务协议》和《隐私政策》」时点击登录，会弹出二次确认弹窗：
   - 标题：**为保障您的权益，请阅读并同意以下协议**
   - 按钮：**[再看看]**（关闭弹窗）/ **[同意并登录]**（自动勾选并继续登录）
7. **「获取验证码」按钮永不换行**：在窄屏（如 320 px 小手机）下，按钮文字「获取验证码」与倒计时态「60 s」均强制单行展示
8. **删除冗余文案**：移除底部「登录即表示您将享受 AI 智能健康陪伴服务」一行说明文字

### LOGO 接入约束（重要）

✨ **本次改造完全不影响 LOGO 的获取链路**：
- 不改 LOGO 接口路径
- 不改请求参数与字段名
- 不改缓存策略与回退逻辑
- 三端原有的 LOGO 调用链路一律保留

仅在登录页中**调整 LOGO 的展示位置 / 尺寸 / 容器样式**（即把它放进 92×92 的白色圆形托盘）。管理后台切换 LOGO 后，登录页能正确刷新。

## 使用说明

### 1. 进入登录页

- **H5 端**：浏览器打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login/)。如果之前已登录，请先在浏览器开发者工具中清除 localStorage（或换无痕窗口）以触发未登录状态自动跳转登录页。
- **微信小程序**：用【微信开发者工具】导入下载并解压后的小程序源码包，编译运行，模拟器中打开 → 默认进入登录页。
- **安卓 / iOS APP**：安装新版本后启动 APP → 默认进入登录页。

### 2. 输入手机号 + 获取验证码

1. 在「+86 │ 请输入手机号」输入框输入 11 位手机号
2. 点击右侧「**获取验证码**」按钮（永远单行显示，窄屏不换行）
3. 收到短信后在「请输入验证码」输入框填入 6 位数字
4. 倒计时态显示为「**60 s**」（同样单行不换行）

### 3. 协议勾选与登录（重点流程）

#### 场景一：已勾选协议
- 点击「**登录 / 注册**」按钮 → 直接发起登录请求

#### 场景二：未勾选协议
- 点击「**登录 / 注册**」按钮 → 弹出二次确认弹窗：

  ```
  ┌─────────────────────────────────────┐
  │  为保障您的权益，请阅读并同意以下协议    │
  │                                      │
  │  您需要阅读并同意《用户服务协议》和    │
  │  《隐私政策》后才能继续登录。          │
  │                                      │
  │           [再看看]   [同意并登录]      │
  └─────────────────────────────────────┘
  ```

- 点击「**再看看**」 → 关闭弹窗，回到登录页（不发起登录请求）
- 点击「**同意并登录**」 → 协议勾选框自动变为已勾选 + 继续完成登录流程（发送 SMS 登录请求）

### 4. 登录成功

- 老用户：登录成功，跳转主页 `/home`
- 新用户：自动注册 + 提示会员卡号 + 跳转健康档案引导页 `/health-guide`
- 双身份用户（同时是用户与商家）：跳出角色选择弹窗（保留原逻辑）

## 注意事项

1. **LOGO 仍由管理后台动态下发**：本次仅调整登录页 LOGO 的展示容器样式（92×92 白色圆形托盘），LOGO 图片本身仍来自管理后台「品牌设置 → 系统 LOGO」的接口。**管理员切换 LOGO 后，登录页会自动刷新显示新 LOGO。**
2. **协议二次确认是「软提示」**：用户点击「同意并登录」后，系统会**自动**把协议勾选框设为已勾选状态并继续登录流程，避免再次中断。「再看看」则保持未勾选状态，关闭弹窗后用户可继续修改输入或手动勾选。
3. **「获取验证码」按钮永不换行**：H5 用 `white-space:nowrap; word-break:keep-all; flex:0 0 auto`，小程序用 `white-space:nowrap`，Flutter 用 `maxLines:1, softWrap:false`，三端策略一致。窄至 320 px 也保持单行。
4. **底部多余文案已删除**：原来的「登录即表示您将享受 AI 智能健康陪伴服务」已从所有三端 DOM / Widget 树移除。
5. **无后端接口变更**：本次纯前端 / Flutter UI 改造，**未触碰后端任何 API**。原有的 `/api/auth/sms-code`、`/api/auth/sms-login`、`/api/settings/logo`、`/api/auth/register-settings` 接口签名与字段全部保持不变。
6. **缓存清理建议**：H5 端如刷新后仍看到旧页面，请按 `Ctrl+F5`（Windows）或 `Cmd+Shift+R`（Mac）强制刷新，或在浏览器隐私窗口中打开。
7. **iOS 安装提示**：本次 iOS IPA 为**未签名**版本，仅供内部测试和体验。正式上架 AppStore 需通过 TestFlight → AppStore Connect 流程，审核周期约 1–3 个工作日。
8. **Flutter 双端共享代码**：安卓 APK 与 iOS IPA 都来自同一份 `flutter_app/lib/screens/login_screen.dart`，所以二者视觉与交互一致。
