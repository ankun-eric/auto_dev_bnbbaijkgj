## 功能简介

本次修复了 bini-health 健康管理系统的 **502 Bad Gateway** 故障。该故障导致管理后台和 H5 用户端均无法访问，页面加载时直接返回 502 错误。

### 故障原因

系统部署使用 Docker 容器化架构，由 Nginx gateway 反向代理转发请求到各业务容器。故障的根本原因是 **Nginx gateway 容器未正确连接到项目的 Docker 网络**：

- Docker Compose 启动时自动创建的网络名称带有项目目录前缀（如 `6b099ed3_..._network`）
- 部署脚本中将 gateway 连接到了名称不同的手动创建网络
- 两个网络名称不匹配，导致 Nginx 无法通过 Docker DNS 解析到后端、管理后台、H5 容器的地址

### 修复措施

1. **即时修复**：将 gateway 容器连接到 docker-compose 实际创建的项目网络，并 reload nginx，所有服务立即恢复
2. **根因修复**：更新部署脚本 `deploy_remote.py`，改为动态检测后端容器所在的网络名称再连接 gateway，避免未来部署时再次出现网络不匹配问题

## 使用说明

### 访问管理后台

1. 在浏览器中打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 输入管理员账号和密码
3. 点击「登录」按钮即可进入后台管理界面

### 访问 H5 用户端

1. 在手机或电脑浏览器中打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)
2. 即可正常浏览和使用 H5 端的各项功能

## 注意事项

- 如果后续重新部署系统后再次出现 502 错误，请检查 gateway 容器是否正确连接到了项目的 Docker 网络
- 可通过服务器上执行 `docker network inspect` 命令确认 gateway 与项目容器在同一网络
- 部署脚本已更新，正常使用 `deploy_remote.py` 部署不会再出现此问题
