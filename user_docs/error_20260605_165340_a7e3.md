### 已尝试的修复措施

- TCP 端口 80/443 同样无应用层响应
- 诊断结论：服务器可能已宕机、Docker 服务崩溃、或云服务商防火墙/安全组规则变更导致入站流量被丢弃

## 建议的排查方向

> 以下为基于错误信息的初步排查建议，仅供参考。

1. **检查服务器运行状态**：登录腾讯云控制台，确认 CVM 实例 `134.175.97.26` 是否处于"运行中"状态。如果已关机或异常，尝试重启实例
2. **检查安全组/防火墙**：确认安全组规则中是否放行了 22/80/443 端口入站流量。如果服务器迁移或安全组被修改，可能导致端口被拦截
3. **检查 VNC/控制台登录**：通过云服务商提供的 VNC 或网页控制台直接登录服务器，检查 SSH 服务是否运行（`systemctl status sshd`），检查 Docker 服务状态（`docker ps`）
4. **网络连通性检查**：从不同网络环境测试 SSH 连接，排除本地网络或 Cloudflare 代理干扰
5. **检查服务器磁盘/内存**：如果磁盘已满或内存耗尽，可能导致 SSH 无法正常接受新连接
