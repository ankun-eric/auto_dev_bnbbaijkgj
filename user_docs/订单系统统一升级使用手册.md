## 一、本次升级做了什么（一句话版）

**彻底合并双轨订单系统。** 原来的"老订单页面（写死假数据）"和"新统一订单页面（真实接口）"已合二为一：所有端都统一走新接口，**待付款、待收货、待使用、已完成、待评价、已取消**等状态实时联动，再也不会出现"角标显示 1 条、点进去却是空"的问题。同时新增了独立的**退款/售后**列表页，一眼看清所有申请中、处理中、已退款、被拒绝的售后单。

## 四、7 个订单 Tab 说明

| Tab | 后端 status | 含义 |
|---|---|---|
| 全部 | （不传） | 所有订单 |
| 待付款 | `pending_payment` | 用户下单未支付 |
| 待收货 | `pending_receipt` | 已支付，等快递 |
| 待使用 | `pending_use` | 已收货/服务类，未消费 |
| 已完成 | `completed` | 服务完成 |
| 待评价 | `pending_review` | 已完成且未评论 |
| 已取消 | `cancelled` | 订单已取消（含退款成功后） |

**退款状态（叠加在上述订单状态之上）**：
- `applied`：用户已申请，等待运营审核
- `reviewing`：审核中
- `returning`：运营要求退回商品，买家寄回中
- `refund_success`：退款成功（订单归入"已取消"）
- `rejected`：审核被拒
- `approved`：审核通过待放款

## 五、常见问题

**Q1：我的老订单看不到了？**
> A：老订单数据表（mock 假数据）不是真实订单，已经下线。如果您有历史真实订单，全部都在新的"订单明细"里，按时间倒序展示。

**Q2：退款状态看不到了？**
> A：请前往**退款/售后**独立页面（H5: `/refund-list`；小程序: 我的 → 退款；App: 我的 → 退款/售后），此前 H5 只展示"申请中"的 Bug 已修复。

**Q3：管理后台老订单菜单不见了？**
> A：运营只需使用「订单明细」（`/admin/product-system/orders`）。此页面包含完整的真实订单数据，支持按 7 种状态过滤、退款审核。

**Q4：App 的 Tab 看起来和网页不一样？**
> A：三端 Tab 已统一为 7 个，如果 App 仍显示旧版，请卸载重装最新版本。

**Q5：退款中订单为什么还在"待收货"里？**
> A：退款中是过程状态，订单仍挂在原 Tab，通过"退款中"角标提示；退款成功后才会迁移到"已取消"。

## 六、注意事项

1. **老 URL 已失效**：以下地址将永久返回 404：
   - `/api/orders/*`（老用户端接口）
   - `/api/admin/orders/*`（老管理后台接口，unified 版保留）
   - H5 `/orders`
   - 管理后台 `/admin/orders`
   - 小程序 `pages/orders/*`
2. **退款流程严格走"用户申请 → 运营审核"**，客服代申请已下线。
3. **三端数据一致**：同一账号在 H5 / 小程序 / App 登录看到的订单完全一致。
4. **角标与列表一致**：如果角标显示 N 条，点进去必定是 N 条，不再存在不一致现象。
