## 功能简介

本次更新包含以下三项改进：

1. **Bug 修复 — 全局 500 错误**：修复了部署腾讯云 COS 存储功能后，所有写操作（保存配置、测试连接、更新上传限制等）均报 500 错误的问题。根因为数据库表字段不同步，已通过 Schema 自动同步机制彻底解决。

## 使用说明

### 一、存储配置管理

#### 1. 进入存储配置页面

1. 使用管理员账号登录管理后台：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 在左侧导航栏中找到并点击「存储配置」菜单项

#### 2. 页面 Tab 说明

进入页面后，您将看到 4 个 Tab 标签：

| Tab | 图标 | 功能说明 |
|-----|------|----------|
| **存储配置** | ☁️ | COS 基础配置（SecretId、SecretKey、Bucket、Region）+ CDN 加速配置 + 保存/测试操作 |
| **上传限制** | ⬆️ | 设置各业务模块的文件上传大小限制 |
| **数据迁移** | 🔄 | 扫描本地文件、启动迁移到 COS、查看进度、重试失败文件 |
| **文件管理** | 📄 | 查看已上传的文件总数统计 |

---

### 二、COS 基础配置（存储配置 Tab）

1. 点击「**存储配置**」Tab
2. 在 **COS 基础配置** 区域中填写：
   - **SecretId**：腾讯云 API 密钥的 SecretId
   - **SecretKey**：腾讯云 API 密钥的 SecretKey（已设置时显示掩码，重新输入将覆盖）
   - **Bucket**：COS 存储桶名称（如 `my-bucket-1250000000`）
   - **Region**：存储桶所在地域（从下拉列表选择）
3. 配置 **3 个分类路径前缀**（均为必填项，不可为空）：
   - **图片路径前缀**：上传图片时的存储目录前缀，默认 `images/`
   - **视频路径前缀**：上传视频时的存储目录前缀，默认 `videos/`
   - **文件路径前缀**：上传其他文件时的存储目录前缀，默认 `files/`
4. 在 **CDN 加速配置** 区域中（可选）：
   - **CDN 域名**：如已配置 CDN，填入加速域名
   - **CDN 协议**：选择 HTTPS 或 HTTP
5. 点击「**保存配置**」按钮保存设置
6. 点击「**测试连接**」按钮验证 COS 是否可达
7. 连接测试通过后，打开「**启用 COS**」开关

## 路径前缀分类机制

上传文件时，系统会根据文件的 MIME 类型自动选择对应的路径前缀：

| 文件类型 | MIME 类型 | 使用的前缀 | 默认值 | 示例存储路径 |
|----------|-----------|-----------|--------|-------------|
| 图片 | image/* | 图片路径前缀 | `images/` | `images/a1b2c3d4.jpg` |
| 视频 | video/* | 视频路径前缀 | `videos/` | `videos/e5f6g7h8.mp4` |
| 其他 | 所有其他类型 | 文件路径前缀 | `files/` | `files/i9j0k1l2.pdf` |

---

## 注意事项

1. **路径前缀不能为空**：3 个分类路径前缀（图片/视频/文件）均为必填项，如果清空将无法保存
2. **先测试后启用**：首次配置 COS 时，请先填写配置信息并点击「测试连接」，确认连接成功后再启用
3. **修改配置后需重新测试**：修改 SecretId、SecretKey、Bucket 或 Region 后，连接测试状态会重置，需重新测试
4. **数据迁移不可逆**：迁移到 COS 后，文件引用 URL 会更新为 COS 地址，请确认 COS 配置正确后再执行迁移
5. **CDN 配置可选**：如果不需要 CDN 加速，CDN 域名可留空，文件 URL 将直接使用 COS 原始域名
