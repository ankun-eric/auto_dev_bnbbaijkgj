本批次修复围绕"统一订单状态体系（V2：12 个订单状态 + 7 个退款状态）"展开，解决两个长期影响经营与体验的 Bug：

| Bug 编号 | 模块 | 现象 | 修复后效果 |

## 二、Bug ①：管理后台订单统计页升级

### 2.1 进入路径

1. 浏览器打开管理后台：<https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin>
2. 使用管理员账号登录
3. 左侧菜单 → **产品系统 → 订单统计**

直链：<https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/statistics>

### 2.2 时间筛选

页面顶部新增统一的时间维度筛选条：

- 快捷按钮：**今日 / 本周 / 本月**（默认"今日"）
- 自定义时间段：选择日期范围后立即刷新

时间筛选会同步影响：
- 12 个订单状态卡片
- 7 个退款状态卡片
- 销售趋势图、分类销售柱状图、退款原因饼图

### 2.3 12 个订单状态卡片（与"订单明细页"完全一致）

| 状态码 | 中文 | 含义 |
|---|---|---|
| pending_payment | 待付款 | 订单已创建但用户未支付 |
| pending_shipment | 待发货 | 实物商品已支付待商家发货 |
| pending_receipt | 待收货 | 已发货等待用户收货 |
| pending_appointment | 待预约 | 服务/到店类商品支付后未预约 |
| appointed | 已预约 | 已选定预约时段，等待履约 |
| pending_use | 待核销 | 等待到店扫码核销 |
| partial_used | 部分核销 | 多份核销码部分已使用 |
| completed | 已完成 | 全部履约完成 |
| expired | 已过期 | 超过有效期未核销 |
| refunding | 退款中 | 用户已申请退款，处理中 |
| refunded | 已退款 | 退款已完成 |
| cancelled | 已取消 | 订单已取消 |

每张卡片显示：**订单数量** + **总实付金额**。

### 2.4 7 个退款状态卡片（对齐 V2 退款体系）

| 状态码 | 中文 |
|---|---|
| none | 无退款 |
| applied | 退款申请中 |
| reviewing | 退款审核中 |
| approved | 退款已批准 |
| rejected | 退款已拒绝 |
| returning | 退款处理中 |
| refund_success | 退款成功 |

### 2.5 顶部总览（筛选时间段内）

- **总订单数**（不含 cancelled）
- **总实付金额**
- **退款单数**（refund_success）
- **退款金额**

### 2.6 注意事项

- 默认载入"今日"数据，如果今天还没有订单，相关卡片显示 0 是正常现象。
- 切换时间段后，所有卡片和图表会同步刷新，无需手动刷新页面。
- 已取消订单不计入"总实付金额"，但仍会出现在 cancelled 卡片里方便回溯。

## 三、Bug ②：用户端"履约方式"中文化

### 3.1 涉及位置

| 端 | 页面 | 入口 |
|---|---|---|
| H5 | 统一订单详情 | <https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/unified-order/[订单ID]> |
| H5 | 商品列表/筛选 | <https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/products> |
| 小程序 | 订单详情、商品列表、商品详情、服务页 | 在微信内"小白健康"小程序中查看 |
| Flutter APP | 订单详情、服务列表、商品模型展示 | 安装最新 APK 体验 |

### 3.2 履约方式中文映射表

| 后端枚举 | 中文标签 |
|---|---|
| on_site | 上门服务 |
| to_store | 到店服务 |
| delivery | 快递配送 |
| in_store | 到店核销 |
| virtual | 线上服务 |
| 其他 / 未知 | 其他服务 |

各端均使用各自语言对应的"公共字典"组件（`fulfillmentLabel`），未来若需要新增履约方式只需修改字典文件即可全端生效，避免重复散落。

### 3.3 用户体验

- 订单详情商品名称下方会以小标签形式展示中文履约方式，颜色区分。
- 列表页商品卡片上的小角标也会显示对应中文。

## 五、技术验证（可选）

后端管理员可通过以下接口验证统计数据：

```
GET /api/admin/orders/statistics?start_at=2026-05-01&end_at=2026-05-04
Authorization: Bearer <admin_token>
```

返回结构示例：

```json
{
  "start_at": "2026-05-01T00:00:00",
  "end_at":   "2026-05-04T23:59:59",
  "summary":  { "total_orders": 0, "total_revenue": 0.0, "total_refund_count": 0, "total_refund_amount": 0.0 },
  "order_status_items": [ { "status": "pending_payment", "label": "待付款", "count": 0, "amount": 0.0 }, ... ],
  "refund_status_items": [ { "status": "none", "label": "无退款", "count": 0, "amount": 0.0 }, ... ]
}
```

`order_status_items` 必返回 12 项，`refund_status_items` 必返回 7 项。
