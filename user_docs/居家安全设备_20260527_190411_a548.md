## 功能简介

「居家安全设备」是 bini-health 平台新接入的「宾尼」系列居家智能硬件绑定与报警体系，覆盖三类设备：

| 类型 | 颜色等级 | 通知文案 |
|---|---|---|
| 紧急呼叫器（type=1） | 🔴 红 | **【紧急】XX 触发了 SOS 呼叫，请立即联系！** |
| 烟雾报警器（type=2） | 🟠 橙 | **【警告】XX 家中检测到烟雾，请确认是否火情** |
| 水位报警器（type=7） | 🟡 黄 | **【提醒】XX 家中检测到漏水，请尽快查看** |

核心能力：
- 用户自助绑定/解绑设备（每次手工输入网关 SN + 设备 SN）；
- 上游报警 → 平台多维通知（站内 / 小程序 / 短信 / AI 电话外呼）；
- 紧急联系人从守护人列表中勾选（主守护人强制锁定 + 其他最多 2 位）；
- 报警记录永久保留，绑定者与主守护人均可查看历史；
- 同设备 5 分钟去重窗口，AI 外呼最多 1 次；
- 管理后台覆盖字典、设备绑定流水、报警流水、回调配置（含连通性测试 + 推送上游）。

## 使用说明（用户端 · H5）

### 1. 进入入口

打开「健康档案」主页，在「今日健康数据」下方可以看到新增的 **「🛡️ 居家安全设备」** 蓝绿渐变卡片，点击进入主页。

### 2. 主页三个 Tab

进入后默认选中 **紧急呼叫器** Tab，可切换到 **烟雾报警器** / **水位报警器**。

### 3. 绑定新设备

1. 点击页面顶部的 **「+ 绑定新设备」** 按钮，弹出绑定窗口。
2. **网关 SN**：手工输入 12 位字母+数字（每次绑定都必须手工输入，不缓存）。
3. **设备 SN**：手工输入 8 位字母+数字，或使用「📷 扫码」（视配置而定）。
4. 点击「确认绑定」。前端会做格式校验：
   - 网关 SN 必须 12 位字母+数字；
   - 设备 SN 必须 8 位字母+数字。
5. 提交后后台异步调上游校验，如果失败，会在站内消息提醒「设备 SN 与网关不匹配，请核对」。

### 4. 解绑设备

在「── 我绑定的设备 ──」区域，每张设备卡片右下角的「解绑」按钮可解绑。解绑后仅对当前用户失效，其他绑定者不受影响；后台保留所有绑定/解绑流水。

### 5. 报警记录与处置

「── 报警记录 ──」区域展示该设备类型下的最新报警，每条记录包含：
- 报警时间（5 分钟内重复会合并标注「共 N 次」）；
- 设备类型；
- AI 外呼状态（等待回调 / 成功 / 失败 / 未发起）；
- 操作按钮：「标记已读」「处置备注」。

点击「处置备注」会弹出输入框，填写后即标记为已处置。

### 6. 紧急联系人

「── 紧急联系人（共用，主守护人 1 + 其他 2 位）──」区域：
- **主守护人**：默认勾选且不可取消（UI 上灰显 ☑ + 黄色「主守护人（强制）」标签）；
- **其他守护人**：用户最多勾选 2 位，超出会自动拦截。
- 该联系人配置为「用户级共用」，3 件设备共用一份。

## 使用说明（管理后台）

入口：左侧菜单 **健康档案管理 → 居家安全设备管理**

页面包含 4 个 Tab：

### 1. 设备类型字典
展示 3 类设备的：上游类型码、平台设备名、颜色等级、通知标题模板、AI 话术模板。

### 2. 设备绑定流水
- 所有用户的绑定/解绑历史（解绑后仍保留）；
- 列：流水 ID、设备类型（带彩色 Tag）、网关 SN、设备 SN、绑定用户、操作类型、校验结果、绑定/解绑时间。

### 3. 报警记录流水
- 所有用户的报警记录；
- 列：报警 ID、设备 SN、设备类型、上报时间、接收时间、关联用户、4 类通知结果（站内/小程序/短信/AI 外呼）、合并次数、处置状态、处置备注。

### 4. 回调地址配置（全平台 1 套）

填写 4 个字段：
- 机构 ID（= 平台 ID）
- 回调地址 URL（我方对外暴露给上游的 URL）
- 鉴权 Token
- 上游基础 URL

按钮：
- **保存**：将配置落库；
- **测试连通性**：探测上游基础 URL 是否合法可达；
- **把配置推送给上游**：调用上游「回调地址设置接口」更新回调地址。

## 关键 API

### 上游报警回调（对外开放，无需鉴权）

```http
POST /api/home_safety/callback/alarm
Content-Type: application/json

{
  "device_sn": "ABCD1234",
  "type": 1,
  "alarm_time": "2026-05-27T18:30:00+08:00",
  "sign": "..."
}
```

返回：
```json
{
  "success": true,
  "matched": 2,
  "created": 2,
  "dedup_skipped": 0,
  "alarm_ids": [101, 102]
}
```

去重规则：同一 `device_sn` 在 5 分钟窗口内的二次报警会被合并到上次记录上（`dedupe_count++`），不再发新通知、不再触发 AI 外呼。

### AI 外呼结果回调（预留）

```http
POST /api/home_safety/callback/ai_call_result
```

本期仅落入日志，不解析业务字段。

## 关键规则

| # | 决策项 | 规则 |
|---|---|---|
| 1 | 报警站内通知对象 | 所有绑定者 + 档案主人的主守护人 |
| 2 | 紧急联系人通知方式 | 站内+小程序+短信全部通知，AI 电话仅在有额度时调用 |
| 3 | 网关 SN 输入方式 | 每次绑定都必须手工输入 12 位 |
| 4 | 紧急联系人归属 | 用户级配置（3 件设备共用一份） |
| 5 | AI 外呼额度扣费 | 档案主人优先，不足走主代付（与守护体系一致） |
| 6 | 解绑规则 | 仅影响自己；后台保留所有绑定流水 |
| 7 | 报警去重 | 同设备 5 分钟内重复只发 1 次，AI 外呼最多 1 次 |
| 8 | SN 校验时机 | 前端格式校验 + 后台异步校验失败给站内提醒 |
| 9 | 紧急联系人来源 | 从守护人列表勾选 |
| 10 | 设备在线状态 | 本期不展示（仅靠报警感知） |
| 11 | 报警归属 | 每个绑定者档案下各生成一条 |
| 12 | 报警保留 | 永久保留 |

## 注意事项

1. **首次使用**：请先在「我守护的人」中建立守护关系（特别是主守护人），紧急联系人才有可勾选项；否则报警时只能通知本人手机。
2. **AI 外呼额度**：与「用药提醒 AI 外呼」共用额度，先扣档案主人额度，不足时走主代付（扣主守护人额度）。
3. **解绑保留**：解绑后历史报警记录仍可在管理后台查询，用户端列表则只显示当前生效设备的报警。
4. **5 分钟去重**：上游 API 一次性高频推送（如 SOS 长按）会被合并为 1 条，避免短信/电话被打爆。
5. **未知设备 SN**：上游推送了未绑定的 device_sn 时，平台会落日志告警但仍返回 200 给上游（不抛错），便于上游稳定接入。
6. **回调路径**：上游应使用 `/api/home_safety/callback/alarm`（推荐，网关代理友好），原 PRD 设计的 `/callback/home_safety/alarm` 仍兼容保留。
7. **管理后台权限**：仅平台管理员可访问；普通用户访问后台 API 会返回 401。

## 数据模型一览

本期落地 5 张表（前缀 `home_safety_`），由 SQLAlchemy 自动建表，无需手动迁移：

| 表 | 说明 |
|---|---|
| `home_safety_device_binding` | 设备绑定（含解绑历史，唯一约束：user_id + device_sn + status） |
| `home_safety_emergency_contact` | 紧急联系人配置（每用户最多 3 条：主守护人 1 + 其他 2） |
| `home_safety_alarm` | 报警记录（含去重键、4 类通知结果、处置状态） |
| `home_safety_callback_config` | 回调配置（全平台单条） |
| `home_safety_ai_call_log` | AI 外呼流水（预留回调字段） |
