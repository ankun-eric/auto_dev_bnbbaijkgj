## 功能简介

本次为 **健康档案 → 我守护的人** 卡片同时修复 3 个互相关联的 Bug，统一一期上线：

| 编号 | 修复点 | 修复前 | 修复后 |

## 使用说明

### 一、查看「档案记录数」（Bug A 修复）

1. 在手机浏览器访问 [健康档案首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile)
2. 切换到「本人」Tab
3. 在页面中部找到「我守护的人」卡片
4. 卡片副标题改为：**`档案记录数 N`**（N = 您所有守护对象名下的医疗档案记录条数之和，包括邀请中、未注册的占位记录）

> ✅ 与修复前的「N 人」相比，新口径更准确反映您"实际管理着多少份档案"。

### 二、查看可邀请额度（Bug B 修复）

「我守护的人」卡片底部新增一行邀请额度展示，按您的会员等级三态自适应：

#### 状态 1：未达上限（普通会员/VIP）
显示：**`可邀请 N 人（已守护 X/上限 Y）  升级解锁无上限`**

- 点击「升级解锁无上限」直接跳转 [会员中心](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center)
- 上限 Y 由后端按 `membership_plans.max_managed` 实时返回（不再写死 10）

#### 状态 2：已达上限
显示：**`已达上限（X/Y），升级会员可邀请更多`**（橙色高亮）

- 整段文字可点击 → 跳转会员中心
- 此时「我守护的人」内部的「+ 发起邀请」按钮也会自动置灰

#### 状态 3：超级 VIP（无上限）
显示：**`已守护 X · 上限 不限`**

- 不再展示升级文案，邀请按钮永久可用

### 三、移除已守护的家人（Bug C 修复）

1. 在「我守护的人」卡片点击 → 进入 [我守护的人列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard)
2. 在「已绑定」区找到要解除的家人
3. 点击该卡片右下角的「**解除守护**」（如该家人为孤儿档案则按钮为「**移除**」）
4. 在二次确认弹窗点击「确认解除/确认移除」
5. ✅ 列表立即更新，并 toast 提示：「已解除守护」/「已移除」
6. 同时您和对方都会收到一条系统通知：「守护关系已解除」

> 💡 **特别说明**：修复前对任意已绑定家人点击移除都会返回「未找到守护关系」错误。本次修复后，无论是 active 状态、孤儿档案，还是已解绑/已过期记录，移除流程都已打通——后端会按真实情况返回准确文案，前端不再使用兜底通用提示。

## 注意事项

### 仍保留的 3 项不可删校验
为防止误删数据，以下 3 种情况依然会拦截「移除」操作并提示明确文案：
1. **该家人已绑定硬件设备** → 「该家人已绑定硬件设备，请先在设备管理中解绑后再移除」
2. **该家人尚有邀请中的邀请记录** → 「该家人尚有邀请中的记录，请先取消邀请后再移除」
3. **该家人存在在途服药计划** → 「该家人存在在途服药计划，请先终止服药计划后再移除」

### 邀请上限的判定口径
卡片显示的"已守护 X"按以下规则计入：
- ✅ 已绑定（active） + 邀请中（pending） + 尚未邀请（never_invited）+ 孤儿档案 → 占名额
- ❌ 暂未响应（rejected）+ 已解绑（unbound）+ 已过期（expired）→ 不占名额
- ❌ 本人不占名额

### 兼容性
- 本次接口为**向前兼容**升级，旧字段（`active_count`、`pending_count` 等）全部保留
- 当后端 v13 接口暂不可达时，H5 会自动回退到旧 v12 接口，保证页面始终可用
