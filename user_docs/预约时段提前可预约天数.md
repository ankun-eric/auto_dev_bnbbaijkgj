## 功能简介

本次修复并增强了商品系统的「**预约联动**」配置：

- **修复前**：商品编辑页选择「预约时段」模式时，**没有「提前可预约天数」字段**，导致顾客端可以无限远地往后挑选预约日期，业务方完全无法控制时段类商品（如理发、上门、体验课）的排期窗口。
- **修复后**：「预约日期」与「预约时段」**两种模式共用** `提前可预约天数`，并新增 `预约起始日（包含今天 / 从明天起算）` 选项；顾客端 H5、小程序、APP 三端按统一公式计算可选日期范围，并由后端兜底校验越权下单。

> 一句话总结：以后无论用「预约日期」还是「预约时段」，都能像设置"最多提前几天预约"+"是否含今天"这样精确控制排期窗口。

## 使用说明

### 一、商家端：在管理后台配置「预约时段商品」

1. 浏览器打开 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)，登录运营/商家账号
2. 进入「商品系统 → 商品列表」
3. 点击「新建商品」（或编辑现有商品），切换到「预约与核销」Tab
4. 在「预约模式」下拉中选择 `预约时段`
5. 此时表单会出现以下字段（修复后新增/调整）：
   - **下单预约方式**（必填）：下单即预约 / 先下单后预约
   - **提前可预约天数**（必填，**本次修复点**）：例如 `7` 表示最多提前 7 天预约
   - **预约起始日**（必填，**本次新增**）：
     - `包含今天` （默认）：可预约日期 = `今天 ~ 今天 + N - 1 天`
     - `从明天起算`：可预约日期 = `明天 ~ 明天 + N - 1 天`
   - **时段列表**（必填）：至少配置 1 个时段，每个时段含 `开始 / 结束 / 容量`
6. 填写完成后点击「保存」，若有项缺失/非法，会弹出明确的提示文案（如「请填写"提前可预约天数"（需大于 0）」）

### 二、顾客端：H5 / 小程序 / APP 选择预约日期

打开商品详情，点击「立即购买」进入下单页：

1. 下单页会自动按以下公式计算可预约日期范围：
   - `include_today = true`（默认）：`今天 ~ 今天 + N - 1 天`
   - `include_today = false`：`明天 ~ 明天 + N - 1 天`
2. 下单页会以一行小字提示，例如：
   - `可预约：5月2日 ~ 5月8日`
   - `可预约：5月3日 ~ 5月9日（不含今天）`
3. 点击「预约日期」选择器，**只能选择上述范围内的日期**，超出区间的日期会被禁用
4. 选完日期后选择时段（已配置容量校验，约满的时段会显示「已约满」并置灰）
5. 点击「提交订单」，若提交日期超出区间（如本地时间被篡改），后端会兜底返回 400 错误：「预约日期超出可预约范围」

### 三、不同模式行为对照

| 预约模式 | 提前可预约天数 | 预约起始日 | 单日最大人数 | 时段列表 |
|---------|----------------|------------|--------------|----------|
| 无需预约 (`none`) | — | — | — | — |
| 预约日期 (`date`) | ✅ 必填 | ✅ 必填 | ✅ 必填 | — |
| **预约时段 (`time_slot`)** | ✅ **本次新增必填** | ✅ **本次新增必填** | — | ✅ 必填 |
| 自定义表单 (`custom_form`) | — | — | — | — |

## 历史数据兼容说明

后端启动时，迁移脚本自动完成以下数据修复（**无需任何人工操作**）：

1. **新增字段** `include_today`：默认值 `true`，所有历史商品均视为「包含今天」
2. **历史 `time_slot` 模式商品** 若 `advance_days` 为空 / 0，**统一刷为 7**（覆盖理发、就诊、体验课等高频短期场景）
3. **历史 `date` 模式商品** 若 `advance_days` 为空 / 0，同样兜底刷为 7（保险起见）

> 服务器上线后已自动执行；本服务器实测：1 条 `time_slot` 商品已成功补值为 7。

## 注意事项

- **API 字段名为 `advance_days` 与 `include_today`**：所有端的 JSON 字段保持一致
- **下单越权校验**：前端禁用区间外日期；后端兜底校验，绕过前端也无法非法下单
- **跨年/跨月计算**：基于自然日 + 服务器时区计算，已通过单元测试覆盖（含跨年场景：12/30 + N=3 + include_today=true → 12/30 ~ 1/1）
- **小程序需重新导入最新代码包** 才能看到本次修复（因为小程序代码缓存在用户微信端）
- **APP 用户需安装本次新打的 APK / IPA** 才能看到本次修复（旧版仍按硬编码 `含今天` 处理，行为与默认值等价，不会出错）
- **管理后台 / H5 已通过服务器更新**，刷新页面即可生效，无需重新安装/导入
