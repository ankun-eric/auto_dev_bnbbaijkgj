## 一、功能介绍

本次更新围绕「居家安全设备」做了三项体验升级：

1. **成员 Tab 统一**：H5 居家安全页面顶部的「本人 / 父亲 / 母亲 / 配偶 / + 添加」成员切换 Tab 与「健康档案」共用同一组组件，数据源统一为家庭成员，体验完全一致。
2. **新增「报警记录」入口**：成员 Tab 下方新增「设备列表 / 报警记录」一级 Tab，可直接在 H5 查看当前成员名下的报警历史。
3. **设备备注**：新绑定设备必须填写"设备备注"（≤ 20 字，如：爸爸家）；老设备未填备注时显示"未命名"，可通过卡片上的"✎ 编辑备注"按钮主动补全。

## 三、H5 用户端操作步骤

### 3.1 切换成员

1. 进入「居家安全」页面。
2. 顶部成员 Tab 默认选中"本人"。
3. 点击其他成员（如"父亲"）即可切换；切换不会变化 URL。
4. 点击「+ 添加」可跳转到家庭成员管理页面新建成员。

> 健康档案首页与居家安全页面的成员 Tab 样式、动效完全一致。

### 3.2 查看报警记录

1. 选择目标成员（默认"本人"）。
2. 点击一级 Tab 的「**报警记录**」。
3. 系统按"触发时间倒序"展示该成员名下所有报警：
   - 🟠 设备类型彩色 Tag（红/橙/黄）  
   - 处理状态：未处理（红点）/ 已处理（绿点）/ 已忽略（灰点）  
   - 相对时间（"刚刚 / N 分钟前 / 昨天 HH:mm / YYYY-MM-DD"），点击/悬停可看绝对时间  
   - 设备备注名；无备注显示"未命名"  
   - 外呼通知状态："✓ 已外呼通知紧急联系人 138****8888"  
   - 设备 SN（辅助信息）  
4. 在"非本人"Tab 下，每条右上角会额外显示触发成员名（如"父亲名下"）。
5. 列表分页加载（每页 20 条），下拉到底自动加载更多。
6. 无数据时显示空状态："暂无报警记录，希望永远用不到这里 🙏"。

### 3.3 绑定新设备（必填备注）

1. 在「设备列表」点击「+ 添加设备」。
2. 表单中**新增了"设备备注"输入框**：
   - 占位文案灰色显示：`如：爸爸家`
   - 右下角实时字数计数：`N/20`
   - 限制 1~20 字，纯空格不允许
   - 未填写时提交按钮置灰，强制点击提示：「请填写设备备注，方便识别（如：爸爸家）」
3. 填写"网关 SN"、"设备 SN"、"紧急联系人手机号"、"设备备注"后，点击「提交」完成绑定。

### 3.4 老设备补填备注

1. 老设备卡片显示灰字「未命名」，右侧出现「✎ 编辑备注」按钮。
2. 点击「✎ 编辑备注」弹出输入框，输入 ≤ 20 字备注后保存。
3. 卡片即时刷新显示新备注；同时报警记录中该设备也会显示新备注。

## 五、注意事项

1. **必填校验**：绑定接口 `POST /api/home_safety/devices/bind` 现在强制要求 `remark` 字段非空且 ≤ 20 字，否则返回 `40001`。
2. **新接口**：`PATCH /api/home_safety/devices/{id}/remark` 用于单独修改设备备注。
3. **接口废弃**：旧的 `GET /api/home_safety/members` 已标记 `deprecated=True`，请改用 `GET /api/family/members`。过渡期内仍可使用，建议尽快迁移。
4. **报警记录响应增强**：`GET /api/home_safety/alarms` 现在返回 `device_remark`、`member_name`、`notify_phone_mask`，以及分页字段 `total / page / size`。
5. **数据迁移**：数据库已自动新增 `home_safety_device_binding.remark` 列（默认 `NULL`），无需手动迁移。

## 六、常见问题（FAQ）

**Q1：为什么我的老设备显示"未命名"？**  
A：本次升级前老设备没有备注字段；为了不打扰你，我们没有强制弹窗让你补填。你可以随时点击卡片上的「✎ 编辑备注」按钮主动补全。

**Q2：成员 Tab 切换为什么 URL 不变？**  
A：与健康档案保持一致，避免页面刷新和返回栈混乱。如需直接进入"报警记录"Tab，可在链接末尾追加 `?tab=alarm`。

**Q3：报警记录里"已外呼通知紧急联系人"是什么？**  
A：当设备触发报警时，系统会自动呼叫绑定的紧急联系人；通话发起成功后会在记录里显示脱敏的手机号，便于你确认通知已经送达。

**Q4：我能编辑别人（父亲、母亲）名下的设备备注吗？**  
A：可以。备注属于设备本身的属性，进入对应成员 Tab 后直接编辑即可，不影响设备权属。
