## 功能简介

会员中心是您查看会员等级、本月配额、套餐对比和完成升级/续费的核心页面。本次升级把**付费用户**进入会员中心后看到的主题色，从原先和普通界面相似的「健康暖橙」基调，全面切换到了**与系统主色一致的真实蓝紫色 `#5B6CFF`**，让付费用户在打开会员中心的第一眼就能强烈感受到「我是 VIP，我和普通用户不一样」的尊贵感与归属感。

### 本次新增/升级要点

1. **顶部 Banner**
   - 付费会员看到深浅三段蓝紫渐变（`#5B6CFF → #6C7BFF → #8B6CFF`）+ 右上角光斑装饰
   - 普通用户看到浅灰背景 + 灰色描边，视觉差异明显
2. **本月配额卡（全新）**
   - 付费会员看到的是蓝紫底色 + 大白字主数值 + 半透明白进度条的高级感配额卡
   - 普通用户看到的是白底 + 蓝紫色前景的克制配额卡
   - 三项配额：AI 外呼提醒、紧急 AI 呼叫、守护他人上限。已用 `-1` 或 ≥9999 自动展示「不限」
3. **等级徽章配色**
   - 「尊享会员」→ 金色渐变徽章
   - 「健康会员」→ 紫色徽章
   - 「普通用户」→ 灰色徽章
4. **三档对比表全新色系**
   - 表头蓝紫底 + 白字
   - 当前所在档位整列高亮（淡蓝紫底）+ 列顶部「当前」小标签
   - 付费列数值统一改为蓝紫调梯度，废弃旧的橙红渐变
5. **状态条**
   - 即将到期（30 天内）：橙黄状态条「您的会员将于 X 天后到期」
   - 已过期：主题降级为浅灰 + 红色状态条「您的会员已于 X 天前到期」
6. **底部吸底 CTA 主按钮**
   - 蓝紫渐变胶囊 + 阴影
   - 文案随状态智能变化：
     - 普通用户 → 「立即开通会员」
     - 健康会员 → 「升级到尊享会员」
     - 尊享会员 → 「续费 1 年」
     - 已过期 → 「立即续费，恢复权益」

### 后端新增接口

- `GET /api/member/quota-usage`：返回当前用户本月已用的 AI 外呼/紧急呼叫次数 + 守护他人已用人数。

## 使用说明

### 一、付费会员体验入口

1. 在 H5 中登录已开通会员的账号
2. 打开右下角「我的」 → 「会员中心」（或直接访问上方「会员中心」链接）
3. 第一眼应当看到：
   - **蓝紫渐变 Banner**：深浅三段渐变 + 用户头像 + 等级徽章 + 「有效期至 YYYY-MM-DD」
   - **本月配额卡（蓝紫底）**：三项配额各自显示 `已用 / 总额`，附带白色进度条
   - **三档对比表**：表头是蓝紫底白字，您当前的档位整列被淡蓝紫底高亮，列顶部带「当前」小标签
   - **底部吸底主按钮**：根据您的等级显示「升级到尊享会员」或「续费 1 年」

### 二、未付费用户体验

1. 用一个普通注册账号登录（未开通过会员）
2. 进入「会员中心」
3. 应当看到：
   - **浅灰背景 Banner**：灰色描边 + 灰色徽章「普通用户」+ 提示文案「开通会员，享 AI 外呼/紧急呼叫等权益」
   - **白底配额卡**：进度条前景为蓝紫色，已用/总额数字为深蓝紫
   - **三档对比表**：「免费会员」列高亮 + 列顶部「当前」标签
   - **底部主按钮**：「立即开通会员」

### 三、即将到期 / 已过期体验

- **即将到期（≤30 天）**：Banner 仍保持蓝紫渐变（强调"您目前还是 VIP"），但顶部覆盖橙黄状态条「您的会员将于 X 天后到期」，右侧带「立即续费 →」。底部 CTA 文案变为续费动作。
- **已过期**：Banner 主题整体回退为浅灰风（标识权益已失效），状态条变红「您的会员已于 X 天前到期」，底部 CTA 文案变为「立即续费，恢复权益」。
- **永不到期 / 长期有效**：不展示任何状态条。

### 四、本月配额展示规则

- **总额数据**：来自 `/api/member/center` 接口，根据您的会员等级返回
- **已用数据**：来自全新 `/api/member/quota-usage` 接口，按月统计
- 任意单项配额若设为「不限」（数据库 `-1` 或 `>=9999`），UI 自动展示「不限」并隐藏进度条
- 接口异常时已用值兜底为 0，不阻塞主流程

## 注意事项

1. **本次仅 H5 端有视觉改动**。微信小程序、安卓 App、iOS App 通过 WebView 加载 H5 自动同步生效，**无需重新安装客户端**。
2. **后端接口完全向后兼容**：`/api/member/center` 字段无任何删改，仅新增 `/api/member/quota-usage` 只读接口。
3. **暗黑模式**：当前主流移动端浏览器在「白色」基底下保持良好对比度，PRD §4.3 中的暗黑模式适配将在后续 dark theme 全站统一时按相同规范扩展。
4. **大字号模式**：所有蓝紫卡片高度自适应，文字按系统大字号缩放后仍满足 WCAG AA 对比度。
5. **真实支付通道**：当前支付为占位模拟流程，正式支付通道接入由后续独立需求承接。
