## 功能简介

本次修复解决了「体检报告 AI 解读 / 报告对比」场景中，用户一次上传多张图片时**仅显示第一张**的问题。现在：

1. **上传时**：后端完整保留所有上传图片的 URL（新增 `file_urls` / `thumbnail_urls` / `original_image_urls` 字段）。
2. **AI 咨询页**：「查看报告原图」按钮支持九宫格缩略图 + 左右滑动大图预览。
3. **报告详情页**：已支持的多图列表展示继续可用，接口返回不再只给第一张。
4. **报告对比场景**：两份报告各自的多张图片均可独立查看。
5. **历史数据**：服务器启动时自动把历史 OCR 记录中的多图 URL 回填到报告表，**用户无感知**。

## 使用说明

### 场景 1：上传新的体检报告（多张图片）

1. 打开 H5 主页，进入 **"体检报告"** 入口：
   [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/checkup](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/checkup)
2. 选择「上传体检报告」并选择 **多张** 图片（最多 5 张，默认上限可在后台调整）。
3. 选择咨询人，点击「开始 AI 解读」。
4. 系统完成 OCR → 自动进入 AI 对话解读页。

### 场景 2：查看已上传报告的全部图片（AI 解读对话页）

1. 进入「AI 解读」对话页后，页面顶部会出现一张蓝色的「🩺 报告解读」卡片。
2. 卡片上会显示：
   - 咨询对象姓名与关系
   - 报告标题
   - 本次上传的 **图片数量**（如"共 3 张"）
   - 一排 **九宫格缩略图**（超过 4 张显示「+N」按钮可展开全部）
3. **点击任意缩略图** 或点击「查看报告原图」按钮，即可进入 **大图预览**：
   - 左右滑动切换不同图片
   - 底部小圆点显示当前位置
   - 点击黑色背景或返回键退出预览

### 场景 3：报告对比场景查看多图

1. 在「历史报告」列表中选择 2 份报告，进入「报告对比」。
2. 对比对话页顶部会出现黄色的「🔄 报告对比」卡片。
3. 卡片内分别显示 **报告 A** 和 **报告 B**，每份报告后面都有独立的
   「查看 N 张原图」按钮，点击即进入该份报告的多图预览。
4. 两份报告的图片互不干扰，可分别独立浏览。

### 场景 4：查看报告详情页全部图片

1. 进入「历史报告 → 报告详情」。
2. 顶部「📄 报告图片」区域以横向滚动条展示所有图片。
3. 点击任意一张即进入大图多图预览。

## 历史数据说明

- **老报告（修复前上传的）**：服务器启动时执行一次性回溯脚本，根据 `ocr_call_records.image_urls` 自动回填历史记录的 `file_urls` / `thumbnail_urls` / `original_image_urls` 字段。
- **对于无法从 OCR 记录匹配到的老报告**：字段会 fallback 为 `[file_url]`（即与修复前行为一致），保证前端不会出现"字段为空"导致的白屏。
- **用户无感知**：不需要任何手工操作，进入对应页面即可看到历史多图（若历史本身只上传了 1 张则仍显示 1 张）。

## 注意事项

1. **图片数量上限**：默认一次最多上传 5 张，Admin 后台可以调整。
2. **浏览器兼容**：
   - 微信内置浏览器、Safari、Chrome 均正常支持多图预览和左右滑动。
   - iOS 设备首次进入预览时若无法滑动，请重新进入页面一次（新版本已修复）。
3. **网络环境**：多图预览会逐张加载大图，移动网络环境下建议等候片刻。
4. **上传失败回退**：如上传过程中某张失败，其他成功的图片仍然会完整保留，不会出现"全部丢失"的情况。
5. **图片缓存**：若对新数据仍看到"只有一张"，可尝试刷新页面或退出重新进入。

## 技术支持

- 后端接口（已扩展 `file_urls` / `thumbnail_urls`）：
  - `GET /api/checkup/reports/{id}` —— 返回 `images` 数组
  - `GET /api/chat/sessions/{id}` —— 返回 `reports_brief[*].file_urls`
- 前端多图组件：`ImageViewer.Multi`（antd-mobile）
- 数据库变更：`checkup_reports` 表新增 `file_urls` / `thumbnail_urls` 两个 JSON 字段；`checkup_report_details` 表新增 `original_image_urls` JSON 字段。
