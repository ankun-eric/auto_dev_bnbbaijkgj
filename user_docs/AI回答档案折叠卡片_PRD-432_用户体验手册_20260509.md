## 二、功能简介（PRD-432 摘要）

每一条 AI 回答消息的**正上方**新增一张「**咨询对象档案折叠卡片**」，让用户在每次看到 AI 回答时，都能立即确认「**这条建议是基于谁的档案给出的**」。

核心交付：

1. **默认折叠态**：单行紧凑展示「咨询对象姓名 · 性别·年龄 · 关键风险标签」+ 完善度（绿圈 100%/黄圈 60-99%/红圈 0-59%）+「30 天内已更新」对勾标识。
2. **展开态（点击卡片）**：以 2 列网格展示 7 项核心档案：
   - 性别、年龄、身高、体重、过敏史、既往史、长期用药
3. **未完善的字段**：以灰色「未填写」展示，并提供「去完善 →」跳转。
4. **三种"无内容"语义区分**：
   - 用户**未填写** → 显示灰色「未填写」 + 跳转完善
   - 用户**确认无** → 显示「无」（如「过敏史：无」）
   - 用户**填写了** → 显示具体内容
5. **长期用药行可点击**：点击「长期用药」整行 → 弹出**半屏底部 Drawer**，列出全部长期药品（药名、剂量、用法、起止日期），底部「去管理用药」按钮跳转 `/medication-plan`。
6. **未选定咨询对象时**：自动 fallback 到「我自己」（即与登录账号绑定的本人档案），不会出现空卡。
7. **30 天更新标识**：若任一关键字段在最近 30 天内被修改，则在右上角显示一个细小蓝色对勾「已更新」。

## 三、使用说明

### 1）H5 端体验路径

1. 打开 [AI 对话首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)，登录后输入任意问题（例：「最近老咳嗽怎么办」）。
2. AI 回答消息出现后，回答**正上方**会立即显示一张折叠卡片：
   - 默认折叠：紧凑摘要 + 完善度圆环 + 30 天对勾
3. 点击卡片任意位置 → **平滑展开** 7 项档案。
4. 找到「长期用药」行 → 点击 → 屏幕下方弹出半屏 Drawer，可看到全部长期药品。
5. Drawer 底部「去管理用药」→ 跳转用药管理页（可继续新增/编辑/停用）。
6. 切换家庭成员（点击底部输入框上方的「为 XX 咨询」按钮）后，新发出的 AI 回答上方的卡片会**立即变更**为新成员的档案；切换前的旧消息卡片仍保留旧成员档案，**不会被错误改写**。

### 2）小程序端体验路径

1. 用微信开发者工具导入下载得到的 `miniprogram_prd432_*.zip` 解压目录。
2. 选择「测试号 / 体验号」编译运行 → 登录账号。
3. 进入「AI 对话」Tab → 发送任意问题。
4. AI 回答上方同样出现折叠卡片，交互一致：
   - 点击卡片 → 展开 7 项；
   - 点击「长期用药」行 → 半屏 popup 弹出（小程序底部抽屉）。

### 3）Flutter App（Android / iOS）体验路径

1. 通过下方「多端安装包」处的 APK / IPA 链接下载安装。
2. 启动 App → 登录 → 底部 Tab「AI 咨询」。
3. 发送一条问题 → AI 回答 Bubble 上方显示卡片（与 H5 视觉对齐）。
4. 点击卡片 → 展开档案；点击「长期用药」行 → `showModalBottomSheet` 半屏弹出用药列表。

## 四、注意事项 / 边界场景

1. **未选定咨询对象**：自动 fallback 到本人；如本人档案也不存在，则卡片折叠态显示「请先完善本人档案 →」，点击直接跳转 `/health-archive`。
2. **空字段 vs 显式无**：若用户在档案完善页主动选择「无既往史」或「无过敏史」或「无长期用药」（对应数据库 `health_profiles.past_history_is_none / allergy_is_none / medication_is_none = 1`），展开时显示「无」；否则显示「未填写」。
3. **数据时效**：卡片每次进入页面/切换咨询对象时**实时拉取**接口；30 天更新标识根据 `health_profiles.updated_at` 判断。
4. **历史消息一致性**：每条消息内嵌 `consultant_target_id`（数据库字段，本期已迁移），保证历史消息上方卡片永远对应当时实际咨询的对象，不会因后续切换而错位。
5. **接口鉴权**：两个新接口均要求登录态；未登录时返回 401。

| `health_profiles` | `allergy_is_none` | TINYINT(1) | 0 | 用户显式声明「无过敏史」 |

## 八、常见问题（FAQ）

**Q1：为什么我的卡片显示「未完善」？**
A：当前所选咨询对象的健康档案 7 项核心字段不全。点击卡片中的「去完善 →」即可直达 `/health-archive` 完善对应档案。

**Q2：为什么有些消息的卡片显示的是「我自己」，但我明明切到了妈妈？**
A：这是正常行为。卡片严格对齐**消息生成时刻**的咨询对象（`chat_messages.consultant_target_id`）。切换咨询对象后**新发出的**消息的卡片才会展示新对象。

**Q3：「过敏史：无」和「过敏史：未填写」有区别吗？**
A：有。「无」是用户在档案页**主动声明**没有过敏史（更可信，AI 会基于此给建议）；「未填写」是用户尚未提供该信息，AI 在生成回答时会做保守推断并提示「为了更精准，建议补全档案」。

**Q4：长期用药 Drawer 里展示的是哪些药？**
A：展示属于该咨询对象、状态为「**进行中（in_progress）**」且「**频次 ≥ 每日服用 / 长期连续 ≥ 7 天**」的药品。一次性的短期药（如急性发热的退烧药）不会出现在长期用药列表中。
