## 功能简介

本次新增两个高度联动的新能力，让"今日要处理的事"在 AI 首页一眼可见：

1. **铃铛红点合并计数**
   - 顶部 🔔 铃铛右上角出现 **红色圆点 + 白色数字**
   - 数字 = 今天**未服用**的用药条数 + 6 种"球在用户脚下"的**待处理订单**条数
   - 数字 > 9 显示 `9+`，数字 = 0 时铃铛**无任何角标**（最简洁）

2. **AI 首页"今日待办"胶囊**
   - 位置：AI 首页 LOGO 下方、"还没有对话记录"提示文案上方
   - 一行两个胶囊：💊 **今日用药** + 🛒 **待处理订单**
   - 数字 = 0 也始终显示（`0 条待服 / 0 条待办`）
   - 数字 > 0 显示红色小圆点；含**紧急事项**（已过点未服用 / 待支付）时红点**脉动**
   - **整块胶囊可点击**，行为等同于点顶部铃铛 → 弹出统一抽屉

3. **统一抽屉「待办事项 (N)」**
   - 自适应高度（最高占屏幕 80%），底部弹出
   - 上下两区块：**💊 用药提醒 (M)** + **🛒 待处理订单 (K)**
   - 用药：`✓ 已服用` 一键打卡；已过点未服用条目显示 `⚠ 已超时` 标签
   - 订单：手风琴式就地展开详情，主操作按钮按状态文案（去支付/去预约/查看核销码/继续预约/确认收货）
   - 处理完毕的条目折叠到「已完成 (N) ▼」分组，**默认折叠**，跨天自动消失

## 计入红点的订单状态（共 6 个）

| 状态码 | 状态名 | 用户要做什么 | 主操作按钮 |
|---|---|---|---|
| `pending_payment` | 待支付 | 去支付 | 去支付 |
| `pending_appointment` | 待预约 | 选预约时间 | 去预约 |
| `appointed` | 已预约 | 按时到店出示核销码 | 查看核销码 |
| `pending_use` | 待核销 | 出示核销码 | 查看核销码 |
| `partial_used` | 部分核销 | 继续使用 / 预约下一次 | 继续预约 |
| `pending_receipt` | 待收货 | 确认收货 | 确认收货 |

> ⚠️ **不计入红点**的状态：`pending_shipment`（待发货，商家在处理）、`refunding`（退款中，平台在处理）、`completed` / `cancelled` / `expired` / `refunded`（终态）。

## 操作步骤

### 1️⃣ 通过顶部铃铛 🔔 查看待办

1. 打开 [AI 助手首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
2. 看到屏幕右侧悬浮的 🔔 铃铛：
   - 右上角**红色数字**就是今天还有几件事要处理
   - 数字 > 9 显示 `9+`
3. **点击铃铛** → 底部弹出「待办事项」抽屉
4. **关闭抽屉**：点空白区域 / 点右上角 ✕ / 下滑手势均可

### 2️⃣ 通过 AI 首页"今日待办"胶囊查看

1. 打开 AI 助手首页，**不发任何对话**（保持空白态）
2. 在 LOGO（小康头像 + 标题）下方、"还没有对话记录"文案上方，看到两个**白底胶囊**：
   - 左：💊 **今日用药** · `X 条待服`
   - 右：🛒 **待处理订单** · `Y 条待办`
3. 胶囊左侧 3px 竖条：用药 = 天蓝（#0EA5E9），订单 = 橙（#F59E0B）
4. 数字旁边的小红点：
   - **静态**红点 = 有待办
   - **脉动**红点 = 含紧急事项（已过点未服用 / 待支付）
5. **点击任一胶囊** → 同样弹出「待办事项」抽屉
6. 点击胶囊会有轻微按压缩放动效（scale 0.97）

### 3️⃣ 在抽屉里完成用药打卡

1. 抽屉打开后，**第一个区块**是「💊 用药提醒 (M)」
2. 每条用药条目展示：
   - 主标题：药品名（如「降压药」）
   - 副标题：`计划时间 · 剂量 · 备注`（如 `08:00 · 1 片 · 饭后`）
   - 右侧：`✓ 已服用` 按钮
3. 如果某条已过点未服用，时间旁会出现 **`⚠ 已超时`** 黄色标签
4. **点击 `✓ 已服用` 按钮** → 立即打卡：
   - 按钮变灰色 `已服用 HH:MM`
   - 红点数字立即 -1（前端乐观更新）
   - 条目自动折叠到「已完成 (N) ▼」分组里
5. ⚠️ **已打卡不支持撤销**（与今日用药页面保持一致，避免误操作）
6. 想看完整用药计划？点击区块右上角「**查看全部 →**」跳转今日用药打卡页

### 4️⃣ 在抽屉里处理订单

1. **第二个区块**「🛒 待处理订单 (K)」按下单时间倒序展示
2. 每条订单展示：
   - 主标题：商品/服务名
   - 副标题：状态标签 + 金额（待预约状态额外提示 `⏰ 待您选时间`）
   - 右侧：主操作按钮（按状态显示对应文案）
3. **点击条目主体** → **就地展开详情**（手风琴模式，同时只展开一条）：
   - 展开内容包含：订单号、商品名、规格、金额、下单时间、门店/地点（如有）
   - 已预约 / 待核销 / 部分核销：额外显示**预约时间 + 核销码**（到店出示）
   - 部分核销：额外显示**剩余次数**
   - 待收货：额外显示**物流公司 + 物流单号**
4. **点击主操作按钮**：
   - **去支付 / 去预约 / 确认收货** → 自动关闭抽屉跳转订单详情页
   - **查看核销码 / 继续预约** → 就地展开，无需离开抽屉

### 5️⃣ 跨页面同步红点

完成以下任何一个操作，回到 AI 首页时红点数字会**自动减 1**：

- 在抽屉里点 `✓ 已服用`
- 在今日用药页打卡
- 在订单详情页支付完成 / 完成预约 / 确认收货

> 实现机制：全局事件总线 + 页面 visibility 监听 双保险。

## 红点细节规则

| 场景 | 红点表现 |
|---|---|
| 数字 = 0 | 铃铛无任何角标；胶囊数字处不显示红点 |
| 数字 = 5 | 红色圆点显示 `5` |
| 数字 = 12 | 红色圆点显示 `9+` |
| 已过点未服用 | 用药胶囊红点**脉动**（医学优先级最高） |
| 含待支付订单 | 订单胶囊红点**脉动**（订单可能 30 分钟内超时） |
| 接口请求失败 | 铃铛无红点（兜底当作 0）；抽屉打开后显示「**加载失败，点击重试**」 |
| 两区块都为空 | 抽屉显示 `✨ 暂无待办事项，一切都很顺利` |
| 单区块为空 | 用药为空：`今日用药已全部完成 ✅`；订单为空：`暂无待处理订单` |

## 注意事项

1. **登录后才能看到红点**：未登录时 `/badge` 接口返回 401，铃铛和胶囊数字都显示 `0`、不显示红点
2. **跨天自动刷新**：所有"已完成"分组里的条目会在**次日 0 点**随"今日"刷新自然消失，无需手动清理
3. **打卡不可撤销**：与今日用药页保持一致，避免误操作
4. **本期不做的能力**：
   - 系统级 App 推送 / 微信模板消息（系统已有相关推送）
   - 首次进入引导浮层（抽屉本身已足够直观）
   - 暗黑模式适配（下版本统一规划）
   - 抽屉打开 / 关闭动画（使用现有默认动画）
5. **后端 API 向后兼容**：旧客户端读取 `medication_unchecked` / `appointment_pending` / `total` 字段仍可工作，新客户端读取 `medication.count` / `medication.has_urgent` / `order.count` / `order.has_urgent` / `order.breakdown` 嵌套对象

## 测试覆盖

后端单元测试已新增 12 个验收用例（与既有 21 个 PRD-439 用例合计 **33/33 全部通过**），覆盖：

- `/badge` 旧字段向后兼容 + 新嵌套字段
- 6 个订单状态的合并计数 + 状态细分（breakdown）
- 终态（pending_shipment / completed 等）不计入红点
- 过点未服用 → `medication.has_urgent = true`
- 存在待支付订单 → `order.has_urgent = true`
- `/appointments` 默认 6 状态 + `status_in` 参数过滤
- 扩展字段 `status / amount / remaining_redeem_count / verification_code / status_text` 等
- 无效 `status_in` 参数自动 fallback 到 6 状态默认集
