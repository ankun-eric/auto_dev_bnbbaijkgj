## 功能简介

本次修复解决了系统无法通过 HTTPS 访问的问题。之前访问 H5 页面、Web 管理后台和安卓端链接时，浏览器显示"无法访问此网站 — ERR_CONNECTION_CLOSED"错误。

### 问题原因

服务器的 gateway-nginx 网关仅配置了 HTTP（80端口）监听，未配置 HTTPS（443端口）的 SSL 支持。而项目的所有访问链接均使用 `https://` 协议，导致浏览器连接到 443 端口时被拒绝。

### 修复内容

1. 为 gateway-nginx 生成了自签名 SSL 证书
2. 在 gateway-nginx 配置中添加了 HTTPS（443端口）server 块
3. 在 gateway docker-compose 中暴露了 443 端口并挂载 SSL 证书目录
4. 添加了静态文件（APK/小程序zip包）的下载路由配置
