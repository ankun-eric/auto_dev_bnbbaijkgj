## 功能简介

本次 v1.3.2 补丁是 v1.3.1「守护人体系列表口径一致 + 真删除 + 配额防护」批次的 **二次清扫**，针对用户上线后实测发现的 3 个未落地干净的漏改点：

| 编号 | 现象 | 修复后效果 |
|------|------|-----------|
| **G1** | AI 首页 → 选择咨询人 → 新建 → 顶部【去邀请】跳到邀请页后，**只有"关系"下拉、没有"姓名"输入框**，提交时被后端 422 拦截，用户卡死 | 邀请页（无 `member_id` 时）现在渲染统一的「姓名 + 关系」两个输入框，UI 与「健康档案 → 我守护的人 → 新建邀请」抽屉完全一致；用户填完即可生成邀请二维码 |
| **G2** | 后端配额计算未把"悬空 pending 邀请"算入已用名额，"反复邀请-取消"可刷大量脏邀请记录但配额数字不涨 | 配额公式升级为 PRD §4.2「蚂蚁阿福派补丁版」：`已用 = family_member 非 deleted 数 + 悬空 pending 未过期数 - 1（本人）`。反复邀请-取消立即对配额数字生效 |
| **G3** | AI 首页 → 选择咨询人抽屉中，"对方已退出"的家人卡片仍按彩色正常态渲染，与"我守护的人"页面灰标体验不一致 | 选择咨询人抽屉新增「对方已退出」灰卡片渲染：浅灰背景 + 灰度头像 + 灰色"对方已退出"胶囊，且不允许选为咨询对象，点击 Toast 提示 |

> 💡 同时本次补齐了「我守护的人」内邀请抽屉的「纯特殊字符拦截」校验（如 "!!!"、"???"、"---"），与「新建家庭成员」抽屉、「邀请页」三端校验字符级一致。

## 使用说明

### 场景 A：从 AI 首页发起新邀请（G1 修复）

> 这是本次主要修复的入口流程。

**操作步骤**：

1. 打开 H5 主页：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)
2. 点击左下角 **「选择咨询人」** 按钮，弹出咨询人选择抽屉
3. 点击抽屉里的 **「+ 新增家庭成员」**，弹出新建抽屉
4. 不直接建档，点击顶部蓝色横幅 **「邀请 TA 成为我守护的人 → 去邀请」**
5. **修复后**：跳转到 `/family-invite/` 页面后，会看到「填写邀请信息」表单：
   - **姓名**（红星 *，必填，最长 20 字符）
   - **关系**（红星 *，必填，例如：父亲、母亲、配偶）
6. 填写完毕后点击 **「生成邀请码」** 按钮
7. 生成成功后展示邀请二维码 + 邀请链接，可保存图片 / 复制链接 / 转发微信好友

> ✅ **校验规则**（与场景 B 完全一致）：
> - 姓名 trim 后非空、长度 1–20、不允许纯特殊字符（如 "!!!"、"???"），但允许 emoji（如 "张妈妈❤"）
> - 关系 trim 后非空、长度 1–10
> - 任一不合规会 Toast 提示，不会提交到后端

### 场景 B：从「我守护的人」页面发起新邀请

**操作步骤**：

1. 打开「我守护的人」页：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard/)
2. 点击顶部 **「+ 发起邀请」** 按钮，弹出抽屉
3. 在抽屉中填写：
   - **姓名**（红星 *，必填）
   - **关系**（红星 *，必填）
4. 点击「生成邀请二维码」即可

> ✅ 本次补齐：本抽屉同样会拦截纯特殊字符（如 "!!!"），与场景 A 体验一致。

### 场景 C：选择咨询对象时跳过"对方已退出"的家人（G3 修复）

**前提**：你曾守护某人 B，B 在自己的 App 里主动退出了守护关系。

**操作步骤**：

1. 打开 H5 主页 → 左下角「选择咨询人」
2. 在咨询人列表里观察 B 的卡片：
   - **修复后**：B 的卡片为 **浅灰色背景 + 灰度头像 + 浅灰胶囊"对方已退出"**，视觉上明显与正常家人区分
   - 灰色卡片排在列表末尾（视觉降权）
3. 尝试点击 B 的卡片：
   - **修复后**：弹出 Toast 提示 **"该对象已退出守护，无法进行 AI 咨询"**，不会切换咨询对象

### 场景 D：配额数字与"反复邀请-取消"压测（G2 修复）

> 此场景多用于验证开发，普通用户不易感知，但配额数字现在更准确。

**预期表现**：

- 进入「我守护的人」页面，顶部卡片显示「我守护：X / Y 人」
- 创建一条悬空 pending 邀请（场景 A 或场景 B 任一） → X 立即 +1
- 撤销该 pending 邀请 → X 立即 -1
- 自然过期（24 小时无响应）→ X 自动 -1
- 不会出现"反复邀请-取消但 X 不变"的脏数据漂移

## 注意事项

1. **缓存刷新**：H5 已重新部署，如发现页面行为仍是旧版，请清除浏览器缓存或强制刷新（Ctrl+F5 / Cmd+Shift+R）。
2. **已有邀请记录**：本次仅修改"创建邀请"路径，已经在数据库里的邀请记录不会被重写。
3. **小程序未同步**：按本次产品决策（D8），小程序端**搁置到下一个补丁批次**，本次只改 H5 + 后端。小程序用户的相关入口暂时维持 v1.3.1 行为。
4. **配额规则升级**：如发现自己的"已用配额"突然增多，是因为新公式把悬空 pending 邀请也算入已用，属于正常修复后的口径变化。可通过撤销不需要的 pending 邀请释放配额。
5. **姓名校验**：本次姓名校验三端字符级对齐 — 场景 A、场景 B、场景 C 三处只要任一拒绝，其他两处也一定拒绝。
6. **关系下拉无默认值**：场景 A（邀请页）的"关系"输入框不带默认值，必须用户显式输入，避免误提交错误关系。
