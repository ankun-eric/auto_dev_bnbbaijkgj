# 统一消息中心 · 铃铛单入口 使用手册

> 版本：FINAL v1.0　｜　适用范围：H5 网页端　｜　更新日期：2026-06-02

## 一、功能介绍

为了让你不再被"今日待办"和"系统通知"两个分散的入口搞晕，我们把它们**合并到了主页右上角的一个铃铛里**。从此：

- **一个铃铛，搞定所有**：点开主页顶栏的 🔔 铃铛，既能看"今天该做的事（待办）"，也能看"系统给你的消息（通知）"，在抽屉里一键切换。
- **红点不再骗人**：铃铛右上角的红点数 = 未完成待办数 + 未读通知数，点进去看到的内容和红点数字完全对得上，**彻底告别"红点有数字、点进去却加载失败"的问题**。
- **概念清清楚楚**：
  - **待办 = 催你办事**：今天该吃没吃的药、待支付 / 待预约 / 待核销 / 待收货的订单。
  - **通知 = 告诉你消息**：家人血压/血糖异常、守护关系变动、家人邀请/授权、设备报警等。

> 同时，旧的"消息通知"页面（`/notifications`）已下线，访问它会自动跳转到新的统一消息中心，无需手动操作。

## 三、操作步骤

### 1. 查看待办（默认显示）

1. 点击主页右上角 🔔 铃铛，弹出抽屉，**默认停在「待办」分段**，标题显示「待办事项 (N)」。
2. 这里展示：
   - **今日用药**：今天需要服用的药，可直接打卡、撤销打卡。
   - **待处理订单**：待支付、待预约、待核销、待收货等，可展开查看详情并跳转处理。
3. 完成对应操作后，待办数量会自动减少，铃铛红点也随之下降。

### 2. 查看系统通知

1. 在抽屉**顶部点击「通知」分段**，切换到通知列表，标题变为「系统通知 (N)」。
2. 列表中每条通知显示：**类型图标**、标题、内容、时间，未读的会**高亮并带红点**。
3. **点击某条通知**：自动标记为已读，并跳转到对应详情页（例如：家人绑定列表、邀请页、订单详情等）。

### 3. 标记已读

- **单条已读**：点击某条通知即视为已读，红点消失。
- **全部已读**：当存在未读通知时，通知分段右上角会出现「全部已读」按钮，点击后清空所有未读红点。

### 4. 访问消息中心（完整列表）

如需查看更完整的历史消息列表，可访问消息中心页面：
`https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/messages`

> 旧地址 `https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/notifications` 已自动重定向到上面的新地址。

## 四、注意事项

1. **待办与通知互不串数据**：待办只放"用药 + 订单"，通知只放"系统消息"，两者数据来源不同，请按分段查看，不会混在一起。
2. **红点 = 待办未完成 + 通知未读**：如果红点不为 0，请分别检查两个分段，把待办做完、把通知读掉，红点即可清零。
3. **加载失败时**：若网络异常导致通知加载失败，列表会显示"加载失败，点击重试"，点击即可重新加载；若本来就没有通知，则显示"暂无系统通知"的空状态，二者已做区分。
4. **设备报警类通知**：属于安全兜底类消息，建议优先处理，不要忽略。
5. **本次范围**：本次升级仅覆盖 H5 网页端；小程序、App 端将在后续版本同步。

## 五、常见问题（FAQ）

**Q1：以前的两个铃铛 / 两个入口去哪了？**
A：已合并为主页顶栏右上角的唯一铃铛。"咨询"旁的红点跳转、旧的 `/notifications` 页面入口均已移除。

**Q2：红点数字和点进去看到的消息对不上怎么办？**
A：本次升级已把红点计数和列表数据统一为同一数据源，二者现在完全一致。如仍有疑问，请下拉刷新或重新进入页面。

**Q3：点了"全部已读"后还能看到这些通知吗？**
A：能。"全部已读"只是清掉红点未读状态，历史通知记录仍保留在通知列表中。
