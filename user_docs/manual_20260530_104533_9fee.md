## 功能简介

本次修复解决了健康档案 - 档案列表页面**一进入即弹出"加载失败：Not Found"** 的故障，并完善了前端的错误兜底体验。

### Bug 现象

- 入口：健康档案首页 → 顶部「档案列表」入口卡片
- 现象：点击进入档案列表页后，页面立即弹出 toast 提示 **"加载失败：Not Found"**（中英混合）
- 后果：档案列表完全不可用，用户无法管理家人档案、邀请、配额等

### 真实根因（已定位）

经远程诊断确认：
- 前端档案列表页调用的是 `GET /api/family/member/state/list`（v2 状态机批量查询接口）
- 该接口在源码中确实存在，但**测试服务器上的后端容器使用的是旧版本镜像**，旧版镜像中的 `app/main.py` 还没有 `app.include_router(family_member_v2.router)` 这段路由注册代码
- 因此请求直接命中 FastAPI 的兜底 404，返回 `{"detail":"Not Found"}`，前端拼接后显示为 "加载失败：Not Found"

## 本次修复内容

### 1. 后端修复（已部署到测试服务器）

- 重新构建并发布后端容器镜像 `6b099ed3-7175-4a78-91f4-44570c84ed27-backend`
- 镜像内的 `app/main.py` 现已包含 `family_member_v2` 路由的注册代码
- 验证：在容器内打印路由表，已能看到以下 v2 路由全部生效：
  - `GET /api/family/member/state/list`（档案列表批量查询）
  - `GET /api/family/member/quota`（家庭成员配额）
  - `GET /api/family/member/{member_id}`（单个成员状态查询）
  - `POST /api/family/member/{member_id}/invite`（重新邀请）
  - `POST /api/family/member/{member_id}/unbind`（解除守护）
  - `DELETE /api/family/member/{member_id}`（统一删除）
- 接口实测结果：通过外部 URL 直接 curl，返回 **HTTP 401 + 中文 `{"detail":"未登录"}`**（不再是 404），证明路由注册已成功生效

### 2. 前端兜底优化（已部署到 H5 容器）

针对档案列表页（`/health-profile/archive-list`）的 `fetchList` 错误处理逻辑做了如下增强：

- 不再把 `err.response.data.detail` 原样拼接到中文 "加载失败：" 前缀后面，避免再次出现 "加载失败：Not Found" 这种中英混合裸露给用户
- 区分 HTTP 状态码给出对应中文提示：
  - **404**：`档案数据接口暂不可用，请稍后重试`
  - **401 / 403**：`登录已过期，请重新登录`
  - **其他/网络错误**：`加载失败，请稍后重试`
  - 后端返回的 detail 仅在**不是 "Not Found"** 时才透传
- 失败时同步清空 `list` 状态，让页面进入「暂无档案」空态，而不是停留在 "加载中…"

## 排查/复现方法

如未来再出现类似"接口路由生产不可达"的问题，可按以下步骤诊断：

1. **抓接口**：浏览器 F12 → Network → 复现页面 → 找红色 4xx 请求，记录完整 URL
2. **判别 404 类型**：
   - FastAPI 的 404 → JSON `{"detail":"Not Found"}`、Content-Type 为 `application/json` → 路由未注册到容器内的 app
   - nginx 的 404 → HTML 页面 → 网关路由前缀错配
3. **检查容器内路由表**：
   ```bash
   docker exec {容器名} python -c "from app.main import app; print('\n'.join([r.path for r in app.routes if hasattr(r,'path')]))" | grep 路由关键词
   ```
4. **如缺失**：表示需要重新构建并部署后端容器（`docker compose build backend && docker compose up -d backend`）
