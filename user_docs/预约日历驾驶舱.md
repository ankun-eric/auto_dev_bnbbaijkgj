## 功能简介

本次将商家 PC 后台的「预约日历」从一个**只读看板**升级为**店长经营驾驶舱 + 轻量操作台**。一屏即可掌握"今日 / 本周 / 本月"预约数量与档期忙闲，月/周/日/资源/列表 5 种视图自由切换，单元格点击即可完成核销、改约、联系顾客、查看详情，加盟体系下门店账号严格只看自己门店数据。

### 核心能力一览

| 能力 | 说明 |
|---|---|
| 顶部 KPI 驾驶舱 | 三张大卡：今日 / 本周 / 本月预约数（不含已取消），点击卡片自动切到对应粒度 |
| 五视图切换 | 月 / 周 / 日 / 资源 / 列表，工具条 Tab 一键切换 |
| 单元格信息密度 | 月/周视图：`预约 X · 已核 Y · 占用 Z% · ¥金额` 4 字段 + 悬浮 tooltip 含取消数 |
| 资源视图 | 横轴时间、纵轴服务项目（甘特式），快速识别哪个项目最热 |
| 多条件筛选 | 服务项目 / 员工 / 状态 / 来源 / 顾客（手机号/姓名）多选组合 |
| 我的视图 | 把"筛选 + 视图类型"组合保存为 `我的视图`，每账号最多 10 个，可设默认 |
| 快捷菜单 | 日/资源/列表视图卡片点击 → 4 个动作 popover：核销 / 改约 / 联系顾客 / 查看详情 |
| 改约面板 | 改时间、改服务项目、改服务员工，三段任改其一即可提交 |
| 顾客自动通知 | 5 个场景（预约成功 / 改约 / 取消 / 行前 D-1 / 行前 H-1）走小程序订阅消息（本期为接口层落地，调度日志全量留痕） |
| 加盟权限隔离 | 门店账号仅看自己门店；总部账号本期不进入预约日历 |

## 操作步骤

### 1. 早会场景：1 屏查看今日 / 本周 / 本月预约数

1. 打开浏览器，访问 [商家 PC 预约日历](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/calendar)
2. 输入门店账号登录
3. 页面顶部即可看到三张 KPI 大卡：**今日预约数 / 本周预约数 / 本月预约数**
4. 数字仅统计「待核销 + 已核销」状态，已取消订单不计入

### 2. 切换 5 种视图

工具条左侧 Tab 一键切换：

| 视图 | 适用场景 | 数据展现 |
|---|---|---|
| 月视图 | 看一整月档期分布 | 7×N 月历，每个日期格子 4 字段（预约/已核/占用/营收） |
| 周视图 | 看本周忙闲 | 周一~周日 7 列，单元格信息口径与月视图一致 |
| 日视图 | 当天精细化排期 | 时间网格（30 分钟粒度）× 服务项目分组 |
| 资源视图 | 找最热项目 | 横轴时间 × 纵轴服务项目（甘特图） |
| 列表视图 | 经典表格化清单 | 日期/时间/顾客/项目/员工/状态/金额/来源/操作 |

> 切换视图时**已选筛选条件保留**，无需重新设置。

### 3. KPI 卡片快速定位

- 点击「今日」卡片 → 自动切到日视图、定位今天
- 点击「本周」卡片 → 自动切到周视图、定位本周
- 点击「本月」卡片 → 自动切到月视图、定位本月

### 4. 多条件筛选

工具条右侧「筛选」按钮 → popover 展开：

- **预约状态**（多选）：待核销 / 已核销 / 已取消 / 已退款 / 其它
- **来源**（多选）：小程序 / H5 / 到店 / 电话 / 后台代下
- **顾客搜索**：搜索框输入手机号或姓名（模糊匹配）

筛选条件确认后立即生效，所有视图同步使用。

### 5. 保存为「我的视图」

1. 设好筛选条件 + 视图类型后，点击工具条「我的视图」下拉
2. 选择「保存当前为我的视图」
3. 弹窗输入名称（≤20 字）+ 是否设为默认
4. 视图列表里一键应用，**单账号最多保存 10 个**
5. 设置默认视图后，下次登录自动加载该视图

### 6. 现场处理：核销 / 改约 / 联系顾客 / 查看详情

在「日视图」、「资源视图」或「列表视图」下，点击预约卡片或行末「操作」下拉：

- **核销**：跳到现有核销流程（扫码/手动二选一）
- **改约**：弹出改约面板，可改时间 / 服务项目 / 服务员工
  - 时间：以营业时段为基础，已被占满的时段自动置灰
  - 服务项目：仅可改为同价或差价兼容的项目
  - 服务员工：本期可空（待员工库接入后启用）
  - 提交后自动给顾客发送「改约通知」小程序订阅消息
- **联系顾客**：一键发起小程序订阅消息（默认渠道）
- **查看预约详情**：跳到订单详情页，可看顾客标签和消费历史

> 月视图和周视图的单元格点击会跳转到日视图（不直接弹快捷菜单），保持视觉清爽。

### 7. 加盟门店权限

- 门店账号登录后**仅看本门店数据**：预约、KPI、单元格、列表、服务项目、员工、顾客
- UI 上**不显示门店切换器**，不感知其他门店是否存在
- 「我的视图」按账号 + 门店双重隔离，跨门店账号不可见
- 总部账号本期暂不进入预约日历（菜单可隐藏或显示"暂未开放"）

### 8. 顾客自动通知（5 个场景全套）

| 场景 | 触发时机 | 模板 |
|---|---|---|
| 预约成功 | 顾客下单或商家代下单成功 | 预约成功通知 |
| 商家改约 | 商家在快捷菜单完成改约 | 改约通知（含原时间 + 新时间） |
| 商家取消 | 商家发起取消 | 取消通知 |
| 行前 D-1 提醒 | 服务开始时间前 24h | 行前提醒 D-1 |
| 行前 H-1 提醒 | 服务开始时间前 1h | 行前提醒 H-1 |

> **顾客未授权小程序订阅消息**：失败静默处理，预约详情页会展示"通知未授权"小标识，便于商家电话兜底。
> **频控与去重**：行前 D-1 / H-1 以「预约 ID + 模板 ID」去重；改约/取消通知必发。

## 注意事项

1. **状态口径**：KPI 数字、cells 单元格 `预约 X` 字段、items 卡片只统计 **待核销 + 已核销**，已取消订单不计入主数字（但月/周视图悬浮 tooltip 中仍展示"取消 X"）。
2. **占用率口径**：`占用率 = 已约时段数 / 当日营业时段总数`，向下取整。门店未维护营业时段时按默认 22 个 30 分钟时段（9:00-20:00）计算；若门店配置了营业时段，自动按配置取数。
3. **营收口径**：单元格 `¥金额` 取自订单实收金额，**已退款不计**。
4. **权限边界**：所有日历接口都强制走门店权限校验。门店 A 账号即使知道门店 B 的 `store_id` 直接调接口也会被服务端 403 拦截。
5. **筛选保存**：「我的视图」按 `账号 × 门店` 隔离，跨账号不可见。同账号同门店超过 10 个会被服务端拒绝创建。
6. **改约约束**：本期改服务项目仅支持同价或差价兼容，跨价格段需走"取消重约"流程。
7. **未在本期范围（v1.1+ 再做）**：环比 / 同比箭头、异常预警中心、Excel 导出、拖拽改约、代客下单、批量动作、PC→H5 二次大改、总部跨店汇总驾驶舱、短信渠道。
8. **小程序订阅消息授权率**：实际发送结果可能为 `no_subscribe`（顾客未授权），属正常情况。商家可在预约详情页看到"通知未授权"标识，电话兜底。
9. **接口路径**：本次新增 9 个接口前缀均为 `/api/merchant/calendar/*` 和 `/api/merchant/booking/*`，参考 [API 接口章节](#接口清单)。
10. **测试覆盖**：本次新增 18 个 pytest 用例，叠加既有 13 个回归用例，远程容器 **31/31 全绿**，验证 KPI 口径、状态合并、排序规则、权限隔离、改约链路、通知日志、内部扫描接口等关键路径。

## 接口清单

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/merchant/calendar/kpi` | 顶部 KPI 三数字 |
| GET | `/api/merchant/calendar/cells` | 月/周/日单元格 4 字段（预约/已核/占用/营收） |
| GET | `/api/merchant/calendar/items` | 日/资源视图卡片列表 |
| GET | `/api/merchant/calendar/list` | 列表视图（分页） |
| GET | `/api/merchant/calendar/views` | 我的视图列表 |
| POST | `/api/merchant/calendar/views` | 创建/保存我的视图 |
| PUT | `/api/merchant/calendar/views/{id}` | 修改我的视图 |
| DELETE | `/api/merchant/calendar/views/{id}` | 删除我的视图 |
| POST | `/api/merchant/booking/{order_item_id}/reschedule` | 改约 |
| POST | `/api/merchant/booking/{order_item_id}/notify` | 联系顾客（手动触发订阅消息） |
| POST | `/api/merchant/internal/calendar/notify-scan` | 行前 D-1/H-1 定时扫描内部接口 |

> 所有接口均要求登录态 + `merchant_dep`（商家 owner / staff）+ `_ensure_store_access` 三重校验。
