## 功能简介

本次修复了管理后台「商品体系 → 商品管理」中的一个严重 Bug：

**修复前**：编辑统一规格商品时，只要从"基本信息"Tab 切换到"预约与核销"等其他 Tab 后保存，商品的售价和库存会被意外写入 0，导致商品价格数据丢失。

**修复后**：无论在哪个 Tab 之间切换，所有已填写的表单字段值都会完整保留。保存时的售价和库存始终为用户实际填写的值，不会因 Tab 切换而被清零。

## 使用说明

### 正常编辑商品流程（修复后）

1. **打开管理后台**：访问 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)，使用管理员账号登录

2. **进入商品管理**：在左侧菜单中依次点击「商品体系」→「商品管理」

3. **编辑商品**：在商品列表中找到需要编辑的商品，点击「编辑」按钮

4. **自由切换 Tab**：编辑弹窗左侧有 5 个 Tab 导航：
   - **基础信息**：商品名称、分类、售价、库存、图片等
   - **标签设置**：症状标签、适用体质
   - **积分设置**：积分兑换、积分价格
   - **预约与核销**：预约模式、核销次数、支付超时等
   - **排序与权重**：推荐权重、排序值、状态

   现在可以在这些 Tab 之间自由切换，所有已填写的数据都会完整保留。

5. **修改预约设置**：切换到「预约与核销」Tab，修改预约模式（如切换为"预约日期"），填写提前可预约天数、单日最大预约人数等信息

6. **保存**：点击弹窗底部的「保存」或「保存并上架」按钮

7. **验证**：重新点击「编辑」打开该商品，确认售价和库存与之前一致

### 验证修复效果的步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 选择一个统一规格商品，记下当前售价和库存 | - |
| 2 | 点击「编辑」，切换到「预约与核销」Tab | 基本信息中的售价和库存不会丢失 |
| 3 | 修改预约模式为「预约日期」，填写相关配置 | 正常 |
| 4 | 点击「保存」 | 保存成功 |
| 5 | 重新点击「编辑」查看该商品 | 售价和库存与步骤 1 记录的一致 |

## 注意事项

1. **仅影响管理后台**：本次修复仅涉及管理后台的商品编辑功能，不影响用户端 H5、APP、小程序等前台展示

2. **多规格商品不受影响**：多规格商品的价格和库存由 SKU 列表管理，本 Bug 原本就不影响多规格商品

3. **历史数据**：如果之前因为此 Bug 导致某些商品的价格或库存被错误清零，需要手动编辑这些商品恢复正确的价格和库存

4. **「下单预约方式」说明**：「下单预约方式」（下单即预约 / 先下单后预约）的选择与此 Bug 无关，可正常使用

## 技术说明

本次修复采用了双保险策略：

- **方案一（根治）**：将商品编辑弹窗的 Tab 切换从"条件渲染"改为"CSS 显隐控制"，确保所有表单字段始终保持挂载状态，切换 Tab 时数据不会丢失

- **方案二（兜底）**：在保存提交时，如果字段值意外为空，会回退到编辑前的原始值，而不是直接写入 0
