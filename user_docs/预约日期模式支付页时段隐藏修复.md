## 功能简介

本次修复了一个 P1 级用户体验 Bug：**预约日期模式（`appointment_mode = date`）的商品在支付/下单页错误地展示了「选择时段」一栏**。该模式按设计只按"天"维度限流，根本无需选择时段。

修复后的效果：

| 预约模式 | 支付页应显示 |
| --- | --- |
| **不预约**（none） | 不显示任何预约控件 |
| **预约日期**（date） | 仅显示「日期 + 门店」，**不再显示「选择时段」** ✅ |
| **预约时段**（time_slot） | 显示「日期 + 时段 + 门店」（保持原有功能） |
| **自定义表单**（custom_form） | 跳转自定义表单页 |

同时修复了订单详情页 / 列表页 / 商家后台订单抽屉对 `date` 模式订单也错误展示"预约时段"行的问题。

## 使用说明（验证修复效果）

### 场景一：date（预约日期）模式商品下单

1. 在商家后台创建一个商品，预约模式选择「**预约日期**」，配置可预约日期范围
2. 用户在 H5 / 小程序 / 安卓 App / iOS App 任一端打开该商品详情，点击「立即购买」
3. 进入支付页，应该看到：
   - ✅ 显示「① 选择日期」
   - ✅ 显示「③ 选择门店」
   - ❌ **不再显示**「② 选择时段」（修复前会错误显示）
4. 选择日期 + 门店后点击「立即支付」，订单应当成功创建
5. 进入订单详情页，应该看到：
   - ✅ 「预约日期」一行
   - ❌ **不再显示**「预约时段」一行
6. 商家在管理后台「产品系统 → 订单管理」打开该订单详情抽屉，「预约信息」卡片同样不再显示"预约时段"行

### 场景二：time_slot（预约时段）模式商品下单（回归验证）

1. 商家后台创建一个商品，预约模式选择「**预约时段**」，配置时段（如 09:00-10:00、10:00-11:00 等）
2. 用户支付页应正常显示：日期 + 时段 + 门店三栏
3. 选择完整后下单，订单详情应同时展示"预约日期 + 预约时段"

### 场景三：none / custom_form 模式（回归验证）

| 模式 | 预期 |
| --- | --- |
| `none`（不预约） | 支付页完全不显示预约控件，下单流程不受影响 |
| `custom_form`（自定义表单） | 下单时跳转自定义表单页（已有逻辑，本次不涉及） |

## 注意事项

1. **历史订单不受影响**：本次修复采用"展示层屏蔽 + 后端忽略"方案，**未对数据库做任何改动**。已有 `appointment_data.time_slot` 脏值的历史 date 模式订单：
   - 数据库中的脏值原样保留（便于将来回滚）
   - 但前端 / 后台展示时一律按 `appointment_mode === 'time_slot'` 判断是否渲染时段，**对用户透明**
2. **后端兜底**：即便用户使用旧版 App 提交订单仍误传 `time_slot` 字段，后端会**主动剥离**，不会写入新订单；旧版用户不会因此报错
3. **时段模式不受影响**：所有 `time_slot` 模式商品的下单与展示流程**完全保持原状**
4. **自动化测试**：本次新增 5 个 pytest 用例，远程容器内 5/5 全部通过：
   - date 模式不传时段 → 200 OK
   - date 模式误传时段 → 200 OK 且字段被剥离
   - time_slot 模式完整提交 → 200 OK 且数据回传
   - none 模式下单 → 200 OK 且不带预约信息
   - 订单详情接口透传 `appointment_mode` → 字段存在
5. **若 App 升级后仍看到时段栏**：极少数缓存问题——卸载重装即可

## 修复的核心改动文件

| 端 | 文件 | 改动要点 |
| --- | --- | --- |
| H5 | `h5-web/src/app/checkout/page.tsx` | `needAppointment` 拆为 `needDate / needTimeSlot`，时段块改 `needTimeSlot` 渲染，提交时 date 模式不传 `time_slot` |
| H5 | `h5-web/src/app/unified-order/[id]/page.tsx` | 订单详情时段标签按 `appointment_mode === 'time_slot'` 渲染 |
| 小程序 | `miniprogram/pages/checkout/index.{js,wxml}` | data 加 `appointmentMode/needTimeSlot`；wxml 时段块加 `wx:if="{{needTimeSlot}}"` |
| Flutter | `flutter_app/lib/screens/product/checkout_screen.dart` | 新增 `_needDate`/`_needTimeSlot` getter，时段 Wrap 包 `if (_needTimeSlot)` |
| Flutter | `flutter_app/lib/screens/order/unified_order_detail_screen.dart` | 时段行按 `item.appointmentMode == 'time_slot'` 渲染 |
| admin-web | `admin-web/src/app/(admin)/product-system/orders/page.tsx` | 抽屉「预约时段」行按 `item.appointment_mode === 'time_slot'` 渲染 |
| 后端 | `backend/app/api/unified_orders.py` | date 模式下从 `appointment_data` 主动剥离 `time_slot`（即便前端误传） |
| 后端 | `backend/tests/test_checkout_date_mode_no_time_slot.py` | 新增 5 个 pytest 用例 |
