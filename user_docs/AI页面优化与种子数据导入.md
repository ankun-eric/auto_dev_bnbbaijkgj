## 一、本次更新做了什么（一句话）

1. 修复了 H5 端两处"健康档案"入口点击后跳到 404 / 旧页面的问题，统一指向新版 `/health-profile`；

3. 在管理后台新增了【系统设置 → 种子数据导入】页面，运营可以一键导入 / 卸载 6 个种子包；
4. 把"健康自查"两个入口合并成一个唯一入口；
5. 清理了"空壳的 36 题模板"孤儿数据，今天再点【中医体质测评】就能看到完整 36 道题。

#### 3.2.1 页面结构

页面以表格形式展示 **6 个种子包**：

| # | 种子包名称 | 包含内容（摘要） |
|---|---|---|
| 1 | 中医体质测评（王琦国标 36 题） | 1 模板 + 36 题 + 9 分型规则 + 1 功能按钮 + 11 默认关键词 |
| 2 | 抑郁筛查 PHQ-9 | 1 模板 + 9 题 + 分型规则 + 追问 chips + CTA |
| 3 | 焦虑筛查 GAD-7 | 1 模板 + 7 题 + 分型规则 + 追问 chips + CTA |
| 4 | 匹兹堡睡眠质量指数 PSQI | 1 模板 + 19 题 + 分型规则 + 追问 chips + CTA |
| 5 | 健康自查 6 维度升级包 | 部位+症状+持续+性质+严重程度+备注 |
| 6 | 9 种体质标签 | 平和质 / 气虚质 / 阳虚质 / 阴虚质 / 痰湿质 / 湿热质 / 血瘀质 / 气郁质 / 特禀质 |

每行的列：种子包名称、简版说明、包含内容摘要、**当前数据库状态**（已导入 / 未导入 / 部分导入 / 已被运营修改 / 未知）、操作按钮。

#### 3.2.2 操作 1：查看完整内容

- 点击行内 **【查看完整内容】**，弹出抽屉式详情，展示：
  - 种子包元信息（名称、code、版本、来源/出处、最近更新时间）；
  - 导入会做什么（逐条列出将要 INSERT/UPDATE 的表名 + 数据量预估）；
  - 数据完整明细（可折叠展示题目原文、分型规则等）；
  - 操作记录（最近一次导入时间、操作人）。

#### 3.2.3 操作 2：一键导入

1. 点击 **【一键导入】**；
2. 弹窗二次确认："将导入 X 个表、Y 条记录，是否继续？"
3. 若库里已有同 code 数据，会追问选择 **覆盖** / **跳过** / **取消**：
   - 覆盖：先删除现有再导入；
   - 跳过：保留现有不动；
4. 成功后提示："已成功导入。如不需要，可在【通用问卷模板管理】和【功能按钮管理】里手动删除，**不会再自动恢复**。"
5. 失败回滚：整个导入事务包裹，任一步失败全部回滚，前端会展示具体错误。

#### 3.2.4 操作 3：一键卸载

1. 点击 **【一键卸载】**；
2. 二次确认："将删除该种子包的全部数据，请确认是否继续？"
3. 执行删除，事务包裹，任一步失败回滚。

#### 3.2.5 状态说明

| 状态 | 含义 |
|---|---|
| 未导入 | 库里完全没有该种子包的核心数据 |
| 已导入 | 库里已有该种子包，且与代码内的标准定义完全一致 |
| 部分导入 | 库里有该种子包，但题目/分型条数与标准不一致（被人改过或漏导入） |
| 已被运营修改 | 库里有该种子包，但部分字段被运营修改过（覆盖会丢失） |
| 未知 | 无法准确检测（建议查看完整内容确认） |

## 四、常见问题（FAQ）

**Q1：以前每次部署都会自动出现的"中医体质测评"按钮和模板，怎么不见了？**

A：这是本次故意关闭的。所有 6 类种子数据现在统一在【系统管理 → 种子数据导入】里**按需导入**。如果当前环境没有该数据，进入种子数据导入页点【一键导入】即可。

**Q2：我之前手动删除过的种子数据，会被这次部署再次写回吗？**

A：**不会**。本次修复后，后端启动时**不再自动插入种子数据**，启动日志会显示 `skipped: by_seed_pack_admin_page` 字样。运营手动删除即真删除。

**Q3：新部署的环境怎么把 36 题问卷加进去？**

A：登录管理后台 → 系统管理 → 种子数据导入 → 找到"中医体质测评（王琦国标 36 题）"行 → 点【一键导入】 → 二次确认 → 完成。

**Q4：体质标签现在能删吗？**

A：能。本次把 9 个体质标签的 `is_locked` 解锁了，运营可以在标签管理页正常增删改。删除前请确认没有商品关联，否则商品的体质标签会失效。

**Q5：健康自查为什么不能在通用问卷模板列表里直接编辑了？**

A：因为之前两个入口的字段双写经常冲突。现在统一收敛到【健康自查问卷模板】专属配置页（功能更全、字段更全）。在通用列表里点击跳转按钮即可前往。

**Q6：旧的 /health-archive 链接被分享出去了怎么办？**

A：旧路由已下线，返回 404。请提醒分享者使用新链接 [/health-profile](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile)。

## 五、注意事项

1. **覆盖操作会丢失定制化修改**：如果某个种子包对应的库内数据已经被运营深度定制（例如改过题目文案、调过分型阈值），点击【覆盖】会**直接丢失这些自定义内容**。如不放心，请先在【查看完整内容】里对比差异，或选择【跳过】。
2. **种子数据导入页仅 super_admin 可见**：菜单默认对超级管理员开放。如果你看不到菜单，请联系超管确认权限。
3. **所有导入 / 卸载会落审计日志**：操作人、时间、IP、动作详情都会写入 `audit_log`，事后可追溯。
4. **首次部署必做**：新环境部署完成后，运营需登录管理后台 → 种子数据导入 → 按需点击各包【一键导入】（旧版本是自动写库，新版本不再自动）。
