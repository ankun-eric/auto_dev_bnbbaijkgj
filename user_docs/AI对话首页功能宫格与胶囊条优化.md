## 一、本次更新亮点

本次升级聚焦于"AI 对话首页"两大模块——**功能宫格**与**胶囊条**，让运营配置更灵活、视觉更现代：

1. **后台拆分双开关**：原"是否启用"开关拆分为「是否推荐」+「是否胶囊」两个独立开关，可分别控制按钮在【功能宫格】和【胶囊条】的显隐。
2. **宫格视觉重构**：AI 对话首页功能宫格升级为「一行 4 列、图标+名称、自动循环背景色」的现代样式，与"菜单模式首页"风格统一。
3. **AI 专属配色池**：新增 5 色循环配色（天蓝青 / 蓝绿青 / 靛蓝 / 紫罗兰 / 青绿），冷色调专为 AI 对话场景定制。
4. **图标三级兜底**：图标显示优先级为「管理员上传图片」 > 「管理员填写 emoji」 > 「按按钮类型自动匹配内置图标」，零空图标。
5. **客户端 5 分钟缓存**：拉取功能按钮接口结果本地缓存 5 分钟，二次进入秒开；后台改动 5 分钟内全端自动生效。

## 三、运维管理员使用指南（管理后台）

### 3.1 打开「功能按钮管理」列表页

1. 浏览器登录 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin)。
2. 左侧菜单依次点击：**AI 咨询配置 → 功能按钮管理**。
3. 列表上你会看到两列**新开关**替代原"是否启用"：
   - **是否推荐**：控制按钮是否出现在 AI 对话首页【功能宫格】
   - **是否胶囊**：控制按钮是否出现在 AI 对话输入框上方的【胶囊条】

### 3.2 快速切换开关（无确认弹窗，即点即生效）

- 在列表对应行直接点击 **「是否推荐」** Switch → 状态立即翻转，右上角弹 Toast「已开启」或「已关闭」。
- 同理点击 **「是否胶囊」** Switch → 状态立即翻转。
- 切换失败时（如网络异常）Switch 会自动**回滚到原状态**，并弹错误提示。
- **不需要二次确认**——直接点即可。

### 3.3 新增功能按钮

1. 列表右上角点击「+ 新增按钮」。
2. 弹窗中填写：
   - 按钮名称、按钮类型、图标 emoji / 图标图片、排序权重等（保持不变）。
   - **「是否推荐」** Switch：默认 OFF，开启后表示"出现在功能宫格"。
   - **「是否胶囊」** Switch：默认 OFF，开启后表示"出现在胶囊条"。
3. 两个 Switch 下方各有灰色小字说明开关含义。
4. 至少需开启一个 Switch，按钮才会在 C 端可见；两个都关闭表示"留底不显示"。
5. 点击「确定」保存。

### 3.4 编辑功能按钮

- 在列表操作列点击「编辑」→ 弹窗回显当前两个 Switch 状态 → 可单独修改任一开关 → 保存。

### 3.5 「是否推荐」与「是否胶囊」组合场景

| 是否推荐 | 是否胶囊 | C 端表现 |
|---|---|---|
| ON | ON | 同时出现在【功能宫格】+【胶囊条】 |
| ON | OFF | 仅【功能宫格】可见 |
| OFF | ON | 仅【胶囊条】可见 |
| OFF | OFF | 全端不可见（按钮数据保留） |

## 四、C 端用户使用指南（H5 / 小程序 / App）

### 4.1 打开 AI 对话首页

- 浏览器或 H5 入口直接访问：[AI 对话首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
- 小程序：底部 Tab「AI 对话」/ 首页 AI 入口卡片
- App：底部 Tab「AI 对话」

### 4.2 功能宫格（新视觉）

- 一行 **4 列固定**，按钮多时**向下自动延展不限行数**。
- 每个按钮：
  - 上方一个**圆角 16px 的彩色背景框**（56×56 px），框内居中显示图标。
  - 下方一行 13px 灰色按钮名称，超长省略号显示。
  - 整体按下时有"轻微缩小"按压反馈。
- 配色按位置自动循环（5 色循环）：
  - 1 - 天蓝青 `#0EA5E9`
  - 2 - 蓝绿青 `#06B6D4`
  - 3 - 靛蓝 `#6366F1`
  - 4 - 紫罗兰 `#8B5CF6`
  - 5 - 青绿 `#14B8A6`
- 图标显示顺序：管理员上传的图片 > 管理员填写的 emoji > 按按钮类型自动匹配的内置图标。

### 4.3 胶囊条（保持现状样式）

- 位于 AI 对话输入框**正上方**横向排列。
- 单胶囊：圆角小图标 + 简短文字。
- 按钮过多时**横向滑动**即可看到后续按钮。

### 4.4 点击按钮的行为（与既有完全一致）

| 按钮类型 | 点击效果 |
|---|---|
| 快捷提问 (quick_ask) | 自动向对话发送预设问题 |
| 外部链接 (external_link) | 跳转外部网页 |
| 健康自查 (health_self_check) | 进入健康自查表单 |
| AI 引导 (ai_chat_trigger) | 注入对话引导语 |
| 报告解读 (report_interpret) | 进入报告解读流程 |
| 拍照识药 (photo_recognize_drug) | 唤起相机识药 |
| 图片/文件上传 (photo_upload / file_upload) | 唤起本地上传 |
| 数字人通话 (digital_human_call) | 启动数字人对话 |

## 七、注意事项

1. **新增按钮默认两个开关都关闭**，必须显式打开至少一个 Switch 才能在 C 端显示。
2. **历史按钮迁移规则**：
   - 升级前「已启用」的按钮 → 升级后两开关均默认 ON。
   - 升级前「未启用」的按钮 → 升级后两开关均默认 OFF。
   - **建议升级后 Review 一次**，关闭不希望出现在宫格的按钮的「是否推荐」，避免宫格过于拥挤。
3. **图标设计建议**：上传的图片图标尽量统一为 28×28 透明背景单色图，避免与其他位置的内置图标视觉割裂。
4. **同一按钮可同时出现在两个位置**：开关互不影响。

## 八、常见问题（FAQ）

**Q1：在后台关闭了某按钮的「是否推荐」，C 端为什么没立即消失？**  
A：客户端有 5 分钟本地缓存，最多 5 分钟内会自动同步；用户也可手动下拉刷新或退出重进。

**Q2：列表里直接点开关有时候转回去了，怎么办？**  
A：表示后端切换失败（多为网络抖动）。前端会自动回滚 UI 并提示错误。再点一次即可。

**Q3：新增按钮时两个 Switch 都是关闭的，按钮怎么不可见？**  
A：这是产品设计——强制管理员**显式选择**曝光位置，避免误推送。打开任一开关后即可。

**Q4：宫格图标颜色我能改吗？**  
A：宫格图标外层背景框颜色为系统 AI 专属 5 色循环，按位置自动取色，无需也无法在后台单独配置。图标本身可在后台「编辑按钮」中替换 emoji 或上传图片。

**Q5：胶囊条最多放几个？**  
A：不限。超出屏宽时**横向滑动**可看完整列表。但建议从用户体验出发不要超过 10 个。
