## 功能简介

本次更新对「用药提醒」页面进行了三项体验优化：

1. **超时打卡文字精简**：将冗长的"已超时，请尽快补打卡"精简为"⚠️ 已超时"，视觉更清晰
2. **全局 Toast 统一为微信风格**：所有页面的操作成功/失败提示统一为居中轻量 Toast（微信风格），1.5 秒自动消失
3. **数据一致性修复**：修复首页待服用数量与今日页面列表不一致的 Bug，确保两处数据始终同步

## 使用说明

### 优化点 1：超时状态显示更精简

**操作路径**：首页 → 点击用药提醒卡片 → 进入今日用药页面

1. 打开 H5 页面后登录账号
2. 进入「用药提醒」页面
3. 查看已超时未打卡的药品条目
4. 现在超时状态显示为 **⚠️ 已超时**（简洁明了），右侧仍有「补打卡」按钮可操作
5. 点击「补打卡」完成补打卡操作

### 优化点 2：轻量 Toast 操作提示

本次优化后，所有操作反馈提示均采用微信风格居中 Toast：

- **位置**：屏幕垂直居中
- **样式**：半透明深色圆角矩形，白色文字
- **时长**：1.5 秒后自动淡出消失
- **不遮挡操作**：Toast 显示期间可正常点击页面其他区域

以下操作可体验新 Toast 效果：
1. 在用药提醒页面点击「打卡」→ 弹出"已打卡"居中 Toast
2. 已打卡条目点击「✓ 完成」按钮撤销 → 确认后弹出"已撤销"居中 Toast
3. 在设备管理页面绑定/解绑设备 → 弹出对应提示 Toast
4. 在病历管理页面保存/删除操作 → 弹出对应提示 Toast
5. 在健康预警页面标记已处理 → 弹出"已标记已处理"Toast

### 优化点 3：数据一致性

- 首页用药卡片显示的"N 条待服用"数字，与点击进入后今日页面的列表条数**完全一致**
- 在今日页面完成打卡后，返回首页数字会立即更新
- 无论是否选择了特定家庭成员，两个页面的数据口径始终保持统一

---

## 注意事项

1. 需要先登录账号才能使用用药提醒功能
2. 补打卡操作没有时间限制，但建议尽快完成以保证服药记录准确
3. 撤销打卡仅限 5 分钟内操作
4. 确认类操作（如撤销打卡）仍保留弹窗确认形式，不受 Toast 优化影响
5. 如首页数字为 0 但确实有用药计划，请检查是否所有时间点已全部打卡完成
