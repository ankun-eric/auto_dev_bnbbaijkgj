# 订单详情页订单地址展示统一 — 用户体验手册

> 版本：v1.0  
> 出具时间：2026-05-05  
> 适用端：用户端 H5 / 微信小程序 / Flutter App（Android / iOS） + 商家管理后台 Web

---

## 1. 本次更新做了什么

我们对**订单详情页的"订单地址"区块**做了一次全平台治理，目标只有一个：

- 用户**看自己的订单**：不再出现"预约门店"和"订单地址"两处显示同一家门店的视觉冗余；
- 商家**看自己的订单**：不再出现"看不到收件人地址 / 看不到门店电话"的信息缺失；
- 三类订单（**到店核销 / 配送快递 / 上门服务**）在用户端、商家端的地址呈现风格**完全统一**。

---

## 2. 三类订单的最终展示效果

### 2.1 到店核销订单（如：到店服务、到店预约项目、到店核销券）

| 端 | 订单详情页【订单地址】区块 | 预约信息·预约门店 |
|----|----------------------------|------------------|
| 用户端（H5 / 小程序 / App） | ❌ **不再显示**（与下方预约门店重复，已彻底隐藏） | ✅ 正常显示门店名称 + 地址 + 一键导航 + 联系商家 |
| 商家管理后台 | ✅ 新增**订单地址（到店核销）** Card：门店名称 + 门店详细地址 + 门店联系电话 | ✅ 正常显示 |

**用户视角**：到店订单详情页的【订单地址】区块**完全消失**，门店信息在【预约信息·预约门店】里查看，**一目了然，不会重复**。

### 2.2 配送 / 快递订单（如：实物商品、营养品、保健器械等）

| 端 | 订单详情页【订单地址】区块 |
|----|----------------------------|
| 用户端（H5 / 小程序 / App） | ✅ 显示**收货地址**：收件人姓名 + 联系电话 + 完整收货地址 + 一键导航 |
| 商家管理后台 | ✅ 显示**订单地址（收货地址）** Card：收件人 + 电话 + 详细收货地址（与现有"快递公司 + 快递单号"并列） |

**商家视角**：以前商家在订单详情里只能看到"快递公司 + 快递单号"，**根本看不到收件人姓名和地址**，发货环节非常不便；现在直接在详情页就能一站式核对完整收货信息。

### 2.3 上门服务订单（如：上门理疗、上门艾灸、上门艾灸理疗等）

| 端 | 订单详情页【订单地址】区块 |
|----|----------------------------|
| 用户端（H5 / 小程序 / App） | ✅ 显示**上门服务地址**：联系人 + 联系电话 + 完整上门地址 + 一键导航 |
| 商家管理后台 | ✅ 同时展示两份地址：①新增统一【订单地址（上门服务）】Card；②保留原有【上门服务地址】Card（保护已有上门作业流不受影响） |

**商家视角**：商家在订单详情里**既能从统一的【订单地址】区块快速识别**（与到店、配送一致风格），**也能从原有【上门服务地址】卡片读详情**，不影响任何已习惯的操作路径。

## 5. 注意事项

1. **客户端缓存**：H5 浏览器请按 `Ctrl + F5` 强制刷新；微信小程序请关闭后重新进入；Flutter App 请下拉订单详情页刷新。
2. **历史订单兼容**：本次修复对修复前生成的订单也生效——后端会按订单类型自动语义化下发地址，不需要补录数据。
3. **0 元订单 / 优惠券抵扣订单**：地址展示规则不变，遵循上面三类订单的风格。
4. **退款 / 已取消订单**：地址区块仍按订单履约类型正常展示，不会因为订单状态变化而消失或新增。
5. **「联系商家」入口不受影响**：到店订单虽然隐藏了【订单地址】区块，但**预约信息中的预约门店行点击 = 联系商家**，底部还有独立的「联系商家」按钮，多入口都可达。
6. **若仍看到旧效果**：可能是某一端缓存未刷新或测试包未升级。请按本节第 1 点彻底刷新；若仍未生效，请将问题反馈给我们。

## 6. 涉及变更摘要（给开发同学的参考信息，普通用户可跳过）

- 后端订单详情接口（`/api/orders/unified/{id}` + `/api/admin/orders/unified` 列表）新增统一字段：
  - `order_address`：按订单类型语义化的结构化地址（`type / contact_name / contact_phone / address_text / store_id`）
  - `order_address_type`：`store / delivery / onsite_service`
  - `store_phone`：商家后台到店订单展示用门店联系电话
- 前端三端（H5 / 小程序 / Flutter）订单详情页：
  - 到店类型：彻底隐藏【订单地址】区块
  - 配送 / 上门：使用统一字段渲染，统一文案与 UI
- 商家后台订单详情 Drawer：
  - 新增统一【订单地址】Card（按类型差异化标题与字段）
  - 保留【上门服务地址】Card 不变（兼容已有上门作业流）
