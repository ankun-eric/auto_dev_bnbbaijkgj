## 功能简介

本次修复解决了通过 `https://newbb.bangbangvip.com` 域名访问项目时出现的 **502 Bad Gateway** 错误。修复后，H5 前端页面、管理后台、后端 API 均可正常访问。

### 问题原因

项目部署在测试服务器 `newbb.test.bangbangvip.com`（101.33.205.59），通过前端代理服务器 `newbb.bangbangvip.com`（43.135.169.167）对外提供访问。

502 错误的根本原因是前端代理服务器的 Nginx 配置中，项目的反向代理规则直接指向了测试服务器的 Docker 容器内部端口（8000、3000、3001），而这些端口并未对外暴露（仅使用 `expose` 而非 `ports`），导致前端代理无法连接到目标服务。

### 修复方案

1. **修正代理路径**：将前端代理服务器的 Nginx 配置从直连内部端口（如 `http://101.33.205.59:3001`）改为通过测试服务器的 Gateway Nginx（HTTPS 443 端口）进行转发（如 `https://newbb.test.bangbangvip.com/autodev/3b7b999d.../`）
2. **修复配置语法错误**：修复了前端代理 Nginx 配置文件中预先存在的花括号缺失问题（影响 `29c7b754` 和 `da347c81` 两个项目的 location 块），使 Nginx 能够正常启动
3. **启用 SSL SNI**：添加 `proxy_ssl_server_name` 和 `proxy_ssl_name` 配置，确保 HTTPS 反向代理时 SNI 匹配正确
