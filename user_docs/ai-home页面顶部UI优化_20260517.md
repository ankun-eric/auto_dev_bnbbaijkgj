## 功能简介

本次仅针对 **AI 对话首页（ai-home）首屏顶部区域**做了三项轻量级视觉与交互优化，让顶部更整洁、信息触达更高效。**不涉及任何功能逻辑改动**：菜单内容、铃铛点击后的页面、消息列表逻辑等均与上一版完全一致。

| 编号 | 优化点 | 用户感受 |
|------|--------|----------|
| ① 悬浮铃铛 | 去掉了原本的圆形蓝色底色，仅保留 🔔 图标本身，并把初始位置从底部上移到「健康贴士轮播卡」的垂直正中 | 屏幕底部不再被一个抢眼的圆形按钮占据；铃铛与顶部 banner 视觉上融合得更自然 |
| ② 汉堡菜单图标（≡） | 由系统字符"☰"改成精致的 SVG 三横线，第 3 条只画一半（左对齐）；整体高度与"小康"字号严格对齐 | 顶栏左侧的菜单图标与"小康"二字形成"等高 + 顶端齐"的视觉组合，更协调、更高级 |
| ③ 汉堡图标右上角红点 | 当有未读系统消息或有待使用订单时，菜单图标右上角自动出现一个 8px 的小红点 | 进入 AI 对话首页第一眼就能知道"有事要看"，不需要再点开抽屉/列表才发现 |

## 使用说明

### 一、查看新版顶部铃铛

1. 打开 [AI 对话首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
2. 视线落到屏幕右侧——你会看到一只 🔔 图标静静地浮在「今日健康贴士」彩色卡片的右侧、卡片**垂直正中**位置
3. 它**没有任何底色 / 圆环 / 阴影框**，就像直接画在了背景上一样
4. **点击铃铛**：弹出"今日待办"抽屉，里面是用药提醒和预约提醒（功能与之前完全一致）
5. **长按 + 上下拖动**：可以把铃铛拖到屏幕上任意垂直位置（仅垂直方向）
6. **复位规则**：拖动后退出页面（比如切到其他 Tab 或返回上级页面），下次再进入 AI 对话首页时，铃铛会**自动回到默认位置**，不再记忆上次的拖动结果

### 二、查看新版汉堡菜单图标

1. 仍然停留在 AI 对话首页，视线落到屏幕**左上角**
2. 你会看到三条等粗、左对齐的横线，**第 3 条只有前两条一半的长度**（像一个"半藏"的菜单图标）
3. 三条横线的**总高度** = 旁边"小康"二字的字号；图标顶端与"小康"顶端在同一水平线
4. **点击图标**：左侧滑出"历史会话"抽屉（与之前完全一致），可以查看/切换历史对话

### 三、查看红点提示

1. 红点出现时机：以下任一条件满足，就会自动亮起小红点
   - 当前用户**有未读的系统消息**（消息中心未读数 > 0）
   - 或当前用户**有处于"待使用"状态的订单**（包括"待预约 / 已预约 / 待核销 / 部分核销"）
2. 红点位置：**菜单图标的右上角外侧**，红点的左下角顶点正好贴着图标的右上角顶点
3. 红点造型：纯红色实心圆，直径 8px，**没有白色描边**
4. **不带数字**：本次只展示"有 / 无"两种状态，不显示具体数字
5. **刷新策略**：**仅在你进入 AI 对话首页那一刻**拉取一次，页面停留期间**不轮询、不实时推送**；下次离开再回来时再次拉取判定

### 四、把红点"消灭"掉的两种方式

- **方式 A**：点击"小康"二字右侧的未读徽标 → 进入 [消息中心](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/messages) → 阅读所有未读系统消息
- **方式 B**：进入 [我的订单](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/unified-orders) → 完成所有"待使用"订单的核销 / 取消

> ⚠️ 注意：只有当 **未读系统消息数 = 0 且 待使用订单数 = 0** **同时**满足时，红点才会消失。两个条件中只要还有一个 > 0，下次进入 AI 对话首页时小红点依然亮起。

| # | 项目 | 规格 |

| ④ | 三条横线间距 | 按等比缩放 ≈ 4px |

| ⑥ | 红点直径 | 8px |
| ⑦ | 红点与图标右上角的位置 | 红点左下角顶点紧贴图标右上角顶点（间距 0px，红点完全溢出在外） |
| ⑧ | 铃铛尺寸 | 36 × 36px（emoji 字号 30px） |
| ⑨ | 铃铛距屏幕右边距 | 16px（保持原值） |
| ⑩ | 铃铛在 banner 区的初始垂直位置 | banner 区域**垂直正中**（top = banner.top + banner.height/2 − 铃铛高度/2） |

## 注意事项

1. **登录态**：未读消息数与待使用订单数都需要用户登录后才能拉取；未登录时小红点不会显示（避免误导）。
2. **接口异常兜底**：如果某个接口请求失败/超时，对应来源的计数视为 0，不影响另一个来源的判定（例如：消息接口失败但订单接口成功且有待使用订单，红点照常亮起）。
3. **不轮询**：进入页面**只拉取一次**红点判定数据。如果你在页面上停留期间后台新增了系统消息或新订单，**当前会话不会刷新红点**；离开本页再回来时才会重新判定。
4. **铃铛拖拽手感**：长按 200ms 后进入拖拽态（标识：略微放大 1.05 倍），仅垂直拖动；松手后**不持久化**位置。
5. **点击热区**：菜单图标视觉变小后，点击热区**仍维持 32 × 32px**，不会影响点按手感。
6. **banner 不存在时**：如果运营后台关闭了健康贴士轮播或本日无 banner 数据，铃铛会回退到屏幕顶部下方约 120px 的位置（避开顶栏即可）。
