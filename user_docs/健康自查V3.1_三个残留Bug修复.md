## 功能简介

本次修复针对 V3 上线后用户实测发现的 3 个未真正生效的问题：

1. **Bug 1：自动下一步开关不生效** — 后台勾选「自动下一步」+「每页 1 题」+「抽屉」后，H5 抽屉单选题选完仍需手动点「下一步」。
2. **Bug 2-A：详情页 6 大区块全为空** — 提交问卷后等待解读完成进入详情页，所有内容区块仅显示「暂无评估」「暂无居家建议」等兜底文案，AI 调用形同失效。
3. **Bug 2-B：详情页咨询人识别错误** — 胶囊已切换为「妈妈 张红」并答题，详情页顶部仍显示「本次回答结合 本人 的健康档案」。

修复后：

- 单选题选完自动跳下一题（0 秒延迟，最后一题不自动提交）。
- 详情页 6 大区块（评估总览 / AI 解读 / 居家建议 / 红线信号 / 异常预警 / 推荐商品）显示真实内容。
- 详情页顶部正确显示「本次回答结合 妈妈（张红）的健康档案」。

## 根因与修复要点（面向技术读者）

| Bug | 根因 | 修复 |
|-----|------|------|
| Bug 1 | 前端 `qnDrawerDisplayForm` 取自按钮默认 `DRAWER_SCROLL`（一屏多题），而 `autoNext` 逻辑只在 `DRAWER_STEPPED`（一题一屏）模式下生效，导致后台配置形同摆设 | H5 在拿到 `render-meta` 后，检测 `auto_next_enabled && questions_per_page=1` 时强制升级为 `DRAWER_STEPPED` |
| Bug 2-A | 后端 `GET /api/questionnaire/answers/{id}` **存在两个同路径 endpoint**！简版（仅返回 ORM 字段）先注册，FastAPI 按"先注册先匹配"始终命中简版，丰富版（含 `ai_full_interpretation / home_care_tips / red_flag_signals / subject_label`）永远走不到 | 将简版改名为 `/answers/{id}/raw`，让 `/answers/{id}` 唯一指向丰富版的 `get_answer_detail` |
| Bug 2-B | H5 抽屉 submit handler 未传 `subject_kind / subject_name / subject_relation / subject_member_id` 4 字段，后端只能按 `consultant_id` 反查兜底，老/虚拟成员场景下会丢字段。同时后端详情页拼装的 `subject_label` 顺序颠倒为「张红（妈妈）」而非需求要求的「妈妈（张红）」 | H5 显式从胶囊上下文取出 4 字段塞进 payload；后端 subject_label 格式改为 `{relation}（{name}）` |

附带强化：AI 异步任务在写库前增加「关键三字段非空」校验，防止「ai_status=done 但内容空」造成的"假 done"。

## 使用说明（终端用户视角）

### 一、健康自查答题（H5）

1. 登录 H5 用户端：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)
2. 进入 AI 对话首页，**左下角胶囊**可切换咨询人为「本人」或「家人（如 妈妈 张红）」
3. 在已切换的状态下，点击宫格或胶囊中的「健康自查」按钮
4. 抽屉弹出，**单选题选完选项后会自动跳到下一题**，无需点击底部按钮
   - 多选题、文本题：仍保留「下一步」按钮（混合模式）
   - 最后一题：不自动提交，需手动点击「提交」按钮
5. 提交后弹出结果卡片，按钮初始为「分析中…」灰色

### 二、查看详情

1. 等待约 5~10 秒，按钮恢复为绿色「查看详情」
2. 点击「查看详情」进入详情页
3. **顶部第二行**：正确显示「本次回答结合 妈妈（张红）的健康档案」（如本人则显示「本人」）
4. **6 大区块**全部显示真实 AI 解读内容（评估总览 / AI 解读 / 居家建议 / 红线信号 / 异常预警 / 推荐商品）

### 三、运营后台配置（管理员）

1. 进入管理后台：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin)
2. 「功能按钮管理 → 健康自查 → 编辑」
3. 「呈现配置」三件套：
   - 容器：选「抽屉」
   - 每页题数：选「1」
   - 「自动下一步」：开启
4. 保存后，无需重启服务即可在 H5 端立即生效

## 注意事项

- **本次未修改数据库历史数据**。升级前已存在的「空」答卷（subject 字段 NULL、AI 字段空）保持原样，详情页对老记录自动走兜底"本人"和"暂无评估"，不报错。
- **本次未涉及小程序 / Flutter / 管理后台前端代码改动**，三端用户无需重新下载安装包。
- **后端 Schema 向下兼容**：4 个 `subject_*` 字段全部为 `Optional`，老版本前端继续走旧 payload 时后端会按"本人"兜底处理，不会 500。
- 详情接口仍兼容 ORM 简版：通过新路径 `GET /api/questionnaire/answers/{id}/raw` 仍可获取 ORM 原对象。

## 风险与回滚

- 若发现新提交问卷出现「6 大区块为空」，先检查后端日志中的 `[hsc-ai-task]`，新增的「关键字段非空校验」会把这种异常落到 `ai_status='failed'` + `ai_failed_reason`，前端可触发「重试解读」。
- 回滚方式：将 `backend/app/api/questionnaire.py` 与 `h5-web/src/app/(ai-chat)/ai-home/page.tsx` 回滚到上一次发布的 commit 即可，无 DB schema 变更，零风险回滚。
