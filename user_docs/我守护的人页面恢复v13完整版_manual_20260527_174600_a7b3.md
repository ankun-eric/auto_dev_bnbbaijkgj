## 使用说明

### 1. 进入「我守护的人」页面

**入口 A：直接访问页面 URL**

- 在 H5 中打开：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard)

**入口 B：从健康档案主页进入**

1. 打开健康档案主页：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile)
2. 点击「我守护的人」卡片
3. 自动跳转至 `/health-profile/i-guard`

### 2. 页面结构总览（自上而下）

```
┌─────────────────────────────────────┐
│ ← 我守护的人      （绿色顶部导航栏）     │
├─────────────────────────────────────┤
│ 您还可邀请 3 位 / 共 5 位  [+ 发起邀请]  │ ← 蓝色配额条
├─────────────────────────────────────┤
│ [守护中 (4)] [待守护 (2)]              │ ← 极简两态 Tab
├─────────────────────────────────────┤
│ 「我」（本人）虚拟行                    │
│  · 编辑档案 / 邀请记录 / AI 外呼额度    │
├─────────────────────────────────────┤
│ 家人卡片 1 ······ 👑 主（金色徽章）     │
│   · 查看档案 / 代付明细 / 解除守护      │
├─────────────────────────────────────┤
│ 家人卡片 2 ······（普通守护人）         │
│   · 查看档案 / 解除守护                │
└─────────────────────────────────────┘
```

### 3. 切换 Tab：守护中 / 待守护

- **守护中（默认）**：显示已经建立守护关系的家人，第一行永远是「本人」虚拟项
- **待守护**：显示待邀请、邀请中、已拒绝、已过期、已解绑等不同生命周期的家人卡片

### 4. 「守护中」Tab 操作

#### 4.1 本人行的三个按钮

| 按钮 | 功能 |
|------|------|
| 编辑档案 | 跳转到 `/health-profile` 编辑自己的档案信息 |
| 邀请记录 | 弹窗查看本人发起的所有邀请历史 |
| 我的 AI 外呼额度 | 跳转到会员中心额度区 `/member-center#quota` |

#### 4.2 主守护人卡片（金色边框 + 「👑 主」徽章）

| 按钮 | 颜色 | 功能 |
|------|------|------|
| 查看档案 | 蓝色 | 跳转到该被守护人档案 |
| 代付明细 | **金色**（独有） | 弹窗查看 AI 呼叫代付开关 + 今日/本月代付次数 + 明细列表 |
| 解除守护 | 红色 | 二次确认后解除共管关系 |

#### 4.3 普通守护人卡片（淡蓝色边框）

| 按钮 | 颜色 | 功能 |
|------|------|------|
| 查看档案 | 蓝色 | 跳转到该被守护人档案 |
| 解除守护 | 红色 | 二次确认后解除共管关系 |

### 5. 「待守护」Tab 操作

不同状态显示不同按钮：

| 卡片状态 | 副标题 | 按钮 |
|----------|--------|------|
| 邀请中（inviting） | 还剩 N 小时 | 查看二维码 + 取消邀请 |
| 已拒绝（rejected） | （已拒绝） | 再次邀请 + 移除* |
| 已过期（expired） | （已过期） | 再次邀请 + 移除* |
| 已解绑（unbound） | （已解绑） | 再次邀请 + 移除* |
| 未邀请（never_invited） | （未邀请） | 发起邀请 + 移除* |

> *「移除」按钮仅在通过 **4 不可删校验** 时才显示（不显示 ≠ 灰掉，是直接不出现）：
> - 当前 active 状态
> - 已绑定智能设备
> - 邀请进行中
> - 有在途服药计划

每张卡片右下角还有「邀请记录」文字链，可弹窗查看该家人的邀请历史。

### 6. 「+ 发起邀请」入口

- 当配额未满时：跳转邀请发起页 `/health-profile?action=invite`
- 当配额已满时：弹出「💝 温馨提示」窗，可选「再想想」或「升级会员」

1. **路径仍是 `/health-profile/i-guard`**：本次修复后访问路径不变，原来的旧版本（fdc3d00a 后的融合版）已被替换为完整 v13 版本。点「我守护的人」卡片直接进入新版，无需中间跳转。
2. **后端接口**：本次使用 `/api/guardian/v13/family/*` 系列共 6 个接口（family/list、invite-history、proxy-pay/toggle、proxy-pay/detail、invite/cancel、remove），后端已挂载并通过烟雾测试（7/7 通过）和自动化测试（9 个 pytest 用例全部通过）。
3. **数据库**：本次未新增任何表或字段，完全复用现有表（`guardian` / `invite` / `proxy_pay_log` 等）。
4. **登录态**：所有接口需要用户登录鉴权，未登录访问会得到 401。
5. **浏览器缓存**：如发现页面没有变化，请清理浏览器缓存或按 `Ctrl + F5` 强制刷新。
