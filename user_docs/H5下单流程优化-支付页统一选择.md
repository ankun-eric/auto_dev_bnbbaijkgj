> 版本：v1.0

> 发布日期：2026-05-02

## 一、为什么要这样优化？

过去你在商品详情页就要先后做出 **「选日期 → 选时段 → 选门店」** 三个决定，
还没决定要不要买，就被一连串选择打断；定位授权也是一进详情页就弹出，体验很重。

本次改版把整条选择链全部 **下沉到支付页**，详情页回归"看看到底要不要买"的纯展示属性，
你点击「立即购买」后，再在一页之内完成所有选择，整体步骤更短、更顺畅。

## 二、功能介绍

### 2.1 商品详情页（更清爽）

- ✅ 只显示：商品图、视频、名称、价格、销量、卖点、规格（SKU）、立即购买按钮
- ❌ 不再出现：预约日期、可预约时段、可用门店

### 2.2 支付页（一站式选择）

支付页按 **"日期 → 时段 → 门店 → 联系人 → 备注"** 的顺序，自上而下完成下单：

| 区块 | 说明 |
|---|---|
| ① 选择日期 | 默认 7 天可预约日期，按商品「提前可预约天数」自动调整 |
| ② 选择时段 | 横向时段按钮，**已满档的时段直接隐藏**，避免误选 |
| ③ 选择门店 | 卡片样式；**第一次点击门店卡片才请求定位**，授权后按距离排序，未授权按字母排序 |
| ④ 联系人手机号 | 自动用账号绑定的手机号回显，可手动修改，自动校验 11 位手机格式 |
| ⑤ 订单备注（选填） | 最长 50 字，方便告知门店特殊需求 |

### 2.3 智能保留

切换门店时，已经选好的日期 / 时段会**自动保留**：

- 在新门店仍可用 → 状态保持（绿色）
- 在新门店不可用 → 时段标红、按钮禁用，并出现"该时段在新门店已不可用，请重选"的提示

### 2.4 满档判定（更严谨）

- 满档容量由门店的 **「单时段最大接单数」** 决定，默认 10
- 占用数 = **已支付** + **下单 15 分钟内未支付** 的订单数
- 商品上的"时段容量"不再生效（已隐藏）

### 2.5 管理后台（门店表单新增字段）

商家管理员可在 **「商家中心 → 门店 → 编辑」** 配置：

- 单时段最大接单数（slot_capacity，默认 10）
- 营业开始 / 结束时间（HH:MM 格式，用于和商品时段做交集，超时的时段自动过滤）

## 五、注意事项

1. **首次定位授权**：H5 上请在「点击门店卡片」时再授权定位；首次进入详情页**不会**再弹定位窗口，更安全。
2. **拒绝定位的兜底**：拒绝后门店会按字母 A-Z 排序，依旧能正常下单，只是看不到距离。可在浏览器站点权限里重新打开。
3. **手机号校验**：联系手机号必须是有效的 11 位（13/14/15/16/17/18/19 开头）数字，否则无法支付。
4. **备注限制**：最多 50 字；超出会自动截断或不允许继续输入。
5. **满档时段被自动隐藏**：商品时段总数减去满档数 = 你看到的时段；若全部被隐藏，请改选其他日期或门店。
6. **门店切换的智能保留**：标红的时段表示在新门店不可用，必须重新选择一个绿色时段后才能支付。
7. **管理后台默认值**：新建门店若未填写营业时间，按 09:00 / 22:00 处理；接单数按 10 处理。
8. **小程序 / APP 同步生效**：本次发布后小程序和 APK 已同步，详情页选择区将隐藏，下单时携带门店和联系人手机号。

## 七、常见问题（FAQ）

**Q1：为什么我看到的时段比商品配置的少？**  
A：时段会经过两次过滤：① 与门店营业时段求交集（不在营业时间的时段会被剔除）；② 已满档的时段直接隐藏。两次过滤后剩下的就是你能选的。

**Q2：为什么"立即支付"按钮一直是灰色？**  
A：检查是否漏选了某一项：日期、时段、门店、联系人手机号 都是必填；备注选填。

**Q3：定位被我误拒绝了，怎么重新授权？**  
A：H5 浏览器右上角的小锁图标 → 站点设置 → 位置 → 改为"允许"；或清除权限后重新点击门店卡片即可。

**Q4：下单 15 分钟没付款会发生什么？**  
A：订单进入"已超时"，系统会自动释放该名额（不再计入占用数），他人可正常下单。
