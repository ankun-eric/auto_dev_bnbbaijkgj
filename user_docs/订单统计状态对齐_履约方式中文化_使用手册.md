## 功能简介

本批次以「一期工程一次性修复」方式上线两条 P1 体验问题，全端联动覆盖：

### Bug ①：管理后台「订单统计」页与「订单明细」状态种类对不齐

升级前订单统计页只展示 7 个订单状态 + 5 个退款状态，缺失订单中台 V2 升级后新增的「待预约 / 已预约 / 部分核销 / 已过期 / 退款中 / 退款审核中 / 退款处理中」等关键状态，导致运营人员看不到完整经营数据。

升级后：
- 完整展示 **12 个订单状态卡片** + **7 个退款状态卡片**，与「订单明细」页**完全对齐**
- 顶部新增**统一时间筛选条**（今日 / 本周 / 本月 / 自定义日期范围，默认"今日"）
- 每张卡片同时显示**订单数量** + **金额合计**，一眼掌握各状态订单分布
- 销售趋势、品类销售排行、退款原因等图表全部随时间筛选条联动刷新

### Bug ②：用户端订单详情「履约方式」直接显示英文枚举

升级前 H5 / 小程序 / Flutter APP 三端的订单详情页，商品下方"履约方式"标签直接显示后端原始枚举（如 `on_site`、`delivery`），用户不知道是什么意思，专业感受损。

升级后：
- 全端统一通过**公共字典**翻译为中文（"上门服务 / 到店服务 / 快递配送 / 到店核销 / 线上服务"）
- 字典文件**单一来源**：每端各一份字典文件，未来再加新枚举只需改 5 个文件，避免散落 switch
- 字典外的未知枚举降级显示为"其他服务"，并在控制台打印警告

## 使用说明

### 一、管理后台「订单统计」页操作步骤

1. 浏览器打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 使用管理员账号登录
3. 左侧菜单进入「**产品系统 → 订单统计**」（也可直接打开 [订单统计页直链](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/statistics)）
4. 顶部时间筛选条：
   - 默认载入"**今日**"数据
   - 点击「本周 / 本月」快捷按钮一键切换
   - 也可以选择**自定义日期范围**（任意起止日期）
5. 中部查看 **12 张订单状态卡片**，按业务流程顺序排列：
   1. 待付款 → 2. 待发货 → 3. 待收货 → 4. 待预约 → 5. 已预约 → 6. 待核销 → 7. 部分核销 → 8. 已完成 → 9. 已过期 → 10. 退款中 → 11. 已退款 → 12. 已取消
6. 卡片右下方显示 **7 张退款状态卡片**：无退款 / 退款申请中 / 退款审核中 / 退款已批准 / 退款已拒绝 / 退款处理中 / 退款成功
7. 顶部"总览"区显示选定时间段内的：
   - **总订单数**（不含已取消）
   - **总实付金额**
   - **退款单数**（仅成功退款）
   - **退款金额**
8. 切换时间段后，下方销售趋势 / 品类销售排行 / 退款原因分布等图表会**自动同步刷新**

## 注意事项

| 类别 | 说明 |
|------|------|
| **数据范围** | 默认载入"**今日**"数据，如果今天还没有订单，相关卡片显示 0 是正常现象，请切到"本月"或自定义"本周"以验证完整功能 |
| **状态对齐** | "订单统计"页与"订单明细"页的状态枚举**100% 对齐**。若发现差异，可能是浏览器缓存导致，请按 Ctrl+F5 强刷 |
| **金额口径** | 卡片金额 = `paid_amount`（实付金额），未支付订单金额为 0；总实付金额排除已取消订单；退款金额仅统计 `refund_status=refund_success` |
| **兼容字段** | `pending_review`（待评价）已迁移为 `completed`，不在卡片矩阵中独立展示，但筛选下拉框中保留以便兼容老数据 |
| **APK 安装提示** | APK 未代码签名，部分浏览器或安全软件可能弹出提示，请点击"保留 / 仍要继续"完成下载安装 |
| **iOS 用户** | 本次仅修改了 Dart 层 UI 文案与公共字典，**iOS 端可在下次大版本一并打包**，不影响功能可用性 |
| **微信小程序** | 项目仓库源码部署，开发者用 [微信开发者工具] 重新导入即可，不需要单独的 zip 包 |
| **新枚举兜底** | 未来后端如新增履约方式但前端字典没及时更新，标签会显示"其他服务"并打印控制台告警，**不会再回退到英文** |
