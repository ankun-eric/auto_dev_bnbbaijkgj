## 功能简介

本次集中修复了「健康档案页 + 用药模块」的 7 个 Bug，让数据口径一致、状态正确、文案准确：

1. **「今日用药」口径统一**：Hero 卡片与提醒列表的「今日总次数」彻底对齐，不再出现「Hero 显示 7，列表显示 9」的差异。
2. **「本月依从」改为「今日完成率」**：用药提醒页 Banner 上的统计指标从「本月」改为「今日」，更贴近实时反馈。
3. **「已超时」状态新增**：超过提醒时间还未打卡的项，时间线上明确标注「已超时，请尽快补打卡」（红色徽标 + 补打卡按钮），不再误显示为「未到时间」。
4. **新增用药计划默认时长**：服用周期默认值由「30 天」调整为「5 天」；备注占位符文案优化为「可填补充说明」。
5. **健康档案「用药计划」卡片正确显示数量**：之前由于字段错误永远显示 0，本次修复后能正确返回该咨询人当前服药中的计划数。
6. **健康档案「用药计划」卡片标题颜色**：由原本的深蓝 `#3B82F6` 调整为天蓝 `#0EA5E9`，与小程序保持一致。
7. **AI 首页铃铛红点 + 今日用药胶囊**：之前因数据源指向了旧表，永远为 0。现已切换到新表，可正确显示未打卡次数。
8. **删除用药计划后实时刷新**（附加）：删除用药计划后，铃铛、Hero、今日用药胶囊会立刻广播刷新，不再保留过期数字。

## 使用说明

### 一、查看 AI 首页「今日用药」胶囊（验证 Bug 7）

1. 用浏览器打开 [AI 首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
2. 登录后，顶部铃铛 🔔 旁如有用药未打卡，会显示红点 + 数字
3. 页面中部「今日待办」胶囊条会显示「今日用药 N 次」实际数字
4. **以前为 0，现在能正确显示**

### 二、健康档案页查看「用药计划」卡片（验证 Bug 5/6）

1. 进入 [健康档案页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile)
2. 向下滚动，找到 4 大卡片区：健康预警 / **用药计划** / 家庭成员 / 我的设备
3. 「用药计划」卡片标题颜色为**天蓝 `#0EA5E9`**（与小程序一致），数字应该等于您当前服药中的计划条数（之前显示 0）

### 三、用药提醒页（验证 Bug 1 / 2 / 3）

1. 进入 [用药提醒页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home/medication-reminder)
2. **顶部 Banner**：
   - 三块统计：「已服用 / 待服用 / **今日完成率**」（最后一项原为「本月依从」）
   - 大字「还有 X 次用药」与 Hero 卡数字应当**完全一致**
3. **时间线**：
   - 已打卡 → 绿色「已服用」
   - 下一次 → 橙色「即将服用」+ 立即服用按钮
   - 已过点未打卡 → **红色「已超时，请尽快补打卡」**+ **补打卡按钮**（点击即可补打）
   - 未到时间 → 灰色「未到时间」
4. **删除用药计划**：在用药计划详情页删除后，回到首页/健康档案，铃铛与今日用药胶囊会立即刷新，不再显示已删除项

### 四、新增用药计划（验证 Bug 4）

#### H5 端
1. 在用药提醒页点击右下角 + 按钮 → 进入新增页
2. 「服用周期」默认显示「**5 天**」（之前默认为 30 天）
3. 「备注」行的灰字占位符显示「**可填补充说明**」

#### 小程序端
1. 进入小程序 → 健康计划 → 用药计划 → 新增
2. 「服用周期」默认 5 天
3. 「备注」灰字与抽屉 placeholder 都显示「可填补充说明」

## 注意事项

1. **数据回滚**：本次 Bug 修复对**已有数据无破坏性**，旧的「本月依从率」字段仍然保留在 API 返回里，仅前端展示切换到「今日完成率」。
2. **超时打卡**：「已超时」是基于实时时间计算的（前端结合服务端返回的 status），不依赖后端定时任务。补打卡按钮调用与正常打卡相同的接口（`POST /api/medication-check-in`）。
3. **小程序 AppID**：体验时使用「测试号」即可，无需正式 AppID。
4. **缓存清理**：H5 在浏览器中体验时如发现旧文案没刷新，请按 `Ctrl+F5` 强制刷新清除缓存。
5. **删除级联策略**：删除用药计划后，**保留历史打卡记录**用于回顾，但当日及未来的统计不再纳入该计划。
