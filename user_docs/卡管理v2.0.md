## 四、新增/修改的 API 端点（v2.0 第 2~5 期）

> 全部以 `https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27` 为前缀。

### 第 2 期：购卡下单
- `POST /api/cards/purchase` — 直购卡（兼容入口）
- `POST /api/orders/unified` — 新增 `product_type=card` 分支

### 第 3 期：动态核销码 + 卡退款 + 核销记录
- `POST /api/cards/me/{user_card_id}/redemption-code` — 生成 60s 动态码
- `GET /api/cards/me/{user_card_id}/redemption-code/current` — 拉取当前 active 码
- `POST /api/staff/cards/redeem` — 门店扫码核销（行锁 + 频次校验 + 项目范围校验）
- `GET /api/cards/me/{user_card_id}/usage-logs` — 用户端核销流水
- `GET /api/admin/cards/{card_def_id}/usage-logs` — 商家端核销流水
- `POST /api/orders/unified/{id}/refund-card` — 卡退款（要求 `remaining_times == total_times`）

### 第 4 期：拆 2 单 + 续卡 + 省钱引导
- `POST /api/orders/unified/checkout` — 混买结算，按 `product_type` 拆 2 单，返回 `[order_a_id, order_b_id, split_group_id]`
- `POST /api/cards/me/{user_card_id}/renew` — 续卡（STACK/RESET/DISABLED）
- `GET /api/products/{id}/savings-tip` — 商品省钱提示
- `GET /api/cards/me/renewable` — 临期 7 天 + 过期 30 天可续卡列表

### 第 5 期：销售看板 + 卡分享海报
- `GET /api/admin/cards/dashboard/summary?start=&end=` — 销量 / 销售额 / 核销次数
- `GET /api/admin/cards/dashboard/trend?start=&end=&granularity=day|week` — 趋势曲线
- `GET /api/cards/{card_id}/share-poster?inviter_user_id=` — 800×1200 海报（缓存 1 小时）

## 五、用户使用流程

### 1) 用户购卡
1. 在 H5 / 小程序 / Flutter App 卡列表选购，或在商品详情页通过"💡 开通卡可省 ¥X"浮条进入卡详情
2. `POST /api/orders/unified { product_type: "card", card_definition_id }` 下单
3. 完成支付（微信 / 支付宝），后端回调时自动激活：写入 `user_cards`（valid_from=now、valid_to=now+valid_days、remaining_times=total_times、status=active），`card_definitions.sales_count +1`
4. 卡详情页若带 `?from_product_id=xxx`，购卡成功后回流商品页

### 2) 用户核销
1. 进入"卡包"找到对应卡 → 点击"使用核销码"
2. 系统生成 200×200 二维码 + 6 位数字（60s 倒计时圆环），自动失效旧 active 码
3. 商家用核销小程序扫码 / 输入 6 位数字 → 即时核销（取消用户二次确认）
4. 后端 `_redeem_user_card` 行锁原子操作：校验状态、项目范围、频次、扣次、写 `card_usage_logs`
5. 卡剩余次数为 0 → status 自动改为 `used_up`

### 3) 用户退款
- 仅当 `remaining_times == total_times`（一次未核销）且 status=active 且未过期时允许整卡退款
- 退款成功：`user_cards.status=refunded`、active 码失效、`card_definitions.sales_count -1`

### 4) 用户续卡
- 卡包列表中临期（剩余 ≤ 7 天）/ 过期（30 天内）的卡显示"续卡"按钮
- 续卡按 `card_definitions.renew_strategy` 三策略生效：
  - **STACK**：续卡次数累加到现有卡，valid_to 延长 valid_days
  - **RESET**：失效旧卡 → 创建新卡（renewed_from_id 关联）
  - **DISABLED**：禁止续卡

### 5) 商家销售看板
- 入口：Admin → 商品体系 → 卡销售看板
- 维度：今日 / 本周 / 本月 / 自定义；3 指标：销量 / 销售额 / 核销次数
- 趋势线：day / week 粒度

### 6) 卡分享海报
- 任意用户访问 `GET /api/cards/{card_id}/share-poster?inviter_user_id=<uid>` → 返回 800×1200 PNG 海报（含卡面、邀请人头像、二维码），1 小时缓存

### 7) 到期触达（自动）
- 后端 cron `backend/app/tasks/card_expiry_notify.py` 每日凌晨 3 点扫描到期前 7/3/1 天的卡 → 写 `messages` 表 + 给小程序用户发送微信模板消息

## 八、关键设计 & 教训

- **核销码双形式 + 60s 自动失效**：QR + 6 位数字，自动失效旧 active 码，避免一码多人用
- **门店扫码即核销（取消二次确认）**：减少误操作但用 `SELECT ... FOR UPDATE` 行锁防并发
- **退款规则严格化**：仅"一次未核销"才能整卡退；已核销任一次即不可退（避免商家利益受损）
- **拆 2 单**：混买场景把卡子单与商品子单拆开，独立支付独立核销，但用 `split_group_id` 串联，便于后台追溯
- **续卡三策略写在卡定义上**：STACK/RESET/DISABLED 由商家配置，运行时按 `card_definitions.renew_strategy` 派发，避免硬编码
- **海报缓存 1 小时**：Pillow 合成相对昂贵，按 `(card_id, inviter_user_id)` 哈希缓存到 backend 容器临时目录
- **测试用 SQLite (aiosqlite) 内存模式**：`backend/app/core/database.py` 的 `_build_engine` 按 `DATABASE_URL` 前缀分流，MySQL 走完整连接池配置，SQLite 跳过 pool_size/max_overflow/init_command —— 这是本期 pytest 能跑起来的关键修复
