## 功能简介

本次更新对数据库中历史上传的所有图片、视频、文件 URL 进行了统一的 Bucket 名称替换，将旧存储桶标识 `xiaokang-1323135906` 批量更新为新存储桶标识 `xiaokang-prod-1420478721`。

### 更新内容

- 全库扫描所有文本类型字段（VARCHAR、CHAR、TEXT、JSON），识别包含旧 Bucket 名称的 URL 并自动替换
- 系统配置表 `cos_configs` 中的 bucket 配置字段已做保护，不会被误修改
- 新上传的文件自动使用新 Bucket，不受影响

### 迁移数据概况

本次迁移共更新 **517 条历史记录**，涉及 23 个表的字段，主要分布在以下模块：

| 模块 | 主要涉及表 | 说明 |
|------|-----------|------|
| 文件存储 | cos_files | 历史上传的文件 URL |
| AI 聊天 | chat_messages | 聊天消息中的图片/文件引用 |
| 订单系统 | order_items | 订单商品图片 |
| 用药识别 | drug_identify_details | 药品识别图片 |
| 体检报告 | checkup_report_details | 体检报告图片 |
| 商品管理 | products | 商品图片和描述 |
| OCR 识别 | ocr_call_records | OCR 识别原始图片 |

## 使用说明

本次更新为**后端数据库维护操作**，对用户端功能无任何影响。您可正常登录使用所有功能：

1. **H5 用户端**：打开上方「H5 用户端」链接，登录后即可正常使用 AI 问诊、健康档案、数字安全绳、商城等全部功能
2. **管理后台**：打开上方「管理后台」链接，使用管理员账号登录后可管理商品、订单、用户等

所有功能页面的图片、文件 URL 均已完成替换，历史数据中的图片/文件可正常加载和查看。

## 注意事项

1. 本次更新仅涉及数据库中已存储的 URL 文本替换，不涉及 COS（对象存储）中的文件实际迁移
2. 系统配置 `cos_configs` 表中的 Bucket 配置值保持不变，已做自动保护
3. 新上传的文件自动使用新 Bucket 生成 URL
4. 如发现任何历史图片/文件无法加载的情况，请联系开发团队排查
