## 五、注意事项

1. **缓存清理**：浏览器若访问过旧版本，请先按 `Ctrl + Shift + R` 强制刷新或清除该域名缓存，确保加载到最新前端资源。
2. **HTTPS 强制**：所有访问请直接使用 `https://` 协议，HTTP 请求会被网关 301 重定向。
3. **路径前缀**：所有 URL 均带有 `/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27` 前缀，**不要去掉**该前缀直接访问根域名（会进入网关默认/其他项目）。
4. **API 调用**：前端代码已统一使用 `…/api/` 前缀；如需手工调用接口，请保留路径前缀。
5. **服务自愈**：所有容器均设置 `restart: unless-stopped`，遇到偶发崩溃会自动恢复；MySQL 容器启用 healthcheck，后端依赖其健康才会启动。
6. **资源回收**：本次部署使用 `docker compose down` 清除了旧容器，挂载卷 `mysql_data`、`uploads_data` 已保留，历史数据不丢失。
