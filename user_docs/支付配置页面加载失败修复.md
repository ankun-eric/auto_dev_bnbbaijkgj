## 功能简介

本次修复彻底解决了管理后台「支付配置」页面**进入即「加载失败」**的严重 Bug。

修复后，超级管理员/管理员账号登录管理后台后，进入「支付配置」页面应能：

- ✅ **正常加载**，1~2 秒内展示 4 个支付通道（微信小程序支付 / 微信 APP 支付 / 支付宝 H5 支付 / 支付宝 APP 支付）
- ✅ 每个 TAB 内能看到对应通道的**配置卡片**（含状态标签、操作按钮）
- ✅ 任何加载错误都会**展示具体根因文案**（如「加密密钥未配置…」），而不再只是笼统的「加载失败」
- ✅ 支持**重试按钮**，无需刷新整页
- ✅ 后端服务**重启后自愈**，缺失通道会被自动补齐

## Bug 根因与修复说明（一句话）

**根因**：服务器首次部署时，4 条支付通道种子记录虽然已写入数据库，但 `created_at` / `updated_at` 两列为 `NULL`（受 MySQL `+00:00` 时区与 sql_mode 影响，`DEFAULT CURRENT_TIMESTAMP` 没生效）；后端 `PaymentChannelResponse` Pydantic v2 严格校验这两个字段必须是 `datetime` 类型，遇到 `None` 直接抛 `ValidationError` → 接口返回 500 → 前端瞬间「加载失败」。

**修复**：本次一次性覆盖了原方案中的 7 个修复点（FIX-1 ~ FIX-7），从数据、加密密钥、路由、序列化兜底、前端文案到部署样例配置全部加固。

### 步骤 3：确认 4 个 TAB 内容能正常展示

页面应在 1~2 秒内加载完毕，看到：

- 顶部 4 个 TAB：**微信小程序支付** / **微信 APP 支付** / **支付宝 H5 支付** / **支付宝 APP 支付**
- 每个 TAB 内有一张配置卡片，含：
  - 通道名称 + 「未启用」标签 + 「配置未完整」标签（**首次部署正常状态**）
  - `notify_url` / `return_url` / 各通道字段（首次未填写时显示「（未填写）」）
  - 右上角「测试连接」「编辑」「启用 / 停用」按钮

### 步骤 4：编辑通道配置（可选）

点击任一通道的「**编辑**」按钮 → 弹出 720px 宽抽屉 → 填入对应通道的密钥参数 → 点击「**仅保存**」或「**保存并测试连接**」。

| 通道 | 需要填入的关键字段 |
|---|---|
| 微信小程序支付 | 小程序 AppID / 商户号 MchID / API V3 密钥 / 商户证书序列号 / 商户私钥（PEM） |
| 微信 APP 支付 | 开放平台 AppID / 商户号 / API V3 密钥 / 商户证书序列号 / 商户私钥 |
| 支付宝 H5 支付 | 应用 AppID / 接入模式（公钥/公钥证书）/ 应用私钥 / 支付宝公钥（公钥模式）或公钥证书 + 根证书（证书模式） |

> 💡 表单中支持**上传 .pem 文件**直接回填，无需手动复制粘贴。

### 步骤 5：启用通道

填写完整字段且测试连接通过后，点击右上角「**启用 / 停用**」开关 → 通道立即上线，C 端下单时会出现该通道。

未配置完整时尝试启用，会弹出明确提示：「**配置不完整，无法启用，请先完成必填字段**」。

## 注意事项

### 1. 加密密钥环境变量

本次修复后，`docker-compose.yml` 中已注入默认 `PAYMENT_CONFIG_ENCRYPTION_KEY`（base64 32 字节）。

**生产环境强烈建议**：

```bash
# 生成新密钥
openssl rand -base64 32

# 写入服务器 .env
echo "PAYMENT_CONFIG_ENCRYPTION_KEY=<上面生成的值>" >> .env
```

### 2. 密钥变更注意

**密钥一旦变更，已加密的旧密文将无法解密**。如需更换密钥，必须先在管理后台中清空所有敏感字段、然后再换密钥并重新填入。

### 3. 后端启动自检

后端启动时会在日志中打印：

```
[支付配置] /api/admin/payment-channels 路由已正确挂载
[支付配置] 检测到 PAYMENT_CONFIG_ENCRYPTION_KEY 环境变量
```

如出现：

```
[支付配置] 环境变量 PAYMENT_CONFIG_ENCRYPTION_KEY 未配置，已使用项目内置 fallback 32 字节密钥
```

说明环境变量未生效，请检查 `docker-compose.yml` 与 `.env` 注入是否正确。

### 4. 错误信息现在更友好

如果未来再次出现加载失败，前端会**直接展示后端 detail**，例如：

- 「加载失败：支付通道列表加载失败：ValidationError: ...」
- 「加载失败：HTTP 500（请联系管理员排查后端日志）」
- 「加载失败：网络异常或服务器无响应，请检查网络后重试」

页面顶部还会出现一个**红色 Alert + 重试按钮**，点击即可重新加载，无需刷新整页。

### 5. 数据自愈

如果数据库中误删了某条通道记录，**下次访问列表接口时会自动补齐**该通道（is_enabled=False / is_complete=False / 字段全空），不影响其他通道的配置。

### 6. C 端用户实际下单

修复完成后，您可以**正常进入支付配置页面**。但要让微信支付与支付宝支付**真正在 C 端下单时跑通**，仍需准备平台方颁发的密钥（详见上面表格）。这些不是 Bug 修复能解决的部分。
