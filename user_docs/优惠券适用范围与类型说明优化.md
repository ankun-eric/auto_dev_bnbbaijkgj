## 1. 功能简介

本次改造仅聚焦在管理后台「优惠券编辑/新增弹窗」的两块用户体验，**不改变优惠券的领取、核销、计算逻辑**。

### 1.1 改了哪两件事

1. **类型说明从「靠记忆」升级为「点开即看」**  
   类型字段右侧新增 ❓ 信息图标，点击即可弹出 4 种类型（满减券 / 折扣券 / 代金券 / 免费试用）的完整说明卡片，包括：核心规则、关键字段、典型场景、配置示例。新人运营无需再翻文档。

2. **适用范围从「手填 ID」升级为「点选 + 排除」**  
   - 「指定分类」改为可视化分类树，多选 + 搜索 + 父类自动包含子类。  
   - 「指定商品」改为弹窗式商品选择器，3 个 Tab（全部 / 快递配送 / 到店服务），支持搜索、分类筛选、跨页保留勾选，已选商品超过 100 时自动引导切换到「指定分类」模式。  
   - 当 scope = 全部 / 指定分类 时，新增「**排除商品**」补丁，最多 50 个，命中商品在订单结算时不享受券折扣。  
   - 表单底部出现实时统计预览：「📊 本券将对 X 个商品 / Y 个分类（共 Z 个商品）生效，已排除 W 个商品。」

### 1.2 关键收益

| 指标 | 改造前 | 改造后 |
|------|--------|--------|
| 配券错率 | 高（手填 ID 极易写错） | 低（点选 + 后端实时校验） |
| 配券效率 | 1 张券约 5~10 分钟（要去商品页查 ID） | 1 张券 30 秒~2 分钟 |
| 培训成本 | 高（新人需翻文档） | 低（页面内嵌说明） |
| 运营场景能力 | 缺「分类全场可用 + 剔除几款特价」 | ✅ 通过「排除商品」原生支持 |

## 2. 操作步骤

### 2.1 入口

1. 访问 [优惠券管理页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/coupons)（侧边菜单：商品体系 → 优惠券）。
2. 点击右上角 **「新增优惠券」** 或在列表行点击 **「编辑」**，弹出编辑弹窗。

### 2.2 看「优惠券类型」是啥意思（F1）

1. 在弹窗内找到 **「优惠券类型」** 字段。
2. 点击字段标签右侧的 **❓ 圆形图标**（鼠标悬停会变成主题绿色）。
3. 弹出「优惠券类型说明」窗口，按 4 张卡片并排列出（窄屏自动改为单列）：
   - 💰 **满减券（full_reduction）** — 满 X 减 Y，强调凑单门槛
   - 🏷️ **折扣券（discount）** — 满 X 享 N 折，建议配合门槛使用
   - 🎫 **代金券（voucher）** — 现金抵扣，可无门槛
   - 🎁 **免费试用（free_trial）** — 整单 0 元，建议必须搭配「指定商品」
4. 每张卡片包含：核心规则 / 关键字段 / 典型场景 / 配置示例 / 风控建议。
5. 点击底部 **「知道了」** 关闭弹窗。

### 2.3 选「适用范围 = 全部商品」（F2）

1. 「适用范围」单选 **「全部商品」**，下方说明：「适合全场满减、节日券」。
2. 不再显示分类 / 商品选择器，仅保留：
   - 一行说明：「该券对全店所有在售商品生效（实物 + 到店服务，虚拟商品本期不纳入）」
   - 「**排除商品**」补丁（可选，最多 50 个）— 见 2.6
3. 实时统计条显示：「📊 本券将对全店 N 个在售商品（实物 + 到店服务）生效，已排除 W 个商品。」

### 2.4 选「适用范围 = 指定分类」（F3）

1. 单选 **「指定分类」**，下方说明：「适合品类清仓、分类专享」。
2. 弹出树形分类选择器，支持：
   - **关键词搜索**：直接输入分类名称定位。
   - **多选 + 父子关系**：勾选父分类自动包含全部子分类（已显示为父级 Tag）。
   - **已选标签区**：每个 Tag 带 ×，可单独移除。
3. **保存时**：必须至少选 1 个分类，否则提示「请至少选择 1 个分类」。
4. **重要业务规则**：保存后系统只存「最上层勾选的分类 ID」，**之后新加入这些分类的商品也自动适用本券**（符合「分类」概念直觉）。

### 2.5 选「适用范围 = 指定商品」（F4 / F5）

1. 单选 **「指定商品」**，下方说明：「适合爆款促销、新品试用」。
2. 点击 **「+ 选择商品」** 按钮打开商品弹窗。

#### 弹窗能力

- **3 个 Tab**：全部 / 快递配送 / 到店服务（虚拟商品本期不展示）。
- **顶部筛选条**：商品名 / 编码模糊搜 + 分类下拉筛选 + 「搜索」按钮 + 「勾选本页」复选框。
- **数据列**：商品图（缩略 32×32）/ 商品名 / 分类 / 价格 / 库存（到店服务显示「-」）/ 履约方式 Tag（绿色到店、蓝色快递）。
- **跨页保留**：切页码、切 Tab、变更搜索条件，**已勾选商品状态完整保留**（前端用 Set 暂存）。
- **顶部计数**：「已选 X / 100」实时刷新；快达上限时变红。
- **已选展开**：「查看 / 移除已选 X 个商品 ▼」点开后可单条移除。
- **达上限保护**：第 101 个 checkbox **置灰禁用**，hover 提示「已达上限，单张优惠券最多关联 100 个商品」；顶部出现红色横幅 + 蓝色快捷按钮「**切换为指定分类模式**」（点击二次确认 → 清空已选 → 关闭弹窗 → 自动切到分类模式 → 焦点放到分类树）。

#### 在主表单的回显

- 已选商品以「**📷 商品图 + 名称 + ×**」的小卡片形式展示在按钮下方，点击 × 可单独移除。
- 编辑历史优惠券时：
  - 商品已被**删除** → 红色 Tag「❌ 商品已不存在(ID:123)」，**不自动剔除**，需要运营手动 ×。
  - 商品已被**下架** → 橙色 Tag「⚠️ 商品已下架(商品名)」，同样保留 ID。

### 2.6 设置「排除商品」（F6，新功能）

> ⚠️ 仅当 scope = 全部 / 指定分类 时显示该区块。scope = 指定商品 时不显示（业务上无此需求）。

1. 找到 **「排除商品（可选，最多 50 个）」** 区块，旁边有 ❓ Tooltip：「设置后，本券对所选范围生效，但被排除的商品不参与」。
2. 点击 **「+ 选择排除商品」**，弹出与「指定商品」相同的弹窗（标题改为「选择要排除的商品」，上限改为 50）。
3. 选完后已排除商品同样以小卡片在主表单展示。
4. **业务校验**：
   - 当 scope = 指定分类 时，排除商品**必须落在已选分类范围内**（含子分类），否则保存报错「排除商品 [XXX] 不在已选分类范围内，无法排除」。
   - 排除商品总数 ≤ 50，否则报错「排除商品最多 50 个」。
5. **业务效果**：用户下单时，命中 `exclude_ids` 中的商品**不计入门槛、不参与折扣**，购物车内其他商品按原逻辑参与计算。

### 2.7 保存时的校验（F9）

| 校验点 | 触发条件 | 提示文案 |
|---|---|---|
| 分类必选 | scope=category 且 scope_ids 空 | 「请至少选择 1 个分类」 |
| 商品必选 | scope=product 且 scope_ids 空 | 「请至少选择 1 个商品」 |
| 商品上限 | scope_ids 长度 > 100 | 「适用商品最多 100 个，建议改用指定分类模式」 |
| 排除上限 | exclude_ids 长度 > 50 | 「排除商品最多 50 个」 |
| 排除合法性 | scope=category 时排除商品不在分类范围内 | 「排除商品 [XXX] 不在已选分类范围内」 |
| 商品/分类存在性 | 提交时校验真实存在 | 「商品 [ID:XXX] 不存在或已删除，请重新选择」 |
| 虚拟商品 | scope=product 包含 virtual 商品 | 「虚拟商品本期不支持作为优惠券适用范围」 |
| 免费试用 + 全店 | type=free_trial + scope=all | 弹出**黄色二次确认**：「免费试用券设为全店生效风险较高，是否继续？」 |

## 3. 注意事项

### 3.1 业务规则速记

| 规则 | 说明 |
|---|---|
| BR-1 | scope=product 模式下不支持「排除商品」（前端隐藏 + 后端忽略） |
| BR-2 | scope=category 时按分类 ID 实时判断商品归属，**新加入分类的商品自动适用** |
| BR-3 | 商品上限 100 / 排除上限 50，**写入 `system_configs` 表，可后台动态调整无需发版** |
| BR-4 | 已删除/已下架的历史 scope_id 在编辑页保留显示（红/橙 Tag），运营可手动剔除 |
| BR-5 | 商品弹窗仅展示 `fulfillment_type IN ('delivery','in_store') AND status='active'` 的商品，**虚拟商品本期不纳入** |
| BR-6 | 优惠券类型说明文案以本次内嵌弹窗为准，不再额外维护文档 |
| BR-7 | 排除商品列表**不会**显示在用户端「领券中心」和「我的优惠券」详情页，仅作为内部计算 |
| BR-8 | 单张优惠券同时设置「指定分类 + 排除商品」时，业务含义为「分类内除排除项以外的商品都生效」 |

### 3.2 权限要求

| 角色 | 能做什么 |
|---|---|
| 普通管理员 / 运营 | 可创建 / 编辑优惠券，使用本次新交互 |
| 运营主管 | 同上 |
| 超级管理员（is_superuser） | 同上 + 已下架/已删除商品的「忽略校验强制保存」权限 + 优惠券下架/上架权限 |
| 商家管理员 | 不涉及（本期优惠券模块仅平台级，商家无入口） |
| 客服 | 仅查看，不可编辑 |

### 3.3 用户端是否受影响

**不受影响。** 用户端（H5、小程序、Flutter App）的：

- 我的优惠券列表
- 领券中心
- 优惠券核销 / 折扣计算

均**无 UI 改动**，本期不涉及移动端发版。但**底层下单价格计算已增加 `exclude_ids` 过滤逻辑**——命中排除商品的订单行不享受券折扣（这是后台业务规则，对前端透明）。

### 3.4 历史数据兼容

- 历史 `scope_ids` 字段为字符串 `"1,2,3"` 时，编辑页打开自动迁移为数组 `[1,2,3]` 并写回。
- 历史 `exclude_ids = NULL` 行为完全不变，旧券计算逻辑不受影响。

### 3.5 灰度与回滚

- **灰度**：本次为管理后台体验改造，**不需要灰度**，全量发布。
- **回滚**：
  - 前端：admin-web 容器回滚到上一个 docker tag 即可，老 UI 立即恢复（手填 ID）。
  - 后端：`exclude_ids` 字段保留即可（NULL 时与旧行为一致），无需 DDL 回滚。

## 4. 常见问题（FAQ）

### Q1：以前手填 `1,2,3` 的旧券，编辑时还能看到原来填的内容吗？

A：可以。打开编辑页时，前端会调用 `/api/admin/coupons/product-picker?selected_ids=1,2,3` 拉取这些商品的详情进行回显（图 + 名）。已删除商品显示红色 Tag，已下架显示橙色 Tag，不会自动剔除。

### Q2：商品弹窗为什么没看到「在线问诊」、「健康咨询」这些虚拟商品？

A：本期商品弹窗强制只展示 `fulfillment_type IN ('delivery', 'in_store')` 的商品，虚拟商品（virtual）本期暂不支持作为优惠券适用范围。如需对虚拟商品做活动，请在后续迭代中跟进。

### Q3：「排除商品」和「指定商品」可以同时用吗？

A：不行。当 scope=指定商品 时，「排除商品」区块自动隐藏（业务上你已经精确指定了商品列表，无再排除场景）。这是 BR-1 规则。

### Q4：上限 100 / 50 改不动怎么办？

A：在 `system_configs` 表中调整：
- `coupon_scope_max_products`（默认 100）
- `coupon_exclude_max_products`（默认 50）

修改即时生效，无需发版重启。

### Q5：选了父分类「食品保健」后，是不是子分类「蜂蜜茶饮」下新加入的商品也会自动适用本券？

A：是的。本期实现的关键设计：scope=category 时，`scope_ids` 只存最上层分类 ID，订单计算时实时通过 `product.category_id IN scope_ids OR parent IN scope_ids` 判断。**这是符合「分类」概念直觉的标准做法**。

### Q6：免费试用券为什么提醒我必须指定商品？

A：风控建议。免费试用 = 整单 0 元，如果 scope=all 等于全店免单，风险极高。系统会在保存时弹**黄色二次确认**：「免费试用券设为全店生效风险较高，是否继续？」。运营可坚持继续，但建议：① 限发指定商品 ② 设置发行总量 ③ 单人限领。
