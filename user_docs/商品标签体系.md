> 适用版本：v1.0（标签体系重构 一期上线）
> 文档发布日期：2026-05-20
> 适用人群：运营人员（后台标签管理 / 商品打标）、C 端用户（小程序 / APP / H5 / Web）

## 一、功能介绍

本次重构将商品标签体系从「自由文本 + 9 选 Checkbox」彻底升级为 **结构化、可复用、按分类组织的标签字典**，全平台共用同一套标签数据。

新体系覆盖 **6 大标签分类**：

| 分类 Key | 中文名 | 说明 |
| --- | --- | --- |
| `constitution` | 体质类 | 9 种中医体质（平和/气虚/阳虚/阴虚/痰湿/湿热/血瘀/气郁/特禀），系统预置并锁定 |
| `symptom` | 症状类 | 与症状相关的标签，初始为空，运营自行新增 |
| `crowd` | 人群类 | 如「孕妇」「中老年」「上班族」 |
| `effect` | 功效类 | 如「补气血」「改善睡眠」 |
| `scene` | 场景类 | 如「换季」「加班」 |
| `contraindication` | 禁忌类 | 如「孕妇慎用」「哺乳期禁用」 |

### 主要功能

1. **后台「标签管理」一级菜单**：6 个 Tab 切换，集中维护全部标签。
2. **商品编辑页 Chip 池选择**：按分类平铺，点击即选/再点即取消，挂载数量无上限、无必填。
3. **实物商品自动隐藏服务相关分类**：履约方式为「实物」时，禁忌类等不适用分类自动隐藏。
4. **C 端筛选 / 详情 / 相关商品 全量切换新体系**：小程序、安卓 APP、苹果 APP、H5、Web。
5. **数据迁移自动完成**：9 种体质已预置并锁定，旧字段已 drop。

## 三、运营操作指南

### 1. 打开「标签管理」菜单

1. 登录管理后台。
2. 左侧导航 → **商品体系** → **标签管理**。
3. 顶部依次显示 6 个分类 Tab：体质类 / 症状类 / 人群类 / 功效类 / 场景类 / 禁忌类。

### 2. 新增标签

1. 切换到要新增标签的分类 Tab（如「症状类」）。
2. 点击右上角「**新增标签**」按钮。
3. 输入标签名、排序值（数字越小越靠前），所属分类自动填入当前 Tab。
4. 点击「确定」，列表实时刷新。

> 同一分类下，标签名不可重复（系统会校验）。

### 3. 启用 / 停用标签

- 列表「状态」列点击开关即可启停。
- 停用后，C 端筛选器不再展示该标签，但商品上已挂载的关联不会丢失（恢复启用后立即可见）。

### 4. 编辑 / 删除标签

- **编辑**：点击操作列「编辑」，可改名、改排序。**体质类的 9 个锁定标签禁止改名、改分类，只能改排序与启停**。
- **删除**：点击操作列「删除」。
  - 如果该标签**已被任意商品引用**，会弹出提示「被 N 个商品使用，请先解除引用」，删除将被拒绝。
  - 如果该标签是体质类的 9 个**锁定标签之一**，删除按钮不可点击（图标变灰）。

### 5. 给商品打标签（商品编辑页）

1. 进入「商品管理」→ 选一个商品 → 点击「编辑」。
2. 切到「**标签设置**」Tab。
3. 按分类查看 Chip 池，**点击 Chip 即选中（高亮）**、再次点击即取消。
4. 分类标题旁实时显示「已选 N」计数。
5. 履约方式为「实物」时，**禁忌类等不适用分类整体隐藏**（无需手动屏蔽）。
6. 保存即可，标签关联实时落库。

> 全部分类**均无必填校验**，可只挂部分分类、也可一个都不挂。

## 四、C 端用户体验

### 1. 商品列表 - 筛选

- 在小程序 / APP / H5 商品列表页，筛选条件由旧的「症状文本框 + 体质 9 选」**升级为按 6 大分类组织的 Chip 选择器**。
- 多个标签筛选条件之间为「**AND**」关系（同时命中）。
- 体质类是核心匹配维度，**用户完成体质识别后，系统会自动加权推荐对应体质标签的商品**。

### 2. 商品详情 - 标签展示

- 商品详情页在标题下方按 **6 大分类分组渲染**该商品已挂载的标签。
- 禁用状态的标签不会显示。

### 3. 相关商品推荐

- 商品详情页底部「**相关商品**」基于「**标签命中数加权**」算法：
  - 命中标签越多越靠前；
  - 同类目 > 跨类目；
  - 销量 / 时间作为兜底排序。

### 4. 个性化首页

- 用户体质识别完成后，体质标签会作为个性化首页推荐的权重输入。
- 用户偏好的标签命中后，相关商品会优先展示。

## 五、API 使用示例

### 1. 获取启用标签（按分类）

```http
GET /api/admin/tags?category=constitution&status=1
Authorization: Bearer <admin_token>
```

### 2. C 端商品列表 - 按标签筛选

```http
GET /api/products?tag_ids=109,110&page=1&page_size=20
```

返回字段中新增：

```json
{
  "id": 20,
  "name": "艾灸疗养服务",
  "tag_ids": [109, 110],
  "tags": {
    "constitution": [{"id":109,"name":"平和体质"}],
    "effect": [{"id":50,"name":"补气血"}]
  }
}
```

### 3. 相关商品推荐

```http
GET /api/products/20/related?limit=6
```

## 六、注意事项

1. **体质类 9 个标签为系统锁定**，**禁止改名、禁止删除、禁止改分类**，仅允许改排序与启停。
   - 若误停用某个体质，C 端筛选与个性化推荐会立刻缺失该体质，请及时恢复。
2. 旧的 `symptom_tags`（自由文本）字段**已被彻底删除**，历史脏数据**不再迁移**——上线后请运营在新「标签管理」中重新补齐症状类、人群类、功效类、场景类、禁忌类标签。
3. 旧的 `service` 分类标签已自动合并到 `contraindication` 分类，`other` 分类标签已被停用（`status=0`），**不影响 C 端展示**。
4. **删除标签前需先解除商品引用**——这是保护机制，避免误删导致 C 端筛选数据缺失。
5. **实物商品自动隐藏禁忌类等服务类分类**——切换履约方式后，UI 会即时变化，无需刷新页面。
6. 标签变更（新增 / 停用）**实时对 C 端生效**，但 APP 端可能因本地缓存有 1～5 分钟延迟。

## 七、常见问题（FAQ）

**Q1：我能删除「平和体质」吗？**
不能。9 个体质标签为系统锁定，仅可改排序与启停。

**Q2：商品上的旧症状标签去哪了？**
旧字段 `symptom_tags`、`constitution_types` 已 drop，体质数据已自动迁移到新 `goods_tags` 表，旧症状自由文本按方案要求**全量丢弃**，请运营在新体系下重新打。

**Q3：实物商品能否打「孕妇慎用」（禁忌类）标签？**
当前一期方案：**实物商品页面隐藏禁忌等服务相关分类区域**。如需为实物商品也支持禁忌类，请提需求评估。

**Q4：标签数量有上限吗？**
单个商品挂载标签数量**无上限**，但建议每个分类不超过 5 个，避免 C 端展示过于拥挤。

**Q5：C 端怎么看到我新加的标签？**
新增并启用后，C 端商品列表筛选器、商品详情页立即可见。APP 端如本地有缓存可能稍有延迟。
