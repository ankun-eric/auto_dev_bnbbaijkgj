## 功能简介

商家以前在预约日历上只能看到每天「N 单」的徽标，要看具体订单必须跳走到订单列表再筛选日期。本次升级后：

- **PC 端**：点击日历中订单数 ≥ 1 的日期单元格，**就地弹出「当日订单弹窗」**，无需跳转
- **H5 端**：点击日期单元格，**底部抽屉**呈现当日订单卡片，配合手机操作习惯
- **统一交互**：弹窗/抽屉内含 5 个状态 Tab（全部 / 待核销 / 已核销 / 已取消 / 已退款），按"待办优先"自动排序，详情字段一应俱全（核销时间、核销码、取消/退款时间、客户手机号 + 拨打/复制等）

## 使用步骤（PC 端）

1. **登录商家后台**：使用商家管理员或店员账号通过 [登录入口](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login) 登录
2. **进入预约日历**：在左侧菜单点击 **「预约日历」**，或直接访问 [PC 预约日历](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/calendar)
3. **点击有订单的日期单元格**：日历上每天会显示「共 N 个预约」徽标 + 上下午晚的密度色块。鼠标移到这些单元格上时，光标会变成手型（pointer）
4. **查看当日订单弹窗**：点击后会立即弹出"当日订单"弹窗（标题示例：`2026年5月4日 周一 · 当日订单（共 5 单）`）
5. **切换状态 Tab**：弹窗顶部有 5 个 Tab：
   - **全部(N)**：默认选中，展示所有订单
   - **待核销(N)**：已支付但未核销的订单（最优先关注）
   - **已核销(N)**：已完成核销的订单
   - **已取消(N)**：用户/商家取消的订单
   - **已退款(N)**：已退款完成的订单
6. **就地展开订单详情**：
   - 点击表格行任意位置即可展开/收起详情面板
   - **同一时刻最多展开一行**，点击其他行会自动收起前一行
   - 详情字段：客户备注 / 服务地点 / 客户手机号（带 📞 拨打 + 📋 复制按钮） / 核销状态 / 核销时间 / 核销码（仅已核销订单显示）
   - 已取消订单额外显示「取消时间」+「取消原因」
   - 已退款订单额外显示「退款时间」+「退款原因」
7. **跳转完整订单**：详情面板底部点击 **「查看完整订单」**，跳转到该订单在订单列表的高亮位置
8. **关闭弹窗**：右上角 **×** + 点击半透明遮罩 + 键盘 **ESC**，三种方式都可关闭

## 使用步骤（H5 移动端）

1. **登录商家 H5**：用同样账号在手机上访问 [H5 登录](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login)
2. **进入移动端预约日历**：从底部导航或工作台进入「预约日历」，或直接访问 [H5 预约日历](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/m/calendar)
3. **点击有订单的日期格**：日历单元格下方有「N单」+ 三色密度块。点击有订单的日期格
4. **底部抽屉弹出**：屏幕底部弹出抽屉，最高占屏 85%，可上下滚动；标题栏显示「日期 · 共 N 单」+ 右上角 **✕** 关闭按钮
5. **状态 Tab**：抽屉顶部固定 Tab 区，5 个 Tab 横向滚动，点击切换；列表区在 Tab 下方独立滚动
6. **订单卡片**：每张卡片显示：
   - 顶部：预约时段 + 状态徽标（颜色：橙色待核销 / 绿色已核销 / 红色已取消 / 紫红已退款）
   - 中部：客户昵称 · 服务项目
   - 底部：📍 服务地点
   - 右下角：「展开详情 ▼」/「收起 ▲」
7. **就地展开**：点击卡片就地展开详情，与 PC 端字段一致；点击「📞 拨打」直接调起手机电话；点击「📋 复制」复制纯数字手机号到剪贴板
8. **关闭抽屉**：点击右上角 **✕** 或点击抽屉外的半透明遮罩

## 注意事项

1. **核销码安全**：核销码是客户的专属凭证，**未核销订单接口不会下发核销码**，前端无法绕过。仅当订单状态变为「已核销」时，商家才能在详情面板中看到核销码（用于事后追溯）
2. **手机号权限**：仅商家管理员、店员等有权限的角色可看到完整 11 位手机号；复制 / 拨打操作会写入操作审计日志，请合规使用
3. **排序规则（强制）**：列表始终按 **待核销 → 已取消 → 已退款 → 已核销** 排列，组内按预约时段升序。设计目的是让你**优先关注待办**，已完成的订单沉到最下方
4. **状态合并**：原本"订单状态" + "核销状态"两个字段合并为单一"状态"字段，5 个枚举值一目了然
5. **统计口径一致**：弹窗里的"共 N 单"与日历上的徽标统计**完全一致**，避免徽标显示 5 单、弹窗里只有 4 单的不一致问题
6. **空状态**：切换到某个 Tab 但当天 0 条记录时，弹窗保持打开，列表区显示「该状态下暂无记录」，可继续切换其他 Tab
7. **0 单日期不响应**：当天没有任何预约的日期格，点击不会弹出弹窗（普通展示）
8. **数据时效**：弹窗打开后**不实时刷新**；如需获取最新状态，关闭再次点击同一日期即可重新拉取
9. **后端鉴权**：未登录访问 API 会返回 401，无门店权限会返回 403，日期格式错会返回 400，已通过 13 项后端非 UI 自动化测试覆盖
10. **多端打包**：本次更新仅涉及商家端 PC + H5 网页，**微信小程序 / 安卓 App / iOS App / Windows 桌面端均零改动**，无需重新下载客户端
