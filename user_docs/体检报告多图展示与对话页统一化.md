> 版本：v2.0（2026-04-23 发布 · Phase 2）
> 适用范围：Web / H5 / 小程序 / Flutter App / Admin 后台

## 一、功能介绍

本次更新完成两件事：

1. **体检报告原图完整展示**：过去上传多张体检报告时，各端只能看到"第一张"，本次修复后，所有上传的原图都能以 **九宫格缩略图** 形式完整展示，点击任一张即可进入 **全屏多图预览**，支持 **左右滑切换 / 双指缩放 / 底部页码**。
2. **对话页统一化**：体检报告解读对话和公共 AI 健康咨询合并为同一个页面，享受 **语音输入 / 语音播报 / 更丰富的输入框** 等完整能力。体检报告特有的"顶部报告卡片 + 查看原图"按钮被无缝迁移到公共对话页中。

### 2.1 H5 / 小程序 / App：查看报告原图多图预览

1. 进入「体检」模块，点击任一条历史报告
2. 在报告详情页下方，可以看到「报告原图（共 N 张）」九宫格
3. 点击任意缩略图进入全屏预览
4. 左右滑动切换图片，双指捏合缩放，顶部显示「当前/总数」

### 2.2 从报告详情进入 AI 解读对话

1. 打开某份报告详情页，底部点击「AI 解读」或「继续对话」
2. 系统自动跳转到公共 AI 健康咨询页（带 `type=report_interpret&auto_start=1`）
3. 对话页顶部会显示「🩺 报告解读：XX 报告（共 N 张）」卡片，点击缩略图可查看完整原图
4. 若是首次进入，AI 会自动开始首条解读回复（auto_start 机制）
5. 语音输入、语音播报、键盘/语音切换等能力都已可用

### 2.3 报告对比场景

1. 在"体检报告列表"中选择两份报告，点击「报告对比」
2. 系统自动跳转到公共对话页（`type=report_compare&auto_start=1`）
3. 顶部显示两张报告卡片（A / B），分别可点击查看各自的原图
4. AI 会对两份报告进行差异解读

### 2.4 Admin 后台：查看 OCR 多图

1. 登录 Admin → 「AI 管理」→「体检报告解读明细」
2. 点击任一条记录右侧「详情」
3. 在抽屉中展开「原始图片（共 N 张）」九宫格
4. 点击任一图片进入全屏预览，支持左右切换

### 2.5 兼容旧链接

所有旧的 `/h5/checkup/chat/:sid` 链接都会被 301 永久重定向到新的
`/h5/chat/:sid?type=report_interpret&auto_start=1`，用户历史收藏 / 外部分享不会失效。

## 三、注意事项

1. **首次上传多图后**：需要等待 OCR 完成才能在列表页和详情页看到全部图片。单次 OCR 失败的图片不会被写入。
2. **Admin 上传上限**：默认 5 张，可在 Admin 的系统设置中调整，超出会被截断。
3. **历史数据**：服务器已执行一次性回溯脚本（`backend/scripts/migrate_checkup_multi_image.py`），历史老报告的 `file_urls / thumbnail_urls` 字段已被填充。如有遗漏，可联系开发组重新执行（支持 `--dry-run` 预演 与 `--undo` 回滚）。
4. **旧对话页废弃**：`/h5/checkup/chat/:sid` 保留 7 天作为回滚兜底，之后将被删除。请以 `/h5/chat/:sid` 为准。
5. **多端打包发布**：Admin + H5 已部署到测试服。微信小程序和 Android / iOS App 需走各自的发布流程（小程序 → 微信平台；App → 商店）。
6. **浏览器兼容**：语音播报在 iOS Safari 需用户主动点击触发一次（Safari 播放策略），属于浏览器行为，非本次引入的问题。

## 四、常见问题 FAQ

- **Q**：为什么老报告还是只显示一张？
  **A**：检查对应 OCR 记录是否只有一张成功，或提请开发组重新执行回溯脚本。

- **Q**：点击旧链接 `/h5/checkup/chat/123` 能否用？
  **A**：能，会自动 301 跳到新页面并带上 `type=report_interpret&auto_start=1`。

- **Q**：大图预览是否支持保存到相册？
  **A**：小程序和 App 原生支持长按保存；H5 用户可长按图片选择"保存图像"。

## 五、回滚方案（运维参考）

- 前端：`git revert` 本次前端改动，重新部署 H5 / Admin。
- 数据：`docker exec {DEPLOY_ID}-backend python -m scripts.migrate_checkup_multi_image --undo`
- 路由：`/h5/checkup/chat/:sid` 的废弃文件仍在 Git 中，恢复 `next.config.js` 旧版本即可。
