## 功能简介

健康档案页（`/health-profile`）顶部的"成员切换条"原本仍是 V1 旧版"蓝色渐变描边头像条"，与新版设计稿不一致。本次按 PRD V2 §F2 规范完整重写：

1. **选中态**：橙色实底 `#FF8A3D` + 白字，更醒目；
2. **未选中态**：浅蓝灰 `#EAF2FF` 整块底色，视觉更柔和；
3. **圆形字徽**：按关系自动显示一个汉字（如"我/爸/妈/儿/女/爱/哥/姐/爷/外"），底色按入档顺序在橙/蓝/绿/紫/粉 5 色中循环；
4. **文案排版**：每个 Tab 内显示两行——上行为称呼或关系（本人/爸爸/老婆…），下行为成员姓名；
5. **末尾"+"按钮**：灰底深灰加号圆形按钮，点击调起"新增家庭成员"抽屉；
6. **排序规则**：本人永远第一，其余按入档时间从早到晚排列；
7. **数据持久化**：徽章配色已落入数据库（`avatar_color_index` 字段），刷新页面、换浏览器、重新登录后同一成员的颜色保持一致。

## 操作步骤

### 1. 进入健康档案页

打开浏览器，访问：

[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile)

首次访问需要登录账号，登录后即进入健康档案页面。

### 2. 查看新版成员 Tab

页面顶部即新版成员切换条，每个 Tab 大小为 `76×96`，包含：
- 顶部圆形字徽（直径 36px，按关系自动取字、按入档顺序取色）；
- 中间一行：称呼或关系（如"本人""老婆""爸爸"）；
- 下方一行：成员姓名（如"张三"，过长自动省略号截断）。

### 3. 切换成员

点击任意一个成员 Tab，该 Tab 会变为**橙色 `#FF8A3D` 实底 + 白字**，下方 Hero 卡及健康数据自动刷新为该成员的数据。

### 4. 新增家庭成员

滑动到 Tab 末尾，可以看到一个**灰底深灰加号**的圆形 + 按钮：
- 点击 "+" 按钮，会弹出"新增家庭成员"抽屉；
- 录入关系（如"儿子""女儿""老婆""爸爸"等）、姓名、性别、生日等信息；
- 保存后，新成员会立即出现在 Tab 末尾，并自动分配一个新的 5 色循环底色（按 `入档顺序 % 5` 取色）。

### 5. 关系字徽对照表

新成员录入时，系统会根据关系自动选取徽章上的单字：

| 关系 | 徽章字 |
|---|---|
| 本人 | 我 |
| 爸爸 / 父亲 | 爸 |
| 妈妈 / 母亲 | 妈 |
| 儿子 | 儿 |
| 女儿 | 女 |
| 老公 / 老婆 / 丈夫 / 妻子 / 伴侣 | 爱 |
| 哥哥 | 哥 |
| 弟弟 | 弟 |
| 姐姐 | 姐 |
| 妹妹 | 妹 |
| 爷爷 | 爷 |
| 奶奶 | 奶 |
| 外公 / 外婆 / 姥姥 / 姥爷 | 外 |
| 其他亲属 | 取关系字段第一个字 |

> 与原 PRD 的差异：**"儿子"与"女儿"分别显示"儿"与"女"**，不再统一显示"娃"。

### 6. 5 色循环底色

字徽底色按入档顺序循环：

| 序号 | 颜色 | 字徽底色 | 字徽字色 |
|---|---|---|---|
| 0 | 橙 | `#FFE8D6` | `#E66A1F` |
| 1 | 蓝 | `#E0EFFF` | `#1F6FE6` |
| 2 | 绿 | `#E8F7EE` | `#1FA168` |
| 3 | 紫 | `#EFE4FF` | `#7E3FE6` |
| 4 | 粉 | `#FFE4EE` | `#E63F86` |

第 6 个成员开始再次从橙色循环。**选中态下字徽会额外加 2px 白色描边**，提升对比度。

## 注意事项

1. **历史数据自动回填**：原有家庭成员在升级后会自动按"入档顺序 % 5"分配 `avatar_color_index`，无需手动操作。
2. **本人永远第一**：无论排序如何变化，"本人" Tab 始终位于最左侧。
3. **被守护角标保留**：右上角"被守护"小角标的显示逻辑保持不变，仅样式微调以贴合新字徽。
4. **未影响其他模块**：本次改动**仅限**健康档案页顶部成员 Tab；Hero 卡、三个快捷入口（今日用药 / 我的设备 / 家庭成员）、`/family` 家庭成员列表页、`/devices/member` 设备列表页、提醒设置抽屉、解绑抽屉等模块**未被改动**。
5. **关系字段是空字符串时**：徽章会取昵称的首字作为兜底，避免显示空白。
6. **多端兼容**：本次为 H5 端样式优化，小程序、安卓 APP、苹果 APP、Windows 桌面端**不在本次改动范围内**，相关接口向后兼容，旧客户端继续可用。
