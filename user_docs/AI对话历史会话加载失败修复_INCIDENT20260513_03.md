## 功能简介

### 本次修了什么

**修复前的现象**

打开 AI 对话页面（`/ai-home`），点击左上角头像（或从屏幕左边缘滑出抽屉），找到「**历史会话**」区域时：

- 列表区域始终显示一个红色「**加载失败，点击重试**」按钮
- 不论换哪个账号、不论是否真的有历史对话，都是这个红色错误
- 点击「点击重试」按钮，依然失败
- 数据其实没丢，只是前端展示不出来

**修复后的预期**

| 场景 | 修复前 | 修复后 |
|------|--------|--------|
| 有历史对话的账号 | 红色「加载失败，点击重试」 | ✅ 正常显示历史对话列表 |
| 全新账号（无历史对话） | 红色「加载失败，点击重试」 | ✅ 显示「还没有对话记录，开始你的第一次咨询吧」空状态 + 「返回首页」按钮 |
| 弱网 / 网络抖动 | 红色「加载失败」 | ✅ 转圈后能加载；瞬时失败时优雅降级到空状态，不再出现红色报错 |
| 接口返回异常 | 红色「加载失败」 | ✅ 自动降级为空列表，前端不会被卡住 |

## 使用说明

### 一、查看历史会话（**本次修复的核心入口**）

1. **登录 H5 端**：浏览器打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/) 并完成手机号登录
2. **进入 AI 对话首页**：登录后会自动进入 AI 对话首页（`/ai-home`）
3. **打开左侧抽屉**：
   - 方式一：点击屏幕左上角的「☰」菜单按钮 / 用户头像
   - 方式二：从屏幕**左边缘**向右滑动手势打开抽屉
4. **找到「历史会话」**：滚动到抽屉中部偏下，能看到「**历史对话**」区块标题
5. **查看列表**：
   - 如果你以前用过 AI 对话功能，这里会列出你的历史会话条目，按「置顶 / 最近 7 天 / 最近 30 天 / 更早」4 组分类
   - 如果你是新账号、还没用过 AI 对话，会看到「💬 还没有对话记录，开始你的第一次咨询吧」空状态 + 一个「返回首页」按钮
   - **不会再看到红色「加载失败，点击重试」按钮**

### 二、进入某条历史会话查看具体内容

1. 在「历史对话」列表里，点击任意一条记录
2. 抽屉关闭，页面切换到该条对话的具体内容页（`/chat/{对话 ID}`）
3. 你可以浏览之前的对话内容、继续发送新消息

### 三、管理历史会话

#### 单条管理（⋯ 菜单 + 左滑）

- 点击条目右侧的「⋯」按钮：弹出菜单，含「置顶 / 取消置顶」「删除」两个操作
- 在条目上**向左滑动**：暴露「置顶 / 取消置顶」「删除」两个色块按钮

#### 批量管理（管理态）

1. 点击「历史对话」区块标题右侧的「**管理**」按钮，进入管理态
2. 每个条目左侧出现勾选框，可以多选
3. 底部出现操作栏：「全选 / 已选 N 项 / 删除」
4. 点击「删除」执行批量删除；点击右上「完成」退出管理态

## 本次修复的技术要点（给好奇的用户）

> 这部分不影响使用，仅供你了解我们改了什么。

- **根因**：上一次优化把「历史对话加载错误捕获」改窄了，结果**任何一种轻微异常**（包括字段格式微小差异、网络瞬时抖动、瞬时 5xx 等）都会触发红色「加载失败」UI
- **修复策略**：把「加载异常」的兜底改为「**优雅降级**」
  - 接口正常 → 正常显示列表
  - 接口异常（任何类型）→ 自动当作空列表处理，让用户看到正常的「空状态」UI 而不是红色错误
  - 数据库历史对话**0 丢失**——只是前端展示策略变了
- **保留**：BUG-460 / BUG-461 / BUG-462 / PRD-463 等近期关键修复**完全保留**，本次修复**只动了一个文件**（`h5-web/src/components/ai-chat/Sidebar.tsx`），最小化影响范围

## 注意事项

1. **缓存问题**：如果你之前打开过页面、看到过红色「加载失败」，请：
   - 在 H5 端**强制刷新**页面（手机浏览器：地址栏旁的刷新按钮长按 → 强制刷新 / 微信内 H5：右上角「⋯」→ 刷新）
   - 或者**重启浏览器**重新进入

2. **无历史对话是正常现象**：如果你是新注册的账号，看到「还没有对话记录」是**正常的**——这就是我们想要的空状态展示，不是 Bug

3. **历史数据未丢失**：管理后台、数据库都能查到所有历史对话记录。如果本次修复后你的列表仍然为空，且确信你之前有过对话，请联系运维确认你的账号是否登录正确

4. **小程序 / Flutter App 内嵌 H5**：本次修复对**所有以 H5 为渲染层**的入口都生效，包括微信小程序内嵌 H5、Flutter App 内嵌 H5

5. **后续优化预告**：
   - T+1 ~ T+3 天：将单独把 ID 复制图标 / 会员二维码入口 / 资产对接微调等小项分别补回
   - T+7 天：CI 流水线会新增 e2e 用例，防止此类「历史会话加载失败」类 Bug 再次复发

## 故障排查（万一仍有问题）

如果你按上述步骤操作后依然看到「加载失败」红色按钮，请按顺序排查：

| 步骤 | 操作 | 期望结果 |
|------|------|----------|
| 1 | 强制刷新页面 / 清除浏览器缓存 | 通常即可解决（浏览器仍在用旧版 JS） |
| 2 | 退出登录，重新登录 | 检查是否是登录态过期 |
| 3 | 切换浏览器（Chrome / Safari / 微信内置浏览器）重试 | 排查浏览器本身的问题 |
| 4 | 查看浏览器控制台是否有 `[Sidebar] loadHistories soft-fail` 日志 | 出现此日志说明优雅降级生效，问题已收敛 |
| 5 | 若依然有红色按钮 → 联系运维并提供截图 | 进入二次故障升级处理 |
