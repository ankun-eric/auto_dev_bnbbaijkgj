## 一、本次升级要点速览

V2.1 优化后，优惠券功能在以下四个方面进行了完整升级：

1. **领券中心更友好** — 已领取/已抢光的券会明确置灰显示，避免重复点击或误操作。
2. **兑换码可追溯** — 运营可按"批次 + 明细"两层视图查看每个兑换码的状态、使用情况，并支持单码 / 整批作废、复制、CSV 导出。
3. **优惠券更安全** — 优惠券一律不可物理删除，统一改为"下架（带原因记录）"，下架仅超级管理员可操作。
4. **预留积分兑换** — 优惠券新增"积分兑换次数上限"字段，为后续积分兑换券功能预埋。

## 二、用户端：领券中心使用说明

### 2.1 入口

- **H5 / Web**：`https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/coupon-center`
- **小程序**：底部"我的" → "我的优惠券"卡片右上角"去领券" → 进入"领券中心"页（页面路径 `pages/coupon-center/index`）
- **App（Flutter）**：首页"会员福利" → "领券中心"

### 2.2 按钮文案与显示规则

V2.1 起，每张券的按钮根据用户和库存状态自动呈现以下四种之一：

| 场景 | 按钮文案 | 状态 | 说明 |
|------|---------|------|------|
| 未登录 | **领取**（绿色） | 可点击 | 点击后引导登录，登录完成自动返回 |
| 已登录 + 未领过 + 库存>0 | **领取**（绿色） | 可点击 | 直接领取，成功后按钮变灰显示「已领取」 |
| 已领过（无论 unused/used/expired） | **已领取**（灰色） | 不可点击 | 每张券每人限领 1 次，永久判定 |
| 未领过 + 库存=0 | **已抢光**（灰色） | 不可点击 | 库存归零提示，已下架的券不会出现在列表中 |

### 2.3 重要提示

- 已领过的券，即使后续使用、过期，仍会在领券中心持续显示「已领取」。
- 已下架的券，**完全不会**出现在领券中心；但若你之前已经领取，仍可在「我的优惠券」中正常使用至过期。
- 一旦操作领取成功，列表会自动刷新，按钮立即置灰，避免重复点击造成误操作。
- 重复点击「领取」时，后端会返回友好提示"您已领取过该券"，并自动刷新按钮状态。

### 2.4 我的优惠券

- **H5**：`https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/my-coupons`
- 三个标签：可使用 / 已使用 / 已过期。
- 即使券被运营下架，已领取的依然在此正常显示，并可使用至过期日。

## 三、运营管理端：优惠券管理升级

> Admin 入口：`https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin`
> 优惠券管理页：`https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/coupons`
> 默认管理员账号（首次登录）：手机号 `13800138000`，密码 `admin123`（已默认开启 `is_superuser`，可执行下架）

### 3.1 优惠券列表新功能

- 新增「上下架筛选」下拉框：全部 / 上架中 / 已下架。
- 列表中已下架的券会显示红色「已下架」标签。
- 顶部的提示栏会根据当前账号权限提示是否可执行下架/上架。

### 3.2 下架优惠券（替代原"删除"）

V2.1 起，"删除"按钮已**完全移除**，统一替换为"下架"：

操作步骤：

1. 进入优惠券列表，定位目标券，点击右侧操作列的 **下架** 按钮。
   - 如当前账号非超级管理员，按钮处于灰色禁用状态，需联系超管处理。
2. 弹窗中选择"下架原因"（必选）：
   - 活动结束 / 配置错误 / 库存调整 / 业务调整 / 其他
   - 选择"其他"时，必须填写详细原因（最少 5 字）。
3. 点击「确认下架」。
4. 系统将该券标记为"已下架"，并记录原因、操作人、时间到 `coupon_op_logs` 表。
5. 已下架的效果：
   - 领券中心**完全不显示**该券。
   - "我的优惠券"中已领的依然显示，可正常使用至过期。
   - 发放记录、统计数据完整保留。

### 3.3 重新上架

1. 在优惠券列表筛选"已下架"。
2. 点击目标券右侧的 **重新上架** 按钮。
3. 弹窗确认后立即上架，领券中心立即可见。

### 3.4 新人券下架强校验

- 系统会检测当前 `NEW_USER_COUPON_KEY` 引用的券，**禁止直接下架**。
- 若尝试下架新人券，弹窗提示：「该券是当前新人券，请先在新人券配置页切换到另一张券再下架」。
- 切换路径：Admin → 优惠券管理 → 选定其他券 → 点击「发放」→ 选择"D 新人券"→ 加入新人券池；然后再下架原券。

### 3.5 删除接口已禁用

`DELETE /api/admin/coupons/{id}` 接口已下线，调用会返回 **HTTP 405 Method Not Allowed**，并提示请使用下架接口。

## 四、F 兑换码批次与明细管理

### 4.1 创建兑换码批次

1. 在优惠券列表，点击目标券的「发放」按钮。
2. 在"发放方式"中选择 **F 兑换码**，确认后进入批次创建窗口。
3. 选择码类型：
   - **A 一码通用**：所有用户使用同一个码。**必须填写"领取人数上限"**（V2.1 强制必填）。
   - **C+ 一次性唯一码**：每码每人只能使用 1 次，生成数量自动等于"领取人数上限"。
4. 可选填：兑换码有效期（独立于券有效期）、第三方合作方。
5. 提交后系统自动生成"批次编号"（格式 `BATCH-YYYYMMDD-XXX`）。

### 4.2 查看批次列表

1. 在优惠券列表，点击目标券右侧的 **兑换码批次** 按钮。
2. 抽屉打开后展示该券的全部批次，10 个核心字段：
   - 批次编号 / 关联券 / 码类型 / 生成时间 / 生成人 / 总数 / 已用 / 未用 / 已作废 / 上限 / 有效期 / 状态。

### 4.3 查看批次明细

1. 在批次列表点击「查看明细」，进入明细抽屉。
2. 一码通用类型：
   - 顶部 4 个统计卡片：兑换码（默认脱敏）、领取上限、已被兑换、剩余。
   - 下方"兑换记录"表格：用户ID / 手机号 / 兑换时间 / 状态。
3. 一次性唯一码类型：
   - 表格列：兑换码（默认脱敏）、状态（未用/已用/已作废）、兑换用户、兑换时间、作废原因、操作。
4. 点击右上角「显示完整码」/「隐藏码」可切换脱敏状态（默认脱敏，需主动确认才显示完整码以保护数据）。
5. 单码可一键复制（仅在显示完整码状态下）。

### 4.4 单码作废

在批次明细中，点击某个未使用兑换码右侧的 **作废** 按钮：

1. 弹窗内必须填写作废原因（必填）。
2. 提交后该兑换码立即失效，无法再被新用户兑换。
3. 已通过该码领到的优惠券**不受影响**，用户可继续使用至过期。
4. 作废操作记录到 `coupon_op_logs`。

### 4.5 整批作废（强二次确认）

在批次列表点击 **作废整批**：

1. 弹窗"批次编号确认"输入框：必须输入完整批次编号（如 `BATCH-20260419-001`）才能提交。
2. 必填作废原因。
3. 点击「确认作废」后整批立即失效：
   - 所有未用码不能再被新用户兑换。
   - 已通过该批次领到的优惠券不受影响。

### 4.6 导出 CSV

在批次列表的某行点击「导出CSV」，浏览器会下载该批次完整码列表（含状态、用户、时间）的 CSV 文件，UTF-8 BOM 编码，Excel 可直接打开。

V2.1 在优惠券创建/编辑表单新增「积分兑换次数上限」字段：

## 七、常见问题

**Q1：我之前领过这张券，但现在过期了，能再领一次吗？**
A：不能。每张券每人限领 1 次，无论是 unused / used / expired，按钮永久置灰显示「已领取」。

**Q2：管理员误操作下架了一张券，已发出去的券会失效吗？**
A：不会。下架只阻止新用户领取/兑换，已领取的用户可继续使用至过期。可在 Admin → 优惠券列表 → 筛选「已下架」→ 点击「重新上架」立即恢复。

**Q3：兑换码批次作废后，已用此码领到券的用户会被回收吗？**
A：不会。作废只阻止新用户兑换，已领到的券保持有效。如确实需要回收，请在「发放记录」抽屉中使用"批量回收"功能。

**Q4：怎么判断哪些是"已下架"的券？**
A：Admin 列表中已下架的券显示红色「已下架」标签，并可通过列表上方的"上下架筛选"快速过滤。

**Q5：为什么我的账号下架按钮是灰的？**
A：下架仅超级管理员（`is_superuser=True`）可操作。请联系系统管理员开通权限，或由超管代为操作。

**Q6：积分兑换次数怎么用？**
A：本期仅做预留，等积分兑换券功能正式上线后启用。当前填写不会影响"免费领取限领 1 次"的判断。
