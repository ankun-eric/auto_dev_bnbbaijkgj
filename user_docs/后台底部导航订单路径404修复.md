## 一、本次修复内容

以前在 **管理后台 → 首页配置 → 底部导航配置** 中：

- 选择「订单」后，系统自动填入的跳转路径是 **`/orders`**。
- 用户点击 H5 / 小程序 / APP 底部导航的「订单」按钮，页面跳转到了 **404 错误页**。

原因是三端实际订单页的路径在 2026 年「订单系统统一升级」时已经改为 `/unified-orders`，但管理后台的默认配置没跟上。

**本次修复后：**

- 管理后台的「订单」下拉选项默认值统一改为 `/unified-orders`；
- 后端启动时自动将数据库里老的 `/orders` 迁移为 `/unified-orders`（幂等安全）；
- Flutter APP 内部的订单路由也跟进改名；
- 用户点击「订单」按钮，三端（H5 / 小程序 / APP）均可正常进入订单列表页。

## 四、注意事项

1. **请先清理浏览器缓存 / 小程序缓存**：在 H5 或小程序上做验证前，建议手动清缓存，避免老的导航配置残留。
2. **后端接口路径未变**：本次只改页面路径（前端路由），`/api/orders/...` 等后端接口保持原样，API 调用方 **无需任何适配**。
3. **老路径 `/orders` 已彻底停用**：如果你有其他地方硬编码了 `/orders` 跳转，需要自行升级到 `/unified-orders`。
4. **幂等迁移**：后端启动时的数据库迁移脚本是幂等的——重复启动不会重复迁移，也不会误伤你手动配置的其他同路径菜单。
5. **APP 需要重新安装**：APP 是原生路由表修改，**必须重装最新包**才会生效；旧包需要卸载后重装。

## 五、故障排查

| 现象 | 可能原因 | 处理建议 |
|------|---------|---------|
| 后台「订单」下拉仍显示 `/orders` | 浏览器缓存未清 | 强制刷新（Ctrl+Shift+R）或清缓存 |
| H5 点击「订单」仍 404 | 数据库迁移未执行 / 数据被手动回滚 | 进入后台重新手动选择「我的订单 (/unified-orders)」并保存 |
| 小程序点击「订单」404 | 小程序端未发布新版本 | 联系研发发布小程序新版本 |
| APP 点击「订单」白屏或 404 | 老版本 APP | 卸载重装最新内部包 |
| 后台日志无 `[bottom_nav_migration]` 输出 | DB 里本来就没有 `/orders` 记录 | 属正常，说明无需迁移 |

## 六、回滚预案（仅限运维/研发）

如上线后发现问题，可按此顺序回滚：

1. **代码回滚**：`git revert` 本次提交，重新部署。
2. **数据回滚**：根据后端日志里的迁移明细（id 列表），执行 SQL：
   ```sql
   UPDATE bottom_nav_config SET path = '/orders' WHERE id IN (<日志中的 id 列表>);
   ```
3. **APP 回滚**：重新出一包还原旧路由（因未上架应用商店，无审核成本）。
