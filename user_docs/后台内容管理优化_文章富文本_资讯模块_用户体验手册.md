# 后台内容管理优化（v8）用户体验使用手册

> 版本：v8（2026-04-21）
> 主题：文章富文本化 + 资讯模块新增 + 首页资讯入口

## 一、概述

本次优化围绕"内容管理"做了三件事：

1. **文章升级为富文本**：文章现在支持图文+视频混排，不再是纯文本；
2. **新增「资讯管理」模块**：独立于文章之外的新内容类型，同样支持富文本；
3. **移除「视频管理」模块**：原独立视频管理已下线（对应数据物理删除），视频内容统一作为富文本内嵌到文章/资讯中；
4. **H5 首页新增「最新资讯」入口**：提升资讯曝光，用户能在首页一眼看到最新内容。

## 三、管理后台使用指南

### 3.1 登录与菜单

1. 打开管理后台地址，使用管理员账号登录。
2. 左侧导航「内容管理」下新增了两个入口，并移除了"视频管理"：
   - **文章管理**（已升级为富文本）
   - **资讯管理**（新）
   - **分类管理**（新）

### 3.2 分类管理（/admin/content/categories）

- 用于维护「文章分类」列表。
- 字段：分类名称、排序、是否启用。
- 支持新增、编辑、启停、删除。
  - 删除时会校验：若该分类下存在文章，删除将被拒绝，需先把文章迁到其他分类。
- 默认初始化 6 个分类（首次启动自动插入）：健康科普、营养饮食、运动健身、心理健康、中医养生、疾病预防。

### 3.3 文章管理（/admin/content/articles）

新增/编辑文章时，关键改动：

| 字段 | 说明 |
| --- | --- |
| 标题 | 必填 |
| 封面图 | 单图上传，走腾讯云 COS |
| 摘要 | 列表与推荐展示用，可选 |
| 文章正文（content_html） | **富文本 HTML 字符串**，支持图文视频混排 |
| 分类 | 下拉选择，动态拉取"分类管理"中启用的分类 |
| 标签 | 逗号分隔，多标签 |
| 作者姓名（author_name） | 可选，为空时显示登录用户名 |
| 置顶（is_top） | 开关，置顶文章在列表中优先展示 |
| 发布时间（published_at） | 可选，支持预设发布时间 |
| 状态 | 草稿 / 已发布 / 归档 |

富文本编辑辅助：
- 编辑器上方提供「插入图片」「插入视频」两个按钮：
  - 选择本地图片/视频后自动上传到 COS，并在正文中插入 `<img ...>` / `<video ... controls>` 标签；
  - 也可直接手写 HTML（`<p>`、`<h2>`、`<strong>` 等）。
- 正文内容以 HTML 字符串存储到 `articles.content_html`，同时兼容老的纯文本字段 `content`；老文章无需手动迁移，前端会自动把旧纯文本包一层 `<p>` 后渲染。

### 3.4 资讯管理（/admin/content/news，新）

字段与文章类似，但**没有"分类"**，改用"标签 + 来源"：

| 字段 | 说明 |
| --- | --- |
| 标题 | 必填 |
| 封面图 | 可选 |
| 摘要 | 列表展示用 |
| 正文 | 富文本 HTML |
| 标签 | 逗号分隔多标签，支持**自动联想**（根据历史使用频率） |
| 来源 | 可选，如「人民日报」「新华社」等 |
| 置顶 | 与文章一致 |
| 发布时间 | 可选 |
| 状态 | 草稿 / 已发布 / 归档 |

交互能力：
- 列表支持按「标题 / 标签 / 状态 / 是否置顶」筛选；
- 支持**发布/下架**与**一键置顶**；
- 删除为软删（设置 status=archived）。

### 3.5 角色权限

新增 `content_editor`（内容编辑）角色：
- 可访问「文章管理」「资讯管理」「分类管理」三个模块。
- 不能操作用户、订单、积分、系统设置等敏感模块。

管理员仍然拥有全部权限，不受影响。

## 四、用户端（H5）新体验

### 4.1 首页「最新资讯」板块

- 位置：首页「健康知识」上方新增「最新资讯」卡片组；
- 显示最近 5 条已发布的资讯，含封面、标题、发布日期；
- 点击「更多」跳转到资讯列表页 `/news`；
- 点击单条跳转到资讯详情页 `/news/:id`；
- 下拉刷新会同时刷新资讯数据。

### 4.2 资讯列表页（/news，新）

- 支持搜索框（按标题/标签/摘要关键词），回车触发；
- 无限滚动加载更多；
- 置顶资讯在列表顶部带红色「置顶」徽标；
- 卡片展示：标题、摘要、标签 ≤2、来源、阅读数、发布日期、封面图。

### 4.3 资讯详情页（/news/:id，新）

- 完整渲染富文本 HTML（图、视频、标题、引用等样式已适配，详见 `globals.css` 中的 `.rich-text`）；
- 头部信息：标题、来源、发布时间、阅读数、标签；
- 封面图单独大图显示；
- 操作：点赞、收藏（与文章一致的接口），以及评论；
- 评论实时刷新。

### 4.4 文章详情页（/article/:id）

- 接入真实后端 API（此前为 mock 假数据）；
- 支持富文本渲染：
  - 若后端返回 `content_html` 则直接渲染；
  - 若只有老的 `content` 纯文本，则自动将每行包一层 `<p>`、`**加粗**` 转 `<strong>`；
- 点赞、收藏、评论使用真实后端接口；
- 列表页 `/articles` 同步接入真实后端 API，并移除了不再使用的"视频"Tab。

## 五、数据迁移说明

服务端冷启动时会自动执行 `_migrate_v8_content()`：

1. `articles` 表补加列：`content_html` MEDIUMTEXT NULL、`author_name` VARCHAR(100) NULL、`comment_count` INT DEFAULT 0、`is_top` TINYINT(1) DEFAULT 0、`published_at` DATETIME NULL；
2. `articles.content` 改为允许 NULL；
3. **物理删除 `videos` 表**，并清理 `comments`/`favorites` 中 `content_type='video'` 的记录；
4. `users.role` 枚举追加 `content_editor`；
5. 首次运行时插入 6 个默认文章分类，并写入 `system_configs` 标记 `v8_content_initialized=1`，保证幂等。

所有迁移是**幂等**的，可重复启动不报错。

## 六、接口清单（新/改）

### 6.1 公开接口（无需登录）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/content/article-categories` | 文章分类列表（已启用） |
| GET | `/api/content/news?page=&page_size=&keyword=&tag=` | 资讯分页列表 |
| GET | `/api/content/news/latest?limit=5` | 首页最新资讯 |
| GET | `/api/content/news/{id}` | 资讯详情（会 +1 阅读数） |
| GET | `/api/content/articles` | 支持按 `is_top` 排序 |
| GET | `/api/content/articles/{id}` | 自动把老纯文本兼容为 HTML |

### 6.2 管理端接口（需 `admin` 或 `content_editor` 角色）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET/POST/PUT/DELETE | `/api/admin/article-categories[...]` | 分类 CRUD |
| GET/POST/PUT/DELETE | `/api/admin/news[...]` | 资讯 CRUD |
| POST | `/api/admin/news/{id}/publish` | 发布/下架 |
| POST | `/api/admin/news/{id}/toggle-top` | 一键置顶/取消 |
| GET | `/api/admin/news/tags/suggest?keyword=` | 标签联想（按历史频次） |

### 6.3 已下线接口

- `GET /api/content/videos`
- `GET /api/content/videos/{id}`
- `GET/POST/PUT/DELETE /api/admin/content/videos[...]`
- 搜索类型 `type=video` 改为 `type=news`。

## 八、快速体验路径

1. 登录管理后台 → **分类管理** → 确认 6 个默认分类；
2. **文章管理** → 新建一篇富文本文章（插入 1 张图片 + 1 段视频）→ 发布并置顶；
3. **资讯管理** → 新建一条资讯，标签填 2 个 → 发布；
4. 打开 H5 首页 → 应看到「最新资讯」板块出现刚发布的资讯；
5. 点击资讯 → 详情页正确渲染图文视频；
6. 点击「健康知识」下的文章 → 详情页富文本正常渲染；
7. 点赞、收藏、评论 → 数据持久化。
