## 一、本次升级要点

本次升级聚焦于"中医养生 · 体质测评"模块的**信息架构精简**与**测评体验优化**，共 4 项核心改动：

| # | 优化项 | 升级前 | 升级后 |
|---|--------|--------|--------|
| 1 | **入口精简** | 同时存在"我的档案""历史记录"两个入口，数据内容重叠、职责模糊 | 统一为**「测评记录」**一个入口，集中展示全部体质测评结果 |
| 2 | **命名规范** | "我的档案"容易与"健康档案"混淆 | 使用**「测评记录」**专属命名，避免与健康档案功能混淆 |
| 3 | **视觉统一** | 两个入口卡片样式不一致 | 统一采用**测评对象标签 + 体质类型 + 一句话描述 + 时间**的卡片样式，左侧带体质主色边框 |
| 4 | **流程可感知** | 答题过程无进度提示，提交后直接跳结果页 | 全流程引入 **StepBar 步骤条**：① 答题 → ② 选择对象 → ③ AI 分析，对齐"健康自查"的体验 |

此外，后端对 `GET /api/tcm/diagnoses` 和 `GET /api/tcm/diagnoses/{id}` 返回增加了 `member_label` 字段，统一按 **"姓名（关系）"** 格式输出（例如 `张三（本人）`、`李四（妈妈）`、`赵六（未设置）`），避免各端自拼字符串引起不一致。

## 三、完整体验路径

1. **进入**：首页 → 中医养生 → TCM 主页
2. **功能分区**：顶部可选「体质测评 / 舌诊 / 面诊」三项
3. **发起测评**：点击「体质测评」→ 进入答题页，顶部 **StepBar 显示第 1 步：答题**
4. **答题**：回答 9 题（完全不符 / 有点符合 / 比较符合 / 非常符合）
5. **选择咨询人**：答完弹出成员选择，**StepBar 前进到第 2 步：选择对象**
6. **AI 分析**：确认后提交，**StepBar 前进到第 3 步：AI 分析**，Loading 提示"AI 正在分析体质..."
7. **查看结果**：自动跳转 6 屏测评结果详情（体质概览、体质特征、饮食建议、运动建议、注意事项、回顾记录）
8. **回看历史**：返回 TCM 主页 → 下方「测评记录」卡片列表 → 点击任意卡片 → 再次进入 6 屏结果详情

## 四、测评记录卡片样式说明

```
┌──────────────────────────────────────────────────┐
│ ┃  🌿   气虚质   [李四（妈妈）]                     │
│ ┃       体质虚弱，建议补气益气调理                    │
│ ┃       2026-04-20 10:30                           │
└──────────────────────────────────────────────────┘
  ↑ 左边框：体质类型对应主色（平和质绿、气虚质蓝等）
```

- **Emoji**：由后端按体质类型返回 `persona_emoji`
- **体质类型颜色**：后端 `persona_color` 十六进制，用于左边框和标签
- **Member 标签**：格式严格为 `姓名（关系）`
- **一句话描述**：后端 `one_line_desc` 字段
- **点击跳转**：`/tcm/result/{id}`（H5）、`pages/tcm-constitution-result/index?id=xxx`（小程序）、`ConstitutionResultScreen`（Flutter）
