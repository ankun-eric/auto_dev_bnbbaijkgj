## 功能简介

本次修复了预约时段选择模块的两处关键 Bug：

1. **已过时段未禁用**：当用户选择"今天"作为预约日期时，已完全结束的时段（结束时间 ≤ 当前时间）现在会置灰不可选择
2. **名额已满仍可下单**：当某个时段的预约名额已满时，前端现在会展示"已约满"标签并禁止点击，后端同时增加了容量校验作为双重保险

## 使用说明

### 预约商品下单流程

1. 在 H5 商城中浏览预约类商品（如"艾灸疗养服务"）
2. 点击商品进入详情页，点击"立即购买"进入确认订单页面
3. 在"预约日期"区域选择日期
4. 在"预约时段"区域查看各时段状态：
   - **正常可选时段**：绿色或白色背景，可正常点击选择
   - **已过时段**（仅当选择"今天"时）：灰色背景、灰色文字，不可点击
   - **已约满时段**：灰色背景、灰色文字，显示"已约满"标签，不可点击
5. 选择可用的时段后，完成其他信息填写，点击"提交订单"

### 时段状态判断规则

| 情况 | 时段状态 | 示例 |
|------|---------|------|
| 预约日期为今天，时段已完全结束 | 置灰不可选 | 当前 14:30，时段 09:00-10:00 → 灰色 |
| 预约日期为今天，时段正在进行中 | 正常可选 | 当前 14:30，时段 14:00-15:00 → 可选 |
| 预约日期为今天，时段未开始 | 正常可选 | 当前 14:30，时段 15:00-16:00 → 可选 |
| 预约日期为明天及之后 | 全部正常可选 | 所有时段均为可选状态 |
| 时段名额已满（任何日期） | 置灰+已约满 | capacity=1 且已有1个有效订单 → 灰色+"已约满" |

### 切换日期刷新

每次切换预约日期时，系统会自动重新查询该日期各时段的可用名额，实时更新时段的可选/已约满状态。

## 注意事项

1. **三端一致**：H5 Web、小程序、Flutter APP 三端的时段禁用逻辑和表现一致
2. **实时更新**：每次切换预约日期时，系统会自动刷新各时段的可用名额状态
3. **后端兜底**：即使前端因网络异常未能正确显示禁用状态，后端在提交订单时仍会进行校验：
   - 已过时段提交会返回错误："该时段已过，请选择其他时段"
   - 名额已满提交会返回错误："该时段名额已满，请选择其他时段"
4. **名额计算规则**：每天每个时段独立计算，已取消的订单不占用名额
5. **降级处理**：如果网络异常导致时段可用性接口请求失败，前端不会阻止用户操作，依赖后端兜底校验保障数据正确性
