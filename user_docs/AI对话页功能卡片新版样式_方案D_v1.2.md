## 功能简介

本次升级把 AI 对话页中所有 AI 回复消息里嵌入的**「功能引导卡片」**（点击可跳转到健康自查、舌象采集、中医体质辨识等功能页）统一切换为**方案 D 新版样式**（图标左 + 标题右 · 宾尼天蓝），与产品参考稿做到像素级一致。

### 核心变更点

1. **顶部品牌色条**：每张卡片顶部新增 4px 高的渐变色条（`#0EA5E9 → #38BDF8`），作为视觉锚点。
2. **新头部排版**：图标 48×48 在左、主标题 20px 加粗在右，横向居中排列，gap 14px。
3. **副标题色块化**：副标题不再是普通灰字，而是浅蓝底色块（`#F0F9FF`）+ 左侧 3px 蓝色竖线，信息层级更清晰。
4. **按钮副说明文字**：12px 灰字居中显示在主按钮正上方，让运营在后台填写的「按钮副说明」字段真正发挥价值。
5. **主按钮统一化**：所有卡片的主按钮文案统一硬编码为「**开始**」，全宽 48px 高，圆角 12px，纯色 `#0EA5E9`。运营无需再到后台配置每个按钮的文案。
6. **取消封面图模式**：本期所有卡片走「图标 + 标题」头部样式，封面图字段（`cover_img`）保留但不展示，避免历史样式碎片化。
7. **一刀切刷新**：历史会话回看时也用新样式渲染，不引入版本字段，长期只维护一套样式。
8. **后台预览能力**：「功能按钮管理」表单新增「预览效果」按钮，弹出 375×667 手机框，实时联动表单字段，1:1 还原 H5 真机效果。

## 使用说明

### 一、运营在后台配置新版功能卡片

1. **进入后台**：点击 [后台·功能按钮管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/function-buttons/) 链接进入。
2. **新建或编辑按钮**：点击列表上方的「新增按钮」或某行的「编辑」。
3. **填写关键字段**（与新版卡片对应关系）：
   - **按钮图标（Emoji）**：会作为卡片头部左侧 48×48 的图标。
   - **卡片标题**：会作为卡片头部右侧的主标题（20px 加粗，过长自动单行省略号）。
   - **卡片副标题**：会作为浅蓝色块内的说明文字（14px / 行高 1.6）。如果留空，整个色块不渲染，按钮区会自然上推。
   - **按钮副说明文字**：会作为主按钮上方居中的小灰字（12px / #64748B）。例如「预计耗时 3-5 分钟 · 数据加密保护」。
4. **预览效果**：点击编辑弹窗右上角的「**预览效果**」按钮，弹出 375×667 手机框，实时显示当前表单字段填写后的卡片真机效果。**修改任意字段，预览即刷新，无需保存**。
5. **保存**：点击「确定」保存按钮配置。新版卡片将在所有 H5 端 AI 对话页（包括历史会话回看）立即生效。

### 二、终端用户在 H5 端体验新版卡片

1. **打开 H5 AI 对话页**：访问 [H5 AI 对话首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home/)。
2. **触发功能卡片**：
   - 直接点击首页的功能按钮（如「健康自查」「中医体质辨识」），或
   - 在对话框中说出关键词（如「我想做体质测评」），AI 会自动弹出对应的功能引导卡片。
3. **查看新版样式**：每张卡片自上而下包含：
   - **顶部色条**：4px 高的蓝色渐变品牌锚点。
   - **头部**：左侧大号图标 + 右侧 20px 加粗标题。
   - **副标题色块**：浅蓝底 + 左侧蓝竖线的说明文字（如有）。
   - **按钮副说明**：12px 灰字居中（如有）。
   - **主按钮「开始」**：全宽蓝色按钮，点击进入对应功能。
4. **点击操作**：
   - 点击主按钮「开始」即可进入对应功能流程（如打开健康自查抽屉、跳转目标页等）。
   - 整张卡片也是可点击的，点击卡片任意空白区域等同于点击主按钮。

## 注意事项

- ✅ **`button_text` 字段已停用**：新版样式中主按钮文案统一为「**开始**」，**后台填写的 `button_text` 字段不再生效**。这是为了保证全端视觉一致性，运营无需再为每个按钮单独配置文案。
- ✅ **`cover_img` 字段已停用**：新版样式取消封面图模式，所有卡片走「图标 + 标题」头部。后台填写的封面图 URL 会被前端忽略。如果未来需要恢复封面图模式，需要单独立项。
- ✅ **历史会话也用新样式**：用户回看历史对话时，旧卡片数据也会用新方案 D 样式渲染。**无版本字段，一刀切刷新**。
- ⚠️ **主标题过长策略**：主标题超长时**单行省略号**，鼠标悬浮（H5 端）会通过 `title` 属性显示完整文本。建议运营把卡片标题控制在 12 个汉字以内，效果最佳。
- ⚠️ **副标题缺失降级**：副标题留空时，整个色块不渲染，按钮区上推；按钮副说明缺失时，该行不渲染，主按钮直接贴在副标题色块下方。
- ⚠️ **图标缺失降级**：图标 emoji 留空时，默认显示 📋。

## 验收基准

- **像素级视觉基准**：严格按照参考稿 `3c87314dcaf9435793c8021dd643baa4.html`（方案 D · 图标左 + 标题右 · 宾尼天蓝）。
- **关键尺寸/颜色**（自上而下）：
  - 卡片：圆角 16px / 阴影 `0 4px 24px rgba(15,23,42,0.08)` / padding `24px 22px 22px`
  - 顶部色条：高 4px / `linear-gradient(90deg, #0EA5E9 0%, #38BDF8 100%)`
  - 头部：flex 横向 / align-items: center / gap 14px / margin-bottom 14px
  - 图标：48×48 / flex-shrink:0 / emoji 字号 28px
  - 标题：20px / weight 700 / `#0F172A` / 单行省略号
  - 副标题色块：bg `#F0F9FF` / border-left 3px `#0EA5E9` / radius `0 8px 8px 0` / padding `12px 14px` / 14px / 行高 1.6 / `#334155`
  - 按钮副说明：12px / `#64748B` / center / margin-bottom 10px
  - 主按钮：全宽 / 48px 高 / 圆角 12px / `#0EA5E9` 纯色 / 白字 16px / weight 600 / letter-spacing 1px / 文案「开始」
