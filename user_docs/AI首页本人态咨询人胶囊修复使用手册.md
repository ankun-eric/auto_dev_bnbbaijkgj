## 二、功能介绍

### 2.1 什么是「咨询人胶囊」

每条 AI 回答的最上方会出现一条**浅灰色圆角胶囊**，作用是清晰告知用户：本次回答结合的是哪位家庭成员的健康档案。

```
┌──────────────────────────────────────┐
│ 👤  本次回答结合 本人 的档案        ⌄ │
└──────────────────────────────────────┘
```

- 左：小人图标
- 中：「本次回答结合 {成员名} 的档案」文案
- 右：展开 / 收起箭头

点击胶囊任意区域可展开 7 项健康档案字段（性别、年龄、身高、体重、既往病史、过敏史、长期用药）。

### 2.2 本人态特殊点

| 场景 | 表现 |
|---|---|
| 折叠态显示文字 | 永远是「本次回答结合 **本人** 的档案」，**不会显示真实姓名** |
| 展开后查看完整档案 | 会显示真实数据（包括真实姓名、性别、年龄等）—— 用户主动点击展开 = 知情同意 |
| 档案未完善 | 顶部显示「档案完整度 X%，点击补全 ›」，点击跳转到健康档案页 |
| 档案已完整 | 顶部显示「档案已完整 ✓」 |

> 🔒 **隐私保护**：折叠态文案永远显示"本人"二字，即使在公开场合截图分享也不会泄露真实姓名/手机号/账号昵称。

## 五、注意事项

### 5.1 隐私

- 折叠态显示的"本人"二字是**前端固定文案**，与后端返回的真实姓名无关
- 即使后端返回了真实姓名（例如"张三"），折叠态也只会显示"本人"
- 真实姓名仅在用户主动点击展开后才会显示在 7 项字段中（属知情同意）

### 5.2 兼容性

本次修复在以下环境验证通过：

- ✅ iOS Safari
- ✅ Android Chrome
- ✅ 微信内置浏览器

### 5.3 接口行为

后端接口 `GET /api/v1/consultant/0/profile_card` 在本人态做了兜底：

| 场景 | 响应 |
|---|---|
| 已登录 + 本人档案完整 | 200 + 7 项真实数据 + `is_self=true` + `percent>0` |
| 已登录 + 本人档案空白 | 200 + 基础结构 + `nickname="本人"` + `percent=0` |
| 已登录 + 本人 FamilyMember 未建立（罕见） | 200 + 兜底基础结构 + `nickname="本人"` |
| 未登录 | 401 |
| 非本人成员 ID 不存在（id > 0） | 404 |

> **关键**：本人档案空白时**绝不返回 404 / 空对象**，避免前端 `!data` 走 `return null` 分支导致胶囊消失（即历史 Bug 的根因之一）。
