## 一、功能简介

把门店订单管理的视角，从原来的**「列表式」**升级为**「时段看板式」**，让店长 / 收银员可以**一眼看清**：
- 当天哪些时段已经被预约满了
- 当天哪些时段还有空位可以接客
- 当天**已经核销**了多少单、收了多少钱

> 💡 **本次升级最大变化**：所有口径**统一改为绝对值**，废除了原来的「占用率 %」字段。
> 老看板里看到的「12:00-14:00 占用 75%」这种相对值，从此不再出现。
> 取而代之的是：**预约 N / 已核 X / 已核金额 ¥Z** 三个直接的数字。

### 三视图一览

| 视图 | 形态 | 适用场景 |
|------|------|----------|
| **日视图** | 9 宫格（每格对应 2 小时时段） | 看单天每个时段的预约/核销情况，最常用 |
| **周视图** | 横向 7 列日期卡片 | 看一周整体预约和已核销趋势 |
| **月视图** | 标准月历 + 点击日期弹出当日详情 | 看整月哪些日期最忙、做月度复盘 |

### 默认行为

- 进入**「订单管理」**菜单时，默认会自动跳到**「预约看板」**页（这是 PRD-02 的核心默认行为）
- 进入预约看板后，默认显示**「日视图」**
- 浏览器会**记住**你上次选择的视图（日/周/月）和**列表/看板偏好**，下次再进会保持
- 如果你点了**「切换到列表视图」**按钮，下次进入订单管理就会停留在列表页（直到你再次主动切回看板）

## 三、操作指引

### 3.1 进入预约看板

1. 打开 [商家管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/login/) 并登录
2. 在左侧导航点击 **「商品体系 → 订单管理」**
3. 系统会自动跳转到 **「预约看板」** 页面（如果你之前选择过列表视图，则会停留列表，可点右上角「切换到预约看板」）

### 3.2 日视图（9 宫格）

页面会展示一个 **3×3 的九宫格**，每个格子对应一个 2 小时的时段：

```
┌─────────────┬─────────────┬─────────────┐
│ 06:00-08:00 │ 08:00-10:00 │ 10:00-12:00 │
├─────────────┼─────────────┼─────────────┤
│ 12:00-14:00 │ 14:00-16:00 │ 16:00-18:00 │
├─────────────┼─────────────┼─────────────┤
│ 18:00-20:00 │ 20:00-22:00 │ 22:00-24:00 │
└─────────────┴─────────────┴─────────────┘
```

每个格子显示三组关键数据：

- **预约：N** — 这个时段一共有多少个预约（含已核销）
- **已核：X** — 这个时段已经核销了多少个
- **已核金额：¥Z** — 这个时段已经核销的订单金额合计（**只算已核销的订单**）

#### 视觉规则

- 已核率高的格子，**底色越绿**（颜色越深表示越忙、回款越好）
- **当前正在进行的时段**会有**绿色边框**高亮，并标记「进行中」
- 完全空闲的格子显示浅灰色 +「暂无预约」
- 凌晨 0:00-6:00 不开放，9 宫格不展示这段时间（如有历史脏数据会在底部显示「⚠️ 凌晨脏数据 N 条」提示）

#### 点击格子 → 抽屉

点击任意一个九宫格 → 页面右侧会**滑出抽屉**，显示该时段的订单列表。每个订单卡片包含：

- 客户姓名 + 完整 11 位手机号（**不脱敏**，方便商家直接联系）
- 服务项目名
- 订单状态（已核销 / 待核销 / 已取消 等）
- **三个操作按钮**：
  - **核销** — PC 端置灰，提示「请到手机端操作」（核销动作收口手机端，避免误操作）
  - **详情** — 跳转到该订单的详情页
  - **📞拨打** — 桌面浏览器自动唤起本机拨号软件 + 复制手机号到剪贴板（双保险，方便手机/电脑都能用）

### 3.3 周视图（7 列卡片）

横向 7 列日期卡片，每张卡片显示：

- `预约：N`
- `已核：X`
- `¥Z`（金额，仅统计已核销订单）

> 💡 **PRD-02 重要变化**：周视图**已删除原「占用 %」字段**，全部改为绝对数字。

点击任意一天的卡片，会自动切换到**「日视图」**展示这天的 9 宫格。

### 3.4 月视图（月历 + 弹窗）

标准月历，每天的格子显示：
- 日期数字
- 该日的预约数（如有）
- 占用色块（已核率越高，绿色越深）

#### 点击日期 → 当日详情弹窗

点击月历上的任意一天，会弹出**「当日订单详情弹窗」**，左右两栏布局：

- **顶部**：当日汇总（总预约 N / 已核 X / 已核金额 ¥Z）
- **左栏**：上午时段（06:00-12:00）的订单卡片，按时段升序排列
- **右栏**：下午+晚上时段（12:00-24:00）的订单卡片，按时段升序排列
- **右上角**：关闭按钮

每张订单卡片含 **6 字段**：
- 预约时段（如 `10:00-12:00`）
- 订单状态（如 `已核销`）
- 客户姓名 + 完整 11 位手机号
- 服务项目名
- 订单金额（如 `¥588`，**整数、不带千分位**）

### 3.5 切换视图

页面顶部有一组按钮：

- 单选按钮组：**[日视图] [周视图] [月视图]** —— 切换三视图
- 日期选择器：选择具体日期 / 周 / 月
- **[今天]** —— 一键回到当前日期
- **[刷新]** —— 重新拉取数据
- **[切换到列表视图]** —— 回到旧版列表式订单管理

> 💡 浏览器会记住你的选择，下次进来还是这个视图。

## 四、口径与规则说明

| 规则 | 说明 |
|------|------|
| **绝对值口径** | 9 宫格、周视图、月视图统一去掉「占用率 %」，全部用绝对值（预约 N / 已核 X / ¥Z） |
| **已核金额口径** | **仅统计已核销的订单**，未核销的订单（待核销 / 已取消 / 已改期）一律不计入 |
| **手机号显示** | 商家端看到的客户手机号**不脱敏**，11 位完整展示（方便商家直接电话） |
| **金额格式** | `¥` + 整数，**不带千分位**，如 `¥1280`、`¥588` |
| **凌晨脏数据** | 0:00-6:00 不开放预约，9 宫格不展示；如有历史脏数据，仅在列表视图可见 |
| **跨日订单** | 22:00-次日 00:00 的订单按起始时间归到第 9 段（22:00-24:00） |
| **核销操作** | PC 端**全部置灰**，必须到手机端操作核销，避免误操作（PRD-05 「核销动作收口手机端」） |

## 五、注意事项

1. **首次访问可能加载稍慢**：日视图 9 宫格首次加载约 1-2 秒（聚合该日所有订单）
2. **视图记忆是浏览器级别**：换电脑、换浏览器、清除 LocalStorage 后会回到默认（自动进看板 + 日视图）
3. **数据非实时**：看板数据是当前请求时刻的快照，新订单进来需要点「刷新」按钮看最新数据
4. **多门店管理**：管理员（admin / super_admin）默认看到**所有门店**汇总；商家角色只能看到所属门店
5. **手机端如何操作**：H5 端的同名页面已具备同样能力（响应式适配），手机扫码登录后访问相同 URL 即可

## 六、常见问题

**Q1：为什么我点「订单管理」直接进了看板，找不到原来的列表？**
A：这是 PRD-02 的预期行为。如果想回列表，点页面右上角**「切换到列表视图」**按钮，浏览器会记住你的选择，下次再进就停留在列表页。

**Q2：周视图为啥没有「占用率 %」了？**
A：PRD-02 R-02-01 规则：全部改为绝对值口径，废除原「占用率 %」字段，避免分母选择带来的歧义（用座位数还是商品时段配额？）。

**Q3：「已核金额」和「订单金额」差别？**
A：**已核金额**只统计已核销的订单，待核销订单的金额**不计入**。这样商家看到的是**真实回款**，而非未实现的应收账款。

**Q4：抽屉里点「📞拨打」按钮没反应？**
A：桌面浏览器一般不会内置电话功能，按钮会**自动复制手机号到剪贴板**+尝试唤起 `tel:` 协议。手机浏览器或装了电话软件的电脑可以直接拨号。

**Q5：可以从看板直接核销吗？**
A：不可以。PC 端的「核销」按钮一律置灰，提示「请到手机端操作」。这是为了避免误操作和保证现场体验（核销人需要现场扫码）。
