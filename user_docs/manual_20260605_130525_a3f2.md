## 功能简介

本次更新对健康档案模块中的家庭成员状态体系进行了以下三项优化：

### 1. 清理 `removed` 旧状态兼容代码

V3 升级前旧 DELETE 接口曾向 `FamilyMember` 表写入 `removed` 枚举值。V3 后已统一使用 `deleted` 或 `unbound`，但代码中仍残留大量兼容逻辑。本次将所有查询过滤条件中的 `removed` 兼容代码全部移除，降低了维护成本。

### 2. 修正被守护人退出关系的状态

之前被守护人主动退出守护关系时，`FamilyMember` 的状态被错误地写为 `deleted/self_deleted`（已删除），导致用户以为档案被删除。现已修正为 `unbound/unbinded`（未绑定/已解绑），退出守护关系后档案仍然保留，用户可以随时重新发起邀请建立守护关系。

### 3. 新增账号注销功能

在「设置 → 账号安全 → 账号注销」页面，用户现在可以正式注销账号。注销前系统会自动检查该账号下是否还有非本人的在册家庭成员，如有则拦截注销并提示先清理家庭成员，避免产生孤儿数据。

## 使用说明

### 一、被守护人退出守护关系

当您作为被守护人需要退出守护关系时：

1. 打开 H5 端，进入「健康档案」模块
2. 在家庭成员列表中找到对应的守护人卡片
3. 点击退出守护关系按钮
4. 确认退出后，该守护人的卡片状态将显示为「已解绑」，但卡片不会消失
5. 您可以随时重新向该用户发起守护邀请，重新建立守护关系

**对比之前的改动**：退出后卡片保留在列表中，状态变为「已解绑」，而不是被删除消失。

### 二、账号注销

> ⚠️ 账号注销为不可逆操作，请谨慎使用。

#### 操作步骤

1. 打开 H5 端，进入「设置」页面
2. 点击「账号安全」
3. 进入「账号注销」页面
4. 输入收到的短信验证码
5. 点击「确认注销」按钮

#### 注销前的自动校验

系统会在注销前自动检查您是否还有其他家庭成员：

- **校验通过**：如果您名下没有非本人的在册家庭成员，系统将执行注销操作：
  - 您的账号将被标记为已注销
  - 所有登录凭证将被清除
  - 您将无法再登录该账号

- **校验拦截**：如果您名下还有 N 位家庭成员，系统会提示：
  > 您当前还有 N 位家庭成员，请先在「健康档案-家庭成员」中删除所有家庭成员后再注销账号。

#### 如何清理家庭成员

1. 进入「健康档案」→「家庭成员」
2. 逐一删除非本人的家庭成员卡片
3. 确认所有家庭成员已删除后，返回「账号注销」页面重新操作

#### 错误提示说明

| 错误信息 | 含义 | 解决方法 |
|---------|------|---------|
| "验证码错误或已过期" | 输入的验证码不正确或已超过有效期 | 重新获取验证码后输入 |
| "您当前还有 N 位家庭成员..." | 名下还有未删除的家庭成员 | 先删除所有家庭成员后再操作 |
| "未登录" | 登录状态已失效 | 重新登录后再尝试 |

## 注意事项

1. **账号注销不可逆**：一旦注销成功，账号将被永久标记为已注销，无法恢复登录。请确认已备份所有重要数据。

2. **家庭成员清理顺序**：注销前必须删除所有非本人的家庭成员。本人档案是注册时自动创建的，用户无法手动删除，也不影响注销。

3. **守护关系退出与删除的区别**：
   - **退出守护关系**：双方关系解除，档案保留，可重新建立关系
   - **删除家庭成员**：档案被标记为已删除，不在列表中显示

4. **后端优化仅影响服务端**：本次更新为后端代码优化，前端页面无需升级即可体验。

5. **默认测试账号**：如需登录体验，可使用 `admin / admin123` 测试账号。
