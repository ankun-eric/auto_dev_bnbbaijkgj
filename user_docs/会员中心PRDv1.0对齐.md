## 3. 各功能操作步骤

### 3.1 免费会员额度配置

> 入口：管理后台 → 会员管理 → **免费会员额度配置**

1. 进入页面后看到 4 个输入框：
   - **守护人数量**（默认 3）—— 该等级用户最多能添加几位被守护人
   - **AI 外呼提醒（次/月）**（默认 5）—— 每月可触发的 AI 外呼次数；填 `-1` 表示不限
   - **紧急 AI 呼叫（次/月）**（默认 3）—— 每月可触发的紧急 AI 呼叫次数；填 `-1` 表示不限
   - **被管理人数上限**（默认 3）—— 该等级用户最多能被几位守护人添加
2. 修改后点击「保存」，提示"保存后所有普通会员的额度立即按新配置生效"
3. 顶部蓝色 Alert 提示「`-1` = 不限」

### 3.2 付费会员套餐管理

> 入口：管理后台 → 会员管理 → **付费会员套餐管理**

#### 新增套餐
1. 点击右上角「新增套餐」按钮
2. 在弹窗中填写：
   - 套餐名称（必填）
   - 套餐说明（选填，textarea）
   - 月价（30 天）/ 年价（365 天）—— **留空表示不展示对应按钮**
   - 守护人数量、AI 外呼提醒、紧急 AI 呼叫、被管理人数上限 —— 4 项均必填，`-1` 表示不限
   - 商城折扣（0~1，选填）—— **仅后台可配，用户端不展示**
   - 是否启用（默认开）
   - **是否推荐（开启后该套餐在用户端显示金色描边 + 推荐角标）**
3. 点击「确定」保存

#### 列表显示
- 推荐套餐整行 **金色高亮 + 黄色"推荐"角标**
- 操作列支持「编辑 / 停用|启用 / 删除」
- 「删除」按钮在该套餐有历史订单引用时会被禁用，只能"停用"

### 3.3 用户管理 - 会员信息卡片

> 入口：管理后台 → 用户管理 → 点击任一用户的「详情」

1. 用户详情抽屉中可看到新增的「会员信息」卡片，展示：
   - 当前会员等级（如「健康会员」/「免费用户」）
   - 有效期（如 `2026-06-25（剩 30 天）`）
   - 守护人：`已用 / 上限`
   - 被守护人：`已用 / 上限`
   - AI 外呼提醒：`剩余 / 上限`
   - 紧急 AI 呼叫：`剩余 / 上限`
2. 卡片底部 3 个操作按钮：
   - **手动延长会员期**：弹窗输入天数（默认 30），确认后会员期 +N 天
   - **手动重置额度**：当月已用额度归零，立即生效
   - **手动降级**：将用户立即降为免费会员，所有付费权益清零（需二次确认）

### 3.4 订单管理 - 履约类型筛选 / 角标 / 退款

> 入口：管理后台 → 商品系统 → **订单管理**

1. 列表上方筛选条件新增 **「履约类型」下拉**：
   - 全部 / 到店 / 配送 / **权益服务（virtual）** / 上门
2. 选中「权益服务」后，列表只显示会员费订单（virtual）
3. 任意 virtual 订单行尾会显示 **紫色「权益服务」角标**
4. 点击「详情」打开订单抽屉：
   - 顶部正常订单字段保留
   - 新增「会员费订单专属信息」卡片，展示：
     - 套餐名称（如「健康会员（年）」）
     - 周期：月卡 / 年卡
     - 生效日 / 到期日
     - 支付方式 / 实付金额
5. 抽屉底部 / 列表操作列新增「退款」按钮：
   - 点击后弹出二次确认
   - 确认后：订单状态变为 refunded、用户立即降级为免费会员、当月额度归零

### 3.5 H5 用户端会员中心联动

> 入口：H5 → 个人中心 → 我的会员，或访问 [/member-center](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center)

- 顶部「我的权益」3 张实卡按新字段渲染：
  1. 守护人数量（`max_managed`）
  2. AI 外呼提醒（`ai_outbound_call_count`）
  3. 紧急 AI 呼叫（`emergency_ai_call_count`）
- 末尾固定 1 张「即将推出」占位卡
- 套餐列表中，后台开启「推荐」的套餐显示 **金色描边 + 推荐角标**
- `discount_rate` 商城折扣字段 **不会** 在用户端任何位置展示

## 4. 注意事项

1. **字段命名一次性物理重构**：旧字段（`plan_code`、`ai_call_quota`、`ai_alert_quota`、`ai_remind_quota`、`max_guardians`、`benefits_desc`、`point_multiplier`、`price_monthly`、`price_yearly`）已从数据库 **物理删除**，老接口字段也已下线。如有外部系统集成 `plan_code` 等老字段，请联系开发同步改造。
2. **`-1 = 不限`**：免费额度配置页与套餐管理表单中，所有"数量类"字段填 `-1` 表示不限。
3. **推荐套餐默认关闭**：新增/迁移过来的套餐默认 `is_recommended=false`，需管理员在后台手动勾选「推荐」才会在用户端显示金色描边。
4. **会员费订单退款不可逆**：退款后用户立即降级、额度归零，请二次确认后再操作。
5. **数据迁移已自动完成**：项目启动时 `schema_sync.py` 会自动执行 ALTER 迁移与数据搬运（`ai_alert_quota → emergency_ai_call_count`、`ai_remind_quota → ai_outbound_call_count`、`price_monthly → price_month`、`max_guardians → max_managed`），无需手动操作。

## 5. 常见问题

**Q1：原来「AI 异常告警额度」对应的数据去哪了？**  
A：自动迁移到「紧急 AI 呼叫（`emergency_ai_call_count`）」字段，老列已物理删除。

**Q2：套餐编码 `plan_code` 还能用来关联订单吗？**  
A：不能。系统统一通过套餐 `id` 主键关联，所有内部业务零依赖 `plan_code`。

**Q3：我配置了「商城折扣」，为什么用户端看不到？**  
A：按 PRD v1.0 决策，`discount_rate` 仅后台可配置，用户端不展示。该字段未来会用于商城下单时计算折扣价。

**Q4：用户被「手动降级」后还能再升回付费等级吗？**  
A：可以。让用户重新购买任意付费套餐即可，付费立即生效。
