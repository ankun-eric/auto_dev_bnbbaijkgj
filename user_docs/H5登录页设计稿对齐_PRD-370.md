## 功能简介

本次升级针对 PRD-363 上线后用户反馈的「H5 登录页线上效果与设计稿严重不符」问题，按附件最终设计稿做了**6 大类**精确对齐修复：

1. **主色升级到 iOS 标准绿 `#34C759`**：由旧版 `#2fb56a` 全面切换到 `#34C759`，与设计稿色号 1:1 对齐。
2. **登录按钮三段渐变**：`#4AD97A → #34C759 → #2BD4C4`（浅绿 → 主绿 → 青色），按钮高度 56px、圆角 28px 全胶囊。
3. **上绿下白分屏卡片式布局**：顶部绿色渐变品牌区 + 白色圆角卡片向上覆盖（圆角 24px），主标题"宾尼小康"32px 加粗白字 + 副标题"AI 健康管家 · 您的私人健康助手"。
4. **删除"欢迎回来 👋"等冗余问候语**：白色卡片顶部直接进入手机号输入框（`padding-top: 32px`），不再插入任何欢迎文案。
5. **协议二次确认弹窗 + 半屏抽屉**：未勾选协议点击"登录/注册"时弹出标题为"服务协议及隐私保护"的居中弹窗，双按钮"不同意 / 同意"；点击协议名打开 80% 屏高的底部抽屉展示协议详情，关闭后弹窗保留。
6. **输入框聚焦视觉反馈**：灰底胶囊在聚焦时变白底 + 2px `#34C759` 描边 + 4px 浅绿光晕，提升交互感。

后端新增 `/api/config/login_ui_version` 远程开关（默认 `v2` 新版），如需回滚到旧版 UI 可一键切换为 `v1`，无需重新发版。

## 使用说明

### 一、H5 登录页操作步骤

1. **进入页面**：在手机或电脑浏览器中打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login/)。
2. **观察布局**：上半部分是绿色渐变品牌区（含 LOGO 圆形托盘 + "宾尼小康" + "AI 健康管家 · 您的私人健康助手"副标题），下半部分是白色圆角卡片，**卡片顶部不再出现"欢迎回来"等多余文字**。
3. **输入手机号**：在第一个输入框输入 11 位手机号，输入框自动变为白底 + 绿描边 + 浅绿光晕。
4. **获取验证码**：点击右侧"获取验证码"按钮（绿色文字）→ 短信发送后变为 "60 s" 倒计时；输入收到的验证码。
5. **登录**：点击下方三段渐变（浅绿→主绿→青色）的"登录 / 注册"全圆角按钮。
6. **协议交互**：
   - **场景 A：已勾选协议**：直接跳转到首页或健康档案完善页。
   - **场景 B：未勾选协议**：弹出居中弹窗"服务协议及隐私保护"，双按钮「不同意 / 同意」。
     - 点击 [《用户服务协议》] 或 [《隐私政策》]：弹出 80% 屏高的底部抽屉，加载协议详情。可下拉关闭或点击右上角 ✕ 关闭，**关闭后弹窗仍然保留**。
     - 点击「同意」：自动勾选页面复选框 + 立即发起登录请求 + 登录成功跳转首页（一气呵成）。
     - 点击「不同意」：弹窗静默关闭，停留登录页，不弹任何 toast。

### 二、登录按钮置灰条件

当手机号未填满 11 位 或 验证码不足 4 位时，「登录 / 注册」按钮自动呈 50% 透明置灰，不可点击。

### 三、回滚开关（运营/运维可见）

后端新增 `GET /api/config/login_ui_version` 接口，默认返回 `{"version":"v2","rollback_supported":true}`。如线上出现严重问题，运营可通过管理端将 `version` 切回 `v1` 触发四端回滚到旧版 UI，无需重新发版。

### 四、协议详情独立页面

协议详情页同时支持以独立 URL 访问（不通过抽屉），方便分享或备案：
- [用户服务协议](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/legal/service-agreement/)
- [隐私政策](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/legal/privacy-policy/)

## 注意事项

1. **缓存清理**：H5 升级使用了 Next.js chunk hash，浏览器会自动加载新 chunk；如发现仍是旧版本，请下拉刷新或清空浏览器缓存。
2. **iOS 安全区域**：H5 顶部绿色渐变已通过 `viewport-fit=cover` + `env(safe-area-inset-top)` 适配 iPhone 灵动岛/刘海屏，绿色背景延伸到状态栏；底部按钮区按 `env(safe-area-inset-bottom)` 自动避让 Home Indicator。
3. **协议详情 iframe 加载策略**：H5 端弹窗中的协议抽屉通过 `iframe` 加载 `/legal/service-agreement` 与 `/legal/privacy-policy` 页面；首次打开可能略有加载延迟，属于正常现象。
4. **回滚预案**：本次新版 UI 默认开启（`version=v2`），如线上核心登录接口错误率 > 1% 或转化率下降 > 15%，可通过 `PUT /api/admin/config/login_ui_version` 切回 `v1` 触发降级，**全程无需重新发版**。
5. **App 端代码已改动但本次未远程构建**：Android/iOS 端代码（`flutter_app/lib/screens/login_screen.dart`）已对齐主色与弹窗，可在 GitHub 仓库 Actions 中手动触发 `android-build.yml` / `ios-build.yml` 获取安装包。
6. **后端无破坏性变更**：本次仅在 `backend/app/main.py` 注册了一个新路由 `login_ui_config.router`，未触碰任何既有接口（`/api/auth/sms-code`、`/api/auth/sms-login`、`/api/auth/register-settings`、`/api/settings/logo` 等保持原样），不影响其他业务。
7. **协议详情正文**：本次协议正文为通用版本（v1.0），如需根据实际运营要求调整条款细节，可直接修改 `h5-web/src/app/legal/service-agreement/page.tsx` 与 `privacy-policy/page.tsx`。
