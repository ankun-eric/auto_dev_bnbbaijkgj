## 功能简介

本次更新为 bini-health 健康管理平台新增了以下功能：

1. **体检报告解读明细**：在管理后台 AI管理 菜单下新增独立页面，管理员可按业务场景查看所有体检报告 OCR 解读记录，包含统计卡片、多条件筛选、详情弹窗
2. **拍照识药记录明细**：在管理后台 AI管理 菜单下新增独立页面，管理员可查看所有拍照识药记录，包含药品名称、分类、用法用量等业务维度
3. **OCR提示词统一管理**：将 OCR 识别场景的提示词合并到「AI配置中心 → 提示词配置」统一管理，新增"体检报告识别"和"拍照识药"两个 Tab
4. **移除原识别记录Tab**：OCR识别配置页面中不再包含识别记录区块（由新的明细页面承接）
5. **数据双写机制**：用户使用体检报告解读或拍照识药功能时，系统同时写入 OCR 调用记录表和对应的业务明细表

## 使用说明

### 一、体检报告解读明细

**入口路径**：管理后台 → AI管理 → 体检报告解读明细

#### 1. 查看统计概览

页面顶部展示 4 个统计卡片：

| 指标 | 说明 |
|------|------|
| 总报告数 | 累计所有体检报告解读记录总数 |
| 今日新增 | 今天新产生的解读记录数 |
| 异常报告数 | 状态为"异常"（含异常指标）的报告总数 |
| 本月解读数 | 当月累计解读记录数 |

#### 2. 筛选数据

使用页面中的筛选区域，可按以下条件过滤数据：

- **时间范围**：选择起止日期
- **用户搜索**：输入手机号或昵称关键词
- **报告类型**：下拉选择（血常规、肝功能、综合体检等）
- **状态筛选**：选择"正常"或"异常"

点击「搜索」按钮应用筛选，点击「重置」清空所有条件。

#### 3. 查看详情

在列表中点击某条记录右侧的「详情」按钮，将从右侧弹出详情抽屉，展示以下完整信息：

- **用户信息**：头像、昵称、完整手机号
- **解读时间**：完整日期时间
- **原始图片**：上传的体检报告原图（支持点击放大查看）
- **OCR原始文字**：OCR 提取出的原始文字内容
- **AI结构化分析**：AI 分析结果的 JSON 格式化展示
- **异常指标列表**：表格形式展示所有异常指标（指标名、实际值、参考范围、异常说明）

> 注意：列表中的手机号显示为脱敏格式（如 138****5678），详情弹窗中显示完整手机号。

### 二、拍照识药记录明细

**入口路径**：管理后台 → AI管理 → 拍照识药记录明细

#### 1. 查看统计概览

页面顶部展示 4 个统计卡片：

| 指标 | 说明 |
|------|------|
| 总识药数 | 累计所有拍照识药记录总数 |
| 今日新增 | 今天新产生的识药记录数 |
| 药品种类数 | 识别出的不重复药品名称总数 |
| 本月识别数 | 当月累计识药记录数 |

#### 2. 筛选数据

- **时间范围**：选择起止日期
- **用户搜索**：输入手机号或昵称关键词
- **药品名称**：输入药品名称进行模糊搜索
- **药品分类**：下拉选择（处方药、非处方药、中成药、保健品）

#### 3. 查看详情

点击「详情」按钮弹出右侧抽屉，展示：

- **用户信息**：头像、昵称、完整手机号
- **识别时间**：完整日期时间
- **原始药品图片**：上传的药品原图
- **OCR原始文字**：OCR 提取的文字
- **AI结构化结果**：药品名称、药品分类、用法用量、注意事项/禁忌的完整展示
- **完整AI数据**：JSON 格式的 AI 原始分析结果

---

### 三、OCR提示词统一管理

**入口路径**：管理后台 → AI配置中心 → 提示词配置

#### 操作步骤

1. 进入「提示词配置」页面
2. 在顶部 Tab 栏中，找到新增的两个 Tab：
   - **体检报告识别**：管理体检报告 OCR 识别后的 AI 结构化分析提示词
   - **拍照识药**：管理拍照识药 OCR 识别后的 AI 结构化分析提示词
3. 点击对应 Tab，在文本框中编辑提示词内容
4. 编辑完成后点击「保存」按钮

> 提示词内容决定了 AI 对 OCR 识别文字的结构化分析质量，建议按照 JSON 输出格式要求来撰写提示词。

---

### 四、OCR识别配置页面变更

**入口路径**：管理后台 → AI管理 → OCR识别配置

本次更新对 OCR 识别配置页面做了以下调整：

1. **移除了"识别记录管理"区块**：原来显示在此页面的识别记录列表已由"体检报告解读明细"和"拍照识药记录明细"两个独立页面完全承接
2. **场景模板编辑中移除了提示词输入框**：提示词已统一迁移到「AI配置中心 → 提示词配置」管理
3. **新增场景自动联动**：当创建新的 OCR 识别场景模板时，系统会自动创建对应的提示词配置条目，并弹出提醒引导前往提示词配置页面完善内容

## 注意事项

1. **权限要求**：所有明细查询和管理功能仅限管理员角色访问
2. **数据脱敏**：列表中用户手机号自动脱敏显示，详情弹窗中展示完整手机号
3. **提示词为空处理**：如果 OCR 场景对应的提示词内容为空，OCR 识别时将跳过 AI 分析步骤，仅返回 OCR 原始文字
4. **数据双写**：用户使用 OCR 功能时，数据同时写入调用记录表和业务明细表，如果明细表写入失败不影响主流程
5. **向后兼容**：现有的 OCR 调用记录数据保持不变，新的明细表从本次部署后开始记录
