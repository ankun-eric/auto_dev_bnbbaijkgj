## 一、Bug 修复简介

### 问题现象

PRD《健康档案页面优化（守护体系收敛与旧页面下线）》v1.0 (2026-05-26) 的源码改造已经完成并 commit 到 master，但**用户在浏览器上仍然看到旧 UI**：

- 「我守护的人」页面上仍能看到「体验全新」按钮
- 转让横幅没有出现
- 旧目录 `guardian-system/`、`v13/` 还能访问

### 根本原因

老的部署脚本对 `h5-web`、`admin-web` 调用了 `docker compose build <svc>`，但服务器上 `docker-compose.yml` 中根本没有这两个 service（H5/admin 是用 `docker run` 独立启动的），命令报错 `no such service`。同时部署脚本未启用 `set -euo pipefail`，错误被**静默吞掉**，CI/调度层完全感知不到失败，于是用户看到的永远是旧版。

### 修复内容（仅运维层，未改业务代码）

1. **新增** `deploy_h5_admin_v2.sh`：H5/admin 容器单容器 `stop → rm → build → run` 部署流水线，全程 `set -euo pipefail`，杜绝静默失败
2. **新增** `rollback_h5_admin_v2.sh`：独立回滚脚本，部署自检失败时自动调用
3. **新增** `_deploy_h5_admin_v2.py`：通过 SSH 上传脚本并驱动远端执行
4. **新增** `_verify_prd_v1.py`：12 条 TC 验收脚本（9 条自动 + 3 条手测清单）
5. **关键技术修正**：在 `docker build` 中**正确传入 `--build-arg NEXT_PUBLIC_BASE_PATH`**（H5 用 `/autodev/{token}`，admin 用 `/autodev/{token}/admin`），原脚本遗漏 build-arg 导致 Next.js basePath 为空，所有路径 404
6. **镜像 tag 双轨**：每次部署前把当前 running 镜像 tag 为 `<repo>:autodev-prev`，新镜像 tag 为 `<repo>:autodev-<git-sha>`；自检失败自动回滚到 prev
7. **健康自检**：部署后 sleep 10s 让 nginx DNS 缓存刷新（valid=10s），然后 30 次 ×3s 探测 `https://{域名}/autodev/{token}/health-profile/i-guard`，要求最终 HTTP 200

### 3.1 浏览器强制刷新（关键第一步）

老 UI 缓存可能仍留在您的浏览器，**请先做一次强制刷新**：

1. 打开手机/电脑浏览器，访问 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard/)
2. 在 PC 浏览器：按 `Ctrl + F5`（Windows）或 `Cmd + Shift + R`（Mac）
3. 在手机浏览器：长按地址栏的刷新图标选择「强制刷新」/「清除缓存并重新载入」

## 四、3 条手测清单（需用户回执）

以下 3 条因为涉及 Tab 切换/弹窗点击交互，自动化成本高，请用户在浏览器手动验证并回复 ✓ / ✗：

- [ ] **TC-04**：在 `/health-profile/` 切换到非本人（被守护人）Tab，确认页面**不**再渲染「我守护的人」卡片
- [ ] **TC-05**：在非本人 Tab 上的「守护 TA 的人」分组，确认每一行都**没有**任何写操作按钮（邀请、移除、转让、解除、提醒均不可见）
- [ ] **TC-06**：在非本人 Tab 上点「守护 TA 的人」任意一位守护人，弹出的详情弹窗包含且仅包含 **5 个只读字段**（如：昵称、关系称呼、角色徽章、代付状态、绑定时间），且没有任何写按钮

回执通过后，本 Bug 修复正式关闭。

## 五、注意事项

1. **本次未发布新客户端**：本次修复仅动了服务器侧的 H5 / admin Docker 容器与部署脚本，**没有任何小程序 / 安卓 APK / iOS / Windows 桌面端代码改动**，因此**无需重新下载/安装**任何客户端。继续使用原有客户端访问 H5 入口即可看到新版 i-guard 页面。
2. **浏览器缓存**：如果强制刷新后仍看到旧版，请清掉该域名的 LocalStorage 与 ServiceWorker 缓存（Chrome 开发者工具 → Application → Storage → Clear site data）。
3. **回滚机制**：本次部署在服务器上保留了 `:autodev-prev` 镜像作为快照。若发现严重问题，运维可在服务器上一键 `bash rollback_h5_admin_v2.sh h5` / `bash rollback_h5_admin_v2.sh admin` 回到上一版本。
4. **后端不动**：本次 Bug 修复未触发后端容器重启。后端的 PRD 改造（`guardian_system_v12.py`：`total_count`、`/transfer/pending|approve|reject|cancel`、旧 `/api/guardian/list` 已删除）在更早的部署中已生效，本次只补齐 H5 / admin 的同步。
