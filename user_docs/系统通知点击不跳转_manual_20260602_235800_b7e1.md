## 功能简介

本次更新调整了【系统通知】的点击行为，**全站统一**：

- **点击通知记录后，只把它标记为"已读"（红点消失），人停留在原地，不再跳转到任何详情页**。
- 所有类型（家人邀请类、家人授权类、订单类、健康提醒类等）均不再跳转。
- 适用范围：H5 首页右上角铃铛 🔔 → 抽屉 →「通知」分页、独立的「消息中心」页面、小程序「消息中心」页面。

之前点击通知会自动打开"家人绑定列表 / 家人邀请页 / 订单详情"等页面，容易打断当前操作；现在点击通知只是"已读确认"，更符合用户实际预期。

## 使用说明

### 场景 1：在 H5 首页通过铃铛 🔔 抽屉查看通知

1. 打开 [H5 首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)，登录后进入 AI 首页。
2. 点击右上角铃铛 🔔 图标，抽屉从底部弹出。
3. 在抽屉顶部切换到「通知」分段，查看系统通知列表。
4. **点击任意一条通知**：
   - 该通知前的未读红点消失，文字样式变为已读样式（颜色变浅）。
   - **抽屉不会关闭，页面不会跳转**，您可以继续点击下一条。
5. 如需一次性把所有通知标记为已读，点击右上角「全部已读」按钮。
6. 阅读完毕后，点击抽屉外的遮罩或右上角 ✕ 关闭抽屉。

### 场景 2：在独立「消息中心」页面查看通知

1. 打开 [消息中心页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/messages)（也可从 H5 内右上角的"三"入口进入）。
2. 列表展示所有系统消息，未读消息带蓝色背景和左侧小红点。
3. **点击任意一条消息**：
   - 红点消失，背景变为白色，标题字重变为常规。
   - **页面不发生跳转**，您继续停留在消息列表，可继续点击其它消息。
4. 列表项右侧若有「重新邀请」按钮（针对家人邀请被拒类消息）：该按钮**仍然保留**原跳转行为，点击会进入家人邀请页。这是显式操作按钮，与"点击通知记录本身"不同，不在本次变更范围内。
5. 如需一次性标记全部已读，点击右上角「全部已读」。
6. 下拉刷新、上拉加载更多等行为不受影响。

### 场景 3：在微信小程序内查看通知

1. 在【微信开发者工具】中导入上方下载的小程序代码包并预览（导入步骤见下方「微信小程序体验」章节）。
2. 进入小程序「消息中心」页面。
3. **点击任意一条通知**：
   - 红点消失。
   - **页面不跳转**，继续停留在消息列表。
4. 「全部已读」按钮等其它功能保持不变。

## 注意事项

- 🔔 **铃铛入口、未读红点角标保留**：本次只调整点击行为，不移除铃铛入口、不影响未读数量徽标显示。
- 🔁 **"全部已读"功能不受影响**：未读列表清空、红点角标归零等仍正常工作。
- 🧩 **「重新邀请」等显式操作按钮保留原行为**：H5 消息中心列表项内的"重新邀请"按钮（针对被拒的家人邀请类消息）仍可正常跳转到家人邀请页。这类带文字标签的"操作按钮"与"点击通知记录本身"是不同的交互，本次变更仅影响后者。
- 📦 **后端数据结构不变**：消息体的 `click_action` / `click_action_params` 字段仍保留，但前端不再消费它们做跳转。后续如需恢复部分类型的跳转，无需改后端，只改前端。
- 🌐 **多端行为一致**：H5（铃铛抽屉 + 独立消息中心）、微信小程序（消息中心）三处入口的点击行为完全一致。
