## 一、功能简介

本次版本针对**商家 PC 端**与**平台后台 admin 端**的订单列表做了一次"减少视觉拥挤、提高浏览效率"的最小化优化：

1. **取消左侧固定列**：原本固定在左侧的 `订单号 / 下单时间 / 商品名称` 三列，现已**全部跟随中间区域横向滚动**，给中间字段释放更多可视空间。
2. **仅保留右侧 `状态 + 操作` 固定**：横向滚动时仍可随时看到订单状态，并直接点击「详情 / 发货 / 批准 / 拒绝」等按钮。
3. **统一列顺序**：从左到右调整为 `订单号 → 下单时间 → 商品名称 → 用户 → 手机 → 数量 → 金额 → 支付方式 → 状态 → 操作`（admin 端在金额与状态之间额外保留原有「退款状态」列；商家端在金额与状态之间额外保留原有「核销码 / 门店 / 预约时间 / 附件」等专属列）。
4. **按 PRD 规范统一列宽**：`订单号 160px / 下单时间 140px / 商品名称 220px(2 行截断+悬停 tooltip) / 用户 140px / 手机 120px / 数量 60px(右对齐) / 金额 100px(右对齐, ¥两位小数) / 支付方式 110px / 状态 100px / 操作 160px`。
5. **新增「用户(昵称+手机后4位)」列**：列形如 `张三(1234)`，方便一眼区分客户。
6. **新增「数量」列**：展示该订单全部商品 quantity 之和。
7. **截断与悬停**：`订单号` 截断后悬停显示完整订单号并支持复制；`商品名称` 最多两行截断后悬停 tooltip 显示完整商品名。

> 本次只改"列固定 + 列顺序 + 列宽"，**没有动**筛选条件、搜索框、分页、订单状态机、操作按钮逻辑，移动端（H5/小程序/APP）订单列表也**未受影响**。

## 二、操作步骤

### 2.1 平台后台 admin 端

1. 打开 [admin 登录页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/login)，使用平台管理员账号登录。
2. 在左侧菜单进入 **商品系统 → 订单**，或直接访问 [订单列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/orders)。
3. 你将看到刷新后的订单表格：
   - 顶部统计卡片、Tab 切换、搜索筛选条件**保持不变**。
   - 表格 **左侧不再有任何"粘住"的列**，所有业务字段（订单号 / 下单时间 / 商品名称 / 用户 / 手机 / 数量 / 金额 / 支付方式 / 退款状态）均跟随表格横向滚动。
   - **右侧固定** `状态` + `操作` 两列，无论横滚到哪里，状态和按钮始终可见。
4. 鼠标移到 `订单号` 单元格，会出现完整订单号 tooltip；点击后面的复制图标即可复制订单号。
5. 鼠标移到 `商品名称` 单元格，会以 tooltip 形式展示完整商品名（默认最多展示两行）。
6. 点击右侧 **详情 / 发货 / 批准 / 拒绝** 按钮（对应订单状态可见），原有的弹窗 / Drawer 全部正常工作。

### 2.2 商家 PC 端

1. 打开 [商家登录页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/login)，使用商家账号登录。
2. 在顶部导航进入 **订单管理**，或直接访问 [订单列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/orders)。
3. 你将看到刷新后的订单表格：
   - 表格 **左侧不再有任何"粘住"的列**，订单号 / 下单时间 / 商品名称跟随中间区域一起横滚。
   - 中间从左到右依次展示：`用户(昵称+手机后4位) / 手机 / 数量 / 金额 / 支付方式 / 核销码 / 门店 / 预约时间 / 附件`。
   - **右侧固定** `状态` + `操作`（详情、附件按钮）。
4. 鼠标移到 `订单号` 单元格，可看到完整订单号并支持复制。
5. 鼠标移到 `商品名称` 单元格，悬停 tooltip 显示完整商品名。
6. 点击右侧 **详情 / 附件** 按钮，原有的订单详情 Drawer / 附件 Modal 全部正常工作（含核销码、门店备注、调整预约时间、附件上传等功能均未变）。

### 2.3 一屏可见性预期

- 在 **1440 × 900**（PC 主流分辨率）及以上分辨率下，**不横滚即可看到** `订单号 → 支付方式` 全部 8 个核心字段 + 右侧固定的 `状态` + `操作`；admin 端因额外保留「退款状态」列，可能需轻微横滚或在 1440 下略微紧凑展示。
- 在 **1280 × 800** 最低分辨率下，至少保证 `商品名称 / 用户 / 手机 / 金额 / 支付方式` 这 5 个核心字段不横滚可见。
- 任何分辨率下，右侧 `状态 + 操作` 都保持固定可见。

## 三、新功能/字段说明

### 3.1 用户列（昵称 + 手机后4位）

- 显示格式：`昵称(尾4位)`，例如 `张三(1234)`。
- 当用户没有填写昵称时，admin 端会显示 `#用户ID`（例如 `#1024(1234)`），商家端会回退为 `用户(1234)` 之类的占位。
- 当用户没有手机号时，括号会被省略，仅显示昵称。

### 3.2 数量列

- 显示该订单所有 OrderItem 的 `quantity` 之和（**右对齐**）。
- 例如订单含 2 件商品 A + 1 件商品 B，则数量列显示 `3`。
- 与详情 Drawer 中"商品明细"表格保持一致。

### 3.3 金额 / 支付方式列

- 金额列：以 `¥` 前缀 + 两位小数显示，例如 `¥199.00`，**右对齐**，便于上下对比。
- 支付方式列：直接显示中文标签（`微信 / 支付宝 / 余额 / 积分`），不再展示渠道说明（详细渠道可在详情 Drawer 中查看）。

### 3.4 订单号复制

- 鼠标悬停在订单号上，会自动显示完整订单号 tooltip。
- 订单号文本右侧带"复制"图标，点击即可复制完整订单号到剪贴板，复制成功后图标会短暂提示"已复制"。

## 四、注意事项

1. **本次仅改 PC Web 端**：移动端（H5、微信小程序、安卓 / iOS APP）的订单列表**未做任何改动**，请勿误以为移动端也会出现新布局。
2. **筛选 / 搜索 / 分页 / 排序 / 导出 / 详情 / 操作按钮 全部不变**：原本能用的功能都还能用。如果发现"列变了之后某个功能跟着失效"，属于 bug，请联系研发反馈。
3. **大数据量横滚**：当前列总宽 admin 端约 1500px、商家端约 1820px，单页数据量越大，横向滚动越明显。表格已配置为表头与表体一同滚动，固定列不会与中间列错位。
4. **浏览器要求**：建议使用 Chrome / Edge / Safari 最新版浏览器，分辨率不低于 1280 × 800。
5. **缓存影响**：上线后如发现页面布局未更新，请在浏览器中按 `Ctrl + F5`（Windows）或 `Cmd + Shift + R`（macOS）强制刷新页面以清除缓存。
6. **无回归 Bug 预期**：本次属于纯 UI 列布局调整 + 后端补字段（user_nickname / user_phone / total_quantity）的最小集合改动，未触及订单状态机、支付链路、退款审核流程等核心逻辑。
