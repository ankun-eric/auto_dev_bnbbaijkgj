# 健康自查 - 档案信息收起与独立保存优化

> 版本：v1.0  
> 适用端：H5 / 微信小程序 / Flutter App（三端同步）  
> 发布日期：2026-04-20

---

## 一、功能介绍

本次优化将"健康自查"功能里"咨询人档案编辑"与"AI 开始分析"两个操作彻底解耦，并大幅降低已有家庭成员的视觉干扰：

1. **默认收起**：已有家庭成员的档案默认以一张"摘要卡片"展示，只显示 **头像/emoji + 姓名 + 性别 + 年龄**，页面更加干净。
2. **独立保存**：当您修改档案字段时，页面会自动出现 **"保存档案"** 按钮；未修改或保存后按钮会自动消失。
3. **智能拦截**：在切换其他咨询人或点击 **"AI 开始分析"** 时，如果您有未保存的修改，系统会弹出三选一提示：
   - **保存并分析 / 保存并切换**：先保存档案再继续
   - **放弃修改并分析 / 放弃修改并切换**：放弃刚才的改动
   - **取消**：留在当前页面继续编辑
4. **实时校验**：姓名、性别、出生日期在失焦/变更时即时校验，保存前再次总校验。
5. **新建成员保持旧体验**：首次添加新家庭成员时档案默认展开，无独立保存按钮，保存在 AI 分析时一并完成。

> 说明：H5、微信小程序、Flutter App 三端**交互逻辑、文案、数据完全一致**，UI 按各端平台规范呈现。

## 三、操作步骤

### 场景 1：快速开始分析（不修改档案）

1. 进入"症状自查"页面。
2. 选择部位/症状/持续时间后点击 **下一步**。
3. 页面展示咨询人列表，**默认已选中"本人"或上次使用的成员**，档案卡片处于"收起态"。
4. 确认是此人后，直接点击 **"AI 开始分析"** 按钮，进入 AI 分析。

### 场景 2：修改档案并独立保存

1. 在咨询人档案卡片上 **任意位置点击**，卡片展开进入编辑模式。
2. 修改想要更新的字段（姓名、性别、出生日期、身高、体重、既往病史、过敏史、遗传病史）。
3. 一旦出现未保存修改，页面底部会出现绿色 **"保存档案"** 按钮。
4. 点击 **"保存档案"**，等待提示 "保存成功"，按钮消失、卡片重新收起。
5. 再次点击 **"AI 开始分析"** 继续后续流程。

### 场景 3：切换咨询人或分析前有未保存修改

1. 您修改了档案但还没点"保存档案"。
2. 点击 **"AI 开始分析"** 或选择另一个家庭成员。
3. 系统弹出 **三按钮对话框**，请选择：
   - **保存并分析 / 保存并切换**：推荐。系统先调用保存接口，成功后继续。
   - **放弃修改并分析 / 放弃修改并切换**：放弃刚才的改动，恢复到加载时的数据。
   - **取消**：留在当前页继续编辑。

### 场景 4：新增家庭成员

1. 在咨询人列表底部点击 **"+ 添加家庭成员"**（H5/Flutter）或 **"➕ 家庭成员"**（小程序）。
2. 选择关系 → 填写基础信息（姓名、性别、出生日期为必填）。
3. 填完直接点击 **"AI 开始分析"**，系统会一并保存新成员并启动分析。

> 新建成员场景档案默认展开，**没有独立"保存档案"按钮**，这是为了让首次填写更顺畅。

## 四、注意事项

1. **必填字段**：姓名、性别、出生日期。若缺失，"保存档案"和"AI 开始分析"都会拦截并标红提示。
2. **重复保存会自动合并**：保存后如果您继续修改，"保存档案"按钮会再次出现；未修改则隐藏。
3. **网络异常**：保存失败时会弹 Toast，数据保留在页面上，可以重试或选择放弃。
4. **年龄展示**：根据出生日期自动计算，出生日期未填则显示 "年龄待完善"。
5. **历史分析不会改动档案**：老版本中点击"AI 开始分析"会悄悄覆盖档案，本次优化后 **AI 分析不再有保存档案副作用**，档案只在您显式点击"保存档案"或三选一弹窗中"保存并分析/切换"时才被写入。

## 五、常见问题

**Q1：为什么看不到"保存档案"按钮？**  
A：档案没有任何修改，或您当前是新建成员场景。修改任何字段即可唤起按钮。

**Q2：切换家庭成员时弹出了弹窗，我点了"取消"会发生什么？**  
A：留在当前成员的编辑页面，不切换、不保存、不丢失数据。

**Q3：AI 分析后我发现档案被改了，这是不是个 Bug？**  
A：本次优化后 AI 分析已**完全不修改档案**。如您在弹窗中选择"保存并分析"，则是按您的明确意愿先保存后分析。

**Q4：Flutter/小程序能用吗？**  
A：可以，三端功能、文案、交互完全一致。
