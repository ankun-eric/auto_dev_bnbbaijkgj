## 一、功能简介

本次更新对「居家安全设备管理」（紧急呼叫器 / 烟雾报警器 / 水位报警器）进行了体验全面升级，解决了以下四个老问题：

1. **设备一眼看不出类型**：管理后台和 H5 现在用红/橙/黄三色区分三类设备。
2. **H5 排版拥挤**：设备按类型分组、卡片留白舒适、序号一目了然。
3. **设备无法按家庭成员区分**：现已支持「本人 / 父亲 / 母亲 / 配偶 / …」按成员隔离。
4. **入口僵化**：管理后台「功能按钮管理」可灵活配置 ai-home 首页快捷入口。

## 三、H5 用户端使用指南

### 3.1 顶部「家庭成员 Tab」

进入 `/home-safety` 后，页面最顶部为一排横向滚动的成员 Tab：

```
┌──────┬──────┬──────┬──────┬─────────┐
│ 本人 │ 父亲 │ 母亲 │ 配偶 │ + 添加  │
└──────┴──────┴──────┴──────┴─────────┘
```

操作要点：

- **默认激活规则**：
  - 直接打开 `/home-safety` → 默认在「本人」
  - 健康档案某成员卡片点进来 → 自动激活该成员
- **切换 Tab 实时刷新设备列表**，**不重载整页**
- 切换 Tab 时浏览器地址栏 URL **不会变化**（设计如此）
  - 副作用：在「父亲」Tab 浏览中刷新页面，会回到 URL 指定的默认 Tab。如需保持父亲 Tab，请用带 `?member_id=父亲ID` 的链接进入。
- 点击「+ 添加」→ 进入家庭成员管理流程

### 3.2 历史数据迁移提示条

第一次更新后，在「本人」Tab 顶部可能会看到一条浅黄底色的提示：

> ⓘ 有 N 台设备默认归属本人，[去调整 →]    [×]

- **仅在「本人」Tab 显示**，其他成员 Tab 不会出现
- **「去调整 →」**：进入「设备归属批量调整」页面，可勾选设备 → 选择新归属成员 → 保存
- **「×」**：永久关闭，刷新后不再显示（同账号换浏览器仍会再次出现，受 localStorage 隔离）
- 若误关，可在「我的 → 设置 → 重置引导提示」处重新唤起

### 3.3 设备卡片新版排版

设备按「紧急呼叫器 → 烟雾报警器 → 水位报警器」三组顺序展示：

- **分组头**：渐变背景色（红/橙/黄）+ 大图标 + 类型名称 + 右侧「已绑定 N 台」
- **卡片左上角**：灰色「序号 / 总数」药丸（如「1 / 2」）
- **卡片右上角**：绿色 / 灰色圆点 + `在线` / `离线` 标签
- **卡片字段**：网关 ID、设备 SN、紧急联系手机、绑定时间、解绑按钮
- **失效设备**：右上角红色「需重绑」标识
- **紧急联系手机未填写**：底部红色提示条
- **空分组**：显示「暂无设备」+ `+ 添加` 按钮

### 3.4 添加设备

点击底部固定按钮「+ 添加设备」：

1. 选择设备类型（紧急 / 烟雾 / 水位）
2. **必填项 - 归属家庭成员**：默认带入当前 Tab 的成员，可改
3. 填写网关 ID、设备 SN、紧急联系手机
4. 提交

> 提交后新设备会自动出现在所选成员的 Tab 下。

### 3.5 调整设备归属

如发现某台设备归属错误（例如本人 Tab 下应当属于父亲）：

- **方式一**：通过迁移提示条「去调整 →」批量调整
- **方式二**：进入设备卡片 → 点击「调整归属」→ 选择新成员 → 保存

> 后端会通过 `PATCH /api/home_safety/devices/{id}/transfer` 完成切换。

### 3.6 健康档案入口

健康档案某成员 Tab 下点击「🛡️ 居家安全」卡片，会自动跳到该成员的居家安全页（顶部 Tab 自动激活到该成员）。

## 四、管理后台使用指南

进入路径：`管理后台 → 紧急呼叫触发源管理 → 居家安全设备管理`，共四个 Tab：

### 4.1 设备类型彩色 Tag

三个 Tab（设备绑定 / 报警记录 / 回调原始记录）的「设备类型」列均统一为彩色 Tag：

| 类型 | 颜色 |
| --- | --- |
| 紧急呼叫器 | 🔴 红色 |
| 烟雾报警器 | 🟠 橙色 |
| 水位报警器 | 🟡 黄色 |

### 4.2 行左侧 3px 细色条

每行最左侧加 3px 宽彩色细条（与设备类型同色），方便整页快速扫读。

### 4.3 「设备绑定」Tab 新增「归属成员」列

- 显示该设备绑定到的家庭成员姓名（如「本人」「父亲」）
- 「本人」用蓝色 Tag、其他成员用青色 Tag
- 由迁移脚本自动归属到「本人」的设备右上角带「·迁」上标，鼠标悬停可见说明
- 未归属的设备显示「未归属」灰色 Tag

### 4.4 历史数据迁移（管理员一次性操作）

发布后，管理员可以调用以下后端接口完成历史数据迁移（一次性、幂等）：

```
POST /api/admin/home_safety/migrate_member_id
```

返回示例：

```json
{
  "success": true,
  "binding_updated": 12,
  "alarm_updated": 35,
  "contact_updated": 8,
  "self_member_created": 5
}
```

接口幂等：重复调用不会重复迁移已处理数据。

## 五、API 接口变更（开发参考）

| # | 接口 | 变更 |
| --- | --- | --- |
| 1 | `GET /api/home_safety/members` | **新增**：返回当前账号下所有家庭成员 |

| 7 | `POST /api/admin/home_safety/migrate_member_id` | **新增**：一次性历史数据迁移 |

## 六、注意事项

1. **设备会"看得见"**：旧设备会自动归到「本人」名下，不会丢失。
2. **告警通知范围**：设备触发告警时，仅通知 **该设备归属成员本人 + 该成员的守护人**，不会再骚扰其他家庭成员的守护人。
3. **删除家庭成员前请先转移设备**：否则系统会拒绝删除。
4. **URL 行为**：切换 Tab 不修改地址栏。如需把当前 Tab 链接分享给他人，请手动复制 `?member_id=xxx`。
5. **提示条永久关闭后想找回**：进入「我的 → 设置 → 重置引导提示」即可重新唤起。
6. **运营配置 ai-home 快捷入口**：管理后台「功能按钮管理」→ 新增按钮 → 跳转 URL 填 `/home-safety` 即可。

## 七、常见问题（FAQ）

**Q1：我之前绑定的设备去哪了？**  
A：默认归到「本人」Tab。如有需要可通过顶部黄色提示条的「去调整 →」批量改归属。

**Q2：为什么切换 Tab 后地址没变？**  
A：这是 v2.1 锁定的策略：URL 是入口语义，刷新页面会回到 URL 指定的默认 Tab。

**Q3：父亲名下的烟雾报警器报警，我会收到通知吗？**  
A：会。父亲名下设备触发告警时，会通知**父亲本人（如有账号）+ 父亲在「守护人」中配置的人员**。

**Q4：紧急联系手机未填写卡片为何是红色提示？**  
A：紧急联系手机用于设备出险时联系，未填会导致 AI 外呼失败，请尽快补全。

**Q5：迁移提示条点叉关闭后是否会再来打扰？**  
A：同账号同浏览器内不再显示；换浏览器或清缓存后会再次显示一次。
