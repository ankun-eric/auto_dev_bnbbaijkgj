## 功能简介

本次更新针对「居家安全设备」页面进行了两项重要的体验优化：

1. **Bug 3 — 成员 Tab 与健康档案完全一致**：把"居家安全"页面顶部的家庭成员切换 Tab 升级为与「健康档案」完全一致的视觉风格——圆形字徽（爸/妈/夫/儿…）+ 关系名 + 辈分分色。无论您在哪个页面，切换家人的体验都是统一的。
2. **Bug 4 — 报警卡片信息齐全 + 可一键标记已处理**：以前点击报警卡片才会弹出 `alert` 弹窗看时间，现在所有信息（精确到秒的报警时间、设备备注、外呼通知状态）**一次性铺出在卡片上**，无需点击；未处理的报警还多了一个「**标记已处理**」按钮，点击后弹出二次确认，避免误触。

## 使用说明

### 一、查看家庭成员

1. 打开「居家安全设备」页面
2. 顶部即可看到**家庭成员 Tab**：
   - **本人**：渐变蓝底圆形字徽，文字为「我」
   - **爸爸 / 妈妈 / 爷爷 / 奶奶 / 外公 / 外婆**：长辈深蓝底圆字徽（爸/妈/爷/奶/外公/外婆）
   - **老公 / 老婆 / 哥哥 / 姐姐 / 弟弟 / 妹妹 / 朋友**：同辈天蓝底圆字徽（夫/妻/哥/姐/弟/妹/友）
   - **儿子 / 女儿**：晚辈青色底圆字徽（儿/女）
   - **其他关系**：灰蓝底字徽
3. **选中态**会有以下变化：
   - 字徽放大 1.06×
   - 字徽底部出现 6px 蓝色阴影
   - 下方关系名变蓝、加粗
   - Tab 底部出现 3px 蓝色下划线
4. 末尾「**+ 添加**」按钮：虚线圆形 + "添加" 文字，点击可跳转到「健康档案」页添加新家庭成员。

### 二、切换设备列表 / 报警记录

成员 Tab 下方有一级 Tab：

- **设备列表**（默认）：按设备类型分组展示（紧急呼叫器 / 烟雾报警器 / 水位报警器），可绑定 / 解绑 / 调整归属 / 编辑备注。
- **报警记录**：展示该家庭成员名下的全部历史报警。

> 直达报警记录：在 URL 末尾追加 `?tab=alarm` 即可。

### 三、读懂一张报警卡片

新版报警卡片每一项信息都一次性展示在卡片上，**无需任何点击**：

```
┌─┬───────────────────────────────────────────────┐
│ │ 🟠 烟雾报警器  ● 未处理         2 小时前    │
│橙│ 客厅烟雾报警器                                │  ← 备注名（主标题）
│ │ 🕐 2026-05-29 18:30:42                       │  ← 秒级精确时间
│ │ ✓ 已外呼通知紧急联系人 138****8888           │  ← 外呼通知（如已通知）
│ │ - - - - - - - - - - - - - - - - - - - - -   │
│ │ SN：A3F2B8C4D5E6              [ 标记已处理 ] │  ← 操作按钮
└─┴───────────────────────────────────────────────┘
```

各字段含义：

- **左侧 6px 色块条**：颜色随设备类型（紧急=红 / 烟雾=橙 / 水位=黄）
- **设备类型 Tag**：彩色圆角标签
- **状态胶囊**：
  - 🔴 未处理（红圆点 + 浅红底 + 深红字）
  - 🟢 已处理（绿圆点 + 浅绿底 + 深绿字）
  - ⚫ 已忽略（灰圆点 + 浅灰底 + 深灰字）
- **相对时间（右上）**：刚刚 / N 分钟前 / N 小时前 / 昨天 HH:mm / YYYY-MM-DD
- **设备备注名（主标题，16px / 加粗）**：例如「客厅烟雾报警器」，没备注时显示「未命名」灰字
- **精确时间行（🕐）**：`YYYY-MM-DD HH:mm:ss`，**秒级**，方便您核对事件发生时刻
- **外呼通知**：若系统已自动外呼通知紧急联系人，会出现绿色 ✓ + 脱敏手机号
- **SN（小字辅助，12px 灰色）**：设备唯一编号
- **「父亲名下」等成员标签**：仅当您在「本人」之外的 Tab 时显示

### 四、标记报警为「已处理」

1. 切换到「报警记录」Tab
2. 在状态为「**未处理**」的卡片底部右侧，会出现「**标记已处理**」蓝色按钮
3. 点击该按钮 → 弹出**二次确认弹窗**：
   ```
   确认标记为已处理？
   处理后将不可撤销，请确认设备已经检查并排除风险。
   [ 取消 ]      [ 确认已处理 ]
   ```
4. 选择：
   - **取消**：不调用任何接口，关闭弹窗
   - **确认已处理**：调用 `PATCH /api/home_safety/alarms/{id}/resolve` 接口
5. 成功后：
   - 卡片状态胶囊立即变绿"已处理"
   - 「标记已处理」按钮消失
   - Toast 提示「已标记为已处理」
6. 若已经处理过的报警再次调用接口，会**幂等返回**，不会重复修改时间，也不会报错。

## 注意事项

1. **二次确认不可绕过**：由于此操作不可撤销，**老人误点也不用担心**——第一次点击只会弹出确认框，必须主动点击「确认已处理」才会真正修改状态。
2. **权限控制**：只有报警所属本人或绑定家庭成员可以执行「标记已处理」。其他人尝试调用接口会收到 `403 forbidden`。
3. **接口幂等**：已处理过的告警再次调用 `PATCH /resolve` 不会报错，而是返回 `message="已处理过"`，方便客户端在弱网重试场景下安全使用。
4. **成员 Tab 数据源统一**：现在「居家安全」和「健康档案」都使用 `/api/family/members`，旧接口 `/api/home_safety/members` 已标记为 `deprecated`（30 天后下线，过渡期内 H5 / 小程序仍可访问，但会同步走 family/members）。
5. **报警分页**：默认 size=20，下拉加载更多。
6. **时间显示口径**：相对时间按"自当前时间起的秒差"计算；精确时间使用浏览器本地时区。

## 常见问题

**Q1：为什么我没看到「标记已处理」按钮？**
A1：仅当 `handle_status === 0`（未处理）时该按钮才会出现。如果报警已经被处理过、或被忽略，则不再显示该按钮。

**Q2：点击卡片为什么没有反应？**
A2：这是本次 Bug 4 的修复点——以前点击卡片会触发系统 `alert` 弹窗，让用户误以为"信息要弹才出来"。现在所有信息都已平铺到卡片上，**点击卡片不再有任何弹窗**。

**Q3：我看不到家人的字徽（显示为灰色待补充）怎么办？**
A3：这是老数据兼容兜底：当家庭成员未填写"关系类型"字段时会显示姓名首字 + 灰蓝底 + 右上角「待补充」黄色角标。请前往「健康档案 → 编辑该成员」补全关系类型即可。

**Q4：「报警记录」Tab 切换成员时是否会重置？**
A4：不会重置 Tab 选择，只会重新加载该成员的报警列表。换句话说，您切换爸爸 / 妈妈 / 儿子时，依然停留在「报警记录」Tab 上。
