## 使用说明

### Bug 1：绑定设备后立即看到新记录

1. 在浏览器中打开 [H5 居家安全设备页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/home-safety/)
2. 选择任一 Tab（紧急呼叫器 / 烟雾报警器 / 水位报警器）
3. 点击页面上方的「+ 绑定新设备」按钮
4. 在弹窗中输入：
   - **网关 SN**：12 位字母+数字（例如 `ABCD12345678`）
   - **设备 SN**：8 位字母+数字（例如 `ABCD1234`）
5. 点击「确认绑定」
6. ✅ 弹窗自动关闭，下方「我绑定的设备」区域**立即**出现刚才绑定的设备卡片，无需手动刷新

> 修复说明：H5 axios 拦截器已将响应 body 提到第一层，原代码使用 `d.data?.groups` 永远取不到值，已修正为兼容写法 `d?.groups ?? d?.data?.groups`，同时显式传入 `activeTab` 避免闭包旧值问题。

### Bug 2：时间字段显示为北京时间

#### 管理后台

1. 打开 [管理后台 · 居家安全设备管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-safety/)
2. 切到「设备绑定流水」Tab，「绑定时间」「解绑时间」列显示为 `2026-05-27 23:10:31` 形式的北京时间
3. 切到「报警记录流水」Tab，「上报时间」「接收时间」列显示为北京时间
4. 切到「回调地址配置」Tab，底部「上次保存时间 / 上次推送上游时间 / 上次测试时间」均显示为北京时间

#### H5 用户端

1. 打开 [H5 居家安全设备页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/home-safety/)
2. 绑定记录卡片中「绑定时间」显示为北京时间
3. 报警记录条目中的时间显示为北京时间

> 修复说明：后端继续使用 `datetime.utcnow()` 存储，但 `isoformat()` 输出时附加 `Z` 后缀显式标识 UTC（如 `2026-05-27T15:08:51Z`）；前端封装 `formatDateTime()` 工具函数，使用 `dayjs` 的 utc + timezone 插件将 UTC 时间转换为 `Asia/Shanghai` 时区并格式化为 `YYYY-MM-DD HH:mm:ss`。

### Bug 3：报警 Tab 表格横向滚动

1. 打开 [管理后台 · 居家安全设备管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-safety/)
2. 切到「报警记录流水」Tab
3. 当列数较多时，表格底部会自动出现横向滚动条，整体表格不再撑破页面宽度
4. 「设备绑定流水」Tab 同样加了滚动容器（`scroll x=1200`），双重保险

> 修复说明：给报警表格加 `scroll={{ x: 1600 }}`、给绑定表格加 `scroll={{ x: 1200 }}`，并给「处置备注」列设置 `width: 200`，避免它自动撑开破坏布局。

## 注意事项

- 本次修复**不需要**用户重新登录或清浏览器缓存，刷新页面即可看到新效果。
- 时间字段的"UTC + Z 后缀"约定是面向**所有客户端**的，老前端（如桌面浏览器收藏的旧标签页）如果还停留在修复前的代码，可能仍显示 UTC 时间；刷新即可恢复正常。
- 若 H5 / 管理后台显示仍异常，请检查浏览器开发者工具控制台是否有 `[HOME-SAFETY]` 前缀的报错日志，可作为后续问题排查依据。
- 管理后台需要先以管理员身份登录；H5 用户端需要先以普通用户身份登录。
