## 功能简介

本次操作为**测试环境的整体重新部署**：在不修改业务代码的前提下，将 Git 仓库 `master` 分支的最新代码（commit `35b8ad8c`）通过远程构建方式重新部署到测试服务器，并验证所有关键链接的可达性。

本次部署涉及以下服务：

- **后端 API**（FastAPI · Python 3.12）—— 业务接口、健康档案、订单、会员等
- **管理后台**（Next.js · admin-web）—— 运营管理界面
- **H5 用户端**（Next.js · h5-web）—— 终端用户 Web 入口
- **数据库**（MySQL 8.0）—— 业务数据存储

部署后所有容器命名均以部署唯一标识 `6b099ed3-7175-4a78-91f4-44570c84ed27` 作为前缀，统一由 gateway-nginx 通过路径前缀 `/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/` 反向代理对外提供 HTTPS 服务。

## 注意事项

1. **测试环境数据可能不稳定**：本次重新部署执行了 `docker compose down`，但 **MySQL 数据卷 `mysql_data` 和上传文件卷 `uploads_data` 已保留**，原有业务数据不会丢失。
2. **登录会话**：旧的 access token 在后端容器重启后通常仍然有效（JWT 模式），但如发生异常请重新登录。
3. **首次访问可能稍慢**：Next.js 标准模式首次进入页面时需要做 SSR 渲染与冷启动，第二次访问会显著加快。
4. **本次未变更客户端**：本次只是把测试环境重新部署一次，**没有修改业务代码**，所以 H5、admin、安卓 APP、iOS APP、小程序均无需重新下载/导入新版本，使用现有版本即可。
5. **微信小程序入口**：小程序部分需要在「微信开发者工具」中导入项目源码体验，本次部署仅刷新了服务端 API，小程序代码包未变更。
6. **服务器 80 端口规则**：所有外部访问必须通过域名 + HTTPS（443 端口），不要直接访问 Docker 容器内部端口（3000、3001、8000、3306）。这些内部端口不对外暴露。
