## 功能简介

本次更新对管理后台「健康计划管理」一级菜单做了**一次性清理与重构**，目的是让菜单更精简、归属更合理：

1. **下线孤岛功能**：彻底删除「健康计划管理 → 通用任务配置」页面、接口和数据表。经过全端代码核查，该功能虽在后台存在维护入口，但 H5 / 小程序 / Flutter App 均未引用其数据，属于"已实现但未接通"的孤岛模块。
2. **打卡数据统计搬家**：把「打卡数据统计」从原来的「健康计划管理」菜单挪到「健康档案管理」菜单下，业务归属更合理。**页面本身的功能、UI、数据完全不变**。
3. **整条「健康计划管理」一级菜单下线**：上面两步完成后，原一级菜单下已无任何二级菜单项，整条菜单从左侧导航中彻底移除。

> 注：用户端 H5、小程序、Flutter App **没有任何代码改动**，普通用户的体验完全 0 感知。

## 操作步骤

### 进入「打卡数据统计」新位置

1. 使用超级管理员/运营管理员账号登录管理后台
2. 在左侧导航栏点开「健康档案管理」一级菜单
3. 点击末尾的「打卡数据统计」二级菜单
4. 即可进入与搬家前完全一致的打卡数据统计页面

页面内容（与搬家前完全相同，0 改动）：

- 顶部 5 个汇总卡片：今日活跃用户、总用户数、活跃计划数、用药提醒数、打卡项数
- 每日打卡趋势：按日期分布的打卡数量
- 每日打卡汇总表：可按用户筛选 + 日期范围筛选，展开后查看明细
- 明细类型：用药打卡、健康打卡、计划任务

### 旧入口的行为

- **「健康计划管理」一级菜单**：在左侧导航中已**彻底消失**，菜单管理与权限管理页面中也已查不到相关条目
- **旧 URL `/admin/health-plan/default-tasks`**：访问后跳到 404 页面（页面已彻底删除）
- **旧 URL `/admin/health-plan/statistics`**：同样跳 404，请改用新地址 `/admin/health-records/statistics`

- ✅ 浏览器中如保留了旧菜单缓存，刷新后即可看到新菜单结构，无需清缓存。
- ✅ 用户端 H5 / 小程序 / Flutter App 本次**无任何代码改动**，普通用户的体验完全 0 感知，无需重新安装或更新客户端。

## 常见问题

**Q1：我以前用「健康计划管理 → 打卡数据统计」收藏的浏览器书签还能用吗？**

A：旧地址会直接 404。请使用新地址 `/admin/health-records/statistics`，或者通过左侧菜单「健康档案管理 → 打卡数据统计」进入。

**Q2：我以前在「通用任务配置」里配置过的预置任务还在吗？**

A：不在了。该功能为孤岛模块，本次清理已经直接 DROP 数据表（产品已确认接受不可恢复）。该功能本来就未被 H5/小程序/App 引用过，没有任何用户实际看到过这些预置任务。

**Q3：本次清理会影响用户端的「健康打卡」和「健康计划」体验吗？**

A：完全不会。用户端 H5 / 小程序 / Flutter App 没有任何代码改动，用户打卡、用药提醒、健康档案、推荐计划列表等所有功能体验保持原样。
