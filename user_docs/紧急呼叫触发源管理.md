## 功能简介

本次更新解决了「紧急呼叫触发源管理」页面的两件事：

1. **修复中文乱码 Bug**：页面上 4 张内置触发源卡片中的中文名称与描述长期显示为乱码（如 `å¥å`），本次彻底修复并以「最终参考版」文案重新写入。
2. **开放内置触发源的可编辑能力**：以前内置触发源完全不能修改，本次开放「修改名称」和「修改描述」两项能力，方便运营按业务变化迭代文案，同时仍然保留以下硬保护：
   - 内置触发源**不可删除**
   - 内置触发源**始终启用**，不允许禁用
   - 内置触发源的 `source_code`、图标、适用设备类型等技术字段**不可被修改**

修复完成后，4 条内置触发源在页面上的最终文案为：

| 触发源编码 | 中文名 | 中文描述 |
|---|---|---|
| `health_data_abnormal` | 健康数据异常 | 当用户的健康数据（如心率、血压、血氧）超出安全阈值时自动触发 |
| `smoke_alarm` | 烟雾报警 | 当家中烟雾传感器检测到烟雾浓度异常时触发紧急呼叫 |
| `water_alarm` | 浸水报警 | 当家中浸水传感器检测到漏水或积水时触发紧急呼叫 |
| `emergency_button` | 紧急按钮 | 当用户按下随身或家中的一键紧急呼叫按钮时立即触发 |

## 使用说明

### 1. 进入「紧急呼叫触发源管理」页面

1. 在浏览器打开 [管理员后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin)
2. 使用管理员账号登录（默认 `13800000000` / `admin123`）
3. 在左侧菜单点击「紧急呼叫触发源管理」，或直接访问 [紧急呼叫触发源管理页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/emergency-sources)

### 2. 查看 4 张内置触发源卡片

进入页面后你会看到：

- 顶部蓝色 Hero 区，展示「总数 / 内置 / 自定义 / 启用中」4 个统计数字
- 下方卡片网格：4 张「内置」蓝色徽章的卡片 + 任何已新增的「自定义」紫色徽章卡片
- 每张内置卡片的右上角都有一个**蓝色铅笔图标**，点击即可弹出编辑窗口
- 内置卡片**不显示右上角的启停开关**（因为内置触发源始终启用）
- 内置卡片**不显示底部"…"操作菜单**（因为不可删除）

### 3. 编辑内置触发源（仅修改名称 / 描述）

1. 在任意一张内置卡片右上角，点击**蓝色铅笔图标 ✏️**
2. 弹出的「编辑触发源」窗口顶部会出现一条蓝色提示：
   > ℹ️ 这是系统内置触发源，仅支持修改名称和描述，且始终启用。
3. 窗口中只显示三项：
   - **触发源编码**（灰色禁用，仅作展示，不可改）
   - **触发源名称**（最长 20 字，必填）
   - **描述**（最长 100 字，必填）
4. 修改完成后点击「确定」，系统会提示「已更新」，卡片实时刷新
5. 即使你通过浏览器开发者工具或 Postman 直接调用接口并附带 `is_enabled=false`、`source_code=xxx`、`sort_order` 等其他字段，**后端会自动忽略这些字段**，保证内置项的核心配置安全

### 4. 新建 / 编辑 / 删除自定义触发源（不受限）

1. 点击页面右上角「新增触发源」按钮
2. 在弹出窗口中填写：触发源编码（小写英文+下划线，如 `gas_alarm`）、名称、描述、适用设备类型、触发条件 JSON、排序、启用开关
3. 自定义触发源的卡片右下角会显示「…」菜单，可「编辑 / 删除」
4. 自定义触发源的右上角显示**绿色启停开关**，可随时切换启用 / 停用

### 5. 试图删除 / 禁用内置触发源时的预期行为

- 页面 UI 上根本看不到删除按钮和启停开关，没有误操作入口
- 如果通过外部工具（Postman / 浏览器开发者工具）调用：
  - `DELETE /api/admin/emergency-sources/{内置id}` → 返回 `403 Forbidden`，提示「内置触发源不可删除」
  - `PATCH /api/admin/emergency-sources/{内置id}/toggle` → 返回 `403 Forbidden`，提示「内置触发源始终启用，不允许禁用」

## 注意事项

1. **仅限管理员**：本功能仅在管理员后台（`/admin/...` 路径下）开放，H5 / 小程序 / Flutter APP 等用户端**不存在编辑入口**，无需担心被普通用户修改。
2. **字段长度限制**：内置触发源的「名称」最长 20 字，「描述」最长 100 字，超出会被表单校验拦截。
3. **中文乱码修复一次性生效**：本次部署已在数据库执行 UPDATE，4 条内置记录已重写为正确的 UTF-8 中文；同时种子脚本也已修复，未来即使重装数据库，也会用正确的中文重新写入（采用 `INSERT ... ON DUPLICATE KEY UPDATE` 幂等模式，自动覆盖任何历史乱码）。
4. **缓存影响**：如果你在数据库 UPDATE 完成前打开过页面，请按 `Ctrl+F5` 强制刷新浏览器以清除前端缓存。
5. **后续扩展**：如未来需要支持「内置触发源也允许禁用」，只需要后端 `is_enabled` 强制锁定那一行去掉，并在前端编辑弹窗里加上启用开关即可，改动极小。
