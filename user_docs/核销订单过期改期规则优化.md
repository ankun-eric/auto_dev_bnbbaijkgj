## 功能简介

本次优化解决了用户长期投诉的"错过 1 次预约即被永久作废"的体验痛点。核心变化如下：

1. **新增商品级"允许改期"开关**：商家可在商品编辑页配置当前商品是否允许用户错过预约时段后改约，默认**允许**。
2. **错过预约时段不再立即作废**：当商品"允许改期"时，用户错过预约最多可自助改约 **3 次**；只有改满 3 次仍未到店时订单才会真正过期。
3. **不允许改期商品仍按旧规则**：电影票、特定场次演出、限定时段课程等"刚性履约"商品的商家可关闭开关，保留"错过即作废"的硬规则。
4. **任意"待核销"状态都可申请退款**：包括已改 3 次仍在等待核销的订单（等同保留无限退款窗口），全部由用户主动发起、平台审核中心统一受理。
5. **改约置灰提醒**：达到 3 次改期上限的订单，"改约"按钮会被置灰并显示"已无法改期"，避免用户误操作。
6. **联系商家兜底入口**：所有订单状态下都展示"联系商家"按钮，点击底部弹窗可一键拨打门店电话或导航到门店地址。
7. **复用现有退款审核中心**：过期单退款申请与普通售后退款合并审核，平台运营人员入口零变化。

## 使用说明

### 一、用户端：错过预约时段后如何改约

#### 1. 打开订单
进入【我的订单 → 待核销】，找到错过预约时段的订单。

#### 2. 查看订单状态
- **状态仍为「待核销」** + 改期次数 N/3 → 仍可继续改约
- **状态显示「已过期」** → 该商品不允许改期（电影票等），无法操作

#### 3. 点击"改约"按钮
- 选择新的预约日期、时段
- 提交后改期次数 +1（页面会显示"已改期 X/3 次"）

#### 4. 达到 3 次后的提示
- "改约"按钮会变成灰色 + 旁边显示"已无法改期"
- 点击灰色按钮会提示：**本订单已达改期上限**

### 二、用户端：申请退款（待核销状态）

无论改期次数是 0、1、2 还是 3，只要订单仍在"待核销"状态：

1. 点击订单卡片底部"申请退款"按钮
2. 填写退款理由（可从预设选项选择："时间冲突无法到店"/"临时有事"/"其他"）
3. 提交后订单进入"退款审核中"，等待平台审核
4. 审核通过：原路退款 → 订单转"已退款"
5. 审核拒绝：订单回到"待核销"，可继续改约或重新申请退款

> ⚠️ **已过期订单不支持申请退款**（点击会提示"已过期的订单无法申请退款"）。

### 三、用户端：联系商家（所有状态可用）

1. 点击订单卡片底部"联系商家"按钮
2. 弹出底部 Action Sheet，显示：
   - 门店名称
   - 门店地址（点击"导航"按钮调起地图）
   - 联系电话（点击"拨打"按钮调起电话）
   - 提示文案："如有疑问可联系商家协商处理"
3. 异常情况：
   - 门店未配置联系电话：电话区域显示"商家未提供联系方式"，拨打按钮隐藏
   - 门店未配置地址：仅显示电话，地址行隐藏

### 四、商家端：配置商品"是否允许改期"

1. 登录【管理后台】→ 商品系统 → 商品列表
2. 点击编辑某个商品
3. 在"销售设置"区块找到 **"允许用户改期"** 开关
   - **打开（默认）**：用户错过预约时段后，可在 App 内自助改约（最多 3 次），不会立即过期
   - **关闭**：错过预约时段后订单立即过期，不可改约、不可退款（适用于电影票/演出票/限定时段课程等）
4. 保存即可生效

### 五、平台端：审核退款申请

1. 进入【管理后台 → 退款审核中心】
2. 查看用户提交的退款申请（含改期次数、距首次预约时长等参考字段）
3. 选择"通过"（原路退款）或"拒绝"（订单回到待核销）
4. 平台保留拒绝权，无统一通过规则

## 注意事项

1. **改期次数按订单累计**：一个订单整单累计 3 次（不是按订单内每个商品分别计数）。
2. **存量数据不动**：本次上线前已经处于"已过期"状态的订单不会被回滚，新规则只对上线后产生的新订单生效。
3. **存量商品默认允许改期**：所有上线前已存在的商品 `allow_reschedule` 字段会被自动填充为 `true`（允许改期），商家可在商品编辑页随时关闭。
4. **退款全部走平台审核**：本次未引入"自动退款"机制，所有退款申请都由用户主动发起、平台审核中心统一受理（保留拒绝权）。
5. **退款时间窗口**：在"待核销"状态下随时可申请，无时间限制；订单一旦真正进入"已过期"状态则不可退款。
6. **联系电话来源唯一**：底部弹窗中的电话**严格取自门店管理 → 联系电话**字段，**不会取商家总部电话**，请商家务必为每个门店配置正确的对客联系号码。
7. **客户端必须更新**：仅 H5 用户端 / 商家管理后台为浏览器访问可即时生效，**微信小程序 / 安卓 APP / iOS APP 必须下载本次新版本**才能看到新功能。
8. **改约自助完成**：商家不参与改约确认环节，由用户在客户端自助选择新时段。
9. **不支持改期商品的兜底**：当商品 `allow_reschedule=false` 时，错过预约即作废且不可退款，"联系商家"按钮仍然展示作为兜底申诉入口。
10. **数据库自动迁移**：服务端启动时已自动为 `products` / `unified_orders` 表新增 3 个字段（`allow_reschedule` / `reschedule_count` / `reschedule_limit`），无需 DBA 介入。
