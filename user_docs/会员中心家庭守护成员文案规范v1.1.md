## 功能简介

本次发版完成 **v1.1 文案规范** 的全产品落地，核心是把"家庭守护成员"名额的展示口径统一为「**含主账号本人在内**的总人数」，消除老版「+1 加工」+「副文案双行」造成的歧义：

- ✅ **数据库字段口径变更**：`max_managed` 由「不含本人」改为「**含本人总人数**」，已通过一次性自动迁移脚本完成现网数据的 `+1` 修正。
- ✅ **权益卡片单行化**：会员中心权益卡片只保留主文案数字，副文案「含本人在内，统一守护全家健康」整行下线。
- ✅ **档案管理列表页顶部小字**：仅在该页面顶部展示一行 12px 灰字「本套餐可管理 N 位家庭成员（含本人）」，作为唯一"含本人"规则告知触点。
- ✅ **Admin 录入提示**：套餐管理、免费会员配额录入框旁文案改为「填几就是几（含主账号本人在内的总人数）」，所见即所得。
- ✅ **前端零加工**：H5 前端不再做 `max_managed + 1` 加工，模板直接展示数据库原值。

## 使用说明

### 1. 用户：在会员中心查看家庭守护成员名额

1. 打开 [会员中心](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center/)
2. 在「我的会员权益」2×2 网格中找到「家庭守护成员」格子
3. 大字显示的数字 **就是您当前套餐可建档的家庭成员总人数（含您自己）**，例如「10」表示包括您本人在内最多 10 位成员
4. 网格下方**不再展示任何副文案**（如「含本人在内，统一守护全家健康」），整体更简洁

### 2. 用户：在档案管理列表页查看名额规则说明

1. 打开 [健康档案 · 档案管理列表页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/archive-list/)
2. 页面顶部、列表标题上方会显示一行 12px 灰色小字：
   - 限额套餐：「本套餐可管理 N 位家庭成员（含本人）」
   - 不限档套餐：「本套餐可管理家庭成员数量不限（含本人）」
3. 下方的「已管理 X 人 / 上限 Y 人」中：
   - **X = 已建档案数（含您本人卡）**
   - **Y = 套餐上限（含本人）**
   - 「还可添加」也自动按含本人口径计算

### 3. 用户：套餐对比页

1. 在 [会员中心](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center/) 滚动到「权益对比」表
2. 「家庭守护成员」一行的数字**直接是各套餐含本人在内的总人数**，无 +1 加工、无「（含本人）」备注

### 4. 管理员：在 Admin 后台录入新套餐

1. 打开 [Admin 套餐管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/membership/plans)
2. 点击「+ 新增套餐」，定位「家庭守护成员总人数（含本人）」字段
3. 字段下方说明文字：
   > 请填写包含主账号本人在内的总人数（用户端原样展示此数字）。例如想让用户最多管理 10 人（含本人），就填 10；-1 表示不限。
4. 填入数字即等于用户端最终展示数字（**所见即所得**，无需再 +1）
5. 同样规则适用于 [Admin 免费会员额度](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/membership/free-quota) 页面

### 5. 管理员：在 Admin 用户管理详情页

- 用户详情卡 / 家庭管理详情 中显示的 `家庭守护成员配额（含本人）` 字段直接展示用户当前套餐的名额数字，无需在脑中做 +1 计算。

## 注意事项

1. **存量数据已迁移**：后端启动时会自动检测并执行一次性 `+1` 迁移脚本（`schema_migration_log` 表内 key=`membership_v11_max_managed_include_self_20260530`），保证现网旧值正确升级；幂等，不会重复执行。
2. **不限档行为不变**：`max_managed = -1` 仍表示不限，无论权益卡片、套餐对比、档案管理列表页都展示为「不限」。
3. **本次涉及"含本人"字样的展示触点**：**仅档案管理列表页顶部**一行小字会出现「（含本人）」字样。会员中心权益卡片、套餐对比、套餐详情、添加家人弹窗等**均不出现** "含本人 / 不含本人" 字样。
4. **后端内部配额校验逻辑保持不变**：v1.1 通过在 `_get_max_guardians` 内部统一 `-1` 转换，保证守护体系等历史模块比较逻辑零改动，确保零回归。
5. **API 响应字段保持不变**：`max_managed` / `quota_max` / `quota_used` 字段名都不变，仅值的语义由"不含本人"切换为"含本人"。前端模板直接 `{{ plan.max_managed }}` 展示即可。
