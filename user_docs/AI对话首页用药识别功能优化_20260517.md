## 功能简介

本次更新围绕 AI 对话首页（ai-home）的「拍照识药」功能做了 4 项体验优化，并把会话空闲超时从 30 分钟放宽为 60 分钟：

| 编号 | 优化点 | 用户感受 |
|------|--------|----------|
| ① 流式输出体验 | 上传图片后 0.5 秒内出现"正在识别中…"占位气泡，文字逐字流式吐出，先报识别进度，再用所选家人姓名做个性化承接 | 不再"卡几秒后一次性出大段文字"，等待变成"AI 实时和我说话" |
| ② 识药卡片增强 | 卡片包含「药品基础信息 / 用法用量 / 安全提示 / 个性化风险提示」四大模块，按钮永远固定在卡片底部 | 长内容不再把按钮挤出屏幕，关键风险（是否能给当前家人吃）一眼看到 |
| ③ 对话历史完整保留 | 点击"加入用药计划""查看药品详情""重新拍照"跳走后再回 AI 对话页，识药消息和卡片 100% 保留，已加入的按钮自动变灰显示「已加入用药计划」，刷新/换设备都还在 | 不会再"返回后发现刚才识别的药卡片不见了" |
| ④ 时区显示 | 历史会话列表里"刚刚发生的会话"显示「刚刚 / 几秒前」，不再误显示"8 小时前" | 时间显示终于和直觉一致 |
| ⑤ 会话超时 | 默认 60 分钟空闲后再次发消息才开新会话；进入 ai-home 默认续上最近一个会话 | 离开半小时回来还是同一个对话，不被频繁打断 |

## 操作步骤

### 步骤 1 · 选择家庭成员

进入 [AI 对话首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home) 后，在顶部点击「咨询对象」选择要识药的家人（妈妈给孩子拍药时务必先选孩子）。

> 当前选中的家人姓名会作为识药卡片"个性化风险提示"的判断依据，比如对该家人是否有过敏冲突、是否属于特殊人群（婴幼儿/老人）等。

### 步骤 2 · 拍照识药

1. 点击页面下方输入栏附近的「拍照」入口（或在功能区点击「拍照识药」）。
2. 对准药盒正面拍摄一张清晰图，光线足、文字可见。
3. 提交后立即可见「正在识别中…」占位气泡。

### 步骤 3 · 等待流式播报

AI 会逐字吐出两段文案：

1. 第一段（识别进度）："正在识别图片中的药品… 已读取药盒文字，正在结合权威药品库做识别…"
2. 第二段（个性化承接）："结合 〔家人姓名〕 的健康档案，我重点帮你分析一下是否适合 TA 服用…"

> 文字必须**逐字流出**，如果你看到大段文字"一次蹦出来"，请清缓存刷新一次，或反馈给我们。

### 步骤 4 · 查看识药卡片

播报结束后，在该气泡下方会出现一张「识药卡片」，自上而下包含：

1. ① **药品基础信息**：通用名 / 商品名 / 规格 / 剂型 / 厂家 / 批准文号
2. ② **用法用量**（仅命中权威库时显示）：成人用法 / 儿童用法 / 服药时机
3. ③ **安全提示**（仅命中权威库时显示）：禁忌人群 / 不良反应 / 相互作用 / 特殊人群
4. ④ **个性化风险提示**：基于当前家人档案给出
   - ✅ **可以服用** — 未发现明显冲突
   - ⚠️ **谨慎服用** — 命中"在服其他药 / 慢病 / 特殊人群"等需注意项
   - ⚠️ **不建议服用** — 命中过敏史等强禁忌

卡片**最底部**固定 3 个按钮：

- **+ 加入用药计划**（主按钮，高亮蓝色）
- 🔍 **查看药品详情**
- 📷 **重新拍照**

> 当个性化风险等级为「不建议服用」时，主按钮自动置灰为「存在用药风险，无法加入」，无法点击。

### 步骤 5 · 加入用药计划

1. 点击「+ 加入用药计划」→ 跳转到「新建用药计划」表单页，药品名会自动填好。
2. 填完保存后**返回 AI 对话页**：
   - 刚才的识药消息和卡片完整保留；
   - 刚才那张卡片上的主按钮自动变成「**已加入用药计划**」，灰色不可点；
   - 即使刷新页面、换台设备登录，状态依然保留。

### 步骤 6 · 重新拍照 / 查看详情 / 中途切换家人

- 点「📷 重新拍照」：会提示从下方拍照入口重新提交；新识药结果**追加**在对话末尾，**不会替换**前一张。
- 点「🔍 查看药品详情」：跳「用药计划列表」并按药名筛选；返回后历史完全保留。
- 任意按钮跳转返回 AI 对话页，**对话历史 100% 保留**。
- 主动切换家庭成员或点击顶部「新建对话」会开新会话（行为同旧版）。

### 步骤 7 · 历史会话时间显示

打开侧边栏「历史会话」，刚发生的会话现在显示「**刚刚 / 几秒前**」，**不再误显示为"8 小时前"**。

## 注意事项

1. **图片要清晰**：药盒正面、文字可读、光线足，否则系统会让你重拍。
2. **务必先选家人**：识药结果的"个性化风险"完全基于当前选中家人的档案，错选成自己可能给孩子按成人剂量。
3. **未命中权威库时**：用法用量 / 安全提示等模块会隐藏，避免编造禁忌；可点「查看药品详情」自行查阅。
4. **「已加入用药计划」是云端持久化**：换设备/重装小程序后仍然保持。
5. **60 分钟空闲超时**：到点不会立即清空，下次再发消息时才开新会话；如要主动开新会话，点击对话栏的「新建对话」入口。
6. **时区**：服务端统一存 UTC，前端按你浏览器/手机的本地时区显示。在中国看到的是北京时间，去其他时区不会再有 8 小时偏移。

## 验收对照

| PRD 验收点 | 服务端实测 |
|------------|------------|
| 上传图片后 0.5s 内出现"正在识别中…" | ✅ 立即吐出 `event: progress`，前端 0.5s 内渲染占位气泡 |
| 文字逐字流式吐出 | ✅ 后端按 6 字符一片 + 40ms 间隔 SSE 推送 |
| 流式包含「识别进度 + 个性化承接」两段 | ✅ 两段播报型文案在 done 之前完整推完 |
| 卡片含 4 个模块 | ✅ DrugIdentifyCard V4 渲染 ①②③④ 四块 |
| 按钮始终在卡片最底部 | ✅ 卡片 70vh 高，内容区可滚，按钮区固定在 `flex-shrink:0` 的底部容器 |
| 跳转返回保留历史 + 按钮变灰 | ✅ message_metadata.added_to_plan 持久化，列表接口回吐 |
| 时区"刚刚"不再显示"8 小时前" | ✅ 公网 smoke 实测 created_at 与 UTC now 差 1.9s，且带 `+00:00` 后缀 |
| 默认空闲超时 60 分钟 | ✅ SessionConfig.idle_timeout_minutes 默认值为 60 |
| 不破坏现有 AI 对话/家庭成员/历史等功能 | ✅ 本地 9 项新增测试 + 28 项 med-plan 回归全部 PASS；公网 ai-home / health-profile / api/chat/sessions 均可达 |
