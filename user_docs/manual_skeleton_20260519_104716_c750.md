## 功能简介

本次更新对 `/ai-home`（"小康" AI 健康助手首页）做了**首屏骨架屏改造**，目标是**彻底消除刷新时的"假数据 → 真数据"视觉跳变**。

### 改造前（问题）

刷新页面 (F5) 时：

1. 页面瞬间先以默认 state 渲染一套"占位 UI"，功能宫格区域会出现 **3 个不再匹配设计的旧兜底按钮**（AI 诊室 / 看报告 / 健康档案）；
2. 约 1 秒后，多个后端接口陆续返回，state 更新触发二次渲染；
3. 用户视觉上看到一次明显的"假数据→真数据"跳变，**最突兀的就是底部功能宫格**。

### 改造后（效果）

1. 刷新后立刻看到一套**像素级还原页面真实结构的灰色骨架屏**（含微光流动 shimmer 动画）；
2. 当**关键接口（功能宫格 + 咨询人）齐备**后，骨架屏在 200ms 内**淡出 + 真实内容淡入**完成切换；
3. **绝不再出现** 3 个老兜底按钮（AI 诊室 / 看报告 / 健康档案）；
4. 弱网环境下骨架屏的微光流动持续显示，最长保护 **5 秒**；
5. 5 秒仍未拿到关键数据 → 骨架屏内显示**"加载失败，点击重试"卡片**，用户可一键重试。

## 使用说明

### 一、最佳体验入口

1. 在手机浏览器打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
2. 完成登录后进入 AI 健康助手首页
3. 用手机或浏览器的**下拉刷新 / F5 / Ctrl+R** 重新加载该页面，体验骨架屏

### 二、骨架屏组成（与真实页面 1:1 对应）

刷新瞬间，您将看到由灰色色块组成、带微光流动效果的整页骨架屏，从上至下：

| 区域 | 骨架形状 |
|------|----------|
| 顶部条 | 左侧汉堡圆/方块 + 中间"咨询人"胶囊 + 右侧 ⋯ 菜单 |
| 欢迎语 | 一长一短两行灰条 |
| 健康贴士卡 | 完整等高（约 130px）灰色圆角卡 |
| **功能宫格** | **2 行 × 3 列**，每格 = 一个圆形图标占位 + 一行文字占位 |
| 推荐问 | 3 条胶囊形（圆角矩形） |
| 底部输入框 | 长条胶囊 + 右侧"发送"按钮 |

> 所有灰条均自带从左到右循环的浅色高光（shimmer），让用户感知"系统正在加载"。

### 三、骨架屏消失时机

| 网速 | 体验 |
|------|------|
| 强网（4G/Wi-Fi） | 通常 200~600ms 内骨架屏淡出，平滑切到真实页面 |
| 慢速 3G | 骨架屏停留期间 shimmer 一直流动，关键数据回来后平滑淡入 |
| 关键接口超过 **5 秒**仍未返回 | 骨架屏中央出现 "⚠️ 加载失败 [点击重试]" 卡片 |

点击"点击重试" → 骨架屏回到 loading 态，重新发起关键接口请求。

### 四、关键接口分类

> 本次改造严格区分"关键 / 次要"接口，决定骨架屏何时消失：

**关键接口**（齐备后骨架屏才会消失）：

- `GET /api/function-buttons` — 功能宫格按钮
- `GET /api/family/members` — 顶部咨询人

**次要接口**（不阻塞首屏，失败静默隐藏对应区块）：

- `GET /api/ai-home-config` — 欢迎语 / 推荐问配置
- `GET /api/h5/home-banners` — 顶部 banner
- `GET /api/medication-reminder/badge` — 铃铛红点
- `GET /api/health-plan/today-todos` — 今日健康任务
- `GET /api/app-settings/chat-idle-timeout` — 闲置超时
- `GET /api/v1/notifications/unread-count` — 未读数
- `GET /api/orders/unified/counts` — 订单数
- `GET /api/chat/sessions?limit=1` — 最近会话
- `GET /api/user/font-setting` — 字号

3. **核心观察点**：
   - 骨架屏中的功能宫格是否为 6 个灰色方块（2×3）
   - 切换瞬间是否出现"AI 诊室 / 看报告 / 健康档案"3 个老兜底按钮——**应当不会出现**

2. **应当只触发胶囊区域的局部 loading，不触发整页骨架屏**

## 注意事项

1. **本次改动仅限 H5 端**：小程序、安卓 APP、iOS APP 不在本次改造范围内。
2. **骨架屏只在首屏出现**：首屏加载完成（一次）后，会话切换、消息收发、咨询人切换等交互均**不会**再触发骨架屏，体验与之前一致。
3. **像素级对齐 = 1:1**：骨架屏与真实页面元素尺寸、位置 1:1，淡入瞬间元素不会位移（如发现位移，可能是后续 UI 改版后骨架屏未同步，请反馈）。
4. **5 秒兜底**：如果某个用户网络环境长期 5 秒内拿不到关键接口，会一直看到"加载失败"卡片——这通常是网络/服务端问题，重试或换网络即可。
5. **次要接口失败静默**：例如推荐问 (`/api/ai-home-config`) 接口异常时，仅推荐问区块会静默隐藏，首页其余部分仍正常显示，**不会再出现任何老的兜底假数据**。
6. **历史缓存 `localStorage`**：本次改造未触碰功能宫格的 5 分钟本地缓存逻辑——首次进入 + 缓存命中时仍可秒开。骨架屏仅控制"接口未返回前的视觉态"。
