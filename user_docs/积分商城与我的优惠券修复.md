## 使用说明

### 一、商城列表【立即兑换】按钮按原因置灰（BUG-2）

**入口**：积分商城 → 商品列表

**改造内容**：商品卡片底部的【立即兑换】按钮，会根据后端返回的"是否可兑换"状态自动呈现：

- ✅ **可兑换**：实心高亮按钮，文案"立即兑换"，点击进入商品详情页直接兑换
- 🚫 **不可兑换（按优先级取最高）**：灰色描边按钮（**仍可点击**），文案根据原因显示：
  - "已下架" — 商品已下架/停用
  - "未开始" — 未到开始时间
  - "已结束" — 已过结束时间
  - "已兑完" — 库存为 0 / 已兑完
  - "已达上限" — 已达单人兑换次数上限
  - "积分不足" / "还差 XX 分" — 当前积分不够

**操作建议**：点击灰色按钮仍可进入商品详情页，详情页会展示更详细的不可兑换原因。

### 二、兑换记录按钮拆分：查看券 + 去使用（BUG-1 / OPT-4）

**入口**：积分商城 → 兑换记录 → 优惠券记录

**改造内容**：每条优惠券兑换记录的右下区域，原先单一【查看我的券】按钮（点击会跳 404）已替换为两个按钮：

| 按钮 | 文案 | 样式 | 显示条件 |
|------|------|------|----------|
| 按钮 1 | **查看券** | 次按钮（描边） | 始终显示 |
| 按钮 2 | **去使用** | 主按钮（实心高亮） | 仅当券状态 = 可用时显示 |

**【查看券】跳转**：
- 跳到「我的 → 我的优惠券」页面
- 自动定位到"**可用**"Tab
- 对应券卡片**自动滚动到可视区** + **高亮闪烁 1.5 秒**（淡黄色描边动画）

**【去使用】跳转**：
- 跳到该券适用的服务列表（详见第三节）
- 顶部带横幅标识"当前已选优惠券：XXX"
- 进入服务详情下单时，结算页**自动勾选**该券

### 三、我的优惠券-可用 Tab 新增【去使用】按钮（OPT-1）

**入口**：我的 → 我的优惠券 → 可用 Tab

**改造内容**：每张可用券卡片右下角新增【去使用】主按钮（其他 Tab：已使用 / 已过期 / 已退款 不显示此按钮）。

**点击后行为**：根据券的 scope 跳转到不同的服务列表：

| 券的 scope | 跳转目标 | 顶部横幅 |
|-----------|---------|----------|
| 指定服务（scope=product）| 服务列表（按券适用商品过滤）| "当前已选优惠券：XXX" |
| 指定品类（scope=category）| 服务列表（按券适用品类过滤）| 同上 |
| 全场通用（scope=all）| 全部服务列表 | "该券全场通用" |

**横幅交互**：右上角"×"按钮可关闭横幅 UI，但**不会清除该券的选择上下文**——继续浏览并下单时，结算页仍会自动带上这张券。

### 四、"免费试用券" → "免费体验券" 全局改名（OPT-2 / OPT-3 / OPT-5）

**改造内容**：
- 用户端 H5 / 小程序 / APP 三端，所有原"免费试用券"字样全部改为"**免费体验券**"
- 管理后台「商品体系 → 优惠券管理」中：
  - 优惠券类型下拉选项："免费试用" → "**免费体验**"
  - 列表类型列、详情类型字段、表单回显，全部同步
- 推送 / 邮件 / 短信 模板文案也已同步改名
- 用户端原先错误显示的英文 `free_trial` 字样已全部汉化

> 💡 **数据库枚举值 `free_trial` 不变**，历史数据正常显示，只改前端展示文案，无任何数据迁移风险。
