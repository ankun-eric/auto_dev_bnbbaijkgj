## 功能简介

本次迭代完成了「体检报告解读 / 报告对比」对话化功能的**接口规范化**升级：

- **新增 6 个符合 Restful 命名规范的接口**，让前端调用的路径更加清晰直观
- **扩展 2 个既有接口**，让聊天会话详情一次性返回"报告 + 咨询人档案"等上下文信息
- **OCR 批量识别**在完成"体检报告识别"后会自动创建一次 AI 解读会话，前端只需一次请求就能拿到 `report_id` 和 `session_id`，大幅简化前端流程
- **老数据懒加载**支持，历史上传但未触发过 AI 解读的报告，可以通过 `ensure-session` 一键补齐会话并继续体验
- **下线 3 个老接口**（返回 410 Gone），引导调用方切换到新的对话化流程

所有旧接口仍然保留兼容（内部复用相同实现），**已发布的前端 App / 小程序不会被打断**；推荐新接入的调用方直接使用新路径。

## 操作步骤（面向终端用户）

### 一、AI 解读一份体检报告（推荐主流程）

1. 打开 H5 首页 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)，完成登录。
2. 进入「体检报告」模块，点击右下角「+」上传报告图片。
3. 选择本次报告对应的**咨询人**（家庭成员档案，如未创建可先新增）。
4. 点击【AI 开始解读】按钮。
5. 系统会自动：
   - 完成 OCR 识别
   - 创建 AI 解读会话
   - 跳转到**对话页**并以 SSE 流式输出首条 AI 解读（通常 3 秒内可见字）
6. 后续可在对话框中继续追问，例如：「这个指标为什么偏高？」「日常饮食要注意什么？」——AI 会保留上下文记忆连续回答。

### 二、对比两份报告的变化

1. 在首页「体检报告」模块点击右上角【🔄 报告对比】。
2. 先选择**咨询人**（只能对比同一个人的两份报告）。
3. 勾选 **2 份**报告，点击【开始对比】。
4. 系统会校验「两份报告必须属于同一咨询人」，校验通过后：
   - 创建一个 `report_compare` 类型的独立会话
   - 跳转到对话页并流式输出第一份"前后差异分析"
5. 同样支持继续追问「哪些指标变好了？」等自然语言问题。

### 三、查看某一份历史报告

1. 在列表中点击任意一份报告，进入**极简详情页**。
2. 页面顶部可查看/编辑报告标题（点击标题右侧的编辑图标）。
3. 底部有两个按钮：
   - 【💬 继续咨询】：跳回 AI 对话页，继续上次的解读会话。
   - 【🔄 找另一份对比】：以当前报告为锚点进入对比流程。
4. 对于历史较早上传、当时尚未开启 AI 解读的老报告，点击【继续咨询】时会**自动懒加载**创建一次 AI 会话，无需重新上传。

### 四、管理员：维护 AI 提示词模板

1. 打开后台管理 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)，使用管理员账号登录。
2. 进入「AI 管理 → 提示词模板」。
3. 本次新增的两类模板：
   - **体检报告解读**（`checkup_report_interpret`）：用于单份报告的 AI 首次解读
   - **体检报告对比**（`checkup_report_compare`）：用于两份报告的 AI 对比分析
4. 每次保存会自动产生一个新版本；可在列表查看/回滚历史版本。

## 接口改造对照表（开发者速查）

### 新增接口（推荐使用）

| 方法 | 路径 | 用途 | 流式 |
|---|---|---|---|
| POST | `/api/checkup/compare/create-session` | 创建报告对比会话（校验同咨询人） | 否 |
| GET | `/api/chat/sessions/{id}/first-message-stream` | 会话首条消息 SSE 流式输出 | 是（SSE） |
| POST | `/api/chat/sessions/{id}/messages-stream` | 用户追问消息 SSE 流式输出 | 是（SSE） |
| GET | `/api/checkup/reports/{id}` | 极简详情页数据 | 否 |
| PUT | `/api/checkup/reports/{id}` | 更新报告标题 | 否 |
| POST | `/api/checkup/reports/{id}/ensure-session` | 老数据懒加载创建 AI 会话（幂等） | 否 |

### 扩展接口

| 方法 | 路径 | 本次变更 |
|---|---|---|
| POST | `/api/ocr/batch-recognize` | 响应新增 `session_id` 字段；体检报告场景内部不再同步 analyze，而是懒加载创建 interpret 会话 |
| GET | `/api/chat/sessions/{id}` | 响应新增 `report_id` / `report_ids` / `family_member` / `reports_brief` 四个字段 |

### 下线接口（返回 410 Gone）

> 以下接口仍保留路由但已下线，调用会返回 HTTP 410。请切换到对话化流程：

| 方法 | 路径 | 替代方案 |
|---|---|---|
| POST | `/api/report/analyze` | 上传后改走 `POST /api/ocr/batch-recognize` + 返回的 `session_id` |
| GET | `/api/report/trend` | 在对话中追问"最近几次指标趋势" |
| GET | `/api/report/compare` | 使用 `POST /api/checkup/compare/create-session` |

### 提示词管理接口（复用现有）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/admin/prompt-templates` | 列出所有 |
| POST | `/api/admin/prompt-templates` | 新建（本次种子时使用） |
| PUT | `/api/admin/prompt-templates/{id}` | 更新（保存后自动建新版本） |

**本次新增种子**：
- `prompt_type = 'checkup_report_interpret'`
- `prompt_type = 'checkup_report_compare'`

## 注意事项

- **鉴权**：除了前端页面，所有 `/api/*` 接口都需要携带 `Authorization: Bearer <token>` 请求头。未登录访问会返回 HTTP 401（属正常）。
- **SSE 流式格式**：`first-message-stream` 与 `messages-stream` 返回 `text/event-stream`，每条事件为 `data: {json}\n\n` 格式；`json` 中 `type` 字段为 `delta`（逐字增量）、`done`（完成）或 `error`（出错）。
- **幂等性**：`POST /api/checkup/reports/{id}/ensure-session` 多次调用同一 `report_id` 会返回相同的 `session_id`，不会重复创建会话，可以安全重试。
- **对比校验**：`POST /api/checkup/compare/create-session` 必须传入恰好 2 份报告 id，且两份报告属于同一 `member_id`；不满足会返回 HTTP 400。
- **向后兼容**：旧接口 `/api/report/interpret/*`、`/api/report/compare/start`、`/api/member/{id}/reports` 仍然保留并正常工作；推荐新接入优先使用本次新增路径。
- **下线接口**：访问 `/api/report/analyze`、`/api/report/trend/*`、`/api/report/compare` 返回 HTTP 410 Gone，响应体会给出新接口的指引 URL。
