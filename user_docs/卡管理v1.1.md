## 功能简介

本次卡管理 v1.1 的核心改动有三块：

1. **admin 表单字段全面补齐**：按 PRD v1.0 原条款补齐 14 个字段（F01~F14），并对默认值、必填、联动逻辑全部钉死。最显著的变化是「有效期默认 365 天」「库存/同款限购旁注明留空 = 无限/不限」「卡内项目必须从已有商品库选」「商家专属卡有商家档案 ID 搜索器」。
2. **新增「卡面设置」配置组**：在 admin 表单中可视化配置卡面的 4 风格 + 8 色板 + 4 显示项，并提供右侧实时预览，所见即所得。
3. **H5 视觉与 admin 完全统一**：H5 的卡列表、卡详情、我的卡包 3 个页面直接读取 admin 端为该卡保存的卡面配置进行渲染，两端 100% 一致。

## 使用说明

### 一、admin：新建一张卡

1. 登录 admin 后台。
2. 进入「商品系统」→「卡管理」。
3. 点击页面右上角的「**新建卡**」按钮，弹出新建抽屉。
4. 抽屉左侧填写「基础信息」（F01~F14），右侧实时预览卡面。
5. 点击「保存」即创建为「草稿」状态；列表中可点「上架」让 C 端可见。

#### 1.1 基础信息字段说明

| 字段 | 默认 | 必填 | 联动/校验说明 |
| --- | --- | --- | --- |
| F01 卡名 | 空 | 是 | 最多 30 字符，输入框右下角有计数 |
| F02 卡形态 | 次卡 | 是 | 切换次卡 ↔ 时卡，下面的「总次数」/「频次」会即时切换 |
| F03 归属范围 | 平台通用 | 是 | 选「商家专属」会出现 F04 商家档案搜索器 |
| F04 商家档案 ID | 空 | F03=商家专属时必填 | 在搜索框中输入手机号/昵称即可下拉搜索现有商家 |
| F05 总次数 | 10 | F02=次卡时必填 | 仅次卡时显示 |
| F06 频次 | 空 | F02=时卡时必填 | 文本输入，例如「每周 3 次」 |
| F07 售价 | 空 | 是 | > 0 |
| F08 划线价 | 空 | 否 | 留空则前端不显示；填写时必须 > 售价 |
| F09 有效期（天） | **365** | 是 | 1~3650 |
| F10 库存 | 空 | 否 | 留空 = 无限发售（旁有提示） |
| F11 同款限购 | 空 | 否 | 留空 = 不限（旁有提示） |
| F12 续卡策略 | 叠加在原卡 | 是 | 仅产品规则字段，本期不影响业务流程 |
| F13 可用门店 | 全门店通用 | 是 | 「全门店通用」开关；关闭后多选门店灰显待第 2 期开放 |
| F14 卡内项目 | 空 | 是 | 多选已有商品（不允许手填） |

### 二、admin：使用「卡面设置」配置卡面

新建/编辑卡的抽屉底部新增「**卡面设置**」配置组，紧跟在「基础信息」之后：

- **卡面样式（4 风格单选）**：ST1 简约现代 / ST2 沉稳商务 / ST3 节日喜庆 / ST4 奢华尊享。
- **卡面背景（8 色板单选）**：BG1 米黄 / BG2 暖橙 / BG3 朱红 / BG4 深蓝 / BG5 天蓝 / BG6 鲜绿 / BG7 沉黑 / BG8 裸粉。**仅可从 8 色板选择，不可自定义颜色**。
- **卡面显示（4 项勾选）**：SH1 卡名（默认勾选）/ SH2 服务内容（默认勾选）/ SH3 价格含划线价（默认勾选）/ SH4 有效期（默认不勾）。

#### 实时预览

抽屉右侧固定一张「**实时卡面预览**」，您调整任何一个选项时，预览卡片即时刷新，所见即所得。

### 三、H5：查看卡列表 / 卡详情 / 我的卡包

H5 端**完全不另起一套视觉**，所有页面均直接读取 admin 端为该卡保存的卡面配置进行渲染：

- **`/cards`（卡专区）**：每张卡 = 一张可点击的"卡片"，使用 admin 配置的卡面渲染；点击进入详情。
- **`/cards/[id]`（卡详情）**：顶部展示一张「大卡」=admin 配置的卡面；下方再展示卡介绍、可用项目、使用规则。
- **`/cards/wallet`（我的卡包）**：每张已持有的卡渲染缩小版卡面 + 剩余次数 / 剩余天数。

## 注意事项

1. **8 色板临时 HEX**：当前 8 色板使用临时 HEX（如 BG1 米黄 #F4E5C2），色卡到位后由开发同学一处常量替换即可，前端和数据库无需调整。
2. **卡面字段默认值**：未指定时，新建卡默认 face_style=ST1、face_bg_code=BG1、face_show_flags=7（卡名+服务内容+价格全开）、face_layout=ON_CARD。
3. **不允许自定义颜色**：admin 表单不会暴露任何取色器；如需新增颜色须修改后端枚举与前端常量。
4. **信息布局**：本期统一固定为「信息在卡面上」（face_layout=ON_CARD），不提供二选一开关。
5. **店铺装修入口已彻底删除**：本次 PRD 明确要求不保留任何"店铺装修"相关文案。
6. **卡管理仅 admin 可见**：店长/商家侧无任何卡管理入口；H5 用户只能浏览/购买（购卡功能将于第 2 期开放）。
7. **强制刷新缓存**：如果在浏览器看不到最新效果，请**强制刷新**（Windows: Ctrl+F5；macOS: Cmd+Shift+R），绕过浏览器/CDN 缓存。
