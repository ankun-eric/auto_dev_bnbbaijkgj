# 健康自查功能优化 V3 - 用户体验手册

> **版本**：v3.0
> **发布日期**：2026-05-21
> **适用端**：管理后台 / H5 / 微信小程序 / Flutter APP
> **本次解决问题**：4 大用户反馈 Bug（自动下一步、详情页空、重新填写、找医生 CTA 404）

---

## 一、本次优化概述

健康自查 V3 集中解决了 **4 个核心体验断链**：

| Bug | 现象 | 修复后体验 |
|---|---|---|
| Bug 1 | 单选题选完后必须手动点「下一步」 | 选完立刻自动跳下一题，**最后一题不会自动提交** |
| Bug 2 | 详情页核心分析全空 + 顶部"本人"文案错误 | 提交后等待 AI 生成完成才能进详情；详情页显示「本人」或「家人姓名（关系）」 |
| Bug 3.1 | 「重新填写」按钮跳回首页后卡片消失 | 改为「返回」按钮，**保留聊天卡片和会话状态** |
| Bug 3.2 | 「找医生咨询」按钮 404 | 后台可配开关 + 文案 + 跳转类型/值，未配置则**不显示** |

## 二、面向终端用户的新体验

### 2.1 H5 / 小程序 / APP 答题流程

1. 在 AI 对话首页选择「健康自查」按钮，进入抽屉式答题。
2. **单选题选完立即跳下一题**（无需点「下一步」）。
3. 多选 / 填空 / 最后一题仍保留「下一步」「提交」按钮。
4. 提交后回到对话页，会出现一张「健康自查结果卡片」：
   - 「查看详情」按钮**初始为灰色**，文案显示「分析中...」。
   - 等待 5~30 秒，AI 完成解读后，按钮**变绿可点**，文案恢复「查看详情」。
   - 若超时或异常，按钮变为「**重试解读**」，点击重新生成。

### 2.2 详情页（健康自查结果）

- **顶部摘要**：
  - 「本次回答结合 **本人** 的健康档案」（本人态）
  - 或「本次回答结合 **妈妈（张红）** 的健康档案」（家人态）
- 若该家人**档案信息较少**，会出现黄色提示「档案较少，建议补全 → [去补全]」。
- 中间 6 大区块全部有内容：评估总览、AI 解读、居家建议、就医警示、推荐商品等。
- **底部按钮**：
  - 「**返回**」：回到 AI 对话页，**完整保留聊天卡片**（包括本次问卷卡和结果卡）。
  - 「**找医生咨询**」（若管理员已配置）：按配置类型自动跳转。

### 2.3 「返回」按钮的智能降级

- 普通场景：直接回到上一页，保留所有上下文。
- 复制 URL 在新窗口打开后点「返回」：自动跳回 AI 首页（兜底）。

## 三、管理员操作指南

### 3.1 进入「功能按钮管理」

打开管理后台（[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/function-buttons](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/function-buttons)），看到按钮列表。

### 3.2 配置「自动下一步」（Bug 1）

1. 找到「健康自查」按钮，点击「编辑」。
2. **容器选择**：必须是「抽屉」。
3. **每页题数**：必须是 1。
4. **自动下一步开关**：仅当上述两个条件满足才能勾选。
5. 保存后，H5 / 小程序 / APP 三端**立即生效**（无需发版）。

### 3.3 配置「结果页 CTA 按钮」（Bug 3.2）

> 新增功能：在结果详情页底部增加一个**自定义跳转按钮**。

1. 在按钮列表，找到任意「问卷类型」按钮，点击操作列的「**结果页配置**」。
2. 弹出对话框，4 个字段：
   - **开关**「在结果详情页显示 CTA 按钮」：默认关闭。
   - **按钮文案**：最长 16 字符，例如「在线问诊」「联系医生」。
   - **跳转类型**（下拉）：
     - `H5 路径`：例如 `/services/consult`
     - `外部 URL`：例如 `https://m.example.com/doctor`
     - `小程序原生页`：例如 `/pages/doctor/index?id=123`（**只在小程序端显示**，H5 / APP 自动隐藏）
     - `医生 ID`：例如 `42`（自动跳到对应端的医生详情页）
     - `科室 ID`：例如 `7`（自动跳到对应端的科室详情页）
   - **跳转值**：根据上面类型填写。
3. 点击「保存」。
4. **关闭开关**即可让按钮立即从各端消失，无需发版。

### 3.4 跨端跳转适配规则（自动）

| 类型 | H5 端 | 小程序端 | Flutter APP |
|---|---|---|---|
| `H5_PATH` | `router.push(value)` | `webview` 打开 | `Navigator.pushNamed` |
| `EXTERNAL_URL` | `window.open` | `webview` | `url_launcher` |
| `MINIPROGRAM_PATH` | **不显示** | `wx.navigateTo` | **不显示** |
| `DOCTOR_ID` | `/doctor/{id}` | `/pages/doctor-detail` | `/doctor` 路由 |
| `DEPARTMENT_ID` | `/department/{id}` | `/pages/department-detail` | `/department` 路由 |

## 四、注意事项

1. **答题中途切换咨询人**：必须重新答一份，否则 subject 信息会与上次不一致。
2. **重试解读**：每次重试都会产生新的 AI 调用，建议先确认网络稳定。
3. **CTA 按钮的小程序原生页**：需要先在小程序后台「业务域名白名单」中配置目标域名（外链才能用 webview 打开）。
4. **历史问卷数据**：升级前已存在的答卷记录默认 `ai_status='done'`，不会触发轮询，行为完全兼容。

## 六、常见问题（FAQ）

**Q1：单选题选完没有自动跳？**
A：请检查后台配置是否同时满足「容器=抽屉、每页题数=1、自动下一步开关=开」。多选题、最后一题不会自动跳。

**Q2：「查看详情」一直灰着不能点？**
A：等待 5~60 秒。若超时，按钮会自动变成「重试解读」，点击即可。

**Q3：点「返回」回到首页后，本次问卷卡片不见了？**
A：本次升级已修复。如果是从外部 URL 打开详情页，按钮会兜底跳到 AI 首页。

**Q4：「找医生咨询」按钮看不到？**
A：管理员未配置或已关闭 CTA 开关。前往后台「功能按钮管理 → 结果页配置」开启。

**Q5：管理员配置后小程序端按钮没出现？**
A：检查跳转类型是否为「小程序原生页」（H5 / APP 会自动隐藏此类型）。

## 七、技术变更（开发参考）

- 数据库新增字段：
  - `questionnaire_answer.subject_kind/subject_member_id/subject_name/subject_relation/ai_status/ai_failed_reason/ai_full_interpretation/home_care_tips_json/red_flag_signals_json/archive_insufficient`（共 10 列）
  - `chat_function_buttons.result_cta_enabled/result_cta_text/result_cta_target_type/result_cta_target_value`（共 4 列）
- 新增接口：
  - `GET /api/questionnaire/answers/{id}/ai-status`
  - `POST /api/questionnaire/answers/{id}/retry-ai`
- `GET /api/questionnaire/buttons/{id}/render-meta` 顶层与 `button` 节点都新增 `result_cta` 节点。
- `GET /api/questionnaire/answers/{id}` 透传 `ai_status / result_cta / archive_insufficient / subject_label`。
- 后端 lifespan 自动执行幂等迁移 `prd_hsc_optim_v3_migration`。
