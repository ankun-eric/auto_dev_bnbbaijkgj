## 使用说明

### 一、顶部"小康"吸顶栏的新行为

进入 AI 对话首页后，您会看到顶部一条 48px 高的绿色吸顶栏，从左到右依次是：

- 左：☰ 汉堡菜单（点击打开历史会话侧边栏）
- 中：**小康**（页面主标题，带未读消息红点徽标）
- 右：⋯ 更多菜单

**新行为（修复后）**：

- 任何时候，无论您在下方做什么（拖动消息列表、点击消息气泡、上下快速滑动产生惯性、长按、回弹等），**顶部栏都会保持完全静止，栏内三个图标的位置不会发生任何抖动或错位**。
- 滑到消息列表顶端再继续往下拉（"橡皮筋"回弹场景），顶部栏也不会被"顶出去一截"露出页面背景。
- 横竖屏切换时，顶部栏依然稳稳吸顶，元素对齐保持正常。

### 二、欢迎面板的新行为

**关键改动：** 欢迎面板（顶部菜单 + 欢迎语 + 九宫格 + 推荐题胶囊）只有一种合法的"自动收起"触发条件——**您在本次会话主动点击发送按钮、消息进入待发送队列后**，面板才会收起为右上角的圆形小康头像悬浮按钮。**任何其他动作都不会让面板收起**。

#### 进入页面：

| 场景 | 进入后欢迎面板状态 |
| --- | --- |
| 全新成员首次进入 | 完整展开 |
| 有历史会话的成员进入 | 完整展开（哪怕历史会话里有大量消息） |
| 切换家庭成员后进入 | 完整展开（每位成员都从"未发消息"状态开始） |
| 切换历史会话后进入 | 完整展开（哪怕该会话内有历史消息） |
| 从其他页（健康计划、订单等）返回首页 | 完整展开（除非本次会话内您已经发过消息） |

#### 进入后的操作：

- **静止等待 5 秒、10 秒甚至 1 分钟** → 欢迎面板始终完整展开，**绝不会"自动闪掉"**。
- **在消息列表区域上下滑动** → 面板不会被收起。
- **聚焦输入框、长按文字、做其他任何手势** → 面板不会被收起。

#### 面板收起的唯一触发器：

- **您在输入框输入文字、点击右侧发送按钮（或按回车）** → 此刻面板立即收起为右上角的圆形小康头像悬浮按钮，消息列表占据主屏。

#### 面板收起后的来回切换：

- **点击右上角的圆形小康头像悬浮按钮** → 面板再次完整下拉展开（覆盖在消息列表上方），顶部还会出现一个 "⌃ 收起" 圆形小按钮。
- **点击 "⌃ 收起" 按钮** → 面板再次收起为右上角小头像。
- 上述展开/收起来回切换可以无限次重复，但"自动收起"只会在您本次会话第一次主动发送消息时触发**一次**。

### 三、多场景一致性

无论您：

- 切换家庭成员（如从"本人"切到"老婆"），
- 切换历史会话（从侧边栏选择某个旧会话），
- 从其他页面（健康计划、商城、订单等）返回首页，

判断标准都是同一个：**本次会话是否已发过消息**。
- 没发过 → 进入页面时欢迎面板完整展开
- 已发过 → 进入页面时欢迎面板保持收起为右上角小头像

各成员、各会话之间的"已发消息"状态彼此独立，不会出现"A 会话发过消息，切到 B 会话被错误收起"的问题。

## 常见问题

**Q：之前进入首页 1~2 秒后欢迎面板会自动消失，我有时候反应慢都看不到欢迎语。这是 Bug 吗？**
A：是的，这个就是本次修复的 Bug-2。修复后欢迎面板会一直展开等您查看，不会再自动收起。

**Q：之前在消息列表上滑动时，顶部"小康"标题感觉有时候会上下抖一下，这正常吗？**
A：不正常，这是本次修复的 Bug-1。修复后顶部栏完全独立于消息滚动区，不会再受任何手势影响。

**Q：我点了"发送"消息后欢迎面板就消失了，怎么再看到它？**
A：点击页面右上角的圆形小康头像，欢迎面板就会重新完整展开。

**Q：切换到其他家庭成员后，欢迎面板又出来了，是不是 Bug？**
A：不是，这是设计上的正确行为。因为切到新成员就相当于开了新会话，本次会话还没发过消息，所以欢迎面板会重新展开等您输入。
