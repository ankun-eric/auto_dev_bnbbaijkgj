## 功能简介

本期把支付宝 H5 通道从"开发期沙盒桩"切换到"正式生产链路"。具体改动包括：

1. **真实收银台** — 后端在生成支付链接时，会调用支付宝官方 `alipay.trade.wap.pay` 接口，返回形如 `https://openapi.alipay.com/gateway.do?...` 的真实收银台 URL，浏览器跳转后由支付宝官方页面承载支付。
2. **真实资金到账确认** — 新增后端异步通知接口 `/api/payment/alipay/notify`：支付宝服务器在用户付款成功后会异步推送通知，后端做 RSA2 验签 + 金额对账 + 幂等处理，最终把订单状态从「待支付」推进为「待核销 / 待发货」等。
3. **同步回跳页轮询** — 用户从支付宝收银台同步回跳到 H5 支付成功页时，前端会做最长 ~10 秒的轻量轮询（间隔 1.5 秒），等待异步通知把订单状态翻新；超时则提示"支付结果以支付宝为准，请稍后刷新查看"。
4. **测试按钮升级** — 后台「支付配置 → 支付宝 H5」的「测试」按钮，从"参数完整性 + 解密自检"升级为真实调用支付宝 `alipay.trade.query` 接口，能立刻反馈"AppID/私钥/公钥配对是否正确""能否和支付宝网关通信"。
5. **配置兼容** — 管理后台原有支付宝 H5 字段（AppID、应用私钥、支付宝公钥/证书三件套、接入模式）**结构完全不变**，本期把"原本只做参数校验的字段真正用起来"。
6. **沙盒桩兜底** — 当 SDK 配置不完整时（开发自测场景），会自动回退到原沙盒桩，避免链路彻底不可用，但**生产配置完整后绝不会触发**。

## 使用说明

### 场景 A — 管理员：填写并测试支付宝 H5 配置

1. 用管理员账号登录后台，进入 [支付配置页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/finance/payment-config)。
2. 在「支付宝 H5」卡片中，依次填写：
   - **应用 AppID**：支付宝开放平台分配的 16 位数字。
   - **接入模式**：选择「公钥模式」或「证书模式」。
     - 公钥模式：填「应用私钥」+「支付宝公钥」两个字段。
     - 证书模式：填「应用私钥」+「应用公钥证书」+「支付宝根证书」+「支付宝公钥证书」四个字段。
   - 私钥/证书内容支持完整 PEM（含 BEGIN/END 头尾）或裸 base64 文本。
3. 点击「保存」，系统会自动加密敏感字段（AES-256-GCM）。
4. 点击「测试」按钮：
   - 出现 ✅ "测试通过：参数正确、网络连通、签名/验签正常" → 配置可用，可以放心启用。
   - 出现 ❌ "签名校验失败，请检查应用私钥/支付宝公钥是否一一对应" → 大概率是私钥与支付宝公钥不是同一对，重新粘贴。
   - 出现 ❌ "AppID 与配置不匹配" → 检查 AppID 是否填错。
   - 出现 ❌ "网络不通：无法连接支付宝网关" → 检查服务器出网防火墙是否放通 `openapi.alipay.com`。
5. 测试通过后，把「支付宝 H5」开关切到「启用」状态。

### 场景 B — C 端用户：浏览器中下单并真实支付

1. 在 H5 用户端选商品 → 进入结算页 → 选择「支付宝」。
2. 点击「立即支付」，浏览器会自动跳转到支付宝官方收银台（地址栏可看到 `openapi.alipay.com/gateway.do?...`）。
3. 在支付宝收银台用支付宝 App 扫码或登录支付（**这是真实扣款**，请确认订单金额和商品名）。
4. 支付完成后，支付宝会自动回跳到 H5「支付成功」页：
   - 在该页停留约 1~10 秒，期间页面会显示"正在确认支付结果，请稍候…"。
   - 一旦后端收到支付宝异步通知，订单状态翻为「已支付/待核销/待发货」，页面会展示绿色对勾 + 订单详情。
5. 如果 ~10 秒后状态仍未翻新，会出现提示"支付结果以支付宝为准，请稍后刷新查看"，并自动跳到订单详情页 — 这种情况一般是支付宝异步通知延迟，刷新订单详情即可看到正确状态。

### 场景 C — 故意取消支付

1. 在支付宝收银台关闭浏览器或返回。
2. 订单保持「待支付」状态，**未发生扣款**。
3. 30 分钟后订单按既有未支付超时治理策略自动取消（这与本期改动无关，沿用既有逻辑）。

### 场景 D — 0 元订单（优惠券全额抵扣）

零元订单**不会**走支付宝收银台，仍走原有「免支付确认」分支，体验完全不变。

## 注意事项

1. **生产网关固定**：后端代码内写死支付宝**生产环境**网关 `https://openapi.alipay.com/gateway.do`，本期不提供管理后台「环境切换」开关。如未来需要切换沙盒，需要修改源码并重新部署。
2. **真实扣款**：付款页面的扣款是真实的资金交易，请管理员先用一笔小额订单（建议 0.01 元）端到端验证，再开放给全量用户。
3. **必须开通「手机网站支付」能力**：支付宝开放平台对应应用必须签约「手机网站支付（alipay.trade.wap.pay）」产品，否则即使测试按钮通过，下单时仍会被支付宝拒绝。
4. **回调白名单**：支付宝开放平台「应用接口加签方式」「回调白名单」必须配置为本项目部署的真实域名（即上方访问链接表中的 `https://newbb.test.bangbangvip.com`），否则异步通知发不到后端。
5. **资金以异步通知为准**：订单状态推进的唯一可信来源是后端收到的支付宝异步通知（`/api/payment/alipay/notify`）。同步回跳页只用于让用户感知，不能直接据此发货。
6. **幂等保证**：同一笔订单即便支付宝重发多次异步通知，订单状态只会被推进一次，不会重复发货 / 重复加积分。
7. **金额对账**：异步通知中的 `total_amount` 必须与本系统订单实付金额一致（允许 0.01 元浮点误差），任何不一致都会被拒绝并要求支付宝重发。
8. **签名校验**：所有异步通知都强制做 RSA2 验签，验签失败一律拒绝。
9. **测试按钮非充值**：「测试」按钮只调用支付宝订单查询接口，**不会产生任何真实扣款**，可以放心多次点击。
10. **密钥安全**：管理后台「支付配置」中的私钥/证书内容在 DB 中以 AES-256-GCM 加密存储，UI 仅展示末 4 位掩码；保存时若敏感字段留空则保留旧值（避免误删）。
11. **沙盒桩兜底**：仅当 `is_complete=false` 或解密失败时才会兜底回退到沙盒桩 URL（开发期/未配置场景），生产配置完整后此兜底不会触发。

## 风险与回退

如上线后发现支付宝接入异常：

1. **第一时间禁用通道**：管理后台「支付配置 → 支付宝 H5」一键关闭开关，C 端 H5 选项立刻消失，用户回落到其它支付方式或暂时无法支付（避免错单）。
2. **代码兜底**：后端代码内保留了沙盒桩函数 `_build_sandbox_pay_url`，且当 SDK 装配失败时会自动回退；紧急时可在源码层删一行 try/except 强制走兜底，避免完全不可用。
3. **回滚部署**：本期改动文件清单 9 个（5 修改 + 4 新增），可通过 git revert 单 commit 回滚到上一版本（v10），上一版本的支付链路仍是 H5 沙盒桩可用。
