## 功能简介

本次改造把「AI 对话模式首页配置」页面从**伪 Tab 锚点滚动**升级为**真 Tab 切换**：

| 维度 | 改造前 | 改造后 |
|---|---|---|
| 页面长度 | 11 张卡片全摊开 | 仅显示当前 Tab 卡片 |
| Tab 行为 | 点击只是滚动到对应卡片 | 真切换：仅渲染当前 Tab |
| 保存粒度 | 全页面"全部保存" | 每个 Tab 独立保存 |
| 未保存提示 | 无 | Tab 标题旁 ● 红点实时提示 |
| 切 Tab 误操作 | 直接丢改动 | 弹「保存并切换 / 放弃 / 取消」三选确认 |
| 重置按钮 | 单个卡片重置 | 每个 Tab 一键重置 + 二次确认 |

新页面共 **6 个 Tab**，按主题归位：

1. **欢迎区** — 主副标题、问候语、头像
2. **首屏内容** — 健康贴士轮播 + 空对话占位 + 推荐问列表
3. **功能宫格** — 7 字段宫格项（最多 6 项）
4. **输入栏** — 占位符、语音、家庭成员咨询胶囊
5. **会话策略** — 7 字段策略 + 空闲超时 + 空会话引导
6. **全局开关** — 9 模块开关 + 浮动健康打卡按钮 + 顶栏与品牌（兼容旧 H5）

## 使用说明

### 操作入口

打开 Admin 后台 → 左侧菜单「首页配置」→「**AI 对话模式首页配置**」。

或直接访问：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-settings/ai-home-config/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-settings/ai-home-config/)

### 1️⃣ 切 Tab：只看自己关心的模块

页面顶部是 6 个 Tab 横排标签。点击任一 Tab，**只**渲染该 Tab 对应的卡片组，避免在长页面里反复滚动。

进入页面默认停留在 **Tab 1「欢迎区」**，刷新页面也回到 Tab 1（不记忆上次位置）。

### 2️⃣ 修改字段：实时显示「未保存」红点

任何字段一旦被修改：
- 该 Tab 标题右上角出现 **● 红点**
- 页面底部吸底操作栏「重置本 Tab」「保存本 Tab」按钮**激活可点**

```
[欢迎区]   [首屏内容 ●]   [功能宫格]   [输入栏 ●]   [会话策略]   [全局开关]
                  ↑                          ↑
              该 Tab 有未保存改动        该 Tab 有未保存改动
```

### 3️⃣ 保存本 Tab：仅提交当前 Tab 字段

点击底部吸底栏的「**保存本 Tab**」按钮：
- 校验通过 → 调用后端 PATCH 接口（按 Tab 涉及的模块顺序逐一保存）
- 顶部显示 ✅「**保存成功**」绿色 Toast，2 秒自动消失
- Tab 标题红点消失，按钮变灰

> **校验失败**会弹出红色 Toast：`字段「XXX」校验未通过：[具体原因]`，并自动滚动定位到首个错误字段。

### 4️⃣ 重置本 Tab：恢复为上次保存值

点击底部「**重置本 Tab**」按钮：
- 弹出二次确认框：`确定要放弃当前修改并恢复为上次保存的值吗？`
- 选择「**确认重置**」→ 该 Tab 所有字段瞬间恢复，红点消失
- 选择「**取消**」→ 关闭弹窗，改动保留

> 「重置」**只影响当前 Tab**，不影响其他 Tab 的草稿。

### 5️⃣ 切 Tab 时未保存：三选确认框

如果当前 Tab 有红点但你点了别的 Tab，弹出确认框：

| 按钮 | 行为 |
|---|---|
| **保存并切换**（蓝色主按钮） | 先保存本 Tab → 成功后跳到目标 Tab，失败则停留 |
| **放弃修改并切换**（红色描边） | 直接丢弃当前 Tab 草稿 → 跳到目标 Tab |
| **取消**（普通按钮） | 关闭弹窗，停留在当前 Tab，改动保留 |

> **跨 Tab 草稿互不干扰**：在 Tab A 的改动不影响 Tab B 的红点状态。
> **多 Tab 同时有改动**也是支持的，每个 Tab 各自管理自己的草稿。

### 6️⃣ 各 Tab 字段速查表

| Tab | 主要字段 | 校验规则 |
|---|---|---|
| 1 欢迎区 | 主标题（含 `{昵称}` 占位符）、副标题、头像、问候语数组 | 主标题 ≤30 字、副标题 ≤50 字 |
| 2 首屏内容 | 健康贴士开关 + 间隔（3~5s）、空对话占位、推荐问列表（最多 20 条） | 推荐问标题 ≤8 字 / 内容 ≤200 字 |
| 3 功能宫格 | 1~6 项，每项 7 字段（主文案/副说明/链接/图标/双色渐变/角标/启用） | 主文案 ≤8 字、副说明 ≤12 字、HEX 色须合法 |
| 4 输入栏 | 占位符、语音/TTS 开关、家庭成员咨询胶囊（含 `{name}` 占位符） | 模板必须包含 `{name}` |
| 5 会话策略 | 单次回答字数（100~5000）、单日免费次数（1~999）、回复风格、上下文记忆轮数、免责声明 | 字数/次数/免责声明各有上限 |
| 6 全局开关 | 9 模块开关 + 浮动打卡按钮配置 + 顶栏与品牌兼容配置 | 跳转链接须 `/` 开头 |

## 注意事项

1. **不记忆 Tab 状态**：进入页面或刷新都默认回到 Tab 1「欢迎区」。这是有意设计，避免运营被昨天遗留的 Tab 干扰。
2. **草稿不持久化**：刷新页面会清空所有 Tab 的草稿。**任何重要改动请尽快「保存本 Tab」**。
3. **保存粒度按 Tab，不是按卡片**：例如 Tab 2 包含「健康贴士」「空对话占位」「推荐问列表」三个卡片，点「保存本 Tab」会**一次性按顺序保存这三个模块**到后端，互相独立但作为一个 Tab 整体提交。
4. **保存失败不会清空草稿**：如果是字段校验失败，错误字段会标红 + 自动滚动定位；如果是网络/后端错误，Toast 会显示具体原因。修复后再次点「保存本 Tab」即可。
5. **顶栏与品牌**已并入 Tab 6「全局开关」**最末位**，作为兼容旧 H5 的配置不主动引导用户配置。设计图中默认 `visible=false`（顶栏隐藏）。
6. **同一字段被另一管理员同时修改**：后端会做版本检测，如出现冲突，请刷新后重新编辑。
7. **多 Tab 编辑互不干扰**：你可以先在 Tab 1 改一些、再切到 Tab 4 改一些（中途会弹三选框，选「放弃修改并切换」可保留 Tab 4 的草稿继续编辑），但建议**改完一个 Tab 立刻保存**。
8. **保存后立即生效**：所有用户端（H5 / 小程序 / Flutter App）的 AI 首页会在用户下次刷新时拉到最新配置（H5 端有 5 分钟本地缓存）。
9. **操作日志可追溯**：右上角「操作日志」可查看每次保存的字段差异、操作人、时间。
10. **不同角色权限**（按推荐方案）：超级管理员所有 Tab 可编辑；运营管理员 Tab 1-5 可编辑、Tab 6 仅查看；内容运营仅 Tab 1-3 可编辑；只读用户全部 Tab 可看不可改（保存按钮置灰）。

## 常见问题

**Q1：我修改了一些字段，但忘了保存就刷新了，能找回吗？**  
A：不能。刷新等于丢弃所有草稿。如需要"暂存"的能力，请先点「保存本 Tab」落库（后台支持版本回滚到任意历史时间点）。

**Q2：我在 Tab 2 的推荐问改了几条，现在想切到 Tab 3，但又怕丢，怎么办？**  
A：直接点 Tab 3 标签。系统会自动弹出三选框，选「保存并切换」最稳妥；或选「取消」继续在 Tab 2 改完再保存。

**Q3：「保存并切换」点完了，结果保存失败怎么办？**  
A：保存失败时**不会切换** Tab，并弹出错误 Toast（如校验未通过）+ 自动滚动到第一个错误字段。修复后可再次保存或点「取消」放弃。

**Q4：「重置本 Tab」会不会清空别的 Tab 的草稿？**  
A：不会。重置严格限定为**仅当前 Tab 涉及的模块字段**恢复到「上次成功保存」的值，其他 Tab 的草稿不受影响。

**Q5：Tab 标题旁的红点什么时候会消失？**  
A：当该 Tab 完成「保存成功」或「重置」后，红点立即消失。
