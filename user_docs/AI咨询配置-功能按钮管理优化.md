# AI 咨询配置-功能按钮管理优化 使用手册

> 版本：v1.0 · 发布日期：2026-05-15
> 适用对象：运营人员（管理后台）+ 终端用户（H5 / 小程序 / APP 端 `AI 对话首页`）

本次优化围绕"**让运营配置更简单、让用户点击更符合直觉**"，对 4 处进行了升级。本手册分两部分：**运营侧**怎么用、**终端用户侧**怎么用。

---

## 一、功能介绍

### 1.1 升级一览

| 升级点 | 之前 | 现在 |
| --- | --- | --- |
| 上传来源 | 相册 / 拍照 / 本机 / 微信 | **相册 / 拍照** |
| 卡片封面图 URL | 表单中存在，但运营基本不用、用户也看不到 | **已移除**，统一渲染「Emoji + 主题色底」头像 |
| AI 回复模式 | 拍照识药独有 3 档下拉，与「关联 Prompt 模板」并存 | **已移除**，三档固化为 3 个系统内置 Prompt 模板 |
| 胶囊点击行为 | 一律把胶囊文字当作"用户消息"发送 | **按按钮类型分发**：upload→ActionSheet、navigate→跳转、quick_ask→可编辑卡片、sdk_call→调 SDK |

### 1.2 三个系统内置 Prompt 模板

| 模板 Code | 模板名称 | 用途 |
| --- | --- | --- |
| `MED_RECOG_FULL` | 识药-完整分析 | 输出药品基本信息 + 用法用量 + 注意事项 + 与已有用药相互作用 |
| `MED_RECOG_ADVICE_ONLY` | 识药-仅用药建议 | 只输出怎么吃（用法用量 / 服用时间 / 注意事项） |
| `MED_RECOG_AUTO` | 识药-AI 自动判断 | 让 AI 根据用户提问深浅自适应输出 |

**特性**：
- 标记为「系统内置」，**不可删除**
- 可以一键"**复制为可编辑副本**"，作为运营自定义模板的起点

## 二、运营侧操作指引（管理后台）

### 2.1 访问入口

- 后台首页：<https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin>
- 功能按钮管理：<https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/function-buttons>
- Prompt 模板管理：<https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/prompt-templates>

### 2.2 配置 upload 类按钮（报告解读 / 拍照识药 / 文件上传 等）

1. 进入「功能按钮管理」→ 点击列表中任一 upload 类按钮的「编辑」按钮。
2. 表单字段相比旧版有 2 处变化：
   - **不再显示**「卡片封面图 URL」字段（无需填写，前端不会再使用此字段）。
   - **不再显示**「AI 回复模式」字段（识药按钮也不再单独有此选项）。
3. 「关联 Prompt 模板」是控制 AI 行为的**唯一字段**。识药场景下推荐选择：
   - 想要完整分析 → 选择 `识药-完整分析`
   - 只关心怎么吃 → 选择 `识药-仅用药建议`
   - 让 AI 自己判断深浅 → 选择 `识药-AI 自动判断`
4. 「Emoji 图标」字段直接决定用户端卡片头像，请填入一个能直观表达功能的 Emoji（如 💊 📄 ❤️）。
5. 「排序值」决定胶囊在用户端的展示顺序（**升序展示**），数值越小越靠前。
6. 保存后立即生效，无需用户重启 App / 刷新 H5 即可看到新顺序与新行为。

### 2.3 使用系统内置 Prompt 模板

1. 进入「Prompt 模板管理」页面。
2. 页面顶部新增「**系统内置识药模板**」区块，展示 3 个内置模板，并打有「系统内置」标签。
3. 点击模板右侧的「**复制为可编辑副本**」按钮：
   - 系统会立即生成一份**新模板**，名称形如 `识药-完整分析（副本）`。
   - 副本属于普通用户模板，可任意修改名称、内容、版本等。
   - 原系统内置模板**不会被修改**。
4. 副本生成后，可在 Prompt 模板列表中找到并继续编辑、关联到任意按钮。

### 2.4 胶囊顺序与显示

- 用户端「输入框上方胶囊条」的顺序**完全由后台「排序值」决定**（升序展示）。
- 不再有"使用频率排序"或"用户置顶"等动态策略。
- 调整排序后，用户下次进入 `AI 对话首页` 即可看到新顺序。

## 三、终端用户侧使用指引

### 3.1 访问入口

- H5 AI 对话首页：<https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home>

### 3.2 胶囊点击行为表

| 胶囊按钮类型 | 点击后行为 | 对话区是否留消息 |
| --- | --- | --- |
| **upload**（拍照识药 / 报告解读 / 拍照上传 / 文件上传） | 弹出选择面板：**「从相册选择」/「拍照」** → 选完图片后，对话区先出现一条「用户：图片缩略图」气泡 → AI 回复 | 是 |
| **navigate**（页面跳转） | 直接跳到目标页面（如「在线问诊」「医生列表」） | 否 |
| **quick_ask**（快捷提问） | 对话区出现一张**可编辑的「快捷提问卡片」**，预填了预设问题，可改可发可取消 | 是 |
| **sdk_call**（SDK 调用） | 直接调用对应能力（体质测评、挂号等） | 否 |

### 3.3 upload 操作步骤（以拍照识药为例）

1. 在 AI 对话首页，点击输入框上方的「拍照识药」胶囊。
2. 弹出底部选择面板，**只有 2 个选项**：
   - 从相册选择
   - 拍照
3. 选择/拍摄图片后，对话区会显示一条「用户：[图片缩略图]」消息。
4. AI 紧接着会按运营配置的 Prompt 模板（如「识药-完整分析」）给出回复。

### 3.4 快捷提问卡片操作（quick_ask）

1. 点击 `quick_ask` 类型的胶囊（如「血压怎么测才准？」）。
2. 对话区会出现一张卡片，结构如下：

   ```
   ┌──────────────────────────────────────────┐
   │  [💡] 血压怎么测才准                       │
   │  ┌────────────────────────────────────┐  │
   │  │ 血压怎么测才准？（可编辑）         │  │
   │  └────────────────────────────────────┘  │
   │                       [取消]   [发送]    │
   └──────────────────────────────────────────┘
   ```

3. 可在文本框内**任意修改**预填问题（最长 500 字，与输入框一致）。
4. 点击 **「发送」**：
   - 当前文本框内容会以「用户身份」发出。
   - 卡片置灰，不可再点。
   - AI 自动给出回复。
5. 点击 **「取消」**：
   - 卡片置灰，不再交互。
6. 如果你**没有操作**卡片就直接在底部输入框发送了其他消息：
   - 卡片仍保留为"待操作"状态但已变灰，避免脏数据。

### 3.5 navigate / sdk_call 胶囊

- 点击后**直接跳页 / 直接调用 SDK**，对话区不会留下任何新消息，符合"按钮即功能"的直觉。

## 四、注意事项

### 4.1 存量数据兼容

- 旧数据中可能残留 `source: ["local","wechat"]` 的来源勾选 —— **前端会自动忽略**这两个值，不会报错也不会出错弹层。
- 旧的「卡片封面图 URL」字段数据仍然保留在数据库，但**用户端不再渲染**，统一用 Emoji 头像。
- 旧的「AI 回复模式」字段（`ai_reply_mode`）仍然保留在按钮记录中，**仅作历史兼容**，新的逻辑只看「关联 Prompt 模板」。

### 4.2 拍照识药按钮的迁移

- 系统**自动**把存量「拍照识药」按钮的 `reply_mode` 映射到对应内置模板：
  - `full` → `MED_RECOG_FULL`
  - `medicine_only` → `MED_RECOG_ADVICE_ONLY`
  - `auto` 或空 → `MED_RECOG_AUTO`
- 仅在按钮**未绑定**任何 Prompt 模板时才会自动写入；如运营已手动绑定，**保留原绑定不动**。

### 4.3 Emoji 头像规格

- 自动渲染为 **40×40px，圆角 8px，背景为按钮主题色 10% 透明度**。
- 如未配置 Emoji，前端会兜底显示 ✨。

### 4.4 浏览器兼容

- H5 端最低支持现代浏览器（Chrome / Safari / Edge 近 2 年版本）。
- 拍照功能在 H5 通过 `<input type="file" accept="image/*" capture="environment">` 调起手机摄像头，桌面浏览器会退化为「选择本地文件」。

### 4.5 小程序 / APP 端

- 小程序端原本就只有「相册 / 拍照」二选，本次没有外观变化。
- Flutter APP 端补充了对 `quick_ask` 类型按钮的支持，点击后会将预设问题作为用户消息发出。

## 五、常见问题（FAQ）

**Q1：以前配的"AI 回复模式"丢了吗？**
A：不会丢，已**自动迁移**到「关联 Prompt 模板」。重新打开按钮编辑页时，「关联 Prompt 模板」字段已经选好了对应的内置模板。

**Q2：系统内置模板可以删吗？**
A：**不可以**删除。但可以「复制为可编辑副本」，副本随便改。

**Q3：胶囊顺序我已经在后台改了，为什么用户端没变？**
A：用户**下次进入 `AI 对话首页`** 即生效。如果用户当前正在 `AI 对话首页` 停留，则需要刷新或返回重进。

**Q4：拍照识药按钮和报告解读按钮表单为什么长得一模一样了？**
A：这是**特意为之**——本期需求 3 的目标就是统一二者表单，避免运营纠结"该填哪个 AI 字段"。

**Q5：quick_ask 卡片用户没点就关页面了，下次回来还在吗？**
A：会话级别保留，刷新或重新进入会话会消失。卡片始终是**一次性**的临时消息。
