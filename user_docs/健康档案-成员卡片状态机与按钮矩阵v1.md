## 功能简介

本次升级针对「健康档案 - 我守护的人」的 4 大顽疾——**5 个删档接口、3 套删除模型、双邀请入口、状态机不统一**——进行了系统性重构，实现了 **4 个统一**：

1. **状态统一**：成员卡片状态机收敛为 **7 种正向状态（S0 ~ S7）+ 3 种反向状态（R1 ~ R3）**
2. **入口统一**：废弃"无档案直接发邀请"入口，所有邀请必须先建档案
3. **按钮统一**：每种状态主按钮 1 个 + 次操作收纳菜单，文案规范一致
4. **接口统一**：删除收口到唯一接口，返回结构化 `reason_code` 错误码

## 7 种状态卡片速查（正向：我守护的人）

| 状态 ID | 标签 | 颜色 | 主按钮 | 次操作（… 菜单） | 备注 |
|---------|------|------|--------|--------------------|------|
| **S0** | 本人 | 蓝色 | 查看档案 | — | 不可删除 |
| **S1** | 守护中 | 绿色 | 查看档案 | 解除守护、邀请记录¹ | **无删除按钮**（必须先解除） |
| **S2** | 未邀请 | 灰色 | **去邀请** | 删除、邀请记录¹ | 唯一保留的"去邀请"入口（卡片上） |
| **S3** | 邀请中 | 橙色 | **查看邀请码** | 取消邀请、邀请记录¹ | **无删除按钮**（必须先取消） |
| **S4** | 已拒绝 | 红色 | **重新邀请** | 删除、邀请记录 | 重新邀请会自动 cancel 旧记录 |
| **S5** | 已过期 | 灰色 | **重新邀请** | 删除、邀请记录 | 同上 |
| **S6** | 已解绑 | 浅灰 | **重新邀请** | 删除、邀请记录 | 同上 |
| **S7** | 已取消 | 浅灰 | **重新邀请** | 删除、邀请记录 | 同上 |

## 核心操作步骤

### 一、查看档案列表

1. 打开浏览器，访问：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/archive-list/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/archive-list/)
2. 顶部统计区展示：
   - **已管理 X 人 / 上限 Y 人**（不含本人）
   - **还可添加 N 人**
   - **守护中 K 人**（仅 S1 状态计入）
3. 右上角「**+ 新增**」按钮：新建档案入口（唯一）
4. 下方为各成员卡片，每张卡片包含：头像、姓名、关系、状态标签、主按钮、更多菜单（⋯）

> 💡 与旧版的差异：
> - **顶部不再有「去邀请」大按钮**（入口 B 已废除）
> - **建档与邀请分两步**：先建档案，再决定是否邀请

### 二、新增档案 + 立即邀请

1. 点击顶部「**+ 新增**」按钮
2. 在弹出的抽屉中填写：
   - **姓名**（必填，1~20 字）
   - **与本人关系**（必填，如父亲/母亲/配偶/子女）
   - **手机号**（必填，11 位）
   - **性别**（必填，男/女）
3. 点击「**保存档案**」，档案立即创建（此时状态为 **S2 未邀请**）
4. 弹出「**立即去邀请？**」选择抽屉：
   - 点击「**立即邀请**」→ 自动生成邀请码并展示
   - 点击「**暂不邀请**」→ 关闭抽屉，列表中新增一张 S2 状态卡片，可随时点击「去邀请」按钮发起邀请

### 三、邀请家人（S2 → S3）

- 在档案列表中，找到状态为「**未邀请**」（S2）的卡片
- 点击主按钮「**去邀请**」
- 系统会自动生成邀请码 + 分享链接
- 弹出邀请码抽屉，点击「**复制邀请链接**」分享给家人
- 家人通过链接接受邀请后，该卡片状态变为 **S1 守护中**

### 四、查看 / 重新邀请（S4/S5/S6/S7）

- 状态为「已拒绝 / 已过期 / 已解绑 / 已取消」的卡片，主按钮均为「**重新邀请**」
- 点击后系统自动执行：
  - 旧的非 cancelled 邀请 → 全部置为 `cancelled`（保留审计痕迹）
  - 新增一条 `pending` 状态的邀请
  - 卡片状态变为 **S3 邀请中**

### 五、查看邀请码（S3）

- 状态为「邀请中」（S3）的卡片，主按钮为「**查看邀请码**」
- 点击后展示：邀请码 + 完整分享链接 + 剩余有效小时数
- 可复制链接分享给被邀请人

### 六、删除档案

- **仅在 S2、S4、S5、S6、S7 状态下，"更多菜单"中才有「删除」选项**
- 点击「⋯」→「删除」→ 二次确认 →「确认删除」
- 系统会同步软删 4 张表的相关记录：
  - `family_member`（档案主表）
  - `health_profile`（健康档案）
  - `family_invitation`（邀请记录 → cancelled）
  - `family_management`（守护关系 → removed）
- 删除成功后，配额立即释放

### 七、解除守护（S1 → S6）

- 仅在 S1 状态可见
- 点击「⋯」→「解除守护」
- 解除后状态变为 **S6 已解绑**，可重新邀请或删除

### 八、取消邀请（S3 → S7）

- 仅在 S3 状态可见
- 点击「⋯」→「取消邀请」
- 邀请记录置为 `cancelled`，状态变为 **S7 已取消**

## 注意事项

### ⚠️ 删除闸门规则

以下情况无法直接删除档案，需先处理对应业务：

| 阻塞条件 | 错误码 | 提示文案 | 处理方式 |
|----------|--------|----------|----------|
| 处于 S1 已绑定 | `HAS_ACTIVE_GUARDIANSHIP` | 请先解除守护关系 | 先点「解除守护」 |
| 处于 S3 邀请中 | `HAS_PENDING_INVITATION` | 请先取消邀请 | 先点「取消邀请」 |
| 名下有绑定设备 | `HAS_BOUND_DEVICE` | 请先解绑该成员名下设备 | 前往「设备管理」解绑 |
| 有在途服药计划 | `HAS_ACTIVE_MEDICATION` | 请先终止服药计划 | 前往「服药管理」终止 |
| 当日删除超 5 次 | `RATE_LIMIT_EXCEEDED` | 操作频繁，请明日再试 | 等到次日 |

### ⚠️ 配额规则

- **档案配额** = `count(family_member where deleted_at is null and is_self=false)`，**所有 S2~S7 状态都占名额**
- **已守护人数** = `count(family_management where status='active')`，**只有 S1 状态计入**
- **本人不计入档案配额**（不含本人）
- 删除档案后，配额立即释放

### ⚠️ 邀请有效期

- 每条邀请的有效期为 **24 小时**
- 24 小时未被接受 → 自动变为 S5「已过期」
- 已过期 / 已拒绝 / 已解绑 / 已取消 → 都可点「重新邀请」

### ⚠️ 入口收口（重要变更）

| 旧入口 | 新流程 |
|--------|--------|
| 健康档案顶部「去邀请」大按钮 | ❌ **已删除** |
| 档案列表顶部「去邀请」大按钮 | ❌ **已删除** |
| AI 首页 → 新建咨询人 | 建档案后弹「立即邀请」抽屉 |
| 卡片上「去邀请」（仅 S2 态） | ✅ 保留 |

### 后端新增接口（统一删除收口）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/family/member/state/list` | GET | 列表：返回当前用户所有 family_member 的状态机 state（S0~S7） |

| `/api/family/member/{member_id}/invite` | POST | 重新邀请（cancel 旧 + insert 新 pending） |
| `/api/family/member/{member_id}/unbind` | POST | 解除守护（S1 → S6） |
| `/api/family/member/admin/cleanup-orphan-invitations` | POST | 一次性清理孤儿邀请（管理员） |

### reason_code 字典

| reason_code | HTTP | 含义 | 用户文案 |
|---|---|---|---|
| `OK` | 200 | 删除成功 | 已删除 |
| `HAS_ACTIVE_GUARDIANSHIP` | 400 | S1 已绑定 | 请先解除守护关系 |
| `HAS_PENDING_INVITATION` | 400 | S3 邀请中 | 请先取消邀请 |
| `HAS_BOUND_DEVICE` | 400 | 名下有绑定设备 | 请先解绑该成员名下设备 |
| `HAS_ACTIVE_MEDICATION` | 400 | 有在途服药计划 | 请先终止服药计划 |
| `RATE_LIMIT_EXCEEDED` | 429 | 当日删除超 5 次 | 操作频繁，请明日再试 |
| `NOT_FOUND` | 404 | 档案不存在或已删 | 该档案不存在或已被删除 |
| `PERMISSION_DENIED` | 403 | 非档案所属用户 | 您没有权限操作该档案 |

## 常见问题

**Q：为什么旧的「去邀请」按钮不见了？**  
A：这是本次重构的核心改造之一。"无档案直接发邀请"入口已废除，所有邀请必须先建档案。这与华为运动健康、微信家庭群、平安好医生、京东健康等主流健康类产品对齐。

**Q：建档时填写的手机号有什么用？**  
A：手机号是邀请家人接受守护关系的关键依据。家人接受邀请后，系统会自动将该手机号匹配到对应账号，完成绑定。

**Q：我删除了档案，但「我守护的人」页面还能看到？**  
A：本次新增的「档案列表」（archive-list）是独立的 v2 页面，使用全新的状态机和统一删除接口。如果你想用旧版列表，可访问 `/health-profile/i-guard/`，但旧版的删除按钮在某些状态下可能与新版行为不一致——**建议统一使用新页面**。

**Q：删除后能恢复吗？**  
A：删除是软删（`status=deleted`），底层数据保留，但 H5 和后台均不再查询展示。如需恢复，请联系管理员通过数据库后台手动处理。
