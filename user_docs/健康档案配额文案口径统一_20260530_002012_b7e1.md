## 功能简介

本次更新对 **「健康档案 / 会员中心 配额数字与文案」** 进行了 **全产品口径统一**，解决了用户长期反馈的「同一页面两个不同数字」「文案累赘 / 看不懂」等问题。

核心变化：

- **后端 `max_managed` 字段**的语义统一为「**可管理的档案总数**」，前端不再做 `+1` 修正
- 前端**不再出现**「含本人 / 不含本人」这类解释性小字
- 健康档案首页本人 Tab 副标题改为更易懂的 **「已管理 X 人，还可添加 Y-X 人」** 格式，与档案列表顶部统计条对齐

> ✅ 本次修复**仅前端**，后端配额校验逻辑不变，存量套餐用户的实际可用容量不受影响。

## 使用说明

### 一、查看会员权益概览（顶部大数字格子）

1. 打开 H5 → 进入「**会员中心**」
2. 看到「**我的会员权益**」板块下方的 4 个大数字格子
3. **「可管理档案」格子**：现在显示后端 `max_managed` 的原值
   - 基础套餐示例：显示 **3**（修复前误显示为 4）
   - 旗舰不限套餐：显示 **不限**

### 二、查看可升级的套餐卡描述

1. 在会员中心向下滚动到「**升级享更多权益**」区域
2. 每个套餐卡的描述行现在显示：
   - 标准格式：**「可管理健康档案 3 人 · AI 外呼 30 次 · 紧急呼叫 3 次」**
   - 不限套餐：**「可管理健康档案 不限 · ...」**
3. 与上方大数字格子的「3」**保持完全一致**（修复前是 4 vs 3 不一致）

### 三、健康档案首页 · 本人 Tab 入口卡

1. 打开 H5 → 进入「**健康档案**」
2. 默认在「**本人**」Tab 下，可看到「**📋 档案列表**」入口卡
3. 副标题现在显示：
   - **未满**：「**已管理 1 人，还可添加 2 人**」
   - **已满**：「**已管理 3 人，还可添加 0 人**」
   - **不限套餐**：「**已管理 N 人**」（不显示「还可添加」）
4. 该副标题与点击卡片后进入的「档案列表」顶部统计条**口径完全一致**

### 四、档案列表 · 顶部统计条

- 顶部继续保持原有的「**已管理 X 人 / 上限 Y 人** · **还可添加 N 人**」展示
- 当 N=0 时点击「**+ 新增**」会提示：
  - **「档案配额已满（上限 3 人），请先删除现有档案或升级套餐」**
  - 不再出现「含本人」之类的字样

### 五、套餐权益对比表

1. 在会员中心继续向下滚动到「**会员权益对比**」表
2. 第一行「**可管理健康档案**」现在直接展示数字（如 3 / 5 / 不限），不再带「（含本人）」括号字样

## 注意事项

1. **后端容量未变更**：本次仅修复前端文案口径。后端配额校验逻辑保持不变，已购买的套餐用户实际可创建的档案数（本人 + 家人）与原先一致，无负向影响。
2. **存量数据无需迁移**：本次发布**零数据迁移、零接口变更**，纯前端发版即可生效。
3. **多端一致性**：本次只改动了 H5 前端（`h5-web/`），不影响小程序、安卓、iOS 等其他终端的代码与体验。
4. **缓存提示**：如打开页面后仍看到旧文案，请尝试**强制刷新**（PC：Ctrl + F5；移动端：清理浏览器缓存或重新进入小程序入口）。
5. **不限套餐展示**：旗舰不限套餐用户在所有页面会看到「**不限**」，不显示具体数字与「还可添加」字样。
