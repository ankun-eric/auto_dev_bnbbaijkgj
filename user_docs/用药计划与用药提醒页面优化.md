## 功能简介

本次对两个页面做了「外观统一 + 文案修正 + 故障修复」三类优化，不改变任何业务逻辑、打卡功能与数据：

1. **页面A（用药计划列表页）顶栏统一**：顶部蓝色顶栏改为整条天蓝色铺满全宽，白色标题居中，与项目其他二级页保持一致。
2. **页面A 标题改名**：顶部标题由「用药提醒」更正为 **「用药计划」**（这个页面本就是用药计划列表，原标题文字有误）。
3. **页面A 修复「已结束」报错**：此前每次点击「已结束」Tab 都会弹出「该列数据异常，请联系客服」。现已修复，能正常展示已结束的用药计划，或在无数据时显示「暂无用药计划」。
4. **页面B（用药提醒页）顶栏统一**：顶部由白底黑字的导航栏，统一改为天蓝色顶栏 + 白色居中标题。标题文字保持 **「用药提醒」** 不变。

## 使用说明

### 一、查看用药计划列表（页面A）

1. 登录后进入「健康档案」，点击进入用药相关入口，打开 **用药计划列表页**。
2. 页面顶部为整条天蓝色顶栏，标题「用药计划」白色居中，左上角为返回箭头。
3. 顶栏下方有三个 Tab：**服药中 / 未开始 / 已结束**，点击切换查看不同状态的用药计划。
4. 每条计划卡片显示药名、剂量、频次、服用时机与用药周期。点击任意卡片可进入该计划详情。
5. 右下角的蓝色「+」浮动按钮可新增用药计划。

### 二、查看「已结束」用药计划（本次修复重点）

1. 在用药计划列表页，点击 **「已结束」** Tab。
2. 现在可以正常看到所有已结束的用药计划（按结束日期由近到远排序，无结束日期的长期计划排在最后）。
3. 如果没有任何已结束计划，页面会显示「暂无用药计划」，**不再弹出报错提示**。

### 三、用药提醒与打卡（页面B）

1. 进入 **用药提醒页**，页面顶部现为整条天蓝色顶栏，标题「用药提醒」白色居中。
2. 顶部 Banner 显示今日日期、剩余用药次数、下一次提醒、已服用/待服用/今日完成率等信息。
3. 若有即将服用的药，会出现橙色高亮卡片，点击「立即服用」即可完成打卡。
4. 下方「今日时间线」按时间顺序列出当日所有用药安排，可逐条打卡或补打卡；打卡后 5 分钟内可撤销。

## 注意事项

1. 本次为 **H5 网页端** 的样式统一 + 文案修正 + 故障修复，**不涉及小程序端、安卓 / 苹果 App 端**，这些端无需更新或重新安装。
2. 本次更新 **不改变任何业务逻辑**：用药计划的新增/编辑、打卡、撤销、Tab 数据口径、提醒规则等均与此前保持一致。
3. 「已结束」Tab 的报错修复属于后端查询故障修复，已在服务器验证：三个 Tab（服药中 / 未开始 / 已结束）均能正常返回数据，不再出现「该列数据异常，请联系客服」。
4. 访问页面需先登录账号；不同账号看到的用药计划数据各自独立。
