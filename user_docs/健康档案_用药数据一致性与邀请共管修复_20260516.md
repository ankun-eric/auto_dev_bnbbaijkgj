## 三、详细操作指引

### 1. 健康档案首屏 Hero 卡片（第 4 格「在用药品」）

打开链接：
https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile

1. 顶部 Hero 区第 4 格现在叫「**在用药品**」，数字 = 当前选中成员（本人或某位家庭成员）名下正在服用的药品总数。
2. **统计口径**（让你心里有数）：
   - 必须 `status = 在用` 的提醒记录；
   - 并且满足以下任一条件即计入：
     - 勾选了「长期服用」；
     - 没有填结束日期；
     - 结束日期 ≥ 今天。
3. **点击第 4 格即可跳转到用药计划列表**，与列表条数完全一致。
4. 从列表回退到健康档案时，Hero 数字会自动重新拉取最新值。

> 切换不同家庭成员卡片，该数字会跟随当前选中成员变化。

### 2. 用药计划列表（双 Tab：今日用药 / 全部用药计划）

打开链接：
https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/medication-plans

- **「全部用药计划」Tab**：列出所有「在用药品」（口径与 Hero 第 4 格完全一致）。
- **「今日用药」Tab**：在用药品 ∩ 今天需要服用的药。
- 「+ 添加用药」表单字段：
  - 药品名称、剂量、用药频次；
  - 每日服药时间（可多条，按 24h 制，例如 `08:00`、`20:00`）；
  - 起始日期、结束日期；
  - 「长期服用」复选框（勾选后忽略结束日期判断）。
- 提交成功后：列表自动刷新；返回健康档案，Hero 第 4 格自动 +1。
- 行内按钮：
  - **启停**：临时停止/恢复提醒，不删除记录；
  - **编辑**：修改任意字段；
  - **删除**：从列表移除（同时从 Hero 数字中扣除）。

### 3. 性别字段中文显示（B3）

不需要任何操作。打开任意家庭成员卡或 AI 咨询档案抽屉，性别都会显示为：

- 男 / 女 / 未设置

> 后端依然存的是英文枚举，对接口对接无影响；仅前端展示统一。

### 4. 邀请家人共管（B4 + B5）

1. 在健康档案的成员区切到某个**家庭成员**卡片（不是本人）。
2. 看「共管与提醒」区，会出现「**+ 邀请共管**」按钮。
3. 点击该按钮，会跳转到邀请页：
   `https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/family-invite?member_id={成员ID}`
4. 在邀请页填写：手机号、关系（如：父亲/母亲/配偶等），点「发出邀请」即可。
5. 对方在自己的客户端收到通知后即可同意。

> **本人卡片不再出现「+ 邀请共管」按钮**：本人无需邀请自己。

#### 4.1 兜底体验（万一你直接打开了无参数的邀请页）

如果你直接打开 https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/family-invite 而没有携带成员参数，会看到：

- 友好文案：「未找到要邀请的成员，请从健康档案的成员卡片重新进入。」
- 一个明显的「**返回健康档案**」按钮，点击即可回到健康档案首屏。

## 四、常见问题（FAQ）

**Q1：Hero 第 4 格显示的数字和列表条数不一致怎么办？**
本次修复后两边走同一数据源、同一口径，理论上不可能不一致。如果遇到，请：
1. 下拉刷新健康档案首屏；
2. 切换其他家庭成员卡再切回来。

**Q2：我之前用「旧版」添加的药为什么列表里没有？**
之前两套用药表（`MedicationReminder` / `MedicationPlan`）并行。本次修复后统一以 `MedicationReminder` 为唯一真相源——之前写在 `MedicationReminder` 的数据全部可见；写在已废弃的 `MedicationPlan` 的旧数据保留但不再显示（如有真实业务数据需要迁移，请联系管理员）。

**Q3：勾选了「长期服用」+ 填了结束日期，哪个生效？**
「长期服用」优先级最高：勾选了即使 end_date 已过期也计入「在用药品」。

**Q4：把药品状态改为「停用 / 删除」后，今日 Tab 还能看到吗？**
不会。停用 / 删除后立刻从「在用药品」里剔除；Hero 数字同步减 1。

**Q5：本人想要邀请别人共管自己怎么办？**
请让对方在他自己的账号里把你添加为「家庭成员」，再由对方在他端发起共管邀请。

## 五、注意事项

1. **数字与列表口径完全一致**：如果发现不一致，请截图反馈，并告知是哪个成员（本人或家庭成员名字）。
2. **长期服用 vs 结束日期**：两者互斥语义——勾长期就不看结束日期，没勾长期就严格按 end_date 判断。
3. **邀请页 URL 必须带 `member_id`**：从健康档案成员卡片进入会自动带；直接复制粘贴旧链接可能丢参数。
4. **性别字段写入仍是英文**：仅展示中文化，前后端字段不变，对接其他系统无影响。
5. **历史 `/api/medication-reminder/plans` 接口保留只读**：老版本前端不会崩，但所有新增/编辑请使用最新版前端。
