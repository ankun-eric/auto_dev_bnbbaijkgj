## 四、对账单管理页使用指南

### 4.1 页面布局（自上而下）

1. 页面标题：**对账单管理**
2. 副标题说明：系统每月 1 日凌晨自动为「上一自然月」的所有已启用机构生成对账单（机构维度 + 门店维度各一份）。
3. 筛选区（默认 4 项 + 折叠 5 项，见 4.2）
4. 工具条：左侧「共 X 条」统计 + 右侧「刷新」「生成对账单」按钮
5. 列表表格（默认 20 条/页）
6. 分页器

### 4.2 筛选（9 项全量筛选）

默认展示常用筛选项，点击「更多筛选」展开剩余项。所有筛选项同时生效（AND 逻辑）。

| # | 筛选项 | 控件 | 默认展示 |
|---|---|---|---|
| ① | 机构 | 下拉单选（可搜索） | ✅ |
| ② | 账单周期 | 月份选择器（YYYY-MM） | ✅ |
| ③ | 状态 | 全部 / 未结清 / 已结清 / 争议中 | ✅ |
| ④ | 关键词 | 模糊匹配：机构/门店/对账单ID | ✅ |
| ⑤ | 门店 | 下拉单选（与"机构"级联） | 折叠 |
| ⑥ | 维度 | 全部 / 机构维度 / 门店维度 | 折叠 |
| ⑦ | 生成时间 | 日期范围 | 折叠 |
| ⑧ | 结清时间 | 日期范围 | 折叠 |
| ⑨ | 金额区间（元） | 两个数字输入框（最小~最大） | 折叠 |

**操作说明**：
- 填写完条件后点击「查询」刷新列表；点击「重置」清空所有筛选。
- 选择「机构」后，「门店」下拉会自动过滤为该机构下的门店。

### 4.3 列表字段

| # | 字段 | 说明 |
|---|---|---|
| 1 | 对账单ID | 系统唯一编号，点击可复制 |
| 2 | 机构/门店名称 | 机构维度显示机构名；门店维度显示「机构名 - 门店名」 |
| 3 | 维度 | 蓝色「门店维度」/紫色「机构维度」标签 |
| 4 | 账单周期 | 例：`2026-03-01 ~ 2026-03-31` |
| 5 | 应结金额（元） | 带千分位，右对齐 |
| 6 | 状态 | 灰色「未结清」/ 绿色「已结清」/ 红色「争议中」 |
| 7 | 生成时间 | 例：`2026-04-01 00:05` |
| 8 | 结清时间 | 未结清时显示 `-` |
| 9 | 操作 | 【上传凭证】【查看详情】 |

### 4.4 排序

- 默认按「生成时间倒序」排列，最新生成的在最上方。
- 点击表头可切换这 4 个字段的排序：账单周期、应结金额、生成时间、结清时间。

### 4.5 空态

当列表为空时显示："当前暂无对账单，点击右上角「生成对账单」可手动创建。"

## 五、手动生成对账单

当定时任务异常、临时新增机构、需要补录历史账单时使用。

### 操作步骤

1. 在列表页右上角点击主按钮「**生成对账单**」。
2. 弹窗表单：
   - **机构（可选）**：不选则对"所有已启用机构"批量生成。
   - **账期月份（可选）**：不选则默认为"上一自然月"。
3. 点击「确定」，系统提示生成条数并刷新列表。

> 💡 **幂等保护**：如果某机构 + 账期的对账单已存在，不会重复生成。

## 六、上传打款凭证（行内操作）

### 6.1 入口与显隐规则

- 列表每行的「操作」列，在状态为 **未结清 / 争议中** 时显示可点击的「上传凭证」。
- 状态为 **已结清** 时按钮置灰并提示"该对账单已结清"（如需修改走查看详情弹窗的「修改凭证」）。

### 6.2 弹窗字段

| 字段 | 必填 | 说明 |
|---|---|---|
| 对账单ID | - | 只读，自动带入 |
| 机构/门店 | - | 只读 |
| 应结金额 | - | 只读，便于核对 |
| 账单周期 | - | 只读 |
| **凭证文件** | ✅ | 详见 6.3 |
| **打款金额**（元） | ✅ | 支持两位小数，≥ 0；与应结金额不一致时会弹窗二次确认 |
| **打款时间** | ✅ | 不能晚于当前时间 |
| **打款备注** | - | 最多 500 字，例："华夏银行对公转账 / 分两笔支付" |

### 6.3 凭证文件上传控件

支持两种**互斥**模式：

- **多张图片模式**
  - 数量：1~5 张
  - 格式：JPG / JPEG / PNG
  - 单张大小：≤ 5 MB
  - 支持：缩略图预览、点击放大、上/下调整顺序、删除
- **单份 PDF 模式**
  - 数量：1 份
  - 格式：PDF
  - 大小：≤ 10 MB
  - 支持：点击「打开预览」跳转查看、删除

**模式切换**：若已上传图片时再上传 PDF，会弹窗确认"已有图片凭证，继续将清空后改为 PDF 模式"，反之亦然。

### 6.4 提交结果

- 点击弹窗底部「确定」提交，成功后：
  - 该对账单状态自动变为「**已结清**」
  - 「结清时间」自动填入当前时间
  - 列表自动刷新
- 支持**重复上传**：再次点击「上传凭证」会带出已有的文件和打款备注，可修改后保存。

## 七、查看详情（行内操作）

### 7.1 入口

点击列表行「查看详情」即可在弹窗中查看，无需跳页。

### 7.2 弹窗内容（三块）

**① 基本信息**

- 对账单ID、机构/门店名称、维度、账单周期、应结金额、状态、生成时间、结清时间。

**② 结算明细（核销记录）**

- 以表格形式展示该账期内的所有业务单据：业务单号、业务类型、发生时间、金额、备注。
- 底部合计：总条数 + 总金额。
- 支持分页翻阅。

**③ 打款凭证**

- 未上传时显示"暂无凭证"。
- 已上传时展示：
  - 图片（点击放大预览）或 PDF（点击在新标签打开）
  - 打款金额、打款时间、打款备注、上传时间、上传人
- 状态为「未结清 / 争议中」时，底部显示「**修改凭证**」按钮，点击回到上传凭证弹窗（带出已有数据）。
- 状态为「已结清」时，凭证**只读**，不可修改。

### 7.3 关闭规则

- 点击右上角 ✕ 或底部「关闭」按钮关闭。
- **点击弹窗外的遮罩不会关闭**，避免误操作丢失数据。

## 八、典型使用流程

### 场景 A：月初管理员日常对账

```
登录管理后台
  → 左侧菜单进入「对账单管理」
  → 默认看到按生成时间倒序的列表
  → 用"机构 + 状态=未结清"筛选出待处理对账单
  → 每一条：
      1) 点击【查看详情】核对结算明细
      2) 线下完成打款后，点击【上传凭证】
         上传图片/PDF，填写打款金额/时间/备注，提交
      3) 列表自动刷新，该行状态变为「已结清」
```

### 场景 B：月初补单

```
发现某机构本月对账单漏生成
  → 右上角点击【生成对账单】
  → 弹窗选择"机构 + 账期月份" → 确定
  → 系统生成该机构在该账期的对账单
  → 列表刷新
```

## 九、注意事项

1. **打款金额与应结金额不一致时**：系统会弹窗确认，避免误录。请反复核对银行回单。
2. **凭证模式互斥**：同一对账单的凭证要么是「一组图片」要么是「一份 PDF」，不能混合。切换模式会清空已上传的文件。
3. **已结清凭证不可修改**：一旦状态变为「已结清」，对应凭证将变为只读，如需修改请联系技术人员走线下申诉流程。
4. **浏览器兼容**：推荐使用 Chrome / Edge 最新 2 个大版本，以获得最佳体验。
5. **权限控制**：仅具备「对账单管理」权限的管理员可见该菜单，并可执行上传/修改操作。
6. **移动端**：后台管理面板对桌面端优化，建议使用 1366×768 及以上分辨率的屏幕。

## 十、常见问题（FAQ）

**Q1：我上传图片时提示"单张图片不能超过 5MB"怎么办？**  
A：使用图片压缩工具（如系统自带工具、TinyPNG 等）压缩后再上传，或改用 PDF 模式上传整页回单。

**Q2：上传 PDF 后预览打不开？**  
A：请确保浏览器允许在新标签打开 PDF；如仍无法打开，可直接将 PDF 下载到本地查看。

**Q3：筛选"金额区间"没有筛选结果？**  
A：请检查"最小值"是否大于"最大值"；金额单位为「元」，不是"分"。

**Q4：「已结清」状态下还能看到之前上传的凭证吗？**  
A：可以。点击「查看详情」即可看到完整的凭证信息，只是不能再修改。

**Q5：手动生成对账单后，数据量很大，列表没有刷新？**  
A：点击工具条右侧的「**刷新**」按钮即可手动刷新；或切换筛选条件后重新查询。
