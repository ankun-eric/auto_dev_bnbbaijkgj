## 功能简介

本次针对「健康档案 → 我守护的人」页面做了 5 项体验与数据一致性优化：

1. **顶部摘要更紧凑**：右上角紧凑显示 `守护人 X/Y`（X = 当前已守护人数，Y = 角色可守护总位数）。
2. **本人卡上方统计文案优化**：清晰展示「守护人：X 人，还可邀请 Y 位，共 M 位」+「本人不占名额」小字。
3. **「尚未邀请」点【移除】真生效**：纯档案（孤儿）卡片点击移除后，数据库同步软删，列表立即刷新，不再出现"提示已移除但还在列表"的怪相。
4. **「已过期」点【移除】不再报错**：服务端改为幂等返 200 + 友好文案，再也看不到"未找到对应的守护关系"系统报错。
5. **外部「我的家人」Tab 建档后即时可见**：通过孤儿档案兜底查询，在外部建的家人档案，「我守护的人」页面会立刻看到对应卡片并统计入摘要。

## 使用说明

### 操作路径

1. 登录 H5 → 进入「健康档案」 → 点击「我守护的人」入口（或直接打开访问链接表中第 2 条）。
2. 顶部蓝色统计栏即可看到：
   - 右上角：`💙 我守护的人  守护人 2/5`（2 表示当前守护人数，5 表示总位数）
   - 中部：`守护人：2 人，还可邀请 3 位，共 5 位`
   - 底部小字：`本人不占名额`

### 各类卡片对应操作

| 卡片类型 | 显示位置 | 主要操作 |
|---|---|---|
| 我（本人卡片） | 顶部 | 编辑档案 / 邀请记录 / AI 外呼额度 |
| 已绑定家人 | 「💙 已绑定」区 | 查看档案 / 共享额度 / 解除守护 |
| 由我代为管理（孤儿档案） | 「📋 未绑定」区 | 发起邀请 / 移除 |
| 邀请中 | 「📋 未绑定」区 | 查看二维码 / 取消邀请 / 邀请记录 |
| 已过期 / 暂未响应 / 已解绑 | 「📋 未绑定」区 | 再次邀请 / 移除 / 邀请记录 |

### 移除操作的新行为（核心改动）

- 点击【移除】 → 二次确认 → 服务器受理后：
  - **真删了一条数据** → 提示「已移除」，列表立即刷新。
  - **数据其实已不存在**（多终端并发 / 重复点击）→ 提示「该记录已被移除，列表已刷新」，**不再弹系统错误**。
- 「已过期」状态的邀请、「尚未邀请」状态的孤儿档案，现在都可以直接点【移除】完成清理，不再卡死。

### 外部 Tab 建档后看「我守护的人」的体验

1. 进入「健康档案」 → 切换到「我的家人」Tab。
2. 点击「+ 添加家人」，填昵称、关系、性别等基础资料，保存。
3. 立刻切回「我守护的人」 → 在「📋 未绑定」区即可看到该家人卡片，标签为「**由我代为管理**」。
4. 顶部摘要 `X/Y` 中的 X 也会同步 +1。

## 注意事项

- **可守护总位数（Y）** 取决于您当前的会员等级：免费版默认 3 位（以后台 `free_member_quota.max_managed` 配置为准）；付费会员按套餐 `max_managed` 字段下发；超级 VIP 显示「无上限」。
- **本人不占名额**：摘要文案中的 X 仅统计「非本人卡片」，本人不计入 X。
- **数据一致性**：「移除」操作对已绑定的家人会执行「解除守护 + 软删」；对孤儿档案会同步删除关联邀请记录；不会误删硬件设备或在途服药计划（仍受 4 项不可删校验保护）。
- **多端幂等**：两台设备同时点击「移除」同一张卡片，两边都会看到友好提示与列表刷新，不会再有"未找到守护关系"的系统报错。
- **脏数据清理**：本次上线已在生产库执行一次性脏数据清理（悬空 invitation 引用、悬空 FamilyManagement、已注销用户残留、(manager_user_id, managed_member_id) 重复条目）。

## 常见问题

**Q1：为什么有些档案显示「由我代为管理」？**  
A：该家人是您在「我的家人」Tab 中自行添加的、尚未通过邀请绑定真实账号的孤儿档案。您可以为其发起邀请，也可以直接移除。

**Q2：点【移除】提示「该记录已被移除」是什么意思？**  
A：表示这条记录此前已被删除（可能是您在另一台设备已操作过，或后台批量清理过）。系统不会报错，而是友好提示并自动刷新列表。

**Q3：摘要里的 X 和卡片数量对不上怎么办？**  
A：X 统计「所有非本人卡片」（已绑定 + 全部未绑定）。如果出现偏差，请下拉刷新页面；如果仍不一致，请联系产品反馈。
