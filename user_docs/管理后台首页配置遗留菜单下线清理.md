## 功能简介

本次更新是一次**纯清理性**改造，目标是把管理后台「首页配置」分组下三个早已失去意义的旧菜单页**物理下线**，并顺手修复一个隐藏近一个月的历史 Bug。

具体涉及四件事：

1. **下线 3 个旧菜单页**
   - 删除「页面风格」配置页（旧菜单式/AI 对话页切换 UI，已无切换价值）
   - 删除「首页菜单管理」（菜单式首页已下线，没有运营需求）
   - 「首页基础设置」改名为「字体配置」并瘦身——只保留字体相关的 5 个字段，其它旧字段（搜索框开关、菜单列数等）的编辑入口移除

2. **侧边栏结构调整（管理后台）**
   - 「AI 对话模式首页配置」→ 改名为「AI首页配置」
   - 「功能按钮管理」从「AI咨询配置」分组迁入「首页配置」分组
   - 「首页配置」分组下最终保留 5 项：AI首页配置 / 功能按钮管理 / 字体配置 / 首页Banner管理 / 公告管理 / 底部导航配置

3. **修复 H5 端 AI 首页 banner 不显示的历史 Bug**
   - 原因：H5 端 ai-home 页面调用了错误的 banner 接口路径 `/api/h5/home-banners`（实际后端路径是 `/api/home-banners`），404 错误被 `catch(() => {})` 静默吞掉，导致 banner 永远为空数组
   - 修复后：路径改正 + catch 升级为 `console.warn` 兜底，banner 区域恢复正常显示

4. **数据库一次性数据治理**
   - 旧 banner 中 `link_url = '/home'` 自动改为 `/ai-home`
   - 指向旧菜单子页 `/home/menu/*` 或 `/menu-mode/*` 的 banner 自动下架（`is_visible=false`）
   - 治理结果完整记录到审计表 `banner_migration_log_20260519`，便于事后回查

> ✅ **重要保证**：小程序和 Flutter App 端**本次完全不受影响**，老首页（菜单式）依然可以正常打开和使用。

## 使用说明

### 1. 管理员侧 — 体验全新的「首页配置」分组结构

1. 在浏览器打开管理后台：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 用管理员账号登录
3. 在左侧侧边栏找到 **「首页配置」** 分组（图标为小房子 🏠）
4. 展开后应当看到以下 5 项（按从上到下顺序）：

   | 菜单项 | 用途 |
   |---|---|
   | **AI首页配置**（原「AI 对话模式首页配置」改名而来） | 维护 AI 对话首页的开场白、推荐问题等 |
   | **功能按钮管理**（从「AI咨询配置」迁入） | 维护拍照识药 / 报告解读 / 健康自查等功能按钮 |
   | **字体配置**（原「首页基础设置」改名 + 瘦身） | 仅保留字体切换 5 个字段的编辑（开关、默认档位、3 档字号） |
   | **首页Banner管理** | 维护首页轮播图 |
   | **公告管理** | 维护首页公告 |
   | **底部导航配置** | 维护底部 Tab 栏 |

5. **确认旧入口已不可达**：
   - 直接访问 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-settings/page-style](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-settings/page-style) 应当 404
   - 直接访问 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-menus](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-menus) 应当 404

### 2. 管理员侧 — 体验「字体配置」瘦身后的形态

1. 在「首页配置」分组下点击 **「字体配置」**
2. 页面标题应为 **「字体配置」**
3. 页面上**只剩一个卡片**「字体切换配置」，包含以下 5 个字段：
   - **字体切换功能**（开关）
   - **新用户默认档位**（标准 / 大 / 超大）
   - **标准档基础字号**
   - **大档基础字号**
   - **超大档基础字号**
4. 修改任一字段，点击「保存」，弹出「字体配置保存成功」即视为成功
5. 旧的「搜索框配置」和「宫格配置」卡片不应再出现

### 3. 管理员侧 — 体验「AI咨询配置」分组

1. 展开侧边栏「AI咨询配置」分组（图标为电话 📞）
2. **「功能按钮管理」** 已不在此分组内（它已迁到「首页配置」分组）
3. 其它 11 项保持不变：部位症状字典、健康自查问卷模板、通用问卷模板管理、AI 外呼配置、数字人形象管理、语音服务配置、TTS语音配置、分享海报配置、视频客服、对话超时配置

### 4. 用户侧 — 体验 H5 AI 首页 banner 恢复显示

1. 在手机或浏览器打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)

3. 滚动到首页正中或上半部，应当能看到 banner 轮播图区域（如管理后台已配置了可见的 banner）
4. 修复前 banner 区域永远为空，**修复后**只要数据库中存在 `is_visible=true` 的 banner，前端就会展示

### 5. 小程序 / Flutter App 用户 — 体验未受影响

1. 打开你常用的微信小程序或 App
2. 进入主页（菜单式老首页）
3. **菜单列表、Banner 轮播图、字体设置都应当正常**，没有任何功能丢失
4. 如果你正在使用的是旧菜单页，体验和上线前完全一致

## 注意事项

1. **后端「读接口」全部保留兼容**：
   - GET `/api/home-config`、GET `/api/home-menus`、GET `/api/home-banners` 全部保留，小程序/Flutter App 老首页可以继续读取
   - GET `/api/app-settings/page-style` 也保留，但**永久返回常量** `{"key":"page_style","value":"ai_chat"}`，待小程序/Flutter 完成 AI 首页改造后随 v1.2 一并清理

2. **后端「写接口」物理下线**：
   - PUT `/api/admin/app-settings/page-style` 已删除（admin 页面已下线，无人调用）
   - 所有 `/api/admin/home-menus` 写接口（POST/PUT/DELETE/sort）已删除
   - PUT `/api/admin/home-config` 保留，但**只接收 font_\* 字段**，传其它字段会被静默忽略

3. **数据库实际变化**（已通过 SSH 验证）：
   - `app_settings` 表：`key='page_style'` KV 已删除（数量 = 0）
   - `system_configs` 表：所有非 `home_font_*` 的 `home_*` KV 已删除（如 `home_search_visible`、`home_search_placeholder`、`home_grid_columns`），仅保留 `home_font_*` 共 5 条
   - `home_banners` 表：原 `link_url='/home'` 的 banner 已自动改为 `/ai-home`；指向旧菜单子页的 banner 已自动 `is_visible=false`
   - `banner_migration_log_20260519` 临时审计表已创建并写入治理快照

4. **历史 Bug 已闭环**：
   - 文档 `docs/BUG_FIX_AI对话首页轮播图菜单不显示_20260427_175500.md` 中描述的 banner 不显示问题，本次一并闭环修复

5. **回滚方案**：
   - 代码层：Git revert 三个 PR（admin-web / backend / h5-web）即可一键回滚
   - 数据库层：可通过 `banner_migration_log_20260519` 审计表手动反向恢复 banner 数据；`home_*` KV 的恢复建议依赖事前 mysqldump 备份

6. **下一步规划**：
   - 待《小程序 & Flutter App AI 首页改造 PRD v1.0》完成后，将启动本 PRD 的 v1.2 收尾：彻底清理 `GET /api/home-menus`、`GET /api/home-config`、`GET /api/app-settings/page-style` 等读接口
