## 功能简介

本次更新完成了**日期时区全面简化优化**，核心改动如下：

### 1. 时间存储统一为北京时间
- 服务器和 Docker 容器时区统一为 `Asia/Shanghai`（北京时间）
- 所有新增记录的时间字段直接使用北京时间存储
- 不再需要在各端做 UTC → 北京时间转换

### 2. 时间显示格式统一
- API 返回的时间格式统一为 `"YYYY-MM-DD HH:mm:ss"`（如 `"2026-06-06 21:19:30"`）
- 前端不再做任何时区转换，直接显示后端返回的时间

### 3. 历史数据迁移
- 数据库中所有旧记录的时间字段已自动加 8 小时（从 UTC 转为北京时间）
- 用户无需任何额外操作，历史数据时间会自动显示正确

## 使用说明

### 时间显示验证

1. **API 时间验证**：访问 [server-time 接口](https://6b099ed3-7175-4a78-91f4-44570c84ed27.noob-ai.test.bangbangvip.com/api/system/server-time)，返回格式为：
   ```json
   {"now_iso":"2026-06-06 21:19:30","now_unix_ms":1780751970423,"timezone":"Asia/Shanghai"}
   ```
   - `now_iso` 为北京时间字符串，格式 `YYYY-MM-DD HH:mm:ss`
   - `timezone` 为 `Asia/Shanghai`

2. **H5 用户端**：打开主页面，所有时间（订单创建时间、健康记录时间、签到时间等）直接显示北京时间

3. **管理后台**：打开管理页面，所有列表和详情中的时间字段直接显示北京时间

### 历史数据确认

数据库中的历史数据已自动完成 UTC → 北京时间转换（加 8 小时），您可以在以下位置验证：
- 订单列表中查看历史订单的创建时间
- 用户管理中查看用户的注册时间
- 健康档案中查看历史记录的时间

## 注意事项

1. **服务器时区**：服务器时区已设置为 `Asia/Shanghai`（北京时间），Docker 容器也已同步设置
2. **历史数据**：数据库中的历史数据时间字段已加 8 小时（UTC → 北京时间），此操作不可逆，如需回滚请使用数据库备份
3. **第三方回调**：支付宝/微信支付等第三方回调的时间字段由对方系统生成，不受本次优化影响
4. **客户端时间**：前端的 `new Date()` 和 `DateTime.now()` 默认使用客户端本地时间，与服务器时区独立
5. **API 兼容**：所有 API 接口的时间字段格式统一为 `"YYYY-MM-DD HH:mm:ss"`，不再包含时区标识
6. **默认账号**：管理员账号 `admin / admin123`，可用于登录管理后台验证时间显示
