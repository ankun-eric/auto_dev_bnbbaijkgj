## 三、功能介绍

### 3.1 预约模式（appointment_mode）

| 模式 | 说明 | 需要配置 |
|---|---|---|
| 无需预约 | 用户下单直接核销使用 | 仅需核销次数、支付超时、有效期 |
| 预约日期 | 用户在指定日期范围内选择一天 | 提前可预约天数、单日最大预约人数 |
| 预约时段 | 用户在固定时段内选择 | 至少 1 条时段（起止 + 容量） |
| 自定义表单 | 按自定义表单字段采集预约信息 | 必须绑定一张预约表单 |

### 3.2 下单预约方式（purchase_appointment_mode）

| 选项 | 说明 |
|---|---|
| 下单即预约 | 用户在下单环节就完成预约选择 |
| 先下单后预约 | 用户先完成购买，再到订单中心发起预约 |

### 3.3 预约表单库

- 可统一管理所有自定义预约表单
- 一张表单可被**多个商品**绑定（不再是 1:1 潜规则）
- 支持启用 / 停用 / 删除（被商品引用的表单禁止删除，会给出提示）

## 四、操作步骤

### 4.1 新建预约表单（可复用）

1. 打开 [预约表单库](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/appointment-forms)
2. 点击右上角【新建表单】
3. 填写表单名称、状态（启用 / 停用），保存
4. 在表格行点击【字段管理】，逐条添加字段：
   - 字段标识（英文，用于存储）
   - 字段显示名（中文）
   - 类型：单行文本 / 多行文本 / 单选 / 多选 / 数字 / 日期 / 时间 等
   - 是否必填、排序
5. 完成后该表单可被任意商品引用

### 4.2 新建"预约日期"商品

1. 打开 [商品管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/products)
2. 点击【新建商品】
3. 在"基础信息"填写名称 / 分类 / 价格等
4. 切到【预约与核销】Tab：
   - **预约模式**选择"预约日期"
   - **提前可预约天数** 必填（如 7）
   - **单日最大预约人数** 必填（如 10）
   - **下单预约方式** 必选（下单即预约 / 先下单后预约）
5. 点击保存

### 4.3 新建"预约时段"商品

1. 预约模式选择"预约时段"
2. 在时段编辑器中添加若干条（至少 1 条）：
   - 起始时间（如 09:00）
   - 结束时间（如 10:00）
   - 单时段容量（如 5）
3. 保存

### 4.4 新建"自定义表单"商品

1. 预约模式选择"自定义表单"
2. 在下拉框中**选择一张已有的预约表单**（可点击"新建表单"跳转到预约表单库再创建）
3. 保存

### 4.5 编辑已有商品的预约配置

1. 在商品列表点击某商品的【编辑】
2. 切到【预约与核销】Tab
3. 改变预约模式后对应联动区会自动展开 / 收起
4. 保存即可

### 4.6 商品列表"表单"按钮的新语义

- **未绑定表单** → 会弹提示"该商品尚未绑定预约表单，请先在编辑中选择或到预约表单库新建"（不再自动偷偷建一张）
- **已绑定表单** → 进入表单字段编辑

## 五、错误提示说明（告别"操作失败"）

过去所有失败都只提示"操作失败"。本次升级后：

| 场景 | 提示内容 |
|---|---|
| 预约模式=预约日期 且未填提前天数 | "预约日期模式下，提前可预约天数必填" |
| 预约模式=预约时段 且时段为空 | "预约时段模式下，至少配置 1 个时段" |

| 枚举值不合法 | "枚举值不合法：xxx" |
| 数据库完整性冲突 | "数据约束冲突，请检查必填字段与唯一性" |

前端在 **提交前会先做一次本地校验**，不合格直接给出红色提示，不再发请求，用户输入体验更顺滑。

## 六、注意事项

1. **老商品兼容**：之前由于枚举不合法，部分历史脏数据会被统一回落为"无需预约"（后端数据清洗）。
2. **表单复用**：同一张表单被多个商品共用时，**修改字段会影响所有引用该表单的商品**，请谨慎。
3. **表单删除**：若想删除某张表单，请先解除所有商品的绑定（编辑商品把预约模式改为其他），再删除；否则会提示"该表单正在被使用，无法删除"。
4. **下单预约方式**仅在"预约日期 / 预约时段 / 自定义表单"3 种模式下有效，"无需预约"会忽略此项。
5. **时段配置**建议：起止时间需满足 `start < end`，否则保存会报错。
6. **刷新**：前端路由为 Next.js 客户端跳转，第一次操作建议硬刷新一次，让新菜单项生效。
