## 一、功能简介

本次升级将**实物商品**与**积分商城**从字段、数据、产品定位三个层面**彻底切开**，让两套体系各司其职：

- **实物商品**：在【商品管理】里维护，走普通下单流程，可参与"付费会员折扣""积分抵扣订单金额"等玩法。**不能、也无需**进入积分商城。
- **积分商城商品**：在【积分商城管理】里独立维护，走纯积分兑换流程。可独立录入**任意类型**的兑换品（虚拟权益/卡券/电子兑换码/实物礼品均可），与实物商品库**零数据关联**。

这意味着运营同学在维护实物商品时不再被"是否进入积分商城""所需积分"等字段干扰，逻辑也更清爽：

| 体系 | 维护入口 | 业务流程 | 是否参与积分商城 |
|------|----------|----------|------------------|
| 实物商品 | 后台 · 商品管理 | 普通下单 / 付费会员折扣 / 积分抵扣订单金额（≤20%） | ❌ 不参与 |
| 积分商城商品 | 后台 · 积分商城管理 | 纯积分兑换 | ✅ 唯一来源 |

## 二、本次具体变化（运营 / 管理员视角）

### 2.1 商品编辑页【会员与积分】Tab

| 字段 | 本次处置 |
|------|---------|
| 是否支持付费会员折扣 | ✅ 保留 |
| ~~是否进入积分商城（纯积分兑换）~~ | ❌ **已删除** |
| ~~积分商城兑换所需积分~~ | ❌ **已删除** |
| 是否支持积分抵扣订单金额 | ✅ 保留 |

> 删除两项后，【会员与积分】Tab 内容由 4 项缩减为 2 项，排版自然收紧。

### 2.2 后端接口

- 实物商品创建 / 编辑 / 详情 / 列表接口**不再返回**也**不再接收** `points_exchangeable` 和 `points_price` 两个字段。
- 老版本 App / 小程序若仍传这两个字段，后端**静默忽略并返回 200**，不会报错（兼容老客户端一个发版周期）。
- 列表筛选参数 `?points_exchangeable=true` 同样会被忽略，返回全部实物商品。

### 2.3 数据库

- 实物商品表 `products` 中的 `points_exchangeable` / `points_price` 两列**已置空（设为 0/false）**。
- 按 PRD 回滚预案，**物理列暂保留**，作为本发版周期的回滚缓冲；下个迭代再评估物理 DROP COLUMN。
- 积分商城侧的兑换记录、订单、库存流水、用户积分流水**全部保留不动**，财务与运营可继续追溯。

### 2.4 用户端（H5 / 小程序 / Flutter App）

- H5 商品列表页：去除"积分可兑"勾选筛选与列表卡片右上"积分兑"角标
- H5 商品详情页：去除"X 积分可兑"标签
- 小程序商品列表 / 详情：去除积分价格行 / "X 积分可兑"标签
- Flutter App 商品详情：去除"积分可兑"小标签（保留"积分可抵"，因属于另一玩法）

## 三、操作步骤

### 3.1 运营 · 编辑实物商品（变化最大的入口）

1. 浏览器登录 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 左侧菜单进入【商品管理 → 商品】
3. 在商品列表点击任意一行的"编辑"按钮
4. 进入编辑弹窗，切换到【会员与积分】Tab，可以看到现在只剩两项：
   - **是否支持付费会员折扣**（保留）
   - **是否支持积分抵扣订单金额**（保留）
5. 按需开关 → 点击"保存"即可

### 3.2 运营 · 维护积分商城商品（流程与本次升级前一致，无变化）

1. 左侧菜单进入【积分商城管理】
2. 进行积分商城商品的新增 / 编辑 / 上架 / 下架等操作
3. 积分商城商品来源**唯一**：由本入口录入；不存在"实物商品同步进来"的可能

### 3.3 普通用户 · 在 H5 端浏览实物商品

1. 打开 [H5 商品列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/products)
2. 顶部筛选栏：履约方式仍可筛选（到店 / 快递 / 虚拟 / 上门），但**"积分可兑"筛选已下线**
3. 商品卡片：不再显示"积分兑"角标
4. 点击商品进入详情：不再出现"X 积分可兑"标签
5. 普通下单流程不变；下单时可勾选"使用积分抵扣订单金额（≤20%）"或"使用付费会员折扣"，二者二选一不可叠加

### 3.4 普通用户 · 体验积分商城（流程不变）

1. 打开 [H5 积分商城](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/points/mall)
2. 浏览商品，使用积分兑换；历史兑换记录、积分流水均可正常查询

## 四、注意事项

1. **不需要做任何数据迁移**：旧的"实物商品 + 进入积分商城 = 是"配置已视为自然失效，**不会向积分商城回写**；同时积分商城商品的兑换历史完全保留。
2. **不存在"实物商品同时在积分商城出现"的可能**：本次从字段层就杜绝了这种配置，运营即便想这么做也没有入口。
3. **积分玩法不冲突**：
   - "积分商城兑换"（纯积分换购）→ 仅积分商城商品
   - "积分抵扣订单金额"（最多 20%）→ 仅普通商城实物商品下单
   - "付费会员折扣"→ 仅普通商城实物商品下单（与积分抵扣二选一）
4. **老版本客户端的兼容**：旧版 App / 小程序如未升级，仍可正常使用（请求里多带的旧字段会被后端忽略）；建议在一个发版周期内全量升级到最新版客户端。
5. **回滚预案**：如线上发生异常需回滚，仅需后端回退接口契约、前端回退 UI；数据库列保留物理列，业务无数据损失。

## 五、常见问题

**Q1：以前在实物商品里设置过"进入积分商城 = 是"的商品，现在还能被用户用积分兑换吗？**
A：不能。该配置已视为自然失效，相应字段已置空。但用户在积分商城中已有的兑换记录、订单、积分流水都完整保留可查。

**Q2：H5 / 小程序 / App 不升级会有问题吗？**
A：不会立即出问题。后端兼容老客户端的请求和参数；只是老版本里的"积分可兑"角标 / 筛选还会显示（但不会再筛出结果）。建议运营层面尽快推动客户端升级。

**Q3：积分商城是否会做同样的瘦身？**
A：本次**只调整实物商品端**，积分商城后台和前端完全未动，原有功能保持不变。

**Q4：商品详情接口里 `points_exchangeable` / `points_price` 字段真的没有了？**
A：是的，后端响应里已彻底移除。Flutter App 中模型字段为兼容性保留，但 `?? false` / `?? 0` 自动兜底，不会因字段缺失崩溃。
