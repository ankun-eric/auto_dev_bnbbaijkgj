## 功能简介

本次改版围绕 **「让健康关键信息一屏可见、漏吃药家人能感知」** 两个核心目标，对 H5 端「我的健康档案」进行全面重构：

1. **信息架构升级**：从「折叠分区式」升级为「成员条 + Hero + 粘性 Tab + 5 个内容区」，关键信息首屏呈现。
2. **视觉对齐基线**：全面对齐 PRD-441/442 的 11 级天蓝色阶 + 病历卡左竖线 + 天蓝阴影 + 中老年友好字号（关键数字 22px）。
3. **6 宫格今日数据**：血压 / 血糖 / 心率 / 睡眠 / 血氧 / 用药打卡 一屏可见，异常项左竖线由蓝变黄 + 右上角「偏高」黄徽章。
4. **手工 + 设备双通道录入**：每个指标详情页都有「✎ 手工填写」（主按钮）与「⌚ 绑定设备」（次按钮）两个入口。
5. **漏打卡代为提醒**（最小版本）：本人漏打卡超过 15 分钟，全部 active 共管者自动收到提醒，文案固定、每天每事件每人最多 1 条。

## 使用说明

### 步骤 1：进入新版健康档案

1. 在浏览器中访问 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile-v2](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile-v2)
2. 未登录时会被引导到登录页，登录后将自动返回新版主页
3. 顶部为天蓝色顶栏「我的健康档案」，下方依次为家庭成员条、基本信息 Hero 卡、粘性 Tab、5 个 Tab 内容

### 步骤 2：在家庭成员条切换查看人

- 进入页面后默认激活「本人」（圆形头像 + 蓝底白字 + 浅蓝描边）
- 点击其它家人头像，整页数据立刻刷新到对应成员
- 头像右上角小蓝点表示该成员已经通过「邀请本人关联」绑定真实账号
- 末尾「+ 添加」按钮跳转到旧版健康档案编辑页继续添加家人

### 步骤 3：阅读「今日数据」6 宫格

- 滚动到「今日数据」区域，看到 6 张迷你病历卡（白底 + 3px 天蓝左竖线 + 天蓝阴影）
- 关键数字采用 22px 大字体，单位 13px，标签 13px，符合中老年阅读习惯
- **异常态视觉**：当血压收缩压 > 140 或舒张压 > 90、空腹血糖 > 7.0、心率 < 50 或 > 100、睡眠 < 6h、血氧 < 95% 时，左竖线由 **蓝色变黄色**，右上角出现 **「偏高」黄色徽章**
- 第 6 格「用药打卡」展示 X/Y 已打卡比例 + 进度条；任意一条用药逾期未打卡，左竖线变黄 + 显示「待打卡」黄徽章
- **点击任意一格即可进入该指标的详情页**

### 步骤 4：在指标详情页录入数据

1. 点击任意指标卡（如「血压」）进入详情页
2. 顶部「当前值」Hero 卡展示最近一次记录（数值、单位、时段、时间）
3. 中部双按钮：
   - **「✎ 手工填写」（主按钮，全宽 60%，蓝色实底白字）** — 弹出录入弹窗，填写数值（如收缩压 / 舒张压）+ 时段（晨/午/晚），点击保存
   - **「⌚ 绑定设备」（次按钮，全宽 40%，白底蓝字蓝边）** — 引导用户到下方「设备绑定」列表
4. 下方「7 天趋势」展示近 7 日主指标的折线趋势
5. 「历史记录」分页列出最近录入：每条带来源胶囊（✎ 手工=蓝胶囊，⌚ 手环=绿胶囊）
6. 「设备绑定」列表：
   - ⌚ **华为 Watch / GT 系列**：点击「立即绑定」即刻通过占位通道绑定（华为开放平台 Key 到账后将自动切真接 OAuth）
   - ⌚ **小米手环 / Watch**：同上
   - 📶 血糖仪 / 💪 血压计 / ⚖️ 体重秤：本期占位，文案「敬请期待」

### 步骤 5：查看用药计划 & 漏打卡提醒

1. 在主页粘性 Tab 中点击「用药计划」或滚动到该区域
2. 每个用药计划单独一张病历卡，展示药品名 + 剂量 + 时间胶囊（绿色 ✓ 已打卡 / 红色 ⏰ 待打卡）+ 本周完成率
3. 卡片底部固定文案：**「ⓘ 漏打卡超 15 分钟将通知共管者」**
4. **代为提醒规则**：
   - 总开关：系统级**固定开启**，用户无法关闭（最小版本）
   - 通知对象：所有 status=active 的家庭共管者
   - 宽限期：固定 **15 分钟**
   - 通知文案（固定）：「家人漏打卡提醒：您正在共同管理的 {昵称} 今日 {HH:MM} 的 {药品名} 已超过 15 分钟未打卡，请联系提醒」
   - 去重：同一漏打卡事件，每个共管者每天最多 1 条

### 步骤 6：查看共管与提醒设置

1. 主页粘性 Tab 切换到「共管」（Tab 4）
2. 卡片内三行：
   - 👥 共同管理者 — 点击查看本档案的所有共管者
   - 📨 邀请本人关联 — 已关联时右侧显示绿色「✓ 已关联」徽章
   - 🔔 漏打卡代为提醒 — 状态固定「开启（15min 宽限）」
3. 底部浅蓝信息块说明完整规则：「系统级固定开启。当本人漏打卡超过 15 分钟时，全部共管者将收到通知，提醒联系本人。」

## 注意事项

1. **数据隔离**：新版与旧版数据完全隔离。新版的「今日数据」「指标详情」走 `/api/health-profile-v3/*` 接口，写入新表 `health_metric_record`；旧版的健康档案编辑、家人管理、共管邀请保持原样不变。
2. **设备绑定占位通道**：本期华为 / 小米手环走的是「占位通道」（直接落库 + 状态标记 active），主要用于打通前后端链路；开放平台 Key 到账后会无缝切换到真 OAuth，已绑用户不需重新操作。
3. **漏打卡扫描周期**：定时任务每 10 分钟扫描一次 + 15 分钟宽限 → 最差通知延迟约 25 分钟，符合最小版本预期。
4. **粘性 Tab 滚动联动**：点击 Tab 会平滑滚动到对应锚点；反向滚动经过锚点 1/3 屏高度时会自动高亮对应 Tab。中老年用户首次进入可能不熟悉，后续版本会增加引导蒙层。
5. **异常阈值默认值**（可后续接入个性化）：
   - 血压：收缩压 > 140 或舒张压 > 90 → 偏高
   - 血糖：空腹 > 7.0 或餐后 > 11.1 → 偏高
   - 心率：< 50 或 > 100 → 异常
   - 睡眠：< 6h → 不足
   - 血氧：< 95% → 偏低
6. **健康标签 / 健康事件流**：本期 Tab 2「健康标签」与 Tab 5「事件流」为基础占位版本，跳回旧版健康档案进行编辑；事件流自动聚合最近的指标录入事件，无需手动操作。
7. **入口位置**：本次只新增 `/health-profile-v2` 路径，旧的 `/health-profile` 仍可访问，方便对比 / 灰度。后续根据反馈可将默认入口切换到新版。
