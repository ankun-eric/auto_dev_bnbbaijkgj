## 二、板块一：门店类别字段

### 2.1 功能介绍

- 新建 / 编辑门店时，新增"**所属类别**"必选下拉；
- 门店列表新增"类别"展示列，支持按类别筛选；
- 类别字典由平台维护（自营门店 / 医疗机构 / 居家服务机构 / 其他），门店新建下拉项实时同步。

### 2.2 操作步骤（管理员）

#### 新建门店

1. 登录 Admin → 菜单【商家管理 → 门店管理】
2. 点击 **新建门店** 按钮
3. 填写：
   - 门店名称、门店编码
   - **所属类别**（必选，默认"自营门店"，可切换为其它类别）
   - 联系人、联系电话、门店地址（可选）
4. 点击 **保存**

> 若忘填"所属类别"，前端会提示"请选择所属类别"，无法提交。

#### 编辑门店

1. 门店列表中点击某行"编辑"
2. 可修改"所属类别"及其它字段
3. **存量未分类门店**在列表以灰色"未分类"标签显示，编辑时必须补选类别再保存。

#### 按类别筛选门店

- 门店列表顶部新增"类别"下拉筛选器，选择后仅显示对应类别的门店；选"全部类别"恢复不过滤。

### 2.3 注意事项

- 类别为平台字典，如需增加新类别，请在【机构类别管理】菜单维护后即可在门店下拉中看到。
- 切换门店类别只影响统计 / 识别维度，不会影响门店上的员工、核销记录等业务数据。

## 三、板块二：商家角色权限体系

### 3.1 四个平台内置角色

| 角色 | 中文 | 创建入口 | 典型权限 |
| --- | --- | --- | --- |
| boss | 老板 | 管理员在 Admin 的 "新建商家账号" 页面创建 | 覆盖该商家所有门店 & 全部 8 个模块 |
| manager | 店长 | 管理员在 Admin 的 "新建员工" 里为某店长选择角色 | 覆盖其授权门店 & 全部 8 个模块 |
| finance | 财务 | 同上 | dashboard / records / messages / profile / finance |
| clerk | 店员 | 同上 | dashboard / verify / records / messages / profile |

### 3.2 八个模块

`dashboard 工作台`、`verify 核销`、`records 记录`、`messages 消息`、`profile 我的`、`finance 财务对账（新）`、`staff 员工管理（新）`、`settings 门店设置（新）`。

### 3.3 管理员操作步骤

#### 3.3.1 新建商家账号（老板）

1. 菜单【商家管理 → 商家账号管理】
2. 点击 **新建主账号（老板）**
3. 填写手机号、初始密码、昵称、授权门店等
4. 保存后，该账号自动拥有 `role_code = boss` 并覆盖 8 个模块

> 显示文案已从"商家主账号"统一改为"**老板**"。

#### 3.3.2 新建员工账号

1. 在 "商家账号管理" 页面点击 **新建员工**
2. 依次填写：
   - 手机号、初始密码、昵称
   - **员工角色**（必选，可选 `店长 / 财务 / 店员`；下拉中**不含"老板"**）
   - 授权门店（多选）
3. 选中角色后，系统会按 §3.2 默认矩阵**自动勾选对应模块**，您仍可以在默认基础上手动微调。
4. 点击保存

> 若在后端直接传 `role_code=boss`，将返回 400 错误："新建员工接口不允许创建老板角色"。

#### 3.3.3 编辑员工 / 切换角色

1. 员工列表中点击某行 "编辑"
2. 修改 **员工角色** 时，系统会弹出二次确认："切换角色将把所有授权门店的模块权限重置为该角色默认值，是否继续？"
3. 确认后，该员工所有门店的模块位将被重写为新角色默认集。

#### 3.3.4 按角色筛选员工

- 员工列表顶部新增 "角色" 下拉过滤器；列表行新增 "角色" 列。

### 3.4 商家端 H5 菜单变化（老板 / 店长 / 财务 / 店员）

- 左侧导航新增 3 个入口，按权限自动显示 / 隐藏：
  - **财务对账**（finance）—— 聚合入口：结算、发票、下载中心、报表
  - **员工管理**（staff）—— 原"员工与权限"重命名
  - **门店设置**（settings）—— 查看门店资料（本期暂为只读，后续开放编辑）

### 3.5 商家端"员工管理" —— 店长可用能力

店长登录 H5，进入【员工管理】后可以：

- 查看本店所有员工及其角色；
- 对本店**下属**（`finance / clerk`）：
  - **编辑权限**：调整模块权限复选框后保存
  - **停用/启用**：切换账号状态
- **无法**新增员工（新增统一由老板在 Admin 完成），UI 不显示"新增"按钮；接口直接返回 403 `E_MANAGER_CANNOT_CREATE`。
- **无法**修改其他店长或老板；**无法**跨店操作。

### 3.6 财务 / 店员登录后的差异

- **财务**：看不到 "核销" 菜单入口，但能进 "财务对账"；
- **店员**：能进 "核销"，但看不到 "财务对账" / "员工管理" / "门店设置"。

## 四、数据迁移说明

- 历史 `owner` 账号自动迁移为 `role_code = boss`
- 历史 `staff` 账号且无 `role_code`，默认迁移为 `role_code = clerk`（店员）
- 已配置好的模块权限（`merchant_store_permissions`）**不会自动重写**，保持原有勾选。管理员如需按新角色重置，只需进入编辑面板重新选择角色并确认即可。
- 历史门店 `category_id` 保持 NULL，列表显示 "未分类"，管理员下次编辑时必须补选类别再保存。

## 五、常见问题

**Q1：我是老板，登录后找不到"员工管理"入口？**
A：清空浏览器缓存后重新登录即可。或检查自己账号的"授权模块"是否包含 `staff`（老板默认都有）。

**Q2：切换员工角色之后之前的授权自定义微调会丢失吗？**
A：会。角色切换时系统会统一按新角色默认模板重写该员工所有门店的权限。切换前请在 Admin 页面的确认弹窗中确认。

**Q3：店长能否给员工调权限？**
A：可以，但**只能对本店的 finance/clerk 下属**调整，且不能新增员工。新员工统一由老板在 Admin 完成。

**Q4：门店类别字典可以自定义吗？**
A：由平台管理员在"机构类别管理"菜单维护，维护后立刻在"新建门店"下拉中生效。
