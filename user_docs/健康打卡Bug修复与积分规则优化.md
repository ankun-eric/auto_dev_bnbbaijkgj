## 功能简介

本次更新包含 **三项改动**，涉及 Bug 修复和功能优化：

1. **Bug 修复 — 健康打卡全部失败**：修复了"打卡送积分"功能上线后导致所有打卡操作（健康习惯、用药提醒、健康计划任务、今日待办）在所有端均提示"打卡失败"的严重 Bug。现在打卡功能已完全恢复正常，且积分发放异常不会影响打卡本身。

2. **积分规则配置页面合并**：后台"积分规则配置"页面原有两个重复的打卡积分配置（通用规则 Tab 和打卡规则 Tab），现已合并为单一页面。删除了"打卡规则"Tab，在"任务奖励"卡片中新增了"健康打卡每日上限"字段，运营人员配置更清晰直观。

3. **完善健康档案送积分**：原"完善个人资料"改名为"完善健康档案"，并实现了积分发放逻辑。用户首次将健康档案中的性别、生日、身高、体重 4 个字段全部填写后，将自动获得一次性积分奖励。

## 使用说明

### 一、健康打卡功能（Bug 已修复）

打卡功能已恢复正常，所有类型的打卡操作均可正常使用：

1. **健康习惯打卡**：
   - 进入"健康打卡"页面，点击任意健康习惯的"打卡"按钮
   - 打卡成功后，系统会自动发放积分（如已在后台配置）
   - 页面会显示获得的积分数和今日积分进度

2. **用药提醒打卡**：
   - 进入"用药提醒"页面，点击"确认服药"完成打卡
   - 打卡成功后同样会获得积分奖励

3. **健康计划任务打卡**：
   - 进入健康计划详情页，点击任务的"打卡"按钮
   - 打卡成功后获得积分

4. **今日待办打卡**：
   - 在"今日待办"页面，点击任意待办项完成打卡

> 💡 **积分规则**：每次打卡获得的积分由后台"健康打卡"字段配置决定。当积分值设为 0 时，打卡不发放积分但功能正常使用。达到每日上限后，打卡仍然成功但不再发放额外积分。

### 二、积分规则配置（管理员操作）

管理员登录后台后，进入"积分管理 → 积分规则配置"：

1. 页面已**去除 Tab 切换**，所有配置项在同一页面展示
2. 在"任务奖励"卡片中可以看到以下配置项：
   - **分享文章**：分享一篇文章获得的积分
   - **完善健康档案**：首次完善健康档案获得的一次性积分（已改名）
   - **评价服务**：评价一次服务获得的积分
   - **健康打卡**：每次健康打卡获得的积分（设为 0 表示关闭打卡送积分）
   - **健康打卡每日上限**（新增）：每日通过打卡可获得的最大积分数（设为 0 表示不限制）
3. 修改配置后，点击右上角"保存规则"按钮即可生效

### 三、完善健康档案获取积分（用户操作）

1. 在 H5 端或 App 端进入"健康档案"页面
2. 编辑"本人"记录，确保以下 4 个字段全部填写：
   - **性别**
   - **生日**
   - **身高**
   - **体重**
3. 点击保存后，如果这是您首次完善这 4 个字段，系统将自动发放积分奖励
4. 积分数量由后台"完善健康档案"配置项决定

> ⚠️ 此积分为**一次性奖励**，每个用户终身只能获得一次。后续再次修改健康档案不会重复发放积分。

## 注意事项

1. **打卡积分配置**：如果后台将"健康打卡"的积分值设为 0，则打卡不会发放积分，但打卡功能本身不受影响
2. **每日上限**：当"健康打卡每日上限"设为 0 时，表示不限制每日积分上限；设为具体数值时，用户当日打卡积分达到上限后，继续打卡仍然成功但不再发放积分
3. **完善健康档案**：必须 4 个核心字段（性别、生日、身高、体重）全部填写才会触发积分发放
4. **异常容错**：即使积分系统出现异常，也不会影响打卡和健康档案保存功能的正常使用
5. **权限说明**：积分规则配置仅管理员可操作；普通用户通过打卡和完善健康档案获取积分
