## 功能简介

本次更新对接「健康档案管理优化方案 PRD v1.0」，把多年沿用的「我守护的人」概念在**资产/配额语境**下统一升级为「健康档案列表」，与京东健康、平安健康、丁香、微医等主流产品对齐——把"档案"作为长期持有的资产单位（**份**），而把"守护人"留给关系/邀请等情感语境继续使用。

主要变化一句话：**"守护人数 X 人"→ "可管理健康档案 X 份（含本人）"**。

### 升级亮点

1. **概念升级**：入口卡片、详情页、面包屑、空状态、达上限提示统一改为「健康档案列表 / 健康档案已满」。
2. **配额口径明确**：本人档案**计入**配额；「档案存在 = 名额占用」「档案删除 = 名额释放」；邀请被拒/退出/过期不会自动释放，需用户手动删除档案才释放。
3. **会员权益清晰**：会员中心权益卡 / 权益对比表展示 "可管理健康档案 X 份（含本人）"，与主流产品口径一致。
4. **达上限弹窗 C 方案**：新弹窗带场景化标题（列表页「健康档案已满」/ 添加成员页「守护人名额已满」），并列出"升级 VIP/SVIP 可管理 X 份"的具体升级路径；如已是最高档则提示"联系客服扩容"。
5. **关系/邀请措辞保留温度**：「添加成员」「邀请 TA 成为我守护的人」「邀请短信」等关系语境保持不变，不丢失产品情感。
6. **零迁移成本**：数据库字段 `max_managed`、接口契约**完全不动**，仅前端文案 + 后端注释更新，运营在管理后台改数值即可调整免费/VIP/SVIP 三档配额。

## 使用说明

### 一、在 H5 用户端体验新文案与新弹窗

1. **打开健康档案首页**
   - 进入：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/)
   - 在「本人」Tab 下，可以看到左侧入口卡片标题已升级为「健康档案列表」，副标题改为「健康档案：X 份（含本人，上限 Y 份）」。

2. **进入健康档案列表详情页**
   - 点击「健康档案列表」入口卡 → 跳转 `/health-profile/i-guard/`。
   - 顶部统计栏标题显示「📋 健康档案列表」，右上角紧凑摘要为「已管理 X/Y 份」，正文显示「已管理 X 份档案（含本人），还可新增 Y 份」。
   - 「+ 发起邀请」按钮位置不变。

3. **触发达上限弹窗（C 方案 + 场景化标题）**
   - **列表页场景**：当配额已满时点击「+ 发起邀请」，弹出标题为 **「健康档案已满（X/Y）」** 的弹窗。
   - **添加成员页场景**：当配额已满时点击某未绑定档案的「重新邀请」按钮，弹出标题为 **「守护人名额已满（X/Y）」** 的弹窗。
   - 弹窗正文列出**升级路径**：「升级 VIP 可管理 N 份」「升级 SVIP 可管理 M 份」「升级后可继续为更多家人建立健康档案」。
   - 弹窗操作：**[查看会员权益]**（跳转 `/member-center#plans`） / **[取消]**。
   - **最高档场景**：如果当前已经是最高档付费会员（再无更高档可升），弹窗文案变体为「您已是最高档会员，可联系客服申请扩容」，操作按钮变为 **[联系客服] / [取消]**。

4. **会员中心权益对比**
   - 进入：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center/)
   - 「我的会员权益」首格的数字会自动换算为 **含本人后的总份数**（例如配置 max_managed=3 → 显示 4 份）。
   - 「升级享更多权益」每个套餐卡描述行：「可管理健康档案 X 份（含本人）· AI 外呼 ... · 紧急呼叫 ...」。
   - 「权益对比」表第一行标签为「**可管理健康档案（含本人）**」，每个套餐数值自动展示 "X 份"（不限档显示为「不限」）。

### 二、关系/邀请措辞保留（不变）

以下场景**继续保留**「守护」措辞，温度不丢失：

- 「添加成员」按钮 / 抽屉标题
- 「邀请 TA 成为我守护的人」按钮文字
- 邀请短信、邀请确认页 中的「守护对象 / 守护人」措辞
- 「守护我的人 / 守护 TA 的人」双向关系视图
- 小程序「家庭守护列表」AI 外呼提醒页

### 三、配额规则一句话

> **「档案存在 = 名额占用，档案删除 = 名额释放」**
>
> - **本人档案**：永久占用 1 个名额（注销账号才释放）。
> - **代管档案 / 邀请中 / 已绑定**：占用 1 个名额。
> - **邀请被拒 / 对方已退出 / 邀请已过期**：档案保留，名额**继续占用**——只有用户在列表页手动「彻底删除」该档案，名额才会释放。
> - **会员升级**：立即可建至新档配额上限，无需重新登录。

### 四、运营在管理后台配置档位配额

1. 进入：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/membership/plans/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/membership/plans/)
2. 在「可管理健康档案」字段填写 **"家人档案数（不含本人）"**，例如：
   - 免费会员：2（用户实际可建 3 份，含本人）
   - VIP：7（用户实际可建 8 份）
   - SVIP：19（用户实际可建 20 份）
   - 不限：-1
3. 用户端展示时系统会自动 +1 显示「X 份（含本人）」，运营无需关心展示细节。

## 注意事项

1. **数据库字段保留**：`max_managed` 字段名在数据库、接口、模型中**完全保留**，本次仅升级前端展示文案与后端注释。如果你有针对该字段名的第三方集成代码，**无需任何改动**。
2. **配额数值由运营在后台配置**：当前已部署的免费 / VIP / SVIP 三档配额请运营按 PRD 第 5 节建议值在管理后台直接配置，**无需改代码**即可生效。
3. **关系/邀请措辞不要改成「档案」**：邀请短信、添加成员按钮、双向关系视图请**保持「守护」措辞**——这是与 PRD §3.2 一致的设计原则，丢失温度会引起用户体感下降。
4. **达上限弹窗触发位置**：
   - **列表页 + 发起邀请**：标题「健康档案已满」
   - **未绑定卡片重新邀请**：标题「守护人名额已满」
   - 两者正文与按钮一致，仅标题区分场景。
5. **不限档显示**：`max_managed = -1` 或 ≥9999 时统一显示「不限」，套餐描述行显示「可管理健康档案 不限」，权益对比表显示「不限」。
6. **会员升级即时生效**：升级到 VIP / SVIP 后无需重新登录，回到列表页刷新即可看到新的配额上限。
7. **小程序需要重新导入**：本次小程序代码已更新，请用最新 zip 重新导入或在开发者工具中点击「同步」按钮拉取最新源码。
8. **多端数据同步**：H5 删档 → 小程序刷新即可看到名额计数实时同步。
