## 功能简介

「**数字安全绳**」是面向独居长辈的关怀模式核心子功能。功能本身（4 个 Bug 修复）已在上一轮完成代码落库，但**部署阶段未把新代码真正同步到测试服务器**——后端 `main.py` 仍是老版本（不含 `safety_rope_v1` 路由注册），H5 容器构建时也未拿到含「数字安全绳」入口的 `care-home/page.tsx` 新版。表现为：13 项端到端冒烟里 12 项失败、4 个 Bug 全部"假修复"。

**本轮修复彻底完成 4 件事**：

1. **后端 `safety_rope_v1` 路由真正注册**：6 个 API（`status` / `config` / `contacts` / `contacts/check-phone` / `checkin` / `contacts/{id}` 增删改查）从 404 变为 401（带 token 即 200）
2. **H5 `/care-home` 数字安全绳入口真正可见**：绿色横卡 `data-testid="care-home-safety-rope-entry"` + 右下角悬浮球 `data-testid="care-home-safety-rope-fab"`
3. **容器无缓存强制重建**：`docker compose build --no-cache backend h5-web` + `up -d --force-recreate`，根治"以为部署成功、实际跑旧镜像"
4. **13 项端到端冒烟 13/13 全过**：包括关系字段 7 芯片、阈值 24/48 选中态、OpenAPI 标签等所有验收点

## 使用说明

### 入口路径一（关怀模式首页主入口横卡 — 推荐）

1. 浏览器打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/care-home](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/care-home)
2. 在「快捷胶囊」（档案/用药/拍照问AI/家人）下方，可以看到一张**绿色横卡**：
   - 左侧 🪢 图标
   - 标题：**数字安全绳**
   - 副标题：每天点一下"我今天平安"，超时自动通知亲友
   - 右侧箭头 `›`
3. 点击横卡 → 进入数字安全绳功能页

### 入口路径二（关怀模式首页右下角绿色悬浮球 — 紧急直达）

1. 同样在 [关怀模式首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/care-home) 下
2. 屏幕**右下角**有两个悬浮球：
   - 红色 SOS 球（最右）
   - **绿色「🪢 安全绳」球**（紧贴 SOS 左侧）
3. 点击绿色悬浮球 → 进入数字安全绳功能页

### 数字安全绳功能页操作

打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/care-safety-rope](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/care-safety-rope) 后：

#### 1. 顶部签到 banner（Bug4 修复）
- 未签到：橙色 banner 提示"今日还未签到"
- 已签到：banner 立刻刷新成两行
  - 第一行：✅ 上次签到 MM-DD HH:mm
  - 第二行：⏰ 下次签到截止 MM-DD HH:mm
- 点击大号按钮「**🪢 我今天平安**」即可签到，banner 立即更新（无需手动刷新）

#### 2. 阈值选择（Bug2 修复）
- 提供两个阈值按钮：**24 小时** / **48 小时**
- 选中态：**绿色填充背景 + 加粗文字 + 右上角白底绿勾 ✓**
- 未选中态：白底灰边、普通文字
- 视觉对比清晰，点哪个亮哪个

#### 3. 紧急联系人管理（Bug3 修复）
- 顶部点「+ 添加」打开抽屉
- 表单字段：
  - **姓名**（必填，最长 50 字）
  - **手机号**（必填、必须为 11 位 13~19 开头、且**必须是 bini-health 已注册用户**）
    - 失焦时自动校验：未注册 → 红色提示"该手机号还未注册 bini-health，请先邀请 TA 注册"
    - 已注册 → 绿色对勾 ✓ + "已识别用户：XXX"
  - **关系**（可选，**7 芯片单选**：子女 / 配偶 / 父母 / 邻居 / 朋友 / 护工 / 其他）
  - **不再有邮箱字段**（避免 SMTP 阻塞导致"添加成功但列表无记录"的旧 Bug）
- 「保存」按钮在校验未通过时灰色禁用
- 保存成功后：
  - 列表立即出现这条记录
  - 对方（被设为联系人的注册用户）会收到一条**站内信**：「【数字安全绳】您被 XXX 设为紧急联系人」
- 最多 3 位紧急联系人

#### 4. 超时预警与解除
- 当连续未签到超过阈值（24/48 小时）：
  - 系统**每 5 分钟**扫描一次
  - 触发预警时，**给所有 matched 联系人发站内信**：
    - 标题：【数字安全绳预警】XXX 超时未签到
    - 内容：含最后签到时间、最后位置、TA 的手机号、行动建议
  - 给本人也发一条「您已超时未签到」站内信
- 重新签到后：
  - 自动解除 alerting 状态
  - 给联系人再发一条「【数字安全绳·已平安解除】XXX」站内信

## 注意事项

1. **本次修复不改动客户端**：本会话仅修复**部署链路**——把已写好的本地代码真正同步到测试服务器并强制重建容器。**没有改动任何小程序/Android/iOS/Windows 桌面端的源代码**，所以**用户无需重新下载安装包**，原有 APK / 小程序包继续可用。
2. **首次访问需登录态**：所有 `/api/safety-rope/*` 接口都需要带 token，未登录访问会返回 401（这是正常现象，不是 Bug）。
3. **手机号校验依赖用户表**：紧急联系人必须是 bini-health 已注册用户。如果手机号未注册，请先邀请 TA 注册。
4. **关系字段白名单**：后端会强制校验关系必须是 7 芯片之一（子女/配偶/父母/邻居/朋友/护工/其他），任何其他值都会被拒绝。
5. **预警仅站内信，不发邮件**：上一轮 v2 锁死版已彻底移除 SMTP 邮件，国内场景不需要。如果联系人没收到预警，请检查是否已注册（matched_user_id 为 NULL 时不会被通知）。
6. **后端镜像构建时间戳**：本次重建后，`6b099ed3-7175-4a78-91f4-44570c84ed27-backend` 容器与 `6b099ed3-7175-4a78-91f4-44570c84ed27-h5` 容器均已重新创建（时间戳为 2026-06-03 17:41 北京时间）。可通过 `docker inspect <container> --format '{{.Created}}'` 验证。
