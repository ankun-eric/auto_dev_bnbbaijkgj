## 功能简介

本次迭代对**积分模块的 6 个页面**做了一次彻底的视觉换肤，目标是让积分模块的视觉语言与 AI 对话首页保持一致，消除「进入积分页面像跳到另一个 App」的割裂感。

### 改造范围（6 个页面，三端同步）

| 页面 | 改造类型 |
|------|----------|
| 积分主页 `/points` | 仅换肤（结构不动） |
| 积分商城列表 `/points/mall` | 换肤 + **新增顶部 banner**（我的积分 + 去赚积分） |
| 积分商品详情 `/points/product-detail` | 换肤 + 底部 CTA 升级（天蓝实心 + 白字 + 8px 圆角） |
| 积分明细聚合 `/points/detail` | 仅换肤（结构不动） |
| 旧版积分流水 `/points/records` | 仅换肤 |
| 兑换记录 `/points/exchange-records` | 仅换肤（顶部装饰条改天蓝渐变） |

### 核心视觉变化（橙金/绿色 → 天蓝色）

- **主操作色**：所有主要按钮、积分数字、激活态 tab、签到按钮 → 由原橙金色（`#FF8A00`）/ 旧绿色（`#1B5E20`、`#2E7D32`、`#4CAF50`）**统一改为天蓝色 `#0EA5E9`**
- **装饰渐变背景**：积分主页顶部、商城 banner、兑换记录顶部条 → **统一改为 `#F0F9FF → #DBEAFE` 浅蓝渐变**
- **强调描边 / 次级元素**：卡片边框、次级按钮描边 → 使用 `#7DD3FC`
- **正向积分数字**（流水中的「+50 积分」）：天蓝 `#0EA5E9`
- **负向积分数字**（流水中的「-100 积分」）：**由红色改为灰色 `#999999`**（PRD 决策：让消耗显得不那么刺眼）
- **商城顶部 banner（全新）**：左侧显示「我的积分 XXXX 分」，右侧两个胶囊按钮「去赚积分 →」（跳积分主页）和「兑换记录」

### 主 CTA 按钮规范

| 项 | 规范值 |
|------|--------|
| 背景 | `#0EA5E9` 实心 |
| 文字 | `#FFFFFF` 白色 |
| 字号 | 16px |
| 圆角 | 8px |
| 高度 | 44px |
| Disabled | 背景 `#BAE6FD`，文字白色 0.6 透明 |

应用位置：商品详情底部「立即兑换」、积分商城卡片「立即兑换」、签到「去签到」按钮等。

## 使用说明

### 1. 体验 H5 端积分模块（最快）

1. 用浏览器访问 [积分主页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/points)
2. 第一眼应看到顶部「我的可用积分」卡片是**浅蓝渐变背景**（不再是绿色），积分数字也是**天蓝色**
3. 点击「积分商城」入口，进入商城页面
4. 商城页面顶部能看到全新的 **banner**：左侧「我的积分 XXXX 分」，右侧两个按钮：
   - 「去赚积分 →」：点击跳回积分主页（继续做任务赚积分）
   - 「兑换记录」：点击进入兑换记录页
5. 浏览商品卡片，确认卡片底部「立即兑换」按钮是**天蓝色实心 + 白字**
6. 点击任一商品进入详情页，底部「立即兑换」按钮是**天蓝色实心 + 白字 + 圆角 8px**

### 2. 验证积分明细页

1. 从积分主页右上角点击「积分明细 ›」进入
2. 顶部 tab 切换到「积分明细」
3. 流水列表中：
   - 正向积分（如「+50 每日签到」）→ **天蓝色 `#0EA5E9`**
   - 负向积分（如「-100 兑换商品」）→ **灰色 `#999999`**（不再是红色 / 橙金色）

### 3. 验证签到功能

1. 在积分主页找到「每日签到」任务
2. 点击「去签到」按钮（天蓝色）
3. 完成签到后按钮变为「已签到」灰色态
4. 顶部积分数字会实时刷新

## 注意事项

1. **三端视觉一致**：H5 / 小程序 / Flutter APP 三端的积分模块视觉表现完全一致，使用同一套配色规范。
2. **功能 0 变更**：本次仅是视觉换肤，**所有积分功能（签到、查询、商城浏览、兑换流程、跳转路径等）100% 保持不变**。
3. **负向积分由红改灰**：流水中的「-100 积分」原本是红色，现在统一为灰色 `#999999`，让消耗显得不那么刺眼。如有疑问可联系客服。
4. **缓存清理建议**：如果在 H5 中仍看到老的橙色/绿色，建议**清空浏览器缓存后强制刷新**，或在浏览器开发者工具中关闭缓存。
5. **小程序需重新导入**：因为小程序代码已重新打包，请下载最新 zip 后导入微信开发者工具体验。
6. **APP 需重新安装**：因为 APP 代码已重新打包，请下载最新安装包后**卸载旧版 → 安装新版**体验。
