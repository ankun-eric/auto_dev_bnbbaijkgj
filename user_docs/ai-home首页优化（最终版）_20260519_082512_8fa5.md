## 功能简介

本次 ai-home（AI 对话首页）优化包含 **3 个模块**，全部聚焦在 H5 端：

1. **识药结果卡片底部 4 按钮统一改造** — 卡片底部按钮统一为「💊 加入用药计划 / 📅 查看用药计划 / ⏰ 今日用药 / 📸 重新拍照」2×2 胶囊布局，置灰、红点、跳转、抽屉、Toast 全部按 PRD 行为对齐。
2. **铃铛位置 & 拖拽优化** — 🔔 铃铛初始位置改为「屏幕靠右 + 垂直正中（viewport 50%）」，拖拽后的位置自动持久化，下次进入页面时恢复。
3. **左上角「汉堡三 + 小康」对齐与样式改造** — 三横线由"全/全/左50%"改为"右50% / 全 / 左50%"的错落感，汉堡水平中线与「小康」文字视觉中线对齐。

## 使用说明

### 模块 1：识药结果卡片 4 按钮

在 ai-home 拍照识药后会出现「药品识别结果」卡片，其底部按钮全新升级。

#### 1.1 按钮排布

```
┌──────────────────────────────────────┐
│ ① 药品基础信息（通用名 / 商品名 / ...）       │
│ ② 用法用量                                  │
│ ③ 安全提示                                  │
│ ④ 个性化风险                                │
├──────────────────────────────────────┤
│ [💊 加入用药计划]    [📅 查看用药计划]     │  ← 上行
│ [⏰ 今日用药]        [📸 重新拍照]         │  ← 下行
└──────────────────────────────────────┘
```

胶囊样式（高度 36px、圆角 999px、浅紫底 + 紫色文字、emoji 在前文字在后），在 iPhone SE / 360px 窄屏下文字自动从 14px 缩小到 12px，保持 2×2 不换行。

#### 1.2 「💊 加入用药计划」

- **正常态**：点击 → 弹出底部抽屉「加入用药计划」（已自动带入药品名）→ 保存 → 抽屉自动关闭 → 按钮立即变为「✓ 已加入」灰态 → 顶部弹出 1.5 秒 Toast「已加入用药计划」。
- **置灰态**：识药失败时按钮变灰，点击无任何响应（无 Toast、无跳转、无震动）。
- **同步**：如果您在「医药计划」页手动删除该药计划，再回到 ai-home 时本按钮会自动变回亮色可点态。

#### 1.3 「📅 查看用药计划」

任何时候都可点。点击直接跳转到 `/ai-home/medication-plans?tab=in_progress`（医药计划三 Tab 列表页，**强制定位到"服药中"Tab**）。

> 即使您正在查看的药已经停药，也会先进入「服药中」Tab，由您自行切换到「已结束」Tab。

#### 1.4 「⏰ 今日用药」（重点交互）

| 场景 | 按钮表现 | 点击行为 |
|------|---------|---------|
| 今天有用药计划 | 亮态 + 右上角 8px 红点 | 从底部弹出「今日待办」抽屉，复用打卡页能力（顶部 Banner / 即将服用 / 时间线 / 撤销） |
| 今天无用药计划 | 置灰（emoji 降饱和、文字中灰、底色浅灰） | 完全无响应 |
| 进入页面瞬间（数据未拉到） | 亮态 + **暂不显示**红点 | （数据返回后再决定红点亮灭，避免闪烁） |

红点规则严格按"今天有计划就亮"：
- 打卡 / 撤销 **不影响** 红点（即使全部打完卡，红点仍亮）
- 跨天（次日 00:00 后）自动按新一天的"是否有计划"重新计算
- 与「健康档案 Hero」的「今日用药」徽标 **完全同源**（共用 `/api/medication/today`）

抽屉支持三种关闭方式：
1. 右上角 ×
2. 点击遮罩
3. 下滑手势

抽屉内的打卡 / 撤销不会让红点闪烁。

#### 1.5 「📸 重新拍照」

任何场景下均可点击（含识药失败场景）。点击触发系统相机或原有拍照流程。

#### 1.6 置灰规则速查

| 场景 | 💊 加入 | 📅 查看 | ⏰ 今日 | 📸 重拍 |
|------|---|---|---|---|
| 识药成功 + 今天有计划 | 亮（或「✓ 已加入」） | 亮 | 亮 + 红点 | 亮 |
| 识药成功 + 今天无计划 | 亮 | 亮 | **置灰** | 亮 |
| 识药失败 + 今天有计划 | **置灰** | 亮 | 亮 + 红点 | 亮 |
| 识药失败 + 今天无计划 | **置灰** | 亮 | **置灰** | 亮 |

### 模块 2：铃铛 🔔（位置 & 拖拽）

- **初始位置**：屏幕靠右（`right: 12px`）+ 视口（viewport）垂直正中（约屏幕高度的 50%）。
- **拖拽**：长按 200ms 后进入拖拽态，可上下拖动；释放后停在新位置。
- **持久化**：拖动后的位置会自动保存到本地。下次进入 ai-home，铃铛在 **上次拖动后的位置** 出现。
- **边界保护**：横竖屏切换、设备宽高变化时，若上次位置已经越界，会自动收回到屏幕安全区内（距边缘至少 8px）。

> 持久化使用浏览器 `localStorage`，key 为 `bini.ai-home.bell.position`。如需清除可在浏览器开发者工具 → Application → Local Storage 中删除。

---

### 模块 3：左上角「汉堡 ☰ + 小康」

- **汉堡三横线**改为"右-全-左"错落感：
  ```
  ━━━━━     ← 第 1 条：50% 长度，右对齐
  ━━━━━━━━━━ ← 第 2 条：全长
  ━━━━━     ← 第 3 条：50% 长度，左对齐
  ```
- 三条横线颜色与「小康」文字一致（深灰色 `#1F2937`）。
- 汉堡水平中线与「小康」文字视觉中线对齐（误差 ≤ 1px）。
- 右上角红点规则保持现状（未读消息 > 0 或待使用订单 > 0 时显示）。

## 注意事项

1. **本次仅 H5 端有改动**：微信小程序、Flutter App（安卓 / iOS）此次未变更，不需要更新这些端。仍可继续使用旧版本。
2. **同源接口**：「今日用药」红点 / 健康档案 Hero「今日用药」徽标 / 「今日用药」抽屉内容 全部来自同一个后端接口 `/api/medication/today`，三者同步亮灭、不会出现"红点亮但抽屉空"的不一致。
3. **置灰按钮**：置灰态点击**完全无响应**——既不会跳转，也不会出现 Toast 或震动反馈。这是 PRD 规定的设计行为，并非 Bug。
4. **铃铛位置清除**：若想恢复"屏幕右 + 垂直正中"的默认位置，请在浏览器 DevTools 中清除 localStorage 的 `bini.ai-home.bell.position` 字段后刷新页面。
5. **窄屏适配**：在 iPhone SE / 360px 宽度等小屏设备上，按钮文字会自动缩小到 12px（emoji 13px），保持 2×2 网格不换行。
6. **跨页面状态同步**：如果您在「医药计划」页里删除了某药，回到 ai-home 时该药对应识药卡片的「💊 加入用药计划」按钮会自动变回亮色（无需刷新）。

## 出现问题怎么办

| 现象 | 排查 |
|------|------|
| 红点没显示 | 确认"今天确实有用药计划"。若刚加入计划，点击「今日用药」按钮抽屉关闭时会触发刷新 |
| 加入计划后按钮没变灰 | 等 1~2 秒（抽屉关闭 → 接口刷新有少量延迟）；若仍不变请退出再进 ai-home |
| 铃铛拖到屏幕外 | 横竖屏切换或刷新页面时会自动收回安全区内（距边缘至少 8px） |
| 「今日用药」抽屉空 | 抽屉里只显示今天到点的提醒。若所有时间点都未到，抽屉可能为空 |
