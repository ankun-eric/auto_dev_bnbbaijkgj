## 功能简介

本次修复解决了系统每次发布/重启时**自动覆盖运营人员手动修改的业务数据**的问题。修复后，以下4类数据将完全由运营在后台自主管理，系统重启不再干预已有数据：

| 修复项 | 修复前的问题 | 修复后的效果 |
|--------|-------------|-------------|
| 商品分类自动同步 | 每次重启，已删除的旧服务分类（专家咨询等）自动"复活"到商品分类中 | 彻底移除自动同步逻辑，商品分类完全由运营管理 |
| 角色模板覆盖 | 每次重启，后台修改的商家角色权限被重置为代码默认值 | 角色模板仅在首次不存在时插入，已有数据不再覆盖 |
| 管理员密码重置 | 每次重启，修改过的管理员密码被重置为 admin123 | 管理员密码仅在首次创建时设定，后续修改不受影响 |
| 短信配置覆盖 | 每次重启，后台修改的短信签名/模板被环境变量覆盖 | 短信配置仅在首次为空时初始化，后续由运营在后台管理 |

## 使用说明

### 1. 商品分类管理（修复项①）

**修复前**：在管理后台删除商品分类后，系统重启会自动从旧服务分类表同步数据，导致已删除的分类"复活"。

**修复后**：商品分类完全由运营自主管理，增删改操作在重启后保持不变。

**操作步骤**：
1. 登录管理后台：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 进入「商品系统 → 商品分类」页面
3. 可自由添加、编辑、删除商品分类
4. 系统重启后，您的修改将被完整保留

### 2. 商家角色权限管理（修复项②）

**修复前**：在后台修改商家角色（老板、店长、财务、店员）的权限模块后，系统重启会将权限重置为代码中的默认值。

**修复后**：角色权限修改在重启后保持不变。4个内置角色（boss/manager/finance/clerk）仅在数据库中完全不存在时才自动创建。

**操作步骤**：
1. 登录管理后台
2. 进入商家角色管理页面
3. 修改任意角色的权限模块配置
4. 系统重启后，权限配置保持修改后的状态

### 3. 管理员密码管理（修复项③）

**修复前**：修改管理员密码后，系统重启会将密码强制重置为 `admin123`。

**修复后**：管理员密码仅在首次创建账户时使用默认值，后续修改的密码在重启后保持不变。

**操作步骤**：
1. 使用当前密码登录管理后台
2. 进入个人设置修改密码
3. 修改后的新密码在系统重启后仍然有效
4. 如需首次登录，默认手机号 `13800000000`，默认密码 `admin123`

### 4. 短信配置管理（修复项⑤）

**修复前**：在后台修改短信签名、模板ID、SDK配置后，系统重启会用环境变量中的值覆盖数据库中的配置。

**修复后**：短信配置仅在数据库中无记录时使用环境变量初始化，后续由运营在后台自主管理。

**操作步骤**：
1. 登录管理后台
2. 进入短信配置管理页面
3. 修改短信签名、模板、SDK 配置等
4. 系统重启后，配置保持修改后的状态

## 注意事项

1. **全新部署（空数据库）**：所有默认数据（管理员账号、内置角色、短信配置等）仍会在首次启动时自动创建，无需手动操作。
2. **管理员密码遗忘**：修复后系统不再自动重置密码，如忘记密码需通过数据库手动重置。
3. **角色模板误删**：如果不小心删除了内置角色模板，可通过手动执行 SQL 重新插入，或联系技术人员恢复。
4. **短信配置清空**：如果将短信配置全部清空后重启，系统会用环境变量重新初始化默认配置。
5. **旧商品分类清理**：已存在于商品分类表中的旧服务分类（如"专家咨询"等）不会自动删除，需要运营人员在后台手动删除不需要的分类。
