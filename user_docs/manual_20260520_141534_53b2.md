## 功能简介

本次更新解决了三个相互关联的体质测评相关 Bug，并附带实现了一系列增强功能：

1. **Bug 1（已修复）—— 加载问卷模板失败**：管理员在「功能按钮管理」编辑「体质测评」按钮时，关联问卷模板下拉框只显示数字 ID 而非名称，并提示「加载问卷模板失败，请重试」。原因是前端拉取了 `page_size=200`，超过后端 `Query(le=100)` 校验上限，FastAPI 直接返回 422。本次修复双向加固：前端改为 `page_size=100`，后端上限放宽至 `le=500`，并按 HTTP 状态码细化错误提示。
2. **Bug 2（已修复）—— 36 题模板真正落库**：服务器启动时迁移脚本会自动检测 `tcm_constitution` 模板下题目数是否为 36，若不为 36 则软删旧题目并批量重写 36 道标准题干（含第 34/35/36 反向计分题）。运维可在 backend 容器日志中 `grep tcm36` 看到 `template_id=X question_count: before=Y after=36` 与 `36 questions OK`。
3. **Bug 3（已修复）—— 卡片正常弹出**：H5 端在点击「体质测评」按钮但发现按钮未关联模板 ID 时，不再静默无反应，而是显式 Toast 提示「该功能按钮未关联问卷模板，请联系管理员前往「功能按钮管理」补充配置」，方便定位问题；同时已有的 INLINE_CHAT 卡片渲染逻辑确认无需改动。
4. **附带增强**：
   - 新增 `POST /api/chat/intent-detect` 关键词 + 意图双兜底接口
   - 在大模型 `system_prompt` 拼装中注入用户最近一次主体质（≤80 字）
   - 新增 `GET /api/questionnaire/answers/{answer_id}/follow-up` 主动追问接口
   - 新增 `backend/scripts/_seed_tcm36.py` 一次性手动补数脚本

## 使用说明

### 一、管理员场景：修复后重新配置「体质测评」按钮

1. **打开管理后台**：在浏览器中打开 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin) 并使用管理员账号登录
2. **进入「功能按钮管理」**：在左侧导航中找到「功能按钮管理」菜单并点击进入
3. **找到「体质测评」按钮**：在按钮列表中找到「体质测评」相关按钮（如「中医体质测评」），点击右侧「编辑」
4. **核对配置**：在编辑弹窗中：
   - 「按钮类型」选择「AI 功能」
   - 「AI 功能子类型」选择「对话内问卷」
   - 「关联问卷模板」下拉框此时应**正常显示完整模板列表**（包含「中医体质测评（王琦 36 题版）」名称及编码）
   - 选择「中医体质测评（王琦 36 题版）」
   - 「问卷展示形态」可选 `INLINE_CHAT`（对话内说明卡片）或 `DRAWER_STEPPED`（一题一屏抽屉）
   - 「弹出说明卡片」开关建议开启（pre_card_enabled=true）
5. **保存**：点击「保存」按钮，提示「保存成功」即可
6. **如出现红字**：
   - 若提示「请求参数超出限制（page_size 上限），请联系运维」→ 联系开发再次扩大后端上限
   - 若提示「登录态失效或无管理员权限，请重新登录」→ 重新登录
   - 若提示「加载问卷模板失败（XXX），请重试」→ 点击重试按钮再次加载

### 二、终端用户场景：体验体质测评

1. **打开 H5 主页**：在手机浏览器或微信中打开 [H5 用户端](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/) 并登录
2. **进入 AI 主页**：登录后默认进入 AI 对话主页
3. **触发方式一（点击按钮）**：在底部胶囊栏或宫格区找到「体质测评」按钮，点击
   - 若按钮已正确配置：对话区会立即出现「我要做体质测评」用户气泡 + AI 回复气泡 + 「中医体质测评（王琦 36 题版）」说明卡片
4. **触发方式二（输入关键词）**：在底部输入框中输入「我要做体质测评」「我是什么体质」「九种体质」等关键词中的任意一句，发送
   - 后端会通过 `/api/chat/intent-detect` 命中关键词，对话区出现同款说明卡片
5. **开始测评**：点击说明卡片上的「开始」按钮，问卷抽屉将从底部弹起，按一题一屏方式逐题作答（共 36 题，预计 5 分钟）
6. **查看结果**：完成全部 36 题后会出现 6 屏结果页（包含主体质、兼夹体质、调理建议等），并在健康档案中留下记录
7. **AI 主动追问**：结果页关闭后，对话流会自动推送一条 AI 消息「您想了解 {主体质} 的具体调理方法吗？」，点击或回复后可深入了解
8. **后续 AI 对话**：之后向 AI 提问健康类问题时（如「我能吃冰淇淋吗」），AI 将自动结合您的主体质给出针对性建议

## 注意事项

1. **历史答卷不会受影响**：本次重建 `questionnaire_question` 时虽然会软删/重新插入 36 道题，但 `questionnaire_answer` 表中答卷的 `answers` JSON 字段独立存储，不依赖 `question_id` 外键，因此**历史答卷数据完全保留**。
2. **模板 ID 可能因环境不同**：部分环境数据库中 `tcm_constitution` 模板 ID 是 2（如本环境），与 PRD 文档中假设的 3 不同，属正常情况，按代码逻辑通过 `code='tcm_constitution'` 自动定位，无需修改任何配置。
3. **意图识别 24h 单用户单次去重**：同一用户 24 小时内已主动做过测评后，AI 不会再次主动推荐测评，避免打扰。
4. **AI 上下文长度控制**：注入到 system prompt 的体质上下文严格控制在 80 字以内，不影响对话长度上限。
5. **page_size 兼容性**：本次后端 `le=500` 已能覆盖一般运营场景下的下拉模板拉取需求；前端实际只请求 `page_size=100`，未达上限。如未来模板数超过 100，需要前端同步上调至 200/500。

## 故障排查

| 现象 | 排查方向 |
|------|---------|
| 管理后台依然提示「加载问卷模板失败」 | 1）按 F12 看 Network → 状态码若仍是 422，确认后端 `questionnaire.py` 容器内 `le=500` 已落（可用 `docker exec ... grep le=500 /app/app/api/questionnaire.py`）；2）若是 401/403，重新登录管理员账号 |
| 启动日志无 `[tcm36] 36 questions OK` | 进入 backend 容器手动执行 `python scripts/_seed_tcm36.py`，看末尾是否输出 `inserted=36` |
| H5 点击「体质测评」按钮无反应 | 1）F12 Console 看是否有 Toast；2）若 Toast 提示「未关联问卷模板」→ 让管理员重新配置；3）确认 `/api/questionnaire/templates` 接口返回的列表中 `tcm_constitution` 模板存在 |
| AI 回答不结合体质 | 1）确认用户已做过 1 次完整测评（答卷 status=completed）；2）确认按钮 `ai_reference_passive=true`（默认 1） |
| 主动追问消息未出现 | 1）确认按钮 `ai_reference_active=true`（默认 1）；2）确认前端调用 `/api/questionnaire/answers/{id}/follow-up` 返回 `enabled=true` |
