## 一、功能简介

本次更新修复了「**删除家庭成员删不掉、还弹出看不懂的报错**」的问题。

修复前：在「健康档案 → 家庭成员」里删除某个成员（例如名叫 **黎明** 的档案）时，如果这个成员名下还挂着病史、过敏史、体检报告等数据，会删除失败，并弹出一句让人摸不着头脑的提示：

> 「关联数据不存在，请检查所绑定的表单/分类是否有效」

用户根本不知道卡在哪、该怎么办。

修复后：系统会在删除前**一次性把该成员名下所有"挡路"的数据全部数清楚**，并用一句清楚的大白话告诉你：

> 「该成员名下还有 3 条既往病史、1 条过敏史、2 份体检报告、1 个服药计划，请先清空后再删除。」

你照着提示把这些数据清理干净，就能顺利删除，再也不用反复试、反复猜。

## 二、本次涉及的数据类别

删除一个家庭成员时，系统会自动排查并统计该成员名下以下各类数据（有就列出、并尽量带上准确条数）：

| 序号 | 数据类别 | 提示示例 |
|------|----------|----------|
| 1 | 既往病史（慢病 + 手术史） | 「3 条既往病史」 |
| 2 | 过敏史（药物 / 食物 / 其他） | 「1 条过敏史」 |
| 3 | 家族病史 | 「2 条家族病史」 |
| 4 | 健康记录（血压、血糖、心率、睡眠、血氧等） | 「12 条健康记录」 |
| 5 | 健康事件 / 时间轴 | 「2 条健康事件」 |
| 6 | 病历卡 / 正式病历 | 「2 份病历」 |
| 7 | 体检报告 | 「2 份体检报告」 |
| 8 | 用药提醒 | 「3 条用药提醒」 |
| 9 | 服药计划 | 「1 个服药计划」 |
| 10 | 中医诊断 / 体质结果 | 「1 条中医诊断记录」 |
| 11 | 健康提醒 | 「3 条健康提醒」 |
| 12 | 报告历史 | 「N 条报告历史」 |

> 说明：能精确数出条数的会带上准确数量；个别技术上确实数不出条数的，会只说类别、不强求数字（保证不误导）。

## 三、操作步骤

1. 打开 H5，进入「**健康档案**」页面。
2. 切换到「**家庭成员**」入口，找到要删除的成员（如「黎明」）。
3. 点击该成员右侧的「**更多**」菜单，选择「**删除**」。
4. 在弹出的「删除档案」确认框中点击「**确认删除**」。
5. **如果该成员名下没有任何挂载数据** —— 直接删除成功，提示「已删除」。
6. **如果该成员名下还有挂载数据** —— 不会再弹出看不懂的报错，而是清楚地一次性列出所有卡点，例如：
   > 该成员名下还有 3 条既往病史、1 条过敏史、2 份体检报告、1 个服药计划，请先清空后再删除。
7. 按提示进入对应模块，把这些数据逐一清空。
8. 全部清空后，再次执行删除即可成功。

---

## 四、注意事项

- **一次看全所有卡点**：如果一个成员同时卡了好几类数据，系统会**一次性把所有原因全列出来**，不会"删一次报一个、清掉再删又冒一个"，省去反复折腾。
- **不会自动偷偷删数据**：为保护你的数据安全，系统**不会**替你自动连同子数据一起删除，需要你确认清理后再删，避免误删重要健康记录。
- **纯文字提示、无跳转按钮**：提示就是一句话，干净直观，按提示去对应模块清理即可。
- **本次仅优化「删除家庭成员」这一个入口**：其他入口（如单独删除某条健康档案）不在本次范围内。
- **本人档案不可删除**：标记为「本人」的档案无法删除，属于正常限制。
- 删除前若该成员仍有**绑定设备 / 进行中的服药计划 / 守护关系 / 待处理邀请**，仍会有对应的「请先 XX」提示，按提示处理即可。

## 五、修复后的预期效果

1. 删除「黎明」这类成员时，不再弹出莫名其妙的报错，而是清楚告诉你「名下还有哪几类、各多少条数据」。
