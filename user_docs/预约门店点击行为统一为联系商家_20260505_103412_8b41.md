## 功能简介

本次更新优化了**订单详情页**中"**预约门店**"行的点击交互，让用户的操作更顺畅、更直观：

- **修复前**：在订单详情页点击"预约门店"那一行，会弹出一个"门店详情"弹层（多一层、信息重复），与底部的"联系商家"按钮行为不一致。
- **修复后**：点击"预约门店"整行，**直接弹出与底部"联系商家"按钮一模一样的弹层**——含**拨打电话**、**查看地址**、**一键导航**三大能力，操作路径完全统一。

修复在**微信小程序**、**H5**、**Flutter App（安卓 / iOS）**三端**同步生效**，行为完全一致。

## 使用说明

### 步骤 1：进入订单详情页

1. 打开 App / 小程序 / H5
2. 登录后，从首页或个人中心点击「**我的订单**」，进入订单列表
3. 在列表中点击任意一个**预约/核销类订单**（如待核销、已预约、部分核销），进入「订单详情页」

### 步骤 2：点击"预约门店"行（本次新交互）

在订单详情页中央，能看到一行展示门店信息的内容（不同端的呈现略有差异）：

| 端 | 位置 | 行内可见内容 |
|----|------|------|
| 微信小程序 | "服务门店" 卡片 | 门店名 / 门店地址 / 📞 电话 / 📍 导航 / 右侧 › |
| H5 | "预约信息" 卡片中的"预约门店"行；以及"订单地址"卡片中的"门店地址"行 | 📍 + 门店名 + 导航按钮 + 右侧 › |
| App（安卓 / iOS） | "订单地址" 卡片的"门店地址"行 | 🏪 + 门店名 + 门店地址 + 导航按钮 + 右侧 › |

**点击"预约门店"行（除内部的导航按钮和电话外）→ 立即弹出"联系商家"弹层**，弹层从底部滑出。

### 步骤 3：在"联系商家"弹层中操作

弹层提供以下三大能力：

1. **📞 拨打电话**：点击商家电话号码 → 直接呼出系统拨号面板
2. **🏠 查看地址**：清晰展示门店详细地址
3. **🧭 一键导航**：点击「导航」按钮 → 唤起高德/百度等地图 App，自动规划路线到门店

完成操作后点击关闭按钮（或弹层外蒙层）即可关闭。

### 步骤 4：验证两个入口完全等价

- 点击"预约门店"行 → 弹出 ✅ 联系商家弹层
- 点击底部"联系商家"按钮 → 弹出 ✅ **同一个**联系商家弹层

两种方式可任选其一，结果完全相同。

## 注意事项

1. **行为完全一致**：本次修复后，两个入口（"预约门店"行 / 底部"联系商家"按钮）触发的弹层是**同一个**弹层，含同样的电话/地址/导航三大能力。
2. **行内子按钮保留原行为**：行内的「导航」按钮、「📞 电话号码」点击行为不变，不会被整行点击吞掉（采用了事件冒泡拦截）。
3. **多种订单状态均生效**：待核销、已预约、部分核销、已过期、已完成等状态均可点击门店行。
4. **数据来源未变**：弹层中的电话/地址/坐标仍来自后端 `GET /api/stores/{id}/contact` 接口，未做改动。
5. **门店详情弹层已下线**：旧的"门店详情弹层"在本次入口已被联系商家弹层替代，避免功能重复带来的认知成本。
6. **H5 端无需用户操作**：H5 已自动部署到服务器，刷新页面即可看到新行为；小程序/安卓/iOS 三端需要重新下载/导入对应的最新包。
