## 三、使用说明

### 3.1 上传体检报告并体验 AI 解读

1. **进入体检报告入口**
   - **H5**：访问 [体检 / AI 报告解读入口](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/checkup)
   - **小程序 / App**：底部 Tab 进入「报告」或「体检」模块
2. **选择咨询人**：在弹出的咨询人选择面板中，选择"本次报告归属哪位家庭成员"（如本人 / 父亲 / 母亲 / 孩子等）
3. **拍照或从相册选择体检报告图片**：可一次选多张
4. **等待 OCR 识别**：本次修复后，单张图片 OCR 期望 5 秒以内完成
5. **自动跳转解读对话页**：跳转后 AI 立即开始流式输出本次报告的解读
6. **继续追问**：在底部输入框可以追问，例如"我这个项目偏高严重吗？"、"日常饮食要注意什么？"

### 3.2 顶部解读卡片的正确用法（Bug-03 修复）

- 进入解读对话页后，顶部「报告解读卡片」展示报告缩略图与解读摘要
- **本次修复后**：原【查看报告原图】按钮已去除，**直接点击缩略图本身**即可弹出大图预览（支持双指缩放、保存）

### 3.3 咨询人锁定规则（Bug-04 修复）

- **AI 报告解读 / 报告对比** 会话中，左下角"咨询人"选择器**置灰、不可点击**，咨询人始终为该报告/这组报告的真实归属人
- **普通 AI 健康咨询**（不带 `type` 参数的会话）**不受影响**，仍可自由切换咨询人
- **历史会话**进入时同样锁定，避免回看旧会话时误改导致 AI 上下文错位

## 七、注意事项

1. **图片压缩参数**：本次修复后三端统一使用 **最大边 ≤ 1920px、JPEG quality 80%、目标体积 ≤ 1MB**。如遇到极少数医院报告字体特别细小、压缩后 OCR 识别准确率下降，请反馈给我们以便后续放宽到 1.5MB。
2. **历史会话兼容**：Bug-04 的"咨询人锁定"已对**历史会话**生效；从「历史报告解读」列表点回旧会话，咨询人同样置灰，不会被误改。
3. **普通 AI 健康咨询不受影响**：不带 `type` 参数的普通问诊会话，咨询人选择器**仍可自由切换**，本次修复未误伤。
4. **服务端无需操作**：H5 与后端已通过 Docker 容器在服务器上自动重建并发布；用户访问 H5 链接 / 调用 API 都已经是最新版本。
5. **Bug-01 兜底机制**：即使将来 `report_interpret` 类型未匹配到专属 prompt 模板，前端也会**回退到通用模板**继续工作，不会再向用户抛"AI 服务未配置"提示，相关异常会写入后端日志便于运维定位。
