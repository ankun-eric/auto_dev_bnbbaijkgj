## 功能简介

本次为「时间显示统一治理」P0 级 Bug 修复，覆盖 H5、管理后台、微信小程序、Android、iOS 五个客户端的所有时间显示位置，统一以北京时间（东八区）为口径。修复亮点：

- **修复主 Bug**：血压卡片（首页）「今日 02:32」少 8 小时的现象，修复后显示真实北京时间「今日 10:32」。
- **修复同类 Bug**：血压详情页顶部"最近一次"、当日趋势图（X 轴小时分布）、近 7 天周视图聚合也都按北京时间口径正确切日。
- **统一时间工具**：四端全部新增/升级 `formatFriendlyTime`（今日 HH:mm / 昨日 HH:mm / X 天前 / MM-DD / YYYY-MM-DD）和 `formatFullTime`（YYYY-MM-DD HH:mm:ss）函数。
- **跨时区一致**：海外用户、UTC 服务器、UTC 浏览器看到的时间结果与国内一致（以北京时间显示）。
- **兼容历史数据**：服务端返回 `"2026-05-30 10:32:41"`（无时区后缀）/ `"2026-05-30T02:32:41Z"`（UTC）/ `"2026-05-30T10:32:41+08:00"`（带时区）三种格式都能解析为同一刻北京时间。

## 使用说明

### 一、H5 端验证（无需下载，直接浏览器访问）

1. 打开 [血压详情页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-metric/blood_pressure)
2. 在血压指标 Tab 中，添加一条手工测量记录（北京时间当下时刻）
3. 提交后，**卡片上"今日 HH:mm"应等于北京时间真实时刻**（修复前会少 8 小时）
4. 同步检查：详情页顶部"最近一次"时间，与历史记录列表第一条时间应**完全一致**

### 二、跨日临界验证（推荐回归测试）

1. 在北京时间 23:50 添加一条记录 → 查看应显示「今日 23:50」
2. 等到次日 0:10 → 再次查看，应显示「昨日 23:50」（按北京时间切日，不依赖设备时区）

### 三、海外/UTC 时区验证

1. 在 Chrome DevTools → Sensors → Location 改为 London/UTC 或 America/Los_Angeles
2. 刷新页面，时间显示**仍应是北京时间**（不再随浏览器时区漂移）

### 四、新增工具函数（开发者参考）

四端均提供以下函数：
- `parseServerTime(input)`：解析后端时间字符串为 Date（兼容 UTC + 无时区后缀 + 带时区后缀）
- `formatFullTime(input)`：北京时间 `YYYY-MM-DD HH:mm:ss`
- `formatFriendlyTime(input)`：友好时间 `今日 HH:mm` / `昨日 HH:mm` / `X 天前` / `MM-DD` / `YYYY-MM-DD`
- `formatDate` / `formatTime` / `formatRelativeTime`：兼容旧调用

## 注意事项

1. **时区基准**：本系统所有时间字段统一以北京时间（东八区）为口径，不再依赖设备本地时区。海外用户登录后看到的时间也是北京时间。
2. **后端老接口兼容**：系统已对所有可能的时间字符串格式做兜底解析（含无时区后缀的老接口字符串），开发者无需担心后端尚未规范化的字段。
3. **禁止裸用 `new Date(serverTimeStr)`**：开发者编写新页面时，必须从对应端的 `datetime` 工具文件导入 `parseServerTime` / `formatFriendlyTime` / `formatFullTime` 等函数，不得直接用 `new Date(...)` 解析后端返回的时间字符串，否则会再次出现时区漂移问题。
4. **第二批位置（下一轮上线）**：本次为「第一批用户高频可见位置」治理，下一轮将覆盖提醒推送时间、报告生成时间、设备同步时间、医生留言时间、操作日志/审计、后台管理等系统类时间位置。
5. **APK / IPA 安装权限**：未签名的应用首次打开会弹出"未知开发者"提示，按系统指引信任后即可使用。
