## 三、功能 1：个人信息页字段回填（PC + H5 双端）

### 3.1 操作步骤

#### PC 端

1. 用商家账号（如老板 6399 / 员工 6366）登录商家 PC 后台
2. 点击右上角头像 → "个人信息"，或直接访问上方"PC 后台 - 个人信息"链接
3. 现在你应该能看到完整的字段值

#### H5 端

1. 在手机浏览器打开"H5 移动端 - 我的"链接，登录账号
2. 在底部"我的"页中，点击「👤 个人信息」
3. 现在你应该能看到与 PC 端**完全一致**的字段值

### 3.2 你将看到的字段（一定有值）

| 字段 | 说明 |
|---|---|
| 姓名 | 用户的真实姓名；若数据库中未填写，自动展示为脱敏手机号（如 `138****6399`），**绝不为空** |
| 手机号 | 登录手机号 |
| 角色 | 老板 / 店长 / 财务 / 员工 |
| 所属门店 | 你被授权运营的所有门店列表 |
| 所属商家 | 你所在的商家名称 |
| 账号状态 | 正常 / 已禁用（**新增**） |
| 创建时间 | 账号创建时间，精确到秒（**新增**） |
| 最近登录 | 上一次成功登录的时间（**新增**） |

> 💡 **注意事项**：如果"姓名"字段显示为脱敏手机号格式（如 `138****6399`），代表系统中你的真实姓名暂未维护，可联系平台客服补全。

## 四、功能 2：H5 店铺信息编辑（仅老板可编辑）

### 4.1 老板操作步骤（H5）

1. 用老板账号登录 H5（手机浏览器打开上方"H5 移动端 - 我的"链接）
2. 在"我的"页面找到新增的「🏪 店铺信息」入口，点击进入
3. 进入后默认是**只读视图**，右上角能看到一个蓝色的「编辑」按钮
4. 点击「编辑」→ 字段切换为可编辑态，底部弹出「取消 / 保存」按钮
5. 修改你想调整的字段，比如：
   - **店铺名称**（2~30 字符，必填）
   - **店铺 Logo URL**（图片链接，≤ 500 字符）
   - **店铺简介**（≤ 200 字符）
   - **店铺地址**（必填）
   - **联系电话**（11 位手机号或固话）
   - **营业时间**（如 `09:00 - 21:00`）
6. 点击「保存」→ 底部出现"保存成功"提示，字段切回只读态
7. PC 端刷新店铺信息页，**修改即时同步可见**

### 4.2 老板操作步骤（PC，可选）

PC 端店铺信息页右上角同样有「编辑」按钮，操作流程与 H5 一致，字段范围完全一致。

### 4.3 不可编辑的字段（敏感字段）

以下字段为只读，**老板和员工都改不了**，需要修改请联系平台客服：

- 营业执照号
- 法人姓名
- 商家 ID

### 4.4 员工角色（看不到"编辑"按钮）

1. 用员工账号（如 6366）进入 H5 / PC 店铺信息页
2. 你将**完全看不到"编辑"按钮**——这是设计如此，不是 bug
3. 页面顶部会有一行温馨提示："员工角色仅可查看店铺信息，如需修改请联系老板。"
4. 即便通过技术手段绕过前端按钮直接调用接口，**后端也会返回 403 拒绝**，不会有任何脏数据写入

## 五、常见问题

### Q1：为什么我的"姓名"还是显示成 `138****xxxx`？

A：说明你的账号在系统中确实未维护"真实姓名"。本次修复保证字段不再空白（用脱敏手机号兜底），但要看到真实姓名，需联系平台客服在用户表中补全 `nickname`/`real_name`。

### Q2：H5 店铺信息保存后，PC 端没看到？

A：请刷新一次 PC 端店铺信息页（Ctrl + F5 强制刷新）。本次实现已在保存后立即清除前端缓存，PC + H5 必然同步。如仍未同步，请联系技术支持。

### Q3：员工想改店铺联系电话，可以吗？

A：不可以。本次产品设计明确：**店铺信息只允许老板修改**，员工只能查看。如老板临时不在，可委托老板远程操作 H5 完成修改。

### Q4：老板修改店铺名称后，会不会影响订单、消息等关联数据？

A：不会。店铺 ID 不变，仅展示名称变更。所有历史订单、消息记录中的店铺名会**实时同步**为最新值。

### Q5：可不可以上传 Logo 图片？

A：本期支持填写 Logo 图片 URL（外链）。如果你需要上传本地图片再生成 URL，请使用 PC 端"门店设置"中的 Logo 上传组件，再把生成的 URL 复制到 H5 这里。本期暂不在 H5 直接上传文件，下期会补齐。

### Q6：什么是"最近登录"时间？

A：你上一次输入账号密码或验证码登录成功的时间。该时间会在每次登录时自动更新，便于商家自查账号是否被异常使用。

## 六、安全机制说明

| 安全维度 | 实现方式 |
|---|---|
| 接口鉴权 | 所有接口必须携带有效 JWT Token，未登录直接 401 |
| 角色越权拦截 | 编辑接口（PUT `/api/merchant/shop/info`）后端强制校验老板角色，员工调用直接 403，不依赖前端按钮 |
| 数据回写完整性 | 后端字段缺失时自动兜底（如姓名→脱敏手机号），杜绝任何空白 |
| 操作日志 | 每次店铺信息修改自动写入服务端日志，包含操作人、时间、修改字段、前后值 |
| 字段校验 | 联系电话校验合法手机号/固话格式；店铺名称限制 2~30 字符；店铺简介限制 200 字符 |
