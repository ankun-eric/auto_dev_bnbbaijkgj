## 功能简介

本次为 bini-health「守护人体系」健康档案新增 **血糖管理模块**，实现完整闭环：

```
录入 → 五档自动判定 → 高/低糖危象预警 → 守护人推送 → 趋势分析 → AI 个性化建议 → 报告导出
```

### 核心能力一览

| 能力 | 说明 |
|------|------|
| 📥 录入 | 4 种测量场景（空腹 / 餐后 2h / 随机 / 睡前）+ 0.1 mmol/L 精度 |
| 🚦 五档判定 | 严重偏低 / 偏低 / 正常 / 偏高 / 严重偏高，按场景自动套用阈值 |
| ⚠️ 危象预警 | 测量值 ≥ 16.7 触发高糖危象，< 2.8 触发低糖危象，强弹窗 + 守护人推送 |
| 👥 守护人协同 | 异常自动通知绑定的守护人，便于远程感知与干预 |
| 📈 趋势分析 | 支持 7 / 30 / 90 天 + 全场景或单场景筛选 |
| 🤖 AI 建议 | 基于近 30 天数据生成个性化饮食/运动建议（含"仅供参考"免责声明） |
| 📄 报告导出 | 一键生成血糖管理报告（含统计 + 趋势 + AI 建议），便于线下面诊 |
| ⏰ 餐后提醒 | 用户可设置餐后 2 小时自动提醒测量 |

## 使用说明

### 第 1 步：进入血糖管理

1. 在浏览器打开 [血糖管理首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/glucose)
2. 如果未登录，会自动跳转登录页，使用 bini-health 已注册账号登录
3. 页面顶部为 **Tab 切换栏**：首页 / 录入 / 历史 / 趋势 / 预警 / AI 建议 / 报告

### 第 2 步：录入血糖

1. 点击顶部 Tab「**录入**」或首页主按钮「**+ 录入血糖**」
2. 填写 3 个字段：
   - **测量值**：0.5 ~ 35.0 mmol/L（精确到 0.1）
   - **测量场景**：四选一（空腹 / 餐后 2h / 随机 / 睡前，系统会根据当前时间智能推荐）
   - **备注**（可选）：200 字内
3. 点击「**保存**」
4. 系统会**立即判定档位**并给出对应反馈：
   - ✅ 正常：返回首页，无任何弹窗
   - ⚠️ 偏低 / 偏高：Toast 轻提示
   - 🟠 严重偏高：守护人自动收到推送（站内信 + APP Push）
   - 🔴 **高糖危象（≥16.7）/ 低糖危象（<2.8）**：**强提示弹窗 + 守护人实时推送**，必须点击「我知道了」才能关闭

### 第 3 步：查看历史

1. 点击 Tab「**历史**」
2. 显示按时间倒序的所有记录，每条含：测量值、单位、档位标签、场景、时间、备注、危象徽章
3. 支持按场景筛选（全部 / 空腹 / 餐后 2h / 随机 / 睡前）
4. 长按或点击「删除」可移除记录（删除后会同步清除该记录关联的预警事件）

### 第 4 步：趋势统计

1. 点击 Tab「**趋势**」
2. 切换时间范围：**7 天 / 30 天 / 90 天**
3. 切换场景：全部 / 空腹 / 餐后 2h / 随机 / 睡前
4. 查看 3 个区块：
   - **关键指标**：平均、最高、最低、测量次数、异常次数、达标率
   - **趋势曲线**：每日均值折线，背景标注「正常区间 3.9–7.8」与参考线 7.8 / 11.1 / 16.7
   - **五档分布**：每个档位的次数与百分比柱状图

### 第 5 步：预警事件中心

1. 首页顶部如有未处理危象，会显示红色警示条「检测到 X 次血糖异常事件，点击查看」
2. 点击警示条或 Tab「**预警**」进入预警详情
3. 每条预警显示：类型（高糖危象/低糖危象/严重偏高/严重偏低）、数值、场景、时间、推送状态
4. 处理完毕后点击「**标记已处理**」，下次首页将不再红色提示

### 第 6 步：AI 个性化建议

1. 点击 Tab「**AI 建议**」
2. 选择分析周期：7 / 30 / 90 天
3. 系统会生成 3 块内容：
   - 📊 **概况**：平均空腹 / 平均餐后 / 异常事件统计
   - 📈 **趋势观察**：与上周对比、波动点提示
   - 💡 **个性化建议**：饮食、运动、复诊建议
4. 底部显著标注：**「此建议仅供参考，不能替代医生诊断」**

### 第 7 步：报告导出 + 餐后提醒

1. 点击 Tab「**报告**」
2. 选择数据范围（7 / 30 / 90 天），系统会生成报告预览
3. 点击「**下载 PDF**」或「**复制分享链接**」即可发给医生（临时链接 7 天有效）
4. 报告下方可配置「**餐后 2 小时提醒**」：
   - 勾选「开启提醒」
   - 设置早 / 午 / 晚餐时间（默认 07:00 / 12:00 / 18:30）
   - 点击「保存提醒设置」

> ℹ️ 餐后 2 小时提醒由浏览器/APP 本地定时器触发，需保持页面打开或安装 APP 端。

## 五档阈值速查（mmol/L）

> 不同场景使用不同阈值，但 **≥ 16.7 高糖危象** 与 **< 2.8 低糖危象** 跨场景统一触发。

| 档位 | 空腹 | 餐后 2h / 随机 / 睡前 | 行为 |
|------|------|------------------------|------|
| 严重偏低 | < 2.8 | < 2.8 | 危象：强弹窗 + 守护人推送 |
| 偏低 | 2.8 ~ 3.9 | 2.8 ~ 3.9 | 提示「血糖偏低，注意补充」 |
| 正常 | 3.9 ~ 6.1 | 3.9 ~ 7.8 | 正常显示 |
| 偏高 | 6.1 ~ 7.0 | 7.8 ~ 11.1 | 提示「血糖偏高，注意饮食」 |
| 严重偏高 | ≥ 7.0 | ≥ 11.1 | 守护人推送 |
| **高糖危象** | **≥ 16.7（任意场景）** | **≥ 16.7（任意场景）** | **强弹窗 + 守护人推送** |
| **低糖危象** | **< 2.8（任意场景）** | **< 2.8（任意场景）** | **强弹窗 + 守护人推送** |

## 注意事项

1. **录入合理范围**：仅接受 0.5 ~ 35.0 mmol/L 的数值，超出会拦截。
2. **危象强弹窗**：高糖危象 / 低糖危象触发后，**必须点击「我知道了」按钮才能关闭弹窗**，不提供右上角 × 关闭、不支持点击遮罩关闭。
3. **守护人推送**：异常自动推送的前提是被守护人已绑定守护人关系；如未绑定，预警会降级为站内信，不影响本人弹窗。
4. **AI 建议仅供参考**：当前 AI 建议基于规则引擎，建议复诊请遵医嘱。
5. **餐后提醒局限**：H5 端的提醒依赖浏览器本地定时器，关闭页面后无法提醒，建议安装 APP 端获得更稳定的后台提醒。
6. **数据隔离**：所有血糖数据按 user_id 隔离，只有本人和被授权的守护人可以查看。
7. **删除不可恢复**：历史记录删除后会同步清除该记录关联的预警事件，操作前请确认。
