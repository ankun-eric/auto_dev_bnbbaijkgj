## 功能简介

「居家安全设备」是 bini-health 平台中支撑老人居家安全监护硬件（华为等厂商）告警回调接入的核心模块。本次修复针对该模块中的两个管理后台 Tab：

1. **回调地址配置**：配置厂商回调地址、Token、上游推送地址，并能主动「推送给厂商」。
2. **回调原始记录**：审计厂商每一次回调的原始入参、解析状态、关联告警。

### 本次修复了什么

修复前打开这两个 Tab 时，右上角会出现红字「加载失败 / 保存失败」，相关接口返回 HTTP 500。修复后：

- ✅ 「回调地址配置」Tab — 进入后能正常加载已有配置并填充表单
- ✅ 「回调地址配置」Tab — 直接点保存能成功提交
- ✅ 「回调原始记录」Tab — 能正常加载列表（暂无回调时显示「暂无数据」，不再有红字）
- ✅ 厂商回调入口持续可用，新一次推送可在「回调原始记录」立即查到

## 操作步骤

### 步骤 1：登录管理后台

打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/) 并使用管理员账号登录。

### 步骤 2：进入「居家安全设备」

在左侧菜单点击「居家安全设备」，进入模块主页面：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-safety](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-safety)。

### 步骤 3：验证「回调地址配置」Tab

1. 切换到「回调地址配置」Tab
2. 预期：页面正常加载已有配置内容并填充表单字段，**不再出现红字「加载失败」**
3. 不修改任何字段，直接点击「保存」按钮
4. 预期：右上角弹出绿色提示「保存成功」，并能立即看到 `last_pushed_at` / `last_push_status` / `last_push_judge_basis` 等字段被服务端更新

### 步骤 4：验证「回调原始记录」Tab

1. 切换到「回调原始记录」Tab
2. 预期：页面正常加载列表
   - 若历史无回调记录：显示「暂无数据」（属于正常空态，**不再出现红字「加载失败」**）
   - 若有历史回调：按 `received_at` 降序展示，可翻页和按条件筛选
3. 点击列表任意一行：弹出 Drawer 详情，展示请求方法、URL、响应状态码、响应体、处理时间、设备 SN 等审计字段

### 步骤 5：（可选）请厂商发起真实回调

可联系厂商使用以下回调地址进行一次真实推送：

```
POST https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/api/home_safety/callback/alarm
```

预期：

- 厂商收到 HTTP 200 响应
- 在「回调原始记录」Tab 点击刷新，能立即看到该次推送的原始报文
- `processed_at` 字段会随业务完成时间被填充

## 注意事项

1. **修复无需重新发布前端**：本次修复仅作用于后端数据库表结构与自动迁移逻辑，**浏览器只需刷新页面即可看到效果**，无需重新登录或清缓存。

2. **本次修复零数据风险**：所有新增字段均为可空（NULL-able），不影响存量数据；如需回滚仅需 `ALTER DROP COLUMN`，但本次修复符合代码意图，无回滚必要。

3. **厂商回调期间未中断**：修复期间厂商可继续向回调入口 POST，原始报文不会丢失。修复完成后，运营即可在「回调原始记录」Tab 看到所有历史与新推送的报文。

4. **防御性自动迁移已生效**：本次随修复一并在 `backend/app/services/schema_sync.py` 中补上了 7 个字段的 ALTER 自动检测逻辑，后续重建后端容器时即使表已存在也能自动补齐缺失字段，避免同类问题再次发生。

5. **若仍出现红字**：请按 F12 打开浏览器开发者工具的 Network 面板，查看具体接口的响应码：
   - 401：未登录或登录过期，请重新登录
   - 403：当前账号无权限，请联系管理员
   - 500：可能其他问题，请联系技术支持并附上 Network 面板截图
