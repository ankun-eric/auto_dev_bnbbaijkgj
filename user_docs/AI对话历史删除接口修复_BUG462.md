## 功能简介

本次修复了 H5 端 AI 对话「历史对话」抽屉中**删除功能不可用**的严重 Bug：

- **修复前**：在 ☰ 抽屉的「历史对话」区内，无论是「单条删除」（⋯ 菜单 / 左滑红色按钮）还是「批量删除」（管理模式下勾选后点底部删除），点完确认后界面里条目会立刻消失，**但屏幕底部总会弹一个灰色提示「删除可能未同步，请稍后刷新」**；只要下拉刷新或者重新打开抽屉，**被删的对话会原封不动地全部"复活"**——实际上一次也没真正删掉。
- **修复后**：删除立即生效，刷新后被删的对话**不会再回来**。删除失败时（如断网）也不再让人困惑——界面会**自动把条目恢复到原位置**，并提示「**删除失败,请稍后重试**」，告诉用户清楚的下一步动作。

## 操作步骤

### 一、单条删除（⋯ 菜单方式）

1. 在 H5 端打开 AI 对话首页：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
2. 点击左上角 **☰** 按钮，打开历史对话抽屉
3. 在「历史对话」区找到要删除的某条对话，点击右侧的 **⋯** 按钮
4. 在弹出的菜单中点击「**删除**」
5. 在二次确认弹窗「**确认删除该对话？**」中点击「**确认**」
6. ✅ 该条对话立即从列表消失，屏幕底部出现「**已删除**」的成功提示
7. 重新打开抽屉或下拉刷新，**该条对话不再出现**

### 二、单条删除（左滑手势方式）

1. 在「历史对话」区找到要删除的某条对话
2. **在该条对话上从右向左滑动**，右侧会出现两个按钮：橙色「置顶/取消置顶」+ 红色「删除」
3. 点击红色「**删除**」按钮
4. 在二次确认弹窗中点击「**确认**」
5. ✅ 效果同上：列表消失 + 成功提示 + 刷新后仍不在

### 三、批量删除

1. 在「历史对话」区右上角点击「**管理**」按钮，进入管理模式
2. 此时每条对话左侧会出现圆形勾选框
3. 点击任意条目即可勾选/取消勾选，可同时选中多条
4. 全部选中时也可点击底部的「**全选**」
5. 选好后点击底部右侧的红色「**删除**」按钮
6. 在二次确认弹窗「**确认删除已选 N 条对话？**」中点击「**确认**」
7. ✅ 多条对话立即消失 + 出现「**已删除**」提示 + 自动退出管理模式 + 刷新后这些对话**不再出现**

### 四、删除失败时的体验（断网 / 后端异常）

如果删除请求因网络异常或后端临时不可用而失败：

- 列表中"消失了一下"的对话会**自动恢复到原来的位置和顺序**
- 屏幕底部出现红色「❌ **删除失败,请稍后重试**」提示
- **批量删除时**：管理模式不会退出、之前的勾选状态也会保留，用户可以原地点「删除」重试，无需重新勾选

## 注意事项

1. **删除是真删除**：本次修复后，删除操作会真正在数据库中将对话标记为已删除，刷新页面后不会复活。请确认后再点删除，目前没有「撤销」操作。
2. **二次确认按钮文案不变**：单条删除仍为「**确认删除该对话？**」，批量删除仍为「**确认删除已选 N 条对话？**」。
3. **置顶/取消置顶不受影响**：本次只修复删除逻辑，置顶逻辑没有任何改变。
4. **不影响其他操作**：发起新对话、读取历史对话、咨询人切换等流程均不受影响。
5. **若仍看到「删除可能未同步」**：强制刷新页面（Ctrl+F5 或 Cmd+Shift+R）以清除浏览器旧缓存的 JS，确保加载到最新的前端代码。

## 修复说明（技术备注）

> 给关心实现细节的用户参考，普通用户可跳过本节。

- **根因**：前端旧代码调用了一个**不存在**的接口 `POST /api/chat/history/delete`，每次都返回 404，进入异常分支弹出"删除可能未同步"的兜底提示；但前端已经做了"乐观更新"先把条目从界面移除，所以用户会以为删除成功了。
- **修复**：将前端 `Sidebar.tsx` 中的删除接口调用改为后端真实存在的两个接口：
  - 单条：`DELETE /api/chat-sessions/{id}`
  - 批量：`POST /api/chat-sessions/batch-delete`，请求体 `{ session_ids: [...] }`
- **失败容错**：在发请求前先保存 `histories` 列表的完整快照；接口失败时一键 `setHistories(快照)` 恢复列表，并显示"删除失败,请稍后重试"的友好提示。
- **变更范围**：仅修改 H5 端单文件 `h5-web/src/components/ai-chat/Sidebar.tsx`，后端无任何改动。
