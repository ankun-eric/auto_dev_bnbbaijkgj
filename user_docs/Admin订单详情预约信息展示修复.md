## 功能简介

本次修复了 Admin 管理后台「商品体系 → 订单明细」模块中**订单详情预约信息不显示**的 Bug。

修复前，管理员在后台打开含有预约信息的订单详情时，预约信息卡片始终不显示。修复后，预约信息卡片能够正确展示以下内容：

- **关联门店**：预约所属门店名称
- **预约日期**：格式化为「YYYY年MM月DD日」
- **预约时段**：如「10:00-11:00」
- **预约状态**：待支付 / 待核销 / 已完成 / 已取消

## 使用说明

### 查看订单预约信息

1. **登录管理后台**：打开 [Admin 管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)，使用管理员账号登录

2. **进入订单明细页面**：在左侧导航栏中，点击「商品体系」→「订单明细」

3. **打开订单详情**：在订单列表中，找到一条包含预约信息的订单（通常为到店服务类订单），点击该行右侧的「详情」按钮

4. **查看预约信息卡片**：在打开的详情抽屉中，基本信息下方会展示「预约信息」卡片，内容包括：
   - **关联门店**：显示预约关联的门店名称（占满整行）
   - **预约日期**：显示预约的日期，如「2026年4月30日」
   - **预约时段**：显示具体的时间段，如「10:00-11:00」
   - **预约状态**：显示当前状态标签（蓝色-待支付/待核销、绿色-已完成、红色-已取消）

## 注意事项

1. **仅含预约信息的订单显示预约卡片**：如果订单不包含预约信息（如纯快递订单），详情中不会显示预约信息卡片，这是正常的

2. **关联门店按需显示**：只有订单绑定了门店时才会显示「关联门店」字段；未绑定门店的订单不显示该字段

3. **预约状态说明**：
   - 待支付：用户尚未完成支付
   - 待核销：用户已支付，等待到店核销
   - 已完成：订单已核销完成
   - 已取消：订单已被取消

4. **数据来源**：预约信息来自用户下单时填写的预约数据，后端 API 已正确返回，本次修复仅涉及管理后台前端展示
