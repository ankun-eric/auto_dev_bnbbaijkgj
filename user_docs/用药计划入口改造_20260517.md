## 一、功能简介

本次"用药计划入口改造"对 bini-health 体系中的 **用药计划** 模块做了入口与职责的系统重组，目标是 **职责清晰、路径最短、数据真实、代码精简**：

1. **健康档案 Hero "在用药品" 格子升级为高频主入口** —— 文案智能切换为：
   - 今日有待服用药 → **"还有 X 次用药"**
   - 今日已全部打卡 → **"今日用药已完成"**
   - 今日无任何用药安排 → **"今日用药 0"**
   - 点击直接进入 **用药提醒打卡页面**，一步直达
2. **新增用药提醒打卡页 `/ai-home/medication-reminder`** —— 蓝色顶部 Banner 展示今日还有几次、下一次提醒；橙色「即将服用」高亮区块直接点"立即服用"打卡；时间线按时间排列显示全部今日用药；右下角浮动 `+` 直接新增；误打卡 5 分钟内长按"✓ 完成"可撤销。
3. **健康档案摘要卡降级为次入口** —— 顶部右上角并排 `[全部]` `[+ 新增]` 双按钮；只显示"服药中"的药品；单条不可点（纯展示）；卡片内可上下滚动（最大高度 480px / 60vh）；无任何用药时只显示居中的 `[+ 添加第一条用药计划]` 大按钮。
4. **用药计划列表页 `/ai-home/medication-plans`** —— 三 Tab 「服药中 / 未开始 / 已结束」，每个 Tab 带数字徽章 `(X)`，默认进入「服药中」，点击单条卡片进入详情/编辑页。
5. **新增表单支持过去日期补录** —— 服药开始日期默认今天，但允许选择任意过去日期（最远 1 年前），用于代录历史用药；选过去日期时下方显示浅灰提示。
6. **彻底删除旧路由** —— `/medication-plans`、`/health-plan/medications`、`/health-plan/medications/add` 全部返回 404；所有跳转统一收敛到 `/ai-home/medication-*`。

## 二、使用说明

### 2.1 通过 Hero 入口（最高频路径）

1. 打开 [健康档案](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile)。
2. 找到顶部蓝色 Hero 卡的 **第 4 格"在用药品"** —— 上方数字若大于 0，下方文案会显示「还有 X 次用药」（X 为今日待打卡数）。
3. **点击该格** —— 立即跳转到「用药提醒页」。

### 2.2 用药提醒页：打卡

进入用药提醒页后，依次会看到：

1. **顶部蓝色 Banner**：今日日期、还有几次大字、下一次提醒时间 + 药品名、三统计（已服用 / 待服用 / 本月依从率）。
2. **即将服用区块**（橙色高亮 + "即将"徽章）：仅显示最近一条待打卡药品，含药名、剂量、服用时机、计划时间、距现在的倒计时。点击「立即服用」按钮 → 立即完成打卡。
3. **今日用药时间线**：按时间点排序，每条都标注状态：
   - 🟢 已服用（绿点 + "✓ 完成"，浅色不可再点）
   - 🟠 即将服用（橙点 + "立即服用"，蓝色按钮可点）
   - 🔵 未到时间（蓝点 + "未到时间"，浅色不可点）
4. **右下角浮动 ⊕**：点击直接跳到新增表单页。
5. **顶部 ←**：返回健康档案，自动定位到「用药计划」区域。

**撤销误打卡**：刚刚点完"立即服用"想反悔？**5 分钟内**长按那条已变成"✓ 完成"的按钮即可撤销（超过 5 分钟则提示「REVOKE_TIMEOUT」无法撤销）。

### 2.3 健康档案摘要卡：管理"在用药品"

回到 [健康档案](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile)，往下滑动找到「用药计划」摘要卡：

- **有用药时**：卡片顶部右上角并排 `[全部]` 和 `[+ 新增]` 两个按钮。
  - 点 `[+ 新增]` → 跳新增表单页。
  - 点 `[全部]` → 跳列表页，默认停留在「服药中」Tab。
  - 卡片正文列出全部"服药中"药品（按创建时间倒序），每条展示：药名 + 状态徽章「服用中」+ 剂量 + 频次 + 服用时机。多条时可上下滑动浏览。
  - **单条用药条目不可点击**（纯展示）—— 要修改请通过 `[全部]` 进列表页操作。
- **无任何用药时**：卡片只显示居中大按钮 `[+ 添加第一条用药计划]`，无其他元素。

### 2.4 用药计划列表页

进入 [用药计划列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home/medication-plans)：

- 顶部三 Tab 切换：**服药中(X) / 未开始(X) / 已结束(X)**，每个 Tab 后括号显示数量。默认进入「服药中」。
- 每个 Tab 内按 **创建时间倒序** 排列。
- 列表项卡片显示：药名（加粗）+ 状态徽章 + 剂量·频次·服用时机 + 起止日期（如 `2025-05-01 ~ 2025-08-01`）。
- **点击整张卡片** → 进入用药详情/编辑页。
- **右下角浮动 ⊕** → 跳新增表单页。
- **顶部 ←** → 返回健康档案。

### 2.5 新增 / 编辑用药计划表单

字段一览：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 药品名称 | 必填 | 1~50 字 |
| 剂量 | 必填 | 数值 + 单位（如 5mg、0.5g、1 片） |
| 用药频次 | 必填 | 每日 1~4 次 / 隔日 1 次 / 按需 |
| 服用时机 | 必填 | 多选：饭前 / 饭后 / 随餐 / 睡前 / 晨起 |
| 服药开始日期 | 必填 | **默认今天；允许选过去（最远 1 年），上限今天 + 1 年** |
| 服药截止日期 | 必填 | 必须 ≥ 开始日期 |
| 备注 | 选填 | ≤ 200 字 |

**关于过去日期的特别说明**：

> 当选择早于今天的开始日期时，日期框下方会显示浅灰色提示：
>
> > "开始日期早于今天，将自动按已过去的天数计算服药记录"
>
> 不会弹窗、不会阻止提交。这个能力支持**补录历史用药、共管者代录、处方导入**等场景。

**提交后**：返回列表页，自动切到对应 Tab（服药中 / 未开始 / 已结束），并把刚刚新增的记录高亮置顶。

## 三、注意事项

1. **历史日期录入不会追溯打卡数据**：补录"开始日期 = 上周一"的药品，不会自动补全上周一到今天的"已打卡 / 未打卡"记录。本月依从率从录入当天开始统计，避免历史空白拉低数据。
2. **撤销打卡的时间窗口**：误点"立即服用"后，**5 分钟内**可长按"✓ 完成"撤销；超过 5 分钟接口会返回 `REVOKE_TIMEOUT` 错误码，无法撤销。
3. **状态自动流转**：系统每日凌晨 00:01 自动扫描所有计划，按"今天 vs 起止日期"自动更新 `未开始 → 服药中`、`服药中 → 已结束`。此外，每次打开用药提醒页和 Hero 计数接口时也会"懒触发"一次扫描，确保你看到的状态总是最新的。
4. **同一药品同时只能有一个"服药中"**：新增时如撞名同名"服药中"计划，系统会返回 409 并弹窗 "您已有一条进行中的【XX】用药计划，是否要去查看/修改？"（沿用上一版规则）。
5. **旧路由已下线**：访问 `/medication-plans`、`/health-plan/medications`、`/health-plan/medications/add` 会直接返回 404。请把所有书签和分享链接更新为新路径（`/ai-home/medication-*`）。
6. **数据隔离**：当前选定的"咨询人"（家庭成员）会贯穿到全部用药相关接口，本人和家人各自独立的用药档案不会串。
7. **移动端优先**：本次入口改造完全基于手机浏览器和微信内置浏览器的体验设计，PC 端可正常访问但页面宽度按手机视口适配。

## 五、常见问题（FAQ）

**Q1：为什么"在用药品"格子的数字会变？**
A：数字 = 今日还有几次"待打卡"。每打卡一次减 1，撤销打卡时加 1。全部打完显示"今日用药已完成"，没有任何用药计划则显示"今日用药 0"。

**Q2：摘要卡上的药能直接点开编辑吗？**
A：不能。摘要卡只做"在管理"展示。要编辑请点顶部 `[全部]` 按钮，进入列表页后再点击对应药品卡片。

**Q3：列表页 Tab 数字是什么时候更新？**
A：每次切换 Tab 或下拉刷新时实时拉取一次后端 count。新增/编辑/停药后回到列表页也会自动刷新。

**Q4：旧的 `/medication-plans` 收藏夹链接还能用吗？**
A：不能。该旧路由的页面文件已彻底删除，访问会 404。请把收藏夹更新为新链接 `/ai-home/medication-plans`。
