## 功能简介

本次集中修复了**积分商城**模块的两枚 P1 级缺陷，全平台一次性发版：

### BUG-A：兑换记录的「查看我的券」按钮跳转 404
- 用户进入「积分中心 → 积分明细 → 兑换记录」点击优惠券类记录的「查看我的券」按钮，原本会跳到一个不存在的 `/coupons` 路径（404 报错），现已修复为跳转到「我的优惠券」页面，并自动定位到「可用」Tab，目标券会高亮闪烁 1.5 秒。
- 同时把单按钮拆分为「查看券」+「去使用」两个按钮：可用券会额外显示「去使用」直达适用服务列表。

### BUG-B：积分商城「可兑换」Tab 过滤无效
- 之前积分商城的「可兑换」Tab 与「全部」Tab 内容几乎一样（漏过滤），用户经常看到"显示能兑换但下单提示已下架/已限购"。
- 本次修复后，「可兑换」Tab 严格按 5 个条件同时满足才展示：
  1. 用户积分 ≥ 商品所需积分
  2. 商品库存 > 0
  3. 商品在「上架中」状态
  4. 商品在有效期内
  5. 用户尚未达到该商品的限购上限
- Flutter 安卓 / iOS 客户端原本没有 Tab 切换 UI，本次也已新增「全部 / 可兑换」双 Tab，与 H5 / 小程序行为完全对齐。

## 使用说明

### 场景 1：从兑换记录里查看 / 使用某张优惠券

1. 进入「积分中心」（顶部导航的"积分"图标，或个人中心的"积分"入口）
2. 点击页面上方的「兑换记录」Tab
3. 在列表中找到优惠券类型的记录（图标为优惠券、金额或折扣信息），底部会看到两个按钮：
   - 【查看券】（橙色描边）—— **始终显示**
   - 【去使用】（绿青渐变实心）—— **仅当券处于"可用"状态时才显示**
4. **点击「查看券」**：自动跳转到「我的优惠券」页面，停在「可用」Tab，目标券卡片会自动滚动到可视区域并以淡黄色描边闪烁 1.5 秒，方便快速识别
5. **点击「去使用」**：自动跳转到该券适用的服务列表，顶部会有横幅提示「当前已选优惠券：XXX」，挑选服务下单时该券会自动勾选

### 场景 2：在积分商城里只看自己真正能兑换的商品

1. 进入「积分商城」（积分中心点击"积分商城"，或主导航对应入口）
2. 默认停留在「全部」Tab，列出所有商品（含库存不足、已下架、达限购等）
3. **切换到「可兑换」Tab**，列表会严格过滤为同时满足 5 个条件的商品：
   - 你的积分够换
   - 商品有库存
   - 商品处于上架中
   - 商品在有效期内
   - 你还没达到该商品的限购上限
4. 此时点进任一商品并兑换，**不会再出现"积分不足/已下架/已限购"等失败提示**
5. 完成兑换后，可在「积分中心 → 兑换记录」查看明细，再走场景 1 即可使用券

### 三端共同验证点（自查清单）

- ✅ 不会再出现 `/coupons` 404 报错页面
- ✅ 「兑换记录 → 优惠券记录」每条至少能看到「查看券」按钮
- ✅ 状态为可用的券额外有「去使用」按钮，已使用 / 已过期的券只剩「查看券」
- ✅ 「可兑换」Tab 不会再展示库存=0 / 已下架 / 已过期 / 已限购的商品
- ✅ Flutter 安卓 / iOS APP 在积分商城页可以看到顶部的「全部 / 可兑换」Tab 切换条

## 注意事项

1. **后端 / H5 已自动更新**：服务端已部署最新代码，浏览器端请**强制刷新一次**（手机浏览器一般是下拉刷新或清除缓存）即可看到修复后的行为。
2. **APP 必须升级**：安卓 / iOS 客户端老版本仍会跳到 404 或看不到 Tab，请务必下载本次新包。
3. **小程序需重新导入**：用微信开发者工具体验时，需把上面新的 zip 包覆盖原有目录，再编译预览。
4. **「可兑换」Tab 严格过滤**：本次修复后，如果你看到「可兑换」Tab 是空的，说明确实当前你的积分余额、库存或限购等条件下没有商品可兑——这是正确行为，不是 Bug。
5. **「去使用」按钮可能不显示**：仅当券状态为"可用（available）"时才显示「去使用」；已使用、已过期、已退款的券只会显示「查看券」。
6. **跨端数据一致**：同一账号在 H5、小程序、APP 上看到的"可兑换"商品列表与"兑换记录"内容会保持一致，5 条件判定结果在三端 + 后端口径完全相同。
