## 一、功能简介

居家安全设备模块包含 **紧急呼叫器 / 烟雾报警器 / 水位报警器** 三类智能硬件，本次更新围绕设备绑定与告警触达做了如下优化：

### 1.1 网关 SN 改名为「网关ID」（8 位）

- 原网关 SN 为 12 位字母+数字混合，前 4 位明文 + 后 8 位 `*` 掩码，用户手工输入门槛较高且容易出错；
- 新「网关ID」改为 **8 位** `[A-Z0-9]`，**大小写不敏感**（统一存大写），8 位明文展示，不再做掩码；
- H5 与管理后台输入框均会 **实时把小写自动转大写 + 自动过滤空格 / 汉字 / 符号等非法字符**；
- 旧 12 位数据会按"前 8 位"自动截断，撞号时仅保留最近一条记录，其余记录被标记为「失效需重绑」。

### 1.2 设备级紧急联系手机（必填）

- 在 **每条** 设备绑定记录上新增 **紧急联系手机** 字段，独立保存，与"账号级守护人"体系**并存**；
- 绑定页默认 **自动带入您的注册手机号**，您也可以手工修改为别的号码（例如自家家里电话）；
- 必填，校验中国大陆 11 位手机号格式 `^1[3-9]\d{9}$`；
- 历史绑定记录（升级前已经存在）的紧急联系手机为空，您再次进入「居家安全设备」页面时，系统会强制弹窗提示您补填，未补填时该号码不参与告警通知。

### 1.3 告警触达增强

收到设备告警时，通知目标号码列表会自动合并以下三类，并 **去重后发送**：

```
本人注册手机号 + 该设备绑定记录的紧急联系手机 + 已启用对应类型的守护人手机号
```

- 站内 / 小程序消息：仅发本人 + 守护人（设备级紧急联系手机不发）；
- **短信 / AI 外呼**：本人 + 设备紧急联系手机 + 守护人，**同一号码自动去重，只发一条**；
- AI 外呼通道本期仍为占位状态（`failed / 本期未对接外呼通道`），通道上线后业务无需改动即自动生效；
- 短信和 AI 外呼并行触发，互不依赖。

## 二、操作步骤（H5 用户端）

### 2.1 绑定一台新设备

1. 打开 H5 首页 → 点击「居家安全设备」入口
2. 在顶部三个 Tab 中选择您要绑定的设备类型（紧急呼叫器 / 烟雾报警器 / 水位报警器）
3. 点击「+ 绑定新设备」按钮，弹出绑定窗口
4. 依次填写：
   - **网关ID（8 位）**：直接手工输入 8 位字母或数字，例如 `ABCD1234`；输入框会自动将小写转大写、过滤非法字符；
   - **设备 SN（8 位）**：在设备机身上的 8 位字母+数字编码；
   - **紧急联系手机（11 位）**：弹窗打开时已自动带入您的注册手机号；如需修改可直接编辑。
5. 点击「确认绑定」即可

> ⚠️ 三项均为必填。如填写格式不正确，系统会用文字提示具体错误：
> - `网关ID 必须为 8 位字母或数字`
> - `设备 SN 必须为 8 位字母+数字`
> - `请输入有效的 11 位手机号`

### 2.2 查看 / 修改紧急联系手机

1. 进入「居家安全设备」页面，在「我绑定的设备」区域，每张设备卡片底部都会显示已脱敏的紧急联系手机，例如 `138****1234`；
2. 点击右侧的「修改」按钮，弹窗中输入新号码，点击「保存」即可；
3. 保存生效后立即在前端刷新展示，**无需短信验证码**。

### 2.3 补填历史设备的紧急联系手机

- 如果您升级前已经绑定的设备没有紧急联系手机，进入页面时会自动弹出强制补填弹窗：
  > 为了保障告警可及时触达，请先补填本设备的紧急联系手机。
- 弹窗中输入正确格式的 11 位手机号，点击「立即补填」；
- 如果点「取消」则会返回上一页，下次再进入页面仍会弹出，直到补填完成为止。

### 2.4 撞号失效设备的重新绑定

- 升级时若发现历史数据中多条绑定截断为同一个 8 位网关ID，会**仅保留最新一条作为有效**，其余条目以**灰色置顶卡片**展示，标注「失效需重绑」；
- 点击卡片上的「重新绑定」按钮可直接打开绑定弹窗进行重新录入。

## 三、操作步骤（管理后台）

### 3.1 「设备绑定流水」列名与列内容

- 原「网关 SN」列已改名为「**网关ID**」，列内 8 位明文（不再做掩码）；
- 状态列新增「失效需重绑」标签，对应历史撞号失效的记录；
- 列表右上角新增三项操作入口：
  1. 「按网关ID 搜索」输入框（8 位，输入自动转大写）
  2. 「显示紧急联系手机」按钮（默认隐藏，可勾选切换显示）
  3. 「导出 CSV」按钮，导出当前列表（含网关ID 与设备紧急联系手机两个新列）

### 3.2 「报警记录流水」列名

- 报警记录流水的「网关 SN」列同步改名为「网关ID」。

### 3.3 「回调地址配置」与「回调原始记录」Tab

- 本次未改动这两个 Tab 的展示逻辑；
- 外部回调接口 `gwId` 长度从 12 改为 8。**若上游对接方还在发送 12 位 gwId**，系统会**自动截断前 8 位**用于匹配；
- 若匹配到的记录已被标记为「失效需重绑」，回调接口会返回 `code=410 / message=device_rebind_required`，提示对接方该绑定需要重新激活。

## 四、外部设备回调接口说明（开发对接）

- 回调地址不变：`POST /api/home_safety/callback/alarm`
- 报文字段 `gwId` 名称保持兼容，**长度由 12 收敛为 8**；
- 兼容期内若 `gwId` 长度为 12 时，系统自动取前 8 位匹配；超过 8 位的部分会被丢弃；
- 若设备绑定记录为「失效需重绑」状态，HTTP 状态码返回 `410`、`code=410`、`message=device_rebind_required`，提醒对接方该设备需要用户重新绑定后才能恢复推送。

## 五、注意事项

1. **网关ID 是大小写不敏感的**：您可以用大小写任意组合输入，但后端统一存储为大写；
2. **紧急联系手机不发站内消息和小程序服务消息**：仅发送短信 + AI 外呼。如果您希望使用站内消息/小程序消息接收告警，请把该联系人加入「守护人体系」；
3. **同一号码自动去重**：当本人、紧急联系手机、守护人三方号码有重复时，同一号码同一告警事件只会收到一条短信；
4. **AI 外呼仍为占位状态**：本期 AI 外呼仍标记为 `failed / 本期未对接外呼通道`，待通道上线后无需任何改动自动生效；
5. **历史绑定记录的紧急联系手机为 NULL** 时，请尽快补填，否则该号码不会参与告警通知。

## 六、本次后端变更摘要（供运维参考）

| 改动点 | 详情 |
|---|---|
| `home_safety_device_binding.emergency_phone` | 新增 VARCHAR(11)，存储设备级紧急联系手机 |
| `home_safety_device_binding.invalid_reason` | 新增 VARCHAR(128)，记录撞号失效原因 |
| `home_safety_device_binding.status=2` | 新增状态值：失效需重绑（仅限历史撞号迁移） |
| `home_safety_device_binding.gateway_sn` | 长度收紧为 VARCHAR(16)（语义为 8 位大写） |
| `home_safety_alarm.device_emergency_phone` | 新增 VARCHAR(11)，告警快照 |
| `home_safety_alarm.notify_targets_json` | 新增 TEXT，去重后的最终通知目标号码 JSON |
| `home_safety_alarm.notify_dedup_skipped` | 新增 INT，记录被去重跳过的条目数 |
| `_sync_home_safety_v2()` | 自动迁移：截断、撞号判定、status 标记，幂等且零业务停机 |

后端新增 4 个 API 端点：

- `GET  /api/home_safety/devices/bind/defaults` — 绑定页默认值（注册手机号 + pattern）
- `GET  /api/home_safety/devices/{id}` — 设备详情
- `PATCH /api/home_safety/devices/{id}/emergency_phone` — 修改设备紧急联系手机
- `GET  /api/admin/home_safety/bindings/search_by_gateway` — 按网关ID 搜索
- `GET  /api/admin/home_safety/bindings/export` — 导出 CSV（含网关ID 与紧急联系手机）
