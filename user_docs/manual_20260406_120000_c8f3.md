## 功能简介

本次更新对系统的「推送管理」模块进行了全面拆分与增强，将原来集中在系统设置中的推送管理功能拆分为三个独立的管理模块，并新增了多项实用功能：

1. **短信管理**：新增阿里云短信服务商支持（原仅腾讯云），新增短信模板管理功能（CRUD），新增发送记录查询与服务商筛选，新增测试发送功能
2. **微信推送管理**：从系统设置中独立出来，成为独立页面，支持微信推送开关、AppID/AppSecret 配置、模板 ID 配置
3. **邮件通知管理**：从系统设置中独立出来，新增邮件发送记录查询，新增测试发送功能，支持 SMTP 配置管理
4. **系统设置页面**：移除「推送管理」标签页，仅保留基础配置、注册设置、协议管理
5. **侧边栏菜单**：「系统管理」下新增「短信管理」「微信推送管理」「邮件通知管理」三个独立菜单项

## 使用说明

### 一、短信管理

#### 入口路径

管理后台 → 左侧菜单「系统管理」 → 「短信管理」

#### 1. 短信配置

进入短信管理页面后，默认显示「短信配置」标签页。

**腾讯云短信配置：**
1. 在「腾讯云短信配置」卡片中，通过右上角的开关控制是否启用腾讯云作为短信服务商
2. 填写 SecretID、SecretKey、SDK AppID、短信签名等信息
3. 点击「保存腾讯云配置」按钮保存

**阿里云短信配置：**
1. 在「阿里云短信配置」卡片中，通过右上角的开关控制是否启用阿里云作为短信服务商
2. 填写 AccessKey ID、AccessKey Secret、短信签名等信息
3. 点击「保存阿里云配置」按钮保存

> ⚠️ 注意：腾讯云与阿里云同一时间只能启用一个。如果启用一个时另一个已启用，系统会弹窗确认是否切换。密钥类字段（SecretKey、AccessKey Secret）保存后不会明文回显，如需修改可重新输入覆盖。

#### 2. 短信模板管理

切换到「短信模板」标签页。

1. 点击「新增模板」按钮，在弹窗中填写模板名称、服务商、模板 ID、模板内容、签名名称、用途场景、变量说明等信息
2. 可通过顶部筛选栏按服务商、用途场景筛选，或按模板名称搜索
3. 列表支持分页浏览，每条模板可点击「编辑」修改或「删除」移除

#### 3. 发送记录

切换到「发送记录」标签页。

1. 查看所有短信发送记录，包括发送时间、手机号（已脱敏）、验证码、服务商、发送状态、失败原因
2. 支持按手机号搜索、按服务商筛选、按发送状态筛选
3. 测试发送的记录会显示「测试」标签

#### 4. 测试发送

切换到「测试发送」标签页。

1. 选择服务商（腾讯云或阿里云）
2. 输入接收手机号
3. 可选填模板 ID（留空则使用配置中的默认模板）
4. 点击「发送测试短信」按钮
5. 发送结果将在下方显示成功或失败提示

### 二、微信推送管理

#### 入口路径

管理后台 → 左侧菜单「系统管理」 → 「微信推送管理」

#### 配置步骤

1. 通过「启用微信推送」开关控制是否启用微信推送功能
2. 填写微信 AppID
3. 填写微信 AppSecret（已设置时显示占位提示，重新输入将覆盖）
4. 填写订单通知模板 ID 和服务通知模板 ID
5. 点击「保存配置」按钮保存

---

### 三、邮件通知管理

#### 入口路径

管理后台 → 左侧菜单「系统管理」 → 「邮件通知管理」

#### 1. 邮件配置

进入邮件通知管理页面后，默认显示「邮件配置」标签页。

1. 通过「启用邮件通知」开关控制是否启用邮件通知功能
2. 填写 SMTP 服务器地址（如 smtp.qq.com）
3. 填写 SMTP 端口（默认 465）
4. 填写 SMTP 用户名（通常为发件人邮箱）
5. 填写 SMTP 密码（已设置时显示占位提示，重新输入将覆盖）
6. 点击「保存配置」按钮保存

#### 2. 发送记录

切换到「发送记录」标签页。

1. 查看所有邮件发送记录，包括发送时间、收件人、邮件主题、发送状态、失败原因
2. 支持按收件人邮箱搜索、按发送状态筛选
3. 测试邮件会显示「测试」标签

#### 3. 测试发送

切换到「测试发送」标签页。

1. 输入收件人邮箱
2. 输入邮件主题（默认为「测试邮件」）
3. 输入邮件内容（可选，默认已填入测试内容）
4. 点击「发送测试邮件」按钮
5. 发送结果将在下方显示成功或失败提示

> ⚠️ 请确保先在「邮件配置」中完成 SMTP 配置后再进行测试发送。

## 注意事项

1. **权限要求**：以上所有消息管理功能仅管理员可操作，普通用户无权限访问
2. **短信服务商互斥**：腾讯云和阿里云短信服务商同一时间只能启用一个，启用其中一个将自动禁用另一个
3. **密钥安全**：所有密钥类字段（SecretKey、AccessKey Secret、SMTP 密码、AppSecret）保存后均不会明文回显，仅显示「已设置」状态
4. **手机号脱敏**：短信发送记录中的手机号会进行脱敏处理（如 138****0000），保护用户隐私
5. **数据迁移**：原系统设置中的推送管理配置数据已自动迁移到新的独立模块中，无需手动重新配置
6. **测试发送**：测试发送功能会真实调用对应的服务接口发送消息，请注意可能产生费用
