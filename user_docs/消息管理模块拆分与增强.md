## 功能简介

本次更新对系统的消息管理功能进行了模块拆分与增强，主要变更如下：

1. **短信管理增强**：新增阿里云短信服务商支持（原仅支持腾讯云），新增短信模板管理功能，发送记录和测试发送增加服务商维度
2. **微信推送管理独立**：将微信推送配置从「系统设置 → 推送管理」中拆出，成为独立的管理页面
3. **邮件通知管理独立**：将邮件通知配置从「系统设置 → 推送管理」中拆出，并新增邮件发送记录和测试发送功能
4. **删除推送管理**：原「系统设置 → 推送管理」标签页已删除，三个模块均已独立

## 使用说明

### 一、短信管理

**入口路径**：侧边栏 → 系统管理 → 短信管理

短信管理页面包含 4 个标签页：

#### 1. 短信配置

页面上下排列展示两个服务商配置区块：

- **腾讯云短信配置**
  - 启用/禁用开关
  - 填写 SecretID、SecretKey、SDK AppID、短信签名
  - 点击「保存腾讯云配置」按钮保存

- **阿里云短信配置**
  - 启用/禁用开关
  - 填写 AccessKey ID、AccessKey Secret、短信签名
  - 点击「保存阿里云配置」按钮保存

> ⚠️ 同一时间只能启用一家服务商。启用一家时，系统会提示"切换服务商将禁用当前已启用的服务商"，确认后另一家自动禁用。

#### 2. 短信模板

管理系统中使用的短信模板：

- **查看模板列表**：展示所有短信模板，包含模板名称、服务商、模板ID、模板内容、签名、场景、状态等信息
- **筛选**：支持按服务商（腾讯云/阿里云）和场景（验证码/通知/营销/其他）筛选
- **搜索**：支持按模板名称搜索
- **新增模板**：点击「新增模板」按钮，在弹窗中填写模板信息
- **编辑模板**：点击列表中的「编辑」按钮修改模板信息
- **删除模板**：点击「删除」按钮，确认后删除模板

#### 3. 发送记录

展示系统所有短信发送记录：

- 新增「服务商」列，显示该条短信使用的服务商（腾讯云/阿里云）
- 支持按手机号搜索、按发送状态筛选、按服务商筛选

#### 4. 测试发送

用于测试短信配置连通性：

- **选择服务商**：下拉选择使用腾讯云或阿里云发送测试短信（可在服务商未启用时测试）
- 输入接收手机号
- 可选填模板ID
- 点击「发送测试短信」查看发送结果

### 二、微信推送管理

**入口路径**：侧边栏 → 系统管理 → 微信推送管理

单页面配置：

1. 打开「启用微信推送」开关
2. 填写微信 AppID
3. 填写微信 AppSecret
4. 可选填写订单通知模板ID、服务通知模板ID
5. 点击「保存配置」按钮

---

### 三、邮件通知管理

**入口路径**：侧边栏 → 系统管理 → 邮件通知管理

邮件通知管理包含 3 个标签页：

#### 1. 邮件配置

配置 SMTP 邮件服务器：

1. 打开「启用邮件通知」开关
2. 填写 SMTP 服务器地址（如 smtp.qq.com）
3. 填写 SMTP 端口（如 465）
4. 填写 SMTP 用户名（发件邮箱地址）
5. 填写 SMTP 密码（邮箱授权码）
6. 点击「保存配置」按钮

#### 2. 发送记录

查看系统发送的所有邮件记录：

- 表格列：发送时间、收件人、邮件主题、发送状态、失败原因、是否测试邮件
- 支持按收件人邮箱搜索、按发送状态筛选

#### 3. 测试发送

验证邮件配置是否正确：

1. 输入收件人邮箱
2. 输入邮件主题（默认"测试邮件"）
3. 输入邮件内容（可选，默认提供示例内容）
4. 点击「发送测试邮件」按钮
5. 查看发送结果反馈

> ⚠️ 发送前请先完成邮件配置，未配置时会提示"请先完成邮件服务器配置"。

### 四、系统设置变更

原「系统设置」页面的「推送管理」标签页已删除。短信、微信推送、邮件通知的配置已分别迁移到独立的管理页面。

当前系统设置页面仅保留：基础配置、注册设置、协议管理。

## 注意事项

1. **权限要求**：以上所有功能仅管理员（admin）角色可访问
2. **服务商互斥**：短信服务同一时间只能启用腾讯云或阿里云中的一家
3. **密钥安全**：所有密钥类字段（SecretKey、AccessKey Secret、AppSecret、SMTP密码）在页面上以密文显示，保存后通过加密存储
4. **测试发送独立**：短信测试发送允许使用未启用的服务商发送，方便测试连通性
5. **数据迁移**：原「推送管理」中的配置数据已自动迁移到新模块，无需重新配置
