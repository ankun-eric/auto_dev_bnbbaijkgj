## 功能简介

本次更新对系统管理中的消息推送功能进行了**模块拆分与增强**，具体变更如下：

1. **短信管理增强**：新增阿里云短信服务支持（腾讯云+阿里云双服务商，同一时间仅一家生效），新增短信模板管理功能
2. **微信推送管理独立**：从原"推送管理"中拆出，成为独立菜单模块
3. **邮件通知管理独立**：从原"推送管理"中拆出，成为独立菜单模块，并新增发送记录和测试发送功能
4. **删除推送管理**：原"系统设置 → 推送管理"标签页已移除

## 使用说明

### 一、短信管理

**入口路径**：系统管理 → 短信管理

短信管理页面包含 4 个标签页：

#### 1.1 短信配置

页面展示**腾讯云**和**阿里云**两个配置区块（上下排列），每个区块有独立的"启用"开关。

**操作步骤**：

1. 进入管理后台，点击左侧菜单「系统管理 → 短信管理」
2. 在「短信配置」标签页中，可以看到腾讯云和阿里云两个配置区域
3. 填写对应服务商的配置参数：
   - **腾讯云**：SecretID、SecretKey、SDK AppID、短信签名
   - **阿里云**：AccessKey ID、AccessKey Secret、短信签名
4. 通过右上角的开关启用/禁用对应的服务商
5. 点击「保存」按钮保存配置

**注意事项**：
- 同一时间**只能启用一家服务商**。启用一家时，另一家将自动禁用，系统会弹出确认提示
- SecretKey、AccessKey Secret 等密钥类字段以密码方式显示，已设置的密钥不会回显明文

#### 1.2 短信模板

管理系统中使用的短信模板，每个模板绑定具体的服务商。

**操作步骤**：

1. 切换到「短信模板」标签页
2. 点击「新增模板」按钮，在弹窗中填写模板信息：
   - 模板名称（如"验证码模板"）
   - 服务商（腾讯云 / 阿里云）
   - 模板ID（在服务商后台创建后获得）
   - 模板内容（如"您的验证码是${code}，5分钟内有效"）
   - 签名名称
   - 用途场景（验证码 / 通知 / 营销 / 其他）
   - 变量说明
3. 支持按**服务商**和**用途场景**筛选，支持按**模板名称**搜索
4. 可对模板进行编辑和删除操作

#### 1.3 发送记录

查看所有短信发送记录。

**操作步骤**：

1. 切换到「发送记录」标签页
2. 表格展示每条短信的发送时间、手机号（脱敏显示）、验证码、**服务商**、发送状态、失败原因、是否为测试发送
3. 支持按手机号搜索、按服务商筛选、按发送状态筛选

#### 1.4 测试发送

用于测试短信服务是否配置正确。

**操作步骤**：

1. 切换到「测试发送」标签页
2. 选择要测试的**服务商**（腾讯云 / 阿里云）
3. 输入接收手机号
4. 可选填模板ID
5. 点击「发送测试短信」

**注意**：测试发送不受配置页"启用"开关限制，即使某服务商未启用，也可以选择它进行测试。如果所选服务商尚未配置参数，系统会提示"请先完成该服务商的配置"。

### 二、微信推送管理

**入口路径**：系统管理 → 微信推送管理

微信推送管理已从原"推送管理"中拆出为独立页面。

**操作步骤**：

1. 进入管理后台，点击左侧菜单「系统管理 → 微信推送管理」
2. 填写以下配置信息：
   - 启用微信推送（开关）
   - 微信AppID
   - 微信AppSecret（密码输入）
   - 订单通知模板ID（选填）
   - 服务通知模板ID（选填）
3. 点击「保存配置」按钮

**注意事项**：
- AppSecret 字段以密码方式输入，已设置时显示"已设置（重新输入将覆盖）"
- 如果之前在"推送管理"中已配置过微信推送，系统已自动迁移数据到新模块

### 三、邮件通知管理

**入口路径**：系统管理 → 邮件通知管理

邮件通知管理已从原"推送管理"中拆出为独立页面，并新增了发送记录和测试发送功能。

页面包含 3 个标签页：

#### 3.1 邮件配置

**操作步骤**：

1. 进入管理后台，点击左侧菜单「系统管理 → 邮件通知管理」
2. 在「邮件配置」标签页中填写 SMTP 服务器信息：
   - 启用邮件通知（开关）
   - SMTP 服务器（如 smtp.qq.com）
   - SMTP 端口（如 465）
   - SMTP 用户名（发件邮箱地址）
   - SMTP 密码（邮箱授权码）
3. 点击「保存配置」按钮

#### 3.2 发送记录

查看系统发送的所有邮件记录。

**操作步骤**：

1. 切换到「发送记录」标签页
2. 表格展示每封邮件的发送时间、收件人、邮件主题、发送状态、失败原因、是否为测试邮件
3. 支持按收件人邮箱搜索、按发送状态筛选

#### 3.3 测试发送

用于测试邮件服务配置是否正确。

**操作步骤**：

1. 切换到「测试发送」标签页
2. 输入收件人邮箱
3. 输入邮件主题（默认"测试邮件"）
4. 可选填邮件内容
5. 点击「发送测试邮件」

**注意**：发送前系统会检查邮件配置是否完整，如未完成 SMTP 配置，会提示"请先完成邮件服务器配置"。

**入口路径**：系统管理 → 系统设置

原"系统设置"页面中的"推送管理"标签页已被删除。短信、微信推送、邮件通知三个功能已各自独立为顶级菜单模块。

当前系统设置页面仅保留：基础配置、注册设置、协议管理。

## 注意事项

1. **权限要求**：以上所有功能仅管理员（admin）可访问。普通用户访问将返回 403 错误
2. **密钥安全**：所有密钥类字段（SecretKey、AccessKey Secret、AppSecret、SMTP 密码）在 API 返回时均已脱敏处理，不会泄露明文
3. **数据迁移**：如果之前在"推送管理"中已配置过微信推送或邮件通知，系统在启动时会自动将数据迁移到新模块的键名下，无需手动操作
4. **短信服务商互斥**：腾讯云和阿里云短信服务同一时间只能启用一家。切换时系统会提示确认
5. **测试发送**：短信测试发送不受"启用"开关限制，方便分别测试两家服务商的连通性；邮件测试发送需要先完成 SMTP 配置
