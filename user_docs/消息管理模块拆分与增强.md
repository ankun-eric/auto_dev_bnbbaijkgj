## 功能简介

本次更新对系统的消息管理模块进行了全面拆分与增强，主要变更如下：

1. **短信管理增强**：新增阿里云短信支持（原仅支持腾讯云），新增短信模板管理功能，发送记录和测试发送均支持按服务商操作
2. **微信推送管理独立**：从原"推送管理"标签页中拆出，成为独立的管理菜单
3. **邮件通知管理独立**：从原"推送管理"标签页中拆出，新增邮件发送记录和测试发送功能
4. **推送管理删除**：原"系统设置 → 推送管理"标签页已删除，三个模块各自独立

## 使用说明

### 一、短信管理

#### 入口

登录管理后台后，在左侧菜单栏找到 **系统管理 → 短信管理**，点击进入。

#### 1. 短信配置

页面包含**腾讯云**和**阿里云**两个配置区块（上下排列）。

**操作步骤：**

1. 在对应服务商区块中填写配置信息：
   - **腾讯云**：填写 SecretID、SecretKey、SDK AppID、短信签名
   - **阿里云**：填写 AccessKey ID、AccessKey Secret、短信签名
2. 通过启用开关选择当前使用的服务商（**同一时间只能启用一家**）
3. 启用一家服务商时，系统会弹出确认提示："切换服务商将禁用当前已启用的服务商，是否继续？"
4. 点击"保存"按钮保存配置

> 注意：密钥类字段（SecretKey / AccessKey Secret）保存后将以脱敏形式显示，不会暴露原文。

#### 2. 短信模板

管理系统中使用的短信模板。每个模板绑定具体的服务商。

**操作步骤：**

1. 点击"新增模板"按钮，在弹窗中填写：
   - 模板名称（如"验证码模板"）
   - 服务商（腾讯云 / 阿里云）
   - 模板ID（在服务商后台创建后获得）
   - 模板内容（如"您的验证码是${code}，5分钟内有效"）
   - 签名名称、用途/场景、模板变量说明
2. 支持按服务商、用途/场景筛选，支持按模板名称搜索
3. 可编辑和删除已有模板（删除需二次确认）

#### 3. 发送记录

查看系统发送的所有短信记录。

- 表格中包含：发送时间、手机号（脱敏显示）、验证码、**服务商**、发送状态、失败原因、是否测试发送
- 支持按手机号搜索、按发送状态筛选、按**服务商筛选**（全部 / 腾讯云 / 阿里云）

#### 4. 测试发送

用于测试短信服务的连通性。

**操作步骤：**

1. 选择**服务商**（腾讯云 / 阿里云）——可选择任意服务商进行测试，不受"启用"开关限制
2. 输入接收测试短信的手机号
3. （可选）填写模板ID
4. 点击"发送"按钮
5. 查看发送结果反馈

> 注意：如果所选服务商尚未配置参数，系统会提示"请先完成该服务商的配置"。

### 二、微信推送管理

#### 入口

左侧菜单栏 **系统管理 → 微信推送管理**。

#### 操作步骤

1. 配置微信推送开关（启用/禁用）
2. 填写微信 AppID
3. 填写微信 AppSecret（保存后脱敏显示）
4. （可选）填写订单通知模板ID和服务通知模板ID
5. 点击"保存配置"按钮

---

### 三、邮件通知管理

#### 入口

左侧菜单栏 **系统管理 → 邮件通知管理**。

#### 1. 邮件配置

**操作步骤：**

1. 配置邮件通知开关（启用/禁用）
2. 填写 SMTP 服务器地址（如 smtp.qq.com）
3. 填写 SMTP 端口（如 465）
4. 填写 SMTP 用户名（发件邮箱地址）
5. 填写 SMTP 密码（邮箱授权码，保存后脱敏显示）
6. 点击"保存"按钮

#### 2. 发送记录

查看系统发送的所有邮件记录。

- 表格包含：发送时间、收件人、邮件主题、发送状态、失败原因、是否测试邮件
- 支持按收件人邮箱搜索、按发送状态筛选（全部 / 成功 / 失败）

#### 3. 测试发送

用于验证邮件配置是否正确。

**操作步骤：**

1. 输入收件人邮箱
2. 输入邮件主题（默认"测试邮件"）
3. （可选）输入邮件内容
4. 点击"发送"按钮
5. 查看发送结果（成功/失败及原因）

> 注意：发送前系统会校验邮件配置是否完整，未配置则提示"请先完成邮件服务器配置"。

### 四、系统设置变更

原"系统设置"页面中的"推送管理"标签页已被删除。当前系统设置仅保留：

- **基础配置**：应用名称、联系方式、功能开关等
- **注册设置**：自助注册会员、微信端注册方式等
- **协议管理**：用户服务协议、隐私政策等

短信、微信推送、邮件通知的配置已各自独立为单独的管理模块。

---

## 注意事项

1. **服务商互斥**：短信管理中，腾讯云和阿里云同一时间只能启用一家。切换时请确认业务影响。
2. **密钥安全**：所有密钥类字段（SecretKey、AccessKey Secret、AppSecret、SMTP密码）在保存后均以脱敏方式显示，不会暴露原始值。
3. **权限控制**：短信管理、微信推送管理、邮件通知管理各有独立的权限控制，管理员可分别授权。
4. **数据迁移**：原"推送管理"中的配置数据已自动迁移到对应的新模块，无需手动操作。
5. **旧接口废弃**：原推送管理统一保存接口（`/api/admin/settings/push`）已废弃，调用将返回 410 错误。
