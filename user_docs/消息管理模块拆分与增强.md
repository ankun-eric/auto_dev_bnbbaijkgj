## 功能简介

本次更新对系统的消息管理模块进行了全面的拆分与增强，主要变更如下：

1. **短信管理模块增强**：新增阿里云短信支持，实现腾讯云/阿里云双服务商配置（同一时间仅一家生效）；新增短信模板管理功能（增删改查）；发送记录增加服务商字段和筛选；测试发送支持选择服务商。
2. **微信推送管理**：从原"推送管理"中拆分为独立菜单，功能和配置与原有一致。
3. **邮件通知管理**：从原"推送管理"中拆分为独立菜单，新增邮件发送记录和测试发送功能。
4. **删除"推送管理"**：原"系统设置 → 推送管理"标签页已移除，三个模块各自独立管理。

## 使用说明

### 一、短信管理

**入口路径**：管理后台 → 左侧菜单"系统管理" → "短信管理"

短信管理页面包含 4 个标签页：

#### 1.1 短信配置

该页面上下排列展示**腾讯云**和**阿里云**两个配置区块。

**操作步骤**：

1. 进入"短信配置"标签页，页面顶部为"腾讯云短信配置"区块，下方为"阿里云短信配置"区块。
2. 每个区块右上角有"启用"开关。**同一时间只能启用一家服务商**——启用其中一家时，系统会弹出确认提示，确认后另一家自动禁用。
3. **腾讯云配置**需填写：SecretID、SecretKey（密码输入框）、SDK AppID、短信签名。
4. **阿里云配置**需填写：AccessKey ID、AccessKey Secret（密码输入框）、短信签名。
5. 填写完成后，点击各区块底部的"保存"按钮保存配置。
6. 密码类字段（SecretKey / AccessKey Secret）保存后将脱敏显示，提示"已设置"。

#### 1.2 短信模板

管理系统中使用的短信模板，支持按服务商和场景分别配置。

**操作步骤**：

1. 切换到"短信模板"标签页，可看到模板列表。
2. **新增模板**：点击右上角"新增模板"按钮，在弹窗中填写模板名称、选择服务商（腾讯云/阿里云）、模板ID、模板内容、签名名称、用途场景（验证码/通知/营销/其他）、变量说明，设置启用/禁用状态。
3. **编辑模板**：点击操作列的"编辑"按钮修改模板信息。
4. **删除模板**：点击操作列的"删除"按钮，确认后删除。
5. **筛选和搜索**：使用顶部筛选条件按服务商、用途场景筛选，或输入模板名称搜索。

#### 1.3 发送记录

查看系统所有短信发送记录。

**表格列**：发送时间、手机号（脱敏显示）、验证码、服务商（腾讯云/阿里云）、发送状态、失败原因、测试标记。

**筛选**：支持按手机号搜索、按服务商筛选、按发送状态筛选。

#### 1.4 测试发送

用于测试短信配置是否正确。

**操作步骤**：

1. 切换到"测试发送"标签页。
2. **选择服务商**（必选）：下拉选择"腾讯云"或"阿里云"，测试时使用该服务商的配置发送，不受启用开关限制。
3. **填写手机号**（必填）：输入接收测试短信的手机号。
4. **模板ID**（可选）：如需使用特定模板可填写，留空使用默认模板。
5. 点击"发送测试短信"按钮，发送结果会实时显示在下方。

### 二、微信推送管理

**入口路径**：管理后台 → 左侧菜单"系统管理" → "微信推送管理"

**操作步骤**：

1. 进入微信推送管理页面，页面展示一个配置表单。
2. **启用微信推送**：顶部开关控制是否启用微信推送功能。
3. **微信AppID**（必填）：填写公众号/小程序的AppID。
4. **微信AppSecret**：填写对应的AppSecret（密码输入框，保存后脱敏显示）。
5. **订单通知模板ID**（选填）：填写订单相关推送模板ID。
6. **服务通知模板ID**（选填）：填写服务相关推送模板ID。
7. 点击"保存配置"按钮保存。

---

### 三、邮件通知管理

**入口路径**：管理后台 → 左侧菜单"系统管理" → "邮件通知管理"

邮件通知管理页面包含 3 个标签页：

#### 3.1 邮件配置

**操作步骤**：

1. 进入"邮件配置"标签页。
2. **启用邮件通知**：顶部开关控制是否启用邮件通知功能。
3. **SMTP服务器**（必填）：填写 SMTP 服务器地址，如 `smtp.qq.com`。
4. **SMTP端口**（必填）：填写端口号，如 `465`（SSL）或 `587`（TLS）。
5. **SMTP用户名**（必填）：填写发件邮箱地址。
6. **SMTP密码**：填写邮箱授权码（密码输入框，保存后脱敏显示）。
7. 点击"保存配置"按钮保存。

#### 3.2 发送记录

查看系统所有邮件发送记录。

**表格列**：发送时间、收件人、邮件主题、发送状态（成功/失败）、失败原因、测试标记。

**筛选**：支持按收件人邮箱搜索、按发送状态筛选。

#### 3.3 测试发送

用于测试邮件配置是否正确。

**操作步骤**：

1. 切换到"测试发送"标签页。
2. **收件人邮箱**（必填）：输入接收测试邮件的邮箱地址。
3. **邮件主题**（必填）：默认为"测试邮件"，可修改。
4. **邮件内容**（选填）：默认为"这是一封测试邮件，用于验证邮件配置是否正确。"，可修改。
5. 点击"发送测试邮件"按钮，发送结果会实时显示在下方。
6. 如果邮件配置尚未完成，会提示"请先完成SMTP配置"。

## 注意事项

1. **服务商互斥**：短信管理中腾讯云和阿里云同一时间只能启用一家，切换时系统会弹出确认提示。
2. **密钥安全**：所有密钥类字段（SecretKey、AccessKey Secret、AppSecret、SMTP密码）在页面回显时脱敏显示，重新输入将覆盖原值。
3. **测试发送不受启用限制**：短信测试发送时选择的服务商不受配置页"启用"开关限制，方便分别测试两家服务商的连通性。
4. **数据自动迁移**：原"推送管理"中的配置数据已自动迁移到新模块对应的键名下，无需手动操作。
5. **权限要求**：上述所有功能仅管理员（admin 角色）可访问，其他角色用户无权操作。
6. **管理员登录**：默认管理员手机号 `13800000000`，密码 `admin123`。
