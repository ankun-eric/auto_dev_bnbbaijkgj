## 一、功能简介

本次更新围绕「商家端的角色体系与员工管理体验」做了一轮系统性治理与优化，包含 **1 项治理性大改造 + 3 项体验优化 + 1 个核心 Bug 修复**：

- **R1（治理）** 把商家角色统一收敛为 **4 个**：老板（boss）、店长（store_manager）、财务（finance）、店员（clerk）。彻底告别历史上"核销员 verifier / staff / owner / manager"等多套命名打架的问题。
- **R2（优化）** 平台 admin 后台员工列表中冗余的「账号」列已合并到「手机号」列，搜索框提示同步改为「请输入手机号搜索」。
- **R3（优化）** 收紧员工"启用 / 冻结"权限：只有 **老板** 可见、可点、可调用；店长、财务、店员一律不可见，后端兜底返回 `无权限操作`。
- **R4（优化）** H5 端员工管理列表加入了**头像角色色彩**：老板金色、店长蓝色、财务橙红、店员灰蓝，一眼区分团队角色。
- **B1（Bug 修复）** 解决商家"我的 → 个人信息"页 8 项字段全部为 `—` 的问题（根因为后端两个同路径接口冲突）；现在 4 角色 × PC + 移动 共 8 个组合下都能正确显示完整字段。

## 二、使用说明

### 2.1 R1：4 角色统一后，登录与角色显示

更新后只有 4 个商家角色：

| 角色 | 英文 code | 中文显示 | 主要权限 |
|---|---|---|---|
| 老板 | `boss` | 老板 | 全模块 + 启停员工 + 商家信息编辑 |
| 店长 | `store_manager` | 店长 | 全模块（除"启停员工"和"商家信息编辑"外） |
| 财务 | `finance` | 财务 | 看账目、报表，不进入员工管理 |
| 店员 | `clerk` | 店员 | 一线作业、订单核销 |

> 历史上的 `verifier` / `staff` / `owner` / `manager` 账号已自动迁移：
> - `verifier`（核销员）→ `clerk`（店员，承担核销职责）
> - `staff` → `clerk`
> - `owner` → `boss`
> - `manager` → `store_manager`
>
> 用户**无需任何手动操作**，重新登录后即可看到正确角色名。

### 2.2 R2：admin 员工列表"账号"列已合并

操作路径：

1. 登录 [admin 后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 进入 [商家账号管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/merchant/accounts)
3. 点击任一商家行末的「员工」按钮 / 抽屉，进入员工列表

变更点：

- 列由原来的「账号 / 手机号 / 角色 / 状态 / 操作」简化为 **「手机号 / 角色 / 状态 / 操作」**
- 顶部搜索框 placeholder 改为 **「请输入手机号搜索」**
- 检索逻辑保持不变（本就按手机号匹配）

### 2.3 R3：员工启停按钮的权限收紧

新规则（H5 商家端，PC + 移动版均生效）：

| 当前登录角色 | 启停按钮可见 | 直接调接口可成功 |
|---|---|---|
| 老板 | ✅ 可见（除自己行无按钮） | ✅ |
| 店长 | ❌ 任何行都不显示 | ❌ 后端 403 `无权限操作` |
| 财务 / 店员 | ❌ 看不到员工管理菜单 | ❌ |

操作流程：

1. 老板登录后进入 [员工管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/staff)
2. 在目标员工行点击「冻结」/「启用」按钮
3. 系统弹窗确认后写入 `account_operation_log`（记录操作人 / 目标人 / 时间 / IP）

兜底文案（接口被绕过时）：

| 场景 | 文案 |
|---|---|
| 非老板调用启停接口 | `无权限操作` |
| 老板试图启停自己 | `无权限操作` |
| 目标用户不存在 | `员工不存在` |

### 2.4 R4：员工头像按角色着色

进入 [员工管理 PC 版](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/staff) 或 [员工管理 移动版](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/m/staff)：

| 角色 | 颜色 | HEX | 显示 |
|---|---|---|---|
| 老板 | 金色 | `#F5A623` | 圆形 + 姓名首字 |
| 店长 | 蓝色 | `#4A90E2` | 圆形 + 姓名首字 |
| 财务 | 橙红 | `#D0021B` | 圆形 + 姓名首字 |
| 店员 | 灰蓝 | `#9B9B9B` | 圆形 + 姓名首字 |

实现细节：

- 形态：纯色圆形 + 白色文字（姓名首字）
- 尺寸：PC 端 40px / 移动端 36px
- 文字：`PingFang SC, system-ui`，16px，500 字重
- 取值规则：`name[0]`；若姓名为空则取手机号尾号末位 1 位

> 注：本次着色仅作用于「员工管理列表」头像，不动「我的」页头像、订单页操作人头像等。

### 2.5 B1：商家"我的"页个人信息已正常显示

操作路径：

1. 商家账号登录 [H5 商家端](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant)
2. 进入「我的」→「个人信息」（[直链](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/profile)）

应能正确看到 8 个字段：

| 字段 | 含义 |
|---|---|
| `name` | 姓名 |
| `phone` | 手机号 |
| `role_name` | 角色（老板 / 店长 / 财务 / 店员） |
| `store_names` | 所属门店（多门店逗号拼接） |
| `merchant_name` | 所属商家名称 |
| `account_status` | 账号状态（启用 / 冻结） |
| `created_at` | 加入时间 |
| `last_login_at` | 最近登录时间 |

> 修复说明：之前后端有两个完全同路径的 `GET /api/merchant/profile` 接口注册到 FastAPI，老接口（仅返回 `nickname/avatar`）会覆盖新接口，导致前端拿到的字段全部 `undefined`，渲染为 `—`。本次直接删除 `backend/app/api/merchant.py:381` 老接口，唯一保留 `account_security.py:246` 的完整 8 字段实现。同时新增了 `backend/scripts/scan_route_conflicts.py` 路由冲突全局扫描脚本，后端启动期也会输出 warning，避免类似问题再出现。

## 三、注意事项

1. **角色兼容期**：本次保留 1 个版本周期的 `owner/manager/verifier/staff` 历史值兼容映射，下个版本会彻底移除。如有定制对接，请尽快切换到 4 角色 code。
2. **店长不再能启停员工**：本次起店长只能"看到员工管理菜单和列表"，但**不会再看到启停按钮**，这是有意收紧。如有业务异议请联系平台。
3. **admin 后台启停按钮不变**：admin 后台对商家员工的启停权限保持不变（平台运营仍可代为操作）。
4. **头像配色不开放自定义**：本次不引入头像图片上传，也不在头像旁加额外角色文字标签（已有"角色"列足以区分）。
5. **个人信息页空值占位**：保持 `—` 占位策略，不做兜底虚假字符串转换。
6. **回滚预案**：如发现严重问题，可执行 `backend/migrations/v1_0_merchant_role_unify_rollback.sql` 回滚数据迁移；后端代码可回退到 `ce573ae` 之前的 commit。

## 四、常见问题（FAQ）

**Q1：我的账号原来是核销员（verifier），现在登录看到角色变成"店员"了，是不是降权了？**  
A：不是降权。本次治理把「核销职责」并入「店员」角色，您原有的核销权限完全保留。除"角色名变了"以外，您的可见菜单、可点按钮均不变。

**Q2：店长能不能临时启停一下员工？**  
A：不能。本次后端会强制返回 `无权限操作`。如需启停，请联系您所在商家的老板账号操作；平台运营也可在 admin 后台代为处理。

**Q3：个人信息页里的「所属门店」是空的怎么办？**  
A：通常是该账号未被绑定到任何门店。请联系老板/管理员将该账号添加到门店成员中。

**Q4：原 H5 上员工头像是统一灰色的，现在变颜色了，可以关掉吗？**  
A：暂不开放关闭开关。后续如有强需求可在「我的 → 个人偏好」中新增开关。
