## 功能简介

本次更新落地了 PRD V3 终版「家庭成员状态模型 & 一人一档治本方案」第一阶段——**应急修复 SOP** 和 **V3 后端基础**，彻底解决「家人 Tab 看不到苏俊林、李 X，但家庭档案能看到」（6399 账号现场反馈）的两侧口径不一致问题，并为前端引入新的「主+子状态」视图模型。

本次主要变化：

1. **过滤口径全面统一**：家人 Tab、健康档案 member 接口、家庭成员配额查询、状态机列表等所有相关接口，已**全部统一**为「同时排除 `deleted` 与 `removed` 软删除标记」。再也不会出现某个接口能看到、另一个接口看不到的情况。
2. **V3 主+子状态视图字段**：`/api/family/members` 接口现在为每个成员卡片返回 5 个新字段：
   - `v3_main_status`：主状态（`unbound` / `bound` / `deleted`）
   - `v3_sub_status`：子状态（`not_applied` / `applying` / `rejected` / `unbinded` / `invited_expired` / `bound` / `self_deleted`）
   - `v3_can_reinvite`：是否显示「重新邀请」按钮
   - `v3_can_edit`：是否显示「编辑」按钮
   - `v3_show_simplified_view`：解绑/已删除成员卡片是否进入"极简视图"
3. **解绑后老人 Tab 极简化**：当某个家人卡片处于「已解绑」或「已删除」状态时，进入该家人的 Tab 后页面只保留 **Hero 卡片** + **「他的守护人」卡片**，不再显示健康看板入口、提醒管理、设备、用药计划等业务模块（PRD 决策点 14~18）。
4. **「重新邀请」按钮**：解绑/已删除成员的 Hero 卡片右上角，"编辑"按钮左侧新增**「重新邀请」**按钮（首次未发起邀请时则显示「邀请」），一键直达邀请二维码页面，与首次邀请走完全一致的流程。

## 使用说明

### 场景 1：查看与某个家人的解绑状态

1. 打开 [健康档案首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/)
2. 在顶部成员条点击对方头像（如「苏俊林」），切换到该家人的 Tab
3. 如果你与该家人**已解除守护**或对方/你已**删除卡片**，页面将自动进入"极简视图"：
   - 顶部 Hero 卡片仍显示（昵称、关系、性别、年龄等基本信息）
   - 中间一张「XX 的守护人」卡片
   - 中间会出现一行温馨提示：**"您与该家人已解除守护关系。如需继续守护，可在 Hero 卡片上点击「重新邀请」。"**
   - 页面其余的健康看板入口、提醒管理、设备、用药计划等业务模块**不再显示**

### 场景 2：重新邀请已解绑的家人

1. 在已解绑家人的 Hero 卡片右上角，找到白底紫字的**「重新邀请」**按钮
2. 点击该按钮，会跳转到邀请二维码页面
3. 让对方扫描二维码，重新建立守护关系
4. 邀请有效期为 **24 小时**（PRD 决策点 4）。对方在 24 小时内扫码同意后，关系自动恢复，家人 Tab 上的卡片重新进入"已绑定"状态，所有业务模块自动恢复

### 场景 3：首次邀请未注册的家人

如果该家人**从未发起过邀请**（V3 子状态 `not_applied`），Hero 卡片右上角的按钮文案会显示为**「邀请」**而非"重新邀请"。点击后流程完全相同。

## 状态机说明（参考）

| V3 主状态 | V3 子状态 | 含义 | 用户可见交互 |
|---|---|---|---|
| `bound` | `bound` | 守护关系建立中 | 编辑按钮、完整健康档案模块 |
| `unbound` | `not_applied` | 还没发起过邀请 | 编辑按钮 + **邀请**按钮 |
| `unbound` | `applying` | 邀请中（24h 倒计时） | 编辑按钮（不显示重新邀请） |
| `unbound` | `rejected` | 对方拒绝 | 编辑按钮 + **重新邀请**按钮 |
| `unbound` | `invited_expired` | 邀请 24h 过期 | 编辑按钮 + **重新邀请**按钮 |
| `unbound` | `unbinded` | 已解除守护 | **极简视图** + Hero 上"重新邀请"按钮 |
| `deleted` | `self_deleted` | 守护人主动删除卡片 | **极简视图** + Hero 上"重新邀请"按钮 |

## 注意事项

1. **本人 Tab 不受影响**：本人卡片的 V3 状态恒为 `bound/bound`，所有原有的健康看板、提醒、用药、设备等模块均完整保留。
2. **删除（彻底删除）与解绑不同**：解绑后只是关系层 cancelled，档案仍在；删除后 family_members 卡片软删（`status='removed'` 或 `'deleted'`），仍可通过"重新邀请"恢复。本次更新只对解绑/删除两种状态启用极简视图。
3. **小程序用户须知**：小程序原生家人 Tab 不消费 V3 字段，**仍然展示老视图**，但因后端过滤已统一，不会再出现"已解绑成员仍出现在档案列表"的现象。下一次小程序例行迭代时会同步引入极简视图。
4. **守护人额度**：每位被守护人最多 3 位守护人（1 主 + 2 普通），重新邀请不会绕过此上限。
