## 功能简介

本次修复了 **短信验证码登录失败** 的 P0 级 Bug。此前所有用户（包括测试号 13800138000）在 H5 端输入正确验证码后均提示"登录失败，请检查验证码"，导致核心登录功能完全不可用。

### 问题根因

部署「家庭健康档案共管」新功能并进行数据库变更后，`health_profiles` 表中出现了同一用户的多条重复记录。登录流程中调用 `is_profile_completed` 函数时，`scalar_one_or_none()` 对多行结果抛出 `MultipleResultsFound` 异常，导致后端返回 500 错误。

### 修复内容

1. **数据查询安全加固**：所有通过 `user_id` 查询 `health_profiles` 的位置均添加了 `ORDER BY` + `LIMIT 1`，防止多行结果导致异常
2. **数据自动去重**：后端启动时自动检测并清理重复的健康档案记录
3. **MySQL 时区统一**：MySQL 容器时区强制设置为 UTC，与 Python 端保持一致
4. **调试日志增强**：验证码校验失败时记录详细的调试信息，便于后续排查
