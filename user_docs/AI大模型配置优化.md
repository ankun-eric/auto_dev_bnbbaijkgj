## 功能简介

本次更新对管理后台的「AI 大模型配置」功能进行了三大优化：

1. **引入预置模型模板**：管理员添加 AI 配置时，可从预置的模型模板中选择（如火山引擎 DeepSeek、腾讯云 DeepSeek），系统自动填充 Base URL 和模型名称，只需填写 API Key 即可完成配置，大幅降低操作门槛。

2. **配置列表一键切换活跃配置**：在配置列表中直接点击「设为活跃」按钮即可切换活跃配置，无需进入编辑页面，操作更加高效。

3. **模板与配置关联同步**：通过模板创建的配置会记录关联关系，当管理员修改模板后，系统会在进入配置列表时弹窗提示是否同步更新，确保配置始终与模板保持一致。

> **权限说明**：AI 大模型配置功能仅管理员可见和可操作，普通用户无感知，系统后台自动使用管理员设定的全局活跃配置。

## 使用说明

### 一、进入 AI 大模型配置页面

1. 使用管理员账号登录管理后台
2. 在左侧菜单中点击「AI 模型配置」
3. 页面顶部显示两个 Tab 页签：**配置列表**（默认）和 **模板管理**

### 二、模板管理

#### 查看模板列表

1. 点击「模板管理」Tab 页签
2. 页面展示所有预置模型模板的表格，包含：图标、模板名称、Base URL、模型名称、说明、状态、操作按钮
3. 系统初始内置两个模板：
   - **火山引擎 DeepSeek-V3.2**
   - **腾讯云 DeepSeek-V3.2**

#### 添加新模板

1. 在「模板管理」Tab 中，点击右上角「添加模板」按钮
2. 在弹窗中填写以下信息：
   - **模板名称**：如「阿里通义千问」
   - **Base URL**：模型 API 的基础地址
   - **模型名称**：模型标识符
   - **图标**：从预置的 10 个图标中选择一个（火山引擎、腾讯云、OpenAI、DeepSeek、百度文心、阿里通义、智谱AI、月之暗面、Anthropic、自定义）
   - **说明**：模板的简要描述
3. 填写完成后点击「确定」保存

#### 编辑模板

1. 在模板列表中找到目标模板，点击「编辑」按钮
2. 在弹窗中修改相关信息
3. 点击「确定」保存
4. 如果有配置关联了此模板，下次进入「配置列表」时系统会弹窗提示是否同步更新

#### 停用/启用模板

- 点击模板操作列中的「停用」按钮，模板状态变为停用（添加配置时不再展示该模板）
- 已停用的模板可通过点击「启用」按钮重新启用

#### 删除模板

- 点击「删除」按钮，确认后删除模板
- **注意**：如果有配置关联了该模板，则无法删除，需先删除相关配置

### 三、添加 AI 配置

#### 通过模板添加（推荐）

1. 切换到「配置列表」Tab
2. 点击右上角「添加配置」按钮
3. 弹出模板选择弹窗，以卡片形式展示所有启用的模板
4. 点击选择一个模板卡片（如「火山引擎 DeepSeek-V3.2」）
5. 系统自动打开配置表单，并自动填充：
   - 服务商名称 = 模板名称
   - Base URL = 模板的 Base URL
   - 模型名称 = 模板的模型名称
6. 您只需填写 **API Key**（以及可选调整 Max Tokens 和 Temperature）
7. 点击「保存」完成配置

#### 自定义添加

1. 在模板选择弹窗中，点击最后一张「自定义配置」卡片
2. 所有字段均为空白，手动填写：
   - 服务商名称、Base URL、模型名称、API Key、Max Tokens、Temperature
3. 点击「保存」完成配置

### 四、切换活跃配置

1. 在「配置列表」Tab 中查看所有配置
2. 当前活跃的配置会显示绿色的「当前活跃」标签
3. 非活跃配置的操作列会显示「设为活跃」按钮
4. 点击「设为活跃」按钮，该配置立即变为活跃配置
5. 切换即时生效，所有用户立即使用新的活跃配置

### 五、模板同步

当管理员修改了某个模板的信息（如 Base URL、模型名称等）后：

1. 下次进入「配置列表」页面时，系统会自动检测是否有未同步的模板更新
2. 如果有，会弹出同步提示弹窗，展示哪些配置关联了已更新的模板
3. 点击「同步」按钮：将模板的最新信息同步到关联配置
4. 点击「忽略」按钮：本次不同步，下次模板再更新时才会再次提示

### 六、编辑/删除配置

- **编辑**：点击配置列表中的「编辑」按钮，修改配置信息后保存
- **删除**：点击「删除」按钮，确认后删除配置

## 注意事项

1. **权限要求**：AI 大模型配置功能仅管理员可见。普通用户在左侧菜单中不会看到「AI 模型配置」入口。

2. **全局活跃配置**：活跃配置为全局共享模式，管理员设定的活跃配置会被所有用户统一使用。

3. **同一时间只有一个活跃配置**：切换活跃配置时，原活跃配置会自动变为非活跃。

4. **API Key 安全**：API Key 在列表中以脱敏方式显示（仅显示前几位 + ****）。编辑配置时，如不修改 API Key，留空即可。

5. **模板停用不影响已有配置**：停用模板后，已通过该模板创建的配置不受影响，只是新建配置时该模板不再展示。

6. **测试连接**：在添加或编辑配置时，可以点击「测试连接」按钮验证 API Key 和连接是否正常。

7. **自定义配置**：如果使用的模型不在预置模板中，可以选择「自定义配置」手动填写所有参数。
