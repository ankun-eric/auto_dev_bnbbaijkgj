## 功能简介

本次修复解决了"报告解读"与"识药"两类首页 AI 功能按钮的核心问题：当运营在后台**功能按钮管理**中以**新体系**配置按钮（`button_type=ai_function` + `ai_function_type=image_capture` + `capture_purpose=interpret_report/identify_medicine`）时，原先因前后端未识别新体系配置，导致：

- 报告解读结果只是普通文字回复，没有结构化卡片
- 「历史报告」页面看不到本次解读记录
- 「档案管理 → 就医资料 → 体检报告」看不到本次报告
- 识药按钮在同样链路上也存在类似问题（且原本根本没有把识药结果归档到「就医资料 → 用药记录」）

修复后，**无论运营在后台用哪种方式配置「报告解读」/「识药」按钮**（老体系、中间兼容体系、新体系），前后端都会准确识别并路由到正确的专用 AI 引擎，并把结果自动归档到对应的「就医资料」记录里。

### 修复后的能力一览

| 链路 | 行为 |
|------|------|
| 报告解读 | 出**结构化解读卡片**；自动在「历史报告」生成 1 条记录；自动在「就医资料 → 体检报告」生成 1 条记录 |
| 识药 | 出**识药卡片**；自动在「就医资料 → 用药记录」生成 1 条记录（**本次新增能力**） |
| 纯上传 / 通用图片 | 走通用 AI 看图，**不**生成任何专用记录（避免误归档） |

## 使用说明

### 场景 1：运营配置按钮（管理后台）

1. 进入 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/) 登录
2. 进入「AI 咨询配置 → 功能按钮管理」
3. 新增/编辑「报告解读」按钮时，可以使用以下**任一**配置方式，前后端都会正确识别（推荐使用新体系 ⭐）：

   | 配置方式 | `button_type` | `ai_function_type` | `capture_purpose` |
   |----------|---------------|--------------------|---------------------|
   | 老体系 | `report_interpret` | — | — |
   | 中间兼容 | `ai_function` | `report_interpret` | — |
   | 新体系 ⭐ | `ai_function` | `image_capture` | `interpret_report` |

4. 「识药」按钮同理：

   | 配置方式 | `button_type` | `ai_function_type` | `capture_purpose` |
   |----------|---------------|--------------------|---------------------|
   | 老体系 | `photo_recognize_drug` | — | — |
   | 中间兼容 | `ai_function` | `medicine_recognize` | — |
   | 新体系 ⭐ | `ai_function` | `image_capture` | `identify_medicine` |

5. 保存配置后**无需重启服务**，前端立即生效。

### 场景 2：终端用户体验报告解读（H5/小程序）

1. 打开 [H5 首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/) 或微信小程序
2. 在首页找到「报告解读」胶囊按钮，点击它
3. 在弹出的菜单中选择「相册」或「拍照」上传一张体检报告图片
4. 等待 AI 完成解读，**应该看到「结构化解读卡片」**（指标解读、健康建议等结构化展示，而不是单纯一段文字）
5. 解读完成后，进入「我的 → 历史报告」可以看到本次解读记录
6. 进入「我的 → 档案管理 → 就医资料 → 体检报告」也可以看到本次记录

### 场景 3：终端用户体验识药（H5/小程序）

1. 打开 [H5 首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/) 或微信小程序
2. 在首页找到「识药」胶囊按钮，点击它
3. 上传药品包装清晰图片
4. 等待 AI 完成识别，应该看到**识药结构化卡片**（药品名、规格、用法用量、风险提示等）
5. 识别完成后，进入「我的 → 档案管理 → 就医资料 → 用药记录」可以看到本次识药记录（**本次新增能力**）

## 注意事项

| 事项 | 说明 |
|------|------|
| 兼容性 | 本次修复**完全向后兼容**，存量「报告解读」「识药」按钮（无论是哪种配置方式）**无需任何后台调整**即可正常工作 |
| 仅对新会话生效 | 本次修复**仅对修复部署之后的新会话**生效。修复前因 Bug 没有归档的历史"文字解读"不会回溯写入「历史报告 / 就医资料」 |
| family_member_id | 解读/识药结果归档时需要「咨询人」(family_member_id)。如果会话未绑定咨询人，识药记录将**跳过写入**「就医资料 → 用药记录」（仅显示在聊天里），日志会输出 warning |
| 去重保护 | 同一聊天会话内反复触发「报告解读」/「识药」**不会重复归档**到「就医资料」(基于 `user_id + member_id + source + session_id` 去重) |
| 浏览器缓存 | H5 用户首次访问修复后版本时，建议 **Ctrl+Shift+R 强制刷新** 一次，以加载最新前端 bundle |
| 小程序需重新导入 | 微信小程序端需要**重新下载并导入最新 zip 包**，旧版小程序不会自动生效 |
| 后台运营操作 | 本次修复**不需要任何运营手动操作**，所有现有按钮在代码上线后立即按新逻辑工作 |
