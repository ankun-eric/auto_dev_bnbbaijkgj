## 功能简介

本次修复解决了 **"解绑后重新邀请成员，对方打开链接时提示'无法处理邀请'"** 的问题。

### 修复前的问题

当管理者解除与某成员的守护关系（解绑）后，再对该成员点击"重新邀请"按钮发起新邀请，对方在微信中打开邀请链接时，页面会显示 **"无法处理邀请"** 的错误提示，导致无法正常完成重新绑定。

### 修复后的行为

修复后，解绑后重新邀请的流程完全正常：

1. 管理者点击"重新邀请" → 系统正确标记该成员状态为"申请中"，并清理旧的无效守护记录
2. 对方打开邀请链接 → 正常显示确认页面（不再出现"无法处理邀请"）
3. 对方点击"同意" → 系统优先复用旧的守护记录而非创建重复记录，绑定成功

## 使用说明

### 场景：解绑后重新邀请成员

#### 第一步：解绑成员

1. 进入「健康档案」→「家庭成员列表」
2. 找到需要解绑的成员，点击进入详情
3. 点击「解除守护」按钮
4. 输入短信验证码，确认解绑
5. 解绑成功，成员状态变为"未绑定"，列表中出现「重新邀请」按钮

#### 第二步：重新邀请

1. 在家庭成员列表中，点击该成员的「重新邀请」按钮
2. 系统自动跳转到邀请页面并生成新的邀请链接
3. 复制邀请链接，通过微信发送给对方

#### 第三步：对方接受邀请

1. 对方在微信中打开邀请链接
2. 页面显示邀请确认信息（邀请人姓名、关系类型等）
3. 勾选同意协议，点击「同意」按钮
4. 绑定成功，双方恢复守护关系

### 错误提示说明

修复后，邀请页面可以准确识别以下错误类型：

- **邀请不存在** — 链接无效或已被删除（新增匹配）
- **邀请已过期** — 超过 24 小时未处理
- **邀请已取消** — 邀请已被撤销
- **您已在守护关系中** — 重复绑定
- **无法处理邀请** — 其他异常情况（兜底）

## 注意事项

1. 邀请链接有效期为 **24 小时**，超时需重新发起
2. 解绑后重新邀请会清理旧的守护记录，确保数据一致性
3. 同一管理者对同一成员只能有一条有效的守护关系
4. 被守护人数上限默认为 **10 人**，被管理人数上限为 **3 人**
5. 本修复仅涉及后端和 H5 前端代码，无需更新客户端 App
