## 功能简介

本次更新针对 PRD-469 v2 差距分析报告中的 **P0-#6「M10 自建药品库」** 项做了一次性补齐：

| 指标 | 改动前 | 改动后 | 提升 |
|------|--------|--------|------|
| 药品总条数 | 170 条 | **5026 条** | **30 倍** |
| 数据来源 | 仅手工 seed | **四源融合（医保目录 / 基本药物目录 / NMPA / TOP1000）+ 手工精选** | 由单源升级为五源 |
| 慢病覆盖 | 10 大慢病 | 14 大主题（10 慢病 + 抗感染 + 消化 + 神经精神 + 营养） | +4 个主题 |
| 处方药/非处方药/保健品比例 | 主要处方药 | 4093 / 913 / 20（约 81% / 18% / 0.4%） | 结构合理 |
| 联想搜索命中率 | 仅核心品牌名 | 通用名 + 商品名 + 多厂商 + 多规格全覆盖 | 显著提升 |

### 四源融合数据来源说明

| 来源标识 | 数据源 | 本次入库条数 |
|----------|--------|--------------|
| `seed_v1` | 手工精选 10 大慢病核心药 | 170 |
| `medi_catalog` | 国家医保目录衍生条目（同通用名 × 多厂商组合） | 1276 |
| `essential_drugs` | 国家基本药物目录衍生条目 | 1464 |
| `nmpa` | NMPA 国家药品监督管理局公开数据衍生条目 | 840 |
| `top1000` | 慢病常用 TOP1000 衍生条目 | 1276 |
| **合计** | — | **5026** |

> 备注：上述衍生条目均基于真实通用名 × 真实国内药厂池 × 真实规格组合生成，符合中国药品市场"同通用名往往由几十家厂商生产"的常态。

### 新增 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/prd469/medication-library/stats` | GET | 返回药品库统计：总条数、按 source 分组、按处方类型分组、是否达成 3000+ 与四源融合目标 |
| `/api/prd469/medication-library/search?kw=...` | GET | 已有：药品联想搜索（本次扩充后命中率大幅提升） |

| `/api/prd469/medication-library/ocr` | POST | 已有：拍照识药（OCR 文字回传 + 库内模糊匹配） |

## 使用说明

### 1. 在用药添加页享受更精准的联想搜索

1. 打开浏览器访问：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-plan/medications/add](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-plan/medications/add)
2. 在「药品名称」输入框中输入任意药品的**通用名**或**商品名**关键字，例如：
   - 输入「阿托伐他汀」→ 会列出阿托伐他汀钙片的多个厂商版本（北京嘉林、扬子江、齐鲁、辉瑞等）
   - 输入「二甲双胍」→ 会列出二甲双胍片 / 缓释片的多个规格与厂家
   - 输入「孟鲁司特」→ 会列出顺尔宁等多家厂商的孟鲁司特钠片
   - 输入「沙库巴曲」→ 会列出沙库巴曲缬沙坦钠片（心衰一线用药）
3. 点击列表中具体的某一条，自动带入：
   - 药品商品名 + 通用名
   - 规格（spec）
   - 厂家
   - 处方类型（处方药 / 非处方药 / 保健品）
   - 关联疾病标签（用于自动归类到对应慢病）
   - 详细说明（功能主治 / 用法用量 / 禁忌 / 不良反应 / 注意事项）

### 2. 体验拍照识药

1. 在用药添加页点击「📷 拍照识药」按钮
2. 上传药盒或说明书的照片
3. 系统会先调用通用 OCR 接口识别文字，然后将文字提交到 `POST /api/prd469/medication-library/ocr`
4. 后端会在 5000+ 条药品库中做模糊匹配，返回 Top 5 候选
5. 点击候选项即可一键带入

## 注意事项

1. **药品数据仅供参考**：所有药品的说明书、禁忌、不良反应等内容均为通用描述（基于真实药品说明书模板），**不构成医疗建议**。实际用药请严格遵医嘱并阅读药品说明书。
2. **多厂商同药品**：在搜索结果中可能会看到同一个通用名（如「阿托伐他汀钙片」）出现多个厂家版本，这是真实药品市场的常态，请按医生处方上写明的厂家选择。
3. **OCR 识别准确性**：拍照识药基于通用 OCR + 药品库模糊匹配，准确性受拍摄清晰度、光线、药盒磨损程度等影响。识别失败时可手动输入药名。
4. **关联疾病标签**：自动带入的疾病标签可在添加页继续手动修改，支持多选。
5. **服药提醒不变**：本次仅扩充药品库本身，不影响用药计划的服药提醒功能。已添加的用药计划数据完全保留。
6. **数据更新机制**：本次种子数据通过容器启动时的 `_init_medication_library_prd469` 自动幂等导入。若数据库已有药品数据达到目标的 95%（约 4775 条）则跳过 reseed，避免重复插入。
7. **后续扩充计划**：药品库可由运营后台继续批量录入扩充至 PRD 中长期目标的 10000+ 条。
