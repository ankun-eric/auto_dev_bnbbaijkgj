## 本次上线内容

本次已将原“用户小程序”和“商家小程序”合并为一个微信小程序，统一账号、登录和基础框架；用户端与商家端的页面、订单、消息、资料仍按角色完全隔离。

本次同时上线了后台商家门店管理、商家账号管理、员工门店授权、商家消息中心、门店选择、角色切换和无权限提示能力。

## 后台准备步骤

1. 登录管理后台，进入 `商家管理 -> 门店管理`，先创建或维护门店信息。
2. 进入 `商家管理 -> 商家账号`，创建商家主账号或员工账号。
3. 如需双身份账号，给同一手机号同时保留普通用户身份与商家身份。
4. 为员工配置可访问门店与模块权限；商家主账号可绑定多个门店。
5. 旧用户账号与旧商家账号不做自动合并，需由后台人工处理后再正式切换。

## 登录与进入规则

1. 未登录用户首次进入时，默认看到用户端首页。
2. 新注册账号默认是普通用户，不会自动获得商家身份。
3. 只有一个身份的账号，登录后会直接进入对应首页。
4. 同时具备普通用户和商家身份的账号，登录后会弹出角色选择框。
5. 双身份账号可在首页或个人中心切换角色，切换到商家端时需要重新选择门店。
6. 没有商家权限的用户进入商家功能时，会进入无权限提示页。
7. 商家单角色账号不能进入用户端功能。

## 商家端使用说明

### 门店选择

1. 进入商家端后，系统会要求先选择本次操作门店。
2. 系统不会记忆上一次门店，下一次进入商家端时仍需重新选择。

### 商家工作台

1. 商家首页显示当前门店概览数据。
2. 扫码/核销入口已并入合并后的小程序工作台。
3. 核销记录页支持按时间范围筛选和分页查看。
4. 商家消息中心与用户消息中心完全分离。

### 角色与权限

1. 商家主账号可管理多个门店。
2. 员工账号按“门店 + 模块权限”控制可见范围。
3. 员工账号也可以同时拥有普通用户身份。

## 关键业务规则

- 用户端和商家端使用不同 `tabbar`。
- 用户端与商家端订单完全独立。
- 用户端与商家端消息完全独立。
- 用户端资料与商家资料完全独立，可使用不同头像和昵称。
- 商家身份只能后台开通，前端不提供自助申请入口。
- 合并版小程序上线后，旧商家小程序作为历史包保留，不再作为主交付入口。
