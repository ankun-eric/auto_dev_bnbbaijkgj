## 功能简介

本次 **v3 全端版** 在 v1（前端工具）+ v2（后端兜底）的基础上，**全端**清扫散写：

1. **后端兜底（v2 已完成，本次复用）**：FastAPI 全局响应类 `PriceFormattedJSONResponse` 拦截所有 datetime 字段，把 naive UTC datetime 强制输出为带 `+00:00` / `Z` 后缀的 ISO 8601 字符串。
2. **H5 / 后台 / 小程序 / Flutter App 散写全量清扫**：把项目内 80+ 个文件、100+ 处散写的 `new Date(变量)` / `.toLocaleString()` / `DateTime.parse(...)` 统一替换为各端 `datetime` 工具中的 `parseServerTime` / `formatDateTime` / `formatFriendlyTime` 等函数。
3. **ESLint 规则防复发**：在 `h5-web/.eslintrc.json` 新增 `no-restricted-syntax` 规则，禁止后续散写 `new Date(变量)` 与 `toLocaleString/toLocaleDateString/toLocaleTimeString`，告警级别为 `warn`。
4. **小程序工具收敛**：`miniprogram/utils/util.js` 中的旧 `formatTime / formatDate / formatRelativeTime` 内部全部委托到 `miniprogram/utils/datetime.js`，旧调用方零侵入。

修复后效果：在中国大陆设备上打开 H5「历史对话」「咨询人切换」「订单时间」等任意页面，新数据的时间显示与北京时间完全一致；后台、小程序、安卓 App、iOS App 同样行为一致。

1. 用浏览器打开 [H5 入口](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)，登录或切换到会员 6399 账户
2. 进入「AI」Tab → 历史对话（或侧边栏的咨询历史列表）
3. 发起一条新对话或进行一次新咨询
4. 查看该新对话的时间显示：**应与当前北京时间一致**，不再偏差 8 小时 ✅
5. 老对话时间可能仍偏差 8 小时（按用户决策，老数据不动）

- **admin-web 后台**：登录管理后台后，进入任意有时间列的列表页（如「优惠券」「订单」「主题配置」「健康检查模板」「商家账号」等），所有时间字段应显示北京时间。
- **微信小程序**：下载 zip → 解压 → 用【微信开发者工具】导入项目 → 编译预览 → 进入「消息」「积分明细」「卡券」「家庭成员」等页面，确认时间显示正确。

## 注意事项

1. **老对话/老订单时间仍可能偏差 8 小时**：按本次决策（不动老数据），仅保证**新产生的数据**时间正确。如需修复历史脏数据，可联系开发同学后续提供数据库迁移脚本。
2. **iOS 未签名 IPA**：未做企业签名或开发者签名，需用 AltStore 等工具自签后安装；签名有效期 7 天，到期需重新签名。
3. **ESLint 规则**：H5 项目中如再次散写 `new Date(变量)` 或 `toLocaleString`，会在 lint 时给出 `[TIMEZONE_V3]` 警告，提醒改用统一工具。
4. **小程序 util.js 兼容**：旧调用 `require('./utils/util').formatTime(t)` 完全兼容，内部已自动委托到统一工具。
5. **后端时间字段格式**：API 出口的 datetime 字段统一为带 UTC 后缀的 ISO 字符串（如 `"2026-05-17T03:30:00+00:00"`）。前端各端工具会自动检测有/无时区后缀，对老接口的裸字符串也会按 UTC 强制解析，**前端展示永远不会偏 8 小时**。
