## 功能简介

本次升级是对 v1.3 的一次**结构性收敛**，没有引入全新功能，目标是把 v1.3 暴露的四类老问题一次性修掉：

| 老问题 | v1.3.1 的解决方案 |
|---|---|
| 「守护中 / 待守护」两个 Tab 老人不秒懂，「待守护」歧义大 | **取消两 Tab，整页一个大列表 + 两区色块卡组（已绑定 / 未绑定）** |
| `/health-profile/v13`、`/i-guard`、`/health-profile` 三个路径混乱 | **统一收敛**：v13 路径自动重定向到新版「我守护的人」 |
| 上限「10 位」写死在代码里，与会员体系脱钩 | 上限 **从 `membership_plans.max_managed` / `free_member_quota.max_managed` 动态读取**，运营可自由配置 |
| 出现「共管 / 代管 / 已拒绝」等专业 / 敏感术语 | **全面术语柔化**：禁用「共管 / 代管」，「已拒绝」柔化为「暂未响应」 |

## 使用说明

### 1. 进入「我守护的人」

打开 H5 主页 → 进入「健康档案」 → 选择「本人」Tab → 点击「我守护的人」卡片
或直接打开链接：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/health-profile/i-guard](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/health-profile/i-guard)

### 2. 页面结构一览

进入页面后从上至下依次为：

1. **顶部统计栏（天蓝色渐变）**
   - 显示：`守护 X 位家人 · 还可邀请 Y 位 / 共 Z 位`
   - 副提示：`本人不占名额`
   - 右侧按钮：`+ 发起邀请`

2. **本人档案卡（固定首位，永远显示）**
   - 头像「我」+ 昵称（本人）
   - 3 个按钮：编辑档案 / 邀请记录 / AI 外呼额度

3. **区域 1：已绑定（X 位）—— 浅天蓝色卡组**
   - 内有所有「已建立守护关系」的家人卡片
   - 按建立时间正序排列（老朋友先）

4. **区域 2：未绑定（Y 位）—— 浅灰色卡组**
   - 内有「邀请中 / 暂未响应 / 已解绑 / 已过期 / 尚未邀请」的家人卡片
   - 卡片右上角小徽标显示子状态
   - 按建立时间正序排列

### 3. 卡片上的操作按钮（按子状态变化）

| 卡片子状态 | 显示的按钮 |
|------|------|
| 已绑定 · 已建立守护 | 查看档案 / 代付明细（主守护人才有） / 解除守护 |
| 已绑定 · 由我代为管理（孤儿档案） | 查看档案 / 移除（4 不可删校验通过时） |
| 未绑定 · 邀请中（还剩 N 小时） | 查看二维码 / 取消邀请 / 邀请记录 |
| 未绑定 · 暂未响应 | 再次邀请 / 移除 / 邀请记录 |
| 未绑定 · 已解绑 | 再次邀请 / 移除 / 邀请记录 |
| 未绑定 · 已过期 | 再次邀请 / 移除 / 邀请记录 |
| 未绑定 · 尚未邀请 | 发起邀请 / 移除 / 邀请记录 |

### 4. 发起邀请

点击顶部的 `+ 发起邀请` 按钮，或卡片上的 `发起邀请` / `再次邀请` 按钮：

- 系统会自动校验配额：当前已占名额 < 配额上限 才可继续
- 若已达上限，弹出「💝 温馨提示」，提示当前 X/X 位已满，并给出「升级会员」入口

### 5. 解除守护

点击已绑定区某张家人卡片的「解除守护」按钮：

- 弹窗提示：`将解除与 XX 的守护关系，TA 的档案数据完整保留`
- 确认后该家人会从「已绑定」移到「未绑定」区

## 配额规则详解

### 上限值来源（动态读取，不再写死）

| 用户类型 | 字段 |
|------|------|
| 普通用户（无会员） | `free_member_quota.max_managed`（默认 3） |
| 会员用户 | `membership_plans.max_managed`（按会员等级取） |

### 占名额口径

| 子状态 | 是否占名额 |
|------|------|
| 本人档案 | ❌ 不占（豁免） |
| 已绑定 · 已建立 | ✅ 占 |
| 已绑定 · 由我代为管理（孤儿） | ✅ 占 |
| 未绑定 · 邀请中 | ✅ 占（24h 后自动 expired 释放） |
| 未绑定 · 尚未邀请 | ✅ 占（防批量建档） |
| 未绑定 · 暂未响应 | ❌ 不占 |
| 未绑定 · 已解绑 | ❌ 不占 |
| 未绑定 · 已过期 | ❌ 不占 |

### 顶部统计公式

- **守护 X 位家人** = 已绑定区卡片数 + 未绑定区卡片数 - 不占名额状态数
- **还可邀请 Y 位** = `max_guardians` - 当前占名额数
- **共 Z 位** = `max_guardians`（即会员等级对应上限）

**举例**：会员上限 9 位，本人 1 + 已绑定 3（含 1 个孤儿）+ 邀请中 1 + 暂未响应 1 + 尚未邀请 2 = 共 8 张档案卡

- 占名额数 = 3 + 1 + 2 = 6
- 顶部文案：「守护 6 位家人 · 还可邀请 3 位 / 共 9 位」
- 副提示：「本人不占名额」

## 注意事项

1. **本人档案永远不占名额**：顶部小字「本人不占名额」即此意，请勿因本人卡片在列表中而误判配额已满。
2. **暂未响应、已解绑、已过期都不占名额**：可以放心保留为「已邀请历史」，方便日后再次邀请。
3. **配额满后**：点击 `+ 发起邀请` 时会弹出温馨提示，提供「升级会员」直达入口。
4. **4 项不可删校验**：当家人卡片满足以下任一条件时，「移除」按钮不显示（沿用 v1.3）：
   - 关系为已绑定（active）
   - 已绑定硬件设备
   - 邀请中状态
   - 存在在途服药计划
5. **路径自动重定向**：旧的 `/health-profile/v13` 路径已自动重定向到「我守护的人」新版，原书签可继续使用。
6. **小程序与 Flutter App**：本次小程序与 App 端只做了术语清理（如「邀请共管」→「邀请守护」），主页面结构未变化，将随下一次发版一并打包发布；如需立即体验完整 v1.3.1 UI，请使用 H5 端。
