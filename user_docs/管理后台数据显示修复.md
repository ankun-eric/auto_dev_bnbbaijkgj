## 三、注意事项

1. **数据为空的情况**：如果某个菜单列表为空，说明该模块暂无业务数据（例如暂无订单），并非系统异常。
2. **搜索功能**：搜索时请输入关键词后按回车或点击搜索按钮，搜索会在当前页面自动刷新结果。
3. **状态操作**：对用户执行"禁用"或"解禁"操作后，列表会自动刷新以显示最新状态。
4. **浏览器兼容性**：推荐使用 Chrome、Edge 等现代浏览器访问管理后台，以获得最佳体验。
5. **登录超时**：如果长时间未操作导致登录超时（Token 过期），系统会自动跳转到登录页面，请重新登录。

## 四、问题修复技术说明

本次修复的核心问题是**前后端字段命名风格不一致**：

- **后端 API** 返回的数据字段使用 `snake_case` 格式（如 `created_at`、`member_level`、`page_size`）
- **前端页面** 期望接收 `camelCase` 格式（如 `createdAt`、`level`、`pageSize`）

修复方案：在前端各个管理页面中增加了**数据映射层**，将后端返回的 snake_case 字段自动转换为前端期望的 camelCase 字段，同时修正了 API 请求参数名称，确保前后端通信一致。
