## 功能简介

本次修复针对一个跨端核心入口 Bug：

**原现象**：在 H5 网页版的 AI 对话主页（`/ai-home`），输入框上方"家庭成员快捷栏"右侧的"**查看档案 ›**"按钮，点击后浏览器跳转到 `/health-records` 这个不存在的页面，立即报 **404**。同样的问题，在微信小程序、安卓 App、iOS App 上以"完善档案"按钮的形式也存在隐患（指向已下线的旧路径 `/health-archive`）。

**修复结果**：所有端的"查看档案 / 完善档案"入口已统一指向新版健康档案页 `/health-profile`，同时数据库中后台保存的旧配置也被迁移脚本自动修正，**用户从 AI 对话主页这条核心入口终于能正常进入健康档案了**。

## 注意事项

1. **H5 网页版**：刷新页面或清浏览器缓存后即可生效，**无需重新下载**。
2. **管理后台**：本次后端启动时自动跑了一次数据迁移脚本，已将历史保存过的旧路径（`/health-records`、`/health-archive`）替换为 `/health-profile`，迁移日志显示 **archive_path 修正 1 条、宫格 target_path 修正 1 条**。
3. **小程序/App**：客户端路由表写在代码里，**必须下载本手册提供的新版本**才能体验修复（旧版本仍会 404）。
4. **管理员特别说明**：管理后台自身的"用户档案查看"页（`/health-records`）与 H5 用户端健康档案是两个独立模块，本次**不受影响**。同样地，后端 API `/api/health-archive/...`（监护人/家庭成员等接口）是接口路径**不是页面路由**，**本次完全不动**。
