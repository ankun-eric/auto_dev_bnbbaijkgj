## 功能简介

当您家人完成体检报告解析、出现异常项目时，系统会通过「贝尼健康」公众号下发模板消息。点击该消息后，您会被引导至 **公众号推送·中转页**，页面会按以下优先级帮您打开异常详情：

1. **优先唤起 App**：如已安装「贝尼健康」App，2 秒内自动唤起，进入 App 异常详情页
2. **降级到小程序**（在微信内）：未装 App 时，提供「在小程序中查看详情」按钮，一键跳转至小程序对应页面
3. **兜底 H5 网页**：最终提供「下载 App」与「在网页查看（功能受限）」两个按钮，确保任何环境都能查看

页面同时会自动上报点击回执（更新推送日志状态为「已点击」）和关键操作埋点（曝光、唤起成功、各按钮点击），方便管理后台跟踪触达效果。

### 核心特性

- ✅ **优先唤起 App**：直接 `location.href` 触发 URL Scheme，不弹系统"打开 App"提示框
- ✅ **2 秒静默降级**：唤起失败后渐显降级按钮，避免闪烁
- ✅ **微信内一键跳小程序**：使用 `<wx-open-launch-weapp>` 官方组件
- ✅ **签名校验防越权**：链接携带 HMAC-SHA256 签名（`sig` + `t`），服务端强制验签
- ✅ **链接失效友好提示**：sig 错误 / logId 不存在 / 解绑老链接均返回友好页
- ✅ **埋点全覆盖**：曝光、唤起成功、降级、各按钮点击共 7 个事件全部上报
- ✅ **微信版本兼容**：自动检测微信 < 6.7.2，隐藏小程序按钮仅展示下载/网页查看
- ✅ **iOS Safari 兼容**：scheme 拦截后静默降级，不报错

## 操作步骤

### 步骤 1：在公众号收到推送消息

当家人体检报告解析完毕且存在异常项时，您会在「贝尼健康」公众号收到一条模板消息：

```
您的父亲张大爷的体检报告有 5 项异常
请尽快查看并联系家人关注健康
```

### 步骤 2：点击模板消息

点击消息后，会进入中转页（页面 URL 形如）：

```
https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/alert-redirect/
  ?logId=12345
  &memberId=678
  &reportId=910
  &t=1716000000
  &sig=a1b2c3d4e5f6g7h8
```

### 步骤 3：等待自动唤起 App（约 2 秒）

页面会显示「正在为您打开 App...」加载提示，并自动尝试唤起 App：

- **如果已安装 App**：浏览器会自动切到 App，落地到 `binihealth://family/alert?...`，您可在 App 内查看异常详情
- **如果未安装 App**：2 秒后会自动进入降级链路（步骤 4）

### 步骤 4：选择降级查看方式

#### A. 在微信内打开（且微信版本 ≥ 6.7.2）

页面会展示三个按钮：

| 按钮 | 行为 |
|------|------|
| 📱 在小程序中查看详情 | 调起微信小程序 `pages/family-alert-detail/index?reportId=...&memberId=...` |
| ⬇️ 下载 App 接收实时提醒 | 跳转到 App 下载页 |
| 🌐 在网页查看（功能受限） | 跳转到 H5 详情页 `/family/alert/{reportId}` |

#### B. 普通浏览器（非微信）

页面会展示两个按钮（无小程序选项）：

| 按钮 | 行为 |
|------|------|
| ⬇️ 下载 App | 跳转到 App 下载页 |
| 🌐 在网页查看 | 跳转到 H5 详情页 |

#### C. 微信版本 < 6.7.2

仅展示「下载 App」+「在网页查看」按钮，并提示：

> 💡 当前微信版本较低，无法直接跳转小程序。请升级微信或在「贝尼健康」公众号底部菜单进入小程序查看。

### 步骤 5：链接失效处理

如发生以下情况，页面会展示友好提示而非空白：

| 情况 | 提示 |
|------|------|
| 链接参数 `sig` 错误（被篡改） | 🔒 链接已失效 |
| `logId` 在数据库中不存在（消息已过期） | 🔒 该消息已失效 |
| 链接缺少必填的 `logId` 参数 | 🔒 链接参数不完整 |

提示页底部会引导用户：「请前往「贝尼健康」公众号底部菜单进入小程序查看最新提醒」。

## 技术细节（管理员/开发者参考）

### URL 参数

| 参数 | 说明 |
|------|------|
| `logId` | `family_alert_log.id`（必填，用于点击回执） |
| `memberId` | `family_member.id`（被守护者档案） |
| `reportId` | `checkup_report.id`（体检报告） |
| `t` | 时间戳（防缓存 + 参与签名） |
| `sig` | HMAC-SHA256 签名前 16 位 hex（防越权） |

### 后端接口

| 接口 | 用途 |
|------|------|
| `POST /api/alert/click-tracking` | 点击回执：将 `family_alert_logs.delivery_status` 由 `sent`/`delivered` 更新为 `clicked`；带 `t+sig` 时强制 HMAC 校验；幂等 |
| `POST /api/alert/verify` | sig 单独校验：前端可在加载时调用，签名失败直接展示 403 友好页 |
| `POST /api/alert/event` | 埋点事件接收：白名单 7 个事件，未知事件返回 400 |

### 7 个埋点事件

| 事件 | 触发条件 |
|------|---------|
| `alert_redirect_view` | 中转页曝光（页面加载完成后立即上报） |
| `alert_redirect_app_launched` | App 唤起成功（`document.visibilityState === 'hidden'`） |
| `alert_redirect_fallback_wechat` | 进入微信降级态（2s 超时未唤起且在微信内） |
| `alert_redirect_fallback_browser` | 进入浏览器降级态（2s 超时未唤起且非微信内） |
| `alert_redirect_click_weapp` | 用户点击「在小程序查看」 |
| `alert_redirect_click_download` | 用户点击「下载 App」 |
| `alert_redirect_click_h5` | 用户点击「在网页查看」 |

### 签名算法

```
raw = "logId={log_id}&memberId={member_id}&reportId={report_id}&t={t}"
sig = hex( HMAC-SHA256(SECRET_KEY, raw) )[:16]
```

服务端复用 `settings.SECRET_KEY` 校验。`memberId` / `reportId` 缺省按 0 计算。

## 注意事项

1. **公众号推送前置条件**：守护者必须**已关注**「贝尼健康」公众号，否则模板消息根本无法下发，自然不会走到本中转页
2. **链接时效**：每条点击链接绑定一个 `logId`，状态机只允许 `sent`/`delivered → clicked` 一次性流转，重复点击同一链接返回幂等响应（`idempotent: true`）
3. **隐私保留策略**：根据 PRD Q-隐私-3，守护关系解绑前下发的旧链接在解绑后**仍可点击查看**详情（不撤回历史推送）
4. **iOS 隐私限制**：iOS 14+ Safari 对 URL Scheme 有静默拦截行为，本中转页采用 2 秒超时静默降级，**绝不弹出"是否打开 App"系统提示**，体验更平滑
5. **微信内务必使用模板消息触发**：直接复制中转页 URL 在微信内打开**不会**唤起 App，因为微信内置浏览器对未知 scheme 的拦截更严格；正常使用路径必须从公众号模板消息进入
6. **签名密钥**：服务端使用 `settings.SECRET_KEY`（生产环境务必通过 `.env` 覆盖默认值），密钥泄露后所有历史链接需重新生成
