### 4. 积分商品详情页（F4）— 看清了再兑

以前在积分商城列表页，点「立即兑换」就直接扣分。现在**改为先进入商品详情页**，看清后再决定。

#### 4.1 进入方式

进入积分商城列表页 → 点击**商品卡片任意位置** → 自动跳到商品详情页。

- 卡片上不再有直接的「立即兑换」按钮（避免误操作）
- 商品详情页路径：
  - H5：`https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/points/product-detail?id=1`
  - 小程序：`pages/points/product-detail?id=1`
  - APP：`/points-product-detail`

#### 4.2 页面包含的内容

自上而下：

1. **商品大图轮播**（支持多图、小圆点分页）
2. **商品名 + 类型标签**（实物 / 优惠券 / 体验服务）
3. **所需积分**（深绿大字）+ 剩余库存
4. **图文详情**（后台运营可用富文本编辑器维护，支持图文混排）
5. **使用须知 / 有效期 / 限制**
6. **底部固定「立即兑换」按钮**

#### 4.3 「立即兑换」按钮的 5 种状态

按钮会智能判断你当前的条件，呈现不同状态：

| 状态 | 触发条件 |
|---|---|
| **已下架**（灰色不可点） | 商品被管理员下架 |
| **已兑完**（灰色不可点） | 库存为 0 |
| **已达兑换上限**（灰色不可点） | 你已经达到"每人限兑 X 次"的上限 |
| **积分不足（差 XX 分）**（灰色不可点） | 你当前积分不够 |
| **立即兑换（消耗 XXX 积分）**（绿色可点） | 一切正常 |

#### 4.4 兑换流程

1. 点击「立即兑换」
2. 弹出二次确认窗口：「确认用 XXX 积分兑换【商品名】吗？」
3. 点击「确认兑换」 → 调用兑换接口
4. 成功 → Toast 提示「兑换成功」 → **自动跳转到「兑换记录」 Tab**
5. 失败 → 直接弹出后端给出的具体原因（例如"积分余额不足"、"库存不足"等），不再是无信息的"操作失败"

## 三、面向运营/管理员（后台）的更新指引

### 1. Bug1 修复：优惠券类积分商品可正常保存

**问题现象（修复前）**：在后台「积分商城 → 商品管理」新增商品，「商品类型」选择**优惠券**，填完积分和库存保存时，页面弹出红字 **"操作失败"**，但无其他信息，完全无法定位。

**修复后**：

- 选择「优惠券」并填写必填字段，可正常保存
- 即使遇到后端 400 报错（例如缺少某个必填项），前端会**完整展示后端错误原因**（如"关联优惠券不能为空"），不再是黑盒的"操作失败"

**后台访问**：  
`https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin`（登录后进入「积分商城 → 商品管理」）

### 2. Bug2 修复：体验服务类与商品库打通

**问题现象（修复前）**：新增商品选择「体验服务」时，「服务类型」是前端硬编码的 4 个选项（专家服务 / 体检 / 中医等），与真实的服务商品库**完全没有关联**。兑换成功后也无法抵扣用户下单的服务订单。

**修复后**：

- 新增 **「关联服务商品」Autocomplete** 下拉，输入关键字自动从商品库（`fulfillment_type=in_store` 到店服务）搜索  
- 选中服务商品后，商品名、图片自动回填（可手动覆盖）
- 用户在积分商城兑换该服务类积分商品 → 我的卡券中会自动生成一张「**XX 服务抵扣券**」，绑定具体服务商品
- 用户下单该服务商品时，系统会识别并应用这张抵扣券
- 兑换记录中的"去使用"按钮，会直接跳到对应的服务商品详情页

### 3. 后台商品编辑器增强

同时为积分商品增加了一系列新字段，运营可更精细地配置：

| 字段 | 作用 |
|---|---|
| **富文本详情 `detail_html`** | 使用 wangEditor 维护图文详情，支持图文混排；终端用户在商品详情页看到此内容 |
| **多图轮播 `images`** | 支持上传多张商品大图，详情页顶部以 Swiper 形式展示 |
| **每人限兑次数 `limit_per_user`** | 0 表示不限；填写正整数则每人最多兑换 N 次，达到上限按钮变灰 |
| **关联优惠券 `ref_coupon_id`** | type=coupon 时必填，对应优惠券表 |
| **关联服务商品 `ref_service_id`** | type=service 时必填，对应 products 表（见 Bug2 修复） |

### 4. 统一错误提示改造

整个积分商城管理页的所有接口 `catch` 块已统一改造：遇到 400/422 等错误时，直接显示后端给出的 `detail` 文案。这意味着未来再有新类型的错误，运营都能第一时间看到具体原因，无需求助技术同学。

## 四、使用数据迁移与兼容性说明

1. **历史积分商品自动迁移**：启动后端时会一次性执行 `points_mall_v31_migration.py`，把过去存在 `description` 字段里的 `ref_coupon_id=xxx; ref_service_id=yyy` 字符串反解析到新字段；同时保留 1~2 个版本的双读兜底，确保老数据无损
2. **老评论回填**：没做数据回刷，靠评论表的 `user_id` 在查询时 JOIN `users` 表实时拿最新头像/昵称；脏数据（无 user_id）继续展示原快照
3. **老链接不下线**：`/points/records`、`/points/exchange-records`、`/points/mall` 继续可访问；「积分明细」页内部也复用这两个数据源

## 六、注意事项

1. **小程序版本**：请确保小程序已更新到最新包，否则可能看不到新增的「积分明细」页和「积分商品详情」页
2. **APP 版本**：Flutter 应用需下载最新 APK / IPA，老版本对新字段已做非空兜底但不会看到 F3/F4 新功能
3. **兑换服务商品**：兑换成功后生成的抵扣券会入账到「我的卡券」，请在下单对应服务商品时主动勾选使用
4. **富文本内容加载**：如商品详情页富文本因为网络问题加载失败，会降级展示商品的 `description` 纯文本
5. **后台创建商品**：再次提醒务必填写所有必填字段（优惠券类需指定 `ref_coupon_id`，服务类需指定 `ref_service_id`），否则会在新错误提示里看到具体说明
