## 功能简介

本次改造将 AI 对话首页（`/ai-home`）的顶栏统一升级为「**☰ + 标题 + 未读徽标 + ⋯**」三段式新版样式，并新增「站内通知未读数红点提醒」能力，让用户进入页面就能第一时间感知是否有新通知。

### 主要变化点

1. **新顶栏强制贴顶展示**：进入 AI 对话首页或开始对话后，新顶栏始终显示在屏幕顶端，后台旧的「topbar 显示开关」不再生效（前端已忽略）
2. **顶栏标题统一为 AI 助手昵称**：标题文字读取后台「AI 对话首页配置 → AI 助手」中的「署名（signature）」字段，运营改一处即可生效全端；如未配置或读取失败，**自动兜底为「小康」**
3. **新增红色未读徽标**：标题文字右上角显示通知中心未读总数（**所有分类合并累加**）：
   - 0 条未读 → 显示**无数字小红点**（直径 8px）
   - 1～99 条 → 显示**具体数字**
   - 100 条及以上 → 显示「**99+**」
4. **点击徽标即可跳转通知中心**，且**不会自动清零**——只有用户在通知中心内逐条已读后才会清零
5. **多端布局结构一致**（H5 / 小程序 / iOS / Android），视觉细节按各端系统规范适配

## 操作步骤

### 1. 进入 AI 对话首页

打开 H5 主页或在小程序 / App 中进入 AI 对话页面，访问路径 `/ai-home`：

- H5 入口：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
- 小程序：底部 Tab 切换到 AI 对话页
- App：点击底部 Tab「AI 对话」

进入后即可看到屏幕最顶端贴顶展示的新版顶栏。

### 2. 查看顶栏布局

```
┌────────────────────────────────────────┐
│  ☰    小康 ⑤                       ⋯  │
└────────────────────────────────────────┘
```

- **左侧 ☰**：点击展开侧边栏，可查看历史会话列表（沿用现状）
- **中间「小康」**：AI 助手昵称（仅展示，不可点击）。文案过长时（超过 8 个汉字）自动截断并加省略号
- **「小康」右上角红色徽标**：未读通知数。`0` 时显示无数字小红点；`1~99` 显示具体数字；`>=100` 显示「99+」
- **右侧 ⋯**：点击展开更多菜单浮层（沿用现状）

### 3. 查看未读消息

如果顶栏右上角出现红色徽标：

1. 点击徽标 → 自动跳转「通知中心」页面
2. 在通知中心**逐条点击已读**或点击「全部已读」清除红点
3. 返回 AI 对话首页时，顶栏会重新拉取最新未读数并刷新徽标显示

> **注意**：跳转到通知中心后**不会自动清零**未读数，必须实际触发已读操作才会减少。

### 4. 修改 AI 助手昵称（管理员操作）

如需修改顶栏标题文案：

1. 登录后台管理 → 「首页设置」→「AI 对话首页配置」
2. 找到「AI 助手」模块的「署名（signature）」字段
3. 修改并保存即可，所有终端的 AI 对话首页顶栏标题会**实时全端生效**

> **说明**：旧的「topbar.title / topbar.visible / topbar.logo」配置项**已被前端忽略**，不再生效。请统一通过 `ai_chat.signature` 配置标题。

## 注意事项

1. **标题截断规则**：顶栏标题超过 8 个汉字自动截断（末尾加 `…`），建议运营在配置时控制在 8 字以内，以保证视觉美观
2. **徽标不挤压标题**：无论未读数是 0 / 5 / 99 / 99+ 等任何位数，标题文字位置始终固定，徽标向右上方延伸，不会挤压标题
3. **数据刷新机制**：未读数仅在**进入页面时**拉取一次；页面停留期间不轮询、不推送。**离开页面再回来视为重新进入**，会再拉一次
4. **接口异常兜底**：未读数接口失败 / 超时时**徽标完全不显示**（既不显示小红点也不显示数字），不会干扰其他功能
5. **未登录态**：未登录用户访问 `/ai-home` 时，徽标不显示，标题正常展示
6. **欢迎区不变**：顶栏下方的「头像 + 早上好，{昵称}！+ 副标题」欢迎区**完全保留**，与新顶栏并存
7. **小程序胶囊按钮兼容**：微信小程序原生右上角胶囊按钮位置已与「⋯」按钮拉开间距，避免遮挡
8. **客户端代码已更新但未重新打包**：本次小程序、iOS / Android App 的代码已同步改造，但本次未触发客户端重新打包流程。如需在小程序 / App 端体验新顶栏，请联系开发同学下次发版时一并打包发布；H5 端无需重新发版，刷新页面即可看到效果

## 接口说明（开发参考）

### 新增接口

**`GET /api/v1/notifications/unread-count`**

聚合返回当前登录用户在通知中心**所有分类**的未读总数（系统消息 + 业务通知 + 公告）。

请求头：`Authorization: Bearer <token>`

响应示例：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "unreadCount": 5,
    "breakdown": {
      "system_messages": 3,
      "notifications": 2
    }
  }
}
```

### 复用接口

- 读取 AI 助手昵称：`GET /api/ai-home-config`，从返回的 `config.ai_chat.signature` 字段中读取
