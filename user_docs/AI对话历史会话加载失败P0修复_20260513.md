## 功能简介

本次修复解决了 H5 端 AI 对话页 **左侧抽屉「历史会话」** 列表加载失败的 P0 线上故障。

**故障现象**（已修复）：用户进入 AI 对话页 → 点击左上角头像打开左侧抽屉 → 历史会话列表区域显示「**加载失败，点击重试**」红色按钮，点击重试仍然失败，导致用户无法查看自己之前的健康咨询记录。

**修复结果**：现在所有账号都能在抽屉中正常加载历史会话列表，包括：
- 有历史对话的账号 → 正常展示完整对话列表（按置顶 / 最近 7 天 / 最近 30 天 / 更早分组）
- 无历史对话的账号 → 友好展示空状态，不再出现「加载失败」红色报错
- 弱网环境下 → 显示加载中或友好提示，不再误报失败

**用户数据 0 丢失**：本次故障仅是接口访问异常，所有历史对话记录均完整保存在数据库中。

## 修复原理（技术说明）

经过线上抓包诊断（详见根因分析），本次故障的真正原因**不在前端 Sidebar 组件**，而在两个层面：

| 层面 | 根因 | 修复 |
|------|------|------|
| **服务器代码同步** | 生产服务器代码停留在 `6fa6a27`（2026-05-09 版本），未拉取 BUG-460/461/462 三个关键修复，导致 backend 实际跑的仍是旧 SQL（`NULLS LAST` MySQL 不支持） → 接口 500 | 服务器 `git pull` 到最新 master，`--no-cache` 重建 backend + h5-web 镜像 |
| **种子数据兼容性** | 拉取最新代码后，启动时 `seed_medication_library` 因数据库已有同名重复药品 → `scalar_one_or_none()` 抛 `MultipleResultsFound` → 启动崩溃 | `drug_crawler.py` 改用 `.scalars().first()` 并加整体容错，启动不再被脏数据阻塞 |

修复后接口 `GET /api/chat-sessions` 在生产稳定返回 HTTP 200。

## 操作步骤（用户视角）

### 步骤 1：登录并进入 AI 对话页

1. 打开 [H5 端登录页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login)
2. 使用手机号登录账号
3. 登录成功后会自动跳转或手动进入 [AI 对话首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)

### 步骤 2：打开左侧抽屉

- 点击页面 **左上角头像**，左侧抽屉会从左侧滑出
- 或者直接从屏幕左边缘 **向右滑动** 也可打开抽屉

### 步骤 3：查看历史会话

抽屉打开后向下滚动即可看到「**历史对话**」区块：

- **有对话的用户**：列表正常展示，按置顶 / 最近 7 天 / 最近 30 天 / 更早 4 组分组，每条会话显示标题、最后一句对话摘要、咨询人胶囊（本人 / 妈妈 / 爸爸等）和最后活动时间
- **无对话的用户**：会展示友好的「暂无历史对话」空状态，**不再出现红色「加载失败」按钮**
- **网络抖动时**：先显示加载中，加载完成或失败后给出友好提示，不会卡在错误态

### 步骤 4：操作历史会话

- **点击会话**：直接进入查看完整对话内容
- **左滑**：触发该条会话的快捷操作（置顶 / 删除）
- **点击右侧 ⋯ 菜单**：弹出更多操作菜单
- **点击右上角「管理」**：进入批量管理态，可勾选多条进行批量删除

## 注意事项

1. **首次访问可能稍慢**：因 backend 镜像刚刚重建，首次冷启动会比较慢（约 5-10 秒），属正常现象
2. **微信小程序 / Flutter App 端**：本次仅修复 H5 端，但因后端接口已修复，**小程序与 App 端的历史会话功能也会同步恢复**（如果之前也有同样故障）
3. **缓存清理建议**：如果在修复后仍看到「加载失败」，请在浏览器中 **强制刷新页面**（Ctrl/Cmd + Shift + R），清除前端 Service Worker 缓存
4. **数据完整性**：所有历史对话数据未受任何影响，0 丢失
5. **接口监控**：修复后 `/api/chat-sessions` 错误率应稳定 < 0.5%
