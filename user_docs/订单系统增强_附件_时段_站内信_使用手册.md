## 1. 商家：营业时间窗 & 并发上限配置

### 入口
管理后台 → 左侧菜单 → **「商家管理」 → 「营业时间&并发上限」**（路径 `/merchant/business-config`）。

### 功能

#### 1.1 选择门店
顶部下拉框选择要配置的门店，下方区域将自动加载该门店当前的设置。

#### 1.2 周营业时间
- 支持周一 ~ 周日 7 天独立配置；
- 每天可设 **多段时间窗**（例如：09:00–12:00、14:00–18:00）；
- 选择「全天休息」即代表当天不营业；
- 表格底部点击「保存营业时间」一键提交。

#### 1.3 日期例外
- 在「日期例外」区域添加单个日期的特殊营业规则；
- 既可以将某日设为休息（节假日歇业），也可以为某日单独设营业时间段；
- 优先级 **高于** 周营业时间。

#### 1.4 商家级并发上限（slot_capacity）
- 「单时段最大并发接单数」适用于该门店下的全部服务；
- 默认 1，可改为任意正整数；
- 该门店在同一时段总订单数不会超过此值。

#### 1.5 服务级并发上限（max_concurrent_override）
- 表格列出该门店下全部服务，可对个别服务覆写并发上限；
- 留空表示沿用门店级 slot_capacity；
- 同时可设置「服务时长（分钟）」用于自动切片。

> 双层并发判定原则：**(已占商家时段数 < slot_capacity) AND (已占该服务时段数 < max_concurrent_override) → 时段可选**

## 2. 用户端：站内消息通知中心

### 入口
H5 → 我的 → 通知中心（路由 `/notifications`）。

### 功能

#### 2.1 红点未读
- 顶部显示未读总数；
- "全部已读"按钮一键清空所有红点；
- 与订单关联的通知会显示订单编号 Tag。

#### 2.2 事件类型标签
通知卡片顶部带有不同颜色 Tag：

| 事件 | Tag 颜色 | 触发场景 |
| --- | --- | --- |
| 订单创建 | 蓝 | 用户下单成功 |
| 已支付 | 绿 | 完成支付 |
| 状态变更 | 蓝 | 商家或系统改变订单状态 |
| 上传附件 | 橙 | 商家上传订单图片/文件 |
| 临近服务 | 红 | 服务开始前 1 小时自动提醒 |
| 已取消 | 紫 | 订单被取消 |
| 已退款 | 紫 | 退款完成 |

#### 2.3 跳转回执
- 点击通知如关联订单，**自动按订单维度清除红点**并跳转至订单详情；
- 系统/健康类通知（不关联订单）则单条标为已读。

#### 2.4 自动调度
- 后端每 5 分钟扫描即将到来的预约（55–65 分钟内开始的），自动发出"临近服务提醒"。

## 3. 用户端：订单取消规则放宽（PRD F10/R7）

- 旧规则：仅 `pending_payment`（待支付）状态可取消；
- **新规则**：`pending_payment` / `pending_appointment` / `appointed` / `pending_use` 状态下，**只要服务时段还没到**，客户都可以一键取消，已支付的订单将立刻置为 `refunded`（已退款）；
- 客户每完成一次取消，系统都会发送一条"已取消"通知到通知中心。

## 4. 商家：订单附件上传

### 限制（按 PRD R2 收紧）
- 单文件上限：**5 MB**（旧 20 MB）；
- 单订单最多 **5 个**附件；
- 支持图片/PDF/常见文档。

### 自动通知
- 商家完成上传后，系统会**自动给订单归属客户发一条站内信**（事件类型：上传附件）；
- 客户在通知中心点击即可跳转到订单详情查看附件。

## 5. 后端 API（增量）

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| POST | `/api/merchant/business-hours` | 保存门店周营业时间 + 例外日期（数组形式） |
| GET | `/api/merchant/business-hours?store_id=` | 获取门店全部时间窗 |
| POST | `/api/merchant/concurrency-limit` | 保存门店级 slot_capacity 与服务级 override |
| GET | `/api/merchant/concurrency-limit?store_id=` | 查询并发上限配置 |
| GET | `/api/services/{product_id}/available-slots?date=YYYY-MM-DD` | 查询某服务在某日的可用时段（含已满置灰） |
| GET | `/api/notifications/unread-count` | 按订单维度返回未读通知统计（用于红点） |
| POST | `/api/notifications/mark-read-by-order` | `{order_id}` 一键清除该订单未读 |
| POST | `/api/orders/attachment-meta` | `{order_ids:[...]}` 批量返回订单附件缩略图与数量，列表展示用 |
