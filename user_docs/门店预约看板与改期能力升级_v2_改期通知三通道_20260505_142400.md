## 功能简介

本次更新（**v2 — 改期通知三通道**）是对前一次「门店预约看板与改期能力升级 v1.0」的能力补齐，重点交付 PRD 中的 **F-11 改期通知三通道全发**：客户在客户端（小程序 / APP / H5）发起改期并成功后，系统会**并行**触发三条通知渠道，确保改期消息能够触达客户：

1. **微信小程序订阅消息** — 通过 `subscribe_message.send` 推送「订单改期通知」模板
2. **APP push** — 通过统一 push 服务（极光 / 个推 / FCM 任选其一）推送
3. **短信** — 通过现有腾讯云 / 阿里云短信服务发送

三通道**并行下发，互不阻塞**：任一通道失败都不会影响其他通道；只有三通道全部失败时，订单详情页会展示「通知发送异常，请联系客户」并将告警信息记录到日志（后续可对接企业微信告警通道）。

## 使用说明

### 一、客户端发起改期（沿用现有规则）

- 入口：进入「我的订单」→ 选择待预约 / 待核销订单 → 点击「改约」按钮
- 第一步：选择目标日期（从明天起，最远 90 天）
- 第二步：选择目标时段（9 段固定切片：06-08 / 08-10 / 10-12 / 12-14 / 14-16 / 16-18 / 18-20 / 20-22 / 22-24）
- 第三步：点击「确认改期」

### 二、改期成功后的三通道通知

改期接口 `POST /api/orders/{id}/appointment` 在返回成功结果时，会**同时**附带本次三通道下发结果：

```json
{
  "message": "预约已确认",
  "status": "pending_use",
  "appointment_time": "2026-05-07T14:00:00",
  "reschedule_count": 1,
  "reschedule_limit": 3,
  "notify_result": {
    "channels": [
      { "name": "wechat_subscribe", "ok": true,  "detail": "订阅消息已下发" },
      { "name": "app_push",         "ok": true,  "detail": "JPush 已下发" },
      { "name": "sms",              "ok": true,  "detail": "短信已下发" }
    ],
    "any_ok": true,
    "all_failed": false
  }
}
```

通知文案（三通道统一）：

> 【XX 健康】您预约的「基础体检套餐」已改期：原 05月06日 10:00-12:00，现 05月07日 14:00-16:00，门店：朝阳门店。如有疑问请联系门店：010-12345678。

### 三、查看改期通知（站内消息）

- **客户端**：H5 / 小程序 / APP 任一端的「我的 → 消息中心」均可看到「您的预约已改期」事件
- **商家端**：订单详情页底部「通知状态」字段展示三通道结果（`已通知` / `通知发送异常`）

### 四、配置三通道凭证

三通道凭证全部从**配置中心 / 环境变量**读取，**未在代码中硬编码**：

| 通道 | 必填环境变量 |
|------|-------------|
| 微信小程序订阅消息 | `WECHAT_MINI_APP_ID`、`WECHAT_MINI_APP_SECRET`、`WECHAT_RESCHEDULE_TEMPLATE_ID` |
| APP push（以极光为例） | `APP_PUSH_PROVIDER=jpush`、`JPUSH_APP_KEY`、`JPUSH_MASTER_SECRET` |
| 短信 | `RESCHEDULE_SMS_TEMPLATE_ID`（沿用现有腾讯/阿里云短信服务的密钥） |
| 通知抬头（可选） | `NOTIFY_BRAND_NAME`（如「XX 健康」） |

> **未配置任何凭证时**：对应通道返回 `ok=false, detail="凭证未配置"`，但不抛异常、不阻塞改期主流程。

## 注意事项

### 改期权限控制

- **改期仅允许客户端调用**：`POST /api/orders/{id}/appointment` 接口仅允许 `customer` 角色调用
- **商家端 PC/H5 无任何改时间入口**：本次升级前已完成审计，所有商家端入口均无「改时间」按钮
- **改期上限 3 次**：单订单最多改期 3 次，超出后按钮置灰、文案改为「改约（已无法改期）」（前端 v1.0 已上线）

### 三通道通知容错

- **任一通道失败不阻塞主流程**：改期接口仍返回 200 + `notify_result`，前端可根据 `notify_result.all_failed` 决定是否在订单详情页显示告警
- **短信通道支持失败重试 1 次**：第一次失败后等待 0.5 秒重试一次（沿用 PRD F-11）
- **小程序订阅消息需用户同步授权**：用户未在小程序内同意接收订阅消息时，该通道返回失败，但其他两通道继续下发
- **APP 未安装则跳过 push**：当 `APP_PUSH_PROVIDER` 为空时，APP push 通道直接跳过

### 兼容性

- **首次填预约日不触发改期通知**：仅当订单已存在预约时间，再次修改时才视为「改期」并触发三通道通知。这与 PRD F-11「改期成功后才发」的语义保持一致
- **核销 / 取消 / 退款不触发改期通知**：本通知仅由改期接口触发，与 `notify_order_cancelled` 等其他事件互不影响
