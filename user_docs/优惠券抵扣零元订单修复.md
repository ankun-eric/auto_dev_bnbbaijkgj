## 功能简介

之前用户在结算页选择优惠券抵扣后，如果实付金额变成 0 元，点击「立即支付」会提示「下单失败」，订单根本无法创建——优惠券全额抵扣 / 免费试用券的核销率几乎为零。

本次修复新增了「优惠券全额抵扣」支付方式归类（枚举值 `coupon_deduction`），并把 0 元单的支付方式字段统一标记为该值。前端三端（H5 / 微信小程序 / Flutter App）的「0 元单旁路」分流逻辑此前已上线，本次后端补全了对账字段写入与数据库枚举扩列：

1. **0 元订单可正常提交**：前端检测到 `paid_amount === 0` 时走 `POST /api/orders/unified/{id}/confirm-free` 而非真实支付通道，后端将订单状态按履约类型推进。
2. **支付方式字段对账友好**：`unified_orders.payment_method` 字段在 0 元单上写入 `coupon_deduction`，财务一条 SQL 即可拿到所有「优惠券全额抵扣」订单。
3. **数据库枚举自动扩列**：项目重启时 `schema_sync` 会自动 `ALTER TABLE unified_orders MODIFY COLUMN payment_method` 把 `coupon_deduction` 加入枚举集合，无需手工迁移。
4. **报表与微信支付订单数解耦**：原本「微信支付订单数」会被 0 元单虚高，修复后只统计真实走渠道的订单。

### 场景 A：H5 用户端使用优惠券全额抵扣下单

1. 在 H5 用户端浏览商品列表，点击商品进入详情页，点击「立即购买」进入结算页 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/checkout](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/checkout)
2. 在结算页点击「优惠券」一栏，选择面值大于等于商品售价的满减券或免费试用券
3. 此时页面底部「实付」会显示「¥0.00」
4. 点击「立即支付」按钮，**修复前**会弹出「下单失败」提示，**修复后**会直接进入「支付成功」页
5. 进入「我的订单」页面，可以看到该订单状态已正确推进（到店履约 → 待使用 / 实物履约 → 待发货 / 上门服务 → 待预约或待使用）

### 场景 B：商家后台查询「优惠券全额抵扣」订单

1. 打开商家管理后台 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)

4. 财务对账时，可使用 SQL `SELECT * FROM unified_orders WHERE payment_method = 'coupon_deduction'` 一次性导出全部记录

### 场景 C：取消优惠券全额抵扣订单

1. 在「我的订单」中找到刚才创建的 0 元订单

4. 取消行为对所有券种统一，免费试用券、满减券回退逻辑相同

## 注意事项

- **优惠券全额抵扣不消耗任何支付通道**，即使所有支付通道（微信 / 支付宝）均关闭，0 元订单仍可正常提交。
- **订单只有 `status = pending_payment` 且 `paid_amount = 0` 时才允许走 0 元单旁路**，其它任何状态调用 `/confirm-free` 接口都会被拒绝（`HTTP 400`）。
- **不能跨用户调用**：`/confirm-free` 接口会校验订单归属，非订单本人将得到 `HTTP 403 forbidden`。
- **重复调用幂等**：第一次调用成功后，订单状态已推进；再次调用同一订单的 `/confirm-free` 会返回 `HTTP 400 invalid_status`。
- **数据库自动迁移**：后端重启会自动执行 `schema_sync`，将 `unified_orders.payment_method` 列扩为 `ENUM('wechat','alipay','points','coupon_deduction')`，**无需 DBA 介入**。
- **历史 0 元订单不会被回填**：本次修复只对新订单生效。如需对历史 0 元订单回填 `payment_method`，可手工执行 `UPDATE unified_orders SET payment_method='coupon_deduction' WHERE paid_amount=0 AND status NOT IN ('pending_payment','cancelled') AND payment_method IS NULL`。
- **报表口径变化**：上线后「微信支付订单数」「支付宝订单数」会从总订单数中扣除「优惠券全额抵扣」单，建议同步更新报表脚本。
- **本次仅后端改动**：H5 / 微信小程序 / 安卓 / 苹果 / Windows 桌面端均**沿用现有版本**，无需重新下载或重新打包。
- **服务端测试**：修复完成后已在远程容器内执行 `pytest tests/test_zero_order_payment_method_v22.py tests/test_h5_pay_link_bugfix.py`，**12/12 全部通过**（4 个新增 + 8 个旧用例）。
- **数据库验证**：远程库 `unified_orders.payment_method` 列类型已确认为 `enum('wechat','alipay','points','coupon_deduction')`。
