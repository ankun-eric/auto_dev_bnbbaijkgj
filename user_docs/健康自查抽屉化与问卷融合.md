## 一、功能简介

本次更新把"老健康自查抽屉"和"新版对话内问卷"两套实现彻底融合成**一套通用问卷架构**，统一收敛到「问卷模板（questionnaire_template）+ AI 功能按钮（chat_function_buttons）」一套架构。
对用户的可感知变化：

| 变化点 | 体验效果 |
|---|---|
| 健康自查回归"底部抽屉"交互 | 点击「健康自查」按钮后，**底部弹出抽屉**（不是把题目插入对话流），一屏多题滚动答题 |
| 按钮可配置"问卷展示形态" | 运营可在管理后台为每个问卷按钮选择 3 种形态之一：抽屉-一屏多题 / 抽屉-一题一屏 / 对话内插入 |
| 题目支持联动 | 选择"头部"部位 → 症状题目只显示头痛/头晕等头部相关；切部位时已选症状自动清空 |
| 提交后图文摘要卡片 | 提交后对话流插入精致的「问卷结果卡片」，含图标 + 字段块 + AI 分析进度提示 |
| 历史记录无变化 | 老健康自查的历史对话仍可正常打开，AI 回复质量保持一致 |

## 二、本次客户端变更

本次更新仅涉及 **H5 端 + 后端 + 管理后台** 改动，**未改动**小程序 / Flutter App / iOS / Android / Windows 桌面端代码。
H5 端代码改动通过容器重建后自动生效，**用户无需重新下载或安装客户端**，刷新浏览器页面即可体验。

> 说明：本次属于 H5 与后端的纯线上更新，没有任何客户端安装包发布，因此手册中也不包含安装包下载章节。

## 三、使用说明

### 终端用户视角（H5 端）

1. **进入 AI 首页**：打开 [AI 首页链接](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)，登录后选择想要使用的咨询档案
2. **点击"健康自查"按钮**：宫格或胶囊条中的「🩺 健康自查」按钮
3. **抽屉从底部上滑弹出**：抽屉中按顺序展示三道题
   - **第 1 题** 不适部位（图标网格，可多选）
   - **第 2 题** 症状描述（会根据第 1 题部位自动过滤）
   - **第 3 题** 持续了多久（单选）
4. **点击底部"提交"按钮**：抽屉自动收起，对话流中插入一张「问卷结果卡片」（标题 + 部位/症状/持续 三个字段块 + "AI 正在为你分析…"提示）
5. **AI 流式回复出现**：紧接卡片后，AI 给出健康建议（基于问卷模板的 prompt）
6. **可以重新填写**：点击结果卡片右上角的 📝「重新填写」按钮，再次弹出抽屉

### 运营视角（管理后台）

#### 创建一个新的"抽屉式问卷按钮"

1. 进入 [问卷模板管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/questionnaire-templates)
2. 点击「新增模板」，填写：
   - **编码 code**：英文唯一标识，如 `sleep_test`
   - **名称**：如「睡眠质量测评」
   - **结果摘要模板**（新字段）：如 `入睡时长：{入睡时长} | 夜醒：{夜醒次数} | 整体：{评分}`，支持 `{题目维度名}` 占位符
3. 点击该模板的「编辑题目」，逐题添加题干、选项、维度（dimension 即占位符）；如需联动可配置「显示条件」和「选项过滤规则」
4. 进入 [功能按钮管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/function-buttons)
5. 点击「新增按钮」，配置：
   - **主类型**：AI 功能
   - **AI 功能子类型**：📝 对话内问卷
   - **关联问卷模板**：选择上一步创建的模板
   - **问卷展示形态**（新字段，三选一）：
     - 🪟 抽屉-一屏多题（默认，**健康自查推荐**）
     - 📑 抽屉-一题一屏（沉浸式体质测评）
     - 💬 对话内插入（轻量级单题）
   - **是否推荐 / 是否胶囊**：决定按钮出现在宫格还是胶囊条
6. 保存后，H5 端用户在 AI 首页立即可以看到这个新按钮，点击后即按所选展示形态弹出问卷

#### 三种"问卷展示形态"对比

| 形态 | 标识 | 适用场景 |
|---|---|---|
| 抽屉-一屏多题 | DRAWER_SCROLL | 题目较少（3-6 题），独立性强（如健康自查） |
| 抽屉-一题一屏 | DRAWER_STEPPED | 题目较多，强调沉浸感，强联动（如体质测评） |
| 对话内插入 | INLINE_CHAT | 轻量级，即时反馈，单题（如快捷确认） |

## 四、注意事项

1. **老健康自查按钮已自动升级**：上线时执行了数据迁移，原有「健康自查」按钮的子类型已升级为 `questionnaire`，并绑定到新的「健康自查」问卷模板，**用户无感知**
2. **问卷模板"来源"标记**：问卷模板列表中现在多了一列「来源」，标识是「系统迁移」（旧健康自查升级）还是「运营新建」
3. **历史对话保留**：用户过往的健康自查对话**不会丢失**，仍可正常查看
4. **联动规则注意**：当用户在症状题选了一些症状后，再去修改部位题，被联动过滤掉的症状会**自动清空**，避免出现"头部 + 胸闷"这种不合理组合
5. **结果摘要占位符**：模板配置中的 `{占位符}` 必须和题目的 `维度（dimension）` 字段一致，否则替换会失败
6. **多端一致性**：本次改动仅在 H5 端启用了新抽屉；小程序 / App 端老健康自查按钮仍走老接口（后端兼容期保留 `/api/health-self-check/*` 接口）
7. **API 接口变更**：
   - 新增 `GET /api/questionnaire/buttons/{button_id}/render-meta` 返回按钮渲染元信息
   - 新增 `POST /api/questionnaire/submit` 抽屉提交（返回结果卡片结构 + AI prompt hint）
   - 老 `/api/health-self-check/*` 接口保留，无破坏性变更

## 五、技术细节（仅供运营/开发参考）

- **数据库变更**：
  - `chat_function_buttons` 加 `questionnaire_display_form VARCHAR(32) DEFAULT 'DRAWER_SCROLL'`
  - `questionnaire_template` 加 `result_summary_template TEXT` + `source VARCHAR(32) DEFAULT 'operator_created'`
  - `questionnaire_question` 加 `display_condition_json JSON` + `option_filter_json JSON` + `layout_hint VARCHAR(32) DEFAULT 'tag_grid'`
- **数据迁移**：上线时自动执行的迁移把老 `body_part_dict.symptoms` 数组写入问卷题目的 `option_filter_json.filter_map`（部位↔症状联动）；把 `health_check_templates.duration_options` 写入持续时间题选项
- **代码备份**：老健康自查的 Controller / Schema / 前端组件副本备份在工作目录的 `legacy_backup/health_self_check/`
- **测试覆盖**：远端 pytest 跑 7 个新增用例，全部通过（含展示形态校验、占位符替换、render-meta 返回、INLINE_CHAT 枚举接收等）

## 六、常见问题（FAQ）

**Q1：点了健康自查按钮没反应？**
A：先检查浏览器控制台是否有 401 错误（需要先登录）；其次确认按钮在管理后台已配置「关联问卷模板」+「问卷展示形态」。

**Q2：抽屉里症状题显示"请先完成上一题"？**
A：这是正确的联动效果——症状题依赖部位题答案。选完部位后症状题会出现可选项。

**Q3：结果卡片显示的字段名是 `{头部}` 而不是真实值？**
A：检查问卷模板的「结果摘要模板」中的占位符名称，必须和题目「维度（dimension）」字段完全一致。

**Q4：管理后台找不到「健康自查」按钮的旧 `button_type=health_self_check`？**
A：上线时已自动升级为 `button_type=ai_function` + `ai_function_type=questionnaire`，按新的方式编辑即可。

**Q5：能否回滚？**
A：可以。代码副本在 `legacy_backup/health_self_check/`；数据库可执行：
```sql
UPDATE chat_function_buttons SET ai_function_type='health_self_check', questionnaire_template_id=NULL, questionnaire_display_form=NULL WHERE ai_function_type='questionnaire' AND questionnaire_template_id=1;
```
