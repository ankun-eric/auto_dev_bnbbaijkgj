## 功能简介

本次更新解决了 AI 对话首页两个体验断层：

### 修复点 1：AI 回复"复制 / 转发 / 语音"操作栏现在会即时显示

**修复前**：与小康 AI 对话，回答流式输出完毕、停止打字后，底部「AI 生成内容仅供参考 + 复制 / 转发 / 语音」这一行**不显示**，需要退出页面再进入、或者去历史记录里才能看到。
**修复后**：流式回答一结束，操作栏立即出现，可直接复制、转发、播放语音，无需任何额外操作。
**影响范围**：普通文字回答、拍照识药、报告解读、健康自查 —— 所有类型的 AI 回复都受益。

### 修复点 2：AI 回复正文不再泄漏"图片裸链接 + 内部协议提示语"

**修复前**：偶尔在 AI 回答正文末尾出现两类不应外露的内容：
1. 长长的图片 URL 文本，如 `https://xiaokang-1323135906.cos.ap-guangzhou.myqcloud.com/images/xxx.png`
2. 内部协议提示语，如 `请参考下面相关附件：\n[附件 xxx.png 已保存到工作目录: .chat_attachments/xxx.png]`

**修复后**：
- 图片 URL **不再显示为文字**，而是渲染为可点击放大的 **80×80 小缩略图卡片**
- 内部协议提示语 **整段过滤掉**，正文干净简洁
- 历史会话里已存在的脏数据也会在前端兜底过滤，看历史记录时也是干净的

1. 用浏览器打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)（手机或 PC 均可）

4. 等待 AI 回答完整出现并停止打字
5. **预期观察**：AI 回答下方立即出现一行：

1. 继续在 ai-home 页面
2. 触发任意会回复图片 URL 的场景（如：让 AI 提供"附件示意图"，或查看已有历史会话里曾包含图片链接的 AI 回复）
3. **预期观察**：
   - 正文里**不会**出现一长串图片 URL 文字
   - 正文末尾**不会**出现「请参考下面相关附件：[附件 xxx 已保存到工作目录…]」之类的提示语

4. 点击任一缩略图：进入全屏图片预览，支持左右滑动查看下一张、双指缩放
5. 在全屏预览状态下**长按图片**：可保存到相册（H5 在微信内 / 浏览器内走原生长按菜单）

### 三、回看历史记录验证

1. 在 ai-home 页面打开侧边栏，进入「历史记录」
2. 任意打开一个含 AI 回复的历史会话

   - 历史的图片 URL 同样渲染为缩略图，仍可正常查看

## 注意事项

1. **后端清洗 vs 前端清洗的关系**
   - 本次同时在 **后端入库前** 与 **前端渲染层** 各做了一道"内部协议附件提示语"清洗。
   - 后端清洗 → 新产生的 AI / 用户消息 `chat_messages.content` 永远干净。
   - 前端清洗 → 兜底覆盖历史脏数据，无需迁移数据库。
2. **图片 URL 识别规则**：识别 `https://…/*.png|jpg|jpeg|gif|webp`（可带 query）和 Markdown 形态 `![](url)` 两种，且自动去重。
3. **复制 / 语音播报内容**：已同步做了 sanitize，不会把「内部协议提示语」误带进剪贴板或 TTS 播报内容。
4. **不影响其他类型卡片**：识药卡片（drug_identify_card）、自查卡片、ChatCard 等结构化卡片走原有渲染路径，本次不修改。
5. **如果缩略图加载失败**：会自动半透明降级（不会显示破图占位），可继续点击重试。
6. **回滚预案**：
   - 前端兜底清洗 + 缩略图渲染可独立回滚（移除 `sanitizeAiContent` 与 `extractImagesFromContent` 调用即可）。
   - 后端 `_sanitize_attachment_hint_safe` 调用点可单独注释关闭。

## 常见问答

**Q1：旧的 AI 回复（数据库中已经存的）会被自动清洗吗？**
A：会，前端渲染层兜底清洗对所有历史消息生效；数据库保持原样不动，避免迁移风险。

**Q2：用户消息中如果包含"请参考下面相关附件…"这种字符串会被过滤吗？**
A：会被后端入库前清洗；但因为必须**三段同时满足**（"请参考下面相关附件" + "[附件 文件名" + ".chat_attachments/路径]"）才匹配，所以用户自然语言里的"请..."完全不会受影响。

**Q3：图片缩略图能保存到相册吗？**
A：能。点击缩略图进入全屏预览后，长按图片即可触发浏览器/微信原生「保存到相册」菜单。

**Q4：如果我用其他对话入口（drug、checkup、chat），本次修复也生效吗？**
A：本次修复聚焦 ai-home 主页面。其他对话页面会因为后端入库层的 sanitize 受益（不再写入脏数据），但前端缩略图渲染暂未铺设到全部入口。
