## 功能简介

本次更新是对宾尼健康 v3.0 的 Bug 修复版本，共修复 10 个问题，覆盖管理后台、守护流程、系统消息和文案优化四个方面。

### 管理后台优化

- **新增设备管理菜单入口**：管理后台侧边栏新增「设备管理」菜单组，包含「设备场景分类」和「设备目录管理」两个子菜单，管理员可以直接在后台管理设备相关功能。

### 守护流程修复

- **解绑后重新邀请恢复正常**：修复了解除守护关系后无法重新发起邀请的问题，现在解绑 → 重新邀请 → 接受邀请的完整流程已恢复正常。
- **接受邀请后系统消息角色修正**：修复了接受邀请后双方收到的系统消息角色错乱问题，现在守护方和被守护方收到的消息文案各司其职。
- **解绑后双方收到系统消息**：补充了解除守护关系后双方的系统消息通知，操作人和对方都能在系统消息列表中看到解绑记录。

### H5 前端体验优化

- **邀请错误页提示更清晰**：当您重复接受邀请、邀请已过期或已取消时，页面会显示不同的标题和描述，让您一眼就知道发生了什么。
- **解绑验证码流程提示区分角色**：管理方解除守护和被守护人移除守护人时，确认提示和操作指引会区分角色，明确告知操作后果。
- **系统消息完整展示**：系统消息列表中的消息内容不再被截断显示省略号，您现在可以完整阅读每一条系统消息。

### 文案统一优化

- 全文统一将「健康数据」「健康提醒」等表述修正为「健康档案」，与产品定位保持一致。

## 使用说明

### 一、管理后台 — 设备管理

1. 打开管理后台：[https://6b099ed3-7175-4a78-91f4-44570c84ed27.noob-ai.test.bangbangvip.com/admin](https://6b099ed3-7175-4a78-91f4-44570c84ed27.noob-ai.test.bangbangvip.com/admin)
2. 使用管理员账号登录（`13800000000` / `admin123`）
3. 在左侧侧边栏找到「设备管理」菜单
4. 展开后可看到两个子菜单：
   - **设备场景分类**：管理四大设备场景（安全守护、健康守护、日夜守护、其他）
   - **设备目录管理**：管理各场景下的设备目录，支持编辑跳转链接和图标

### 二、守护流程 — 邀请与管理

#### 邀请家人加入守护

1. 在 H5 用户端进入「家庭成员」或「健康档案」页面
2. 点击「添加成员」或「发起邀请」，输入对方手机号
3. 对方将收到邀请链接，点击后进入 family-auth 页面
4. 如果对方已是您的家庭成员，页面会显示 **「您已在守护关系中」**
5. 如果邀请已过期，页面会显示 **「邀请已过期」**，请让对方重新发送

#### 解除守护关系

**管理方解除守护（入口 B）**：
1. 进入「健康档案」→「家庭成员列表」
2. 点击要解除的成员旁的解除按钮
3. 确认框会提示：「解除守护后，您将无法管理 {成员姓名} 的健康档案」
4. 输入短信验证码确认
5. 解绑成功后，双方都会在系统消息中收到通知

**被守护人移除守护人（入口 C）**：
1. 进入「健康档案」→「我的守护人」
2. 点击要移除的守护人旁的移除按钮
3. 确认框会提示：「移除后，{守护人姓名} 将无法管理您的健康档案」
4. 输入短信验证码确认
5. 移除成功后，双方都会在系统消息中收到通知
### 三、系统消息

1. 在 H5 用户端底部导航进入「消息」页面
2. 系统消息列表中每条消息的内容都会**完整展示**，不再出现末尾省略号
3. 接受邀请后的消息：
   - **守护方**会看到：「XXX 已同意加入您的家庭成员，成为您的守护对象」
   - **被守护方**会看到：「您已同意 XXX 的守护邀请，对方可管理您的健康档案」
4. 解绑后的消息：双方都会收到「守护关系已解除」的通知

## 注意事项

1. **解绑后重新邀请**：解绑操作会立即生效，如需重新建立守护关系，需要重新发起邀请并等待对方接受。
2. **短信验证码**：解除守护关系需要短信验证码，验证码 5 分钟内有效。
3. **系统消息保留**：所有守护关系的变更（邀请接受、解绑）都会在系统消息列表中保留记录。
4. **设备管理入口**：仅管理员可以访问设备管理功能，普通用户看不到该菜单。
5. **管理后台访问**：管理后台需要管理员账号登录，默认账号密码见上方访问链接章节。
