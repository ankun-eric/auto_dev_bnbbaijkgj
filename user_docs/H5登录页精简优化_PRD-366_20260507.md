## 一、功能简介

本次更新对 **H5 客户端登录页** 进行了视觉精简，让登录流程更聚焦、更清爽：

- **去除了**手机号验证码下方的灰色提示文字「未注册手机号验证后将自动创建会员 / 当前未开放自助注册，仅支持已注册账号登录」——这条提示对真实用户帮助有限，反而让登录区显得拥挤。
- **去除了**登录页底部「其他登录方式」整块（含微信渠道卡片）——因实际并未真正提供其他登录通道，点击后只会弹「暂不支持」类提示，属于伪功能，给用户造成误导。
- **保留了**所有真正必要的元素：手机号输入、验证码输入、获取验证码按钮、登录按钮、用户协议勾选、邀请码识别行（携带 `?ref=xxx` 时显示）。

最终效果：登录页视觉更干净，登录主动作（输入手机号 → 获取验证码 → 登录）成为唯一视觉主线，提升首屏转化体验。

## 三、使用说明

### 3.1 进入登录页

1. 用手机浏览器（或在微信内）打开 [H5 登录入口](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login)
2. 看到精简版登录卡片：顶部 LOGO + 「宾尼小康」标题 + 「AI 健康管家 · 您的私人健康助手」副标
3. 卡片内只剩 4 个核心元素：手机号输入框、验证码输入框（含「获取验证码」按钮）、登录按钮、协议勾选

### 3.2 登录主流程（保持不变）

1. **输入手机号**：在第一个输入框输入 11 位手机号（前缀已固定为 `+86`）
2. **获取验证码**：点击右侧「获取验证码」按钮，短信验证码会发送到该手机号；按钮进入 60 秒倒计时
3. **输入验证码**：将收到的 6 位验证码填入第二个输入框
4. **勾选协议**：勾选下方「我已阅读并同意《用户服务协议》和《隐私政策》」复选框
   - 若未勾选直接点登录，会弹出二次确认弹窗，点击「同意并登录」即可
5. **点击「登录 / 注册」按钮**：
   - 已注册手机号 → 提示「登录成功」并进入首页
   - 未注册手机号且后台开启自助注册 → 自动创建会员账号，提示「注册成功！您的会员卡号: xxxx」并进入健康指引页
   - 未注册手机号且后台关闭自助注册 → 弹出「无法注册」提示，建议联系管理员开通账号

### 3.3 邀请码场景（保留）

如果通过邀请链接打开（URL 携带 `?ref=邀请码`），登录卡片中部会出现一行**绿色提示**：

```
🎉 已识别邀请码：DEMO123
```

注册时该邀请码会自动绑定上下级关系。可用 [带邀请码的演示链接](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login?ref=DEMO123) 体验。

## 四、注意事项

### 4.1 用户侧需注意

| 场景 | 处理方式 |
|------|----------|
| 看不到「未注册手机号会自动创建会员」提示 | 这是本次优化的**预期行为**，提示行已删除。是否能注册由后台开关决定，登录时按接口反馈即可 |
| 看不到「微信登录」入口 | 这是本次优化的**预期行为**。当前版本仅支持手机号 + 短信验证码登录，第三方登录方式如有需要请联系管理员 |
| 输入未注册手机号点登录 | 后台**开启自助注册**时自动建会员；后台**关闭自助注册**时弹「无法注册」提示，请联系管理员开通账号 |
| 未勾选协议直接点登录 | 弹出二次确认弹窗，点击「同意并登录」即可继续；点击「再看看」则停留在登录页 |
| 邀请码识别行不显示 | 确认 URL 是否带了 `?ref=xxx` 参数；不带参数时本身就不显示，是预期行为 |
| 横屏 / iPad 大屏访问 | 登录卡片正常居中显示，下方不再有「其他登录方式」残留区域 |

### 4.2 兼容性说明

| 维度 | 已验证 |
|------|--------|
| 操作系统 | iOS 12+ / Android 8+ |
| 浏览器 | 微信内置浏览器、Safari、Chrome、各 App WebView |
| 屏幕方向 | 竖屏 / 横屏 |
| 屏幕宽度 | 375px ~ 430px 主流手机尺寸 |

### 4.3 业务规则不变

- 后端 `RegisterSettings` 注册开关业务能力**完全保留**：自助注册开/关的实际业务规则不变，只是**前端不再静态展示**对应文案
- 接口契约不变，与登录相关的 `/api/auth/sms-code`、`/api/auth/sms-login`、`/api/auth/register-settings` 等端点行为不变
- 邀请码逻辑、协议勾选拦截逻辑、图形验证码与短信风控策略均不变
