## 功能简介

**PRD-442 菜单模式 · 晴空诊室风格改造** 是 bini-health 全端视觉规范统一升级项目，本次交付完成了：

1. **菜单模式 11 级天蓝色阶 Design Tokens**：与 AI 模式（PRD-441）色板完全统一，避免新老用户在 AI 模式与菜单模式间切换时的视觉割裂感。
2. **6 屏核对版原型**：登录页 / 菜单首页（九宫格） / 菜单聊天页 / 健康档案 / 历史会话抽屉 / 个人中心 — 严格按设计稿落地，可直接打开浏览器查看像素级效果。
3. **13 类通用组件 demo**：Button / Card / Input / Tab / Drawer / Toast / Modal / List / Chip / Avatar / Badge / Skeleton / Empty —— 一份独立 HTML 单页，开发与设计可对照实现。
4. **三端共用 Design Tokens**：H5 端 `design-tokens.css`、小程序端 `menu-mode-tokens.wxss`、Flutter 端 `menu_mode_theme.dart`（含 ThemeData + 4 个常用 Widget），三端共用同一组 11 级色阶值。
5. **完整 PRD 文档**：9 章节涵盖背景目标、落地范围、Tokens、组件库、三端拆解、灰度发布、验收标准、风险清单。

> **本次为视觉规范基线交付，零业务逻辑改动**，现有所有 H5 / 小程序 / Flutter 业务页面行为完全不变；后续将通过单独 PRD 任务把 6 屏改造逐端落地。

## 注意事项

### 关于灰度发布
本次为「视觉规范基线」交付，未在生产环境开启全量灰度。后续 6 屏与二级页落地完成后，将按 PRD-442 §6 灰度方案分四档发布：内测 0.1% → 灰度 1（10%）→ 灰度 2（30%）→ 全量。

### 关于「晴空诊室」风格
- **统一性**：菜单模式与 AI 模式（PRD-441）共用同一 11 级天蓝色阶（hex 完全一致），新老用户切换无视觉割裂感。
- **病历卡感**：所有 AI 输出/重要信息块统一为「白底 + 左侧 3px sky-400 竖线 + 4 级天蓝色阴影」，强化医疗专业感。
- **渐变三层级**：A1 顶栏（最浅）→ 主按钮（中段）→ Hero（最深），形成清晰视觉层级。
- **中老年友好**：正文最小 14px，关键指标 36px，对比度通过 WCAG AA 级。

### 关于辅助色严格控量
- ✅ `--success` 仅用于：在线状态 / 成功 Toast / 达标徽章
- ✅ `--danger` 仅用于：未读红点 / 错误 Toast / 危险操作 / 严重告警
- ✅ `--member` 仅用于：会员金徽 / 会员标签 / 付费等级
- ❌ 严禁滥用强饱和度色，避免破坏整体晴空感

### 关于本期范围
- ✅ 本期交付：Tokens + 6 屏原型 + 13 类组件 demo + PRD 文档
- ❌ 本期不做：暗黑模式、6 屏在线上业务页面落地、二级页改造（注册/家人管理/订单详情等）、业务逻辑变更
- 📅 后续 PRD-443+ 将分批落地 H5 / 小程序 / Flutter 三端的 6 屏与二级页

### 关于浏览器兼容性
- 推荐 Chrome / Edge / Safari 最新版本
- 移动端推荐 iOS Safari、Android Chrome、微信内置浏览器
- IE 不再支持
