## 功能简介

本次更新修复了**顾客订单改约时弹窗不会自动关闭、且返回订单列表后看到的还是旧时间**的痼疾问题。

修复前（Bug 行为）：

- 顾客点【确认改约】→ 接口已经成功 → Toast 闪一下消失 → ❌ **改约弹窗不关闭，遮罩一直挡着屏幕，必须手动点遮罩才能关掉**
- 关掉弹窗后回到订单列表，看到的还是**旧时间**，必须**下拉刷新**才能看到新时间

修复后（预期体验）：

> 顾客点【确认改约】→ 弹窗瞬间关闭 → 屏幕中央出现「改约成功」轻提示 → 看到订单详情页时间已变 → 提示自动消失 → 返回列表也是新时间。**整个过程不需要任何手动操作。**

本次修复在三个顾客端（H5 / 微信小程序 / Flutter App）**同步落地、一次性上线**，行为完全一致。

## 使用说明

### 场景 1：顾客通过订单详情页改约

1. 登录顾客账号，进入【我的订单】列表页
2. 点击任意一个**待核销 / 已预约 / 部分核销**状态的订单进入详情页
3. 点击【改约】按钮 → 弹窗弹出
4. 选择新的预约日期与时段，点击【确认改约】
5. **接口成功后**：
   - ✅ 改约弹窗**瞬间关闭**（不需要再点击遮罩）
   - ✅ 屏幕中央出现「改约成功」Toast，按默认时长（约 1.5 秒）显示完毕后自然消失
   - ✅ 订单详情页时间立即更新为新时间
6. 点击返回 / 系统返回到订单列表页
7. **列表中该订单卡片显示的预约时间已是新时间**，无需手动下拉刷新

### 场景 2：顾客通过订单列表「改约」快捷入口改约

1. 在【我的订单】列表，找到要改约的订单卡片
2. 直接点击该卡片右下角的【改约】按钮（H5 端、Flutter App 列表中的快捷按钮）
3. 跳转到详情页 + 自动打开改约弹窗
4. 选择新的日期与时段、确认 → 与场景 1 完全一致

### 场景 3：失败兜底（行为不变）

- 如果改约接口返回失败（如时段已过期、达到改期上限等）：
  - ❌ **弹窗不关闭**（保持原内容，方便重选）
  - 🔔 Toast 显示具体失败原因（如「今日剩余时段已过」「本订单已达改期上限」等）
  - 顾客可以重新选择或点击遮罩取消

## 注意事项

1. **改约失败时弹窗不关闭是预期行为**：失败场景需要给顾客重选机会，弹窗会保留并通过 Toast 提示具体失败原因（如时段已过期、达到改期上限）。
2. **可改约状态**：改约功能仅对「待核销 / 已预约 / 部分核销 / 待预约」状态的订单可用，其他状态（已完成、已取消、退款中等）不会显示改约入口。
3. **改约次数上限**：每个订单的可改约次数由商品配置决定（一般为 3 次），达到上限后改约按钮置灰，详情页弹窗会显示「本订单还可改期 X 次」。
4. **改约可选范围**：改期可选范围为「明天起 90 天内」（不含今天）。
5. **服务器时间双保险**：改约接口会根据**服务器时间**校验时段是否过期，即便手机本地时间被人为调快，已过去的时段仍会被后端拒绝，并返回 `RESCHEDULE_TIME_EXPIRED`。
6. **列表自动刷新机制**：
   - **H5**：通过浏览器的 `pageshow` / `visibilitychange` / `focus` 事件 + `localStorage` 标志触发，**用户从详情页 BFCache 返回 / 切回标签页 / 浏览器后退**都能正确触发刷新
   - **小程序**：通过 `globalData.unifiedOrdersNeedRefresh` 标志，列表页 `onShow` 时消费
   - **Flutter App**：通过全局 `UnifiedOrdersRefreshNotifier`（基于 Flutter `ValueNotifier`），订单列表订阅其变化自动刷新
