# AI 对话首页白屏与会话 422 修复（Bug-419）

> 修复日期：2026-05-08  
> 关联：PRD-411（AI 对话首页 Tab 化）/ PRD-414（AI 对话页优化 v1.1）  
> 类型：线上紧急修复（一击到底，不留尾巴）

---

## 1. 问题现象

PRD-411 上线后，H5 用户端出现两个并发线上故障：

1. **现象 A：AI 对话首页大面积白屏**  
   进入「AI 对话」首页（`/ai-home`）后，欢迎语、健康小贴士、九宫格功能区、推荐问题、底部输入框等整个页面区域不可见，仅留下顶部状态栏。

2. **现象 B：点击发送 → "创建会话失败" 提示**  
   用户在 AI 首页随意输入文字点击发送，提示「创建会话失败，请检查网络或登录状态」，浏览器控制台出现 **422 Unprocessable Entity**。

### 根因（一句话总结）

- **后端**：`POST /api/chat/sessions` 把 `session_type` 标成必填，但缺少友好兜底，遇到任何字段疏漏直接 422，且错误信息为英文，难以排查。
- **H5 ai-home**：历史代码请求体里把 `family_member_id` 错写成了 `member_id`，并完全漏掉 `session_type`；同时整页未做防御性渲染，配置接口任一字段缺失就会让整个 React 子树卸载，呈现整屏空白。

---

## 2. 修复内容

### 后端（4 项加固）

| 编号 | 改动 | 文件 |
|---|---|---|
| **B-1** | 全局 422 错误中文化处理器（保留 `errors[]` 兼容前端 axios） | `backend/app/main.py` |
| **B-2** | `session_type` 改为可选并提供 `health_qa` 兜底；建立 `general/qa/chat/...` 等别名→标准枚举值映射 | `backend/app/schemas/chat.py`、`backend/app/api/chat.py` |
| **B-3** | `family_member_id` 缺省时自动取用户「本人 (`is_self=True`)」家庭成员，没有则取最早创建的一条 | `backend/app/api/chat.py` |
| **B-4** | 关键路径增加 INFO 日志：兜底命中、字段映射、最终落库参数全部打日志 | `backend/app/api/chat.py` |

> 同时显式接收旧字段 `member_id` 并自动归一化为 `family_member_id`，老客户端版本无需重发即可继续可用。

### H5 用户端（11 项加固）

| 编号 | 改动 | 文件 |
|---|---|---|
| **H-1** | `ai-home` 创建会话字段对齐：`session_type='health_qa'`、改用 `family_member_id` | `h5-web/src/app/(ai-chat)/ai-home/page.tsx` |
| **H-2/3** | 抽取统一工具 `createChatSession(payload, options)`，所有入口必须经过此工具 | `h5-web/src/lib/chat-session.ts`（新增） |
| **H-4** | 新增 `SectionErrorBoundary` 组件，区块级错误隔离，错误时回退到空占位而不是整页白屏 | `h5-web/src/components/SectionErrorBoundary.tsx`（新增） |
| **H-5** | `ai-home` 关键 6 个区块（welcome/health_tips/func_grid/recommend/empty_placeholder/floating_button/topbar）全部包裹 `SectionErrorBoundary` | `ai-home/page.tsx` |
| **H-6** | `aiHomeConfig` 全部读取改为 `obj?.field ?? defaultValue` 安全访问 | `ai-home/page.tsx` |
| **H-7** | 新增内置默认配置 `FALLBACK_CONFIG`，`/api/ai-home-config` 失败/超时也保证 UI 不空 | `ai-home/page.tsx` |
| **H-8** | 配置加载改为「按一级 key 深合并」(`mergeWithFallback`)，避免接口返回部分字段时把内置默认整段抹掉 | `ai-home/page.tsx` |
| **H-9** | 体质（tcm）入口的 `session_type` 由 `'constitution'` 修正为合法枚举 `'constitution_test'` | `h5-web/src/app/tcm/page.tsx` |
| **H-10** | 症状（symptom）、对话侧边栏（ChatSidebar）、Tab AI 入口三处全部切换到统一工具 | `symptom/page.tsx`、`components/ChatSidebar.tsx`、`(tabs)/ai/page.tsx` |
| **H-11** | 工具内部 `try/catch` 兜底 + `Toast` 提示，**不再向上抛异常**，从根本上杜绝同类整页崩溃 | `lib/chat-session.ts` |
