## 使用说明

### 一、微信小程序支付流程

#### 用户端（小程序）
1. 用户在商品下单页面提交订单
2. 选择「微信支付」作为支付方式
3. 系统调用微信支付 JSAPI 接口，弹出微信支付收银台
4. 用户完成支付（输入密码/指纹）
5. 支付成功后，微信服务器异步通知后端，订单状态自动更新为「已支付」

#### 管理端配置
1. 登录管理后台，进入「支付配置」页面
2. 找到「微信小程序支付」通道，点击编辑
3. 填入以下必填字段：
   - 小程序 AppID
   - 商户号（mch_id）
   - API V3 密钥
   - 商户证书序列号
   - 商户私钥（PEM格式）
4. 保存后点击「测试连接」，系统会调用微信支付 API 验证联通性
5. 测试通过后，点击「启用」开关

### 二、微信支付异步回调

系统自动处理微信支付结果通知，无需人工干预。回调路由为：
`POST /api/pay/notify/wechat_miniprogram`

系统会自动：
- 验签（使用微信平台证书公钥）
- 解密回调报文（AEAD_AES_256_GCM）
- 更新订单支付状态
- 幂等处理（同一订单号重复回调不重复处理）

### 三、退款操作流程

#### 用户申请退款（小程序/H5）
1. 在订单详情页点击「申请退款」
2. 选择退款原因（如：商品质量问题、不想要了等）
3. 上传凭证（可选）
4. 系统自动校验订单是否在可退款期限内（支付后 15 天内）
5. 提交申请后等待管理员审核


#### 管理员审核与退款（管理后台）
1. 登录管理后台，在侧边栏点击「退款管理」
2. 在退款列表查看所有退款申请，包括：
   - 订单编号、用户信息
   - 退款金额、退款原因
   - 申请时间、退款状态（待审核/已通过/已驳回/退款成功/退款失败）
3. 点击「审核」查看退款详情
4. 审核通过后，在操作界面：
   - 选择「全额退款」：退还订单全部实付金额
   - 选择「部分退款」：输入退款金额（不能超过订单实付金额）
   - 点击「确认退款」，系统调用微信支付退款 API
5. 退款成功后：
   - 订单退款状态更新为「退款成功」
   - 订单关联的所有核销码全部作废
   - 退款金额和退款单号记录到退款记录中

#### 退款失败处理
- 若退款 API 调用失败（如商户余额不足），后台会显示具体失败原因
- 管理员可手动重试退款
- 退款失败不会影响订单状态，可反复重试直到成功

### 四、支付宝 H5 退款

管理与操作流程与微信支付退款完全一致：
- 管理员审核 → 选择全额/部分退款 → 调用支付宝退款接口 → 更新订单状态 → 核销码作废
- 退款失败同样显示原因，支持手动重试

## 注意事项

1. **退款时限**：仅支持支付后 15 天内的订单申请退款，超期的订单会提示「该订单已超过可退款期限（支付后 15 天），无法申请退款」
2. **核销码处理**：退款成功后，订单下所有核销码将全部作废（一刀切），即使只退了部分金额
3. **商户余额**：微信支付退款需要商户账户有足够余额，余额不足时退款会失败，需充值后重试
4. **部分退款**：部分退款金额不能超过订单实付金额
5. **平台证书**：系统启动时会自动获取微信支付平台证书，证书过期前会自动更新
6. **敏感信息安全**：商户私钥、API V3 密钥等敏感信息使用 AES-256-GCM 加密存储，编辑时掩码显示
