## 功能简介

本期上线的"卡功能 - 第 1 期"主要包含三大块：

1. **平台运营 admin 端**：可在「管理后台 → 商品体系 → 卡管理」中**创建、编辑、上下架、删除**卡定义。
2. **C 端用户 H5 端**：可在「卡专区」浏览所有已上架的卡，进入"卡详情页"查看完整规则；登录后可在「我的卡包」查看自己持有的卡。
3. **后端 API 完整骨架**：4 张数据表 + 5 个枚举 + 30+ API 路由，已通过 17 个 pytest 单元测试 + 7 项生产环境 E2E 测试，零 Bug。

> 📌 **本期边界**：本期不包含「购卡下单」、「双确认核销」、「销售数据看板」、「拆 2 单订单结构」、「续卡逻辑」、「商品详情页可用卡推荐位」、「支付页购卡引导」、「卡分享海报与邀请奖励」等功能。这些将在第 2~5 期陆续推进，本期为整套卡体系的**地基与可视化基础**。

## 平台运营（admin）使用说明

### 1. 进入卡管理页面

1. 用管理员账号登录管理后台：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/login](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/login)
2. 左侧菜单选择 **商品体系 → 卡管理**
3. 即可看到所有卡的列表（顶部支持按卡名搜索 + 按状态过滤）

### 2. 新建卡

1. 在卡管理页面右上角点击「新建卡」按钮
2. 填写以下必填字段：
   - **卡名**：例如"美业 10 次卡"
   - **卡形态**：次卡（共享次数池） 或 时卡（按时间段）
   - **归属范围**：平台通用 / 商家专属（专属时需填商家档案 ID）
   - **售价 / 划线价**：售价必填，划线价用于详情页"原价"展示
   - **总次数**（仅次卡）：例如 10
   - **频次（仅时卡）**：例如"每天 1 次"或"每周 3 次"
   - **有效期**：自购买成功起算 N 天（默认 365 天）
   - **库存**：留空 = 无限发售
   - **同款限购**：留空 = 不限
   - **续卡策略**：叠加在原卡 / 新发独立卡（暂存策略，第 4 期生效）
   - **可用门店**：本期仅支持「全门店通用」，「指定门店列表」将在第 2 期开放
   - **卡内项目**：从已有的普通商品中勾选作为本卡可核销的项目（必须先绑定后才能上架）
3. 点击「保存」后，卡进入「草稿」状态，C 端不可见

### 3. 上架 / 下架卡

- 在列表中点击行末的「上架」按钮 → 卡状态变为 active，C 端可见
- 已上架卡可点击「下架」按钮 → 卡状态变为 inactive，C 端不再展示但用户已购买的卡仍正常使用
- 仅「草稿」状态的卡可以删除；已上架/下架的卡不可删除（数据安全保护）

### 4. 编辑卡内项目

- 在已存在的卡上点击「编辑」可修改卡的所有字段，包括卡内绑定的项目
- ⚠️ **修改卡内项目仅对新购卡生效，老卡按购买时项目快照执行**（已写入数据模型，第 2 期购卡时会自动生成快照）

## C 端用户使用说明

### 1. 浏览卡列表

1. 在 H5 上访问：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/cards](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/cards)
2. 列表展示所有已上架的卡，每张卡展示：
   - 紫色"次卡/时卡"角标
   - 平台通用 / 商家专属标签
   - 卡名、售价、划线价、有效期、可用项目数、已售量

### 2. 查看卡详情

- 在卡列表点击任意卡片进入卡详情页
- 详情页展示：卡名、价格、形态描述、卡介绍、可用项目列表、使用规则
- 如果用户已持有该卡：会以黄色提示条显示"您已持有此卡（剩余 N 天到期）"
- 当持卡剩 ≤ 7 天时，提示"可到期续卡"（续卡功能在第 4 期开放）
- 底部「立即购卡」按钮目前会提示"购卡功能将在第 2 期开放"

### 3. 我的卡包

1. 登录 H5 后，访问：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/cards/wallet](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/cards/wallet)
2. 顶部 4 个 Tab：
   - **全部**：所有持卡（含活跃/过期/已退）
   - **未使用**：刚购买、剩余次数 = 总次数的次卡
   - **使用中**：已使用过至少 1 次但未用完的卡
   - **已过期**：超过有效期的卡（系统会自动懒迁移过期状态）
3. 每张持卡展示：卡名、剩余次数 / 总次数、有效期、剩余天数、卡形态
4. 点击卡片可跳转回卡详情页

> 📌 第 1 期通过开发者后台直接植入测试卡来体验"卡包"功能；正式购卡链路在第 2 期开放后会自动填充用户的卡包。

## 注意事项

1. **本期为骨架版本**：购卡下单尚未开放，详情页底部的「立即购卡」按钮仅作展示用途，点击后会提示"将在第 2 期开放"。
2. **测试数据**：admin 创建的卡只有上架后才会出现在 C 端列表中；草稿状态的卡仅 admin 可见。
3. **过期懒迁移**：当用户访问"我的卡包"时，系统会自动把已超过 valid_to 但状态仍为 active 的卡刷为 expired，确保统计数据准确。
4. **权限隔离**：所有写接口（创建/编辑/上下架/删除）严格限定 admin 角色，普通用户访问会返回 403。
5. **退款规则**：本期未实现退款，预计第 3 期接入现有 unified_orders 退款流程（未使用全额退、已使用不退）。
