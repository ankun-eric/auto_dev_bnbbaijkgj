## 三、详细操作步骤

### 1. BUG1：上传头像 / 管理员配置 OSS

**普通用户：**
1. 打开 H5「我的」页 → 点击头像
2. 选择本地图片 → 等待上传成功提示
3. 头像应立即显示为新图片，不再出现"裂图"

**管理员配置 OSS 桶（仅超级管理员）：**
1. 登录后台 → 进入「系统设置 → 存储配置」
2. 公开桶：填写桶名、Region、SecretId、SecretKey、访问域名
3. 私有桶：填写桶名、Region、SecretId、SecretKey、签名 URL 有效期
4. 点击「测试连接」按钮 → 系统会真实上传一个 1KB 测试文件并校验读写权限
5. 测试通过后点「保存」即生效（无需重启服务，约 5 分钟内全节点同步）

**注意事项：**
- SecretKey 在保存后会以 `****1234` 形式掩码显示，需修改时点旁边的「修改」按钮重新输入
- 所有配置变更会写入审计日志（操作人、IP、修改字段名）
- 普通管理员看不到「存储配置」菜单

---

### 2. BUG2-A：我的优惠券 - 长期有效与即将到期提醒

1. 进入「我的优惠券」页面
2. 顶部三个 Tab：未使用 / 已使用 / 已过期
3. 在「未使用」Tab 下：
   - **绿色「长期有效」标签**：表示该券永不过期，可以放心积攒
   - **红色「即将到期」标签 + 红色到期日期**：表示该券将在 7 天内到期，请尽快使用
   - 普通有效期券：显示正常黑色到期日期

**注意事项：**
- 「未使用」Tab 不再显示已过期的券，过期券请到「已过期」Tab 查看
- 下单时可选择的优惠券与「未使用」Tab 列表完全一致

---

### 3. BUG2-B：文章收藏

1. 在文章详情页底部点击「收藏」图标
2. 出现 Toast 提示「已收藏」或「已取消收藏」
3. 立即去「我的收藏」页面 → 切到「知识」Tab → 应能看到刚收藏的文章

**注意事项：**
- 取消收藏后，下次进入文章详情页收藏按钮会自动变为未收藏状态
- 收藏失败时会有错误 Toast 提示，请检查网络后重试

---

### 4. BUG3-B：积分日常任务

1. 进入「积分中心」页面
2. 上方显示总积分；下方显示「今日任务」列表
3. 三种状态：
   - **加载中**：显示骨架屏（短暂闪烁）
   - **加载失败**：显示「加载失败，点击重试」+ 重试按钮
   - **暂无任务**：显示空状态图 + 文案「暂无任务」
   - **正常**：显示完整任务列表（如签到、浏览商品等），完成后显示对勾

**注意事项：**
- 任务加载失败不影响积分汇总数据的显示
- 点击「重试」即可重新加载任务列表，无需刷新整页

### 5. BUG4：中医养生 AI 咨询

入口：H5 / 小程序 / App 的「中医养生」页面，提供三种诊断方式。

**A. 体质测评（9 题问卷）**
1. 点击「开始体质测评」→ 完成 9 题问卷
2. 系统计算体质类型并展示结果页
3. 点击「向 AI 咨询调理方案」→ **弹出选择咨询对象（家人）**
4. 选好家人 → 点击「确认」
5. 进入 AI 聊天页，AI 会主动说："您好，根据您的体质测评，我已识别出您是【XX 质】，下面为您详细介绍调理方案……"

**B. 舌诊**
1. 点击「上传舌头照片」→ 拍照或从相册选择
2. 上传成功后 → **弹出选择咨询对象**
3. 选好家人 → 点击「确认」
4. 进入 AI 聊天页，AI 会基于舌象图片分析体质并给出调理建议

**C. 面诊**
1. 点击「上传面部照片」→ 拍照或从相册选择
2. 上传成功后 → **弹出选择咨询对象**
3. 选好家人 → 点击「确认」
4. 进入 AI 聊天页，AI 会基于面相图片分析气血状况并给出调理建议

**注意事项：**
- 三种入口都需要先选择"咨询对象"（即为家中哪位成员咨询），系统会把答案 / 图片 / 体质类型自动带入聊天上下文
- 关闭"选择咨询对象"弹窗即可取消本次咨询，已上传的图片不会丢失，可重新发起
- 同一咨询对象的不同次咨询历史会分别保留

## 四、常见问题

**Q1：我以前的头像还是裂图怎么办？**
A：本次只修复了"新上传"的逻辑。已经裂图的旧头像，请重新上传一次即可恢复。

**Q2：我有一张长期有效券，为什么还是在"已过期"Tab？**
A：长期券（无到期时间）应出现在"未使用"Tab。如果仍在"已过期"，请下拉刷新或重新登录一次。

**Q3：日常任务一直显示"加载失败"怎么办？**
A：先检查网络；如果其他页面正常但任务持续失败，请联系管理员检查服务端 `/api/points/tasks` 接口日志。

**Q4：选完咨询人后聊天页空白？**
A：本次已修复此问题。若仍遇到，请截图聊天页 URL（含 type 和 msg 参数）反馈给客服。
