## 1. 功能简介

「地图配置」是系统管理下新增的**一级菜单**，把整个系统使用的高德地图相关参数集中起来，由超级管理员/系统管理员自助维护。本次更新带来以下能力：

- **集中配置**：高德地图的 3 个 Key（Server / Web JS / H5 JS）+ 安全密钥 + 默认地图参数（默认城市、中心点经纬度、缩放级别）统一在一个页面维护。
- **即配即生效**：点击保存后系统所有使用高德地图的位置自动使用新配置，**无需重启、无需运维介入**。
- **自检能力**：「测试连接」按钮逐项验证 3 个 Key 是否可用，并记录最近 5 次测试历史。
- **平滑过渡**：上线后保持向下兼容——数据库无配置时自动读取原有环境变量；保存过一次后系统自动切换到数据库配置。
- **安全引导**：Web/H5 端 Key 必然下发到浏览器/客户端（这是高德官方设计），页面提供「复制当前域名」按钮 + 域名白名单提示文案，引导管理员到高德控制台配置白名单。

## 3. 页面布局

「地图配置」页面采用单页分区布局，从上到下分为 4 个区块 + 底部操作区：

### 区块 ① 地图服务商

显示当前使用的服务商。本期固定为「高德地图」（不可修改），下拉框已禁用。后续如需扩展百度地图、腾讯地图等多供应商，会在这里加入选项。

### 区块 ② Key 配置（核心）

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| Server Key（服务端用） | ✅ 是 | 用于后端调用高德开放接口（地理编码、POI 搜索、静态地图等）。仅在后端持有，不会下发到客户端。|
| Web JS Key（管理后台用） | ✅ 是 | 管理后台浏览器端加载高德 JS API 用，**会下发到浏览器**。|
| H5 JS Key（用户端 H5/APP/小程序用） | ✅ 是 | 用户端加载高德地图用，**会下发到 H5/APP/小程序**。|
| 安全密钥 SecurityJsCode | 否 | 高德要求与 Server Key 配套使用的安全密钥，仅后端用，不下发。|

> 🔐 **安全提示**：Web JS Key 和 H5 JS Key 在浏览器/客户端可被用户在 F12 / 抓包工具中看到，这是高德官方的设计，无法在我方系统层面规避。**请务必到[高德开放平台控制台](https://console.amap.com/dev/key/app)为这两个 Key 绑定本系统的域名白名单**，否则 Web/H5 端会报 `USERKEY_PLAT_NOMATCH` 错误。
>
> 页面上 Web JS Key 和 H5 JS Key 输入框右侧各有「复制当前管理后台域名 / H5 端域名」按钮，点击即把当前域名复制到剪贴板，方便粘贴到高德控制台。

### 区块 ③ 默认地图参数

| 字段 | 默认值 | 说明 |
|------|--------|------|
| 默认城市 | 北京 | 新建门店或预约下单时的默认定位城市 |
| 默认中心点经度 | 116.397428 | 默认地图中心点（经度范围 -180 ~ 180）|
| 默认中心点纬度 | 39.90923 | 默认地图中心点（纬度范围 -90 ~ 90）|
| 默认缩放级别 | 12 | 拖动滑块 3-18，越大越接近街道级 |

### 区块 ④ 测试连接 + 最近 5 次测试记录

- 点击右上角【测试连接】按钮：
  - 系统会**逐项**测试 3 个 Key：
    - **Server Key（地理编码）**：调用一次"地理编码"接口（查询"北京天安门"的坐标）。
    - **Web JS Key（脚本加载）**：模拟加载一次 Web JS API 脚本。
    - **H5 JS Key（脚本加载）**：模拟加载一次 H5 JS API 脚本。
  - 每个 Key **分别显示**状态：
    - ✅ 正常：绿色对勾
    - ❌ 失败：红色叉，并附带错误码（如 `INVALID_USER_KEY`、`USERKEY_PLAT_NOMATCH`）
  - **单 Key 超时按 10 秒控制**，超时显示「失败：超时」。
  - 测试结果**自动写入日志**，显示在页面底部「最近 5 次测试记录」表格中。

「最近 5 次测试记录」表格列：时间、操作人、Server / Web / H5 各 Key 状态、整体是否通过。

### 底部操作

- **【保存】**：保存所有修改并立即在全系统生效。
- **【取消】**：恢复为最近一次保存的值（不会丢失你尚未输入的内容之外的字段）。

## 4. 操作步骤

### 4.1 第一次配置（推荐流程）

1. 用超级管理员账号登录管理后台。
2. 进入「系统管理 → 地图配置」。
3. 在区块 ② 中：
   - 粘贴你在高德开放平台申请的 **Server Key**。
   - 粘贴 **Web JS Key**，点击右侧「复制当前管理后台域名」按钮，到[高德开放平台](https://console.amap.com/dev/key/app)对应 Key 的「服务平台 → 安全配置」中粘贴域名加入白名单。
   - 粘贴 **H5 JS Key**，同样点击「复制当前 H5 端域名」按钮并到高德控制台配置白名单。
   - （可选）粘贴 **SecurityJsCode**。
4. 在区块 ③ 中调整你的业务默认地图参数（如城市改为深圳，中心点改为某门店附近，缩放级别 13~14）。
5. 点击底部【保存】按钮。
6. 出现 Toast 提示：「地图配置已保存，全系统已生效。已打开的页面刷新后生效」。
7. 在区块 ④ 点击【测试连接】，确认 3 个 Key 全部 ✅。

### 4.2 切换 / 续费 Key

1. 进入页面，旧 Key 会显示在输入框中（明文，便于复核）。
2. 直接覆盖输入新 Key。
3. 点击【保存】，立刻全系统生效。
4. 点击【测试连接】确认新 Key 可用。

### 4.3 仅修改默认地图参数

例如把默认城市从北京改为上海：

1. 进入页面，区块 ② 的 Key 字段保持不变（不要清空，否则保存会失败）。
2. 在区块 ③ 修改「默认城市」为上海，调整经纬度到上海市中心。
3. 点击【保存】。

## 5. 配置生效范围

本次保存的配置将作为**全系统唯一的地图配置数据源**，被以下所有使用点自动读取：

| # | 使用端 | 使用位置 | 用途 |
|---|--------|----------|------|
| 1 | 管理后台 | 商家管理 → 门店管理 → 新建/编辑门店 | 地图选点、坐标拾取、地址解析 |
| 2 | 用户端 H5 | 预约下单 → 选择门店 → 点击地图 | 显示门店位置、查看周边、导航跳转 |
| 3 | 用户端 H5 | 门店列表 / 门店详情 | 静态地图缩略图 |
| 4 | 用户端小程序 | 预约下单选门店、门店详情 | 显示门店位置、导航 |
| 5 | 用户端 APP（Android / iOS） | 门店详情、下单选店 | 显示门店位置、导航 |
| 6 | 后端服务 | 地址 ↔ 坐标转换、POI 搜索、距离计算 | 服务端调用 |

> 💡 切换配置后，**已经打开**的浏览器页面或 APP 由于本地缓存可能仍在使用旧 Key。**新打开的页面/重启的 APP** 会立刻取到新值。这属于正常现象，建议在保存配置时同步通知运营团队让用户做一次刷新/重启。

## 6. 兼容性说明

系统按以下顺序读取地图配置：

1. **优先读数据库 `map_config` 表**（通过本菜单保存）
2. 数据库为空 → **回退读环境变量**（`AMAP_SERVER_KEY` / `AMAP_WEB_JS_KEY` / `AMAP_H5_JS_KEY` / `AMAP_SECURITY_JS_CODE`）
3. 环境变量也为空 → 标记为「未配置」，相关地图能力降级（按现有降级策略走 OpenStreetMap 或显示「地图未配置」提示）

> ✅ 上线时**不需要运维做任何动作**。老的环境变量配置依然能继续工作。**一旦管理员在新菜单里保存过一次配置**，则全系统**优先使用数据库配置**，环境变量自动停用。
>
> 页面顶部会有一段灰色提示告诉你当前正在使用「数据库配置（更新于 xxxx）」还是「环境变量兜底」。

## 7. 注意事项

1. **三个 Key 必填**：Server Key / Web JS Key / H5 JS Key 都必须填写，否则保存按钮会报错。
2. **域名白名单是关键**：Web/H5 Key 一定要到高德控制台配置好域名白名单，否则前端控制台会报 `USERKEY_PLAT_NOMATCH`，地图会不显示。
3. **测试连接是真实调用**：点击「测试连接」会真实调用高德的接口，注意频次（默认 30 次/分钟限流）。
4. **缓存影响**：保存新 Key 后，已打开的用户端页面/APP 因为浏览器或客户端缓存可能仍在使用旧 Key，需要刷新或重启 APP 后才会取到新值。
5. **测试日志会持续累计**：「最近 5 次」只是页面展示限制，完整审计日志保留在后端数据库 `map_test_logs` 表中。
6. **权限严格控制**：只有超级管理员和系统管理员可见此菜单和操作，普通运营/客服/商家无法访问。

## 8. 常见问题排查

| 现象 | 原因与处理 |
|------|------------|
| 保存按钮一直加载中 | 检查浏览器网络是否正常、后端服务是否在线 |
| 保存返回 400「Server Key 为必填项」 | 区块 ② 的某个 Key 输入框为空，请补齐 |
| 保存成功但 Web 端还是空白地图，控制台报 `USERKEY_PLAT_NOMATCH` | 你的 Web JS Key **没有**在高德控制台配置当前域名白名单 |
| 测试连接 Server Key 失败：`INVALID_USER_KEY` | Server Key 写错或已被高德停用，重新复制 |
| 测试连接 Server Key 失败：`DAILY_QUERY_OVER_LIMIT` | 当日调用次数已超出高德配额，建议升级套餐 |
| 测试连接 H5 JS Key 失败：超时 | 服务端无法访问 `webapi.amap.com`，检查网络出口 |
| 切换 Key 后用户反馈"地图还是旧的" | 让用户**强制刷新**浏览器（Ctrl+F5）或**完全退出并重新打开** APP |
