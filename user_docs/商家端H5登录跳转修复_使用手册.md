> 发布日期：2026-04-25  
> 适用端：商家端 H5（手机浏览器）、商家端 PC 网页、管理后台  
> 基础访问域名：`https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27`

## 一、这次修了什么？

本次修复解决了 **商家在手机浏览器登录商家端 H5 后被莫名其妙踢到「用户端登录页」** 的问题，同时顺手修复了两个相关的小问题。

| 编号 | 现象 | 修复后表现 |
|---|---|---|
| ① | 在 `/merchant/m/login` 用商家账号登录，登录成功后短暂看到工作台，随即被跳到 **用户端** 登录页 | 登录成功后**稳定停留在商家工作台** `/merchant/m/dashboard`，不再被踢 |
| ② | 直接打开 `/merchant/m/login`，被自动跳到 PC 版商家登录页 `/merchant/login` | 直接渲染 **移动版商家登录页**，URL 稳定停留在 `/merchant/m/login/` |
| ③ | 用「非商家账号」在商家登录页登录，会先进入工作台再被踢到用户端 | 登录页直接弹提示 **"该账号不是商家账号，请使用商家账号登录..."**，不会跳工作台 |
| ④ | 管理后台 (admin-web) 登录态失效时，也会被踢到用户端 `/login` | 登录态失效时跳到管理后台自己的登录页，不再跨端跳转 |

## 二、功能介绍

### 2.1 四端登录页"各回各家"

| 端 | 登录页 URL | 登录态失效时跳转目标 |
|---|---|---|
| **C 端用户 H5** | [`/login/`](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/login/) | `/login` |
| **商家端 H5（手机）** | [`/merchant/m/login/`](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/m/login/) | `/merchant/m/login` |
| **商家端 PC（电脑）** | [`/merchant/login/`](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/login/) | `/merchant/login` |
| **管理后台 admin** | [`/admin/login/`](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/login/) | `/login`（admin 自身的登录路径） |

核心规则：**在哪个端触发登录态失效，就跳回哪个端的登录页**，不再发生"商家端被踢到用户端"的串台现象。

### 2.2 登录时的身份预检

商家登录接口返回后，前端会立即校验账号的 **商家身份标签**：
- 合法角色：`owner`（店主）/ `store_manager`（店长）/ `verifier`（核销员）/ `finance`（财务）/ `staff`（员工）
- 合法身份：`merchant_owner` / `merchant_staff`

只有校验通过才会保存 token 并跳入商家工作台；否则直接在登录页弹窗提示，避免用户体验到"闪进工作台又被踢"的诡异感。

## 三、操作步骤

### 3.1 商家手机登录（最常见场景）

1. **打开浏览器**，访问：
   ```
   https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/m/login/
   ```
   - 在手机上会直接渲染商家移动登录页；
   - 在电脑上也能正常打开（不会被自动跳走）。
2. 输入**商家手机号 + 密码**（或切到短信验证码方式）。
3. 点击"**登录**"按钮。
4. **若是商家账号**：提示"登录成功"，自动进入商家工作台 `/merchant/m/dashboard`。
5. **若账号只绑定了一个门店**：直接进入工作台；**若绑定多个门店**：先进入门店选择页 `/merchant/m/select-store`，选一家门店后再进入工作台。
6. **若不是商家账号**：登录页直接弹出红色错误提示"该账号不是商家账号...", 停留在登录页，允许重试。

### 3.2 商家 PC 登录

1. 电脑浏览器访问：
   ```
   https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/login/
   ```
2. 若用手机 UA 访问此 URL，会被自动识别为手机并重定向到 `/merchant/m/login`（与移动端体验一致）。
3. 登录成功后进入 PC 版商家工作台。

### 3.3 管理后台 admin 登录

1. 访问：
   ```
   https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/login/
   ```
2. 使用管理员账号登录。
3. 登录态失效时会自动跳回 admin 自己的登录页，不会跨到用户端。

## 四、常见使用场景 & 预期表现

| 场景 | 操作 | 预期结果 |
|---|---|---|
| **A. 商家首次登录** | 打开 `/merchant/m/login/` → 输入商家账号密码 → 点登录 | Toast "登录成功" → 自动进入 `/merchant/m/dashboard`，不会被踢回用户端 |
| **B. 商家刷新工作台** | 登录后在工作台按 F5 刷新 | 保持登录态，继续停留在工作台 |
| **C. 登录态过期** | 登录后长时间不操作，再次发请求 → 后端返回 401 | 自动跳回 `/merchant/m/login/`（**不是**用户端 `/login/`） |
| **D. 非商家账号误登商家页** | 用普通用户账号在 `/merchant/m/login` 登录 | 登录页直接弹红色提示 "该账号不是商家账号..."，不跳工作台 |
| **E. 用户端退出重登** | C 端用户在 `/profile` 退出 | 跳回 C 端登录页 `/login/`（行为保持不变） |
| **F. admin 登录态过期** | 后台管理员 token 过期 | 跳回 `/admin/login/`（**不是**用户端 `/login/`） |

## 五、注意事项

### 5.1 商家账号配置要求
要能进入商家工作台，测试账号必须满足：
1. 账号状态：**启用**；
2. 角色：`owner` / `store_manager` / `verifier` / `finance` / `staff` 其中之一；
3. 至少绑定 **1 个门店**；
4. 有登录密码（或支持短信登录）。

任一项缺失都会在登录时弹出"该账号不是商家账号..."的提示。

### 5.2 缓存清理建议
如果本次升级前你的浏览器已经缓存了旧版 JS，建议：
- **手机**：浏览器设置里清理缓存，或长按刷新选择"硬刷新"；
- **电脑**：按 `Ctrl + Shift + R`（macOS：`Cmd + Shift + R`）强刷。

### 5.3 手动验证清单（可选）

| 步骤 | 预期 |
|---|---|
| 1. 浏览器访问 `/merchant/m/login/` | 页面直接渲染，URL 停在 `/merchant/m/login/` |
| 2. 商家账号登录 | Toast "登录成功" → 进入工作台，**不被踢回 `/login/`** |
| 3. 工作台刷新 | 保持登录态 |
| 4. 浏览器 F12 → Application → 清除 `merchant_token` → 刷新工作台 | 跳回 `/merchant/m/login/`（不是 C 端 `/login/`） |
| 5. 用非商家账号登录 `/merchant/m/login` | 弹红色提示"该账号不是商家账号..."，不跳工作台 |

### 5.4 三端 Token 隔离
- 商家端使用 `localStorage.merchant_token`；
- C 端用户使用 `localStorage.token`；
- admin 后台使用 `localStorage.admin_token`。

三者互不串扰。退出某一端只会清除该端自己的 token，不影响其他端登录状态。

## 六、故障排查

### Q1：我是商家账号，登录后还是被踢到用户端？
- 清浏览器缓存强刷一次；
- 若仍复现，F12 打开 Network，复制登录接口 `/api/merchant/auth/login` 的返回体（JSON），确认 `merchant_role` 字段值是否在合法列表里。

### Q2：弹出"该账号不是商家账号"？
- 账号数据问题。联系管理员在 admin 后台为该账号补配 `owner` 角色并绑定门店。

### Q3：直接访问 `/merchant/m/login` 还是被跳到 `/merchant/login`？
- 本次修复后已彻底解决。若仍复现，通常是浏览器缓存了旧版 HTML，强刷即可。

### Q4：admin 后台退出后跳到了奇怪的页面？
- 本次修复已将 admin 的 401 跳转锁定到自身登录页。如仍有问题，清 admin 端 cookie 后重登。
