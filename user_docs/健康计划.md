## 功能简介

本次重做了「**健康计划（健康打卡）**」功能。

一句话说清楚：**用户自己创建打卡计划（如「每天喝 8 杯水」「每天走 6000 步」），每天回来点一下打卡，看自己坚持了多久、完成得怎么样。**

### 本次重点变化

- 落地页直接进入「打卡首页」，**不再经过「用药提醒/健康打卡/自定义计划」三块的总目录页**
- 顶部为蓝紫色横幅，显示进行中计划数、今天已打卡数、本周完成率
- 计划卡片支持「打卡 / 已打卡」按钮 + 右上角「⋯」菜单（编辑 / 删除）
- 删除计划支持二次确认弹窗，且会同步清除该计划的全部打卡记录
- 支持「每天」与「每周 X 次」两种打卡频率（**不做一天多次/几点几分**）
- 允许补打「最近 3 天」内漏掉的打卡
- 提供「我的打卡成果」页：连续天数 + 月历高亮 + 各计划完成率柱状图
- 后台管理端：管理员**仅可查看**用户计划与打卡统计，**只读不可编辑**

## 使用说明

### 1. 打开健康计划页

- 在 H5 端，直接访问 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-plan](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-plan)
- 在 App 或小程序中，从你已配置的菜单入口进入，点击「健康计划」

> 入口由用户在菜单管理中自行配置，本次开发**不在首页加任何浮动按钮**。

### 2. 落地页（打卡首页）

进入后第一眼看到，自上而下：

1. **顶部蓝紫色横幅**：显示「健康计划」与一句小副标题
2. **总览卡**：
   - 进行中计划数
   - 今天已打卡数
   - 本周完成率
3. **「我的计划」列表**：每个计划一张卡片
4. **右下角浮动「+」按钮**：点击进入新建计划页
5. 右上角的「查看成果 ›」：点击进入成果页

### 3. 创建打卡计划

1. 在打卡首页右下角点击 **「+」** 浮动按钮
2. 进入「新建计划」页，填写以下三项：
   - **计划名称**（如「每天喝 8 杯水」「每天走 6000 步」）
   - **打卡频率**：选择「每天」或「每周 X 次」（X 可在 1~7 之间填写）
   - **计划起止时间**：开始日期必填，结束日期可不填（不填即视为长期）
3. 点击「保存」即创建成功，返回打卡首页

### 4. 完成今日打卡

- 在「我的计划」列表中，每张卡片右侧有「**打卡**」按钮
- 点击一下，今日完成，按钮立即变为灰色「**已打卡**」
- 同一天**不能重复打卡**（再次点击会提示已完成）

### 5. 编辑或删除计划

1. 在计划卡片右上角点击 **「⋯」** 按钮
2. 弹出菜单中选择：
   - **编辑**：进入编辑页，所有字段会预先回填，改完保存
   - **删除**：弹出二次确认，确认文案为「确定删除该计划吗？该计划的打卡记录也会一并清除。」，确认后**删除计划及其所有打卡记录**

### 6. 补卡（最近 3 天）

- 如果某天忘记打卡，可在 **3 天内**补打
- 4 天前及更早的日期**不允许补卡**
- 当天和未来日期**不允许补卡**（当天直接打卡即可，未来不可预打卡）

### 7. 查看打卡成果

1. 在打卡首页右上角点击「**查看成果 ›**」
2. 进入「我的打卡成果」页，可看到：
   - **连续天数**（连续每天都至少打了一次卡的天数）
   - **累计打卡总数**
   - **本月完成率**
   - **月历视图**：打过卡的日子高亮显示
   - **各计划完成率柱状图**

## 注意事项

1. **入口配置**：本次开发**不再添加首页浮动按钮**，进入「健康计划」需用户自行在菜单管理中配置链接 `/health-plan`，或直接访问该 URL。
2. **频率限制**：本期仅支持「每天」「每周 X 次」两档，**不支持「一天几次/几点几分」**。如有刚性提醒诉求，请使用「用药提醒」功能。
3. **补卡限制**：补卡仅允许补**最近 3 天**内的，且不能补当天或未来。
4. **删除影响**：删除计划会**同步清除该计划的全部打卡记录**，请确认后再操作。
5. **不做提醒推送**：本次未做订阅消息/App 推送，养成习惯靠用户自驱。
6. **后台管理只读**：管理员可查看「谁创建了哪些计划、打卡统计」，不可编辑。
7. **历史数据**：本次为重做，**不迁移老数据**，从零开始记录。
