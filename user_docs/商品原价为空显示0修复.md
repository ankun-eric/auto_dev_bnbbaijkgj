## 功能简介

本次修复了商品管理模块中**原价为空时显示为 0** 的 Bug。

**修复前的问题**：在管理后台新建商品时，如果不填写原价（原价是选填项，用于划线价展示），保存后系统会将原价存储为 `0`，导致：
- 再次编辑时，原价输入框显示 `0` 而非空白
- 数据库中无法区分"用户未填写原价"和"原价确实为 0"

**修复后的效果**：
- 新建商品不填原价 → 数据库存储为 `NULL`（空值）
- 编辑商品时原价为空 → 输入框显示空白（而非 0）
- 用户端商品详情/列表 → 原价为空时完全不展示划线价
- 历史数据中原价为 0 的商品 → 已自动清洗为 `NULL`

## 使用说明

### 管理后台 — 商品管理

1. **访问管理后台**：打开 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)，使用管理员账号登录
2. **进入商品管理**：在左侧菜单中找到「商品体系」→「商品管理」
3. **新建商品（不填原价）**：
   - 点击「新建商品」按钮
   - 填写商品名称、售价等必填项
   - **原价输入框保持为空**（不填写任何内容）
   - 点击「保存」
4. **验证修复效果**：
   - 保存成功后，再次点击该商品的「编辑」按钮
   - **原价输入框应显示为空白**（不再显示 `0`）
5. **新建商品（填写原价）**：
   - 如果需要设置划线价，在原价输入框中输入价格（如 `99.99`）
   - 原价应大于售价，保存后可正常展示

### 用户端 H5 — 商品浏览

1. **访问用户端**：打开 [用户端 H5](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)
2. **查看商品列表**：浏览商品列表页面
   - 有原价的商品：显示售价 + 划线原价（如 ~~¥99.99~~）
   - 无原价的商品：仅显示售价，不显示划线价
3. **查看商品详情**：点击任意商品进入详情页
   - 有原价且大于售价：价格区域显示划线原价
   - 无原价：价格区域仅显示售价

## 注意事项

1. **原价字段为选填**：创建商品时，原价不是必填项。只有需要展示划线价时才需要填写原价
2. **原价应大于售价**：如果填写了原价，原价必须大于等于售价，否则无法保存
3. **历史数据已自动清洗**：本次更新后，系统启动时会自动将历史数据中原价为 0 的商品修正为空值（NULL），无需手动处理
4. **多规格商品**：多规格商品的原价由各 SKU 独立管理，商品级别的原价在多规格模式下不生效
