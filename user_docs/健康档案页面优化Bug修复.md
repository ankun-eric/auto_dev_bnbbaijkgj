## 功能简介

本次更新修复了健康档案页面（`/health-profile`）的 **6 个 Bug**，涉及用药计划区域、用药提醒卡片、Hero 卡片、健康看板入口、成员 Tab 切换和子页面返回定位等多个模块的数据展示和交互体验问题。

## 使用说明

### Bug 1 修复：用药计划区域优化

**修复内容：**

1. **统计数字一致性**：首页用药计划标题栏后面的统计数字现在与点击"全部"进入的列表页记录总数完全一致。系统通过统一的成员过滤参数（`consultant_id`）确保两端数据口径一致。

2. **移除重复按钮**：展开用药计划列表后，底部不再显示多余的"+ 新增用药"蓝色按钮，只保留标题栏右侧的"+新增"文字链接。

3. **列表页导航栏美化**：用药计划列表页顶部导航栏已添加蓝色渐变背景（与其他页面风格统一），文字改为白色，视觉效果更统一。

4. **Tab 统计修正**：列表页"服药中"Tab 后面的括号数量现在正确显示实际记录数，不再显示为 `（0）`。

**操作步骤：**
1. 打开健康档案页面
2. 向下滚动找到"用药计划"区域
3. 点击展开查看用药计划列表
4. 确认标题栏统计数字与列表条数一致
5. 点击"全部 ›"进入列表页，确认 Tab 数字和列表条数一致

### Bug 2 修复：用药提醒卡片信息修正

**修复内容：**

1. **"下一次用药"显示修正**：用药提醒卡片中"下一次用药"现在正确显示待服用药品的名称和计划时间（如"阿司匹林 · 12:00"），不再显示错误的统计文案。

2. **"还剩X次"统计修正**：剩余次数现在准确统计当前选中成员今日的剩余用药次数。

3. **成员切换联动**：切换不同成员 Tab 后，用药提醒信息会随之正确更新为对应成员的数据。

**操作步骤：**
1. 打开健康档案页面
2. 向下滚动到"今日健康数据"区域
3. 查看用药提醒卡片，确认"下一次"显示的是具体药品名和时间
4. 切换到其他成员 Tab，确认用药提醒信息随之更新

### Bug 3 修复：Hero 卡片紧凑化

**修复内容：** 个人信息 Hero 卡片的整体间距和内边距已缩小，视觉上更加紧凑，首屏可展示更多内容。

- 外层内边距从 `12px 16px` 缩减为 `8px 16px`
- 卡片内边距从 `20px` 缩减为 `14px 16px`
- 头像与文字信息间距从 `16px` 缩减为 `12px`
- 健康标签区域顶部间距从 `16px` 缩减为 `10px`
- 编辑按钮更精致

### Bug 4 修复：健康看板入口优化

**修复内容：**

1. **本人 Tab 也显示看板入口**：健康看板入口不再仅限"其他成员"Tab，本人 Tab 下也会显示"查看我的健康看板 →"按钮。

2. **位置调整**：所有成员 Tab 下，健康看板入口统一放在 Hero 卡片**下方**，邀请区域上方，布局更合理。

**操作步骤：**
1. 打开健康档案页面
2. 在"本人"Tab 下确认可以看到"查看我的健康看板 →"按钮
3. 切换到其他成员 Tab，确认同样显示健康看板入口
4. 点击按钮可正常跳转到对应成员的健康看板页面

### Bug 5 修复：成员切换自动回顶

**修复内容：** 点击切换成员 Tab 后，页面会自动滚动到最顶部，不再停留在上一个 Tab 的滚动位置。

**操作步骤：**
1. 打开健康档案页面
2. 向下滚动页面到中间位置
3. 点击另一个成员 Tab
4. 确认页面自动回到顶部

### Bug 6 修复：子页面返回 Tab 定位

**修复内容：** 从健康档案页面进入子页面（如用药计划全部页、新增用药页、用药提醒页等）后，返回时会自动恢复到离开时所在的成员 Tab，不再默认回到"本人"Tab。

**操作步骤：**
1. 打开健康档案页面
2. 切换到"其他成员"Tab
3. 点击用药计划的"全部 ›"进入列表页
4. 点击返回按钮
5. 确认页面自动恢复到"其他成员"Tab

## 注意事项

1. **登录要求**：健康档案页面需要用户登录后才能访问。
2. **数据依赖**：用药计划相关功能需要先添加用药计划才能看到数据展示效果。
3. **多成员场景**：Bug 6 的返回 Tab 定位功能需要至少有 2 个家庭成员才能体验（本人 + 至少 1 个家人）。
4. **浏览器兼容**：建议使用 Chrome 或 Safari 浏览器访问 H5 页面。
