### 2. NMPA 种子数据建议预先准备

执行计划要求 ~1000 条 `seed_drugs.json`，建议在新会话前由用户或运营离线准备（从 NMPA 公开数据集、中国药典、百度百科药品库等渠道汇总），单纯依靠 AI 现场编造 1000 条结构化药品数据不可靠（剂量/规格/适应症等专业字段易错）。

### 3. 引用点扫描必须先做

执行计划 §3 明文要求：删除任一旧路径前，必须先输出 `path_references_audit.md`。建议新会话第一步先扫描 + 输出对照表 + 用户人工确认 + 再切换 + 最后删除（小程序 `pages/ai/index`、Flutter `screens/ai/ai_home_screen.dart` 风险最高）。

### 4. 多模态剥离影响面评估

`grep "enable_vision\|multimodal\|upgrade_messages_to_multimodal\|build_vision_message_content"` 全仓扫描，输出全部调用点的 import + 函数签名，再决定剥离策略。`backend/app/services/ai_service.py` 是项目核心服务文件，**直接删函数有连环风险**，应改为：先标记 `@deprecated` + 抛 `RuntimeError`，灰度一周再实删。

### 5. SSE 接口建议先做单接口 PoC

`POST /drug-explain` 是 SSE 流式接口，与现有 `chat.py` 的 SSE 实现保持一致较好。建议新会话开工时第一个先做这个 PoC + curl 联调通过，再做剩余批量接口。
