## 三、操作步骤

### 步骤一：进入下单页（F-01 / F-02 / F-03）

1. 在商品详情页选择规格后点击「立即购买」。
2. 在下单页：
   - 若该商品需要选择门店，「选择门店」卡片右侧显示**📍 导航**按钮（F-01）。
   - 若需要上门服务地址或收货地址，地址卡片右侧显示**📍 导航**按钮（F-02 / F-03）。
3. 点击按钮 → 弹出底部「打开地图」选择面板：高德 / 百度 / 腾讯 / Apple Maps / 网页地图（H5）。
4. 选中后自动跳转对应地图 App 进行导航 / 搜索。

### 步骤二：进入订单详情页（F-04 / F-05 / F-06）

1. 「我的订单」中点开任意订单。
2. 顶部「订单地址」卡片中：
   - 门店地址行右侧显示**📍 导航**（F-04）。
   - 收货 / 上门服务地址行右侧显示**📍 导航**（F-05 / F-06）。
3. 点击按钮，体验同上。

### 步骤三：无坐标地址的兜底体验

- **H5**：弹出「打开地图」面板时仍可选择任一地图，URL Scheme 自动转为关键字搜索（高德 `?q=`，百度 `wd=`，腾讯 `keyword=`，Apple Maps `?q=`）。
- **小程序**：因 `wx.openLocation` 必须坐标，此场景下会**自动复制完整地址到剪贴板**并弹出提示「已复制地址，请在地图 App 中粘贴搜索」。
- **Flutter**：与 H5 一致，调用各地图 App 的关键字搜索 Scheme。

## 四、关键设计点

1. **视觉一致**：按钮采用品牌主绿 `#52c41a`、12% 透明背景、12px 字号、圆角 16px，与商家详情页导航按钮完全一致，确保用户认知一致。
2. **零额外接口**：复用既有 `MapNavSheet` / `wx.openLocation` / `MapNavUtil`，不新增网络请求。
3. **500ms 防抖**：三端均加入连续点击 500ms 防抖，避免误触多次拉起地图 App。
4. **关键字降级**：坐标缺失自动转关键字搜索，保证地址可用性 100%。
5. **隐私安全**：不记录用户坐标 / 地址日志，不上报埋点。
6. **零后端业务改动**：仅在 `UnifiedOrderResponse` 中追加 6 个只读字段（`store_address`、`store_lat`、`store_lng`、`shipping_address_text`、`shipping_address_name`、`shipping_address_phone`），从已有数据库字段透传，不新增表 / 接口 / 外部依赖。
