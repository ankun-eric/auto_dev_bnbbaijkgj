## 建议的排查方向

> 以下为基于错误信息的初步排查建议，仅供参考。

1. **服务器资源问题**：Next.js 构建（npm install + TypeScript 编译 + Webpack 打包）消耗大量内存和 CPU，可能导致服务器 OOM（Out of Memory）或 Docker daemon 崩溃。建议检查服务器内存配置，必要时增加 swap 空间或在构建时限制资源使用（`docker compose build --no-cache --memory=2g h5-web`）

2. **服务器重启**：如果服务器确实因 OOM 崩溃，需要通过云控制台（腾讯云）重启服务器。重启后 docker daemon 应自动恢复，之前所有容器（backend/admin/h5）应通过 `restart: unless-stopped` 自动启动

3. **构建优化**：建议将构建任务放在本地 CI 环境完成，服务器只负责拉取预构建的镜像；或使用 GitHub Actions / 阿里云容器镜像服务（ACR）进行远程构建 + 推送，服务器拉镜像

4. **网络问题**：确认服务器到阿里云 Codeup 的网络连接正常，以及 npm 镜像源（npmmirror.com）在国内的可达性
