## 功能简介

本次 v1.1 位置调整补丁，在 v1.0 已上线的「会员中心位」基础上，**新增**了第二处投放位置——**健康档案-档案管理列表页顶部**。两处使用同一张视觉完全一致的蓝色渐变大卡片，但主按钮点击后的行为不同：

- **会员中心位**：点击「邀请家人」按钮 → **跳转到邀请家人流程页**（沿用 v1.0）
- **健康档案位**：点击「邀请家人」按钮 → **从底部弹出"新增家人档案"半屏抽屉**（v1.1 新增）

同时，本次还做了一项视觉微调：**主标题强制单行**——主流屏 16pt、窄屏（iPhone SE/mini，375px 以下）自动降到 15pt 兜底；若套餐名过长则尾部自动省略号截断，保证版式整洁。

## 使用说明

### 场景 A：在会员中心位邀请家人（沿用 v1.0）

1. 打开会员中心：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center)
2. 在页面上方可以看到一张**蓝色渐变大卡片**，包含：
   - 家人手拉手图标（半透明白色）
   - 主标题：`<您的套餐名> · 可管理 N 位家人`（单行展示）
   - 用量行：`已管理 X / 上限 Y`
   - 辅助说明：`邀请家人，共享健康守护`
   - 右下角白色「**邀请家人**」按钮
3. 点击右下角白色「**邀请家人**」按钮 → 自动跳转到「邀请家人流程页」
4. 按照原有流程完成邀请即可

### 场景 B：在健康档案位新增家人档案（v1.1 新增）

1. 打开健康档案-档案管理列表：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/archive-list](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/archive-list)
2. 顶部仍保留灰色小字「本套餐可管理 N 位家庭成员（含本人）」
3. 在灰色小字**下方**，您会看到**同款蓝色渐变大卡片**（与会员中心位视觉完全一致）
   - 原先的「📋 档案列表 / 已管理 X 人 / 上限 Y 人 / + 新增」白色概览块已**完整替换**为新卡片
4. 点击卡片右下角白色「**邀请家人**」按钮 → **从屏幕底部弹出"新建档案"半屏抽屉**
5. 在抽屉中填写：
   - 家人姓名（如：张妈妈 / 李叔叔）
   - 与本人关系（如：父亲 / 母亲 / 配偶 / 子女）
   - 手机号（11 位）
   - 性别
6. 点击「保存档案」 → 档案创建成功后，可选择「立即邀请」或「暂不邀请」
7. 选择「立即邀请」会立即生成邀请码并展示，复制链接分享给家人即可
8. 选择「暂不邀请」也可稍后在档案卡片上重新发起邀请

### 场景 C：已达档案配额上限时

当您「已管理人数」已达套餐上限时（如 `已管理 10 / 上限 10`）：

- 两处位置的「邀请家人」白色按钮都会变为**半透明禁用状态**，点击无任何反应
- 卡片中会出现一行黄色警示文字：`⚠ 已达上限，请 升级套餐`（"升级套餐"为可点击文字链）
- 点击「升级套餐」链接：
  - 会员中心位：滚动到本页底部的「升级享更多权益」区
  - 健康档案位：跳回会员中心页的套餐档位区

### 场景 D：不限档套餐用户

如果您的套餐为「不限家人数」（如尊享版），则：

- 两处位置都始终展示正常态卡片
- 用量行显示：`已管理 X 人 · 不限上限`
- 「邀请家人」按钮始终可用，永远不会出现达上限态

## 注意事项

1. **两处卡片视觉完全一致**：从渐变色、圆角、阴影、字号、按钮、用量行到辅助说明，两处卡片像素级一致；唯一差异在于主按钮点击后的下游动作。
2. **数据来源完全一致**：两处卡片显示的 `已管理 / 上限` 数字均来自同一份后端接口，与档案管理列表页顶部灰色小字中的数字保持完全一致。
3. **窄屏适配**：在 iPhone SE / iPhone mini（屏宽 375px 以下）等窄屏设备上，主标题字号会自动降至 15pt 单行；如果您的套餐名过长，会以尾部省略号方式截断，权益短语（如「· 可管理 10 位家人」）保留。
4. **小程序与 App 同步生效**：由于小程序、Android App、iOS App 都通过 WebView 加载 H5，本次更新**无需重新下载或更新客户端**，您只需重新打开页面即可看到最新效果。
5. **新建档案抽屉行为不变**：健康档案位「邀请家人」按钮拉起的"新增档案"抽屉，与原先「+ 新增」按钮拉起的抽屉**完全是同一个**——字段、校验、提交流程、完成回跳全部不变。
6. **达上限态时抽屉不会弹出**：健康档案位在达上限态下，点击「邀请家人」按钮**不会**弹出新建档案抽屉（按钮已禁用），需要先升级套餐或删除某位现有家人后才能继续新增。

## 异常处理

| 场景 | 处理 |
|------|------|
| 卡片用量数据与顶部灰条不一致（理论上不应发生） | 以**顶部灰条接口**为准（前端兜底） |
| 健康档案位点击「邀请家人」抽屉拉不起来 | 极少数情况；可下拉刷新页面后重试 |
| 套餐名过长（>16 字符） | 主标题尾部省略号自动截断，权益短语完整保留 |
| 在抽屉内新增家人成功后 | 列表自动刷新，卡片用量行自动更新到最新数字 |
