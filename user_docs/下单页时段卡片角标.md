## 功能简介

### 改造前 vs 改造后

```text
当前（改造前）：                    改造后（本次）：

┌────────────────────┐              ┌──────────────┐橙█ 已结束
│ 11:00-12:00 已结束 │              │ 11:00-12:00  │   ← 贴边橙色直角小色块
└────────────────────┘              └──────────────┘      紧贴右上两条边、无圆角
   ↑ 文字撑长卡片                       ↑ 卡片宽度统一

┌──────────────────┐                ┌──────────────┐橙█ 已满
│ 14:00-15:00 已满 │                │ 14:00-15:00  │   ← 贴边橙色直角小色块
└──────────────────┘                └──────────────┘      紧贴右上两条边、无圆角
```

### 一句话总结

- **视觉**：所有时段卡片宽度统一，整页整齐紧凑；「已满」「已结束」状态一眼可识别
- **颜色**：背景橙色 `#FF9500` + 文字 `#FFFFFF`，柔和而不过度警示
- **形状**：直角矩形（无圆角），紧贴卡片右上两条边（`top:0` / `right:0`），卡片 `overflow:hidden`
- **交互**：不可约卡片整体置灰（`#F5F5F5` 底 + `#E0E0E0` 边框 + `#999999` 文字），完全禁止点击、键盘 Tab 不可聚焦、鼠标显示 `not-allowed`
- **优先级**：同一时段同时满足「已结束」与「已满」时，按「已结束」优先显示（PRD §5.1）

## 使用说明

### 步骤 1：打开「确认订单」页

1. 在 H5 / 小程序 / App 首页进入 **健康商城** 或 **服务列表** 模块
2. 选一款**需要到店预约**的商品（如「经络按摩」、「艾灸理疗」等 time_slot 模式商品）
3. 点击 **「立即预约」/「立即购买」** 进入下单页

### 步骤 2：查看时段卡片

下单页的「预约信息 → 预约时段」区域会以 **3 列网格** 展示所有时段卡片。观察以下三种状态：

| 状态 | 卡片背景 | 卡片边框 | 文字颜色 | 角标 | 是否可点击 |
|------|----------|----------|----------|------|------------|
| **可约**（默认） | 白色 | 浅灰 | 深色 `#333` | 无 | ✅ 可点击选中 |
| **已满** | 浅灰 `#F5F5F5` | 浅灰 `#E0E0E0` | 次级灰 `#999999` | 右上角橙色贴边小色块「**已满**」 | ❌ 不响应点击 |
| **已结束** | 浅灰 `#F5F5F5` | 浅灰 `#E0E0E0` | 次级灰 `#999999` | 右上角橙色贴边小色块「**已结束**」 | ❌ 不响应点击 |

### 步骤 3：验证角标视觉

- **角标颜色**：背景 `#FF9500` 标准橙 + 文字 `#FFFFFF` 白色
- **角标形状**：直角矩形、**无圆角**
- **角标位置**：**紧贴**卡片右上两条边（`top:0` / `right:0`），无外边距
- **角标字号**：10~11px，比时段文字略小
- **角标内边距**：上下约 `2px`，左右约 `6px`
- **卡片裁切**：卡片本身 `overflow:hidden`，角标绝对不超出卡片范围

### 步骤 4：验证交互

- 点击 **已满 / 已结束** 的灰色卡片 → **无任何响应**（不弹 Toast、不变色、不选中、不跳转）
- 点击 **可约** 的白色卡片 → 卡片变绿选中
- 用键盘 Tab 跳转焦点 → 灰色卡片自动跳过（`aria-disabled="true"`）
- 鼠标悬停（仅 H5）→ 灰色卡片显示 `not-allowed` 光标

### 步骤 5：当日全部不可约时

即使**当日所有时段都不可约**（全满或全已结束），页面也会完整列出全部灰色卡片，不会隐藏，也不会额外插入「当日已约满」兜底文案。用户通过「满屏皆灰」的视觉效果即可自行判断，选择切换到其他日期即可。

### 步骤 6：「已结束」优先级验证

当前时间已过某时段的结束时间，且该时段也已被约满时：**卡片显示「已结束」角标**（PRD §5.1 规定）。这一优先级在后端 `_derive_slot_status` 工具函数与前端 `deriveStatus` / `_slotStatus` 里均已保证。

## 注意事项

1. **仅视觉与交互改造，业务规则零变更**：什么算「已满」、什么算「已结束」完全沿用现有规则（PRD §8.1），本次不动。
2. **后端接口新增 `status` 字段但向后兼容**：即使旧客户端还未更新，接口照样返回 `is_available` + `unavailable_reason` 老字段，不会报错。
3. **前端有本地兜底**：若后端因时间差未标 `ended`，前端 `deriveStatus` / `_slotStatus` 会根据「当天 + `end <= nowHM`」自行判定，保证「已结束」优先级。
4. **不可约卡片不响应点击**：即使用户在网络延迟时绕过前端校验，后端 `create_unified_order` 也会对 `full` / `ended` 时段返回错误码拒绝下单（PRD §5.4）。
5. **日期模式下的日历置灰**：本次改造不涉及日期模式（`appointment_mode === 'date'`）的 picker 提示，沿用现有「（已约满）」红字小提示。日期级别后端也已同步返回 `status` 字段，后续如需扩展改造可直接沿用。
6. **服务端自动化测试**：本次新增 7 个集成用例 + 5 个单元参数化用例，加上既有 9 个 slot_grid 用例，**远程容器 pytest 21/21 PASS**，覆盖「status 字段存在性」、「available / full / ended 取值映射」、「ended 优先于 full」、「当天已过 end 时间自动 ended」等核心契约。
