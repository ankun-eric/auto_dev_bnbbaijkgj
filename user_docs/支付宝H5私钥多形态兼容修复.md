## 功能简介

本次修复彻底解决了「**在管理后台「支付配置」页面粘贴支付宝官方密钥工具产出的应用私钥后，
点击「测试」一定会报「应用私钥格式不被支持」**」的阻断性 Bug。

修复前的痛点：

- 支付宝官方密钥工具会**同时生成两份私钥文件**：
  - `应用私钥PKCS8.txt`（PKCS#8 格式）
  - `应用私钥RSA2048.txt`（PKCS#1 格式）
- 老逻辑只接受 `PKCS8.txt` 内容，运营人员若误粘 `RSA2048.txt` 内容（首次使用极易出现），
  系统会直接报错「应用私钥格式不被支持，请使用 PKCS8.txt 不是 RSA2048.txt」。
- 即使粘对了 `PKCS8.txt`，**带上 `-----BEGIN PRIVATE KEY-----` 头尾**也会被一刀切短路报错。
- 运营人员要在两份相似的 `.txt` 文件之间反复确认，每次新增商户、轮换密钥都会再踩坑。

修复后：

- 后端**自动识别并归一化**支付宝官方工具产出的**任何一种**合法私钥形态：
  - PKCS#8 中间 Base64（`应用私钥PKCS8.txt` 直接复制）
  - PKCS#8 完整 PEM（含 `-----BEGIN PRIVATE KEY-----` 头尾）
  - **PKCS#1 中间 Base64**（`应用私钥RSA2048.txt` 直接复制）✅ 本次新增
  - PKCS#1 完整 PEM（含 `-----BEGIN RSA PRIVATE KEY-----` 头尾）✅ 本次新增
  - 上述任何一种 + 不可见字符（BOM / 零宽空格 / 全角空格 / 换行 / Tab）
- 入库前**统一转换为 PKCS#8 PEM**，下游 SDK 不需要任何调整。
- 报错文案更友好：误粘公钥时给出「您粘贴的是公钥不是私钥」的针对性提示，
  内容截断时给出「不完整，请重新复制」提示，乱码时给出通用「无法识别」提示。
- 管理后台的输入框 placeholder 与下方提示也同步改写为友好引导：
  > 💡 把支付宝密钥工具生成的「应用私钥」文件内容直接粘贴进来即可：
  > PKCS8.txt 或 RSA2048.txt 都支持，含/不含 BEGIN/END 头尾都行。

## 使用说明

### 场景一：首次配置支付宝 H5 通道

1. 用浏览器打开管理后台：
   [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 使用管理员账号登录，左侧菜单进入「**支付配置**」。
3. 找到「**支付宝 H5 支付**」一行，点击右侧「**配置**」按钮，弹出右侧抽屉。
4. 填写以下字段：
   - **应用 AppID**：支付宝开放平台分配的 16 位数字
   - **接入模式**：选择「公钥模式」或「证书模式」（按贵公司支付宝商户后台配置情况选择）
   - **应用私钥（PEM）**：⭐ 重点字段
   - **支付宝公钥** / **应用公钥证书** 等：按所选接入模式填写对应字段
5. 在「应用私钥（PEM）」输入框，**直接打开**支付宝密钥工具产出的 `应用私钥PKCS8.txt`
   或 `应用私钥RSA2048.txt`（**两份文件任选其一即可，不必分辨**），
   `Ctrl+A` → `Ctrl+C` 全选复制，回到管理后台 `Ctrl+V` 粘贴进输入框。
6. 也可以直接点击输入框右上角的「**上传 .pem**」按钮选择文件，系统会自动读取内容。
7. 点击下方「**保存**」，再点击「**测试**」按钮：
   - 测试通过会显示「测试通过：联通性 OK」（实际文案以系统返回为准）
   - 测试失败会显示具体的友好文案，按提示处理即可

### 场景二：轮换密钥

支付宝商户出于安全考虑会定期更新 RSA2048 密钥对。轮换时操作完全相同：

1. 在支付宝开放平台密钥工具重新生成新的 RSA2048 密钥对
2. 把新生成的 **应用公钥** 内容上传到支付宝商户后台
3. 在管理后台「支付配置 → 支付宝 H5 → 配置」抽屉中，
   把新的 `应用私钥PKCS8.txt` **或** `应用私钥RSA2048.txt` 内容粘贴到「应用私钥」框
4. 保存 → 测试 → 通过即完成密钥轮换

### 场景三：误粘了「公钥」内容

如果不小心把「应用公钥」或「支付宝公钥」内容粘贴到了「应用私钥」框，
系统会立刻给出针对性提示：

> 您粘贴的内容看起来是「公钥」而不是「私钥」。请检查文件是否为应用私钥（PKCS8.txt 或 RSA2048.txt）。

按提示重新选择正确的私钥文件即可。

## 注意事项

1. **私钥安全**：「应用私钥」是商户绕过支付宝身份核验的关键凭据，
   请勿截图/粘贴到群聊/邮件等不受控渠道。系统已在所有日志中**屏蔽**私钥原文。
2. **必须是 RSA 2048 位**：支付宝目前仅支持 RSA2048 算法的密钥对。
   如果误用 RSA1024 或非 RSA 算法生成的密钥，仍会被拒绝并给出明确提示。
3. **粘贴时不要切断**：如果从微信 / 企业微信 / 浏览器复制，请先确认整段
   内容完整。系统会自动剔除 BOM / 零宽空格 / 全角空格等不可见字符，
   但**不会**自行补全被截断的内容。
4. **「测试」按钮**：本次修复后，「测试」按钮会真实地向支付宝网关发起一次
   `alipay.trade.query` 调用（用一个绝不可能存在的订单号），从而验证
   网关连通性 + 签名链路是否正常。如果显示网络不通，请检查服务器出网/防火墙。
5. **接入模式不要乱切**：「公钥模式」与「证书模式」对应的支付宝商户后台
   配置完全不同，请确认贵公司在支付宝商户后台的实际接入模式后再选择。

## 常见问题（FAQ）

**Q1：我之前已经粘了 `PKCS8.txt` 内容并保存成功，现在还要重新粘贴吗？**
不需要。本次修复**完全向后兼容**，对 PKCS#8 用户体验毫无变化。
原本能用的私钥继续可用，原本能保存的配置继续有效。

**Q2：粘了 `RSA2048.txt` 内容会和粘了 `PKCS8.txt` 在数据库里存的形态不一样吗？**
不会。两种粘贴方式落库前都会被**统一标准化为 PKCS#8 PEM 字符串**，
再 AES-256-GCM 加密入库。下游支付 SDK 拿到的永远是同一种规范化形态。

**Q3：如果我手上只有完整的 PEM 文件（含 `-----BEGIN ... PRIVATE KEY-----`
和 `-----END ... PRIVATE KEY-----` 头尾），可以直接粘贴吗？**
可以。本次修复后，**含/不含头尾两种粘贴方式都支持**，PKCS#8 / PKCS#1
两种格式也都支持。系统会自动识别。

**Q4：我看到原来的报错文案里写「请用 PKCS8.txt 不是 RSA2048.txt」，
现在还会出现这条提示吗？**
不会。这条带歧义的二次引导文案已彻底删除。修复后只会在
**确实无法识别**的情况下提示「请确认您粘贴的是支付宝密钥工具生成的
『应用私钥』内容（PKCS8.txt 或 RSA2048.txt 任一即可）」。
