## 一、功能简介

本次更新一次性打通了 AI 对话模式从「后台配置」到「H5 展示」的运营闭环，让运营同学只在一处维护内容，全端同步生效；让终端用户在对话页能直接看到、点击高频快捷指令并弹出对应卡片。本次主要改进：

1. **后台「功能按钮管理」图标 Emoji 化**：图标由"上传图片 + 维护 URL"升级为"点选 Emoji"，含 800+ Emoji + 关键字智能推荐。
2. **后台「AI 对话首页配置 → 功能宫格」Tab 简化**：内容编辑器整体移除，仅保留显示开关、最大数量、列数三个控制项，并提供"前往功能按钮管理"快捷跳转。
3. **H5 AI 对话首页宫格**：数据源切换为「功能按钮管理」表（`chat_function_buttons`），按 `sort_weight` 排序，取前 `max_count` 个，按 `cols` 列数渲染。
4. **H5 AI 对话详情页胶囊条**：在输入框上沿新增一条横向可滑动的胶囊条，点击胶囊即可在对话流插入对应卡片（上传 / 跳转 / SDK 调用 / 追问四种）。键盘弹起时自动隐藏。
5. **4 种卡片组件激活**：上传卡片、跳转卡片、SDK 调用卡片、追问卡片正式接入，按按钮类型自动调度。
6. **8 个新字段全部生效**：`prompt_template_id` / `external_url` / `preset_prompt` / `auto_user_message` / `card_title` / `card_subtitle` / `card_cover_image` / `button_sub_desc` 在卡片渲染中全部读取并使用。
7. **埋点真正打满**：宫格曝光、宫格点击、胶囊曝光、胶囊点击、卡片曝光、卡片按钮点击、表单提交、卡片失败 8 类事件全部上报。
8. **数据迁移**：旧 `icon_url` 数据已自动迁移到新 `icon` 字段，按按钮名关键字智能推荐 Emoji；`func_grid` 配置自动补全 `visible`/`max_count`/`cols`。

## 三、操作步骤

### 3.1 管理员：在「功能按钮管理」中配置一个新按钮

1. 打开 [功能按钮管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/function-buttons)。
2. 点击右上角「新建」按钮，弹出编辑窗。
3. 填入「按钮名称」（必填，最长 20 字）。
4. 点击「**选择 Emoji**」按钮，弹出 Emoji 选择面板：
   - 顶部有「推荐 Emoji」一行，会根据您填入的按钮名关键字自动推荐（如"用药提醒"会推荐 💊 🩺 ⏰）。
   - 也可通过分类 Tab 浏览 800+ Emoji，或直接搜索关键字。
   - 点击任一 Emoji 即可回填到输入框，无需手动复制。
5. 选择「按钮类型」（共 7 类：`medication_recognize` / `external_link` / `video_consult` / `quick_ask` / `prompt_template` / 等）。
6. 根据按钮类型，填写相应字段（表单会按类型条件展示）：
   - 卡片标题 / 副标题 / 封面图 / 主按钮说明 / 预设话术 / 跳转 URL / 关联模板 …
7. 设置「是否启用」与「排序权重」（数字越小越靠前）。
8. 点击「保存」即生效，**无需手动同步**到 H5。

### 3.2 管理员：调整宫格显示参数

1. 打开 [AI 对话首页配置](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-settings/ai-home-config)。
2. 点击「**功能宫格**」Tab。
3. 设置：
   - **是否显示宫格**：关闭后 H5 整个宫格区不显示。
   - **最大数量**：1~12，控制宫格最多显示几个按钮（按 sort_weight 升序取前 N）。
   - **列数**：2 / 3 / 4 列，控制每行布局。
4. 如需新增 / 删除 / 修改快捷指令，请点击「**前往功能按钮管理 →**」按钮跳转。
5. 点击下方「保存本 Tab」生效。

### 3.3 终端用户：在「AI 对话首页」点击宫格

1. 打开 [AI 对话首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)。
2. 在欢迎区下方看到「功能宫格」（按配置的列数和数量渲染）。
3. 每个宫格卡片显示：Emoji（36-40px）+ 按钮名（14px 加粗）+ 副说明（12px 灰字，若有）。
4. 点击任一宫格：
   - **追问类（quick_ask）**：自动将预设话术作为用户消息发送，AI 立即回复。
   - **跳转类（external_link，配置了 URL）**：直接跳转到目标页面。
   - **其他类（上传 / SDK 调用 / 带卡片的跳转）**：在对话流中插入一条 AI 消息卡片，您可点击卡片内的主操作按钮（如"上传照片"/"立即前往"/"立即开始"）。

### 3.4 终端用户：在「AI 对话详情页」使用胶囊条

1. 进入任意 AI 对话会话页（如：https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/chat/your-session-id ）。
2. 输入框上沿可见一条**横向单排可滑动**的胶囊条，包含全部启用的快捷指令。
3. 胶囊样式：浅灰底全圆角，Emoji 16px + 按钮名 13px。
4. 操作：
   - **左右滑动**：胶囊条超出屏宽时可滑动查看全部胶囊。
   - **点击胶囊**：在当前对话流中插入对应卡片（不新建会话），按类型自动调度。
   - **聚焦输入框（键盘弹起）**：胶囊条整体隐藏，让出空间。
   - **键盘收起**：胶囊条恢复显示。

### 3.5 终端用户：与对话流中的卡片交互

四种卡片均作为 AI 消息气泡渲染在对话流中：

| 卡片类型 | 主按钮 | 行为 |
|---|---|---|
| **上传卡片**（upload） | 上传照片 | 弹出图片选择器 → 上传 → 自动识药并把结果作为下一条 AI 消息显示 |
| **跳转卡片**（navigate） | 立即前往 | 跳转到 `external_url`（站内 router.push / 站外 window.location.href） |
| **SDK 调用卡片**（sdk_call） | 立即开始 | 触发对应 SDK（如视频客服），完成回调后插入下一条 AI 消息 |
| **追问卡片**（quick_ask） | 立即提问 | 把 `preset_prompt` 作为用户消息发送，AI 立即回复 |

主操作按钮**点击一次后会置灰**，再次触发同一功能请再点一次胶囊（会插入新的卡片消息）。

## 四、注意事项

1. **数据源唯一性**：本次起，H5 端的快捷指令内容（图标 / 名称 / 类型 / 8 字段）**只来源于「功能按钮管理」**。「AI 对话首页配置 → 功能宫格」只控制显示参数，不再编辑内容。
2. **宫格 vs 胶囊数量差异**：
   - 宫格：受 `max_count` 限制（默认 6，可调 1~12）。
   - 胶囊条：不限数量（全部启用都显示，多就左右滑）。
   - 如希望隐藏某个按钮，请在「功能按钮管理」把 `is_enabled` 关掉，**不通过前端**做隐藏。
3. **Emoji 必填**：新建 / 编辑功能按钮时，Emoji 图标必填（旧数据已由迁移脚本自动填好，最差兜底 📌）。
4. **公开接口不暴露已禁用按钮**：`/api/function-buttons?is_enabled=true` 仅返回启用按钮，运营草稿 / 临时关闭按钮**不会**被泄露。
5. **键盘 focus 监听**：极端机型可能未触发，胶囊条会保留显示，不会影响输入。
6. **低版本系统 Emoji 显示**：iOS 13- / Android 70- 老机器可能把部分新 Emoji 显示为方块（行业通病），业务可接受。
7. **回滚兼容**：旧 `icon_url` DB 列保留（不写入），`func_grid.items` 保留（不读取）；如需回退，可通过数据库恢复旧字段。

## 五、常见问题（FAQ）

**Q1：我在功能按钮管理新增了一个按钮，H5 看不到？**
A：① 确认按钮的「是否启用」是开启的；② 确认按钮的「Emoji」字段已选；③ 如显示在宫格中需 `sort_weight` 排序靠前并在 `max_count` 范围内；④ 刷新 H5 页面。

**Q2：宫格区什么时候显示"内置 3 项兜底"？**
A：当后端 `/api/function-buttons?is_enabled=true` 接口异常或返回空数组时，宫格区显示内置 3 项（AI 问诊 / 体检报告 / 健康档案）。胶囊条则在此场景下整体不显示。

**Q3：胶囊条点击后没反应？**
A：① 打开浏览器控制台看是否有 JS 报错；② 确认按钮的 `button_type` 是 7 种之一；③ 如是 `quick_ask` 类型，确认 `preset_prompt` 字段已填写。

**Q4：旧的 icon_url（图片 URL）数据怎么办？**
A：本期上线后已通过启动期幂等迁移脚本自动转换：根据按钮名关键字推荐对应 Emoji 填入新 `icon` 字段；全部未命中时填默认 📌。运营可以在「功能按钮管理」按需手动调整。

**Q5：埋点上报失败会影响业务吗？**
A：不会。所有埋点调用都包裹了 try/catch，失败时静默忽略，不影响主流程。
