## 功能简介

本次更新围绕「让 AI 健康管家更主动、更专业」这一目标，对 AI 对话首页做了三项体验优化，并新建了一个用药提醒管理页：

1. **顶部更紧凑**：标题栏「☰ 小康」整体靠左，告别"飘"在中间的视觉感
2. **🔔 提醒替代健康打卡**：原"健康打卡"入口升级为「🔔 提醒」铃铛，点击弹出**今日待办**抽屉，一次看完今日要吃的药 + 待核销的预约
3. **AI 回答档案胶囊**：AI 每条回答顶部内嵌一个浅灰胶囊「回答结合 XX 档案 ⌄」，点击展开档案详情，与回答正文融为一体（参考晓医风格）
4. **用药提醒管理页**（新建）：独立页面承载用药计划的增/删/改/启停，未来可扩展运动、喝水等其他类型提醒

历史「健康打卡」页面已下线，老链接进入会自动跳转到 AI 首页并提示「健康打卡已升级为提醒」。

## 使用说明

### 1️⃣ 顶部标题栏（更紧凑）

打开 AI 对话首页，可以看到顶部的「☰」汉堡图标和「小康」标题之间的间距已经收紧到 8px，整体靠左显示。右上角的「⋯」更多按钮位置不变。

### 2️⃣ 🔔 提醒入口与今日待办抽屉

- 在 AI 对话首页右下角，原来的「健康打卡」浮动按钮**已被原位替换**为 **🔔 提醒** 铃铛按钮
- 当有未处理事项时（用药未打卡 + 待核销订单），铃铛右上角会出现**红色圆形数字徽标**（10 条及以上显示「9+」，0 条不显示徽标）
- 点击铃铛 → 从屏幕底部滑出**「今日待办」抽屉**，无 Tab，上下两段：

#### 📦 上段：用药提醒
- 标题右侧显示「未打卡数」红色徽标，以及「⚙ 设置」按钮
- 列表每条显示：
  - 💊 药品名称
  - 🕐 服用时间（如 08:00、14:00、20:00）
  - 💉 剂量（如「1 片」「5ml」）
  - 📝 备注（可选，如「饭后服用」）
  - 右侧 **「✓ 已服用」** 按钮
- 点击「✓ 已服用」→ 按钮变灰、文字变为「已服用 HH:MM」（显示打卡时间），整条变为弱化样式
- **可撤销**：再次点击该条 → 取消打卡，恢复未打卡样式
- 点击「⚙ 设置」→ 跳转到「**用药提醒管理页**」（详见 4️⃣）

#### 📅 下段：预约提醒
- 标题右侧显示「待核销订单数」红色徽标，以及「⚙ 设置」按钮
- 列表每条显示：
  - 服务名称（如「专家号 · 心内科 · 张医生」）
  - 预约时间（如「2026-05-12 09:30」）
  - 服务地点 / 形式（线下科室名 / 线上视频）
  - 状态标签：「待核销」
- **点击单条预约** → 抽屉**不关闭**，**就地在该条下方展开订单详情卡片**（手风琴式，仅一条可同时展开）。再次点击该条或点击其他条则切换
- 点击「⚙ 设置」→ 跳转到「**我的预约订单列表页**」

#### 🧹 抽屉关闭
- 下拉抽屉头部、点击灰色遮罩、点击右上角关闭按钮均可关闭抽屉
- 关闭时所有展开项自动收起，下次打开默认全部收起

### 3️⃣ AI 回答档案胶囊

- 当你已选定咨询对象（如自己 / 妈妈 / 孩子等家庭成员）后，**每条 AI 回答气泡顶部**会出现一个浅灰胶囊条「回答结合 XX 档案 ⌄」
- **默认收起**，点击胶囊 → 就地展开档案详情（咨询对象的健康档案信息），不跳页
- 再次点击 → 收起。「⌄」图标在展开后旋转 180° 变成「⌃」
- 用户消息气泡不显示档案胶囊
- 未选定咨询对象的回答不显示档案胶囊

### 4️⃣ 用药提醒管理页（新页面）

入口：
- 「今日待办」抽屉 → 用药提醒区块右上角「⚙ 设置」
- 也可直接访问：[/medication-plans](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/medication-plans)

页面元素：
- 顶部返回按钮 + 标题「用药提醒管理」
- 用药计划卡片列表（每张卡片包含：药名、服用时间、剂量、备注、启停开关、编辑、删除）
- 右下角「+ 新增用药计划」按钮
- 空态：「还没有用药计划，点击右下角添加一条」

#### 新增用药计划
1. 点击右下角「+ 新增用药计划」
2. 在弹出的表单中填写：
   - **药品名称**（必填）：如「美托洛尔缓释片」
   - **剂量**（必填）：如「1 片」「5ml」
   - **服用时间**（必填）：可添加多个时间点，如 08:00 / 14:00 / 20:00
   - **备注**（可选）：如「饭后服用」「忌酒」
   - **启停**：默认启用
3. 点击「保存」→ 计划立即出现在列表中，并在「今日待办」抽屉的用药提醒中显示

#### 编辑/删除/启停
- 编辑：点击卡片上的「编辑」按钮 → 复用同一表单
- 删除：点击「删除」按钮 → 确认后从列表中移除（同时移除今日待办中的该条目）
- 启停：切换卡片上的「启用/停用」开关。停用后该计划不再进入今日待办，但保留计划数据，可随时再次启用

### 5️⃣ 健康打卡页面已下线

- 原「健康打卡」入口已彻底从首页移除
- 老链接 [/health-plan/checkin](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-plan/checkin) 进入会自动跳转到 AI 首页并 Toast 提示「健康打卡已升级为提醒」
- 历史打卡数据**保留在数据库中**，不会丢失，但前端不再展示

## 注意事项

### 数据隔离与权限
- 所有用药提醒、订单数据按当前登录用户隔离，**严禁跨用户读取**
- 家庭成员代管理场景下：在已选定咨询对象的会话中操作，需校验家庭关系，避免越权访问他人数据
- 用药计划的增删改查、打卡操作均走鉴权后端接口

### 徽标计算规则
- 🔔 铃铛徽标 = 用药未打卡条数（今日） + 待核销订单条数
- 用药提醒区块徽标 = 用药未打卡条数（今日）
- 预约提醒区块徽标 = 待核销订单条数
- 三者上限均为 **9+**，等于 0 时不显示

### 空态规则
- 用药提醒为空 → 上段显示空态文案「暂无用药提醒」，下段正常渲染
- 预约提醒为空 → 下段显示空态文案「暂无预约提醒」，上段正常渲染
- 两段都为空 → 各自显示各自的空态文案（不再合并为整体空态）

### 跨日凌晨自动重置
- 每次打开抽屉时，前端会拉取「今日」的最新打卡状态，凌晨 0 点跨日后会自动重置「✓ 已服用」标记
- 若抽屉持续打开期间跨日，建议关闭后重新打开以拉取最新状态

### 异常处理
- 用药打卡接口失败 → 自动回滚已变灰的样式 + Toast 提示「打卡失败，请重试」
- 预约订单接口失败 → 区块显示加载失败占位 + 重试按钮，不影响用药区块渲染
- 抽屉打开期间断网 → 显示离线提示，已加载的数据保留可读
- 用药计划被外部端删除 → 抽屉刷新时该条目自动消失，已打卡日志保留作历史记录

### 兼容性
- 已兼容主流移动端浏览器（iOS Safari ≥ 14、Android Chrome ≥ 90）
- 已兼容微信内嵌浏览器
- 抽屉、胶囊、徽标在 320px ~ 480px 屏宽下均无错位

## 常见问题

**Q：原来的健康打卡数据丢了吗？**  
A：没有。历史打卡数据完整保留在数据库中，仅前端不再展示。后续如需数据回溯可联系运营。

**Q：用药打卡和原来的健康打卡是同一套数据吗？**  
A：**不是**。用药提醒是新起的一套数据（`medication_plans` + `medication_logs` 两张表），与原健康打卡完全独立、互不影响。

**Q：我可以为家人（家庭成员）单独设置用药提醒吗？**  
A：可以。在用药提醒管理页新增计划时，会自动关联当前会话的咨询对象。在不同咨询对象之间切换时，今日待办和管理页的数据会自动按对象隔离展示。

**Q：未登录用户能看到提醒入口吗？**  
A：未登录用户首页不展示「🔔 提醒」徽标的数字；点击图标会弹出登录引导而非抽屉。

**Q：预约订单列表为什么有的不显示？**  
A：「预约提醒」严格只展示**状态 = 待核销**的订单（`status = pending_use` 或 `appointed`）。待支付、已核销、已取消、已退款等状态的订单不会进入此列表，可在「我的预约订单」页面查看完整列表。
