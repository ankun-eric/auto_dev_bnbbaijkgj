## 功能简介

本次修复将「健康档案 PRD v1.0」的设计完整落地，覆盖 6 处此前完全未生效的硬伤：

1. **【P0】入口卡跳转切换**：健康档案首页本人 Tab 的「健康档案列表」卡片，原本仍跳到旧的 `i-guard` 页面，现已统一跳转到新版「档案列表」（archive-list）。卡片名称同步改为「档案列表」。
2. **【P0】旧 IGuard 页面整体下线**：所有指向 `/health-profile/i-guard/` 的入口（书签、外链、二维码）会自动跳转到新版档案列表。原本旧页面顶部「+ 发起邀请」大按钮、卡片上无视 S2 才显示规则的「发起邀请」按钮已经一并消失。
3. **【P0】会员中心配额文案修正**：原文案「可管理健康档案 X+1 份（含本人）」已经修正为「可管理健康档案 X 人（上限 Y 人，不含本人）」，与 PRD §5.2 完全一致。
4. **【P1】档案列表排版修复 + S0 本人卡 AI 外呼额度入口**：卡片右下角主按钮的 `width:100%` 强制全宽问题已修复（不再和「⋯」按钮挤压）；本人卡 S0 状态下新增「📞 AI 外呼额度」次按钮，点击弹抽屉展示常规外呼/紧急呼叫的已用/总额/剩余三组数据，并提供「前往会员中心」的升级入口。
5. **【P1】健康档案首页「去邀请 ›」大按钮删除**：根据 PRD §4.1，所有发起邀请的入口都收口到「档案列表」中按 7 态状态机决定是否在卡片上显示。健康档案首页顶部的常驻大按钮已删除。
6. **【P2】AI 首页「+ 新建咨询人」改造**：在 AI 首页咨询人选择器中点击「+ 新增咨询人」建档成功后，会自动弹出「立即去邀请」抽屉，引导用户进入档案列表完成邀请流程。

此外还顺手补齐了三个体验细节：

- **邀请记录改为本页内抽屉**：原 archive-list 的「邀请记录」点进去会跳到旧 i-guard 页（已下线），现已改成本页内的抽屉，自动从后端拉取该成员的全部邀请历史。
- **S4/S5/S6/S7 卡片不再误显示「剩余 Xh」**：仅在 S3「待接受」且邀请码仍有效时才展示剩余时长。
- **统一删除接口的 reason_code 错误提示**：删除按钮在 S1（守护中）/ S3（邀请中）/ 有绑定设备等场景会按服务端返回的结构化原因显示精准提示。

## 使用说明

### 1. 入口：从健康档案首页进入档案列表

1. 打开 [H5 主入口](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/) 完成登录。
2. 进入「健康档案」（底部 Tab 或 AI 首页菜单）。
3. 在本人 Tab 看到的左侧白色卡片名称应为「档案列表」（不再是「健康档案列表」）。卡片副标题为「{X} 人 / 上限 {Y} 人（不含本人）」。
4. 点击卡片，**直接进入新版档案列表**（archive-list），不再去旧的 IGuard 页面。

### 2. 档案列表页核心交互

进入 [档案列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/archive-list/) 后：

- **顶部统计区**：左侧显示「📋 档案列表 · 已管理 X 人 / 上限 Y 人 · 还可添加 N 人」，右侧是「+ 新增」按钮（自适应宽度，不再挤压）。
- **卡片流**：每个家庭成员一张卡片，按 7 态状态机渲染：
  - **S0 本人**：右下角同时显示「📞 AI 外呼额度」（次按钮）+「查看档案」（主按钮）。
  - **S1 守护中**：状态标签蓝色，主按钮「查看档案」，「⋯」菜单含「解除守护」（不含「删除」）。
  - **S2 未绑定**：状态标签灰色，主按钮「去邀请」，「⋯」菜单可「删除」。
  - **S3 待接受**：状态标签橙色，主按钮「查看邀请码」，副标题显示「· 剩余 Xh」，「⋯」菜单含「取消邀请」（不含「删除」）。
  - **S4 已拒绝 / S5 已解绑**：状态标签红色/灰色，主按钮「重新邀请」，「⋯」菜单可「删除」。
  - **S6 已过期 / S7 已取消**：状态标签浅灰，主按钮「重新邀请」，「⋯」菜单可「删除」。

### 3. 新建档案 + 立即邀请

1. 点击右上角「+ 新增」按钮（如果配额已满会提示「档案配额已满，请先删除现有档案」）。
2. 填写姓名、关系、手机号、性别后保存。
3. 保存成功后立即弹出「立即去邀请？」抽屉：
   - 点击「立即邀请」 → 后台立刻为该档案创建邀请并展示邀请码 + 分享链接，可点击「复制邀请链接」。
   - 点击「暂不邀请」 → 留在档案列表，稍后可在该卡片上点「去邀请」重新发起。

### 4. 本人卡 AI 外呼额度抽屉

1. 在档案列表的 S0 本人卡片右下角点击「📞 AI 外呼额度」按钮。
2. 抽屉弹出，自动从后端拉取当前套餐的额度信息：
   - 当前套餐名称
   - 常规 AI 外呼：已用 X / 总额 Y / 剩余 Z
   - 紧急 AI 呼叫：已用 X / 总额 Y / 剩余 Z
3. 点击「前往会员中心」可一键跳转到会员中心查看升级套餐。

### 5. 邀请记录抽屉

1. 在任意有过邀请历史的成员卡片上点击「⋯」，选择「邀请记录」。
2. 抽屉中按时间倒序展示全部邀请记录：邀请码、状态、发起时间、过期/接受/取消时间。
3. 不再跳转到任何其他页面，**完全在档案列表页内完成**。

### 6. 会员中心新配额文案

打开 [会员中心](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/member-center/) 后，每个套餐卡片下方的描述行应显示：

> 可管理健康档案 **X 人（上限 X 人，不含本人）** · AI 外呼 N 次 · 紧急呼叫 M 次

而不是旧版的「可管理健康档案 X+1 份（含本人）」。

### 7. AI 首页咨询人选择器 — 立即去邀请

1. 在 [AI 首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home/) 顶部点击当前咨询人头像/名称，打开「选择咨询人」抽屉。
2. 拉到底部，点击「+ 新增咨询人」按钮。
3. 在弹出的「新增家庭成员」表单中填写信息并保存。
4. 保存成功后立即弹出「立即去邀请？」抽屉：
   - 点击「立即去邀请」 → 关闭选择器，自动跳转到档案列表，您可以在新建的卡片上发起邀请。
   - 点击「暂不邀请」 → 关闭抽屉，留在咨询人选择器。

## 注意事项

1. **缓存刷新**：H5 通过 Nginx 反向代理，Next.js 静态资源带有 hash 后缀，因此**普通刷新即可看到新版本**。如果手机浏览器/微信中仍是旧版本，请尝试「下拉刷新」或「关闭后重新打开」。
2. **旧 i-guard 链接**：所有指向 `/health-profile/i-guard/` 的旧链接（包括二维码、邀请短信内嵌链接）现在会自动客户端跳转到 `/health-profile/archive-list/`，**用户无需做任何手动迁移**。
3. **AI 外呼额度抽屉**：如果后端额度接口尚未提供精确字段，抽屉会显示 `-` 兜底，并仍然提供「前往会员中心」入口。
4. **删除规则保持不变**：S1（守护中）需要先「解除守护」、S3（邀请中）需要先「取消邀请」、有绑定设备时需要先解绑设备，**不可强删**。这些规则由后端通过 `reason_code` 返回结构化错误，前端按文案精确提示。
5. **配额计算口径**：`X = count(family_member where deleted_at is null and is_self=false)`，**不再 +1 含本人**。所有副标题、统计条、会员中心配额文案都已统一。
