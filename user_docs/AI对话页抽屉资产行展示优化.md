## 功能简介

本次对 **AI 对话页左上角抽屉**中的「积分 / 优惠券 / 订单 / 收藏」四格资产展示进行了视觉、口径和顺序三方面的优化：

1. **视觉风格统一**：四格全部采用「上方大号数字 + 下方小字文案」的统一风格，**去除原本优惠券和订单格的 emoji 图标（🎫 / 📦）**，告别"两格有图标、两格没图标"的视觉混乱。
2. **订单数字口径对齐**：抽屉「订单」格数字 ≡ 订单列表「待收货 Tab + 待使用 Tab」之和，杜绝以前出现过的"待收货 1 + 待使用 7 共 8 条，但抽屉却显示 6"这种数字对不上的偏差问题。
3. **顺序优化**：四格顺序由 `积分 / 优惠券 / 订单 / 收藏` **调整为** `积分 / 优惠券 / 收藏 / 订单`，把高频访问的「订单」放在最右侧便捷位置。
4. **订单点击智能定位 Tab**：点击「订单」格不再单纯跳到"全部 Tab"，而是根据您当前待收货 / 待使用的数量自动定位到对应 Tab，减少手动切 Tab 的成本。

## 使用说明

### 一、进入抽屉并查看资产行

1. **打开 H5 主页**：在浏览器中访问 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)（需先登录账号）。
2. **打开抽屉**：点击页面**左上角的 ☰ 图标**，从左侧滑出抽屉。
3. **观察资产行**：抽屉顶部用户名片块下方就是一行 4 格的资产展示，顺序为：

   ```
   [ 积分 ]   [ 优惠券 ]   [ 收藏 ]   [ 订单 ]
      120         3            5         2
   ```

   每格都是 **「大号数字 + 下方小字文案」** 的统一样式，没有任何 emoji 图标。

### 二、数字展示规则

不同格按以下规则展示数字：

| 范围 | 积分 | 优惠券 / 收藏 / 订单 |
|---|---|---|
| 0 | 显示真实值 | 显示 `0` |
| 1 ~ 9 | 显示真实值 | 显示真实数字 |
| ≥ 10 | 显示真实值（例如 `120`、`9999`） | 显示 `9+` |

> 解释：积分通常为较大数值，按真实值展示更直观；优惠券、收藏、订单是"提醒类"指标，超过 10 一律用 `9+` 截断显示更清爽。

### 三、点击各格的跳转目标

| 格子 | 点击跳转 |
|---|---|
| 积分 | 积分中心 `/points-center` |
| 优惠券 | 我的券包 `/my-coupons` |
| 收藏 | 我的收藏 `/my-favorites` |
| 订单 | 我的订单 `/unified-orders` **+ 智能定位 Tab**（详见下方） |

### 四、订单格点击的智能定位规则

点击「订单」格时，会根据您当前待收货与待使用订单的数量，自动选中最合适的 Tab：

| 条件 | 默认进入的 Tab |
|---|---|
| 待收货 > 待使用 | **待收货 Tab** |
| 待收货 < 待使用 | **待使用 Tab** |
| 待收货 == 待使用 且 都 > 0 | **待收货 Tab**（业务紧急度更高） |
| 待收货 == 0 且 待使用 == 0 | **全部 Tab**（无紧急事项，看完整列表） |

实际效果举例：
- 待收货 1 单、待使用 7 单 → 抽屉显示 `8`，点击进订单列表 → 自动停在「待使用」Tab；
- 待收货 3 单、待使用 3 单 → 抽屉显示 `6`，点击进订单列表 → 自动停在「待收货」Tab；
- 待收货 0、待使用 0 → 抽屉显示 `0`，点击进订单列表 → 停在「全部」Tab。

### 五、验证抽屉数字与订单列表对得上

为了让您可以亲自验证"口径已对齐"：

1. 打开抽屉，记下「订单」格数字（假设是 5）。
2. 点击「订单」格，跳转到 [统一订单列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/unified-orders)。
3. 查看顶部 Tab 上的数字：「待收货 Tab」上的数字 + 「待使用 Tab」上的数字 = 5，**应当恒等于抽屉里的「订单」格数字**。
4. 如果三处对不上，请截图反馈给后台运维。

## 注意事项

1. **必须先登录账号**：未登录状态下打开 AI 对话页会被引导到登录页；只有登录后抽屉才会显示真实的资产数据。
2. **首次打开略有加载延迟**：抽屉中的积分 / 优惠券 / 订单 / 收藏 四格数据是**分别独立请求**后端接口，进入抽屉时可能有 1~2 秒的"先显示 0、再被真实数字替换"的过程，属于正常现象。
3. **任一接口失败仅影响单格**：如果某一格的后端接口异常，仅该格保持 `0` 显示，**不会**影响其他三格的正常加载。
4. **订单数字口径与 PC 端可能不一致**：本次仅优化了 H5 端抽屉的展示口径，如有 PC / 小程序 / APP 端的相同入口，数字口径与本次抽屉**不一定完全相同**，请以本次手册描述的 H5 抽屉为准。
5. **9+ 截断仅用于显示**：实际的真实数量可在点击进入对应列表页（券包 / 收藏列表 / 订单列表）后看到精确值。
