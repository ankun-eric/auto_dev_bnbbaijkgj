## 使用说明

### 场景一：用户在 H5 / 小程序 / APP 下单时使用优惠券

1. 进入任意商品详情页，点击「立即购买」进入下单页
2. 找到「优惠券」一栏，点击进入券选择弹窗
3. 现在弹窗中**只会显示对本单真正可用的券**：
   - 已过期的券 → 不出现
   - 已下架的券 → 不出现
   - 适用其他商品的券 → 不出现
   - 不满足门槛的券 → 不出现（免费试用券除外）
4. 选择想用的券，圆圈点亮即选中
5. 若选中的是「免费试用券」，会看到应付金额自动变成 ¥0
6. 点击「立即支付」完成下单

### 场景二：后台管理员创建免费试用券

1. 用管理员账号登录 [后台管理页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 进入「商品系统 → 优惠券」
3. 点击「新增优惠券」
4. 填写券名称
5. 「优惠券类型」选择「**免费试用**」
6. 这时你会发现页面上**「使用门槛金额」「优惠金额」「折扣率」三个字段全部隐藏**
   - 同时弹出蓝色提示：*"免费试用券：凭券免费试用指定商品，整单按 0 元结算，无需设置门槛金额和优惠金额。建议搭配「指定商品」使用并设置发行总量、单人限领，避免被刷。"*
7. 「适用范围」建议选「**指定商品**」，挑选要试用的商品
8. 设置「发行总量」「单人限领」（防风控刷券）
9. 点击「确定」完成创建

### 场景三：圆圈对齐效果验证

打开任意端的下单页，在弹出的优惠券选择弹窗中，可以看到：

- **H5 端**：每张券前的圆圈与文字主体（券名 + 副文字）整体垂直居中
- **小程序端**：弹窗里每张券前**新增了 radio-dot 圆点指示器**，与金额、文字三者垂直居中
- **Flutter Android / iOS**：每张券前的 Material Radio 圆点与标题、副标题整体居中

三端视觉完全统一。

## 注意事项

1. **本次的"展示层屏蔽"为非破坏性修复**：后端不会主动清理已发放的"不适用"券，仅在下单页接口层面把它们过滤掉。已发到用户钱包的券都还在，可在「我的优惠券」里看到。
2. **创单兜底（防绕过）**：即便用户绕过前端、直接调用 `/api/orders/unified` 提交一张不适用的券，后端会返回 **422 «优惠券不适用本单»**，订单不会创建成功。这是一道安全闸门。
3. **免费试用券风控**：建议在后台为免费试用券同时设置：「适用范围 = 指定商品」 + 「发行总量上限」 + 「单人限领 1 张」。否则可能被薅羊毛。
4. **历史脏数据**：之前误填了 condition_amount > 0 的免费试用券记录会被本次新逻辑自动按"整单 0 元"结算，无需手工修正。
5. **客户端版本提示**：未升级到本次新版的旧 APK / 旧小程序仍会调用老接口 `/api/coupons/mine`，会看到全部领过的券（不过滤）。**强烈建议下载最新版本**，体验本次的过滤效果。
