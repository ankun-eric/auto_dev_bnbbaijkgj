## 一、本次优化做了什么

我们把【健康档案首页】里的血糖卡片做得跟血压卡片一模一样，并且重做了一个全新的【血糖详情页】，把原来独立的【血糖管理】整页**物理下线**，所有血糖相关功能都从「档案首页 → 血糖卡片」一条路进入。具体亮点：

1. **🩸 血糖卡片对齐血压**：标题、严重程度胶囊、主数值、测试时间、测试手段（设备测量 / 手动录入）全部和血压卡片同款排版，看起来更整洁。
2. **🆕 全新血糖详情页**：参考血压详情页布局，包含——
   - 顶部主卡片（数值 + 严重程度 + 来源胶囊）
   - 📏 目标范围参考（空腹 3.9–6.1、餐后2h <7.8、睡前 4.4–6.7，使用医学标准，用户不可修改）
   - 🤖 AI 解读本次血糖（点击触发，抽屉弹出）
   - 📈 趋势图（今天 / 7天 / 30天 三档，默认 7 天；空腹蓝/餐后橙/睡前紫**三色分线**）
   - 🤖 AI 解读趋势（点击触发，抽屉弹出）
   - 📋 历史记录（点击任意记录可修改测量类型，严重程度会自动重算）
   - ⚙️ 更多功能（血糖预警、测量提醒、健康报告入口）
3. **➕ 录入入口统一**：进入详情页后，右上角「+录入」按钮打开抽屉式录入界面，包含「手动录入」/「设备读取」两个 Tab；未绑定设备时显示「去绑定」按钮一键跳转。
4. **🗑 旧整页下线**：原来的 `/glucose` 整页已经被替换成跳转兜底，访问会自动跳到健康档案首页。

## 二、操作步骤详解

### 2.1 在健康档案首页查看血糖卡片

1. 打开 [健康档案](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile)
2. 在「今日健康数据」区域，看到 2×2 的指标卡片，左上是「💓 血压」，右上是「🩸 血糖」
3. 血糖卡片字段说明：
   - **标题**：🩸 血糖
   - **严重程度胶囊**（彩色，右上角）：偏低 / 正常 / 偏高 / 危象
   - **主数值**：最新一次测量数值 + `mmol/L` 单位
   - **时间·来源行**（卡片底部）：例如「今日 11:30 · 设备测量」
4. **点击卡片** → 自动进入【血糖详情页】

### 2.2 进入血糖详情页

详情页从上到下依次为：

#### ① 主卡片
- 显示最新一次的数值（超大字号）
- 时间·来源行（例：今日 11:30 · 手动录入）
- 严重程度胶囊 + 来源胶囊（🔵 设备测量 / ⚪ 手动录入）

#### ② 目标范围参考
- 空腹 3.9–6.1 mmol/L
- 餐后 2h < 7.8 mmol/L
- 睡前 4.4–6.7 mmol/L

#### ③ AI 解读本次血糖
- 点击「🤖 AI 解读本次血糖」按钮 → 弹出抽屉
- 抽屉中显示基于本次血糖的个性化建议
- 同一条记录 5 分钟内点击会直接返回缓存结果

#### ④ 趋势图
- **档位切换**：右上角三个 Tab：【今天】【7天】【30天】，默认选中 7 天
- **三色分线**：空腹（深蓝）、餐后2h（橙）、睡前（紫）三条线分别绘制
- **参考虚线**：3.9（灰）/ 7.8（橙）/ 11.1（红）目标线
- **空数据**：显示提示「暂无数据，点击右上角录入」
- 点击「🤖 AI 解读趋势」可得到针对所选时间段的整体趋势建议

#### ⑤ 历史记录
- 每条记录展示：测量时间 · 数值 · 测量类型胶囊 · 测试手段胶囊 · 严重程度胶囊
- **点击记录** → 弹出"修改测量类型"窗口；修改后严重程度自动重算

#### ⑥ 更多功能（即将上线）
- 🔔 血糖预警设置
- ⏰ 测量提醒
- 📄 健康报告

### 2.3 录入一条血糖数据

1. 进入详情页后，点击右上角「**+ 录入**」按钮
2. 从底部弹出录入抽屉，顶部有两个 Tab：

   **A. 手动录入**（默认）
   - 数值：输入数字（合理范围 0.5–35 mmol/L）
   - 测量类型：选择 ○空腹 / ●餐后2h / ○睡前 / ○随机
   - 备注：选填
   - 点击「保存」

   **B. 设备读取**
   - 顶部展示绑定状态：
     - 若已绑定血糖仪：自动带出数值（蓝色虚线框提示），用户选测量类型后保存
     - 若未绑定：显示「暂未绑定设备 → 去绑定」按钮，点击跳转到【我的设备】页

### 2.4 修改某条历史记录的测量类型

例：你把"7.5 mmol/L"错误标成了「餐后2h」（在餐后看是"正常"），其实它是空腹测的（在空腹看就是"严重偏高"了）。

1. 在详情页【📋 历史记录】区域，点击对应那一条
2. 弹窗中选择正确的测量类型（如「空腹」）
3. 点击「保存」→ 系统会自动按新场景重新计算严重程度

## 三、严重程度判定规则（医学标准）

按**最新一条记录**对应的测量类型套用以下标准（单位 mmol/L）：

| 测量类型 | 偏低 | 正常 | 偏高 | 危象 |
|---|---|---|---|---|
| 空腹 | < 3.9 | 3.9 – 6.1 | 6.1 – 7.0 | > 7.0 或 < 2.8 |
| 餐后2h | < 3.9 | 3.9 – 7.8 | 7.8 – 11.1 | > 11.1 或 < 2.8 |
| 睡前 | < 4.4 | 4.4 – 6.7 | 6.7 – 10.0 | > 10.0 或 < 2.8 |
| 随机 | < 3.9 | 3.9 – 11.1 | 11.1 – 16.7 | > 16.7 或 < 2.8 |

颜色档与血压一致：
- 🔵 正常 → 蓝色胶囊
- 🟡 偏低 / 偏高 → 黄色胶囊
- 🟠 危象 → 橙色胶囊（同时触发预警事件并通知守护人）

## 四、注意事项

| 注意项 | 说明 |
|---|---|
| 历史数据已重置 | 本次上线时已按 PRD §五要求，**物理删除所有用户历史血糖记录**（含告警事件、health_metric_record 中的 blood_glucose 子集）。库快照已做，保留 30 天，需要回滚请联系运维 |
| 测试手段判定规则 | 按**最新一条记录的 source 字段**判定：含 `device` / `glucometer` 显示 🔵 设备测量；其它显示 ⚪ 手动录入 |
| 设备自动同步默认类型 | 设备自动同步的记录默认测量类型为「随机」；用户可在历史记录中点开修改为更精确的类型，严重程度会自动重算 |
| AI 建议仅供参考 | AI 解读基于规则引擎给出，**不能替代医生诊断**，重要异常请及时就医 |
| 旧 `/glucose` 链接 | 不再可用作功能页，但访问会自动跳到健康档案首页，不会出现 404 死链 |
| 卡片不显示测量类型 | 健康档案首页的血糖卡片**不显示**「空腹/餐后/睡前」小标签（保持与血压卡片字段一致），完整测量类型信息保留在详情页历史记录中 |

新增/修改端点：
- `GET /api/glucose-v1/latest`（最新一条血糖记录，用于卡片展示）
- `PATCH /api/glucose-v1/records/{id}/scene`（修改测量类型并重算 level）
- `POST /api/glucose-v1/admin/purge-all?confirm_token=...`（一次性清空所有用户血糖数据，运维用）
