## 功能简介

本次健康档案页面进行了一次大规模收敛优化，目标是**简化入口、统一统计口径、补齐转让链路、净化非本人 Tab、下线旧页面与旧接口**。具体如下：

1. **「我守护的人」统计数字改为「所有」**：卡片右上角的人数现在统计**所有 status** 的守护记录（包括 active + cancelled），与用户预期一致。
2. **「我守护的人」卡片一步直达新页面**：旧页面 `/guardian-system` 已下线，卡片点击直接跳转至 `/health-profile/i-guard`，无中转。
3. **非本人 Tab 净化展示**：查看 TA 的健康档案时，「我守护的人」卡片**完全隐藏**，腾出版面。
4. **非本人 Tab 改名 + 只读**：「守护我的人」在非本人 Tab 改名为「守护 TA 的人」，所有写操作（邀请/移除/转让/解除）全部隐藏，点击仅弹出只读详情。
5. **转让链路补齐**：`/health-profile/i-guard` 顶部新增「待确认转让」横幅，**接收方**可同意 / 拒绝，**发起方**可取消，覆盖完整的转让闭环。
6. **i-guard 列表按状态分组**：默认按「守护中（active）/ 待守护（其他状态）」两态分组展示，原 v13 试验页已合并并下线。
7. **历史代码统一下线**：旧前端页面 `/guardian-system` 与 `/health-profile/v13`、旧 C 端后端接口 `/api/guardian/list`、`/api/guardian/i-guard`、`/api/guardian/transfer/*`、`/api/guardian/invitations/records`、`/api/guardian/alert-quota`、`/api/guardian/priority`、`/api/guardian/alert/simulate-serial-call` 全部移除。`admin-web` 后台守护关系查询（`/api/admin/guardian/relations`）继续保留。

## 使用说明

### 1. 查看「我守护的人」统计总数

1. 在浏览器中打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile) 并登录账号。
2. 默认进入「本人 Tab」（顶部头像被选中时 = 本人）。
3. 找到「我守护的人」卡片，右上角数字现在等于您**所有守护关系**的总条数（含主守护人 / 普通守护人 / 已解除关系等）。

### 2. 一步直达「我守护的人」新页面

1. 在本人 Tab，点击「我守护的人」卡片任何位置。
2. 会**直接跳转**到 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard)，不再经过任何旧页面或"体验全新"按钮中转。
3. 列表分两栏显示：
   - **✅ 守护中**：当前正在生效的守护关系，可进行「查看档案 / 提醒设置 / 守护管理」三类操作。
   - **⏸ 待守护 / 已解除**：non-active 状态的历史记录，只展示信息，不可操作。

### 3. 处理「主守护人转让」待确认

转让分两个角色：

#### 3.1 接收方（您被指定为新主守护人）

1. 进入 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/i-guard)。
2. 页面顶部会出现橙色横幅：「🔔 XXX 申请将 YYY 的主守护人转让给您」。
3. 点击「同意」按钮：您立即成为主守护人，横幅消失。
4. 点击「拒绝」按钮：申请作废，发起方将收到拒绝通知。

#### 3.2 发起方（您发起的转让等对方确认）

1. 进入同一页面，顶部会显示浅蓝色提示：「⏳ 您发起的主守护人转让待 XXX 确认」。
2. 如对方迟迟未回应或不想转让了，可点击「取消转让」按钮主动作废。

### 4. 非本人 Tab 的「守护 TA 的人」只读详情

1. 在健康档案页面顶部头像区，切换到任意**家庭成员**（非本人 Tab）。
2. 您会发现「我守护的人」卡片**已隐藏**，仅剩下「守护 TA 的人」一张卡片（绿色 💚 图标）。
3. 点击该卡片，弹出底部抽屉，列出 TA 名下所有守护人：每个守护人有头像首字、姓名、关系（如「父亲」「配偶」）、加入时间，主守护人额外有金色 👑 徽章。
4. 点击列表中的任意守护人，可查看其只读详情（头像 / 姓名 / 关系 / 主守护人标识 / 加入时间共 5 字段），底部提示「该信息仅供查看，您无权对此守护关系进行修改」。
5. 抽屉中**不出现**任何「邀请 / 移除 / 转让 / 解除 / 提醒设置」等写操作按钮。

### 5. 旧入口已 301/404

- 直接访问旧地址 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/guardian-system](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/guardian-system) 将返回 404。
- 直接访问旧 v13 地址 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/v13](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/v13) 同样返回 404。
- 旧 C 端接口（如 `/api/guardian/list`、`/api/guardian/i-guard`、`/api/guardian/transfer/initiate` 等）也均已 404。后台管理「守护关系查询」（admin-web）继续保留正常。

## 注意事项

- 本次更新**仅修改 H5 与后端**，**未触碰**微信小程序、安卓 APP、苹果 APP、Windows 桌面端等其他终端代码，因此**无需重新下载/安装任何客户端**，已安装的小程序与 App 继续可用。
- 接收方同意转让后，原主守护人立即降为普通守护人；如对方未在 24 小时内确认，转让申请自动过期。
- 「我守护的人」总数包含已解除（cancelled）记录，便于您回顾历史；「守护中」一栏单独显示真正生效的关系条数。
- 「守护 TA 的人」抽屉仅支持只读查看，如需修改守护关系（例如解除某守护人），请使用对应被守护人本人的账号或通过主守护人入口操作。
- 浏览器若访问旧地址出现 404 属于正常下线行为，刷新到新入口即可。
