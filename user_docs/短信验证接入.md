## 功能简介

本次更新为系统接入了**腾讯云短信服务（AppKey 认证模式）**，使用 SDKAppID + AppKey 通过腾讯云旧版 HTTP API 发送短信验证码。主要改动：

1. **AppKey 认证模式**：新增基于 SDKAppID 和 AppKey 的短信发送通道，无需配置 CAM 密钥（SecretId/SecretKey），降低接入门槛
2. **自动配置初始化**：系统启动时自动将预设的腾讯云短信凭证（SDKAppID、AppKey、签名、模板ID）写入数据库，确保开箱即用
3. **登录验证模板预置**：自动创建"登录验证"短信模板（模板ID：2201340），内容为「{1}为您的登录验证码，请于{2}分钟内填写，如非本人操作，请忽略本短信。」
4. **双模式兼容**：系统同时支持 CAM 密钥模式和 AppKey 模式，优先使用 CAM 密钥，不可用时自动回退到 AppKey 模式

## 使用说明

### 一、用户端 — 短信验证码登录

#### 1. 进入登录页面

打开 H5 用户端或 APP，进入登录页面。

#### 2. 发送验证码

1. 在手机号输入框中输入 **11 位手机号码**
2. 点击 **「获取验证码」** 按钮
3. 系统调用腾讯云短信接口发送 6 位数字验证码
4. 手机将收到来自「**呃唉帮帮网络**」的短信，内容格式为：

   > XXXXXX 为您的登录验证码，请于 5 分钟内填写，如非本人操作，请忽略本短信。

#### 3. 输入验证码登录

1. 在验证码输入框中填入收到的 **6 位数字验证码**
2. 勾选「已阅读并同意用户协议和隐私政策」
3. 点击 **「登录」** 或 **「登录/注册」** 按钮

#### 4. 首次登录（自动注册）

- 如果该手机号从未在系统中注册过，验证码验证通过后将**自动创建账户并登录**
- 新用户可以在「个人中心」随时完善个人资料

---

### 二、管理后台 — 短信配置管理

#### 1. 查看短信配置

1. 使用管理员账号登录管理后台
2. 在左侧菜单中点击 **「系统管理」→「短信管理」**
3. 默认显示 **「短信配置」** 标签页，可以查看当前腾讯云短信配置：
   - SDKAppID
   - 签名名称
   - 默认模板ID
   - AppKey（已配置时显示）
   - 配置状态（启用/禁用）

#### 2. 修改短信配置

1. 在短信配置页面，选择「腾讯云」服务商区域
2. 可修改以下字段：
   - **SDKAppID**：腾讯云短信应用 ID
   - **AppKey**：应用密钥
   - **签名名称**：短信签名（需与腾讯云审核通过的签名一致）
   - **默认模板ID**：默认使用的短信模板
   - **CAM 密钥**（可选）：SecretId 和 SecretKey，配置后优先使用 SDK 方式发送
3. 点击 **「保存」** 使修改生效

#### 3. 查看短信模板

1. 切换到 **「短信模板」** 标签页
2. 系统已预置一个「登录验证」模板：
   - **模板名称**：登录验证
   - **模板ID**：2201340
   - **服务商**：腾讯云
   - **用途场景**：登录（login）
   - **模板变量**：验证码（6位数字验证码）、有效时间（验证码有效分钟数，默认5）

#### 4. 测试发送短信

1. 切换到 **「测试发送」** 标签页
2. 选择服务商为「腾讯云」
3. 输入目标手机号
4. 选择短信模板（如「登录验证」）
5. 填写或确认模板参数（验证码值、有效分钟数）
6. 点击 **「发送测试短信」**，预览确认后发送
7. 查看发送结果（成功/失败及详情）

#### 5. 查看发送记录

切换到 **「发送记录」** 标签页，可查看所有短信发送的历史记录，包括：
- 手机号（脱敏显示）
- 发送状态（成功/失败）
- 服务商
- 模板ID
- 发送时间
- 错误信息（如有）

## 技术架构说明

### 认证模式优先级

系统在解析短信配置时按以下优先级选择认证方式：

1. **数据库配置 — CAM 密钥**：如果数据库中的 SmsConfig 包含 SecretId 和 SecretKey，使用腾讯云 SDK 方式发送
2. **数据库配置 — AppKey**：如果数据库中配置了 AppKey 和 SDKAppID（但无 CAM 密钥），使用旧版 HTTP API 发送
3. **环境变量 — CAM 密钥**：环境变量中的 SecretId/SecretKey
4. **环境变量 — AppKey**：环境变量中的 SDKAppID/AppKey

### AppKey 发送流程

1. 生成随机数 `random` 和当前时间戳 `time`
2. 使用 `appkey`、`random`、`time`、`mobile` 计算 SHA256 签名
3. 向 `https://yun.tim.qq.com/v5/tlssmssvr/sendsms` 发送 POST 请求
4. 校验返回结果的 `result` 字段是否为 0

## 注意事项

1. **验证码有效期**：每条验证码有效期为 **5 分钟**，请在收到短信后及时填写
2. **发送间隔**：同一手机号 **60 秒内只能发送一次**验证码，重复请求会返回「发送过于频繁」提示
3. **验证码格式**：验证码为 **6 位纯数字**
4. **短信来源**：验证码短信签名为「**呃唉帮帮网络**」，请注意识别
5. **签名一致性**：管理后台配置的签名名称必须与腾讯云控制台中审核通过的签名完全一致，否则发送会失败
6. **模板一致性**：模板ID必须是腾讯云控制台中已审核通过的模板，模板参数个数和格式需匹配
7. **账号统一**：所有端（H5、APP、小程序）使用同一手机号登录的是同一个账户
8. **未收到短信**：如果长时间未收到验证码，请检查手机是否设置了短信拦截，或在管理后台「发送记录」中查看是否有错误信息
