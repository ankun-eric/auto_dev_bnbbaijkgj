## 功能简介

本次更新在已上线的「通用问卷框架」之上，全量落地以下 4 套问卷的内容、规则与体验：

| 问卷 | 题量 | 来源 | 本次动作 |
|------|------|------|---------|
| 中医体质测评（王琦 36 题） | 36 题 | ZYYXH/T157-2009 | 复用已有 36 题题库 · 接入新框架 · chips/CTA 后台可配 |
| 健康自查（升级版） | 部位/症状/时间 + 3 个新维度 | 自研 | 新增「症状性质 / 严重程度 / 症状补充备注」3 维度（均可跳过） |
| 情绪测评 PHQ-9 | 9 题 | Pfizer 量表 · 国家卫健委推荐 | 全新落地 · 题库 + 算分 + crisis CTA |

- chips 默认 3 个、CTA 默认 2 个（最多 4 个），运营在后台编辑模板时即可调整
- 所有 chips / CTA 的文案、跳转链接、行为类型完全后台可控，5 分钟内热更新生效（无需发版）
- **PHQ-9 第 9 题 ≥ 1 分时**，结果页强制插入「心理援助热线 400-161-9995」CTA（最高优先级）

## 操作步骤

### 一、用户端体验：4 个问卷统一入口

1. 打开 [AI 对话主页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
2. 首屏功能卡片区，点击对应问卷卡片：
   - 「中医体质测评」→ 36 题，约 5 分钟
   - 「健康自查」→ 升级版 6 维度（部位 / 症状 / 持续时间 / 症状性质 / 严重程度 / 备注）
   - 「情绪测评」→ PHQ-9 + GAD-7 组合（16 题，3-5 分钟）
   - 「睡眠测评 PSQI」→ 19 题，约 5 分钟
3. 进入沉浸式抽屉答题流程，逐题作答
4. 提交后 AI 自动进入对话流，依次展示：
   - 屏 1：**结果汇总卡片**（含主结论、可视化雷达图/仪表盘、详情入口）
   - 屏 2：**AI 解读文字**（带「本次回答结合 XX 的档案」开场白）
   - 屏 3：**3 个 chips 追问**（如：调理方法 / 饮食禁忌 / 适合运动）
   - 屏 4：**2 个 CTA 按钮**（如：「为我生成调理计划」+ 「推荐茶饮套装」）

### 二、健康自查 6 维度升级（新增 3 个可跳过维度）

| 步骤 | 维度 | 交互形态 | 是否可跳过 |
|------|------|---------|-----------|
| 1 | 选择部位 | 16 部位 emoji 网格（保留原交互，不引入人体图） | ❌ 必填 |
| 2 | 症状清单 | 按选中部位联动症状 | ❌ 必填 |
| 3 | 持续时间 | 5 档：今天才有 / 这两三天 / 一周内 / 一个月内 / 超过一个月 | ❌ 必填 |
| **4（新）** | **症状性质** | 8 个「大白话 + 表情」选项（🫀 一跳一跳的疼 / ☁️ 闷闷的疼 / 🔥 像火烧 / ...） | ✅ 可跳过 |
| **5（新）** | **严重程度** | VAS 0-10 滑块 + 表情提示（0 😀 → 10 😱） | ✅ 可跳过 |
| **6（新）** | **症状补充备注** | 文本框 + 语音录入（复用 VoiceWave 组件）| ✅ 可选 |

> ⚠️ 健康自查结果页**不做**红/橙/黄/绿医疗级硬判定，改为 AI 自然语言建议（居家观察 / 注意事项 / 软性就医提示），规避医疗合规风险。

### 三、PHQ-9 抑郁筛查（9 题，3-5 分钟）

1. 入口：AI 对话主页 → 「情绪测评」卡片 → 进入 PHQ-9 部分
2. 每题选项：完全没有（0 分）/ 几天（1 分）/ 一半以上的天数（2 分）/ 几乎每天（3 分）
3. 算分等级：

| 总分 | 等级 |
|------|------|
| 0-4 | 没有或最少抑郁 |
| 5-9 | 轻度抑郁 |
| 10-14 | 中度抑郁 |
| 15-19 | 中重度抑郁 |
| 20-27 | 重度抑郁 |

4. **特别提醒（自动触发）**：当**第 9 题（自伤念头）≥ 1 分**时，结果页**强制**插入「立即拨打心理援助热线 400-161-9995」CTA，作为最高优先级按钮置顶展示。

### 四、GAD-7 焦虑筛查（7 题，2-3 分钟）

入口、选项规则同 PHQ-9。算分等级：

| 总分 | 等级 |
|------|------|
| 0-4 | 没有或最少焦虑 |
| 5-9 | 轻度焦虑 |
| 10-14 | 中度焦虑 |
| 15-21 | 重度焦虑 |

### 五、PSQI 睡眠质量评估（19 题，5 分钟）

1. 入口：AI 对话主页 → 「睡眠测评」卡片
2. 题目按 7 个成分分组：
   - C1 主观睡眠质量 / C2 入睡时间 / C3 睡眠时间 / C4 睡眠效率
   - C5 睡眠紊乱（9 题频率题）/ C6 催眠药物 / C7 日间功能障碍
3. 总分 0-21，等级：

| 总分 | 等级 |
|------|------|
| 0-5 | 睡眠质量好 |
| 6-10 | 睡眠质量一般 |
| 11-15 | 睡眠质量差 |
| 16-21 | 睡眠质量很差 |

4. 结果可视化采用 **7 维度雷达图**（与体质测评雷达图组件复用）。

### 六、运营后台 chips / CTA 配置

1. 登录 [后台管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin)
2. 进入「问卷管理」 → 选择对应问卷模板 → 编辑
3. 在模板编辑页可配置：
   - **followup_chips_json**：追问 chips 列表（数量、文案、顺序，1~6 个）
   - **cta_list_json**：CTA 按钮列表（1~4 个，含 label / action / target_url / style）
4. 保存后即时生效（前端最多 5 分钟内热更新）

## 注意事项

### 1. 量表合规说明

- PHQ-9 / GAD-7 / PSQI **三个量表均为公开免费量表**，非商用临床使用无版权问题
- 量表题干**字字对应官方版本**，本次发布**未做任何题目改编**
- 所有问卷结果页**仅作筛查参考**，**不构成诊疗依据**，需提示用户「如不适请及时就医」

### 2. PHQ-9 第 9 题危机干预（强制行为）

- 当用户在 PHQ-9 第 9 题（"有不如死掉或用某种方式伤害自己的念头"）选择「几天 / 一半以上的天数 / 几乎每天」（即 score ≥ 1）时：
  - 后端响应中 `phq9_crisis = true`
  - `cta_list` 数组**第一个元素**强制插入「立即拨打心理援助热线 400-161-9995」按钮（style=danger，置顶展示）
  - 该 CTA **不可被后台配置覆盖**（mandatory=true）

### 3. 健康自查语音录入

- 步骤 6 的语音录入**复用已有组件**，**无需**重新开发或接第三方语音 API：
  - H5：`h5-web/src/components/design-system/VoiceWave.tsx`
  - 小程序：`miniprogram/pages/digital-human-call/index.js` 中的 `recorderManager` 录音方案
  - Flutter：`flutter_app/lib/screens/ai/chat_screen.dart` 中的语音输入接入
- 步骤 4 / 5 / 6 全部为**可跳过**维度，跳过后该维度为空，AI 解读时会提示用户补充

### 4. 数据合规

- 健康自查的「症状补充备注（含语音转写）」涉及用户健康信息，存储与传输走平台已有的医疗数据加密通道
- 健康自查结果页**不做**红/橙/黄/绿四级风险等级仪表盘，避免医疗器械合规风险

### 5. chips 与 CTA 协议要点

- chips **不带**「本次回答结合 XX 的档案」开场白（`render_meta.include_archive_prefix = false`）
- 用户点击 chip 后的二轮回答**必须重新带上**该开场白
- 点击后 chip 自动置灰，不可重复点击
- chat_messages 顺序固定：`questionnaire_result_card → text → followup_chips → cta_buttons`，全部 `sender = "ai"`

### 6. 推荐 CTA action 类型

CTA 配置中 `action` 字段支持以下类型：

| action | 含义 |
|--------|------|
| `generate_health_plan` | 跳转到健康计划生成 |
| `open_shop` | 跳转到商城 / 商品详情 |
| `open_service` | 跳转到服务预约 |
| `open_consultation` | 打开专家咨询 |
| `external_link` | 外链（如心理援助热线 H5 / tel: 协议） |

### 7. CTA target_url 占位符

CTA 的 `target_url` 支持 `{main_type}` 占位符（如 `/shop/category/tea?tag={main_type}`），后端在响应时会自动渲染为用户的主体质（如 `痰湿质`）。
