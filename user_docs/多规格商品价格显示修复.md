## 功能简介

本次修复解决了**多规格商品在用户端列表页和搜索结果页价格显示为 ¥0** 的严重 Bug。

### 问题描述

在管理后台创建多规格商品（如配置了多个 SKU，最低售价 ¥1300）后，管理后台列表页能正确显示「¥1300起」，但切换到用户端（H5 网页端、Flutter APP 端、微信小程序端）的服务列表页和搜索结果页时，该商品价格显示为 ¥0。

### 修复内容

- 后端 API 层：商品列表和搜索接口自动计算多规格商品的 SKU 最低售价，返回正确价格
- H5 网页端：服务列表页价格展示改为使用最低价，多规格商品显示「¥最低价 起」
- Flutter APP 端：服务列表页价格展示逻辑同步修复
- 微信小程序端：服务列表页价格展示逻辑同步修复

## 使用说明

### 修复后的价格显示规则

| 商品类型 | 显示格式 | 示例 |
|----------|----------|------|
| 单规格商品 | ¥售价 | ¥500 |
| 多规格商品（SKU 价格不同） | ¥最低价 起 | ¥1300 起 |
| 多规格商品（SKU 价格相同） | ¥统一价 | ¥1000 |

### 验证步骤

1. **H5 网页端验证**
   - 打开 H5 用户端链接：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)
   - 进入「服务」Tab 页面
   - 查看多规格商品的价格，应显示为「¥最低价 起」格式
   - 在服务页搜索多规格商品，搜索结果中价格同样显示正确

2. **Flutter APP 端验证**
   - 下载并安装最新版 APK
   - 打开 APP，进入服务列表页
   - 查看多规格商品价格，确认显示为「¥最低价 起」

3. **微信小程序端验证**
   - 下载小程序代码包并导入微信开发者工具
   - 进入服务页面
   - 查看多规格商品价格，确认显示为「¥最低价 起」

4. **单规格商品回归验证**
   - 在各端查看单规格商品，价格应正常显示，无「起」字

## 注意事项

1. **仅影响列表页和搜索结果页**：本次修复仅涉及服务列表页和搜索结果页的价格展示，不影响商品详情页和下单支付流程
2. **"起"字显示规则**：只有当多规格商品的各 SKU 价格不同时才会显示「起」字；如果所有 SKU 价格相同，则不显示「起」字
3. **管理后台无影响**：管理后台的商品列表页价格显示保持不变
4. **商品详情页无影响**：用户端各端的商品详情页价格展示不受影响，保持原有正确的逻辑
