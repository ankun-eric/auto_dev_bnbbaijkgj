## 一、本次修复了什么

这一轮针对"健康自查"业务做了 7 项修复 + 体验升级：

| 编号 | 问题 / 改动 | 涉及端 |
|------|------------|--------|
| B-1 | 问卷"症状补充备注"题：原来题目说明被同时显示在输入框上方"小标题"和输入框内的 placeholder 里，文字重复。现在两者拆分：上方只显示"小提示文案"，输入框内显示"请输入您想补充的内容（选填）" | H5 |
| B-2 | 完成问卷后的结论卡片解读说明里，"本次回答结合的对象"在选了家人档案时被错显示成"本人"。现在会正确显示为"妈妈（母亲）的健康档案" | 后端 / H5 |
| B-3 | H5 健康自查"查看详情"按钮过去偶发白屏（`Application error: a client-side exception has occurred`）。现在做了"地毯式加固"：所有字段空值兜底、整页 ErrorBoundary、图片加载失败兜底、id 不存在时友好提示 | H5 |
| B-4 | 管理后台「通用问卷模板管理」列表里，"健康自查"那一行原来只有"前往专属配置页"按钮、看不到标准"编辑"按钮。现在已恢复为与其他问卷一致的「题目/分型 + 编辑 + 删除」三按钮 | 管理后台 |
| B-5 | 老的「健康自查问卷模板」「部位症状字典」两个独立菜单/页面合并下线（数据已迁入通用问卷模板），后端老表 `health_check_template`、`body_part_dict` 已具备一键删除能力（默认 dry-run 校验通过，DROP 由运维一键开关控制） | 管理后台 / 后端 |

## 二、操作步骤

### 普通用户视角（H5 端）

#### 场景 1：本人做一次健康自查
1. 打开 [H5 主页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)。
2. 进入 AI 对话页，左上角档案胶囊保持"本人"。
3. 点击"健康自查"功能卡片，依次回答：部位 → 症状 → 持续时间 → 严重程度 → 性质 → 补充备注（最后一题为可选）。
4. 在最后一题"症状补充备注"页面：
   - 输入框上方的小提示文字是题目本身的说明（例：受凉后加重 / 每天下午 3 点准时出现）。
   - 输入框内的 placeholder 是统一的"请输入您想补充的内容（选填）"。
   - 两者不再重复。
5. 提交后会看到 AI 对话流中依次出现：
   - 一张「健康自查报告」卡片（被测人下方写"本人"）。
   - 一段 AI 解读文字，第一句即"本次回答结合 **本人** 的健康档案。"
   - 三个追问按钮（chips）。
6. 点击卡片右下角"查看详情 ›"按钮 → 进入「健康自查结果详情页」（6 区块：摘要 / 答题记录 / AI 解读 / 居家建议 / 就医警示 / 推荐商品）。

#### 场景 2：为家人（妈妈）做一次健康自查
1. 进入 AI 对话页，把左上角档案胶囊切换为"妈妈"。
2. 重复"场景 1"的步骤 3~6。
3. **关键差异**：
   - 结果卡片里"被测人"显示为「**妈妈（母亲）**」。
   - AI 解读文字第一句变为"本次回答结合 **妈妈（母亲）** 的健康档案。"
   - 详情页顶部摘要条下方也会出现"本次回答结合 **妈妈（母亲）** 的健康档案"。

#### 场景 3：查看历史结果详情
1. 打开任意一份历史答卷的详情链接，例如 [/health-self-check/result/1](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-self-check/result/1)。
2. 若 ID 真实存在：正常渲染 6 区块。
3. 若 ID 不存在：页面友好提示「该结果不存在或已被删除」，**不会白屏**。
4. 若网络异常：页面提示「结果暂不可查看，请稍后再试」并提供"重新加载"按钮，**不会白屏**。
5. 若某个商品封面图加载失败：仅该图位置变为空白，**不会影响整页渲染**。

### 运营 / 管理员视角（后台）

#### 场景 4：编辑健康自查模板
1. 打开 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/) 并登录。
2. 左侧菜单进入「AI咨询配置 → 通用问卷模板管理」。
3. 找到 code 为 `health_self_check` 的一行 → 右侧操作列现在有 **"题目/分型 / 编辑 / 删除"** 三个按钮（与体质测评、PHQ-9 等其他问卷一致）。
4. 点击 **"编辑"** → 在弹出窗口里可以直接修改名称、说明、AI 解读 Prompt 模板、AI 开场白、结果摘要模板、问卷完成后体验配置、AI 追问关键字段（key_field_codes）等。
5. 老的「健康自查问卷模板」「部位症状字典」两个菜单已从侧边栏移除。

#### 场景 5：使用「占位符速查表」编写 Prompt
1. 进入任意问卷模板的编辑抽屉。
2. 在 **"AI 解读 Prompt 模板"** 输入框下方有一个浅蓝色的折叠面板「📋 占位符速查表（共 21 项）」。
3. 点击展开后会显示一张表格：
   - **占位符**：如 `{user_name}` / `{family_member_age}` / `{chronic_diseases}` / `{bmi}` 等。
   - **含义**：每个占位符代表的字段。
   - **适用范围**：通用（蓝）/ 档案类（绿）/ 仅健康自查（橙）。
   - **示例值**：直观参考。
4. 复制需要的占位符（如 `{family_member_name}`）粘贴到上面的 Prompt 模板里即可。
5. 运行时取不到值时，会自动替换为"未填写"，不会报错。

#### 场景 6：核对老表下线状态（可选 / 运维向）
- 当前默认是 **dry-run 模式**：每次后端启动时会校验"新表已承接老表数据"，并备份老表行数到日志，但**不真正删除**。
- 如需真正 DROP TABLE，运维设置环境变量 `HSC_LEGACY_OFFLINE_DROP=1` 后重启容器，迁移脚本会自动执行 DROP（并写入 `prd_phase_state.hsc_legacy_offline_v1 = done` sentinel，幂等）。
- 启动日志里 `[migrate] hsc_legacy_offline_v1: 迁移完成 stats=...` 行可以查看校验结果与备份信息。

## 四、注意事项

1. **家人主体识别**：B-2 的修复依赖前端在切换档案时把"咨询对象 ID"正确传给后端。如果您看到家人态结果卡仍显示"本人"，请检查 AI 对话页左上角胶囊是否真正切换到了对应家人，再重新发起一次健康自查。
2. **详情页加固范围**：本次仅强化了 H5 端的 `/health-self-check/result/{id}` 页面。小程序 / Flutter App 端的同类详情页**未做改造**（小程序与 App 端本次代码无任何变更，沿用现有客户端版本即可）。
3. **占位符速查表本身不做"一键插入"**：根据需求设计，速查表是只读纯文本展示，需要手动选中复制后粘贴到 Prompt 模板输入框。
4. **老表下线属"保留拆除工具"**：默认 dry-run，不真正 DROP。运维需要明确开关 `HSC_LEGACY_OFFLINE_DROP=1` 才会执行物理 DROP TABLE。即使 DROP 后再次启动也是幂等的（sentinel 记录 status=done）。
5. **本次不影响 APP / 小程序**：本轮代码改动仅涉及 H5、管理后台、后端三个 Web 端，已通过 docker compose 重建部署。无需重新下载或更新任何客户端版本。
