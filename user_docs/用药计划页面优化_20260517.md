## 功能简介

本次「用药计划」核心模块重构，目标是把 H5 端与微信小程序端打造成 **风格统一、视觉清爽、医嘱明确、跨端一致** 的同款页面。

主要亮点：

1. **顶部医嘱提示条**：每次进入「新增/编辑用药计划」时，页面顶部常驻一条黄底警示条，文案"用药请遵循医嘱"，提醒用户合规用药。
2. **整页"标题+值同行"列表行**：所有字段都改为左侧标题、右侧当前值、最右侧 `›` 箭头的统一行样式，扫一眼即可看全表单。
3. **服用周期升级**：从原"持续 N 天"模型升级为 **开始日期 + 结束日期 + 长期服用开关** 三字段结构；表单页双行展示主值（起止日期）+ 副值（共 N 天 / 长期服用）。
4. **服用时机 Tag-B 视觉**：5 个标签（饭前 / 饭后 / 空腹 / 随餐 / 睡前）改为浅蓝底圆角标签，选中态填充主色。
5. **主色统一为 #0EA5E9**：H5 旧的 `#4A9EE0` 全部替换；小程序相关页面主色一并统一。
6. **小程序老数据无感知迁移**：用户原来的「早上/中午/下午/晚上/睡前」时段会自动按规则映射到新枚举（早上→饭前，中午/下午/晚上→饭后，睡前→睡前）。
7. **备注统一为单行 30 字限制**：右下角实时显示字数 `XX / 30`，超出无法继续输入。

## 使用说明

### 一、新增一个用药计划（以 H5 端为例）

1. **进入"用药计划"列表**：从「健康档案」主页面 → 在用药品（Hero 第 4 格）→ 点击「+」浮动按钮，或者直接打开链接 [新增用药计划](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home/medication-plans/new)。
2. **顶部黄色提示条**：进入后首先会看到「⚠️ 用药请遵循医嘱」常驻提示，无法关闭，这是设计的合规提醒，请知悉。
3. **逐项填写表单**：
   - **药品名称**：点击该行 → 在弹出抽屉里输入药品名，输入 ≥ 2 个字时会弹出"联想下拉"显示候选药品，可点击候选项或继续手动输入。
   - **用药时间**：点击该行 → 抽屉先选择"每日服药次数（1/2/3/4 次）"，下方动态显示对应数量的"时间点"行，点击单条可调整该时间点的具体时:分。
   - **每次剂量**：点击该行 → 双轮滚动选择器，左轮为数量（¼ / ½ / 1 / 2 / 3…），右轮为单位（片 / 粒 / mL / 滴…）。
   - **服用周期**：点击该行 → 上方为"开始日期 / 结束日期"两个日期框；下方为「长期服用」开关。
     - **普通态**：直接选择起止日期，下方提示"共 N 天"。
     - **长期态**：打开"长期服用"开关后，结束日期框灰色显示"长期"，下方提示「长期服用，无固定结束日」。
     - 校验规则：结束日期不能早于开始日期；开始日期可向前回溯 90 天用于历史补录；结束日期最远不超过开始日期 + 3 年。
   - **服用时机**：5 个标签单选（饭前 / 饭后 / 空腹 / 随餐 / 睡前）。
   - **备注**：选填，30 字以内，右下角实时显示字数。
4. **保存**：点击下方主色（蓝色）"保存"按钮，所有必填字段满足后会跳回上一页并提示「添加成功」。

### 二、编辑或删除一个已有计划

1. 在用药计划列表中点击任意一条卡片进入详情/编辑页。
2. 修改任意字段（操作方式同新增）。
3. 底部左侧有「删除」按钮（红色边框），点击后弹窗确认即可彻底删除。

### 三、跨端一致性

- H5 与微信小程序两端的"字段名、字段顺序、占位文字、标签项"**完全一致**，操作思路通用。
- 截图并排放对比，肉眼应当看不出明显差异（仅基础控件由各端原生组件实现）。

## 注意事项

1. **医嘱提示条不可关闭**：根据合规要求，医疗类产品需常驻医嘱提醒，请勿尝试关闭。
2. **长期开关与结束日期**：
   - 打开长期开关后，结束日期会被强制清空（保存到后端为 null）。
   - 关闭长期开关后，结束日期会自动设置为「开始日期 + 30 天」，可继续手动修改。
3. **历史补录**：新增计划时支持选择过去日期作为开始日期（最远 90 天），方便补录已开始的疗程。
4. **保存校验**：所有必填字段（药品名称 / 用药时间 / 每次剂量 / 服用周期 / 服用时机）必须填写完整，否则保存时会弹出「请填写完整信息」提示。
5. **提醒规则**：
   - 普通计划：结束日期当天 23:59 后停止提醒。
   - 长期计划：永远提醒，直到手动归档 / 删除。
6. **APP 端暂未改动**：本次仅 H5 + 小程序两端发版，APP 端（安卓 / 苹果）保持原状（橙色主色 `#FA8C16`），将在下一迭代单独立项做"全局换肤"。
7. **深色模式**：本次不考虑深色模式。
8. **网络异常**：保存失败时，已填写的内容会保留在表单中，不会被清空，可重试。
