## 二、功能 1：60 分钟自动刷新

### 2.1 它是什么

进入 AI 首页时，系统会自动判断**距离上次对话已经多久**：

- 距上次对话 **不超过 60 分钟** → 继续展示原有会话，您可接着上次的话题聊
- 距上次对话 **超过 60 分钟** → 自动清空旧对话，进入全新会话（与蚂蚁阿福、晓医的体验一致）

### 2.2 触发时机（全端覆盖）

下面任意一种"重新进入页面"的场景都会触发判断：

| 平台 | 触发场景 |
|---|---|
| H5 | 打开页面、切换到本 Tab、从其他网页切回、手机锁屏后亮屏 |
| 微信小程序 | 每次进入聊天页（onShow） |
| 安卓 / 苹果 APP | 从后台切回前台、从其他页面返回 AI 首页 |

### 2.3 操作步骤

1. 在浏览器或 App 中打开 AI 首页：[/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home)
2. 与小康正常聊天
3. 离开页面 60 分钟以上后再回来 → 自动进入全新欢迎页
4. 您仍可通过左上角的"历史会话"找回之前的聊天记录

### 2.4 注意事项

- 该刷新机制**不会删除您的历史聊天记录**，只是不再默认加载到当前页面
- 60 分钟阈值由运营后台统一配置，未来可能根据数据调整

## 三、功能 2：切换咨询人「三重提示」

### 3.1 它是什么

当您在左下角切换咨询人（例如从「本人」→「妈妈」）时，系统会以**三种方式同时提示您**，让您绝对不会错过切换：

```
┌─────────────────────────────────────┐
│   ① 屏幕中央 Toast：「已切换为 妈妈 咨询」（2 秒消失）    │
│   ② 聊天流插入系统消息：「—— 现在开始为 妈妈 提供健康咨询 ——」（永久留痕） │
│   ③ 顶部 5 秒撤销横条：「返回上一会话」按钮          │
└─────────────────────────────────────┘
```

### 3.2 操作步骤

1. 在 AI 首页 / 聊天页面，点击底部「切换咨询人」入口
2. 选择目标家庭成员（如：妈妈）
3. 切换瞬间会**同时**看到 3 个提示
4. 如果切错了，**5 秒内**点击顶部"返回上一会话"即可一键回滚（会话 ID、咨询对象、消息列表全部还原）
5. 5 秒后未撤销 → 切换永久生效

### 3.3 注意事项

- ⚠️ **撤销期内（5 秒内）60 分钟刷新计时会暂停**，避免撤销之后会话又被刷新
- 切换咨询人后**不会加载该咨询人的旧对话**，直接进入空白新会话
- 顶部 4 个功能卡片、待办事项会自动切换为该咨询人对应的数据
- 系统消息气泡（②）作为聊天流的一部分**永久保留**，方便您事后回看时分清切换边界

## 四、功能 3：右下角「小康头像悬浮球」

### 4.1 它是什么

聊天进行过程中，原本在顶部的「4 个功能入口 + 今日用药」会被消息逐渐顶到屏幕外。
我们新增了一个**右下角小康头像悬浮球**（48×48 px，带主题绿描边和呼吸光晕动画），让您聊到任意位置都能**一键唤出健康服务面板**。

### 4.2 首次引导

第一次进入 AI 对话页时，悬浮球左侧会自动弹出气泡：

> 健康服务入口在这里，随时点开

3 秒后自动消失，并且**只出现一次**（已在本机记录）。

### 4.3 操作步骤

1. 进入 AI 首页，开始与小康聊天
2. 注意右下角的「小康头像球」（位于输入框上方 48 px）
3. 点击悬浮球 → 从右下角弹出**健康服务面板**
4. 面板内完整复刻了顶部区：
   - 个性化欢迎语
   - 4 个功能入口卡片（症状自查 / 体征录入 / 报告解读 / 用药管理）
   - 今日用药卡片
5. 点击面板内任一入口即可跳转到对应功能
6. 关闭面板：点击面板外区域、右上角 ×、或再次点击悬浮球

### 4.4 注意事项

- 悬浮球与右上角铃铛分工：
  - **铃铛** = 被动通知（消息中心）
  - **悬浮球** = 主动调用（健康服务功能）
- 面板内**不重复显示待办**，避免与铃铛冲突
- 在小程序端使用 cover-view 实现，确保不会被聊天气泡遮挡

## 五、埋点与数据观测

为了持续优化体验，本次新增了 11 个埋点事件（全部已上线 v4 后端）：

| 事件 | 含义 |
|---|---|
| refresh_triggered | 60 分钟刷新成功触发 |
| refresh_skipped | 进入页面但未到刷新阈值 |
| switch_consultant | 切换咨询人成功 |
| switch_undo_clicked | 点击 5 秒撤销 |
| switch_undo_expired | 5 秒未撤销，切换永久生效 |
| floating_ball_shown | 悬浮球首次展示 |
| floating_ball_clicked | 点击悬浮球 |
| floating_ball_panel_action | 点击面板内功能入口 |
| first_guide_shown | 首次引导气泡展示 |
| refresh_anomaly | 反向监测刷新失效（理论上应为 0） |
| system_message_visible | 切人系统消息气泡曝光 |

**用户无需关心这些事件**，它们仅在后台用于产品改进。

## 六、常见问题 FAQ

**Q1：我刚才的对话怎么不见了？**
A：如果离开页面超过 60 分钟，系统会自动开启全新对话。原有聊天记录可在**左上角"历史会话"**中找到。

**Q2：切换咨询人后，能不能看到这位家人之前的聊天？**
A：基于隐私和数据治理考虑，切换后默认不加载该家人的历史对话。如需查看，请在"历史会话"中按家人筛选。

**Q3：悬浮球能不能关闭？**
A：当前版本悬浮球默认常驻。后续如有需要可以在"设置 - 显示设置"里增加开关（v4.1 规划）。

**Q4：5 秒撤销后会怎样？**
A：会话 ID、咨询对象、消息列表、左侧历史抽屉条目**四项一起回滚**，等同于"切换从未发生"。

**Q5：我用 H5、小程序和 APP 三端，体验一致吗？**
A：完全一致。三端使用的是同一份后端配置（`session_refresh_minutes=60`），文案、UI、撤销时间都已对齐。
