## 功能简介

本次更新修复了一个 P0 级线上问题：admin 后台中点击左侧菜单「商家体系 → 订单明细」后，浏览器整页白屏并仅显示一段纯文本「Gateway OK」，导致订单明细功能完全无法使用。

修复后的核心变化：

1. **订单明细页彻底独立**：左侧菜单「订单明细」点击后**直接渲染订单列表页**，不再做任何"视图记忆+自动跳转"，彻底告别白屏问题。
2. **看板入口收敛**：预约看板的入口由「商家管理 → 门店列表」中各门店行末的「预约看板」按钮**专属承载**；订单列表页右上角原有的「切换到预约看板」按钮已**完全移除**，避免双入口造成的混乱。
3. **看板兜底友好化**：直接访问预约看板而未选择门店时，不再返回 nginx 兜底白屏，而是显示一个友好的提示页面，告知用户「请先选择门店」并提供「前往门店列表」按钮。

## 使用说明

### 场景 1：查看订单明细列表

1. 登录 admin 后台
2. 点击左侧菜单 **「商家体系」 → 「订单明细」**
3. 页面**直接进入订单列表**，正常展示订单数据，所有原有功能（搜索、筛选、分页、状态切换、退款审批、详情抽屉、批量操作等）保持不变

> ✅ 修复确认点：点击订单明细菜单后，浏览器**不再出现 "Gateway OK" 白屏**，也不会发生任何意外跳转。

### 场景 2：查看某门店的预约看板

预约看板已改为按门店维度，必须先选门店：

1. 登录 admin 后台
2. 点击左侧菜单 **「商家管理」 → 「门店列表」**
3. 在门店列表中找到目标门店所在行
4. 点击该行末尾的 **「预约看板」按钮**
5. 进入对应门店的预约看板页面，可在「日视图 / 周视图 / 月视图」之间切换查看

> 💡 提示：订单列表页右上角不再有「切换到预约看板」按钮，预约看板入口现统一收敛在门店列表中。

### 场景 3：意外直接访问预约看板（不带 storeId 参数）

如果通过浏览器收藏夹、外部链接、或手动地址栏输入等方式直接访问预约看板地址但**未带门店 ID 参数**，页面将显示一个友好的提示卡片：

- 标题：**「请先选择门店」**
- 说明：「预约看板按门店维度查看，请到「商家管理 → 门店列表」点击对应门店的「预约看板」按钮进入。」
- 按钮：**「前往门店列表」**（点击后跳转到门店列表页）

此场景下页面**不再出现 "Gateway OK" 白屏**，整体体验保持平滑。

## 注意事项

1. **历史用户视图记忆已自动清理**：本次修复后，旧的浏览器 localStorage 偏好值 `bini_orders_view_preference` 会在用户进入订单明细页时自动被清除。无需用户手动操作。
2. **缓存刷新**：如果上线后部分用户仍偶发遇到白屏，请提示用户做一次 **F5 强制刷新**（或 Ctrl+F5）即可加载最新前端资源（admin-web 的静态资源 hash 已变化，浏览器会自动加载新版）。
3. **回归确认范围**：本次仅改动 admin-web 前端，**后端、数据库、网关、H5、小程序、安卓 APP、iOS APP** 均不涉及，无需任何配套发布或回归。
4. **入口路径**：今后预约看板的唯一推荐入口为「商家管理 → 门店列表 → 任一门店 → 预约看板按钮」，请运营/客服在内部话术与外部说明中保持一致。
