## 二、登录页变化（三端通用）

### 2.1 新增图形验证码

- 登录页面**密码输入框下方**多了一行**图形验证码**：
  - 左侧是输入框，右侧是 4 位字符的彩色图片（混入数字 + 大小写字母，已避开 0/O、1/l 等易混字符）。
  - **点击图片即可换一张**；若提交失败，验证码也会自动刷新。
- 验证码与账号、密码一起提交。任一项错误都会统一提示 **"账号、密码或验证码错误"**（不区分具体错的是哪一项，避免被人摸出账号是否存在）。
- 图形验证码 5 分钟内有效，超时请点击图片重新获取。

### 2.2 失败次数防护

为防止恶意刷登录，新增以下风控规则：

| 维度 | 阈值 | 锁定时长 |
| --- | --- | --- |
| **同一 IP** 连续登录失败 | 5 次 / 15 分钟 | 锁 **15 分钟** |
| **同一手机号** 连续登录失败 | 5 次 / 15 分钟 | 锁 **15 分钟** |

- 锁定期内即使账号、密码、验证码全部正确，系统也会提示 **"操作过于频繁，请 15 分钟后再试"**。
- 登录成功后失败计数清零。

### 2.3 商家端短信登录已下线

- **商家 PC 后台**与**商家 H5 后台**已**移除"短信验证码登录" Tab**，仅保留"手机号 + 密码 + 图形验证码"。
- 登录页**不再显示「忘记密码？」入口**：
  - **员工忘记密码** → 找老板用「重置密码」功能（见第六章）。
  - **老板忘记密码** → 联系平台 admin 在「商家账号」列表里点击该老板的「重置密码」。

> 提示：消费者 C 端（小程序 / 消费者 H5 / App）仍保留短信验证码登录，本次不变。

## 三、admin 平台后台 - 个人信息 / 修改密码（M1）

### 3.1 入口

登录后右上角下拉菜单中可看到：

```
…业务菜单
个人信息
修改密码
退出登录
```

### 3.2 个人信息（只读）

页面展示：**姓名 / 手机号 / 角色 / 所属商家**，**全部只读**，无编辑按钮。如需修改请联系超级管理员（页面底部有灰色提示）。

### 3.3 修改密码

表单字段：

- **原密码**（必填）
- **新密码**（必填）— 长度 ≥ 8 位、必须**同时包含字母和数字**
- **确认新密码**（必填）— 必须与新密码一致

校验失败提示：

| 错误类型 | 提示 |
| --- | --- |
| 原密码错误 | "原密码错误，请重新输入" |
| 新密码强度不足 | "新密码至少 8 位，且需包含字母和数字" |
| 两次密码不一致 | "两次输入的新密码不一致" |

提交成功后会**强制踢出当前登录态**，跳转到 admin 登录页，请使用新密码重新登录。

## 四、admin "新建商家账号" 页面精简（M2）

打开链接：<https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/merchant/accounts>

### 4.1 表单只保留 4 个字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 姓名 | ✓ | 老板真实姓名 |
| 手机号 | ✓ | 商家端登录账号 + C 端绑定手机号 |
| 登录密码 | ✓ | 强度规则同上：≥8 位、字母+数字 |
| 所属商家 | ✓ | 选择已有商家或新建 |

> 已移除：用户端昵称、商家端昵称、"同时开通商家身份"勾选项、商家身份类型、角色、状态。
> 创建后角色固定为"老板"、状态固定为"启用"。

### 4.2 列表页列展示

只显示：**姓名 / 手机号 / 所属商家 / 角色（老板）/ 状态 / 操作**。

操作列包含：编辑 / 启用-停用 / 删除 / **重置密码**。

### 4.3 admin 重置老板密码

1. 在列表行点击「重置密码」。
2. 弹窗内二选一：
   - **使用默认密码**（默认）：系统重置为该老板**手机号后 6 位**。
   - **自定义密码**：手动输入新密码（强度规则同上）。
3. 点确定后立刻生效，**该老板所有登录态被踢出**，下次登录时必须强制改密。

### 4.4 「批量导入商家账号」已下线

菜单中不再显示批量导入入口，请改为单个新建。

### 7.2 「编辑权限」 → 「查看权限」（修复回显 Bug）

- 列表操作列原"编辑权限"按钮已**统一改名为「查看权限」**。
- 进入页面后：
  - 所有勾选框**只读 disabled**，但能清晰看到已勾选的模块。
  - 不再显示底部「保存」按钮。
  - 顶部提示："角色权限由系统统一配置，此处仅可查看。"
- 已**修复进入页面后所有 checkbox 都没勾上**的 Bug，现在能正确回显该员工角色对应的权限。

### 7.3 「新增员工」按钮 — 仅老板可见

| 角色 | 是否能看到「新增员工」按钮 | 可创建的角色 |
| --- | :---: | --- |
| 老板 | ✓ | 店长 / 财务 / 店员 |
| 店长 / 财务 / 店员 | ✗（按钮不渲染） | — |

后端也加了校验：非老板调用创建接口直接返回 403。

### 7.4 创建员工表单字段

**PC 端：**
姓名*、手机号*、角色*（店长/财务/店员）、所属门店*（多选）、头像、入职日期、备注（≤200 字）。

**H5 端（精简）：**
姓名*、手机号*、角色*、所属门店*（单门店商家自动隐藏该字段）。

> 单门店商家：所属门店自动选中并隐藏。

### 7.5 初始密码

- **不需要老板手动输入密码**。
- 系统自动设置为：**该员工手机号的后 6 位**。
- 创建成功提示："员工创建成功，初始密码为手机号后 6 位，请告知员工首次登录修改密码。"
- 员工**首次登录会被强制弹窗改密**（无需"原密码"，只需"新密码 + 确认新密码"），未修改前不可使用其它功能。

### 7.6 启用/停用规则（已修 Bug）

- 老板可对店长/财务/店员启停。
- 店长可对**本店**的财务/店员启停（无法操作其它店长或跨店员工）。
- **已修复 H5 启停开关"弹回原状态"的 Bug**：现在仅在后端确认成功后才更新开关状态。
- 老板/店长被 admin 停用 / 删除后，**手机号对应的 C 端用户身份不受影响**，可继续作为消费者登录使用。

### 7.7 「查看权限」可见性

店长 / 财务 / 店员均**可点开「查看权限」**（只读模式）。仅"创建 / 重置密码 / 启停"等写操作做角色控制。

## 八、老板重置员工密码（M6）

### 8.1 入口

商家 PC 后台 + 商家 H5 后台员工列表，**每行操作列新增「重置密码」按钮**，与「启用/停用」并列。**仅老板可见**。

### 8.2 操作步骤

1. 点击「重置密码」 → 弹出确认弹窗："确定要重置 **{姓名}** 的登录密码吗？"
2. 弹窗内二选一：

   | 方式 | 说明 |
   | --- | --- |
   | **默认密码**（默认选中） | 重置为该员工**手机号后 6 位** |
   | **自定义密码** | 老板手动输入新密码（强度规则同上） |

3. 点确定 → 后端执行重置 → toast：**"密码已重置，请告知员工新密码"**。
4. 该员工**所有登录态（PC + H5）被踢出**。
5. 该员工**下次登录时强制改密**，页面会清晰提示**"您的密码已被管理员重置"**。

### 8.3 不发送短信通知

- 系统**不会自动发短信**通知员工。
- 请老板线下口头告知员工新密码。
- 后端会落操作日志（操作人、被操作员工、重置时间、重置方式），便于事后审计。本期无 UI 查询页面。

## 十、注意事项 & 常见问题

1. **图形验证码看不清？** 直接点击图片可换一张。
2. **登录提示"账号、密码或验证码错误"？** 三项任一错误都会统一提示，请逐项核对，注意验证码区分大小写。
3. **被 IP 锁定？** 等 15 分钟自动解锁，期间任何登录请求都会被拒。
4. **首次登录被强制改密？** 这是因为该账号是新建或被管理员重置过。请在弹出的页面填写新密码完成首次设置。
5. **员工说开关切换不上？** 本次已修复该 Bug，如再次遇到请刷新一次或检查角色权限（店长不能启停其它店长）。
6. **老板自己忘了密码怎么办？** 联系平台 admin，在 admin 后台「商家账号」列表为该老板执行「重置密码」。
7. **C 端用户和 B 端老板/员工是同一个手机号怎么办？** B 端被停用 / 删除不会影响该手机号的 C 端身份，C 端依然可以正常以消费者身份登录。
