## 功能简介

「家庭体检异常·守护推送」让子女（守护者）能在**第一时间**收到老人（被守护者）体检异常的合并推送，全链路在小程序/App 内闭环。

本期上线核心能力：

1. **守护者集合算法**：自动识别"建档人 ∪ 双向共管对方 − 被守护者本人"，覆盖三大典型场景（子女拍照建档零参与、子女邀请老人接受、老人邀请子女接受）
2. **聚合推送 + 5 分钟去重**：每份体检报告对每位守护者只推 1 条聚合消息，5 分钟内同 `report_date` 重复触发自动去重
3. **虚拟老人档案 + 注册迁移**：子女可先为未注册的老人建虚拟档案；老人注册后弹窗确认即可接管，自动建立守护关系
4. **管理后台三件套**：异常文案模板 CRUD（三占位）、异常阈值配置、推送记录查询与 CSV 导出
5. **UI 文案统一改名"共管 → 守护"**：所有用户可见处，包括小程序、H5、系统消息、邀请页等
6. **关注公众号引导**：小程序接受邀请成功页强制引导关注「贝尼健康」公众号

## 使用说明

### 一、子女发起邀请并守护老人（场景 A / B）

1. 子女打开小程序 → 进入「我的家人」（或健康档案 → 守护与提醒），点击「邀请家人守护」
2. 系统生成邀请二维码并自动分享文案为「邀请你守护家庭健康档案」
3. 老人扫描二维码 → 自动跳转到「守护邀请确认页（/family-auth）」
4. 老人点击「同意」 → 系统建立守护关系并弹出**强制关注公众号引导**
5. 老人长按公众号二维码识别并关注后，点击「已关注，打开首页」
6. 双方均收到系统消息：「XXX 已成为您的守护者」 / 「您已成功守护 XXX」

### 二、老人发起邀请请子女守护（场景 C - 老年大学主流）

1. 老人在老年大学课上学着自己注册并建档
2. 在小程序「我的家人」点「邀请家人守护」
3. 子女扫码 → 进入接受邀请页 → 同意 → 完成守护关系
4. 现在子女是守护者，老人是被守护者，与场景 A/B 角色镜像

### 三、虚拟老人档案 → 老人注册后接管

1. 子女拍照上传老人的体检报告，但老人尚未注册：子女在小程序「我的家人」时**填写老人手机号**作为虚拟档案标识
2. 老人后续在小程序自助注册（手机号 + 验证码）
3. 注册成功后小程序首页弹出「家庭档案接管」入口 / 直接跳转 `/pages/family-migration/index`
4. 老人在该页面查看待接管档案（含子女昵称、关系、档案信息）
5. 点击「确认接管」 → 系统：① 把虚拟档案的 `member_user_id` 绑定到老人；② 自动建立守护关系（子女为 manager）
6. 老人也可选择「拒绝」，档案保留在子女账号下不动

### 四、体检异常聚合推送

1. 被守护者的体检报告解析完成（OCR + 异常项判定 ≥ 1 项）
2. 系统自动触发 `/api/internal/checkup/parsed` 内部回调
3. 守护者集合算法计算出全部守护者
4. 5 分钟同 `report_date` 去重通过后，对每位守护者下发**聚合 1 条**异常推送
5. 守护者在「我的消息」或公众号 / App 推送可看到："您的父亲张大爷的体检报告有 5 项异常，请及时关注"
6. 点击消息进入 `checkup-detail` 查看详情

### 五、管理员后台

1. 登录管理后台 → 左侧菜单「系统消息管理」展开
2. 三个新菜单：
   - **异常文案模板**：管理 5 种内置模板，支持插入占位符 `{relationship}` `{nickname}` `{count}`，预览框实时渲染示例
   - **异常阈值配置**：增删改查 `metric_code / metric_name / severity / lower_bound / upper_bound` 等，本期全部全局生效
   - **推送记录**：按时间区间 / 守护者 / 被守护者 / 通道 / 状态 / 是否点击多维筛选，支持「导出 Excel」（CSV 含 BOM，Excel 直接打开）

## 注意事项

1. **测试账号**：13800138000 / 13800000001 / 13800000002 三个手机号是后端预置测试号，验证码固定为 `123456`，便于联调
2. **守护者上限**：单个被守护者最多 3 位守护者，建档人 + 双向共管对方累计计算
3. **强制关注公众号**：未关注公众号时，公众号通道推送会失败，但小程序订阅消息正常下发；首页会有红条提示
4. **5 分钟去重**：基于 `(member_id, report_date)` 内存窗口；多次解析同份报告只推 1 次
5. **管理后台导出**：默认导出近 7 天数据，单次最多 1 万条，CSV 含 BOM 头，Excel 直接打开中文不乱码
6. **解绑通知**：守护者退出 / 被守护者踢人，双方均收到系统通知；历史数据不撤回，仅停止接收新推送
7. **App 通道**：App 模板消息推送通道接入留作下个迭代（与本期小程序订阅消息同套数据，无缝复用）
