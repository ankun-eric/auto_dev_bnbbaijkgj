## 功能简介

本次更新实现了**退款与核销的互斥业务保护机制**，解决了用户在申请退款期间核销码仍可被使用的业务漏洞。

核心功能包括：

1. **退款申请中的订单禁止核销** — 用户提交退款申请后，核销码立即锁定，商户无法扫码核销
2. **退款被拒/撤回后自动恢复核销** — 退款被管理员拒绝或用户主动撤回后，核销码自动恢复可用
3. **退款成功后核销码标记已退款** — 退款成功后核销码保留但标记为"已退款"，仅供历史查询
4. **已核销订单退款需人工审核** — 有核销记录的订单申请退款，系统自动标记为需人工审核
5. **退款审核页展示核销详情** — 管理后台退款审核时展示核销进度、明细和操作人信息
6. **商户端友好提示** — 核销被拦截时向商户展示明确的提示信息
7. **用户可撤回退款申请** — 新增撤回退款功能，撤回后核销码自动恢复

## 使用说明

### 一、用户端操作指南（H5/小程序/Flutter App）

#### 1. 查看核销码状态

进入订单详情页，核销码区域会根据退款状态展示不同内容：

- **正常状态**：核销码正常显示，可出示给商户扫码核销
- **退款处理中**：核销码置灰，显示"退款处理中，核销码暂时不可用"
- **已退款**：核销码置灰，显示"该订单已退款，核销码已失效"
- **退款被拒**：核销码恢复正常，可继续使用

#### 2. 申请退款

1. 进入订单详情页
2. 点击"申请退款"按钮（仅在无退款/退款被拒时显示）
3. 填写退款原因和金额
4. 提交退款申请
5. 提交后核销码立即锁定，商户无法核销

> **注意**：如果订单中已有核销记录（即部分或全部商品已核销），退款申请会被标记为"需人工审核"，管理员会根据核销情况决定退款金额。

#### 3. 撤回退款

1. 在退款状态为"退款申请中"时，订单详情页会显示"撤回退款"按钮
2. 点击"撤回退款"，确认后退款申请将被撤回
3. 核销码自动恢复可用状态

### 二、商户端操作指南（核销小程序 / Flutter App）

#### 1. 正常核销

1. 打开核销小程序或 App 中的核销功能
2. 扫描用户出示的核销二维码
3. 确认订单信息后点击"确认核销"
4. 核销成功

#### 2. 核销被拦截

当用户的订单正在退款处理中时，商户扫码后会看到：

- **退款处理中**：橙色警告提示"该订单正在退款处理中，暂时无法核销"
- **已退款**：红色提示"该订单已退款，核销码已失效"

此时无法进行核销操作，需等待退款流程结束。

### 三、管理员操作指南（管理后台）

#### 1. 审核退款申请

1. 登录管理后台 → 订单明细 → "退款申请"标签页
2. 找到待审核的退款申请
3. 点击"批准退款"或"拒绝退款"

#### 2. 查看核销详情

当退款订单有核销记录时，审核弹窗中会显示：

- **核销进度**：如 "3 / 10 次 (30%)"
- **核销明细**：每次核销的时间、门店和操作人
- **审核退款金额**：管理员可自行填写退款金额（默认为订单实付金额）

#### 3. 退款审核决策

- **批准退款**：订单退款成功，核销码标记为"已退款"
- **拒绝退款**：订单恢复正常，核销码自动恢复可用

## 注意事项

1. **退款与核销不可同时进行**：一旦提交退款申请，核销码立即锁定，直到退款流程结束
2. **已核销订单退款需人工审核**：有核销记录的订单不支持自动退款，必须由管理员审核
3. **退款金额由审核人决定**：系统不做自动计算，管理员根据核销情况自行决定退款金额
4. **撤回退款有时效限制**：只有在"退款申请中"状态才能撤回，进入审核后无法撤回
5. **核销码不会被删除**：退款成功后核销码保留在系统中，标记为"已退款"，供历史查询
