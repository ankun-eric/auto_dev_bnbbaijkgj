## 功能简介

本次升级正式将平台"旧版积分会员等级"体系全面废弃，由全新的「付费会员套餐」替代，并对积分用途做了重要扩展：

1. **旧"积分会员等级"体系整体废弃**：原本依据消费/积分自动划档的"普通/银卡/金卡/钻石"等级标签全部取消，不再影响用户能享受到的服务。
2. **后台「会员等级」菜单已下线**：管理员侧不再有该入口（旧 API 保留兼容期，OpenAPI 标记为 `deprecated`，预计 1~2 个版本后整体清理）。
3. **新版「付费会员套餐」上线**：支持守护版 / 家庭版 / 年度版等不同套餐，权益更清晰（AI 电话告警、AI 用药提醒、守护人扩容、商城折扣等）。
4. **积分系统完整保留并扩展**：
   - 签到/打卡/健康任务赚积分流程不变
   - 积分商城（纯积分兑换）业务流程不变
   - **新启用**：在商品支持的前提下，普通商城下单可按比例使用积分抵扣金额，**单笔最多抵扣订单金额的 20%**
   - **不可用于付费套餐购买/续费**
5. **商品后台"积分设置" Tab 改名为「会员与积分」**，并新增 3 个独立开关：
   - 是否支持付费会员折扣（`is_member_discount_eligible`）
   - 是否进入积分商城（`points_exchangeable`）
   - 是否支持积分抵扣订单金额（`points_deductible`）

## 使用说明

### 一、普通用户体验流程

#### 1. 查看付费会员套餐（H5 用户端）

1. 打开 [H5 用户端首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/) 并登录账号
2. 进入"我的"页面，原本展示"会员等级"标签的位置不再显示历史等级
3. 若已开通付费会员（守护版/家庭版/年度版），头像下方会自动出现对应"付费会员套餐"标签
4. 进入"客服中心"，搜索关键字"会员"，可看到新的付费会员说明文案

#### 2. 收银台下单 —— 会员折扣 + 积分抵扣二选一

1. 在 H5 端任意普通商城商品下单（不是积分商城兑换，也不是付费会员套餐购买）
2. 进入收银台（`/checkout`）后：
   - **付费会员折扣卡片**：仅当商品后台勾选了「是否支持付费会员折扣」且您是付费会员（套餐折扣率 < 1）时显示
   - **积分抵扣卡片**：仅当商品后台勾选了「是否支持积分抵扣订单金额」且您积分余额 > 0 时显示
3. 两个开关**互斥**：
   - 打开"会员折扣"会自动关闭"积分抵扣"（页面顶部提示"已切换为会员折扣，积分抵扣已关闭"）
   - 打开"积分抵扣"会自动关闭"会员折扣"（提示"已切换为积分抵扣，会员折扣已关闭"）
4. 积分抵扣自动按 **20% 上限** 计算：例如商品总价 100 元、用户有 10000 积分，最多只能抵扣 20 元（消耗 2000 积分），余额仍保留 8000 积分
5. 底部支付栏会清晰展示"小计 / -¥X（会员） / -¥Y（积分） / 合计"

#### 3. 付费会员套餐购买（不允许积分抵扣）

1. 进入付费会员套餐购买/续费页面（路径取决于运营配置入口）
2. 选择目标套餐（如守护版）并选择月度/年度计费周期
3. **即便您拥有大量积分，付费会员套餐的购买/续费金额也不会被积分扣减**（PRD v1.1 § 九 决策项）

### 二、商家/运营管理员后台流程

#### 1. 旧"会员等级"菜单已下线

- 登录 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin)
- 左侧"积分体系"菜单下只保留：**积分规则 / 积分商城**
- 不再有"会员等级"子菜单（已下线）
- 旧 API `/api/admin/points/levels`（GET/POST/PUT/DELETE）及 `/api/points/level` 保留兼容，OpenAPI 中均标记为 `deprecated=True`，调用时会写一条 WARNING 日志，便于灰度期监控

#### 2. 新的"付费会员套餐"管理

- 左侧"会员管理"菜单下：
  - **付费会员套餐**：CRUD 不同套餐（plan_code / 名称 / 月度年度价 / AI 额度 / 守护人上限 / 商城折扣率 / 权益说明 / 是否启用）
  - **免费会员额度**：单行配置，设置免费会员的 AI 额度与守护人上限

#### 3. 商品编辑：「会员与积分」Tab

- 在「商品管理 → 商品 → 编辑」中
- Tab 已由旧名「积分设置」更名为 **「会员与积分」**
- 内含 3 个独立开关：
  | 开关 | 含义 |
  |---|---|
  | 是否支持付费会员折扣（`is_member_discount_eligible`） | 勾选后，付费会员下单本商品时按其所在套餐的全局折扣率结算 |
  | 是否进入积分商城（`points_exchangeable`） | 勾选后，本商品同时进入积分商城以纯积分兑换 |
  | 是否支持积分抵扣订单金额（`points_deductible`） | 勾选后，普通商城下单时允许使用积分抵扣金额，受 20% 上限约束 |

  > 收银台执行规则：会员折扣 + 积分抵扣 **二选一不可叠加**，默认推荐用户力度更大的一项。

## 注意事项

1. **现有用户的积分余额完全保留**：本次升级不会清空、不会冻结用户积分，原有的签到/任务/邀请等积分获取流程全部保留。
2. **旧 API 兼容期**：`/api/admin/points/levels` 与 `/api/points/level` 仍然可调用，但已标 `@deprecated`。前端如果还在调用，请在下一个迭代切换为 `/api/admin/membership/plans`、`/api/membership/me`。
3. **历史用户的"积分会员等级"字段**：用户表的 `member_level` 字段保留在数据库中（v1.1 与 v1.0 保持一致的"立即清空" 决策），但客户端 UI 已不再消费它（h5-web `UserInfo.memberLevel` 标 `@deprecated`；miniprogram `formatMemberLevel` 退化返回空字符串）。
4. **付费会员套餐购买/续费不接受积分抵扣**：即使 H5/小程序发出 `points_deduction` 字段，后端订阅接口也只会按套餐价计费。
5. **回滚预案**：旧"会员等级"菜单与 API 在第一阶段仅做下线/标 deprecated，可在 30 分钟内通过 nginx + 菜单配置恢复显示；商品 `is_member_discount_eligible` 默认 false，回滚只需禁用下单逻辑分支即可。
