## 二、功能简介

「支付配置」用于集中管理平台所有支付通道的接入参数：

- **4 个内置通道**
  - `wechat_miniprogram` 微信小程序支付（端：小程序）
  - `wechat_app` 微信 APP 支付（端：APP）
  - `alipay_h5` 支付宝 H5 支付（端：H5）
  - `alipay_app` 支付宝 APP 支付（端：APP）
- **AES-256-GCM 加密**：所有敏感字段（API 私钥 / 商户私钥 / 支付密钥 / 应用证书 / 应用私钥 / 应用公钥证书 / 支付宝公钥证书 / 根证书 / 平台证书 / V3 Key / API V2 Key 等）均以 `ENC::AES256::` 前缀加密落库，密钥来自环境变量 `PAYMENT_CONFIG_ENCRYPTION_KEY`（未配置时使用项目内置 32 字节 fallback）。
- **完整性 + 启用守卫**：未配置完整时，启用开关无法打开（API 返 400），下单时若指定通道未启用或未完整，统一返回 `{"code":4001,"message":"该支付方式暂未开通..."}`。
- **测试连接**：在配置完整时执行参数自检（保留 wechatpayv3 / alipay-sdk 占位接入点），并打 `last_test_at/last_test_ok/last_test_message` 时间戳。
- **订单字段透出**：`unified_orders` / `orders` 表新增 `payment_channel_code` + `payment_display_name`，订单详情接口返回 `payment_method_text`（格式 `{显示名}（{端名}）`），admin-web / H5 / 小程序 / Flutter App 四端订单详情统一在「订单状态」前展示。

## 四、使用说明（管理员）

### 4.1 进入支付配置

1. 浏览器打开 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)，使用管理员账号登录。
2. 左侧菜单点击「支付配置」（图标为信用卡），即进入 [支付配置](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/payment-config) 页面。

### 4.2 配置支付通道

1. 页面顶部 4 个 Tab：微信小程序 / 微信 APP / 支付宝 H5 / 支付宝 APP，按需切换。
2. Tab 主体显示该通道的：
   - **状态摘要卡**：「已启用 / 未启用」「已完整 / 未完整」「最近一次测试结果」
   - **字段视图**：所有字段以掩码形式显示（敏感字段显示 `****` + 后 4 位，普通字段如 AppID/MchID 显示尾 4 位）
   - **启用开关**：未完整时开关不可打开，会弹出提示
   - **测试连接按钮**、**编辑按钮**

### 4.3 编辑通道参数

1. 点击「编辑」打开右侧 720px Drawer。
2. 必填项：
   - **显示名称**：终端展示给用户的支付方式名称（如「微信支付」「支付宝」）
   - **AppID / MchID / API Key / API V3 Key 等**：按通道字段规格填写
3. **支付宝双 access_mode**：
   - **公钥模式**：填「应用私钥」+「支付宝公钥」
   - **公钥证书模式**：填「应用私钥」+「应用公钥证书」+「支付宝公钥证书」+「支付宝根证书」（4 个证书字段任意一个长文本可点「上传 .pem 文件」按钮自动回填）
4. **notify_url**：右侧「还原默认」按钮可一键恢复成系统推荐回调地址。
5. 底部 3 按钮：
   - **取消**：关闭 Drawer 不保存
   - **仅保存**：写入 DB 后关闭，不触发联通测试
   - **保存并测试连接**：先保存再调测试接口，结果在状态摘要卡中实时刷新

### 4.4 启用通道

1. 字段填写完整后，状态摘要卡「已完整」会显示绿色。
2. 切换「启用开关」到打开状态，后端会校验完整性，未完整会拒绝并 toast 错误。
3. 启用成功后，C 端收银台 / 下单接口即可指定 `channel_code` 走真实支付流程。

### 4.5 测试连接

- 任意时刻点击「测试连接」按钮，后端会做参数完整性 + 签名工具自检（占位真实 SDK 调用），返回成功/失败和消息。
- 结果会回写到 `last_test_at` / `last_test_ok` / `last_test_message` 字段并在状态摘要卡显示。

## 八、注意事项

1. **敏感数据安全**
   - 所有敏感字段以 AES-256-GCM 加密落库，密钥来自环境变量 `PAYMENT_CONFIG_ENCRYPTION_KEY`，**生产环境必须通过 docker-compose / Kubernetes Secret 等方式安全注入**，不要写在仓库里。
   - 一旦密钥变更，旧密文将无法解密，请妥善保管并做好密钥版本规划。
2. **测试连接说明**
   - 当前测试接口为「参数完整性 + 签名工具自检」轻量实现，并未真实联通微信支付 / 支付宝平台。
   - 代码中已保留 `wechatpayv3` / `alipay-sdk` 真实 SDK 调用的占位注释，生产环境正式上线前请接入并执行真实联通验证。
3. **未启用通道兜底**
   - 用户下单时若指定的 `channel_code` 处于「未启用」或「未完整」状态，后端会返回 `{"code":4001,"message":"该支付方式暂未开通..."}`，前端请对此错误码做友好提示。
4. **掩码规则**
   - 敏感字段：`****` + 后 4 位；如果原值不足 4 位，整段用 `****`。
   - 普通字段（AppID / MchID 等）：尾 4 位掩码，前面也做适度遮蔽，降低泄露风险。
5. **角色权限**
   - 「支付配置」菜单仅在用户角色为 `admin` 时显示；后端 admin 接口同步用 `require_role('admin')` 守卫，前后端双层防护。
6. **客户端兼容性**
   - 老订单（无 `payment_channel_code`）的 `payment_method_text` 字段为 `null`，前端在该字段为 `null` 时回退显示原 `payment_method` 文案，保证向后兼容。
