## 功能简介

本次修复批次共解决 **2 个 UI/交互层 Bug**，覆盖 admin 后台与三大客户端（H5 / 微信小程序 / Flutter App）：

1. **Bug 1 — 营业时间结束下拉重复出现 22:00**
   admin 后台「门店管理 → 编辑/新增门店」中，营业时间的「截止时间」下拉框，原本会重复展示两次 `22:00` 选项，造成用户视觉错乱与选择歧义。本次将多余的那条 `22:00` 移除，整个下拉范围保持 `07:00 – 22:00`，所有时间唯一不重复。

2. **Bug 2 — 客户端订单"改约"按钮文案过长撑破布局**
   当订单已达到改期次数上限（默认 3 次）、不可再次改约时，所有客户端订单列表与订单明细页底部的"改约"按钮，原文案 `改约（已无法改期）` 太长，会换行/截断/撑破按钮容器布局。本次将文案统一精简为 `改约` 两个字，**置灰样式保留**（视觉上明确不可用），点击行为继续保留 Toast 提示「本订单已达改期上限」。

## 使用说明

### 验证 Bug 1（营业时间结束下拉去重）

1. 浏览器打开 [Admin 后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 使用管理员账号登录
3. 左侧菜单进入「商家管理 → 门店管理」
4. 点击列表中任一门店行的「编辑」按钮（或顶部「新增门店」按钮）
5. 在表单中找到「营业时间」字段
6. 点击右侧「截止时间」下拉框
7. **预期看到**：下拉选项从 `07:30` 到 `22:00`，每个时间只出现一次，**`22:00` 不再重复出现**
8. 同时点击左侧「开始时间」下拉，从 `07:00` 到 `21:30` 也是唯一不重复

### 验证 Bug 2（改约按钮文案统一）— H5 端

1. 浏览器打开 [H5 我的订单](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/unified-orders/)
2. 使用客户账号登录
3. 找到一笔已达改期上限的订单（已改约 3 次的订单）
4. 在订单卡片右下角观察「改约」按钮
5. **预期看到**：按钮文字仅为 `改约` 两个字，置灰显示，不撑破卡片布局
6. 点击该按钮 → 弹出 Toast「本订单已达改期上限」
7. 进入该订单详情页，底部操作区按钮同样为 `改约`（置灰）+ 同一 Toast

### 验证 Bug 2 — 微信小程序端

1. 按下方「微信小程序体验」章节，把最新 zip 导入【微信开发者工具】
2. 模拟器登录后，进入「我的 → 我的订单」
3. 找到已达改期上限的订单，按钮文案应为 `改约`（置灰）
4. 点击 → wx.showToast「本订单已达改期上限」

### 验证 Bug 2 — Flutter App（Android / iOS）

1. 按下方「安卓端 / iOS 端体验」章节安装最新包
2. 启动 App → 登录 → 「我的 → 我的订单」
3. 同样验证按钮文案为 `改约`（置灰）+ 点击 SnackBar 提示
