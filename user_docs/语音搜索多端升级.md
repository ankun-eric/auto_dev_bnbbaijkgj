# 比尼健康 — 语音搜索多端升级 使用手册

## 功能概述

本次升级为语音搜索功能增加了搜索来源追踪（`source` 参数），支持区分文本搜索与语音搜索，并在三端（H5、微信小程序、Flutter App）统一传递该参数。

---

## 各端使用说明

### H5 端

**访问地址**：https://newbb.test.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/search

**语音搜索流程**：
1. 打开搜索页面，点击搜索框右侧的 **麦克风图标**
2. **长按**语音按钮开始录音（需授权麦克风权限）
3. 录音中会显示波形动画和计时器（最长 15 秒）
4. **松开**按钮结束录音，系统自动进行语音识别
5. 识别成功后，文字自动填入搜索框，并开始 **2 秒倒计时**自动搜索
6. 倒计时期间可手动编辑文字或直接搜索
7. 搜索结果页会自动标记本次搜索来源为"语音"

**文本搜索**：
- 直接在搜索框输入文字并搜索，搜索来源自动标记为"文本"

### 微信小程序

**语音搜索流程**：
1. 在首页或搜索页面，点击 **麦克风图标**
2. **长按**语音按钮开始录音（使用微信内置语音识别 WechatSI 插件）
3. 松开按钮后，微信自动完成语音转文字
4. 识别结果填入搜索框，2 秒后自动执行搜索
5. 搜索来源正确标记为"语音"

**注意事项**：
- 小程序使用微信原生的 `RecorderManager` 和 `WechatSI` 插件
- 手动输入、选择热门标签或点击搜索联想时，搜索来源会重置为"文本"
- 首次使用需授权麦克风权限

---

### Flutter App

**语音搜索流程**：
1. 打开搜索页面，点击搜索框右侧的 **麦克风图标**
2. **长按**语音按钮开始录音
3. 松开按钮后，音频发送至后端 ASR 接口进行识别
4. 识别结果填入搜索框，2 秒后自动搜索
5. 搜索来源标记为"语音"

## API 变更说明

### GET /api/search

新增查询参数 `source`：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| source | string | "text" | 搜索来源：`text`（文本搜索）或 `voice`（语音搜索） |

**行为说明**：
- `source` 参数不影响搜索结果，仅用于日志记录
- 非法的 source 值会被规范化为 `text`
- 搜索响应中不包含 `source` 字段
- 屏蔽词命中时，`source` 也会正确记录到搜索日志

## 技术要点

1. **后端**：`unified_search` 函数新增 `source` 查询参数，值通过白名单校验（仅 text/voice），其他值规范化为 text
2. **H5**：`doSearch` 函数接受 source 参数，语音搜索成功后传 `voice`，URL 中通过 `source=voice` 传递给结果页
3. **小程序**：使用实例属性 `_pendingVoiceSource` 追踪语音来源，手动输入/标签/联想点击时重置
4. **Flutter App**：`_doSearch` 接受 source 参数，通过 `Navigator.pushNamed` 的 arguments Map 传递
