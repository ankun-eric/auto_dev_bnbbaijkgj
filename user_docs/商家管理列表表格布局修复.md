## 功能简介

本次修复解决了商家后台「商家管理 → 门店管理」列表页在 1366×768 笔记本小屏上的**布局破版**问题：

- **修复前**：表格列宽自动撑大，最右侧「操作」列（编辑 / 营业管理 / 停用）冲出表格容器，把页面右侧边框撑破，"停用 / 启用"按钮难以点击。
- **修复后**：
  - 表格的所有列都有合理的固定宽度。
  - 当窗口宽度不够（如 1366 笔记本）时，**表格内部自动出现横向滚动条**，整页不再被撑破。
  - 在 1920×1080 等大屏下，所有列自然铺开，不会出现多余滚动条。
  - 操作列三个按钮在 200px 内整齐展示。

同时对商家管理域下的其他列表页一并加固（门店账号、商家分类、门店内营业管理页），杜绝同类问题再次发生。

## 使用说明

### 一、查看修复效果（门店管理列表）

1. 在浏览器中打开 [门店管理列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/merchant/stores)。
2. 商家账号登录后进入「商家管理 → 门店管理」。
3. 在 **1366×768** 等小屏笔记本上观察：
   - 表格收在容器内，**右侧页面边框完整不再破版**。
   - 表格内部出现横向滚动条，左右滚动可以看到隐藏的列。
   - 「操作」列的三个按钮（编辑 / 营业管理 / 停用 / 启用）整齐排列在一行内。
4. 在 **1920×1080** 大屏上观察：
   - 所有列一次性展开，无横向滚动条。

### 二、查看连带加固的列表页

修复同样应用到了以下列表页（即使将来增加 1~2 列也不会破版）：

| 列表页 | 入口 | 加固说明 |
|--------|------|----------|
| 门店账号 | 商家管理 → 门店账号 | 全列显式宽度 + scroll.x = 1200，操作列 320px |
| 商家分类 | 商家管理 → 商家分类 | 全列显式宽度 + scroll.x = 1100，操作列 200px |
| 门店内营业管理 | 门店管理列表行 → 营业管理按钮 → 服务级并发上限表 | 全列显式宽度 + scroll.x = 800 |

### 三、表格交互（与修复前一致，未受影响）

- ✅ 排序、筛选、分页等操作行为不变。
- ✅ 编辑、营业管理、停用 / 启用、删除、重置密码等按钮的点击 / 弹窗行为不变。
- ✅ 列表加载中、空数据、多页数据三种状态下都能正确展示。

> 在 1920×1080 大屏上内容区可用宽度 ≈ 1700 px > 1400 px，**不会出现横向滚动条**；在 1366×768 小屏上 ≈ 1100 px < 1400 px，**表格内部自动出滚动条**。

## 注意事项

1. **缓存刷新**：首次访问修复后的页面，浏览器可能缓存了旧的样式资源，请按 `Ctrl + Shift + R`（Windows）或 `Cmd + Shift + R`（Mac）强制刷新一次。
2. **无需重新登录**：本次修复不涉及登录态、Token、Session，刷新页面即可看到效果。
3. **端到端无影响**：
   - 后端、数据库、H5、小程序、APP 端均**零改动**。
   - 不影响其他业务模块（订单、商品、积分等）的列表页。
4. **小屏体验最佳实践**：分辨率 ≥ 1366 即可正常使用；如果分辨率更小（如 1280×720），表格内部滚动条会更频繁出现，但布局仍可保持完整。
5. **浏览器兼容**：Chrome / Edge / Firefox / Safari 主流浏览器全部兼容。
