## 功能简介

本次修复了「健康档案 → 睡眠 → 详情页 → 历史记录 → **全部**」页面长期存在的两个 Bug：

1. **列表显示残缺**：每条睡眠记录此前只显示成 `13:01  - h`，睡眠时长与深睡时长丢失，日期渲染异常；本次修复后与睡眠详情页历史记录**完全一致**，正确展示「**X 小时 · 深睡 Yh**」+ 状态档位胶囊 + 来源/日期行。
2. **编辑保存异常**：之前点击「⋯ → 修改」弹出的编辑弹窗里没有睡眠对应的输入框，强行录入也无法保存。本次修复后，弹窗中新增「**睡眠时长（小时）**」与「**深睡时长（小时，选填）**」两个输入框，保存后立即按睡眠双值结构正确写回后端，**更新原记录、即时刷新列表**。

根因是四指标通用「全部历史」模板（H5）漏掉了"睡眠"这种双值结构，本次将睡眠纳入模板的全部四处硬编码逻辑：列表数值格式化、状态档位配置、编辑字段配置、保存 value 拼装。

## 使用说明

### 步骤 1：进入睡眠全部历史页

1. 打开 H5 主入口 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/) 并登录
2. 进入「健康档案」首页
3. 点击「睡眠」卡片进入睡眠详情页
4. 在历史记录区域，点击右上角「**全部**」按钮，进入「睡眠全部历史」页面

### 步骤 2：查看修复后的列表展示

进入「睡眠全部历史」页后，每条记录将以以下形式完整展示：

- **第一行**：`07:20  8 小时 · 深睡 2.5h` （时间 + 睡眠总时长 + 深睡时长）
- **右侧档位胶囊**：根据睡眠时长自动着色
  - **睡眠充足**（7~9 小时）：蓝色
  - **睡眠偏少**（6~7 小时）：黄色
  - **睡眠不足**（< 6 小时）：橙色
  - **睡眠偏多**（> 9 小时）：黄色
- **第二行**：`手工录入 · 2026-06-01` （数据来源 + 测量日期）

### 步骤 3：使用筛选功能

页面顶部的筛选区现已正确支持睡眠四档：

- 📅 日期范围：近 7 天 / 近 30 天 / 近 90 天 / 全部
- 🩺 状态档位：全部 / 睡眠不足 / 睡眠偏少 / 睡眠充足 / 睡眠偏多
- 📡 数据来源：全部 / 手工录入 / 智能设备同步

### 步骤 4：修改一条睡眠记录

1. 在列表中找到某条**手工录入**的记录（设备同步的记录仍保持只读，不显示「⋯」按钮）
2. 点击该行最右侧的「**⋯**」三点按钮
3. 在弹出的底部操作面板中选择「**修改**」
4. 编辑弹窗会显示：
   - **睡眠时长（小时）**：必填，例如 `7.5`
   - **深睡时长（小时，选填）**：选填，例如 `2`，留空也可
5. 录入数值后点击「**保存**」按钮
6. 提示「**已更新**」后，列表自动刷新，原记录被覆盖（不会新增重复记录）

### 步骤 5：删除一条睡眠记录

1. 在列表中点击某条手工录入记录右侧「⋯」
2. 选择「**删除**」
3. 在二次确认弹窗中点击「**确认删除**」
4. 提示「**已删除**」后列表自动刷新

## 注意事项

1. **睡眠时长校验规则**：
   - 总睡眠时长必须在 **0 ~ 24 小时**之间
   - 深睡时长不可超过总睡眠时长
   - 校验未通过时会有 Toast 提示，记录不会被保存

2. **档位标准统一**：睡眠档位判定与「睡眠详情页」已上线版本**完全一致**（仅按总睡眠时长一个维度判定），整个产品链路标准统一。

3. **设备同步记录只读**：智能手环/智能床垫等设备同步上来的睡眠记录会被标记为「设备同步」并显示锁定提示，**无法在线编辑**；如需修正，请重新测量。

4. **多端自动同步**：因为微信小程序与安卓 / 苹果 App 都通过 `web-view` 内嵌同一份 H5，所以本次修复在 H5 部署后**所有端立即同步生效**，无需重新打包或重新安装。

5. **其他四个指标不受影响**：本次仅补全睡眠在通用模板中的缺失逻辑，血压、血糖、心率、血氧四个指标的全部历史页**保持原有逻辑不变**。
