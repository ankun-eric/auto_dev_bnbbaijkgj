## 二、功能介绍（运营侧）

### M1. 商品三态流转 + 字段锁定

商品在其生命周期中只会处于以下三种状态之一：

- **草稿（draft）**：新创建的商品默认草稿态，用户不可见，所有字段可自由编辑。
- **在售（on_sale）**：用户端可见、可兑换。此时部分关键字段会被锁定（见下方锁定清单）。
- **已下架（off_sale）**：用户端完全不可见；后台列表默认隐藏，可通过筛选查看；已兑换用户在"我的兑换记录"中仍可查看该商品。

**在售状态下的锁定字段**（置灰 + 悬停提示）：

| 字段 | 锁定 | 说明 |
|---|---|---|
| 商品类型 | 🔒 | 一旦在售不可切换（防止已持券用户和商品错乱） |
| 关联优惠券 / 关联服务 | 🔒 | 已兑换用户持券中，不可改 |
| 积分价格 | 🔒 | 公平性保障 |
| 有效期开始时间 | 🔒 | — |
| 商品标题、主图、轮播图 | ✏️ 可改，自动留痕 | 修改会写入「修改历史」 |
| 富文本详情、库存、单人限购、排序权重 | ✏️ 自由改 | 不留痕 |

**库存防呆**（仅对优惠券类商品）：

- 填写库存时下方会实时展示 📎 `当前关联优惠券总量：XX 张（仅供参考，实际库存以下方填写为准）`
- 若填写值超过优惠券总量，**实时弹出黄字警告**："⚠️ 当前填写 200，优惠券总量仅 150，超出 50 张，兑换时可能因券不足而失败。"
- **保存时若仍超量**，弹窗二次拦截确认。

### M2. 修改历史「历史」Tab

商品编辑弹窗内新增「修改历史」按钮（表格每一行"历史"动作）。

- 记录范围：仅针对**在售状态下可改但留痕**的字段——商品标题、主图、轮播图。
- 字段筛选：支持按"商品标题 / 主图 / 轮播图"筛选。
- 默认按时间倒序。
- 图片类字段展示缩略图，原图可另开查看。

---

### M3. 复制新建 + 无缝替换

当运营需要修改已在售商品的锁定字段（如积分价格、商品类型）时，可使用"复制新建"功能。

**操作步骤**：

1. 在「积分商城商品管理」列表中，对一个**在售**商品点击操作栏的【复制新建】。
2. 系统会自动创建一份**草稿副本**（商品名会自动加后缀"(副本)"），进入编辑页可任意修改包括锁定字段在内的所有内容。
3. 编辑完成后点击【发布】，系统会：
   - 新商品状态变为"在售"；
   - **原商品自动下架**，并被打上 `被《新商品名》替代` 的标签。
   - 原商品的 `replaced_by_goods_id` 指向新商品。

**对 C 端的影响**：已兑换过原商品的用户，在「我的兑换记录」中点击【去看替代款】会自动跳转到新商品详情页。

### M4. wangEditor v5 富文本编辑器 + 模板库（T1~T4）

商品编辑页「详情」字段在本期使用加强版 TextArea，并提供 4 套图文卡片模板一键插入。

**模板列表**：

| 模板 | 用途 |
|---|---|
| T1 商品亮点卡 | 开篇介绍核心卖点（图标 + 3 条亮点）|
| T2 使用说明卡 | 说明如何使用（步骤条 1-2-3-4）|
| T3 有效期提示卡 | 强调时效（高亮色块 + "X 天内有效"）|
| T4 注意事项卡 | 风险告知（黄色警示框 + 条目列表）|

**操作步骤**：

1. 打开商品编辑弹窗，找到「详情（富文本）」。
2. 点击下方的【T1 亮点卡】【T2 使用说明卡】【T3 有效期提示卡】【T4 注意事项卡】任一按钮。
3. 预置 HTML 会自动插入到当前编辑区，你可以继续自由编辑。
4. 大小限制：详情总大小 ≤ 50KB（约 5000 字 + 10 张图外链）。

> 浏览器兼容：Chrome / Edge / Firefox 最新 3 个主版本。

## 三、功能介绍（用户侧）

### M5. 用户端列表交互优化

进入 [积分商城列表页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/points/mall) 即可体验：

#### 1）全部 / 可兑换 Tab

- **全部**：展示所有在售商品（默认）。
- **可兑换**：只展示"我的积分 ≥ 商品积分 **且** 库存 > 0"的商品。
  - 🧠 兜底规则：若"可兑换"数量 < 3 件，自动降级为"只要库存 > 0"，避免新用户积分少时 Tab 空荡。
- 右上角红点：当"可兑换"Tab 有商品时亮起，提醒用户有可立即兑换的商品。

#### 2）卡片改版

每张商品卡片：

- **商品主图 + 类型标签**：直观可见。
- **商品名 + 积分价**：文字清晰。
- **库存提示**（仅在库存紧张时显示）：
  - `仅剩 X 件`（红字，当 `stock ≤ 10` 时显示）；
  - `已兑完`（当 `stock = 0` 时显示）。
- **立即兑换按钮**：卡片底部主色按钮，直接点击可触发兑换流程。
- 点击卡片的空白区域则进入商品详情页。

#### 3）按钮五种状态（覆盖全部场景）

| 场景 | 文案 | 交互 |
|---|---|---|
| 在售 + 库存 > 0 + 积分够 | 立即兑换 | 点击直接走兑换流程 |
| 在售 + 库存 > 0 + 积分不够 | 立即兑换（outline）| 点击提示"积分不足，还差 X 积分" |
| 在售 + 库存 = 0 | 已兑完 | 置灰禁用 |
| 已下架 + 有替代商品 | 去看替代款 | 跳转到替代商品详情页 |
| 已下架 + 无替代商品 | 已下架 | 置灰禁用 |

#### 4）智能排序

列表按以下优先级排序：

1. 库存 > 0 的商品排前（库存 0 沉底）；
2. 同组内：用户积分够兑换的排前；
3. 同组同能力内：按运营配置的「排序权重」（越大越靠前）；
4. 最后按新上架优先。

#### 5）"使用按钮"智能跳转

在 [我的兑换记录](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/points/exchange-records) 中：

- 若已兑换商品仍在售 → 按原逻辑展示（服务券去预约、优惠券看我的券、实物看订单）。
- 若已下架 **且** 有替代商品 → 展示【去看替代款】（橙色按钮），点击直接跳到替代商品详情页。
- 若已下架 **且** 无替代 → 【已下架】灰色禁用按钮。

## 四、操作步骤速查

### 运营：创建一个新商品并上架

1. 进入 [管理后台 → 积分商城商品管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/points/mall)；
2. 点击【新增商品】→ 填写名称、类型、积分价、库存、主图等 → 保存为**草稿**；
3. 在列表中点击该草稿的【发布】按钮 → 状态变为"在售"；
4. 用户端自动可见并可兑换。

### 运营：修改在售商品的价格（需走"复制新建"）

1. 在商品列表找到该在售商品，点击【复制新建】 → 得到一个草稿副本；
2. 打开副本编辑页，修改积分价等锁定字段；
3. 点击【发布】→ 新商品上架，**原商品自动下架并打"被替代"标签**；
4. 已兑换过原商品的用户，点使用按钮会自动跳转到新商品。

### 运营：查看某个商品的修改历史

1. 在商品列表找到目标商品，点击【历史】；
2. 弹出的修改历史 Tab 中可按字段筛选，查看每次修改的前后对比。

### 用户：快速找到我能兑换的商品

1. 打开 [积分商城](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/points/mall)；
2. 顶部切到【可兑换】Tab；
3. 看到红点说明有能兑换的商品，直接点商品卡片的【立即兑换】即可。

## 五、注意事项

1. **三态迁移兼容**：已上架的旧商品自动标记为"在售"，未上架的自动标记为"草稿"，不会丢失任何数据。
2. **库存字段只能减到已兑换数**：若已有 50 人兑换，库存不能改为小于 50。
3. **排序权重**：越大越靠前，建议重点推广商品填 ≥ 100。
4. **已下架商品用户侧完全不可见**，但"我的兑换记录"里仍有商品快照（标"已下架"）。
5. **小程序端**：如需更新发布，请下载 [小程序最新包](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/downloads/miniprogram_latest.zip) → 在微信开发者工具中导入该 zip → 上传。
6. **浏览器兼容**：后台仅支持 Chrome / Edge / Firefox 最新 3 个主版本，**不支持 IE 11 及以下**。
7. **富文本详情**总大小限制 50KB，超出会被拦截。
8. **"可兑换"Tab 兜底规则**：若你没看到"可兑换"里有积分不够的商品，这是正常的——只有当可兑换数量 < 3 时才会降级兜底展示。
