## 功能简介

本次上线全新的「付费会员体系」，全面取代旧有的「积分会员等级」（按积分自动划档）模式。新体系的关键能力：

- **付费订阅模型**：管理员在后台配置多套套餐（守护版、家庭版、年度版……），用户在用户端付费订阅，获得 AI 电话外呼额度、AI 提醒额度、守护人数量上限、商城折扣等可量化权益。
- **免费会员兜底**：未付费用户默认享有平台兜底的免费额度（异常告警 3 次/月等），由后台「免费会员额度配置」页面统一维护。
- **商品级会员/积分双开关**：商品编辑页【会员与积分】Tab 内新增「是否支持付费会员折扣」开关，与已有的「积分商城兑换」「积分抵扣订单金额」并列。
- **收银台二选一**：单笔订单中「会员折扣」与「积分抵扣」**不可叠加**，由用户在收银台显式选择，系统默认推荐力度更大的一项。
- **积分抵扣 20% 上限**：单笔订单使用积分最多抵扣订单金额的 **20%**，超出部分前端拦截。
- **套餐购买/续费不允许积分抵扣**。

## 使用说明

### 1. 后台：配置付费会员套餐

1. 进入管理后台 → 左侧菜单【会员管理 → 付费会员套餐】
2. 点击右上角「新增套餐」按钮，填写以下字段后保存：
   - **套餐编码**：唯一标识，建议使用英文（如 `guardian` / `family` / `annual`）
   - **套餐名称**：展示给用户的中文名称（如「守护版」）
   - **月度价格 / 年度价格**：年度价格可不填
   - **AI 电话告警额度**、**AI 异常告警额度**、**AI 外呼提醒额度**：单位为「次/月」
   - **守护人数量上限**：该套餐持有者作为被守护人时的上限
   - **商城折扣率**：取值 0.01~1.00，例如 `0.9` 表示商城下单 9 折
   - **套餐权益描述**：将展示在用户端的会员卡片与权益详情页（支持纯文本或 HTML）
   - **排序**：值越小越靠前
   - **是否启用**：关闭后用户端不可购买
3. 已上线的套餐可随时编辑；点击「下线」会软下线（已购用户继续享受到期，不再可被新用户购买）。

### 2. 后台：配置免费会员额度

1. 进入【会员管理 → 免费会员额度】
2. 编辑各字段后点击「保存」即可：
   - AI 电话告警额度（次/月）
   - AI 异常告警额度（次/月）—— 注意：异常告警电话本身平台兜底完全免费，本字段仅控制 App 内告警次数限制
   - AI 外呼提醒额度（次/月）
   - 守护人数量上限
   - 免费会员权益说明

### 3. 后台：商品的【会员与积分】Tab

1. 进入【商品管理 → 商品列表】，新建或编辑任意商品
2. 切换到 **「会员与积分」** Tab（原名「积分设置」，本次升级改名）
3. 三个独立开关 + 一项数值：
   - **是否支持付费会员折扣**：勾选后，付费会员下单本商品时按其所在套餐的全局折扣率执行。
   - **是否进入积分商城（纯积分兑换）**：勾选后，本商品同时显示在【积分商城】，由【积分商城管理】另行配置兑换所需积分。
   - **积分商城兑换所需积分**：仅当勾选了上一项时生效。
   - **是否支持积分抵扣订单金额**：勾选后，普通商城下单本商品时可用积分按系统当前比例抵扣订单金额，单笔最多抵扣订单金额 20%；与「付费会员折扣」**不可叠加**，由用户在收银台二选一。

### 4. 用户端：付费会员相关 API

> 用户端 UI（H5 / 小程序 / APP）将分批接入以下 API；本次上线先开放接口能力。所有接口需登录用户态（除「公开套餐列表」外）。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/membership/plans` | 用户端可购买套餐列表（仅返回启用中的套餐） |
| GET | `/api/membership/me` | 当前用户的会员状态：`is_paid_member` / `discount_rate` / 各类额度 / 到期时间 |
| POST | `/api/membership/subscribe` | 订阅套餐，参数：`plan_id`、`billing_cycle`（monthly/yearly）|
| POST | `/api/membership/cancel` | 取消订阅，立即降级为免费会员 |
| POST | `/api/membership/calculate-discount` | 收银台计算优惠：返回可选的「会员折扣」/「积分抵扣」选项 + 系统推荐项 |

### 5. 收银台优惠计算逻辑（重点）

调用 `POST /api/membership/calculate-discount`，请求体：

```json
{ "product_id": 1, "quantity": 2 }
```

返回结构（节选）：

```json
{
  "product_id": 1,
  "quantity": 2,
  "original_price": 200.0,
  "is_paid_member": true,
  "user_points": 5000,
  "member_discount_eligible": true,
  "points_deductible": true,
  "options": [
    { "type": "member_discount", "label": "付费会员折扣（守护版 0.9）",
      "discount_amount": 20.0, "final_price": 180.0 },
    { "type": "points_deduction", "label": "积分抵扣（最多抵扣订单金额 20%）",
      "discount_amount": 40.0, "final_price": 160.0, "use_points": 4000 }
  ],
  "recommended": "points_deduction"
}
```

要点：
- 当商品同时支持两种优惠时，`options` 内同时返回；前端在收银台**强制二选一**展示，**不可叠加**。
- `recommended` 字段标识系统默认推荐项（按抵扣金额取大者）。
- 积分抵扣金额封顶 = 订单金额 × 20%；超过则按上限计算 `use_points`。
- 套餐购买与续费的下单流程**不调用本接口**，套餐订单永不允许积分抵扣。

## 注意事项

1. **旧「会员等级」菜单已下线**：管理后台左侧菜单中原【积分体系 → 会员等级】已隐藏，依积分自动划档的旧会员体系正式废弃。`member_levels` 数据表保留只读以兼容历史数据，业务不再读取。
2. **旧用户**：本期为全新付费体系开局，**没有任何历史付费用户**。免费用户保持原有行为，仅在订阅成功后获得付费权益。
3. **过期处理**：用户付费过期 / 主动取消订阅后，立即降级为免费会员；按免费额度执行。重新付费后立即恢复付费权益。
4. **下线套餐**：运营把套餐设为「停用」后，已购用户继续享受本周期内的权益，到期后无法续费同套餐，需引导用户切换其他套餐。
5. **商品级开关互不强制互斥**：「付费会员折扣」「积分商城兑换」「积分抵扣订单金额」是 3 个独立开关。`member_discount_eligible = true` 且 `points_deductible = true` 的商品在收银台由用户二选一；`points_exchangeable = true` 的商品默认进入【积分商城】，与普通下单相互独立。
6. **新建商品默认值**：`is_member_discount_eligible` 默认 `false`，由运营按需勾选；其他既有字段保持原默认值。
7. **数据库迁移幂等执行**：后端容器启动时会自动检测并执行 `_sync_membership_v1` 迁移（创建 `membership_plans` / `user_membership_subs` / `free_member_quota` 三张表，并给 `products` 表添加 `is_member_discount_eligible` 字段）。重启不会破坏已有数据。
