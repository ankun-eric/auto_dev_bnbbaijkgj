## 功能简介

本次升级针对门店端订单管理体验进行了一次深度改造，核心变化包括：

1. **订单管理新增「预约看板」视图**：从"翻列表找单"升级为"看时段管业务"
   - **日视图**：3×3 九宫格，对应一天 9 个固定时段（06-08 / 08-10 / ... / 22-24，每段 2 小时）
   - **周视图**：横向 7 列日期卡片，每张卡片显示「预约 N / 已核 X / ¥收入」三行（删除原"占用 %"）
   - **月视图**：标准月历，点击任意日期 → 弹出"上午 / 下午+晚上"左右两栏订单卡片
2. **全平台统一固定 9 段时段**：以「每段 2 小时、最早 06:00 起」固定 9 段为唯一时段切片，不再读取门店营业时间动态生成
3. **改期权完全归客户**：商家端 PC / H5 全无"改时间"入口；客户端三端（小程序 / APP / H5）保留并强化改期能力
4. **PC 端禁止发起核销**：9 宫格抽屉中的"核销"按钮在 PC 端置灰，仅手机端（H5 移动版 / 核销小程序）可发起核销
5. **9 宫格字段精简为绝对值口径**：统一显示「预约 N / 已核 X / 已核金额 ¥Z」，去掉占用率百分比
6. **金额格式统一**：纯数字 + ¥（如 `¥1280`），不带千分位

## 使用说明

### 一、进入预约看板

1. 打开商家端后台：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 使用门店账号登录
3. 进入左侧菜单「商品系统 → 订单管理」
4. 在订单列表页右上角，点击「**切换到预约看板**」按钮
5. 也可直接访问 [预约看板直达链接](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/orders/dashboard/)

### 二、日视图（9 宫格）操作

- 顶部工具条提供「日 / 周 / 月」三视图切换
- 默认进入「日视图」，展示当天 9 个固定时段：

  ```
  06:00-08:00 | 08:00-10:00 | 10:00-12:00
  12:00-14:00 | 14:00-16:00 | 16:00-18:00
  18:00-20:00 | 20:00-22:00 | 22:00-24:00
  ```

- 每格显示：**时段标题 / 预约 N / 已核 X / 已核金额 ¥Z**
- 当前进行中的时段会有「绿色边框 + 进行中」标签高亮
- **点击任一格** → 右侧滑出抽屉，展示该时段所有订单：
  - 客户姓名、完整 11 位手机号
  - 服务项目名称、订单状态
  - **操作按钮组**：
    - 「核销（请到手机端）」按钮：PC 端**置灰**
    - 「详情」按钮：跳转订单详情页
    - 「复制电话」按钮：一键复制手机号到剪贴板，便于商家直接拨打

### 三、周视图操作

- 顶部周次选择器，可前后切换周
- 主体横向 7 列卡片，按周一到周日排列
- 每张卡片显示「预约 N / 已核 X / ¥收入」三行
- **点击任一日期卡片** → 跳转到该日的「日视图」9 宫格

### 四、月视图操作

- 顶部月份选择器
- 主体为标准月历，每个日期格内显示当天「预约 N + 颜色提示」
- **点击任意日期** → 弹出「当日订单详情」弹窗：
  - 顶部：当日汇总（预约 / 已核 / 已核金额）
  - **左栏：上午（06:00-12:00）订单卡片**，按时段升序
  - **右栏：下午+晚上（12:00-24:00）订单卡片**，按时段升序
- 每张订单卡片显示：时段 / 客户姓名 / 完整 11 位手机号 / 项目 / 金额 / 状态

### 五、客户端改期（沿用现有规则）

C 端客户在小程序 / APP / H5 三端均可发起改期：

- 入口：「我的订单 → 订单详情 → 修改预约」按钮
- 选择新日期（明天起最远 90 天）
- 选择时段（按固定 9 段展示，每行 3 个）
- 提交即可

**改期规则**：
- 单订单最多改期 3 次（沿用现有上限）
- 改期到的目标日门店必须营业（休息日置灰不可选）
- 时段必须在门店营业时间内
- **不校验单时段容量**（允许超约，由门店人工协调）
- 商品级 `allow_reschedule = false` 时按钮置灰

### 六、改期通知

客户端改期成功后，平台**并行下发三通道通知**：
- ✅ 微信小程序订阅消息
- ✅ APP push（iOS + Android）
- ✅ 短信

文案模板（统一）：

> 【bini-health】您预约的「{服务项目名}」已改期：原 {原时段}，现 {新时段}，门店：{门店名}。如有疑问请联系门店：{门店电话}。

任一通道成功即视为已触达；全部失败时订单详情页显示"通知发送异常"，并触发企业微信告警。

## 注意事项

### 时段切片规则（重要变化）
- 全平台所有"按时段管理"的入口，统一使用固定 9 段切片
- 凌晨 00:00-06:00 不开放，对应该时段的历史脏数据会在 9 宫格底部显示"⚠️ 凌晨脏数据 N 条"提示，可在列表视图查看处理

### PC 端核销限制
- PC 端**禁止发起核销**：9 宫格抽屉与订单列表中的"核销"按钮一律置灰
- PC 端**仅保留**：「商家端核销记录页」+「Admin 平台核销管理页」**查看 / 审计**功能
- 核销请使用手机端 H5 移动版或核销小程序的扫码 / 输码功能

### 改期入口
- **商家端 PC / H5 无任何"改时间"入口**（本期已做兜底审计）
- 商家无法擅自更改预约时间，改期权 100% 归客户
- 后端接口 `PUT /api/orders/{id}/reschedule` 仅允许 `customer` 角色调用，`merchant` / `admin` 角色调用一律返回 403

### 已核金额口径
- 9 宫格、周视图、月视图弹窗的「已核金额」**仅统计已核销订单**
- 未核销订单（含待核销 / 已取消 / 已改期）**全部不计入**

### 数据范围
- 商家账号：仅可查看自己所属门店的看板数据
- 平台超管账号：可通过 `?store_id=N` 切换查看任一门店

### 浏览器兼容
- Chrome 90+ / Edge 90+ / Safari 14+ 均测试通过

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| 9 宫格点击格子无响应 | 刷新页面或检查网络连接 |
| 抽屉里"核销"按钮点不动 | 这是预期行为，请到手机端核销 |
| 月视图弹窗加载慢 | 当日订单量较多时正常，等待 1-2 秒 |
| 改期后客户没收到通知 | 检查订单详情页的"通知状态"，全失败时联系运营人工电话联系 |

## 接口清单（开发参考）

本次新增 / 变更接口（完整 OpenAPI 文档可访问 `/api/docs`）：

- `GET /api/merchant/dashboard/time-slots` 公开接口，返回固定 9 段时段配置
- `GET /api/merchant/dashboard/day?date=YYYY-MM-DD` 日视图 9 宫格数据
- `GET /api/merchant/dashboard/week?date=YYYY-MM-DD` 周视图 7 天数据
- `GET /api/merchant/dashboard/month?year=YYYY&month=M` 月视图日聚合数据
- `GET /api/merchant/dashboard/month-day?date=YYYY-MM-DD` 月视图某日左右两栏订单
- `GET /api/merchant/dashboard/slot/{date}/{slot_no}` 9 宫格抽屉时段订单列表

所有看板接口需 `Bearer token`（`merchant` / `admin` / `super_admin` 角色），未登录返回 401。
