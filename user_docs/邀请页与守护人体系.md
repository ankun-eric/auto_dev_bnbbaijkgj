## 功能简介

本次新增/升级守护人体系，让用户清晰理解"主守护人 / 普通守护人 / 付费 / 免费"四种身份组合，规范了主守护人转移流程、电话告警串行外呼策略、AI 外呼额度归属、邀请凭证有效期等关键规则。

主要功能：

1. **守护人角色体系**
   - 主守护人（第 1 个绑定的守护人，全局唯一，承担电话告警的首要顺位）
   - 普通守护人（其他守护人，按优先级依次外呼）
   - 角色徽章在 H5 与管理后台均可见

2. **数量上限按会员等级**
   - 免费会员：最多 **3** 位守护人
   - 付费会员：最多 **10** 位守护人
   - 单个用户最多守护 **10** 人（防灰产）

3. **主守护人转移流程**
   - 现任主守护人在 App 中发起转移
   - 系统通知被守护人确认
   - 被守护人点"同意"后立即切换，并向三方推送通知
   - 新主守护人无需主动确认（已通过初次授权）

4. **异常告警串行外呼**
   - 主守护人优先拨打
   - 60 秒无应答 → 第 2 顺位 → 60 秒无应答 → 第 3 顺位…
   - 付费守护人直接收电话
   - 免费守护人有 5 次/月免费电话告警额度，用完降级为推送 + 短信

5. **AI 外呼提醒**
   - 谁设置谁付费（设置者扣自己的 AI 外呼额度）
   - 与异常告警相互独立

6. **邀请凭证规则**
   - 有效期 24 小时
   - 单次有效（接受后立即失效）
   - 已过期 / 已作废可一键重新发送
   - 双向邀请同时存在时，"先确认者生效"

7. **管理后台守护关系查询**
   - 运营/客服可按手机号、昵称、是否主守护人 等条件查询全平台守护关系
   - 显示付费/免费会员状态、优先级、绑定时间

## 使用说明

### 1. 查看我的守护人

1. 打开 H5 浏览器，进入 [守护人体系页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/guardian-system)
2. 顶部"我的守护人"Tab 显示所有守护您的人
3. 每个守护人卡片上会有徽章：
   - ⭐ **主守护人** / 普通守护人
   - 付费 / 免费
4. 主守护人始终排在第 1 位

### 2. 查看我守护的人

1. 进入"我守护的人"Tab
2. 列表显示您正在守护的所有被守护人
3. 显示您在其中是"主守护人"还是普通守护人

### 3. 发起主守护人转移

> 适用场景：现任主守护人因出国、工作变动等原因希望将主守护人身份转给其他守护人。

1. 进入"我守护的人"Tab
2. 点击想要转移的被守护人卡片
3. 在弹出的操作菜单中选择「将主守护人身份转给Ta」
4. 弹出确认框，点击「确定」即发起申请
5. 系统会向被守护人推送通知
6. 等待被守护人在 App 中点「同意」即可完成转移

> 注：被守护人同意后，新旧主守护人 + 被守护人三方都会收到"主守护人已更新"通知。

### 4. 被守护人确认主守护人转移

1. 收到"主守护人转移申请"通知后，打开 [守护人体系页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/guardian-system)
2. 页面顶部会出现待确认转移卡片
3. 点击「同意」即可生效，或点击「拒绝」取消

### 5. 查看异常告警电话额度

1. 进入"守护人体系页面"，页面顶部显示本月剩余电话告警额度
2. 免费会员：5 次/月 ；付费会员：不限次
3. 额度即将用完时（剩 1 次或更少）会出现红色提示

### 6. 查看邀请记录

1. 进入"邀请记录"Tab
2. 每条邀请显示状态徽章：
   - **待确认**（蓝色）：在 24 小时有效期内
   - **已生效**（绿色）：已被接受
   - **已过期**（黄色）：超过 24 小时未被接受
   - **已作废**（灰色）：被取消或被其他邀请覆盖
3. **已过期 / 已作废**的邀请可点击「重新发送」一键生成新邀请凭证

### 7. 管理后台查询守护关系（仅管理员）

1. 管理员登录 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin)
2. 进入 [守护关系查询](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/guardian-relations)
3. 可按以下条件筛选：
   - 关键词：手机号 / 昵称（模糊匹配）
   - 仅主守护人 / 仅普通守护人
4. 表格列：守护人、角色徽章、会员等级、被守护人、优先级、绑定时间

## 注意事项

1. **主守护人唯一性**
   - 每个被守护人**只能**有 1 个主守护人
   - 第 1 个绑定的守护人**自动**成为主守护人
   - 历史数据已通过迁移脚本自动补齐主守护人字段

2. **会员等级与角色相互独立**
   - 是否付费**不影响**主/普通守护人的角色身份
   - 付费只影响"AI 电话外呼额度"和"被守护人可绑定的守护人数量"

3. **异常告警完全免费**
   - 由平台兜底，不扣任何人的 AI 外呼额度
   - 但免费会员仅有 5 次/月免费电话额度，用完后降级为 App 推送 + 短信

4. **App 推送 + 短信对所有守护人始终免费、始终发送**

5. **邀请凭证 24 小时过期**
   - 被点击预览（未点确认）不消耗，可重复打开
   - 任意 1 人点"确认接受"后链接立即失效

6. **双向邀请冲突时遵循"先确认那条生效"**
   - 另一条自动作废
   - 双方都会收到通知"已通过另一条邀请建立关系"

7. **未列入本期范围的开放点**
   - 被守护人注销账号 → 守护关系处理
   - 主守护人手机号变更 → 是否需重新授权
   - 未成年人能否成为守护人 / 被守护人
   - 守护关系数据导出与审计

## 关键 API 接口（开发者参考）

| 接口 | 说明 |
|------|------|
| `GET /api/guardian/list` | 我的守护人列表（含角色徽章 + 上限） |
| `GET /api/guardian/i-guard` | 我守护的人列表 |
| `POST /api/guardian/transfer/initiate` | 发起主守护人转移 |
| `POST /api/guardian/transfer/{id}/approve` | 被守护人同意转移 |
| `POST /api/guardian/transfer/{id}/cancel` | 取消转移请求 |
| `GET /api/guardian/transfer/pending` | 查询待确认转移 |
| `POST /api/guardian/priority` | 调整守护人外呼优先级 |
| `GET /api/guardian/alert-quota` | 查询本月告警额度 |
| `POST /api/guardian/alert/simulate-serial-call` | 模拟串行告警外呼（含降级策略） |
| `GET /api/guardian/invitations/records` | 邀请记录列表（含状态徽章） |
| `GET /api/admin/guardian/relations` | 后台守护关系查询 |
