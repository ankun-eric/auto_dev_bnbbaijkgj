## 1. 功能简介

本次优化在「管理后台 → 商家管理 → 门店管理」列表页对表格列冻结策略做了细微但关键的调整：

- **门店名称**列：左固定（横向滚动时贴在最左侧，始终可见）
- **状态**列：右固定（横向滚动时贴在最右侧，始终可见）
- **操作**列：右固定（横向滚动时贴在最右侧，始终可见）

带来的核心体验提升：

1. **小屏（如 1366×768 笔记本）**下，无需再左右拖动滚动条就能直接看到「门店名称」并点击右侧「编辑 / 营业管理 / 停用启用」三个高频操作。
2. **行级身份始终在场**：滚动表格中部信息时，左侧的门店名称依然可见，再也不会出现"不知道自己在操作哪一家门店"的情况。
3. **冻结/滚动区分明**：AntD Table 自带的阴影分隔会清晰提示哪部分是冻结区、哪部分是可滚动区。
4. **大屏无副作用**：屏幕足够宽（≥1920px）时表格不出现横向滚动条，固定列阴影自动隐藏，整体视觉与改造前完全一致。

> 本次为**纯前端表格列配置调整**，不改任何字段、不改任何接口、不改任何数据。

## 2. 使用说明

### 2.1 进入门店列表

1. 在浏览器中打开 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/) 并登录
2. 左侧菜单点击「商家管理 → 门店管理」，或直接访问 [门店管理列表](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/merchant/stores/)

### 2.2 查看固定列效果

1. 在 1366×768 等中小屏窗口下打开门店列表页
2. 按住表格底部的水平滚动条向右拖动
3. 观察体验：
   - **门店名称**始终贴在表格最左侧不动，并伴随一道淡淡的右侧阴影分隔线
   - **状态**和**操作**始终贴在表格最右侧不动，并伴随一道淡淡的左侧阴影分隔线
   - 中间的「门店编码 / 所属类别 / 联系人 / 联系电话 / 地址 / 营业时间」会在固定区之间正常滑动

### 2.3 边缘阴影自动隐藏

- 滚动条拖到**最左端**时：左固定列的右投阴影**自动消失**（避免无意义的阴影）
- 滚动条拖到**最右端**时：右固定区的左投阴影**自动消失**

### 2.4 大屏体验

在 1920×1080 等大屏下，如果整张表格的总宽度 ≤ 浏览器可视宽度，则不会出现横向滚动条，两侧分隔阴影自动不显示，视觉与未启用固定列时完全一致。

### 2.5 现有功能完全保留

以下所有功能保持原有行为，未受任何影响：

- ✅ 按类别筛选
- ✅ 「显示已停用」开关
- ✅ 「新建门店」
- ✅ 行级「编辑」「营业管理」「停用 / 启用」
- ✅ 已停用门店的灰化样式（`store-row-inactive`）
- ✅ 表格 loading、分页等

## 4. 注意事项

1. **无需重新登录**：本次仅前端 UI 调整，登录 token、权限、Cookie 完全沿用。
2. **强制刷新建议**：浏览器若有强缓存，请在门店列表页 `Ctrl + Shift + R`（Windows / Linux）或 `Cmd + Shift + R`（macOS）做一次强制刷新，确保加载到新版前端代码。
3. **回滚成本极低**：本次改动仅 1 个文件（`admin-web/src/app/(admin)/merchant/stores/page.tsx`）的 4 行变更，去掉新增的 `fixed: 'left'` 与 `fixed: 'right'` 三处属性即可恢复原状，零风险。
4. **后续扩展（不在本次范围）**：「商家分类」「商家账号」「营业配置」等其他列表页若也希望应用同样的冻结策略，可单独立项统一处理。

## 5. 技术细节（开发同学参考）

- 改动文件：`admin-web/src/app/(admin)/merchant/stores/page.tsx`
- 改动方式：在 AntD `<Table>` 组件的 columns 配置上：
  - 「门店名称」列追加 `fixed: 'left' as const`
  - 「状态」列追加 `fixed: 'right' as const`
  - 「操作」列追加 `fixed: 'right' as const`
- 保留原有 `scroll: { x: 1400 }` 兜底
- 阴影、边缘自动隐藏均由 AntD Table 内置实现，无需自行编写 CSS
- 不新增依赖、不新增组件、不改后端
