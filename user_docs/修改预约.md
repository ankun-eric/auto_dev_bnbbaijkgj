## 使用说明

### 场景 1：你的订单已经预约过，但想改一个新的时间

1. 在用户端【订单列表】或【订单详情】里，找到一笔状态为「**待核销 / 已预约 / 部分核销**」的订单。
2. 进入订单详情，会在底部看到【**修改预约**】按钮（注意：以前是"修改预约时间"，现在统一为"修改预约"，4 个字）。
3. 点击【修改预约】，会弹出"选择预约时间"窗口：
   - 上方是【预约日期】，点击后选择您方便的日期（默认明天，最远 90 天）。
   - 下方是【预约时段】，每行 3 个时段方块（这次专门做了固定 3 列的优化），点击选中即可。
4. 点击【确认预约】，提示"预约成功"后窗口自动关闭，订单详情会刷新展示新预约时间。

### 场景 2：你购买的商品仅按日期预约（不需要时段）

- 弹窗只会显示【预约日期】，**整块时段选择 UI 已隐藏**。
- 选完日期后直接【确认预约】即可，不会再要求你选时段。

### 场景 3：你购买的商品使用"自定义预约表单"

- 点击【修改预约】后，**不会**弹出普通弹窗，而是直接跳转到自定义表单页面。
- 你可以重新填写表单内容并提交，表单字段以商家配置为准。

### 场景 4：从短信/分享链接进入订单（仅 H5）

- 链接形式如 `https://...../unified-order/123?action=appointment`
- 进入页面后，"修改预约"弹窗会**自动**打开，无需再次点击按钮。

### 体验步骤

1. **下载压缩包**：点击上方链接，将 zip 压缩包下载到本地电脑。

3. **下载微信开发者工具**：如尚未安装，请前往 [微信开发者工具官方下载页面](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html) 下载并安装。

   - 在「目录」栏点击浏览，选择第 2 步解压后的文件夹（即包含 `app.json` 的那一层目录）。
   - 「AppID」栏可填入项目的 AppID，或选择「测试号」进行体验。
   - 点击「导入」按钮。
6. **预览体验**：导入成功后，开发者工具会自动编译并在模拟器中展示小程序界面。
7. **重点体验路径**：

## 注意事项

- **预约模式说明**：商家可在商品后台为商品配置 4 种预约模式，本次修复让前端三端都能根据后端透传的 `appointment_mode` 字段联动展示：
  - `none`（无需预约）：不显示"修改预约"按钮。
  - `date`（按日期）：弹窗仅显示日期选择，时段块整块隐藏。
  - `time_slot`（按时段）：弹窗显示日期 + 时段（每行 3 个）。
  - `custom_form`（自定义表单）：跳转到自定义表单页重填。
- **状态范围**：以下订单状态可使用"修改预约"功能：
  - 待核销（pending_use / appointed）
  - 部分核销（partial_used）
  - 待预约（pending_appointment，对应"立即预约"入口）
- **退款处理中**：如果订单已申请退款且未驳回，"修改预约"按钮会自动隐藏。
- **后端兼容**：date 模式下提交时不再携带 `appointment_time` 字段以外的时段参数，后端已兼容。
- **小程序提交注意**：本次小程序代码改动较多，建议管理员重新走一次微信审核流程后再发布生产版本（测试号体验直接导入即可）。

## 故障自查清单（如果遇到问题）

| 现象 | 可能原因 | 解决办法 |
|------|----------|----------|
| 按钮点击仍然无反应 | 客户端没更新到最新版本 | 重新下载本页提供的 APK / IPA / 小程序 zip |
| 弹窗内还是看到 6 个/8 个时段一行 | 缓存了旧版本 | H5 强制刷新（Ctrl+F5），APP 卸载重装 |
| 按日期商品的弹窗仍显示时段块 | 后端 `appointment_mode` 字段没拿到 | 用最新 APP / H5 + 后端最新版本（已部署） |
| 自定义表单商品点按钮没跳转 | 前端版本与后端不匹配 | 检查 APP / 小程序版本是否为本次发布版 |
| 弹窗打开了但确认预约报错 | 选的日期超出 90 天范围或商品库存不足 | 选 90 天内的日期，或联系商家确认库存 |
