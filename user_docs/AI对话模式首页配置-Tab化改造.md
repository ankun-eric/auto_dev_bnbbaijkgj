## 功能简介

「AI 对话模式首页配置」原本是一个超长单页，10 张配置卡片全部摊开在同一页面，运营每次想改一个小字段都要在长页面里反复滚动定位，体验差且容易误操作。

**本次改造把"伪 Tab"升级为"真 Tab 切换"**——每次只显示当前 Tab 对应的卡片，配合「独立保存 + 草稿提示 + 切换确认 + 重置」的编辑闭环，避免误操作丢失改动。

### 一句话改动

> 6 个真 Tab + 卡片重新归位 + 红点未保存提示 + 切 Tab 三选确认 + 吸底独立保存/重置 + 重置二次确认。

### 6 个 Tab 与卡片归位

| Tab 序号 | Tab 名称 | 包含卡片 |
|---|---|---|
| 1 | 欢迎区 | 欢迎区配置 |
| 2 | 首屏内容 | 健康贴士轮播 + 空对话占位、推荐问列表 |
| 3 | 功能宫格 | 功能宫格（7 字段） |
| 4 | 输入栏 | 输入栏配置 |
| 5 | 会话策略 | 会话策略（7 字段）、空闲超时与空会话引导 |
| 6 | 全局开关 | 全局开关（9 总开关）、浮动健康打卡按钮、**顶栏与品牌（并入末位）** |

> 「顶栏与品牌」作为兼容旧 H5 的配置，被并入 Tab 6「全局开关」末位，不再单独占位。

## 操作步骤

### 1. 进入页面

1. 用浏览器打开 [Admin 后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/) 并登录
2. 进入「首页设置 → AI 对话首页配置」（或直接打开 [AI 对话首页配置页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-settings/ai-home-config/)）
3. 页面默认停留在 Tab 1「欢迎区」

### 2. 修改某个模块的字段

以「修改欢迎区主标题」为例：

1. 当前默认就在「欢迎区」Tab，无需切换
2. 在「主标题」输入框中修改文案（如 `早上好，{昵称}！`）
3. 一旦内容与上次保存值不同，**该 Tab 标题右侧会出现一个红点 ●**，提示有未保存改动
4. 同时**底部吸底操作栏**的 `重置本 Tab` 与 `保存本 Tab` 两个按钮从置灰变成可点击
5. 点击 `保存本 Tab` 按钮 → 顶部弹出绿色 Toast「保存成功」（2 秒自动消失），红点立即消失

### 3. 在多个 Tab 之间切换

#### 场景 A：当前 Tab 没有改动

直接点击其他 Tab 名称，立即切换到目标 Tab，无任何提示。

#### 场景 B：当前 Tab 有未保存改动

点击其他 Tab 名称时，**会弹出"未保存的修改"确认框**，提供 3 个按钮：

| 按钮 | 行为 |
|---|---|
| **保存并切换**（实色主按钮） | 先保存当前 Tab 的所有改动，保存成功后自动切换到目标 Tab；保存失败则停留在当前 Tab 并显示错误 |
| **放弃修改并切换**（红色描边） | 直接丢弃当前 Tab 的所有改动，恢复成上次保存值，切换到目标 Tab |
| **取消**（文本按钮） | 关闭弹窗，停留在当前 Tab，所有改动保留 |

> 多个 Tab 同时有草稿时，每个 Tab 各自的红点 ● 同时显示，互不影响。

### 4. 重置当前 Tab 的改动

1. 在某个 Tab 内做了若干修改后
2. 点击底部吸底操作栏的 `重置本 Tab` 按钮
3. 弹出"确认重置"弹框 → 点击 `确认重置`（红色主按钮）
4. 该 Tab 的所有字段恢复成"上次成功保存"的值；红点消失；按钮重新置灰

> 如果当前 Tab 没有任何改动，`重置本 Tab` 与 `保存本 Tab` 按钮均为置灰状态，不可点击。

### 5. 字段校验失败的处理

点击 `保存本 Tab` 时，系统会做完整的字段校验：

- **校验通过** → 走正常保存流程
- **校验不通过** → 顶部 Toast 显示首个错误的具体原因（如「字段「welcome.main_title」校验未通过：主标题为必填且不超过30字」）；该字段输入框边框变红、下方显示错误文案；页面自动滚动到该字段附近

主要字段校验规则：

| 字段 | 限制 |
|---|---|
| 欢迎区 - 主标题 | 必填，≤30 字 |
| 欢迎区 - 副标题 | ≤50 字 |
| 健康贴士 - 轮播间隔 | 3~5 秒 |
| 空对话占位 - 主标题 | ≤20 字 |
| 推荐问 - 标题 | ≤8 字 |
| 推荐问 - 实际发送内容 | ≤200 字 |
| 功能宫格 - 数量 | 1~6 项 |
| 功能宫格 - 主文案 | 必填，≤8 字 |
| 功能宫格 - 副说明 | ≤12 字 |
| 功能宫格 - 渐变色 | 须为合法 HEX（如 `#5B6CFF`） |
| 输入栏 - 家庭咨询模板 | 启用时必须包含 `{name}` 占位符 |
| 会话策略 - 单次回答字数 | 100~5000 |
| 会话策略 - 单日免费次数 | 1~999 |
| 会话策略 - 免责声明 | ≤100 字 |

### 6. 查看保存历史

点击页面右上角的 `操作日志` 按钮，进入 [配置操作日志页面](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/home-settings/ai-home-config/logs/)，可看到每次保存的字段差异、操作人、IP 等明细。

## 注意事项

1. **刷新即清空**：刷新页面会丢弃所有 Tab 的草稿，并跳回 Tab 1「欢迎区」。请记得保存后再刷新。
2. **不记忆 Tab**：进入页面始终默认停留在「欢迎区」，不会自动切到上次离开的 Tab；URL 也不带 `?tab=xxx` 锚点。
3. **保存只影响当前 Tab**：每个 Tab 的保存动作彼此独立，A Tab 的保存不会影响 B Tab 的草稿。
4. **重置只影响当前 Tab**：`重置本 Tab` 仅恢复当前 Tab 字段，其他 Tab 的草稿保留。
5. **保存失败保留草稿**：网络错误、字段校验失败时，草稿不会被清空，您可继续修改后再次保存。
6. **顶栏与品牌兼容用**：v1.0 设计图无顶栏，「顶栏与品牌」卡片仅作为兼容旧 H5 的配置保留在 Tab 6 末位，正常情况下无需调整。
7. **权限差异**：超级管理员可编辑全部 6 个 Tab；运营管理员对 Tab 6 通常仅有查看权限；只读用户全部 Tab 的保存/重置按钮均置灰。
8. **配合 H5 验证**：每次保存后，可打开 [H5 AI 对话首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/h5/) 实际查看配置效果。

## 异常处理

| 场景 | 期望表现 |
|---|---|
| 保存接口请求中再次点击保存 | 按钮显示 loading，期间不可重复点击 |
| 网络断开 | Toast 提示具体错误原因，草稿保留 |
| 字段校验失败但用户强行切 Tab | 选择"保存并切换"时会因校验失败而保存失败，停留在当前 Tab，红字段提示 |
| 同字段被另一管理员同时修改 | 后端冲突检测会返回错误，前端 Toast 提示，可刷新后重试 |
