## 功能简介

本次升级把「AI 咨询配置 → 功能按钮管理」做了一次**架构级重构**，彻底解决"每加一个问卷类业务就要前后端 5 端联动改代码"的历史包袱。

### 核心改动一句话

> AI 功能子类型由原来的 7 个"按业务划分"枚举，升级为 **5 个永久稳定的"按交互形态划分"** 子类型；运营在管理后台**通过模板配置**即可上线新的问卷类业务，**无需开发**。

### 新的 5 项 AI 功能子类型

| 子类型 | 含义 | 关键配置字段 |
|---|---|---|
| 📝 **对话内问卷** `questionnaire` | 健康自查 / 体质测评 / 睡眠测评等 | 关联问卷模板 |
| 📷 **图像采集** `image_capture` | 识药 / 上传 / 报告解读 | 采集用途（identify_medicine / upload / interpret_report） |
| 📄 **文件上传** `file_upload` | 通用文件上传 | 文件类型限制 |
| 💬 **AI 对话触发** `ai_dialog_trigger` | 直发指定 Prompt 给 AI | 预设话术 |
| ⚡ **快捷提问** `quick_ask` | 直发问题给 AI | 快捷问题文案 |

### 新增"对话内说明卡片"开关

- 字段：`pre_card_enabled`（默认 **开**）
- 开启后：点击按钮 → 对话区先插一张说明卡片（标题/封面/描述）→ 用户主动点卡片按钮才真正开启功能
- 关闭后：点击按钮直接进入功能流程

## 操作步骤

### 1. 创建一个新问卷模板（运营无需开发参与）

1. 进入 [通用问卷模板管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/questionnaire-templates)
2. 点击右上角 **「+ 新建问卷模板」**
3. 填写：
   - **编码 code**：业务英文标识，如 `sleep_test_v1`（一旦保存不可修改）
   - **名称**：如「睡眠质量测评」
   - **模板说明** / **开篇引导文**：用户答题前看到的引导文案
   - **预计分钟数**、**是否允许返回上一题**、**是否打乱题目顺序**
   - **报告布局**：standard（标准）/ radar（雷达图）/ score_bar（分数条）
   - **AI 解读 Prompt 模板**：含 `{占位符}` 的 Prompt，答完后 AI 用它分析
   - **状态**：启用 / 停用
4. 保存后回到列表，点击该行的 **「题目/分型」** 进入题目编辑抽屉

### 2. 给模板加题目

1. 在题目抽屉中切换到「**题目**」Tab，点击 **「+ 新增题目」**
2. 系统默认创建一道单选题（选项 A/B 各预置一个）
3. 点击该题右侧 **「编辑」**，在弹出的 JSON 编辑器中修改：
   - `question_type`：`single_choice` / `multi_choice` / `text`
   - `title`：题干文案
   - `subtitle`：副说明
   - `options`：`[{label, value, score, tags}]` 格式数组（计分用）
   - `dimension`：所属维度（计算维度分数用，可空）
   - `required`：是否必填
4. 保存即生效

### 3. 配置分型规则（用于判定用户结果归属）

1. 切换到「**分型规则**」Tab，点击 **「+ 新增分型」**
2. 系统默认创建一个分数区间为 `[0, 999]` 的分型，点 **「编辑」**：
   - `code`：分型编码，如 `yin_xu` / `headache`
   - `name`：分型名称，如「阴虚体质」
   - `rule_type`：
     - `score_range`：分数区间，`rule_config = {"min": x, "max": y}`
     - `dimension_max`：选择最高分维度，`rule_config = {"dimension": "xxx"}`
     - `tag_match`：标签命中（最小版直接落第一个分型）
   - `rule_config`：JSON 格式规则配置
3. 保存即生效

### 4. 在功能按钮里关联问卷模板

1. 进入 [功能按钮管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/function-buttons)
2. 新建或编辑一个按钮
3. **按钮类型**选「🤖 AI 功能」
4. **AI 功能子类型**只剩 5 项可选：
   - 选「📝 对话内问卷」→ 下方出现「关联问卷模板」下拉，选刚才建的模板即可
   - 选「📷 图像采集」→ 下方出现「采集用途」单选（识药/上传/报告解读）
5. 「对话内说明卡片」开关：建议保留默认「开」，UX 更友好
6. 保存即可在 AI 对话首页生效

### 5. 用户答题与查看报告（用户端体验流程）

1. 用户在 AI 对话首页点击问卷类按钮（如「健康自查」）
2. （若 `pre_card_enabled=true`）对话区先弹一张说明卡片，用户主动点「开始」
3. 抽屉弹起来展示问卷题目，用户逐题作答
4. 提交后系统自动按题目 `score` 计分、按 `dimension` 累加维度分、按规则判定分型
5. 报告页（API `/api/questionnaire/answers/{id}/report`）返回：
   - 头部用户信息
   - AI 解读（流式）
   - 分型详情（描述 + 维度可视化）
   - 推荐商品 / 推荐门店 / 推荐服务（按分型匹配）

## 兼容性与数据迁移

### 数据迁移行为（自动执行，幂等）

后端启动时会自动执行 `prd_questionnaire_v1_migration.py`，做如下事情：

1. `chat_function_buttons` 表加 3 列：`questionnaire_template_id` / `capture_purpose` / `pre_card_enabled`
2. 旧字段 `pre_card_for_navigate` 的值自动回填到 `pre_card_enabled`
3. 老的子类型自动映射到新枚举：
   - `health_self_check` → `questionnaire`（自动绑定到内置 `health_self_check` 模板）
   - `report_interpret` → `image_capture` + `capture_purpose=interpret_report`
   - `photo_upload` → `image_capture` + `capture_purpose=upload`
   - `medicine_recognize` → `image_capture` + `capture_purpose=identify_medicine`
4. 预置两个内置模板：`health_self_check`（健康自查）+ `tcm_constitution`（中医九型体质测评），方便运营即开即用

### API 向后兼容

- 老的 `health_self_check` / `photo_upload` 等枚举值在 backend 校验集中**仍被接受**（兼容期保留）
- 已有客户端（H5/小程序/Flutter）无需任何修改即可继续运行
- 新接口 `/api/questionnaire/*` 完全独立，不影响旧的 `/api/health-self-check`、`/api/constitution` 等老路径

## 注意事项

- **问卷模板 code 一经创建不可修改**：因为 H5/小程序/Flutter 端可能按 code 索引模板。如确需调整，请新建模板后再切换按钮绑定
- **题目顺序由 `sort_order` 升序决定**：建议使用 10、20、30 步长方便插入
- **`rule_type=score_range` 分型规则需覆盖全分数段**：避免出现"某些用户答题完无法归类"的情况，建议至少配置一个兜底分型（如 `min:0, max:999`）
- **本次仅完成「后端 + admin-web」改造**：H5 端的通用 `QuestionnaireDrawer` / 报告页 `/questionnaire/result/[id]` 在下一版本提供；当前 H5 仍走旧的健康自查 / 体质测评抽屉，不受影响
- **小程序与 Flutter APP 本次未改动**：API 完全向后兼容，旧客户端继续可用
