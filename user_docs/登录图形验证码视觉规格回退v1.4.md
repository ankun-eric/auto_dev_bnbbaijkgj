> 版本：v1.4 · 上线日期：2026-04-26

## 一、功能介绍

为了让登录页验证码恢复到大家长期熟悉的「小而瘦、清晰、笔画利落」的样子，我们把图形验证码的**视觉规格完全回退到 v1.0 老版本**，并彻底清理了 v1.1 / v1.2 / v1.3 期间引入的「画布放大、2× 高 DPI、字号放大」等改造。

| 维度 | v1.3（旧） | **v1.4（本次）** |
|---|---|---|
| CSS 显示尺寸 | 160 × 60 | **160 × 60** |
| PNG 物理像素 | 320 × 120（2×） | **160 × 60（1:1，取消 2× DPI）** |
| 字号 | 96px | **38px** |
| 字符撑满画布 | ~80% | **~63%（与 v1.0 一致）** |
| 单张 PNG 体积 | 6 ~ 10 KB | **≤ 3 KB（实测约 1.5 KB）** |

> 业务校验逻辑（4 位字符、不区分大小写、5 分钟有效、一次性、接口字段）**完全保持不变**——前端无需修改任何调用代码，老用户也无任何适应成本。

## 三、操作步骤

### 1. 后台 admin 登录

1. 浏览器打开「后台 admin 登录页」。
2. 在右侧的验证码输入框旁，会看到一张 **160 × 60** 的小巧验证码图片。
3. 输入账号、密码，并照着图片输入 4 位验证码。
4. **如果看不清，直接点击图片**即可换一张新的。
5. 点击「登录」即可进入后台。

### 2. 商家 PC / H5 登录

1. 浏览器打开「商家端 H5 / PC 登录页」。
2. PC 端：验证码图片同样显示为 **160 × 60**，与 admin 端规格一致。
3. 移动端：验证码图片会按比例自适应（宽度区间 120 ~ 220），保持长宽比、不变形。
4. 输入手机号 / 账号、密码、验证码，点击「登录」。

### 3. 刷新验证码

- 任何页面，**点击验证码图片本身**即可刷新一张新验证码。
- 输错验证码后会自动刷新一张新图，请按新图重新输入。

## 四、视觉变化要点（管理员/产品请关注）

1. **物理像素 = 显示像素**：浏览器 DevTools 查看 `<img>` 元素时，渲染尺寸与 PNG 自身分辨率均为 160 × 60。
2. **字符更瘦更清晰**：字号回到 38px，字符高度约占画布 60% ~ 65%，与 v1.0 完全一致。
3. **载入更快**：单张 PNG 实测约 **1.5 KB**（最大不超过 3 KB），登录页验证码刷新比 v1.3 快 3~5 倍。
4. **高分屏下的「轻微锯齿」属正常**：这是为了与 v1.0 视觉完全一致而做的取舍，**不视为 bug**。

## 五、注意事项

| 场景 | 说明 |

| 高分屏（2K / 4K / Retina） | 字符可能略显「轻微锯齿」——这是与 v1.0 一致的预期表现，不影响识别。 |
| 接口字段未变 | API 路径、请求/响应字段与 v1.3 完全一致，第三方系统集成无需调整。 |

## 六、常见问题（FAQ）

**Q1：为什么觉得验证码「变小了」？**
这是有意的回退。v1.3 把字号放到 96px、PNG 渲染到 320×120 后再缩放显示，看上去字符粗壮但「画框」没变大，反而引起「没更新」的错觉。v1.4 完全回到 v1.0 的小而清晰风格，字符笔画细腻易辨认，与历史习惯一致。

**Q2：还能识别清楚吗？**
能。字符高度约占画布 60%~65%，4 位非易混字符（已剔除 0/O、1/I/l 等），加上少量旋转和干扰曲线，识别率与 v1.0 持平。

**Q3：手机上验证码是不是变小了？**
没有。H5 移动端会在 120 ~ 220 宽度区间按比例自适应，不会被压扁或变形，看起来与桌面端规格一致。

**Q4：会不会影响已经做好的接口对接？**
不会。本次只动**视觉规格**，接口 URL、请求/响应字段、缓存策略、限流策略**全部保持原样**。
