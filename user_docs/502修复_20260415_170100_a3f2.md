## 功能简介

本次修复解决了项目所有页面（H5 用户端、管理后台）出现 **502 Bad Gateway** 错误的问题。

### 问题原因

项目的 4 个 Docker 容器（数据库、后端、H5 前端、管理后台）运行在独立的 Docker 网络 `6b099ed3-7175-4a78-91f4-44570c84ed27-network` 中。Gateway Nginx 容器作为反向代理需要加入该网络才能将外部请求转发到项目容器。

当项目容器被重新创建时（如执行 `docker compose up --build`），Docker 网络会被重建，导致 Gateway 容器与项目网络的连接断开。Gateway 的 Nginx 使用 Docker 内置 DNS（`resolver 127.0.0.11`）解析容器名称来转发请求，一旦不在同一网络中，DNS 解析失败，即返回 502 Bad Gateway。

### 修复方式

执行 `docker network connect 6b099ed3-7175-4a78-91f4-44570c84ed27-network gateway` 命令，将 Gateway 容器重新加入项目的 Docker 网络，恢复 DNS 解析和请求转发能力。

3. **使用安卓 APP**：APP 内的接口请求同样已恢复正常，无需重新安装

## 注意事项

- 如果后续项目容器被重新创建（如重新部署），需要再次将 Gateway 加入项目网络
- 如果再次出现 502 错误，可在服务器上执行：`docker network connect 6b099ed3-7175-4a78-91f4-44570c84ed27-network gateway`
- 本次修复不涉及任何代码改动，仅为服务器 Docker 网络配置恢复
