## 功能简介

本次优化把「我的订单」与「退款/售后」的入口、文案、状态体系做了一轮深度统一：

1. **「我的」入口图标更直观**：第 4 个图标「待评价」改为「已完成」，进入后看到所有已完成的订单，未评价的订单可以直接发起评价
2. **三端文案完全一致**：H5 全部订单、H5 退款独立列表、后台订单管理这三处的退款相关文案与筛选逻辑统一为「待审核 / 处理中 / 已完成 / 已驳回」4 个文案；告别了过去"同一件事三种叫法"的混乱
3. **15 天评价时效**：新增评价时效控制——订单完成后 15 天内可发起评价，超期按钮自动置灰为「评价已过期」；已评价的订单永久可"查看评价"
4. **售后撤销能力**：新增「撤销申请」能力，当售后处于"待审核"时用户可一键撤销，订单恢复到原状态
5. **退款/售后入口仍保留独立列表页**，但与全部订单 Tab 数据范围、筛选、卡片样式完全一致，体验"无缝切换"

| 3 | 🎫 | 待使用 | 全部订单 → 待使用 Tab |
| 4 | ✅ | **已完成**（原"待评价"） | 全部订单 → **已完成** Tab |
| 5 | ↩️ | 退款/售后 | 退款/售后**独立列表页** |

> 第 4 个图标的红点（角标）= 您所有已完成订单中**未评价**的数量。已全部评价或没有完成订单时不显示红点。

### 2. 全部订单页（一级 Tab × 二级 Tab）

打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/unified-orders](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/unified-orders)：

#### 一级 Tab（共 6 个）
- 全部 / 待付款 / 待收货 / 待使用 / **已完成** / **退货/售后**

#### 「已完成」Tab 卡片右下角按钮逻辑

| 评价状态 | 距完成时间 | 按钮 | 说明 |
|----------|------------|------|------|
| 未评价 | ≤ 15 天 | 「去评价」 | 高亮可点击，跳转评价页 |
| 未评价 | > 15 天 | 「评价已过期」 | 灰色不可点击 |
| 已评价 | 任意 | 「查看评价」 | 跳转评价详情 |

#### 「退货/售后」一级 Tab → 二级筛选（共 5 个）

打开「退货/售后」一级 Tab 后，下方会显示二级筛选条：

> 全部 / **待审核** / **处理中** / **已完成** / **已驳回**

每个筛选含义：

| 筛选 | 含义 |
|------|------|
| 待审核 | 您已申请、客服未处理 |
| 处理中 | 客服已通过、退款打款中、退货寄回中 |
| 已完成 | 退款已成功打款 |
| 已驳回 | 客服驳回了您的申请 |

- 即使售后被驳回，订单仍保留在该列表中（保留完整历史）
- 同一订单多次发起售后时，列表按**最新一次**售后状态归档显示

### 4. 撤销售后申请（新功能）

当您的售后申请处于「**待审核**」状态时，订单卡片底部会出现 **橙色「撤销申请」按钮**：

1. 点击「撤销申请」
2. 弹出确认浮层："确认撤销本次售后申请？撤销后订单将恢复原状态"
3. 点击「确认撤销」
4. 撤销成功后会提示「已撤销」，并自动刷新订单列表
5. 订单从售后列表移除，回到原状态（如待使用、待收货等），您可继续正常使用或再次申请售后

> ⚠️ **撤销限制**：仅在「待审核」时可撤销。一旦客服已通过/已驳回/已退款，您的「撤销申请」按钮将自动消失。

---

### 5. 评价时效提示

- 订单完成时间起 **15 天内**可发起评价（按钮显示「去评价」）
- 超过 15 天后按钮自动置灰为「评价已过期」，不可点击
- 后端会同步校验时效——即使绕过前端，超期评价请求也会被拒绝
- 已成功评价的订单，按钮变为「查看评价」，**永久可点击查看**

---

### 6. 后台客服/运营操作（管理员视角）

打开 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/orders](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/orders)：

- 顶部退款流程筛选下拉框含 **5 个选项**：全部售后 / 待审核 / 处理中 / 已完成 / 已驳回（与客户端完全一致）
- 订单列表「退款状态」列文案统一为：待审核 / 处理中 / 已完成 / 已驳回
- "退款申请" Tab 默认筛选所有"待审核"订单，方便客服批量处理
- 订单详情中仍保留 `refund_status` 完整审计字段，含审核人、审核时间、退款流水号等明细

## 注意事项

1. **三端文案统一**：客户端 H5、退款独立列表、后台管理三处的退款筛选下拉文案与逻辑 **100% 一致**，告别"同一件事三种叫法"
2. **「已完成」Tab 不含已退款订单**：退款完成的订单归档在"退货/售后 → 已完成"，不会重复出现在"已完成"Tab
3. **15 天评价时效后端二次校验**：防绕过前端发起评价
4. **撤销不影响审计**：撤销售后申请后，后台仍保留"曾发起过售后并撤销"的审计记录（refund_status / 操作日志）
5. **小程序/App 暂未更新**：本次仅 H5 + 后台两端做了改造，小程序与原生 App 仍走旧文案逻辑（向下兼容，不影响使用）
6. **底层数据兼容**：后端保留 `refund_status` 7 个值作为审计字段，新增 `aftersales_status` 4 值逻辑参数作为统一筛选维度，旧客户端继续传旧参数仍可正常工作
