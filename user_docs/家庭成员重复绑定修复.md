## 功能简介

在「健康档案 - 家庭成员 / 守护绑定」模块中，管理者可以为家人创建多个家庭成员档案，并分别生成邀请二维码邀请对方扫码绑定。

**本次修复的问题**：

- 修复前：管理者建立了 A、B 两个家庭成员档案并各自发出邀请。同一个客户先扫码绑定了 A，之后又能扫 B 的二维码再次绑定成功，导致同一个人在同一管理者名下被重复绑成了两条记录。
- 修复后：系统在绑定前会校验「这个人是否已经在当前管理者名下」。一旦命中，立即拦截并提示，确保**一个人在同一管理者名下只占一个名额**。

**修复口径**：

- 「同一个人」的判定标准：**用户 ID 相同 或 手机号相同**，任一命中即视为同一人。
- 唯一性范围：**同一管理者名下唯一**。别的管理者发的邀请不受影响，仍可正常绑定（不做全局唯一，不会误伤）。
- 拦截提示文案：**"您已是该家庭的成员，无法重复绑定。"**
- 覆盖范围：所有产生绑定关系的入口（扫码接受邀请、反向守护邀请、虚拟成员迁移确认）均已统一加上判重，从根上堵死重复绑定。

## 使用说明

本功能对终端用户而言是**无感的自动校验**，无需任何额外操作，正常使用即可享受修复效果。

### 管理者侧（发出邀请的人）

1. 进入 H5「健康档案 / 家庭成员」页面。
2. 为家人创建家庭成员档案（如成员 A、成员 B），分别生成邀请二维码。
3. 将二维码发给对应的家人扫码绑定。

### 被邀请者侧（扫码绑定的人）

1. 扫描管理者发来的邀请二维码，进入接受邀请页面。
2. 点击「接受邀请 / 确认绑定」。
3. 绑定结果说明：
   - **首次绑定该管理者**：绑定成功，成为该管理者名下的家庭成员。
   - **已是该管理者名下成员后再扫该管理者的其他二维码**：系统会拦截并提示 **"您已是该家庭的成员，无法重复绑定。"**，不会重复占用名额。
   - **扫描其他管理者发来的二维码**：不受影响，可正常绑定。

### 行为对照表

| 场景 | 修复前 | 修复后 |

| S1 被管理者解绑 A 后，再扫 B 二维码 | ✅ 成功 | ✅ 成功（解绑后释放名额，可重新绑定） |

## 注意事项

- **本次为后端修复，无需重新安装或更新任何客户端**（H5、微信小程序、安卓 App、苹果 App 均调用同一后端接口，后端上线后全端自动生效）。
- 「同一人」既看用户 ID 也看手机号，因此即使同一个人换了账号但手机号一致，也会被识别为重复并拦截。
- 唯一性仅限**同一管理者名下**。同一个人被多个不同管理者各自邀请绑定，仍属正常，不受影响。
- 存量历史脏数据（修复上线前已产生的重复绑定）不会自动清理，如有需要请管理者手动解绑多余记录。
- 解绑后该名额会被释放，被解绑的人可以重新接受该管理者的邀请绑定。
