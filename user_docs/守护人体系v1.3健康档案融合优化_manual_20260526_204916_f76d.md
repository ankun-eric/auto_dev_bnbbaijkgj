## 功能简介

「守护人体系 v1.3」是对 v1.2 的健康档案视图进行的一次重大优化，目的是让老人和家属都能"秒懂"：

### 三大核心改进

1. **极简信息架构**：最外层 Tab 仅保留「守护中 / 待守护」两态，砍掉容易混淆的「我代管」中间态
2. **解耦数据模型**：把"守护关系状态"和"邀请生命周期"拆分为两个独立字段，让卡片操作按钮逻辑更清晰
3. **统一扣费心智**：AI 呼叫额度采用「先本人 → 后主代付」主流模式，默认代付开关 ON

### 同时支持的能力

- 主守护人金色「主」徽章（卡片右上角醒目展示）
- 4 项不可删除校验（守护中 / 已绑设备 / 邀请中 / 在途服药）→ 按钮直接不出现
- 邀请记录单向视图（"我对该被守护人发起过的全部邀请"）
- AI 呼叫代付明细（今日/本月代付次数 + 列表）
- 邀请上限弹窗：温馨提示 + 跳转升级会员

## 使用说明

### 一、进入「我守护的人」v1.3 新版

1. 登录 H5 → 顶部导航 → 「健康档案」
2. 进入 [我守护的人 v1.3 新版](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/health-profile/v13)
3. 或在旧版页面（i-guard）顶部点击橙色横幅「✨ 体验全新「守护中 / 待守护」两态视图」

### 二、查看两个 Tab

- **守护中**：当前所有 `active` 关系，含本人行 + 被守护人卡片
- **待守护**：所有 `not_active` 关系（未邀请 / 邀请中 / 已拒绝 / 已解绑 / 已过期）

每个卡片底部会显示一段子状态文字，例如：
- `共管 · 2026-03-15 建立`（已守护）
- `待守护（邀请中） · 还剩 18 小时`（邀请中）
- `待守护（已拒绝） · 创建于 3 天前`（已拒绝）

### 三、邀请流程

1. 顶部点【+ 发起邀请】或卡片【发起邀请 / 再次邀请】
2. 二维码页上方会显示"您还可邀请 X 位 / 共 N 位"
3. 当人数已满时弹出 **💝 温馨提示** 弹窗，可点击「升级会员」跳转

### 四、查看二维码 / 取消邀请

仅 `邀请中（inviting）` 状态的卡片显示这两个按钮：
- 【查看二维码】：再次显示原二维码（15 分钟内有效）
- 【取消邀请】：作废邀请，后续可【再次邀请】

### 五、再次邀请

适用于 `已拒绝 / 已解绑 / 已过期` 三种状态。点击后会生成 **全新的二维码 + 新邀请记录**（历史邀请记录全部保留留痕）。

### 六、移除（4 不可删校验）

只有满足以下全部条件时，卡片才会显示【移除】按钮（不显示比灰显更干净）：

| 序号 | 不可删除情形 |
|---|---|
| 1 | 守护关系=active（守护中）|
| 2 | 已绑定硬件设备 |
| 3 | 邀请生命周期=inviting（邀请中）|
| 4 | 存在在途服药计划 |

点击【移除】会弹出二次确认：

> 移除后，该档案将从您的列表中消失。若 TA 是您自行添加的家人档案，所有资料将一并删除，此操作不可恢复。

### 七、解除守护

仅守护中 Tab 的卡片有【解除守护】按钮。点击后弹出确认：

> 将解除与 TA 的共管关系，TA 的档案数据完整保留

解除后卡片会从「守护中」移到「待守护」Tab。

### 八、代付明细 + 代付开关（仅主守护人可见）

主守护人在被守护人卡片上可见【代付明细】按钮。点击后弹窗显示：

- **AI 呼叫代付开关**（默认 ON）：可随时切换
- **今日 / 本月代付次数**
- **代付记录列表**（呼叫类型、时间）

权限规则：
- 普通守护人调用 `POST /api/guardian/v13/family/proxy-pay/toggle` 会返回 **403 仅主守护人可执行此操作**
- 被守护人本人调用则视为本人写自身数据，允许

### 九、邀请记录（被守护人行）

每张「待守护」卡片右下角的【邀请记录】按钮，会展示「我对该被守护人发起过的全部邀请」（单向、时间倒序）：

```
2026-05-26 14:20  发起邀请 → 已拒绝
2026-05-20 09:11  发起邀请 → 已过期
2026-05-15 18:05  发起邀请 → 已取消
```

## 扣费规则（v1.3 统一逻辑）

| 呼叫类型 | 被呼叫人 | 扣费顺序 |
|---|---|---|
| AI 外呼提醒 | 被守护人 | **先扣本人 → 不够扣主**（代付 ON 时） |
| 紧急 SOS | 被守护人 | **先扣本人 → 不够扣主**（代付 ON 时） |
| AI 外呼 / SOS | 普通守护人自己 | 仅扣自己（与主无关） |
| AI 呼叫（任意类型）| **孤儿档案被守护人**（无账号） | **直接扣主**（无本人额度可扣）|

### 通知规则

- 切换为主代付时：被守护人收到「您本月 AI 呼叫额度已用完，已由家人为您代付，请放心使用 ❤️」
- 主额度也耗尽：呼叫失败 + 主收到「您为 XX 的代付额度已用尽，建议升级会员」

| 接口 | 方法 | 说明 | 鉴权 |

| `/api/guardian/v13/family/remove` | POST | 移除卡片（含 4 不可删校验）| 登录 |

## 注意事项

1. **v1.3 与 v1.2 并存**：v1.2 的 `/health-profile/i-guard` 页面仍可用，并在顶部添加了跳转入口指向 v1.3
2. **小程序 / Flutter 端**：本次 v1.3 仅完成 后端 + H5 端 的实现，小程序与 APP 端将在后续迭代中跟进
3. **代付开关默认值**：所有主守护人对所有被守护人的代付开关 **默认 ON**（无显式记录视为 ON）
4. **4 不可删校验**：不满足条件时按钮**直接不出现**（不是灰显），界面更干净
5. **邀请 24h 自动过期**：邀请状态有效期 24 小时（二维码本身 15 分钟），过期后可【再次邀请】
6. **历史数据兼容**：v1.2 的 `is_primary_guardian` / `priority_order` 字段完全复用，无新建数据模型
