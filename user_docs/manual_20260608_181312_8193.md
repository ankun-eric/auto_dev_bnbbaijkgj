## 功能简介

本次更新对"本人健康档案"模块进行了底层加固，核心改动如下：

### 1. 杜绝"一人多档案"的脏数据问题

**问题背景**：业务规则要求每个注册用户只能有一条"本人健康档案"。但此前因数据库层面缺乏强约束，以及多个接口在查不到档案时"偷偷新建"，导致个别用户出现了两条本人档案的脏数据。

**修复内容**：
- **数据库层面**：`family_member_id` 字段的 NULL 改为 0 来标识"本人"，并添加 `(user_id, family_member_id)` 联合唯一索引，从物理层面杜绝重复记录
- **代码层面**：4 处"查不到本人档案就偷偷新建"的接口，全部改为"查不到就返回友好错误提示"，不再自动创建
- **查询层面**：全局 `scalar_one_or_none()` 改为 `scalars().first()`，即使数据库已有脏数据也不会导致系统崩溃（500 错误）

### 2. 用户可见的体验变化

当用户尝试操作本人健康档案但档案尚未创建时，系统将返回**引导式文案**而非无提示静默创建：

| 场景 | 返回状态码 | 提示文案 |
|------|-----------|---------|
| 更新健康档案时档案不存在 | 404 | "健康档案不存在，请联系客服" |
| 完善本人档案时档案不存在 | 404 | "请先完成注册流程，再完善健康档案" |
| 接受家庭成员邀请时本人档案不存在 | 400 | "请先完善本人健康档案，再来接受邀请" |

## 使用说明

### 正常使用流程（不受影响）

对于已正常注册的用户，本次更新**完全透明**，所有功能照常使用：

1. **注册**：新用户注册时自动创建本人健康档案（`family_member_id = 0`）
2. **更新档案**：在「健康档案」页面编辑档案信息，正常保存
3. **接受邀请**：接受家庭成员邀请时，系统正常合并健康档案数据
4. **管理成员**：家庭成员档案的增删改查不受影响

### 异常场景处理（本次新增）

如果遇到以下提示，说明本人档案未正确创建，请按提示操作：

**场景一：更新健康档案时提示"健康档案不存在，请联系客服"**
- 原因：注册时本人档案未成功创建
- 处理：联系客服，由后台协助处理

**场景二：完善档案时提示"请先完成注册流程，再完善健康档案"**
- 原因：尚未完成完整的注册流程
- 处理：返回注册页面完成注册，系统将自动创建本人档案

**场景三：接受邀请时提示"请先完善本人健康档案，再来接受邀请"**
- 原因：本人档案尚未创建或未完善
- 处理：先前往「健康档案」页面完善本人档案，再返回接受邀请

## 注意事项

1. **历史脏数据已清理**：数据库中已存在的重复本人档案记录，已自动清理（保留最早创建的记录）
2. **数据库唯一约束**：`(user_id, family_member_id)` 联合唯一索引已生效，从物理层面保证不会再出现重复记录
3. **注册流程是唯一入口**：本人健康档案**只在注册时创建**，其他任何接口不再自动创建，从源头切断脏数据产生路径
4. **正常使用无影响**：对于已正常注册的用户，所有功能照常，无需任何额外操作
