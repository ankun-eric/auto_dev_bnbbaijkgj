## 修复概要

### 问题现状
测试环境上 `/api/brain-game/*` 所有接口返回 404，省市区街道数据在 `brain_game_regions` 表中不存在。

### 根本原因
代码已存在于 Codeup 远程仓库中，但服务器 Docker 镜像未包含最新版本的 brain_game 模块代码。

### 修复内容
1. **服务器拉取最新代码**：通过 `git pull` 将包含 brain_game 模块的最新代码同步到服务器
2. **重新构建 Docker 镜像**：使用 `docker compose build --pull` 重新构建 backend 和 h5 镜像
3. **种子数据自动初始化**：backend 容器启动时检查 `brain_game_regions` 表为空则自动写入 10 个城市的完整省市区街道数据（257 条记录）
4. **数据去重**：清理了因多次调用 sync-seed 接口导致的重复数据

## 数据库表说明

种子数据写入的表：**`brain_game_regions`**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 自增主键 |
| adcode | VARCHAR(12) | 高德行政区划编码 |
| name | VARCHAR(128) | 名称（如"天河区"） |
| level | VARCHAR(16) | 级别：province/city/district/street |
| parent_adcode | VARCHAR(12) | 父级 adcode |
| center | VARCHAR(64) | 中心经纬度 |
| synced_at | DATETIME | 同步时间 |

- **当前记录数**：257 条（去重后）
- **数据覆盖**：广东省 10 个城市的完整四级行政区划

---

## 注意事项

1. **种子数据幂等性**：`POST /api/brain-game/regions/sync-seed` 接口目前已增加幂等保护，重复调用不会导致数据重复
2. **自动初始化**：每次 backend 容器启动时，lifespan 函数会检查表是否为空，若为空则自动写入种子数据
3. 高德 API 同步接口 `/api/brain-game/regions/sync` 依赖高德 API Key，需要配置真实 Key 后才能使用
