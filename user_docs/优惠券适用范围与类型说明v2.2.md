## 功能简介

本次修复让此前已合并到代码仓库的「优惠券适用范围 & 类型说明」v2.2 优化在测试服真正生效。

之前因部署链路命中了 Docker layer cache，造成"外壳新（chunk hash 是新的）、内核旧（chunk 内容仍是旧版本）"的假部署，导致以下 5 个 v2.2 新元素在弹窗中全部缺失。本次以「git 强制同步 → 清空 .next 缓存 → `docker build --no-cache --pull` 全量重建 admin-web → `up -d --force-recreate` 容器」的流程一次性修复完毕。

修复后，新增/编辑优惠券弹窗将呈现以下 5 处 v2.2 新元素：

1. **优惠券类型 ❓ 图标**：点击查看 4 种类型的差异、可叠加规则、计算逻辑
2. **「按分类」模式的树形多选**：替代手填分类 ID 文本框
3. **「按商品」模式的弹窗选择器**：支持搜索、分页、批量勾选
4. **独立的「排除商品」区块**：可单独配置排除商品名单
5. **底部实时统计预览条**：实时显示"本券将对 N 个分类（共 M 个商品）生效，已排除 K 个商品"

## 使用说明

### 步骤 1：以无痕窗口打开管理后台

为避免任何浏览器缓存干扰：

1. 打开浏览器**无痕/隐私窗口**（Chrome：Ctrl+Shift+N；Edge：Ctrl+Shift+N；Safari：Cmd+Shift+N）
2. 访问 [https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin)
3. 使用账号 `13800000000` 登录

### 步骤 2：进入优惠券管理

1. 在左侧菜单选择「商品体系」→「优惠券管理」
2. 直接访问页面：[https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/coupons](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/product-system/coupons)

### 步骤 3：体验「新增优惠券」弹窗（验证 5 个新元素）

点击右上角「新增优惠券」按钮，弹窗内应同时出现以下全部 5 处新元素：

| # | 新元素 | 验证位置 | 操作方式 |
|---|--------|----------|----------|
| 1 | ❓ 类型说明图标 | 表单顶部「优惠券类型」标签右侧 | 点击图标，弹出类型说明弹窗（4 张并排卡片） |
| 2 | 分类树多选 | 「适用范围」选「按分类」时 | 树形控件，多级勾选，可半选 |
| 3 | 商品弹窗选择器 | 「适用范围」选「按商品」时 | 点击「选择商品」按钮，弹出搜索 + 分页 + 批量勾选 |
| 4 | 排除商品区块 | 弹窗中部独立区块 | 点击「添加排除商品」按钮调起选择器，已选商品以 Tag 显示 |
| 5 | 实时统计条 | 弹窗底部（绿色背景轻提示） | 修改适用范围后即时刷新数字 |

### 步骤 4：体验「编辑优惠券」弹窗

1. 在列表中任选一条已有优惠券
2. 点击操作列的「编辑」按钮
3. 验证同样 5 个新元素均存在

### 步骤 5：保存验证（可选）

新建一张测试优惠券：

- 适用范围选择「按商品」，点击「选择商品」勾选若干商品
- 在「排除商品」中加入 1~2 个商品
- 保存后，按 F5 刷新页面，再次点「编辑」打开，验证以下字段持久化：
  - `scope_product_ids` 为多选数组（不是单一字符串）
  - `exclude_product_ids` 字段被正确保存
  - 折扣计算时正确按 v2.2 规则排除「排除商品」

## 注意事项

1. **请使用无痕/隐私窗口验证**：如果用日常浏览器，可能因强缓存导致仍看到旧界面。无痕窗口排除一切浏览器侧干扰。
2. **页面 chunk 文件名**：本次部署后，coupons 页面的 chunk 实际文件为 `page-b2e4d3a9da7a9ad9.js`。如果将来部署有变更，文件名 hash 会再次变化，但只要内容真正更新这是预期行为。
3. **若仍看到旧界面**：请尝试以下顺序操作
   - 先在 F12 → Network 中关闭"Disable cache"前再开启
   - 再用 Ctrl+Shift+Delete 清空"过去 1 小时"的缓存
   - 仍不行时，换一个浏览器（或使用无痕窗口）
4. **关于本次修复的工程化建议**：本次"假部署"的根因是 `docker compose build admin-web` 命中了 layer cache。后续测试服首次部署应统一使用 `docker compose build --no-cache --pull admin-web`，避免类似问题再次发生。已在 `deploy/_fix_coupon_v2_2_no_cache_20260503.py` 中固化此参数。
