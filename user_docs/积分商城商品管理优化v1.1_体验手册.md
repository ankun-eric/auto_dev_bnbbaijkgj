## 一、功能简介

本次升级带来 5 个模块的全面优化：

- **M1 商品三态流转 + 字段锁定**：商品新增「草稿 / 在售 / 已下架」三态；在售状态下的关键字段（分类、类型、券关联、积分价、SKU 等）锁定保护，防止数据错乱；库存字段具备优惠券同步防呆（实时黄字 + 保存二次确认）。
- **M2 修改历史 Tab**：对在售状态下仍可修改但需留痕的字段（标题、主图、轮播图）自动记录每次修改的前后值、操作人与时间，后台可按字段筛选查看。
- **M3 复制新建 + 无缝替换**：一键复制在售商品生成草稿，修改锁定字段后发布即可自动下架原商品并打「被替代」标签；已兑换用户点击"使用按钮"时智能跳转到新商品。
- **M4 详情模板库**：商品详情富文本编辑新增 T1 亮点卡 / T2 使用说明卡 / T3 有效期提示卡 / T4 注意事项卡 四套一键插入模板，大幅提升运营编辑效率。
- **M5 用户端列表交互优化**：顶部「全部 / 可兑换」Tab + 红点提示；卡片新增「立即兑换」快捷按钮、库存紧张提示；全局智能排序（库存 0 沉底 + 积分够排前）；使用按钮 5 种状态覆盖全场景。

## 三、使用说明（运营侧）

### 3.1 商品三态与字段锁定（M1）

商品生命周期只会处于三种状态之一：

- **草稿（draft）**：用户不可见，全部字段可自由编辑；
- **在售（on_sale）**：用户可见、可兑换；部分字段被锁定；
- **已下架（off_sale）**：用户侧完全不可见；后台默认隐藏，可通过筛选查看；已兑换用户在「我的兑换记录」中仍能看到商品快照。

**在售状态下的锁定字段一览**：

| 字段 | 锁定 | 行为 |
|------|------|------|
| 商品分类 / 商品类型 / 关联优惠券(或服务)ID / 积分价格 / SKU / 有效期开始 | 🔒 锁 | 置灰 + 悬停提示"商品在售中，此字段不可修改。如需修改，请先【下架商品】或使用【复制新建】" |
| 商品标题 / 主图 / 轮播图 | ✏️ 可改，留痕 | 修改自动写入「修改历史」 |
| 详情 / 库存 / 单人限购 / 排序权重 / 每日限兑 / 有效期结束（仅可延后） | ✏️ 自由改 | 不留痕 |

**库存防呆（优惠券类商品）**：
- 库存输入框下方实时提示「📎 当前关联优惠券总量：XX 张（仅供参考，实际库存以下方填写为准）」；
- 若填写值 > 优惠券总量，失焦即时黄字警告"⚠️ 当前填写 X，优惠券总量仅 Y，超出 Z 张…"；
- 保存时若仍超量，弹窗二次拦截【取消 / 确认超量保存】。

### 3.2 修改历史（M2）

1. 进入 [管理后台 → 积分商城商品管理](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/points/mall)；
2. 对一条商品点击操作列的【历史】按钮；
3. 弹窗中默认按时间倒序展示每一次标题/主图/轮播图修改，支持按字段筛选；
4. 图片类字段展示缩略图，点击可查看大图。

### 3.3 复制新建 + 无缝替换（M3）

当运营需要修改在售商品的锁定字段（如积分价格、券关联）时：

1. 在商品列表对**在售**商品点击【复制新建】→ 系统自动创建草稿副本（标题自动加"(副本)"后缀）；
2. 进入副本编辑页，修改任意字段（包括原本锁定的字段）；
3. 点击【发布】→ 新商品上架 + 原商品自动下架并打上「已被《新商品名》替代」标签；
4. 原商品的 `replaced_by_goods_id` 指向新商品，已兑换用户在使用按钮点击时自动跳转到新商品详情页。

### 3.4 详情模板库（M4）

商品编辑页「详情」Tab 中，运营可直接点击预置按钮一键插入 4 套图文卡片：

| 模板 | 用途 |
|------|------|
| **T1 商品亮点卡** | 开篇介绍核心卖点（图标 + 3 条亮点）|
| **T2 使用说明卡** | 说明如何使用（步骤条 1-2-3-4）|
| **T3 有效期提示卡** | 强调时效（高亮色块 + "X 天内有效"）|
| **T4 注意事项卡** | 风险告知（黄色警示框 + 条目列表）|

插入后 HTML 片段可继续自由编辑。详情总大小限制 ≤ 50KB。

## 四、使用说明（用户侧）

### 4.1 全部 / 可兑换 Tab（M5）

进入 [积分商城](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/points/mall)（H5 或小程序）：

- **全部**（默认选中）：展示全部在售商品；
- **可兑换**：只展示"我的积分 ≥ 商品积分 **且** 库存 > 0"的商品；
  - 🧠 兜底规则：若"可兑换"结果 < 3 件，自动降级为"只要库存 > 0"，避免新用户积分少时 Tab 空荡；
  - 🔴 红点：当"可兑换"Tab 有商品时，Tab 右上角亮红点（无数字），无商品时不亮。

### 4.2 卡片与「立即兑换」按钮

每张商品卡片：

- 主图 + 商品名 + 积分价 + 类型徽标；
- 库存紧张红字提示：仅在 `stock ≤ 10` 时显示"仅剩 X 件"；`stock = 0` 显示"已兑完"；
- **「立即兑换」快捷按钮**：点击直接触发兑换；
- 点击卡片空白区进入详情页。

**按钮 5 种状态**：

| 场景 | 按钮文案 | 交互 |
|------|----------|------|
| 在售 + 库存 > 0 + 积分够 | 立即兑换 | 点击直接走兑换流程 |
| 在售 + 库存 > 0 + 积分不够 | 立即兑换（置灰非禁用） | 点击提示"积分不足，还差 X 积分" |
| 在售 + 库存 = 0 | 已兑完 | 置灰禁用 |
| 已下架 + 有替代商品 | 去看替代款 | 跳转到替代商品详情页 |
| 已下架 + 无替代商品 | 已下架 | 置灰禁用 |

### 4.3 智能排序

列表按以下优先级：
1. 库存 > 0 的商品排前（库存 0 沉底）；
2. 同组内：用户积分够兑换的排前；
3. 同组同能力内：按运营配置的排序权重（越大越靠前）。

### 4.4 「使用按钮」智能跳转（兑换记录）

在 [我的兑换记录](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/points/exchange-records) 中：

- 若已兑换商品仍在售 → 按原逻辑（服务券去预约、优惠券看我的券、实物看订单）；
- 若已下架 **且** 有替代商品 → 按钮变为【去看替代款】（橙色），点击跳转到替代商品详情页；
- 若已下架 **且** 无替代 → 按钮变为【已下架】灰色禁用。

## 六、注意事项

1. **三态迁移兼容**：已上架的旧商品自动映射为「在售」，未上架的自动映射为「草稿」，历史数据不会丢失。
2. **库存只能减到已兑换数**：若已有 50 人兑换，库存不能改为小于 50，会被后端拦截。
3. **已下架商品用户侧完全不可见**，但「我的兑换记录」中仍能查看商品快照（标「已下架」）。
4. **"可兑换" Tab 兜底**：当你看到"可兑换"Tab 里也有积分不够的商品时，这是正常的——只有当原始可兑换数 < 3 时才会降级展示。
5. **后台浏览器兼容**：Chrome / Edge / Firefox 最新 3 个主版本，**不支持 IE 11 及以下**。
6. **富文本详情限制**：总大小 ≤ 50KB（约 5000 字 + 10 张图外链）。
7. **小程序体验前请务必下载本次最新包**：如需在真机体验，请下载上方「微信小程序体验」章节的 zip 包并重新导入，否则可能看不到 M5 新功能。
8. **管理后台的"复制新建"是修改锁定字段的唯一无感路径**：请勿通过下架→再上架的方式修改锁定字段，否则该商品的销量和评价都会归零。
