## 功能简介

本次更新对**订单状态机**进行了简化，核心变化如下：

1. **状态合并**：把原本的"已预约（appointed）"和"待核销（pending_use）"合并为统一的"**待核销**"状态，订单生命周期从 5 段精简到 4 段。
2. **核销码即时可见**：用户填写预约日期后，**立即**展示核销二维码，**不再**等待预约日 00:00 才能看到。
3. **支持提前核销**：商家扫码核销时，**不再**校验"是否到达预约日"，用户可在预约日前到店即时使用。
4. **改约/退款入口全程可见**：在"待核销"阶段，用户可随时修改预约时间或申请退款（已部分核销/退款流程中除外）。
5. **新增 T-1 18:00 到店提醒**：预约日**前一天 18:00**自动给用户推送"明日到店提醒"，替代原 00:00 的"订单可核销"提醒。
6. **数据兼容**：历史 `appointed` 订单已**全量迁移**为 `pending_use`，旧订单功能完全兼容。

## 使用说明

### 用户端（H5 / 小程序 / APP）

1. **下单 → 选预约日**：购买商品后进入订单详情，点击"立即预约"，选择到店日期。
2. **核销码立即可见**：预约日选定后，订单状态变为"待核销（预约 X月X日）"，**核销二维码立即展示**在订单详情页中部。
3. **到店核销**（支持提前到店）：在预约日当天或之前，到店时向商家出示二维码，商家扫码即可完成核销。
4. **修改预约**：在核销前的任意时刻，点击订单详情底部的"修改预约时间"按钮，可重新选择到店日期。
5. **申请退款**：在核销前（未部分核销时），点击"申请退款"提交退款申请。
6. **T-1 18:00 提醒**：预约日前一天 18:00，系统会自动推送一条到店提醒（站内消息 + 服务号模板消息）。

### 商家端（Admin 后台）

1. **登录**进入"产品系统 → 订单管理"。
2. **待核销 Tab**：合并了原来的"已预约"和"待核销"两个 Tab，列表按预约时间升序排列，方便按到店时间安排接待。
3. **订单详情**：状态显示为"待核销（预约 X月X日）"，点击对应订单可查看详细信息。
4. **扫码核销**：在"扫码核销"页面，扫描用户出示的二维码，**不再**校验是否到达预约日，可立即完成核销。
5. **预约日历**：预约日历功能保持不变，仍按 `appointment_time` 字段统计到店人数。

## 注意事项

1. **历史订单兼容**：本次部署时已自动把所有 `appointed` 订单更新为 `pending_use`，老订单的核销码、改约、退款流程完全兼容，**无需用户重新下单**。
2. **00:00 自动状态翻转已下线**：以前 `appointed → pending_use` 的定时任务（00:00 触发）已**移除**；保留 R2（核销日 24:00 后未使用 → 退回 `pending_use` 等待补预约）逻辑。
3. **改约限制**：处于"部分已核销"或"退款流程中"的订单**不可**修改预约时间，需先终止退款流程或联系客服。
4. **T-1 提醒去重**：T-1 18:00 提醒按订单维度去重，同一订单同一预约日只会发送一次。
5. **小程序导入**：导入开发者工具时若出现"AppID 不匹配"，可改为"测试号"模式即可正常预览。
