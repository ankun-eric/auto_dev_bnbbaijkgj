## 一、Bug 概述

### 1. 现象

健康档案 v1.1（本人资料完善弹窗与抽屉）部署后，所有用户访问健康档案页面时出现：

```
Application error: a client-side exception has occurred
(see the browser console for more information).
```

浏览器控制台实际报错：

```
ReferenceError: Cannot access 'tn' before initialization
```

### 2. 影响范围

| 端 | 是否受影响 | 范围 |
|---|---|---|
| H5（Next.js） | ✅ 是 | `/health-profile/` 全部用户白屏 |
| 小程序 | ❌ 否 | 不受影响 |
| 安卓 APP | ❌ 否 | 不受影响 |
| iOS APP | ❌ 否 | 不受影响 |

### 3. 复现条件

- 老账号 100% 复现
- 新账号 100% 复现
- 三项资料已填齐也复现

---

## 二、根因（实锤）

**根因不是循环依赖，而是 React 组件函数体内变量声明顺序问题导致的 TDZ（暂时性死区）。**

`h5-web/src/app/health-profile/page.tsx` 中：

- v1.1 在第 603 行新增了一个 `useEffect`，其依赖数组中使用了 `selectedMember`
- 但 `selectedMember` 的 `useMemo` 声明位于第 697 行（在该 useEffect **之后**）
- 在 React 函数组件中，`const`/`let` 声明的变量**没有 hoisting**，函数体严格按代码顺序执行
- 开发模式（HMR）下宽松容错，看不出来；
- 生产构建（Next.js / webpack / terser 压缩）下，混淆器把 `selectedMember` 命名为 `tn`，
  执行到第 603 行 useEffect 时立即触发：
  **`ReferenceError: Cannot access 'tn' before initialization`**，
  React 错误边界吞掉异常并渲染为白屏 + "Application error"。

### 验证

| 项 | 修复前 | 修复后 |
|---|---|---|
| chunk 中 `tn` 出现次数 | 46 次 | 3 次 |
| `tn` 首次出现是声明位置 | ❌ 否（先被读取） | ✅ 是（先声明） |
| `selectedMember` 在 chunk 中混淆名 | `tn`（声明在第 20051 字节） | `eQ`（声明在第 12029 字节，紧随 useState 之后） |
| `/health-profile/` 加载 | 白屏 + client-side exception | 正常渲染档案页 |
| `pageerror` 监听 | `ReferenceError: ...` | 0 条 TDZ 错误 |

## 三、修复内容

### 1. 改动文件

| 文件 | 改动 |
|---|---|
| `h5-web/src/app/health-profile/page.tsx` | 把 `selectedMember`（`useMemo`）与 `memberQs` 的声明上移到**所有 `useEffect` / `useCallback` 之前**（紧接 useState 区段）；删除原位置的重复声明 |

### 2. 改动详情

修复前：

```text
useState 区段（第 ~220-318 行）
   ↓
useCallback 区段（第 ~320-595 行）
   ↓
useEffect 区段（第 ~597-695 行）   ← 这里有 useEffect 引用了 selectedMember
   ↓
const selectedMember = useMemo(...)   ← 声明在使用之后！TDZ 死区
```

修复后：

```text
useState 区段（第 ~220-318 行）
   ↓
const selectedMember = useMemo(...)   ← 上移到所有 useEffect 之前
const memberQs = ...
   ↓
useCallback 区段
   ↓
useEffect 区段                       ← 引用 selectedMember 没问题
   ↓
（原位置删除重复声明）
```

## 六、注意事项

1. **下次开发 Next.js 客户端组件时**：
   - 所有 `useMemo` / `useCallback` 声明的"派生值"应**放在所有 `useEffect` 之前**
   - 即顺序：`useState → useMemo/useCallback（派生值）→ useCallback（fetch 等异步）→ useEffect`
   - 这样可以从根本上避免本类 TDZ 问题在生产构建中复现。
2. **关于"内核 + 适配壳"重构**：
   - 原修复方案中提议把 `AddMemberDrawer` / `CompleteSelfProfileDrawer` 拆出 `MemberFormCore`。
   - 实测两个抽屉**没有循环依赖**（互不 import，只共享 `lib/family-relation.ts` 中的纯工具函数），不是真正的根因。
   - 本次仅做**最小化、零业务风险**的修复（声明顺序上移），不做大重构。
3. **服务端版本**：
   - h5-web 容器（`6b099ed3-7175-4a78-91f4-44570c84ed27-h5`）已 docker rebuild 并重启
   - 新 chunk 文件：`page-a06679189bbc2754.js`（原 `page-f418831bb0ec35d1.js` 已废弃）
