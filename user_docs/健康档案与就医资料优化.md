## 功能简介

本次更新围绕健康档案、就医资料、AI 首页数据一致性三大方向：

- **健康档案 4 卡片重构**：健康预警（橙红 #FF6B35）、用药计划（蓝 #3B82F6）、家庭成员（绿 #10B981）、我的设备（紫 #8B5CF6）
- **健康预警体系**：体检异常 / 用药异常 / 设备异常 / 手动上传识别异常四类来源统一汇总，24 小时窗口自动合并相同指标
- **就医资料模块**：4 分组承载病例单 / 体检报告 / 药物 / 其他，支持多文件（图片 + PDF）上传，30 天回收站
- **AI 首页今日用药数字修正**：剩余 / 已完成 / 无计划三态文案分别为 `N` / `0 今日已完成` / `0 今日无用药`
- **旧页面清理**：`/devices/member` 物理下线，统一指向 `/devices`；"共管与提醒"、"打开提醒设置"、"暂未守护任何人..." 区域全部移除

## 使用说明

### 一、进入健康档案

1. 打开 App 或 H5 → 底部 Tab 进入「健康档案」
2. 顶部为成员 Tab：「我 / 爸爸 / 妈妈 / + 添加家庭成员」，切换 Tab 即可查看不同成员的健康数据
3. 在 Hero 区下方将看到全新 **4 卡片**：

   - 🟠 **健康预警** — 显示当前成员未处理预警条数
   - 🔵 **用药计划** — 显示有效期未结束的用药计划数
   - 🟢 **家庭成员** — 显示家庭成员总数（含本人）
   - 🟣 **我的设备** — 显示已绑定设备数

4. 当预警 > 0 时，Hero 下方会显示**红色横幅**「您有 N 条未处理预警，立即查看 →」，点击直达预警列表
5. 非本人 Tab 下，「家庭成员」与「我的设备」卡片右上角带 🔒 图标，仅可查看不可管理

### 二、查看 / 处理健康预警

1. 健康档案点「健康预警」卡片，或点击红色横幅
2. 列表页支持：
   - **双 Tab 切换**：未处理 / 已处理（默认未处理）
   - **类型筛选**：全部 / 体检 / 用药 / 设备 / 手动
   - **严重程度**：高 🔴 / 中 🟠 / 低 🟡（左侧色条 + 标题前图标）
   - **24 小时合并**：同成员 + 同类型 + 同指标 24h 内再次触发会合并到一条，标题不变，副标显示「近 X 小时累计 N 次」
3. 点击任一条预警 → 底部抽屉弹出**关联原始数据**、AI 建议、跳转完整详情入口
4. 点击「标记已处理」即可关闭预警（终态，不可撤销）；列表顶部「全部标记已处理」支持批量

### 三、上传 / 管理就医资料

1. 健康档案页向下滚动找到「就医资料」区域，4 分组卡片纵向排列：
   - 📋 病例单
   - 🔬 体检报告
   - 💊 药物
   - 📦 其他
2. 点击任一分组的「+ 上传」按钮，进入上传抽屉：
   - **选择分组**（默认沿用入口分组）
   - **填写标题**
   - **选择资料日期**
   - **添加文件**：图片（≤10MB）或 PDF（≤20MB），单份资料最多 9 个文件，共享一个标题与日期
3. 点击「保存」，资料即归入对应分组
4. 在分组列表中点击资料行 → 进入详情页：
   - 顶部：标题（可编辑）、来源色块（绿=AI 体检 / 蓝=AI 药物 / 灰=手动）、资料日期
   - 中部：原始文件网格预览，点击进入全屏，支持左右滑动浏览同分组邻近文件
   - AI 解读区：AI 来源资料显示结构化解读；手动上传显示「暂无 AI 解读（该资料为手动上传），可去【健康档案】或【用药识别】重新解读」
   - 备注：可编辑
   - 操作：编辑标题/备注 · 删除（进回收站）

### 四、回收站

1. 任一资料删除后会进入对应成员的「回收站」，保留 30 天
2. 回收站入口位于就医资料末尾「🗑️ 回收站 ›」
3. 每条资料显示剩余天数；30 天到期前 3 天将通过 App 消息中心 + 系统推送提醒
4. 操作：
   - **恢复**：回到原分组
   - **彻底删除**：立即物理清理，不可撤销

### 五、AI 首页今日用药数字修正

进入 AI 首页（`/ai-home`），右上角"今日用药"显示规则：

- 剩余待服用 > 0 → 显示数字 `N`
- 今日有计划但全部已打卡 → 显示 `0` + 文案「今日已完成」
- 今日无任何计划 → 显示 `0` + 文案「今日无用药」

铃铛数字保持原规则（未读消息 + 未处理订单），**不**包含健康预警，避免误干扰。

### 六、家庭成员 Tab 切换权限

| 模块 | 本人 Tab | 非本人 Tab |
|------|----------|------------|
| 健康预警 | 可查看 + 标记 | 可查看 + 标记 |
| 用药计划 | 可查看 + 可管理 | 可查看 + 可管理（含代打卡） |
| 家庭成员 | 可管理 | **只读** 🔒 |
| 我的设备 | 可绑定/解绑 | **只读** 🔒 |
| 就医资料 | 可上传 + 删除 | 可上传 + 删除 |

## 注意事项

- 「健康预警」**不会**叠加到铃铛数字中，避免与消息/订单数字混淆
- 「就医资料」资料归属严格按当前成员 Tab 隔离；AI 体检报告解读 / AI 药物识别结果会自动归集到对应"咨询对象"的资料夹，并 Toast 轻提示「已保存到 XX 的就医资料」
- 同一份资料若用户在多处分别识别，会重复归集，由用户自行手动删除（**不**自动去重）
- 提醒设置统一进入"家庭成员 → 该成员 → 提醒设置"管理；原健康档案"打开提醒设置"入口已移除
- 旧路径 `/devices/member` 会自动跳转到 `/devices`，老版本 App 也兼容
- 「我的设备」原"共管与提醒"区域整块下线，旧规则已清空

## 接口对照

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/health-archive-v5/overview` | GET | 4 卡片 + 预警横幅一次拉取 |
| `/api/health-alerts` | GET | 预警列表，参数 `status / alert_type / member_id` |
| `/api/health-alerts/{id}/resolve` | POST | 标记单条已处理 |
| `/api/health-alerts/resolve-all` | POST | 全部标记已处理 |
| `/api/medical-records` | GET / POST | 就医资料列表 / 新建 |
| `/api/medical-records/{id}` | GET / PATCH / DELETE | 详情 / 编辑 / 软删 |
| `/api/medical-records/trash` | GET | 回收站列表 |
| `/api/medical-records/{id}/restore` | POST | 恢复 |
| `/api/medical-records/{id}/permanent` | DELETE | 立即彻底删除 |
| `/api/medication-plans/hero-count` | GET | 今日用药 hero 文案（新增 `ai_home_number / ai_home_label`） |
