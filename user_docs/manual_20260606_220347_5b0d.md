## 使用说明

### 一、家庭成员守护关系（核心修复）

#### 1. 管理方查看被守护人健康档案（Bug-2 修复）

修复前：管理方切换到被守护人 Tab 后，健康档案数据为空或不完整，且修改数据后对方看不到变更。

修复后：管理方可以正常查看和编辑被守护人的完整健康档案（包括身高体重、过敏史、用药记录等），修改后对方可立即看到更新。

**操作步骤：**

1. 打开 H5 用户端，登录后进入「健康档案」页面
2. 在顶部 Tab 栏切换到被守护人的名字
3. 查看该成员的完整健康数据（身高、体重、过敏史、用药记录等）
4. 尝试修改某项数据（如身高/体重），保存后要求被守护人打开自己的 App 确认

#### 2. 解除守护后极简视图（Bug-3 修复）

修复前：解除守护关系后，管理方页面仍然显示全部详细数据，没有正确切换到极简视图。

修复后：解除守护后，管理方只能看到 Hero 卡片 + 守护人信息 + 健康看板（只读），无法再查看详细数据。

**操作步骤：**

1. 进入「健康档案 → 家庭成员」页面
2. 点击某个成员的「解除守护」按钮
3. 输入短信验证码确认解绑
4. 返回该成员页面，确认只能看到简化的只读视图

#### 3. 接受守护邀请后系统消息（Bug-6 修复）

修复前：双方收到的系统消息角色搞反了（管理方被告知"已成为守护者"，被守护人被告知"可以管理对方"——逻辑完全相反）。

修复后：系统消息角色正确：
- 管理方（邀请人）收到：「XXX 已同意加入您的家庭成员，成为您的守护对象」
- 接收方（被邀请人）收到：「您已同意 XXX 的守护邀请，对方可管理您的健康档案」

**操作步骤：**

1. 管理方通过邀请链接邀请某用户加入家庭守护
2. 被邀请人点击链接接受邀请
3. 双方各自进入「消息」页面查看系统消息，确认角色文案正确

#### 4. 解绑后双向通知（Bug-7 修复）

修复前：任意一方解除守护关系后，双方都不会收到系统消息通知。

修复后：解绑后双方都会收到系统消息：
- 操作方收到：「您已解除与 XXX 的家庭健康档案守护关系」
- 对方收到：「XXX 已解除与您的家庭健康档案守护关系」


### 二、管理后台「设备管理」菜单（Bug-1 修复）

修复前：管理后台侧边栏缺少设备管理入口，虽然设备分类管理和设备目录管理的页面已开发完成，但无法从导航栏进入。

修复后：侧边栏新增「设备管理」菜单组，包含：
- **设备场景分类管理**：管理四大设备场景（健康监测、居家安全、运动健康、智能穿戴）
- **设备目录管理**：管理设备型号、品牌、跳转链接和图标

**操作步骤：**

1. 打开管理后台，登录后查看左侧侧边栏
2. 找到「设备管理」菜单组
3. 点击「设备场景分类」进入设备分类管理页面
4. 点击「设备目录管理」进入设备目录管理页面

### 三、邀请/绑定错误提示优化

#### 重复绑定提示（Bug-5 修复）

修复前：重复接受绑定邀请时，错误页标题永远显示"无法处理邀请"，用户不知道具体原因。

修复后：根据不同错误原因显示不同标题：
- 重复绑定 → 「您已在守护关系中」
- 邀请过期 → 「邀请已过期」
- 邀请已取消 → 「邀请已取消」

**操作步骤：**

1. 尝试接受一个已过期的邀请链接，确认页面显示「邀请已过期」
2. 尝试接受一个已被取消的邀请链接，确认页面显示「邀请已取消」

#### 解绑验证码提示区分角色（Bug-8 修复）

修复前：管理方和被守护方执行解绑操作时，确认弹窗提示文案相同。

修复后：根据不同角色显示不同提示：
- 管理方（入口 B - 家庭成员列表）：「解除守护后，您将无法继续管理TA的健康档案，接收TA的健康提醒。此操作不可恢复，请谨慎操作。」
- 被守护方（入口 C - 我的守护人）：「解除守护后，TA将无法继续管理您的健康档案，接收您的健康提醒。此操作不可恢复，请谨慎操作。」

**操作步骤：**

1. 以管理方身份进入「健康档案 → 家庭成员」，点击解绑，查看弹窗提示
2. 以被守护方身份进入「健康档案 → 我的守护人」，点击解绑，查看弹窗提示（文案应不同）

### 四、系统消息展示优化（Bug-9 修复）

修复前：系统消息列表中的消息内容被 CSS 截断显示为省略号，用户无法看到完整内容。

修复后：系统消息列表完整展示每条消息的全部内容，不做截断。

### 五、文案统一（Bug-10 修复）

修复前：多处系统消息和通知中混用「健康数据」「健康提醒」，不够准确。

修复后：全部统一为「健康档案」，包括解绑验证码弹窗、系统消息通知等所有文案。

## 注意事项

1. **账号登录**：系统使用手机号+密码方式登录，默认管理员账号请参考部署配置。
2. **短信验证码**：解除守护关系时需要输入短信验证码，确保是本人操作。
3. **数据同步**：建立守护关系后，管理方和被守护方的健康档案数据会自动合并同步。
4. **解绑不可恢复**：解除守护关系后无法恢复，需重新发起邀请建立关系。
5. **网络要求**：系统部署在 HTTPS 环境，请确保访问时使用 `https://` 协议。
6. **浏览器兼容**：H5 用户端和管理后台支持主流现代浏览器（Chrome、Edge、Safari 等）。
