## 功能简介

bini-health 商品体系支持两种下单预约方式：

| 配置 | 行为 |
|------|------|
| **下单即预约**（`purchase_with_appointment`） | 用户在下单页选择服务日期/时段，付款即等于已预约 |
| **先下单后预约**（`appointment_later`） | 用户下单时**不需要**选择时间，付款后订单进入「待预约」状态，由用户在订单详情页发起预约 |

本次修复前，「先下单后预约」商品在 H5 / 小程序 / APP 下单页**仍被强制要求选择预约日期+时段**，与产品语义冲突。本次修复后：

1. 三端下单页对「先下单后预约」商品**完全隐藏**预约时间选择控件，直接进入支付流程
2. 后端订单接口对该类型商品**忽略**前端误传的 `appointment_time`，避免脏数据写库
3. 付款成功后订单自动进入 **`pending_appointment`（待预约）** 状态
4. 三端订单详情页新增**待预约横幅**和**「立即预约」**按钮+弹窗，用户付款后随时可发起预约
5. 商家后台订单列表已支持「待预约」筛选和状态展示

## 使用说明（用户视角）

### 场景一：在 H5 浏览「先下单后预约」商品并完成付款

1. 在手机浏览器打开 [H5 首页](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/)
2. 注册/登录账号
3. 进入「商品市场」→ 找到一个**先下单后预约**型商品（例如 ID=20 的体检套餐）
4. 点击「立即购买」，进入下单页
5. **关注点**：下单页**不再展示**「预约日期/预约时段」表单——这是本次修复的关键体验改善
6. 选择门店（如适用）→ 填写联系手机号 → 点击「立即支付」完成付款
7. 付款成功后跳到订单详情页

### 场景二：在订单详情页发起预约

1. 进入「我的订单」→ 找到刚下单的订单（状态显示为「**待预约**」紫色徽章）
2. 订单详情页**顶部**显示横幅：「🗓️ 您还未预约服务时间，请尽快选择您方便的时间　[立即预约]」
3. 点击横幅或底部「立即预约」按钮，弹出「选择预约时间」弹窗
4. 在弹窗中：
   - 选择**预约日期**（默认明天，可点击修改，最多 90 天内）
   - 选择**预约时段**（08 个常用半天时段供选择）
   - 点击「确认预约」
5. 提交成功后弹窗关闭，订单状态变为「**已预约**」，预约信息显示在订单详情页中部

### 场景三：在小程序/安卓 APP/苹果 APP 上的相同操作

体验流程与 H5 一致：
- 下单页对「先下单后预约」商品隐藏预约表单
- 订单详情页顶部出现待预约横幅
- 点击「立即预约」按钮弹出日期+时段选择弹窗

请按本手册末尾的「微信小程序体验」/「安卓端体验」/「iOS 端体验」章节下载并安装对应客户端体验。

### 场景四：商家后台查看「待预约」订单（管理员）

1. 用管理员账号登录 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/)
2. 进入「商品体系」→「订单管理」
3. 上方状态 Tab 已包含「**待预约**」分页，点击即可筛选所有付款后未预约的订单
4. 订单详情中可查看用户最终的预约日期/时段（用户预约后状态会自动从「待预约」流转为「已预约」）

## 注意事项

| 维度 | 说明 |
|------|------|
| 兼容性 | 老订单（修复前已下单）保持原状，不做任何回填；新逻辑只对修复后下单的订单生效 |
| 商品配置 | 商家在「商品体系 → 商品管理」编辑商品时，可在「下单预约方式」字段切换「下单即预约」和「先下单后预约」 |
| 退款 | 「待预约」状态下订单可正常申请退款，流转规则保持原有逻辑 |
| 兜底 | 即使旧版前端意外传入 `appointment_time`，后端会主动忽略并写 INFO 日志，不影响订单创建 |
| 状态机 | 订单状态依次为：`pending_payment` → `pending_appointment`（先下单后预约场景）→ `appointed` → `pending_use` → `completed` |
| 容器版本 | 服务器端代码版本：`7c5ead5`（master 分支）；APK / IPA / 小程序 zip 均为 2026-05-03 20:34 构建 |
| 验证测试 | 后端非UI自动化测试在服务器上 44/44 全部通过（`tests/test_book_after_pay_bugfix.py` 6 用例 + 相关回归 38 用例） |
