## 功能简介

本次更新围绕"门店营业时间 / 营业范围"完整修复了以下问题，并做了体验升级：

1. **修复"假成功" Bug**：管理后台"门店编辑"页填写营业时间 / 经营范围后，提示成功但实际未入库——本次完全修复，所有字段全部并入主保存接口的事务，整体提交或整体回滚。
2. **营业时间录入方式升级**：从"手填 HH:MM 文本框"统一升级为 **下拉时间选择器**，30 分钟一档，可选范围 **07:00 – 22:00**，开始时间必须严格早于结束时间。
3. **商家后台 H5 同步对齐**：商家自己维护店铺信息时，营业时间也用同款时间选择器，并新增"营业范围"编辑入口。
4. **存量预约温和提示**：保存营业时间后，如果存在预约时段已不在新营业范围内的未核销订单，会弹一条信息"您有 X 单预约时间已不在新营业范围内，建议联系客户"，**不会**影响订单本身的核销链路。
5. **数据迁移**：服务发布时自动把现网历史的 `business_hours` 字符串（如 `"09:00 - 22:00"`）解析回填到 `business_start` / `business_end`，无法解析的留空，等商家上线后自行选填。

## 一、平台管理员：维护门店营业时间和经营范围

### 入口

登录 [管理后台](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/) → 左侧菜单 **商家管理 → 门店管理**。

### 操作步骤

1. **新建门店**
   - 点击右上角 **新建门店** 按钮。
   - 必填项：门店名称、所属类别、地图选点、单时段最大接单数、**营业时间（开始 + 结束）**。
   - **营业时间**：下拉选 **开始时间** 和 **结束时间**，每档 30 分钟，范围 07:00 – 22:00。系统强制要求结束晚于开始，否则按钮置灰且提示。
   - 选填项：联系人、联系电话、**经营范围**（多选商品分类）。
   - 点击 **确定** 一次性保存所有字段；任何字段失败则整体回滚，不会出现"看似成功实则未入库"。

2. **编辑现有门店**
   - 在门店列表点击 **编辑**。
   - 现有的营业时间会自动回填到下拉选择器；现有的营业范围会自动回填到多选框。
   - 修改任何字段后点击 **确定** 保存。
   - 如果本次修改了营业时间，且当前门店存在预约时段已不在新营业范围内的未核销订单，会自动弹出一条信息：**"您有 X 单预约时间已不在新营业范围内，建议联系客户"**。这些订单仍然可以照常核销，**不会**被强制取消，请管理员视情况联系客户。

3. **校验规则速览**
   - 营业时间必填，留空时按钮置灰且红字提示。
   - 开始 ≥ 结束时，红字提示"结束时间必须晚于开始时间"，按钮置灰。
   - 仅允许 30 分钟整点（XX:00 / XX:30），范围 07:00 – 22:00。
   - 经营范围选填；不填仅意味着在"商品-推荐门店"配置页不会被自动推荐，**不影响下单链路**。

## 二、商家自己：通过商家后台 H5 维护店铺设置

### 入口

商家老板登录商家后台 H5 后，进入 **店铺设置**：
- PC 版：[/merchant/store-settings](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/store-settings)
- 移动版：[/merchant/m/store-settings](https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/merchant/m/store-settings)

> **注意**：仅"老板"角色可以修改店铺设置；店长 / 财务 / 店员只能查看。

### 操作步骤

1. **查看模式**
   - 默认进入查看模式，可看到完整的店铺信息（店铺名称、Logo、简介、地址、电话、**营业时间**、**营业范围**等）。
   - 营业时间统一展示为 `HH:MM - HH:MM`（如 `09:00 - 22:00`）。
   - 营业范围以多个商品分类名拼接展示（如 `健康检查 / 中医理疗`）。

2. **编辑店铺信息**
   - 点击右上角 **编辑** 按钮进入编辑模式。
   - **营业时间**：下拉选开始时间 + 下拉选结束时间，30 分钟一档，07:00 – 22:00。
   - **营业范围**：下拉多选商品分类（PC 端）或选择器多选（移动端）。
   - 修改完成后点击 **保存**。
   - 同样会做"开始 < 结束 / 30 分钟整点 / 范围内"的前后端双重校验。
   - 如果存量预约受影响，也会有弹窗提示。

3. **取消编辑**
   - 点击 **取消** 即可恢复到编辑前的内容，不保存任何改动。

## 三、修改营业时间后，存量订单怎么办？

> 用户发现门店明明营业，却选不到任何可预约时段；或反过来非营业时段也能下单——这都是营业时间字段没正确写入导致的。本次修复后这类问题不复存在。

### 业务规则

- **新预约**：用户在 H5 / 小程序 / App 下单时，预约时段会按**新的营业时间**过滤，超出的时段不会出现。
- **存量未核销预约**：
  - 已下单但未核销的订单，**仍然按原营业时段照常核销**，不会因门店改了营业时间而被取消。
  - 但如果新营业时间收窄、导致部分预约时段已不在新范围内，系统会在保存成功后弹一条提示，建议商家自行联系受影响的客户。
- **核销操作**不受营业时间字段变化影响，门店店员可继续按原流程核销已下单订单。

## 四、注意事项

1. **管理员与商家维护字段保持一致**：管理后台的"营业时间 / 营业范围"和商家后台 H5 的字段都指向同一份数据库记录，任何一端修改、另一端立即可见。
2. **历史数据迁移**：服务上线后，系统会自动把历史的 `business_hours` 字符串（如 `"09:00 - 22:00"`）解析回填到新字段。**异常字符串**（含中文、含多段时间等）无法解析，对应门店的营业时间会留空，需要商家上线后自行重新选择并保存——前端会强制要求必填。
3. **兼容期**：旧的独立保存接口 `PUT /api/admin/stores/{id}/business-scope` 在本版本仍可用，但响应里会返回 `deprecated: true` 标识，**1 个版本周期后下线**，建议 H5 / 小程序等历史前端尽快切换到新主保存接口。
4. **时间选择粒度**：本次将粒度固定为 **30 分钟**。如未来业务需要 15 分钟、5 分钟等更细粒度，可联系平台扩展。
5. **超长营业**：本次默认范围 07:00 – 22:00 覆盖常规门店场景，**不允许跨日**。如有夜场 / 24 小时门店等特殊需求，需要平台单独开通。

## 五、出现问题怎么办？

| 现象 | 排查 / 处理 |
|------|--------------|
| 编辑门店时弹"营业结束时间必须晚于营业开始时间" | 重新选择，确保结束时间晚于开始时间 |
| 编辑门店时弹"营业开始时间必须按 30 分钟为粒度" | 重新选择，开始 / 结束都必须是 XX:00 或 XX:30 |
| 编辑门店时弹"营业开始时间必须在 07:00 – 22:00 之间" | 选择 07:00 – 22:00 区间内的时间 |
| 保存成功后弹出"X 单预约……建议联系客户" | 这是温和提示，订单本身仍可正常核销，建议联系受影响客户 |
| 商家后台编辑入口看不到"编辑"按钮 | 当前账号不是"老板"角色，仅老板可改 |
| 历史门店营业时间为空 | 此门店的旧 business_hours 字符串无法自动解析，需要重新选填一次 |
