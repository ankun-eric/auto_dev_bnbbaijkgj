## 一、功能介绍

本次升级针对「用药计划」全链路做了 5 项交互优化，让你在添加 / 编辑用药时少走弯路：

1. **重新拍照更顺手**：识药卡片上有「重新拍照」按钮，点了之后弹出底部抽屉，可以选「拍照」或「从相册选择」，识别失败还会有专门的失败抽屉引导你「再试一次 / 手动录入 / 取消」。
2. **必填字段红字提示**：忘填必填项点保存时，对应字段标题会变红、下方出现红色错误文字，页面自动滚动到第一个出错的位置，一目了然。
3. **重复药品智能拦截**：同一服用人下同一药品不再允许重复添加。命中已存在时弹「该药已加入用药计划，是否重新编辑？」，点「确定」直接进入编辑模式，避免脏数据。
4. **编辑全字段回显**：进入编辑模式后，药品名、剂量、频次、周期、服用时机、服用人、备注等全部字段都会回显已有值，无需重填。
5. **备注字段全端统一**：所有入口的「备注」字段统一为 200 字上限的多行文本，占位文案统一为「可填写服用提醒、注意事项等（选填）」，右下角带字数计数。

## 四、操作步骤

### 场景 1：拍照识药 → 重新拍照 → 加入用药计划

1. 进入 **AI 首页**，点击底部相机图标，对药盒拍照。
2. 等待识别结果，AI 会弹出**识药卡片**。
3. 若不满意（如照片模糊），点击卡片上的「重新拍照」按钮。
4. 弹出**底部抽屉**：
   - 选「📷 拍照」 → 调起系统相机；
   - 选「🖼 从相册选择」 → 调起系统相册。
5. 完成后系统进入「识别中…」状态。
6. 识别成功 → 直接显示识药卡片；识别失败 → 弹出**失败提示抽屉**：
   - 「再试一次」 → 回到第 4 步的抽屉；
   - 「手动录入」 → 抽屉**平滑左滑**切换为一张空白「新增用药」表单，左上角可点「‹ 返回」滑回；
   - 「取消」 → 关闭抽屉。
7. 在识药卡片上点「加入用药计划」，弹出加入抽屉，填写完成后点「保存」。

### 场景 2：必填字段红字提示

1. 在「新增用药」表单中，故意留空某些必填字段（例如「药品名称」或「服用频次」）。
2. 点击「保存」。
3. 你会看到：
   - 屏幕顶部弹出 Toast 提示「请填写完整必填项」；
   - 所有未填字段的**标题变红**，下方出现红色提示文字（如「请填写药品名称」）；
   - 页面**自动滚动到第一个红字字段**。
4. 点击 / 修改任一红字字段，该字段红字**立即消失**。
5. 全部填好后再次点击「保存」即可。

### 场景 3：重复药品智能拦截

1. 假设你已经为「自己」加入过「阿莫西林胶囊」。
2. 再次尝试为「自己」加入同名药品「阿莫西林胶囊」（前后空格、大小写、全半角不同也算同一个药）。
3. 点击「保存」，系统会先调用重复判定接口。
4. 命中后弹出确认弹窗：
   ```
   提示
   该药已加入用药计划，是否重新编辑？
   [取消]    [确定]
   ```
5. 点击「取消」 → 留在当前新增表单，可以改名后继续保存。
6. 点击「确定」 → 自动跳转到该药品已存在记录的**编辑页**，全部字段已回显。
7. 修改后点「保存」 → 后端做 `UPDATE`，**不会**新增重复记录，**历史服药打卡曲线完整保留**。

### 场景 4：编辑模式回显

1. 在「健康档案 → 用药计划」列表中，点击任一条已有记录。
2. 进入编辑页，确认以下字段全部回显已保存值：
   - 药品名称、规格/剂量、服用频次、服用周期；
   - 服用时机（早 / 中 / 晚 / 睡前 等）；
   - 服用人、备注。
3. 修改任意字段后点「保存」。
4. 列表中的记录被**就地更新**，列表条数不变。

### 场景 5：备注字段统一

1. 在任一入口（识药抽屉 / 健康档案 / 首页快捷入口）打开「新增用药」表单。
2. 找到「备注」字段：
   - 标题：「备注」；
   - 占位文案：「可填写服用提醒、注意事项等（选填）」；
   - 多行文本，默认 3 行；
   - 右下角字数计数：`当前字数 / 200`。
3. 输入超过 200 字时**自动被截断**（小程序为前端控制，Web/H5/App 均同），不会保存超长内容。

## 五、注意事项

1. **重复判定维度**：以「药品名称 + 服用人」为准。名称会被后端做 trim / 大小写 / 全半角统一处理，因此「阿莫西林」「阿莫西林 」「AMOXICILLIN」会被判为同一药品。
2. **历史数据**：本次升级会在后端启动时执行一次**幂等迁移**，把同一服用人 + 同一药品的重复 active 记录保留最新一条、其余软删；**服药打卡历史不动**，历史曲线照常显示。
3. **编辑保存的打卡逻辑**：
   - 主表记录 `UPDATE`，不新增；
   - 已生成的打卡记录保留；
   - 服用时机 / 频次变化的新打卡，从保存时刻之后按新设置生成。
4. **重新拍照入口**：**仅**出现在识药结果卡片上，「加入用药计划」抽屉顶部和失败抽屉里**不会重复**展示该按钮。
5. **失败抽屉手动录入**：进入「手动录入」步骤后，左上角有「‹ 返回」可滑回失败步骤改用「再试一次」重拍；右上角「✕」直接关闭整个抽屉。
6. **备注 200 字**：上限为 200 字符（中英文合计），超出禁止输入；旧字段（部分端原为 30 字）已统一升级，存量数据不受影响。
7. **App 端**：本次 Flutter App 已支持「重复药品弹窗 + 备注 200 字 + 必填字段验证」；「重新拍照失败抽屉滑动切换」的完整动画在 Flutter 端按现有相机入口处理，体验对齐 H5。

## 六、常见问题（FAQ）

**Q1：保存时一直提示「该药已加入用药计划」，但我没添加过？**  
A：可能是某个家庭成员名下已加过同名药品。请检查「服用人」选择是否正确，或在「健康档案 → 用药计划」列表中按服用人筛选查看。

**Q2：编辑保存后，历史打卡记录会丢吗？**  
A：**不会**。本次升级明确保留服药打卡历史表（`medication_check_in`）数据，曲线 / 历史照常显示，只是用药计划设置部分被覆盖。

**Q3：重复判定接口的响应时长？**  
A：P95 < 300ms（已通过 pytest 验证）。如出现 5s 以上无响应，可能是网络问题，前端会自动降级走原 `POST /api/health-plan/medications` 接口，命中重复时后端会回 `409` 同样会弹「重新编辑」提示。

**Q4：备注超过 200 字怎么办？**  
A：客户端会**禁止继续输入**，并在右下角字数计数变红提示。已有的超长备注（如有）后端会兼容显示。
