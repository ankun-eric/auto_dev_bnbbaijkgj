## 一、本次更新带来什么？

本次升级聚焦【AI 咨询配置 → 功能按钮管理】模块，让运营和最终用户都能更顺畅地使用 AI 对话首页的功能按钮。

| 维度 | 升级前的体验 | 升级后的体验 |
| --- | --- | --- |
| 排序 | 宫格和胶囊共用一个 `sort_weight`，运营改了宫格，胶囊也跟着乱 | **宫格、胶囊各自独立排序**，互不影响 |
| 排序操作 | 只能手填数字，容易输错 | **置顶 / 上移 / 下移** 一键操作，不再手填数字 |
| 按钮类型 | 9 种枚举名称混乱（"外部链接"含义不明） | 合并为 **2 大类**：`页面跳转` + `AI 功能`（再选子类型） |
| 点击行为 | 体质测评、健康自查直接执行；拍照识药弹卡片，行为不一致 | **AI 功能统一**：点击 → AI 开场白（可选）→ 卡片 → 操作 |
| 视觉风格 | AI 功能卡片是靛紫色，与首页天空蓝割裂 | **统一改为天空蓝**（`#0EA5E9`），白底浅蓝描边、实色填充按钮 |
| 老配置 | 改造时担心要重做 | **自动迁移**：`sort_weight` 自动复制到两个新字段，老 9 种类型自动归到新两大类 |

## 三、运营操作指南（管理后台）

### 3.1 切换"宫格视图 / 胶囊视图 / 全部按钮"

打开 <https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/admin/function-buttons>，列表顶部有 **三个 Tab**：

- **宫格视图**：仅展示「在功能宫格显示」的按钮，按 `grid_sort` 升序
- **胶囊视图**：仅展示「在胶囊条显示」的按钮，按 `capsule_sort` 升序  
- **全部按钮**：展示全部按钮（兼容老逻辑）

> 注意：每个按钮可同时出现在宫格和胶囊里，**两个排序值各自独立**。

### 3.2 调整顺序：置顶 / 上移 / 下移

每行右侧操作列新增三个按钮：

- **⏫ 置顶**：把当前行排到第 1 位（其余行依次后移）
- **🔼 上移**：与上一行交换顺序；列表第一行此按钮被禁用
- **🔽 下移**：与下一行交换顺序；列表最后一行此按钮被禁用

点击后系统会**事务性地重新编号**当前视图下所有按钮（1, 2, 3…），保证序号不会乱。**不再支持手填数字**。

### 3.3 编辑弹窗：选择按钮类型

按钮编辑弹窗里：

1. **第一步选主类型**（单选）：
   - 「页面跳转」：合并了原来的"外部链接"和"数字人通话"
   - 「AI 功能」：合并了原来的拍照上传/文件上传/报告解读/拍照识药/AI 对话/快捷提问/健康自查 7 种

2. **若选「页面跳转」**：
   - 必填【跳转地址】：以 `http://` / `https://` / `/` / `pages/` 开头
   - 可选【先弹卡片再跳转】开关：默认关闭=直接跳转；开启=先弹引导卡片，用户点卡片按钮再跳

3. **若选「AI 功能」**：
   - 必选【AI 功能子类型】：从 7 个子类型中选一个
   - 必填【卡片标题】、【卡片说明】、【主按钮文案】
   - 可选【卡片配图】、【AI 开场白】

### 3.4 AI 开场白（新增字段）

- **配置位置**：编辑弹窗 → 卡片配置区域 → 「AI 开场白」
- **效果**：用户点击按钮后，AI 会先在对话区**冒一句话**（你填的开场白），再弹出操作引导卡片
- **留空策略**：留空时不冒开场白，直接弹卡片

### 3.5 老数据自动迁移

系统**首次启动**时会自动把所有按钮迁移到新结构：

- 把每行的 `sort_weight` 复制到 `grid_sort` 和 `capsule_sort`
- 老的 7 种 AI 类型按映射表自动回填 `ai_function_type` 子类型
- 兜底：若有未识别的老类型，自动归到 `ai_function` + `ai_dialog_trigger`，并写入日志

> 运营无需任何操作；本次部署日志已确认 **14 条排序补齐 + 3 条类型回填，0 条兜底**。

- 进入 <https://newbb.test.bangbangvip.com/autodev/6b099ed3-7175-4a78-91f4-44570c84ed27/ai-home>
- **宫格区域**（顶部）：按 `grid_sort` 升序展示

- 后台调整顺序后，**最迟 30 秒**用户下次进入即可看到新顺序

### 4.2 点击 AI 功能按钮的统一交互链路

```
用户点击按钮
   ↓
（如配置了 AI 开场白）AI 在对话区冒一句话
   ↓
对话区弹出操作引导卡片（天空蓝风格）
   ↓
用户点击卡片上的主按钮
   ↓
执行真正动作（拍照 / 上传 / 测评 / 触发 AI 对话…）
```

### 4.3 点击页面跳转按钮的两种行为

- **开关关闭（默认）**：直接跳转目标页面（外部链接或内部页面），与原来一致
- **开关开启**：先弹一张引导卡片做二次确认，用户点卡片按钮才真正跳转——适合外部链接做风险提示、关键页做二次确认

## 五、注意事项

1. **排序变更生效时机**：列表接口 30s 内强一致，无需发版、无需审核。
2. **并发冲突**：两个运营同时点击排序按钮，后提交者会收到「顺序已被其他人调整，请刷新」提示。
3. **客户端兼容**：旧版本 App / 小程序若拿到未知的 `button_type` 或 `ai_function_type`，会**忽略该按钮**而不是报错，请提示老用户升级。
4. **AI 开场白限长**：建议不超过 60 个汉字，避免在小屏端溢出。
5. **跳转地址校验**：必须是 `http(s)://` 或 `/` 或 `pages/` 开头，否则后台保存时会报错。
6. **AI 功能必填项**：`card_title` / `card_description` / `card_button_text` 三项任一为空，保存时阻断；`ai_opening` 留空允许（直接弹卡片）。
7. **不再支持手填排序数字**：编辑弹窗中已隐藏 `grid_sort` / `capsule_sort` 字段，统一通过列表的「置顶/上移/下移」操作。
8. **不做拖拽排序、不做后台预览**：本期由运营自行打开 H5 验证视觉，专注核心体验。

## 六、常见问题（FAQ）

**Q1：宫格和胶囊里同一个按钮，能各自排不同顺序吗？**  
A：可以。`grid_sort` 和 `capsule_sort` 完全独立。

**Q2：原来手填的数字现在去哪了？**  
A：已自动复制到 `grid_sort` 和 `capsule_sort`，原顺序完全保留。

**Q3：升级前配的"拍照识药"按钮还能用吗？**  
A：能。后台已自动归到 `AI 功能 → 拍照识药`，行为完全一致；运营如需修改文案/卡片，进入编辑弹窗即可看到所有新字段。

**Q4：我希望某个外部链接弹个风险提示再跳转，怎么配？**  
A：编辑该按钮 → 主类型选「页面跳转」→ 打开「先弹卡片再跳转」开关 → 填写卡片标题/说明/按钮文案/可选 AI 开场白 → 保存。

**Q5：AI 开场白和卡片说明文字有什么区别？**  
A：AI 开场白是 AI 头像气泡里的一句话，卡片说明文字是卡片正文。一段引导用户、一段解释功能，二者互补。
