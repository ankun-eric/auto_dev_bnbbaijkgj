# 后台菜单管理 - 图标选择器扩充优化 产品需求文档（PRD）

> 版本：v1.0  
> 日期：2026-04-09  
> 状态：已确认，待开发  
> 产品经理：小白 AI

---

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「首页配置 → 首页菜单管理」中已有一套 Emoji 图标选择器，支持 **7 大分类浏览**（医疗健康、运动健身、食物饮料、自然植物、活动物品、符号标志、动物）、**中文关键词搜索**（基于 260+ Emoji 中文标签映射）和**智能推荐**（根据菜单名称匹配推荐图标）。

但在实际使用中发现以下问题：

1. **Emoji 覆盖范围不足**：当前仅有 7 个分类，缺少科技数码、人物表情、交通工具、建筑地标、天气时间、办公学习、金融商务等常见分类
2. **中文标签映射不完整**：输入"机器人"等常用关键词搜索不到对应 Emoji（如 🤖），标签库覆盖面有限
3. **分类颗粒度不够**：部分常用 Emoji 被遗漏在所有分类之外，用户无法通过分类浏览找到

本次优化旨在：**大幅扩充 Emoji 分类和中文标签映射，让管理员在设置菜单图标时有更丰富、更全面的选择**。

### 1.2 目标用户

- 管理后台运营人员（管理员角色）

### 1.3 核心价值

- 图标分类从 7 个扩充至 **14 个**，覆盖绝大多数使用场景
- 中文标签映射从 260+ 个扩充至 **450+** 个，关键词搜索命中率大幅提升
- 保持现有搜索、推荐、选择逻辑不变，零学习成本

### 1.4 需求边界（不做的事）

| 明确不做 | 原因 |
|---------|------|
| 自定义图标上传 | 当前 Emoji 扩充后已能满足需求，不引入图片上传复杂度 |
| 已有菜单图标批量刷新/迁移 | 已有菜单图标保持不变，仅新增/编辑时体验优化 |
| 搜索算法升级（模糊匹配、AI 搜索等） | 当前搜索逻辑足够，标签库扩充后搜索结果自然更丰富 |
| 最近使用/收藏图标功能 | 暂不增加复杂度，后续迭代考虑 |

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| F01 | 新增 7 个 Emoji 图标分类 | P0 | 科技数码、人物表情、交通工具、建筑地标、天气时间、办公学习、金融商务 |
| F02 | 扩充 EMOJI_CATEGORIES 数据 | P0 | 每个新增分类填充 20-30 个常用 Emoji |
| F03 | 扩充 EMOJI_LABEL_MAP 中文标签 | P0 | 为所有新增 Emoji 添加 4 个中文关键词标签 |
| F04 | 扩充 EMOJI_KEYWORD_MAP 关键词映射 | P1 | 为新增分类相关的菜单名称场景补充关键词映射 |
| F05 | 兼容性保障 | P0 | 已有菜单图标不受影响 |

### 2.2 功能详细描述

#### F01: 新增 7 个 Emoji 图标分类

**分类排列顺序**：保持现有 7 个分类顺序不变，7 个新增分类依次追加在末尾。

#### F02: 扩充 EMOJI_CATEGORIES 数据

在 `admin-web/src/app/(admin)/home-menus/page.tsx` 文件的 `EMOJI_CATEGORIES` 常量中追加 7 个新分类对象，每个分类包含 20-30 个精选 Emoji。

**选择原则**：
- 每个分类优先选择辨识度高、在不同操作系统/浏览器渲染一致的 Emoji
- 避免与现有分类中的 Emoji 重复（少量跨分类出现的通用 Emoji 允许重复，如 📱 在"活动物品"和"科技数码"中都合理出现）
- 单个分类 Emoji 数量控制在 20-30 个，避免过于臃肿

#### F03: 扩充 EMOJI_LABEL_MAP 中文标签

为所有新增的 Emoji 在 `EMOJI_LABEL_MAP` 中添加中文关键词标签映射。

**标签规则**（与现有规则一致）：
- 每个 Emoji 对应 **4 个中文标签**
- 第 1 个标签为 Emoji 的最直接中文名称（如 🤖 → "机器人"）
- 第 2-4 个标签为相关联想词（如 🤖 → "AI", "智能", "自动化"）
- 标签应覆盖用户可能搜索的常见词汇

**重点补充示例**（用户明确反馈缺失的）：

| Emoji | 中文标签 |
|-------|---------|
| 🤖 | ['机器人', 'AI', '智能', '自动化'] |
| 🎨 | ['画板', '调色板', '绘画', '艺术'] |
| 🖼️ | ['画框', '图片', '相框', '艺术'] |
| 🛰️ | ['卫星', '太空', '通讯', '航天'] |
| 🔋 | ['电池', '充电', '电量', '能源'] |
| 🖨️ | ['打印机', '打印', '复印', '办公'] |
| ⌨️ | ['键盘', '打字', '输入', '编程'] |

#### F04: 扩充 EMOJI_KEYWORD_MAP 关键词映射

在 `EMOJI_KEYWORD_MAP`（用于菜单名称智能推荐）中补充新增分类相关的关键词组，示例：

| 关键词 | 推荐 Emoji |
|--------|-----------|
| 科技/技术/智能/AI | 🤖💻🖥️📡🛰️⚙️ |
| 交通/出行/驾驶 | 🚗🚌🚲🚀✈️🚇 |
| 建筑/房屋/场所 | 🏠🏢🏫🏪🏛️🏗️ |
| 天气/气象/温度 | ☀️🌧️❄️🌈🌪️🌡️ |
| 办公/文档/报告 | 📝📊📋💼📎🗂️ |
| 金融/理财/支付 | 💰💳📈🏦💹💸 |
| 教育/课程/培训 | 📚🎓🧑‍🏫📖✏️📝 |
| 机器人/自动/智能 | 🤖⚙️🔧💻🧑‍💻🔬 |

#### F05: 兼容性保障

- 已有菜单记录的 `icon_content` 字段存储的 Emoji 字符不受影响
- 现有 7 个分类中的 Emoji 列表和标签保持不变，仅做追加
- 搜索逻辑、推荐逻辑、选中交互逻辑均保持不变
- 面板 UI 布局保持不变（520px 宽度，10 列网格，分类 Tab 切换）

## 3. 非功能需求

### 3.1 性能要求

| 项目 | 要求 |
|------|------|
| 图标面板打开速度 | < 200ms（纯前端数据，无网络请求） |
| 搜索响应时间 | < 100ms（前端内存匹配，标签库扩充后仍保持毫秒级） |
| 分类切换 | 即时响应，无卡顿 |

### 3.2 兼容性要求

- 新增 Emoji 需在主流浏览器（Chrome 90+、Firefox 90+、Edge 90+、Safari 14+）中正常渲染
- 避免使用过新的 Emoji 版本（建议不超过 Emoji 14.0），确保跨平台显示一致

### 3.3 可维护性要求

- 新增分类和标签与现有数据结构完全一致，方便后续扩展
- 建议在代码注释中标注 Emoji 版本范围和更新日期

## 4. 影响范围

### 4.1 前端改动

| 文件 | 改动内容 |
|------|---------|
| `admin-web/src/app/(admin)/home-menus/page.tsx` | 扩充 `EMOJI_CATEGORIES`（+7 分类）、`EMOJI_LABEL_MAP`（+190 标签）、`EMOJI_KEYWORD_MAP`（+10 关键词组） |

### 4.2 后端改动

**无**。本次为纯前端数据扩充，无后端接口变更、无数据库迁移。

### 4.3 部署影响

- 仅需重新构建和部署 `admin-web` 前端容器
- 无数据迁移，无配置变更

## 6. 交互设计说明

### 6.1 分类 Tab 栏

由于分类从 7 个增加到 14 个，Tab 栏可能超出面板宽度：

- **方案**：分类 Tab 栏支持**横向滚动**，当分类数量超出可视区域时，左右可滑动浏览
- 或者采用**两行排列**，第一行显示前 7 个，第二行显示后 7 个
- **建议**：采用横向滚动方案，保持界面简洁，与主流 Emoji 面板体验一致

### 6.2 其他交互

保持不变：
- 搜索输入框位置和功能不变
- 智能推荐区位置和逻辑不变
- Emoji 网格布局（10 列，36x36px）不变
- 选中高亮样式（蓝色边框 + 浅蓝背景）不变
- Modal 宽度（520px）不变
