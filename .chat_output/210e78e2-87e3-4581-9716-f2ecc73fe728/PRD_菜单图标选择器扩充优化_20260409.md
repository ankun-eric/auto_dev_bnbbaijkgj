# 后台菜单管理 - 图标选择器扩充优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「首页配置 → 首页菜单管理」中的 Emoji 图标选择器已实现基础功能（7 大分类浏览、260+ 中文标签搜索、智能推荐），但在实际使用中发现以下问题：

1. **Emoji 种类覆盖不足**：现有 7 个分类（医疗健康、运动健身、食物饮料、自然植物、活动物品、符号标志、动物）虽然涵盖了核心场景，但缺少科技数码、人物表情、交通工具、建筑地标、天气时间、办公学习、金融商务等常见分类
2. **中文标签映射不够全面**：当前 260+ 条标签映射中，部分高频搜索词无法命中（如搜索"机器人"无结果），用户找不到想要的图标
3. **分类维度单一**：现有 7 个分类偏向健康管理垂直场景，作为通用菜单图标选择器，分类覆盖面不够广

本次优化旨在：**大幅扩充 Emoji 图标库和中文标签映射，新增多个常用分类**，让管理员在设置菜单图标时有更丰富、更全面的选择。

### 1.2 目标用户

- 管理后台运营人员（管理员角色）

### 1.3 核心价值

- 将可选 Emoji 图标从当前约 260 个扩充至 **500+** 个，覆盖更广泛的使用场景
- 将图标分类从 7 个扩充至 **14 个**，新增 7 个常用分类
- 扩充中文标签映射库，让中文搜索覆盖率从约 70% 提升至 **95%+**
- 保持现有功能和体验不变，已有菜单图标不受影响

#### F02: 每个新增分类的 Emoji 内容

每个分类需填充 **20~30 个** 精选常用 Emoji，选择标准：

1. 优先选择跨平台兼容性好的 Emoji（Unicode 12.0 及以下版本优先）
2. 优先选择含义明确、辨识度高的 Emoji
3. 避免与现有分类中的 Emoji 大量重复（少量重复可接受，如 💻 同时出现在"活动物品"和"科技数码"中）

## 3. 页面/界面设计

### 3.1 页面结构与导航

本次优化不新增页面，仅对现有 **Emoji 图标选择器 Modal（EmojiPickerModal）** 的内容进行扩充。

### 3.2 界面变更说明

#### 3.2.1 分类标签栏

**现状**：7 个分类标签横向排列
```
[医疗健康] [运动健身] [食物饮料] [自然植物] [活动物品] [符号标志] [动物]
```

**优化后**：14 个分类标签，因数量增多，标签栏需支持横向滚动或自动换行
```
第一行: [医疗健康] [运动健身] [食物饮料] [自然植物] [活动物品] [符号标志] [动物]
第二行: [科技数码] [人物表情] [交通工具] [建筑地标] [天气时间] [办公学习] [金融商务]
```

**交互规则**：
- 分类标签区域允许自动换行显示，确保所有分类一目了然
- 当前选中的分类标签高亮显示（与现有逻辑一致）
- 默认展示第一个分类"医疗健康"

#### 3.2.2 Emoji 网格区域

- 保持现有的网格布局（10 列，每个 Emoji 36x36px）
- 每个分类下展示该分类的所有 Emoji
- 选中某个 Emoji 时蓝色高亮（与现有逻辑一致）

#### 3.2.3 搜索功能

- 保持现有搜索逻辑不变
- 搜索时遍历扩充后的 `EMOJI_LABEL_MAP` 进行匹配
- 搜索结果因标签库扩充自然更加丰富

#### 3.2.4 智能推荐

- 保持现有推荐逻辑不变
- 推荐范围自动扩大（因新增了更多 Emoji 和标签映射）

## 6. 非功能性需求

### 6.1 性能要求

- Emoji 选择器 Modal 打开时间不超过 **200ms**
- 中文搜索响应时间不超过 **300ms**（现有 300ms debounce 机制保持不变）
- 分类切换无明显卡顿

### 6.2 兼容性要求

- 新增 Emoji 需在以下平台正常显示：
  - Chrome 90+、Firefox 90+、Safari 14+、Edge 90+
  - Windows 10+、macOS 11+、iOS 14+、Android 10+
- 优先使用 Unicode 12.0 及以下版本的 Emoji，确保最大兼容性
- 对于较新的 Emoji（Unicode 13.0+），需标注可能在旧系统上显示为方块的风险

### 6.3 安全要求

- 无安全相关变更（纯前端静态数据扩充）

---

## 7. 业务规则与约束

1. **已有分类不删不改**：原有 7 个分类的名称、顺序、内含 Emoji 保持不变
2. **新分类追加在末尾**：7 个新增分类按"科技数码 → 人物表情 → 交通工具 → 建筑地标 → 天气时间 → 办公学习 → 金融商务"顺序排列
3. **允许合理重复**：同一个 Emoji 可以出现在多个分类中（如 💻 既在"活动物品"也在"科技数码"），搜索时只返回一次（去重）
4. **标签库只增不删**：现有的 260+ 条标签映射不做修改，仅新增
5. **默认分类不变**：打开 Modal 默认展示"医疗健康"分类

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 新增 Emoji 在旧操作系统上显示为方块 | 前端正常显示（方块也是有效字符），不影响功能；建议优先使用兼容性好的 Emoji |
| 分类标签栏在窄屏幕上超出宽度 | 标签栏支持自动换行，确保所有分类可见 |
| 搜索结果过多（超过 50 个） | 保持现有逻辑，全部展示（用户可通过滚动查看） |
| 搜索无结果 | 保持现有提示（"未找到匹配的 Emoji"） |
