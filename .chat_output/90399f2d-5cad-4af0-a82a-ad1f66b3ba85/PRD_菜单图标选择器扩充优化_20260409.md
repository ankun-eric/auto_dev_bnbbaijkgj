# 菜单图标选择器扩充优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「首页配置 → 首页菜单管理」中，Emoji 图标选择器已具备以下能力：

- **7 大分类浏览**：医疗健康、运动健身、食物饮料、自然植物、活动物品、符号标志、动物
- **中文搜索**：基于 260+ 条 Emoji 中文标签映射，支持搜索框输入中文关键词查找 Emoji
- **智能推荐**：根据菜单名称自动匹配推荐 Emoji（40+ 组关键词映射）

存在的问题：

1. **分类覆盖面不足**：现有 7 个分类偏向医疗健康和生活场景，缺少「科技数码」「人物表情」「交通工具」「建筑地标」「天气时间」「办公学习」「金融商务」等常用分类
2. **Emoji 标签库不够丰富**：部分常用 Emoji（如 🤖 机器人）未被收录，搜索"机器人"无结果
3. **可选图标总量有限**：管理员在浏览和搜索时可选项不够多，有时找不到贴合需求的图标

本次优化旨在：**大幅扩充 Emoji 分类和中文标签映射库**，让管理员在选择菜单图标时有更丰富、更全面的选项。

### 1.2 目标用户

- 管理后台运营人员（管理员角色）

### 1.3 核心价值

- 新增 7 个图标分类，从 7 个扩充至 14 个，覆盖更多使用场景
- 大幅扩充 Emoji 中文标签映射库，从 260+ 条提升至 500+ 条
- 搜索"机器人""电脑""汽车""学校"等常见关键词均可命中对应 Emoji
- 已有菜单图标不受影响，向后兼容

## 6. 非功能性需求

### 6.1 性能要求

- 新增 Emoji 数据为前端静态常量，不增加网络请求
- 标签库从 260+ 条扩充至 500+ 条，搜索仍为前端内存过滤，响应时间 < 50ms
- 分类切换、Emoji 渲染无卡顿

### 6.2 安全要求

- 无新增安全风险（纯前端改造）

### 6.3 兼容性要求

- 确保新增 Emoji 在 Chrome、Edge、Firefox 主流浏览器中正确渲染
- 不同操作系统的 Emoji 展示风格可能不同（系统原生渲染），属于正常现象
- 部分较新的 Emoji（如 🛰️、🪙 等）在旧系统上可能显示为方框，属于系统限制

---

## 7. 业务规则与约束

1. 新增分类中的 Emoji 如果与现有分类有重复（如 📱💻 已在「活动物品」中），允许在多个分类中出现，方便用户从不同维度查找
2. 搜索逻辑保持不变：用户输入关键词 → 匹配 `EMOJI_LABEL_MAP` 中包含该关键词的所有 Emoji → 展示搜索结果
3. 智能推荐逻辑保持不变：根据菜单名称匹配 `EMOJI_KEYWORD_MAP` 中的关键词组
4. 分类标签栏中，新增的 7 个分类排列在现有 7 个分类之后
5. 已有菜单的 Emoji 图标保持不变，不受本次优化影响
