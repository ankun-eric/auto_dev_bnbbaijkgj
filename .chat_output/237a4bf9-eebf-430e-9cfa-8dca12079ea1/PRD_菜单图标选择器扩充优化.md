## 1. 需求背景

### 1.1 现状分析

当前管理后台「首页配置 → 首页菜单管理」中，Emoji 图标选择器已具备以下能力：

- **7 大分类浏览**：医疗健康、运动健身、食物饮料、自然植物、活动物品、符号标志、动物
- **中文关键词搜索**：基于 `EMOJI_LABEL_MAP` 的 260+ 条 Emoji 中文标签映射
- **智能推荐**：根据菜单名称通过 `EMOJI_KEYWORD_MAP`（40+ 组关键词）匹配推荐图标

### 1.2 存在问题

1. **Emoji 分类覆盖面不足**：现有 7 个分类缺少科技数码、人物表情、交通工具、建筑地标、天气时间、办公学习、金融商务等常见类别
2. **中文标签映射不够全面**：当前约 260 条标签，许多常用 Emoji 未收录（如 🤖 机器人、🎨 画板、🖼️ 图片等），导致搜索"机器人"等关键词无结果
3. **关键词映射组不够丰富**：`EMOJI_KEYWORD_MAP` 虽有 40+ 组，但新增分类的相关关键词尚未覆盖

### 1.3 核心目标

- **大幅扩充 Emoji 库**：将可选 Emoji 从现有约 260 个扩充至 500+ 个
- **新增 7 个图标分类**：让分类浏览覆盖更多使用场景
- **同步扩充中文标签映射**：确保新增的每个 Emoji 都有对应的中文搜索标签，提升搜索覆盖率
- **同步扩充关键词映射**：为新增分类补充智能推荐关键词组

## 4. 不做的事项（Scope Out）

| 不做项 | 原因 |
|--------|------|
| 不支持自定义图标上传 | 当前扩充 Emoji 库已能满足需求 |
| 不批量刷新已有菜单图标 | 已有菜单图标保持不变，避免影响线上展示 |
| 不修改搜索逻辑 | 现有搜索逻辑（中文关键词模糊匹配）无问题，只需扩充标签库即可 |
| 不修改智能推荐逻辑 | 现有推荐逻辑无问题，只需扩充关键词映射组即可 |
| 不修改 Emoji 选择面板 UI | 现有分类 Tab + 搜索 + 推荐的 UI 交互无需变更 |

## 7. 技术要点

### 7.1 修改文件

仅涉及 **1 个文件**：

```
admin-web/src/app/(admin)/home-menus/page.tsx
```

### 7.2 修改范围

| 修改点 | 位置 | 修改内容 |
|--------|------|---------|
| `EMOJI_KEYWORD_MAP` | 第 16-65 行附近 | 追加 10+ 组新关键词映射 |
| `EMOJI_CATEGORIES` | 第 86-159 行附近 | 追加 7 个新分类及对应 Emoji 数组 |
| `EMOJI_LABEL_MAP` | 第 163-536 行附近 | 追加 150-200 条新 Emoji 中文标签 |

### 7.3 注意事项

1. 新增的 Emoji 如果已存在于现有分类或标签中，无需重复添加
2. 部分 Emoji 可能在多个分类中出现（如 📱 同时在"科技数码"和"活动物品"中），这是允许的
3. 标签库中每个 Emoji 只需出现一次（`EMOJI_LABEL_MAP` 中 key 唯一）
4. 确保新增内容不破坏 TypeScript 类型定义

| 影响维度 | 评估 |
|---------|------|
| 前端包体积 | 增加约 10-15KB（纯文本常量数据），影响可忽略 |

| 搜索性能 | 标签库从 260 条增至 450+ 条，300ms 防抖 + 同步遍历，影响可忽略 |
| 后端接口 | 无任何后端修改，纯前端变更 |
| 其他页面 | 不影响任何其他页面 |
