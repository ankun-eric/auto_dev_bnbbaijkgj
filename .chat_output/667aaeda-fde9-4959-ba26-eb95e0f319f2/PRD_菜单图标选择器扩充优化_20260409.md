# 后台菜单管理 - 图标选择器扩充优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「首页配置 → 首页菜单管理」中，Emoji 图标选择器已经具备分类浏览（7 大分类）、中文关键词搜索（260+ 标签映射）和智能推荐功能。但在实际使用中发现：

1. **Emoji 品类覆盖不足**：现有 7 个分类（医疗健康、运动健身、食物饮料、自然植物、活动物品、符号标志、动物）偏向健康管理领域，缺少科技、人物、交通、建筑、天气、办公、金融等常见类别
2. **中文标签映射缺口**：部分常用 Emoji（如 🤖 机器人）未纳入标签库，导致搜索关键词匹配不到
3. **整体可选数量有限**：管理员在配置非健康类菜单（如"AI 助手"、"在线学习"、"订单管理"等）时找不到合适图标

本次优化旨在：大幅扩充 Emoji 分类和中文标签库，让管理员在选择菜单图标时有更丰富、更全面的选择。

### 1.2 目标用户

- 管理后台运营人员（管理员角色）

### 1.3 核心价值

- 将可选 Emoji 从 ~260 个提升至 **500+**，覆盖绝大多数常见使用场景
- 新增 7 个图标分类，分类总数从 7 个提升至 **14 个**，分类浏览更高效
- 同步扩充中文标签映射库至 **500+** 条，关键词搜索覆盖率大幅提升
- 扩充关键词映射组（`EMOJI_KEYWORD_MAP`），智能推荐匹配更精准

## 3. 页面/界面设计

### 3.1 页面结构与导航

本次优化不新增页面，仅修改现有 Emoji 选择器弹窗（`EmojiPickerModal` 组件）的内容数据。

### 3.2 现有界面调整说明

#### Emoji 选择器弹窗

**调整点：** 分类标签栏从 7 个增加到 14 个

- **现有行为**：分类标签栏横向排列 7 个标签（医疗健康、运动健身、食物饮料、自然植物、活动物品、符号标志、动物）
- **调整后**：分类标签栏横向排列 14 个标签，新增分类追加在末尾
- **交互方式**：保持不变（点击标签切换分类，网格浏览 Emoji）
- **布局适配**：标签栏可能需要支持横向滚动或换行显示，以容纳 14 个分类标签。建议采用**横向滚动 + 左右箭头**或**自动换行（flex-wrap）**方式处理

#### 搜索功能

- **逻辑不变**：保持现有 300ms debounce + `EMOJI_LABEL_MAP` 匹配逻辑
- **效果提升**：因为标签库扩充，相同关键词的搜索结果会自然增多

#### 智能推荐

- **逻辑不变**：保持 `getRecommendedEmojis` 函数现有逻辑
- **效果提升**：因为 `EMOJI_KEYWORD_MAP` 扩充了新关键词组，推荐匹配命中率会提升

## 6. 非功能性需求

### 6.1 性能要求

- Emoji 选择器弹窗打开时间应保持在 **200ms 以内**（数据量翻倍后注意渲染性能）
- 搜索响应仍保持 **300ms debounce**，搜索结果渲染应在 **100ms 以内**
- 如 Emoji 总量超过 500 导致渲染卡顿，可考虑对非当前分类进行懒加载

### 6.2 安全要求

- 无新增安全要求（纯前端静态数据变更）

### 6.3 兼容性要求

- 新增 Emoji 需确保在主流浏览器（Chrome 90+、Firefox 90+、Safari 14+、Edge 90+）中正确渲染
- 部分较新 Emoji（如 🛜 Wi-Fi 符号）在旧系统上可能显示为方块，选用时优先选取兼容性好的 Emoji
- 移动端浏览器（iOS Safari、Android Chrome）也需确保正常显示

## 7. 业务规则与约束

1. **纯增量原则**：只增加新数据，不修改或删除任何已有的分类、Emoji 或标签映射
2. **标签质量**：每个新增 Emoji 的中文标签应准确、常用，优先选择用户最可能搜索的关键词
3. **分类归属**：每个 Emoji 必须属于至少 1 个分类；跨分类重复出现是允许的
4. **去重考虑**：搜索结果如有重复 Emoji 需去重（现有逻辑已支持）

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 新增 Emoji 在用户系统上无法渲染（显示为方块） | 不影响功能，管理员可选择其他 Emoji 替代 |
| 分类标签栏超出一行显示 | 采用 flex-wrap 或横向滚动，确保所有分类可达 |
| 搜索无结果 | 保持现有行为——显示"没有找到匹配的 Emoji"提示 |
| 已有菜单使用了旧 Emoji | 完全不受影响，保持原样 |

## 10. 补充说明

### 不做的事项（明确排除）

- ❌ 不支持自定义图标上传
- ❌ 不做已有菜单图标的批量刷新/替换
- ❌ 不修改搜索算法（不做模糊匹配、拼音搜索等）
- ❌ 不增加"最近使用"或"常用图标"功能
- ❌ 不接入第三方图标库（如 iconfont、Ant Design Icons）

### 实现建议

1. 所有变更集中在 `admin-web/src/app/(admin)/home-menus/page.tsx` 文件
2. 分别扩充 `EMOJI_CATEGORIES`、`EMOJI_LABEL_MAP`、`EMOJI_KEYWORD_MAP` 三个数据结构
3. 分类标签栏 UI 需验证 14 个分类的显示效果，必要时调整布局
4. 建议在扩充后进行一次完整的搜索覆盖率测试，确保常见关键词都能搜到结果
