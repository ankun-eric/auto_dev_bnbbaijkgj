# PRD：用户端页面 UI 视觉一致性优化

- **版本**：v1.0
- **创建日期**：2026-04-19
- **产品负责人**：小白 AI（资深产品经理）
- **优先级**：P0 / 紧急（本周内上线）
- **涉及端**：H5 用户端、微信小程序、Flutter App（三端全覆盖）

---

## 一、需求背景

当前 bini-health 用户端各功能页顶栏样式不统一：
- 「AI 健康咨询」页已采用绿→青渐变顶栏 + 白色文字，视觉突出且有品牌感
- 其他核心功能页（报告解读、健康档案、用药参考、健康自查、中医养生、健康计划等）仍为白底黑字的默认导航栏，缺乏品牌识别度

另有 3 处体验细节问题影响用户感知：
- AI 对话气泡中 AI 头像未使用品牌 LOGO，弱化品牌露出
- 「我的」页顶部空白过多，信息密度低
- 首页"服务菜单—商品分类"字号偏小，中老年用户阅读吃力

本次优化目标：**以"AI 健康咨询"顶栏为视觉基准，统一 8 个核心功能页的品牌化顶栏样式，并修复 3 处体验细节。**

## 二、设计规范（视觉基准）

### 2.1 统一顶栏规范

取色来源：`h5-web/src/app/chat/[sessionId]/page.tsx` 第 1442 行

| 项 | 规范 |
|---|---|
| 背景色 | `linear-gradient(135deg, #52c41a, #13c2c2)`（绿→青渐变） |
| 高度 | `48px`（保持与 AI 咨询一致） |
| 下边框 | 无（`--border-bottom: none`） |
| 标题区布局 | **LOGO 图标 + 标题文字**，整体水平居中 |
| LOGO 样式 | 品牌主 LOGO 图标部分（不显示英文文字），约 `20×20px`，圆角 |
| LOGO 与标题间距 | `6px` |
| 标题文字 | 白色 `#fff`、字号 `16px`、字重 `500`（medium） |
| 左侧返回按钮 | 保留，白色图标；若原本有其他左侧按钮（如侧边栏）也保留，图标统一白色 |
| 右侧操作按钮 | 保留（分享/更多/字体设置等），图标统一白色 |
| 状态栏适配 | 顶栏使用 `safe-area-inset-top` 自动撑高，文字不穿透状态栏 |

### 2.2 AI 对话 AI 头像规范（仅对话气泡中的 AI 头像）

| 项 | 规范 |
|---|---|
| 形态 | 圆形白底 `#FFFFFF` |
| 内容 | 品牌 LOGO 图标居中（不裁切） |
| 尺寸 | 保持现有容器尺寸（28~32px 圆形），LOGO 内容占比约 70% |
| 描边 | `1px solid #E5E7EB`（浅灰，避免纯白背景与浅色气泡融在一起） |

### 2.3 「我的」页顶部留白

- 顶部空白区高度 **收紧约 30%**
- 实施方式：在当前顶部 padding 基础上 `× 0.7`，向下取整为 4 的倍数

### 2.4 首页"服务菜单—商品分类"字号

- 字号由当前 `12px` 调整为 **`14px`**
- 行高同步提升至 `20px`，避免多行拥挤

## 四、交互细节与边界

1. **深色状态栏文字**：顶栏变为深色渐变后，需将状态栏前景色设为白色（iOS `light-content`、Android status bar icons 白色）
2. **下拉刷新**：若页面有下拉刷新，loading 圆圈颜色改为白色，避免在绿底上看不见
3. **键盘弹起**：顶栏不受键盘影响，固定在顶部
4. **无 LOGO 降级**：LOGO 资源加载失败时，仅显示标题文字居中（保持布局不塌陷）
5. **分享页 / 无登录态**：共享报告、共享对话等无鉴权页同样套用品牌顶栏，但右上角操作按钮按原规则隐藏
6. **暗黑模式**：本次不支持暗黑模式特化，按亮色方案统一

## 七、风险与注意事项

1. **色值精确性**：绿→青渐变需严格使用 `#52C41A → #13C2C2`、角度 `135deg`，各端实现细节不同需视觉验收
2. **小程序自定义导航栏**：改为自定义后需自行处理胶囊按钮避让（右上角 87px 安全区）
3. **Flutter AppBar 高度**：Material 默认 56px，需显式设置 `preferredSize: Size.fromHeight(48)` 保持一致
4. **LOGO 素材**：需产品/设计提供适配白底圆形使用的 LOGO 图标版本（带透明背景的 PNG / SVG）
5. **回归范围**：涉及 9 个高频功能页，发布前务必完成全链路点击回归
