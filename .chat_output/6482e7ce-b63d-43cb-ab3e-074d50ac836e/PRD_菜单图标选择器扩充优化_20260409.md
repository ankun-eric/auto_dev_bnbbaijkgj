# 后台菜单管理 - 图标选择器扩充优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「首页配置 → 首页菜单管理」中已实现了一套 Emoji 图标选择器，支持 7 大分类浏览（医疗健康、运动健身、食物饮料、自然植物、活动物品、符号标志、动物）、中文关键词搜索（260+ Emoji 中文标签映射）和基于菜单名称的智能推荐功能。

但在实际使用中发现，**可选图标的覆盖范围仍然不够丰富**。例如搜索"机器人"时无法找到对应的 🤖 Emoji——因为当前的分类和标签库中未收录科技数码、人物表情、交通工具、建筑地标、天气时间、办公学习、金融商务等常见类别的 Emoji。

本次优化的核心目标：**大幅扩充 Emoji 图标库和中文标签映射，新增多个图标分类**，让管理员在设置菜单图标时拥有更全面、更丰富的选择。

### 1.2 目标用户

- 管理后台运营人员（管理员角色）

### 1.3 核心价值

- 将 Emoji 分类从当前 7 个扩充至 **14 个**，覆盖更广泛的使用场景
- 大幅扩充 Emoji 中文标签映射库（从 260+ 扩充至 **500+**），搜索覆盖率显著提升
- 同步扩充关键词推荐映射，让智能推荐匹配更多菜单名称
- 已有菜单的 Emoji 图标保持不变，零影响
- 不支持自定义图标上传，保持功能简洁

## 6. 非功能性需求

### 6.1 性能要求

- 扩充后的 Emoji 常量数据量约增加一倍，但总量仍在 KB 级别，对页面加载性能无可感知影响
- 搜索仍采用 300ms debounce + 内存遍历，500+ 条映射的搜索耗时仍在 1ms 内

### 6.2 安全要求

- 无新增安全风险（纯前端静态数据扩充）

### 6.3 兼容性要求

- 新增的 Emoji 字符需确保在主流浏览器（Chrome 90+、Safari 14+、Firefox 90+、Edge 90+）中正确显示
- 部分较新的 Emoji（如 Emoji 14.0+ 字符）在旧系统中可能显示为方块，但这是系统层面问题，不影响数据存储和功能使用
- 移动端 WebView 同样需要验证 Emoji 渲染效果

## 7. 业务规则与约束

| 编号 | 规则 | 说明 |
|------|------|------|
| R01 | 已有菜单图标不受影响 | 已设置的 Emoji 图标保持不变，不做任何迁移或刷新 |
| R02 | 搜索逻辑不变 | 保持现有的 `includes` 模糊匹配逻辑，仅因标签库扩充而自然增加搜索覆盖率 |
| R03 | 推荐逻辑不变 | 保持现有的关键词匹配 + 最多 8 个推荐的逻辑，仅扩充关键词组 |
| R04 | 不支持自定义图标上传 | 本期不做自定义图标上传功能 |
| R05 | 不做已有图标批量刷新 | 不提供一键替换已有菜单图标的功能 |
| R06 | Emoji 可跨分类存在 | 同一 Emoji 允许出现在多个分类中 |

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|---------|
| 用户搜索新增 Emoji 对应的关键词但未找到 | 不应发生——所有新增 Emoji 都配置了中文标签。若发现遗漏，后续补充标签即可 |
| 新增 Emoji 在某些旧设备上显示为方块 | 不影响功能使用，图标数据正常存储。管理员可选择其他能正常显示的 Emoji |
| 14 个分类 Tab 在窄屏幕上换行过多 | Modal 宽度为 520px（固定），分类 Tab 使用 flexWrap 自适应换行，极端情况下可能换 3 行，不影响使用 |
| 搜索结果过多 | 保持现有逻辑——搜索结果直接全部展示在网格中，用户滚动查看即可 |
