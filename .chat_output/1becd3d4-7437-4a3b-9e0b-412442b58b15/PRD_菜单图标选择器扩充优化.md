# 后台菜单管理 - 图标选择器扩充优化 产品需求文档（PRD）

> 版本：v1.0
> 日期：2026-04-09
> 项目：bini-health 管理后台
> 需求类型：功能增强

---

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「首页配置 → 首页菜单管理」中，Emoji 图标选择器已具备分类浏览、中文搜索、智能推荐等核心能力。但在实际使用过程中，管理员反馈**可选图标种类不够丰富**——例如搜索"机器人"无法找到对应的 Emoji（🤖），说明当前的 Emoji 收录和中文标签映射尚有大量常用场景未覆盖。

**本次优化聚焦一个核心目标：大幅扩充 Emoji 图标库和中文标签映射，让管理员在设置菜单图标时有更丰富、更全面的选择。**

### 1.2 目标用户

- 管理后台运营人员（管理员角色）

### 1.3 核心价值

- 将 Emoji 分类从现有 **7 个**扩充至 **14 个**，新增科技数码、人物表情、交通工具、建筑地标、天气时间、办公学习、金融商务共 7 个分类
- 将中文标签映射库从现有 **260+ 个** Emoji 扩充至 **500+ 个**，中文搜索覆盖面大幅提升
- 搜索"机器人"、"电脑"、"汽车"、"学校"、"太阳"、"钱"等常用关键词都能找到对应图标
- 不改变现有的交互方式和搜索逻辑，零学习成本

### 1.4 需求范围

| 做什么 | 不做什么 |
|--------|----------|
| 扩充 Emoji 中文标签映射库 | ~~不支持自定义图标上传~~ |
| 新增 7 个图标分类 | ~~不做已有菜单图标的批量刷新~~ |
| 补充各分类下的 Emoji 数量 | ~~不修改搜索逻辑和交互方式~~ |
| 保持已有菜单图标不受影响 | ~~不涉及后端/数据库改动~~ |

## 3. 页面/界面设计

### 3.1 变更范围

本次优化**不改变现有的交互方式和界面布局**，仅在以下方面有变化：

#### 3.1.1 分类标签栏调整

**变更前**：7 个分类标签，一行可以完整显示

```
[医疗健康] [运动健身] [食物饮料] [自然植物] [活动物品] [符号标志] [动物]
```

**变更后**：14 个分类标签，需要支持横向滚动或自动换行

```
[医疗健康] [运动健身] [食物饮料] [自然植物] [活动物品] [符号标志] [动物]
[科技数码] [人物表情] [交通工具] [建筑地标] [天气时间] [办公学习] [金融商务]
```

**建议方案**：分类标签采用 `flex-wrap: wrap` 自动换行布局，间距 4~8px，确保所有分类一目了然。

#### 3.1.2 Emoji 网格区域

- 每个分类下的 Emoji 数量增加，但因为已有滚动支持（最大高度 300px），无需额外调整
- 网格布局保持不变（10 列，36x36px）

### 3.2 页面结构

无新增页面。仅修改 `admin-web/src/app/(admin)/home-menus/page.tsx` 内的常量数据和分类标签栏样式。

## 6. 非功能性需求

### 6.1 性能要求

- 新增的 Emoji 数据仍为前端静态常量，不增加额外网络请求
- 标签库从 260 扩展到 500+，搜索仍为前端内存过滤，响应时间仍 < 50ms
- 分类切换、Emoji 网格渲染无性能影响（每个分类 20~30 个 Emoji，远低于需要虚拟滚动的阈值）

### 6.2 安全要求

- 无新增安全风险（纯前端改造）

### 6.3 兼容性要求

- 新增的 Emoji 字符需确保在 Chrome 80+、Edge 80+、Firefox 78+ 中正确渲染
- 部分较新的 Emoji（如 🧮🪙🧾）需要 Emoji 14.0+ 支持，在极旧系统上可能显示为方块——属于可接受的降级
- 已有菜单数据完全不受影响

---

## 7. 业务规则与约束

1. **向后兼容**：已有菜单的 Emoji 图标保持不变，不受影响
2. **交互不变**：搜索逻辑、智能推荐、选择面板的交互方式均保持现状
3. **分类可重复**：同一个 Emoji 可以出现在多个分类中（如 📱 同时在"活动物品"和"科技数码"）
4. **标签完整性**：所有新增 Emoji 必须配上至少 2 个中文搜索标签，确保中文搜索可达
5. **分类顺序**：新增 7 个分类排列在现有 7 个分类之后，保持现有分类的顺序不变

| 场景 | 处理方式 |
|------|----------|
