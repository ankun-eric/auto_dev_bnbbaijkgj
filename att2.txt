“宾尼小康”AI健康管家APP - 完整需求分析文档

## 1. 项目概述

### 1.1 产品名称
宾尼小康

### 1.2 产品定位
一款基于人工智能的个人及家庭健康管理助手，通过AI对话、报告解读、健康监测与提醒等功能，为用户提供便捷、专业、个性化的健康服务。**通过“健康积分”激励用户坚持健康行为（按时吃药、坚持锻炼等），积分可直接抵扣消费金额**，形成“健康行为→健康积分→消费激励”的正向循环。

### 1.3 目标用户
- **普通用户**：有日常健康咨询、体检报告解读、健康管理需求的个人。
- **家庭用户**：需要管理家庭成员（如老人、小孩）健康数据的用户。
- **亚健康/慢病人群**：需要长期监测指标、用药提醒和健康指导的用户。
- **健康服务消费者**：有口腔、体检、专家、养老等健康服务需求的用户。
- **门店/服务商核销员**：需要处理线下订单核销的工作人员。

### 1.4 产品核心价值
1. **AI健康助手**：7×24小时专业健康咨询，精准推荐服务。
2. **健康数据管理**：家庭成员健康档案、设备同步、报告解读。
3. **健康行为激励**：通过健康积分激励用户坚持用药、锻炼等健康习惯。
4. **一站式健康服务**：健康食品、口腔、体检、专家、养老等全品类服务。
5. **线上线下闭环**：线上购买、线下核销，无缝履约。

## 3. 详细功能说明

### 3.1 模块一：AI健康咨询

#### 3.1.1 对话交互
- **欢迎语**：首次进入对话页面，展示欢迎语（如：“你好，我是你的AI健康管家小康，有什么可以帮你的吗？”）。
- **再次进入提示语**：非首次进入时，展示简短提示语（如：“Hi，今天感觉如何？”或上次未完成的对话提示）。
- **多轮对话**：支持上下文理解，精准识别用户意图，能够根据连续对话进行深入分析。

#### 3.1.2 商品/服务卡片推荐
- **卡片挂载机制**：AI根据用户提问，在回复内容下方自动推荐相关商品或服务卡片。
- **精准推荐逻辑**：
  - 关键词匹配 + 意图识别
  - 地理位置推荐：获取用户位置权限后，优先推荐所在城市的商品/服务
  - 示例：用户问“口腔有问题，是否有专家推荐？”仅推荐口腔科专家
- **卡片内容**：标题、简短描述、价格、标签、行动按钮（“查看详情”、“立即预约”）。

#### 3.1.3 预约流程
用户点击卡片后触发：
- **留资预约**：填写姓名、电话等，提交后由客服联系。
- **在线支付预约**：选择服务时间、确认订单后进入支付流程（支持微信/支付宝）。

#### 3.1.4 对话页面快捷按钮
在输入框上方/下方配置功能入口，每个按钮前带小图标：
- 拍报告 📷：唤起相机/相册，上传报告进行解读
- 打电话 📞：拨打客服热线
- 我的订单 📄：跳转至订单页面
- 健康档案 👤：跳转至健康档案
- 提醒设置 ⏰：跳转至提醒设置页

#### 3.1.5 对话记录管理
- **历史记录入口**：对话页面顶部“历史记录”图标。
- **记录列表**：按时间倒序展示会话，显示标题、时间、摘要。
- **搜索**：支持关键词搜索历史对话。
- **继续对话**：点击记录进入详情，AI能基于上下文继续回答。
- **删除**：支持左滑单条删除、批量删除，删除前确认。

#### 3.1.6 语音输入（按住说话）
- **按钮**：输入框右侧麦克风图标，点击后展开“按住说话”按钮。
- **录音状态**：
  - 未按下：按钮文字“按住说话”
  - 按下时：按钮变为红色录音状态，显示“松手发送”，有音量波形动画
  - 上滑取消：手指上滑离开按钮区域，提示“松手取消”
- **语音转文字**：松开后自动转文字，用户可编辑后发送。
- **权限管理**：首次使用请求麦克风权限，拒绝时提示并引导设置。
- **异常处理**：录音失败、网络异常、语音过短/过长均有对应提示。

#### 3.1.7 回答消息交互按钮
每条AI回答底部右侧显示三个小图标，顺序为：
- **复制** 📋：点击复制回答内容到剪贴板，提示“已复制到剪贴板”。
- **分享** ↗️：点击弹出分享面板，提供两种方式：
  - 分享到微信：生成图文卡片，发送给微信好友或群聊
  - 复制链接：生成专属链接，复制后可粘贴到微信、朋友圈等分享
- **语音播报** 🔊：点击将回答文本转为语音播放（TTS），再次点击停止；支持后台播放、设置语速音调

**分享页面**：包含对话内容、推荐卡片、底部引导下载APP按钮，已安装APP可唤起。

#### 3.1.8 相关提问建议
- **功能**：每次AI回答后，自动展示N条相关提问建议（默认N=3，后台可配置1-5）。
- **展示位置**：交互按钮下方，以标签组形式呈现。
- **点击行为**：点击建议标签，自动填充并发送问题，AI继续回答。

### 3.4 模块四：健康提醒服务

健康提醒服务不仅是提醒工具，更是**健康积分的核心获取渠道**。用户完成健康行为后获得健康积分。

#### 3.4.1 提醒类型与积分奖励
| 提醒类型 | 完成行为 | 积分奖励规则 | 说明 |
|----------|----------|--------------|------|
| **用药提醒** | 用户点击“我已服药”确认 | 每次获得X积分（后台可配置） | 鼓励按时服药，培养依从性 |
| **锻炼计划提醒** | 用户点击“已完成锻炼”确认 | 每次获得Y积分（后台可配置） | 鼓励坚持运动，可设置单次或累计奖励 |
| **签到提醒** | 每日点击签到 | 签到获得Z积分，连续签到递增 | 培养每日打开APP习惯 |
| **健康管理提醒** | 如“喝水提醒”点击确认 | 每次获得少量积分 | 鼓励健康生活习惯 |
| **自定义提醒** | 用户自定义提醒并确认完成 | 可设置积分奖励（可选） | 灵活满足个性化需求 |

#### 3.4.2 完成确认交互
- 每个提醒推送通知，点击后进入提醒详情页。
- 页面显示提醒内容，底部设置“已完成”按钮。
- 点击“已完成”后，系统记录完成行为，自动发放对应健康积分。
- 用户可在【健康积分明细】中查看积分获取记录。

#### 3.4.3 积分发放规则（后台可配置）
| 配置项 | 说明 | 默认值示例 |
|--------|------|------------|
| 用药提醒单次积分 | 每次确认用药获得 | 10积分 |
| 锻炼计划单次积分 | 每次确认锻炼获得 | 20积分 |
| 签到单次积分 | 每日签到获得 | 5积分 |
| 连续签到奖励 | 连续7天额外奖励 | 50积分 |
| 喝水提醒单次积分 | 每次确认获得 | 2积分 |
| 每日积分上限 | 单日最多获得的健康积分 | 100积分 |

| | 健康咖啡 | 功能性咖啡、低因咖啡、添加保健成分的咖啡饮品等 | 到店自提 / 外卖配送 |

| **养老服务** | 居家照护 | 上门护理、家政服务、康复理疗等居家养老服务 | 上门服务核销 |
| | 住院陪护 | 医院内陪护服务，包括日常照料、病情观察、陪检等 | 医院现场核销 |
| | 适老化改造 | 居家环境适老化评估与改造，如扶手安装、防滑处理等 | 上门服务核销（可分阶段） |
| | 养老规划 | 养老财务规划、养老生活规划、养老政策咨询等 | 线上/线下咨询核销（人工） |
| | 养老社区对接 | 养老社区参观预约、入住咨询 | 人工跟进，无需核销 |

#### 3.5.2 服务展示与筛选
| 功能点 | 描述 |
|--------|------|
| **分类导航** | 在服务预约页面顶部或左侧，展示服务分类标签：健康食品、口腔、体检、专家、养老等 |
| **健康食品二级** | 健康食品、健康咖啡 |
| **养老服务二级** | 居家照护、住院陪护、适老化改造、养老规划、养老社区对接 |

| **筛选排序** | 按价格、距离、销量、评分 |
| **地理位置推荐** | 优先推荐附近门店；上门服务推荐覆盖区域；养老社区推荐所在城市及周边 |

#### 3.5.3 购买与核销流程（通用）
1. 用户浏览/搜索服务，进入详情页。
2. 查看介绍、价格、门店信息、用户评价。
3. 选择规格/数量/预约时间（预约类服务）。
4. 进入订单确认页，选择健康积分抵扣、优惠券，完成支付。
5. 支付成功生成核销二维码，订单状态“待核销”。
6. 到店/上门时，用户出示二维码，核销员扫码核销。
7. 订单状态变为“已完成”。

#### 3.5.4 各服务特殊处理
| 服务类型 | 特殊说明 |
|----------|----------|
| **健康食品/健康咖啡** | 支持自提或快递/外卖（需运费），配送订单无需核销 |
| **口腔/体检/专家** | 预约类，需选择具体时段，支付后生成二维码 |
| **居家照护/住院陪护** | 预约类，需选择日期、时长、人员要求，支持按时计价，上门核销 |
| **适老化改造** | 预约评估，可设置定金+尾款，改造完成核销 |
| **养老规划** | 咨询类，可免费或收费，预约后由规划师确认，咨询完成人工标记完成 |
| **养老社区对接** | 用户提交参观预约（姓名、电话、人数、意向时间、需求），客服人工联系确认，参观后标记完成。不强制系统核销，可选择性生成二维码用于到访记录 |

### 3.7 模块七：健康积分体系

#### 3.7.1 健康积分定义
- **核心理念**：健康积分是用户通过坚持健康行为（按时吃药、坚持锻炼、每日签到等）获得的奖励，体现用户的健康自律。
- **积分价值**：健康积分可直接用于购买商品/服务时抵扣支付金额，形成“健康行为→健康积分→消费激励”的正向循环。

#### 3.7.2 健康积分获取渠道
| 获取渠道 | 说明 |
|----------|------|
| **用药提醒完成** | 按时服药确认后获得 |
| **锻炼计划完成** | 完成锻炼计划确认后获得 |
| **每日签到** | 签到后获得 |
| **健康管理提醒完成** | 如喝水、久坐提醒确认后获得 |
| **自定义提醒完成** | 用户自定义提醒确认后获得 |
| **注册送积分** | 新用户注册自动赠送（健康积分启动奖励） |
| **任务奖励** | 如完成报告解读、邀请好友等额外任务奖励 |
| **第三方兑换** | 银行等第三方平台积分兑换为本平台健康积分 |

#### 3.7.3 健康积分抵扣规则（后台配置）
| 配置项 | 说明 | 示例 |
|--------|------|------|
| **抵扣比例** | 多少健康积分抵扣1元 | 100积分 = 1元 |
| **单笔订单抵扣上限** | 最高抵扣金额或比例 | 最高抵扣订单金额的30% |
| **健康积分有效期** | 积分自获取之日起有效期 | 365天有效 |
| **适用商品范围** | 支持积分抵扣的商品/服务类型 | 全部商品 / 指定分类 |
| **用户等级差异化** | 不同等级用户抵扣比例不同（可选） | VIP用户：80积分=1元 |

#### 3.7.4 用户端展示
| 功能点 | 描述 |
|--------|------|
| **健康积分余额** | 在【我的】页面顶部显著位置展示，显示“健康积分：XXX”，带有积分图标（🏆或❤️） |
| **积分明细** | 点击进入明细页，展示积分获取和使用记录，每条记录注明来源（如“按时用药+10分”、“锻炼打卡+20分”、“订单抵扣-50分”） |
| **积分过期提醒** | 积分即将过期时推送通知，引导用户使用 |

#### 3.7.5 下单抵扣流程
1. 用户在订单确认页面，系统自动计算可用健康积分及最高抵扣金额。
2. 显示“健康积分抵扣”选项，展示当前积分可抵扣金额。
3. 用户勾选使用积分，可手动调整抵扣金额（不超过上限）。
4. 系统实时计算应付金额 = 订单总价 - 积分抵扣金额。
5. 用户确认支付，系统扣除相应健康积分。
6. 支付成功，订单生成，积分明细记录扣除记录。

#### 3.7.6 第三方积分兑换接口
- **功能**：供第三方系统（如工商银行）调用，将外部积分兑换为本平台健康积分。
- **接口参数**：用户手机号、兑换积分数量。
- **流程**：第三方发起请求，系统校验后增加用户健康积分，返回结果。
- **安全认证**：API Key + 签名。

## 8. 业务流程总览

### 8.1 用户端核心流程

```
用户注册/登录
    ↓
[AI健康咨询] ←→ [健康提醒服务] → 完成提醒 → 获得健康积分
    ↓                                              ↓
[报告解读]                                       [服务预约]
    ↓                                              ↓
[健康档案管理] ← [健康设备连接]              [我的订单] → 使用积分抵扣 → 支付
                                                      ↓
                                                 生成核销二维码
                                                      ↓
                                                 到店/上门核销
                                                      ↓
                                                 订单完成
```

### 8.2 健康积分激励闭环

```
健康提醒推送
    ↓
用户完成健康行为（按时吃药、坚持锻炼等）
    ↓
获得健康积分
    ↓
积分积累
    ↓
购买健康服务时使用积分抵扣
    ↓
享受服务，保持健康
    ↓
继续收到健康提醒（正向循环）
```

## 9. 总结

本需求分析文档V17.0完整定义了“宾尼小康”AI健康管家APP的全部功能，核心特点如下：

1. **AI健康咨询**：智能对话、精准推荐、语音输入、历史记录、分享传播、相关提问建议。
2. **健康数据管理**：报告解读、档案管理、设备连接、AI初筛报告。
3. **健康行为激励**：健康提醒完成获得“健康积分”，积分可直接抵扣消费，形成正向激励闭环。
4. **一站式健康服务**：健康食品、口腔、体检、专家、养老服务（含居家照护、住院陪护、适老化改造、养老规划、养老社区对接），线上购买、线下核销。
5. **完整商业闭环**：后台管理、核销端、订单管理、积分体系全覆盖。
6. **开放能力**：第三方积分兑换接口、AI初筛报告推送接口。
