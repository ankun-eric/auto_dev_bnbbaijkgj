# OCR全局设置拆分 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「AI管理 → OCR识别配置」页面中，每个 OCR 厂商 Tab 下包含了 4 个配置区块：厂商配置、成功率统计、AI 处理配置（场景模板）、上传限制配置。其中"AI 处理配置"和"上传限制配置"是全局参数，不属于某个 OCR 渠道的专属设置，放在 OCR 厂商 Tab 下不够合理。

本次需求将这两块配置拆出来，在「AI管理」菜单组下新增一个独立的"OCR全局设置"页面进行管理，同时对"AI 处理配置（场景模板）"进行简化重构——去掉 OCR 厂商偏好字段、将提示词从提示词配置表迁移后内嵌到场景模板中，使每个场景只需管理"场景名称 + AI 提示词"两个字段。

### 1.2 目标用户

管理后台管理员

### 1.3 核心价值

- 让配置结构更加合理——全局参数归全局页面，厂商配置归厂商页面
- 简化场景模板管理流程——从"跨两个页面操作"简化为"一个表单完成"
- 降低管理员的认知负担——去掉不常用的 OCR 厂商偏好字段，减少概念理解成本

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 新增"OCR全局设置"页面 | 在「AI管理」菜单组下新增子菜单 | P0 | 页面包含两个 Tab |
| F02 | 场景模板管理 Tab | 场景模板的 CRUD 操作 | P0 | 原"AI处理配置"拆出 + 简化 |
| F03 | 上传限制 Tab | 上传限制配置的查看/编辑 | P0 | 原"上传限制配置"拆出，功能不变 |
| F04 | 场景模板简化 | 去掉 ocr_provider 字段，提示词内嵌 | P0 | 每个场景只有"名称 + 提示词" |
| F05 | 数据迁移 | 提示词迁移 + 清理 | P0 | 提示词配置表中 OCR 记录迁入模板表 |
| F06 | 原"OCR识别配置"页面改造 | 移除场景模板和上传限制区块 | P0 | 只保留厂商配置 + 统计 + 测试 |
| F07 | OCR 厂商自动选择策略调整 | 全部走自动选择 | P0 | 不再支持场景级别指定厂商 |

### 2.2 功能详细描述

#### F01：新增"OCR全局设置"页面

- 在管理后台左侧导航菜单「AI管理」组下，新增 **"OCR全局设置"** 子菜单项
- 菜单位置：排在"OCR识别配置"之后
- 点击后进入一个包含两个 Tab 的页面

#### F02：场景模板管理 Tab

- Tab 名称：**场景模板管理**
- 展示形式：卡片列表（保持现有风格）
- 每个场景卡片显示：
  - 场景名称
  - AI 提示词摘要（截断显示，鼠标悬浮可看完整内容）
  - 操作按钮：编辑、删除（预设场景不可删除）
- 新建/编辑场景弹窗包含：
  - 场景名称（必填，不可重复）
  - AI 提示词（必填，普通多行文本输入框 TextArea）
- 预设场景（"体检报告识别"、"拍照识药"）不可删除，但可编辑提示词

#### F03：上传限制 Tab

- Tab 名称：**上传限制**
- 功能与现有完全相同：
  - 单次最多上传张数（max_batch_count）
  - 单张图片最大尺寸（max_file_size_mb，单位 MB）
- 表单展示，底部保存按钮

#### F04：场景模板简化

**需移除的字段：**
- `ocr_provider`（OCR 厂商偏好）— 所有场景统一走自动选择策略
- `ai_model_id`（AI 模型 ID）— 不再支持场景级别指定 AI 模型

**需新增的字段：**
- `prompt_content`（AI 提示词）— 虽然模型已有此字段，但原先由提示词配置表（AiPromptConfig）管理，现在改为直接在场景模板表中管理

**简化后的场景模板字段：**
- `id`（主键）
- `scene_name`（场景名称）
- `prompt_content`（AI 提示词）
- `is_preset`（是否预设场景）
- `created_at`（创建时间）
- `updated_at`（更新时间）

#### F05：数据迁移

启动时执行一次性数据迁移（幂等）：

1. 读取 `ai_prompt_configs` 表中 `chat_type` 为 `ocr_checkup_report` 的记录，将其 `system_prompt` 写入 `ocr_scene_templates` 表中"体检报告识别"场景的 `prompt_content` 字段
2. 读取 `ai_prompt_configs` 表中 `chat_type` 为 `ocr_drug_identify` 的记录，将其 `system_prompt` 写入 `ocr_scene_templates` 表中"拍照识药"场景的 `prompt_content` 字段
3. 对于自定义场景（`ocr_custom_{id}` 格式），同样将对应 `AiPromptConfig.system_prompt` 迁移到场景模板的 `prompt_content` 字段
4. 迁移完成后，删除 `ai_prompt_configs` 表中所有 `chat_type` 以 `ocr_` 开头的记录
5. 在「AI配置中心 → 提示词配置」页面中移除 `ocr_checkup_report` 和 `ocr_drug_identify` 两个 Tab

#### F06：原"OCR识别配置"页面改造

拆走场景模板和上传限制后，原页面只保留：
- 厂商配置（百度/腾讯/阿里云 Tab，每个 Tab 下的密钥配置 + 启用/禁用）
- 成功率统计（每个厂商的调用统计数据）
- 测试 OCR（上传图片测试 OCR 识别结果）
- 测试完整流程（上传图片测试 OCR + AI 完整链路）

#### F07：OCR 厂商自动选择策略调整

- 所有场景模板统一使用自动选择策略：先用首选厂商，失败后自动切换到其他可用厂商
- `_call_ai_with_scene` 函数不再读取场景模板的 `ocr_provider` 字段
- 后端 recognize/batch-recognize 接口中 OCR 厂商选择逻辑不再参考场景模板

## 3. 页面/界面设计

### 3.1 页面结构与导航

管理后台左侧菜单变更：

```
AI管理
├── AI模型配置
├── AI对话记录
├── 知识库管理
├── 检索策略配置
├── 兜底策略配置
├── OCR识别配置          ← 移除场景模板和上传限制区块
├── OCR全局设置          ← 新增
├── 体检报告解读明细
└── 拍照识药记录明细
```

### 3.2 各页面功能说明

#### "OCR全局设置"页面

- 路由：`/ocr-global-config`
- 顶部标题：OCR全局设置
- 两个 Tab：
  - **场景模板管理**（默认 Tab）
    - 卡片列表展示所有场景模板
    - 右上角"新增场景"按钮
    - 每张卡片显示场景名称、提示词摘要、操作按钮
    - 点击"编辑"弹出 Modal，包含场景名称输入框和 AI 提示词 TextArea
    - 预设场景的删除按钮禁用
  - **上传限制**
    - 表单展示：单次上传张数（InputNumber）、单张大小上限（InputNumber，单位 MB）
    - 底部"保存"按钮

#### "OCR识别配置"页面（改造后）

- 移除原有的"AI处理配置"卡片区块
- 移除原有的"上传限制配置"卡片区块
- 保留：厂商 Tab（百度/腾讯/阿里云）、成功率统计、测试 OCR、测试完整流程

## 4. 数据模型设计

### 4.1 数据实体与字段

#### `ocr_scene_templates` 表（修改）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, 自增 | 主键 |
| scene_name | String(100) | UNIQUE, NOT NULL | 场景名称 |
| prompt_content | Text | NULLABLE | AI 提示词（迁移后直接在此管理） |
| is_preset | Boolean | DEFAULT FALSE | 是否预设场景 |
| created_at | DateTime | DEFAULT NOW | 创建时间 |
| updated_at | DateTime | DEFAULT NOW, ON UPDATE | 更新时间 |

**移除的字段：**
- `ai_model_id`（Integer, FK） — 不再支持场景级别指定 AI 模型
- `ocr_provider`（String(50)） — 不再支持场景级别指定厂商

#### `ocr_upload_configs` 表（不变）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, 自增 | 主键 |
| max_batch_count | Integer | DEFAULT 5 | 单次最多上传张数 |
| max_file_size_mb | Integer | DEFAULT 5 | 单张图片最大尺寸(MB) |
| updated_at | DateTime | DEFAULT NOW, ON UPDATE | 更新时间 |

#### `ai_prompt_configs` 表（清理 OCR 相关记录）

迁移完成后，删除以下 `chat_type` 的记录：
- `ocr_checkup_report`
- `ocr_drug_identify`
- `ocr_custom_{id}` 格式的所有记录

### 4.2 实体关系

- `OcrSceneTemplate` 与 `AiPromptConfig` 的关联关系解除
- `OcrSceneTemplate` 与 `AIModelConfig` 的外键关系移除（`ai_model` relationship 删除）
- `OcrSceneTemplate` 成为独立实体，自包含所有场景配置信息

## 5. 接口需求

### 5.1 接口清单

现有接口迁移到新路由（或保持原路由不变，仅调整请求/响应结构）：

| 编号 | 方法 | 路径 | 说明 | 请求参数 | 返回数据 |
|------|------|------|------|----------|----------|
| A01 | GET | /api/admin/ocr/scenes | 获取所有场景模板 | 无 | `[{id, scene_name, prompt_content, is_preset, created_at, updated_at}]` |
| A02 | POST | /api/admin/ocr/scenes | 创建新场景 | `{scene_name, prompt_content}` | 新建的场景模板对象 |
| A03 | PUT | /api/admin/ocr/scenes/{id} | 更新场景 | `{scene_name?, prompt_content?}` | 更新后的场景模板对象 |
| A04 | DELETE | /api/admin/ocr/scenes/{id} | 删除场景 | 无 | `{detail: "删除成功"}` |
| A05 | GET | /api/admin/ocr/upload-limits | 获取上传限制 | 无 | `{id, max_batch_count, max_file_size_mb, updated_at}` |
| A06 | PUT | /api/admin/ocr/upload-limits | 更新上传限制 | `{max_batch_count?, max_file_size_mb?}` | 更新后的上传限制对象 |

**接口变更说明：**
- A02（创建场景）：请求体新增 `prompt_content` 字段，移除 `ai_model_id` 和 `ocr_provider`；不再自动在 AiPromptConfig 表中创建关联记录
- A03（更新场景）：请求体新增 `prompt_content` 字段，移除 `ai_model_id` 和 `ocr_provider`
- A01（列表）：响应新增 `prompt_content` 字段，移除 `ai_model_id` 和 `ocr_provider`
- A05/A06（上传限制）：接口路径和功能完全不变

## 6. 非功能性需求

### 6.1 性能要求

- 数据迁移脚本在应用启动时执行，需在 5 秒内完成
- 页面加载时间与现有 OCR 配置页面一致

### 6.2 安全要求

- 所有接口保持 admin 角色鉴权（`require_role("admin")`）
- 与现有 OCR 识别配置权限一致

### 6.3 兼容性要求

- 数据迁移脚本需幂等，多次执行不产生副作用
- 迁移后后端 `_call_ai_with_scene` 函数需优先从场景模板的 `prompt_content` 读取提示词，不再查询 AiPromptConfig 表

---

## 7. 业务规则与约束

1. 场景名称全局唯一，不可重复
2. 预设场景（`is_preset=true`）不可删除，但可编辑提示词
3. 去掉 `ocr_provider` 后，所有场景的 OCR 厂商选择统一走自动策略（首选厂商优先 → 成功率自动切换）
4. 提示词迁移后，创建新场景不再在 `AiPromptConfig` 表中生成关联记录
5. `_call_ai_with_scene` 函数直接使用场景模板的 `prompt_content`，不再查询 AiPromptConfig 表
6. 如果 `prompt_content` 为空，使用默认提示词："请根据以下OCR文字内容进行结构化整理，返回JSON格式。"

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 创建场景时名称重复 | 返回 400，提示"场景名称已存在" |
| 删除预设场景 | 返回 400，提示"预设场景不可删除" |
| 编辑/删除不存在的场景 | 返回 404，提示"场景不存在" |
| 数据迁移时 AiPromptConfig 中无对应记录 | 跳过迁移，保持场景模板 prompt_content 现有值或为空 |
| prompt_content 为空时调用 AI | 使用默认提示词兜底 |
| 上传限制配置表无记录 | 自动创建默认记录（max_batch_count=5, max_file_size_mb=5） |

## 10. 补充说明

### 10.1 改造影响范围

| 模块 | 影响 |
|------|------|
| 后端模型（OcrSceneTemplate） | 移除 `ocr_provider`、`ai_model_id` 字段和 `ai_model` 关系 |
| 后端 Schema（ocr.py） | 移除 `ocr_provider`、`ai_model_id`，新增 `prompt_content` 字段 |
| 后端 API（ocr.py） | 修改场景 CRUD 接口，修改 `_call_ai_with_scene` 函数逻辑 |
| 后端初始化（init_data.py） | 新增数据迁移函数，移除 `_init_ocr_prompt_configs` |
| 后端 schema_sync | 新增列移除和数据迁移的同步函数 |
| 管理后台 layout | 新增"OCR全局设置"菜单项 |
| 管理后台 OCR 配置页面 | 移除场景模板和上传限制区块 |
| 管理后台新页面 | 新增"OCR全局设置"页面 |
| 管理后台提示词配置页面 | 移除 `ocr_checkup_report` 和 `ocr_drug_identify` Tab |
| C端（H5/小程序/App） | 无影响，不涉及前端改动 |

### 10.2 不涉及改动的部分

- 上传限制配置功能本身不变，只改位置
- C 端（H5、小程序、Flutter App）不受影响
- OCR 厂商配置（百度/腾讯/阿里云密钥管理）不受影响
- OCR 成功率统计不受影响
- OCR 测试功能不受影响
- 体检报告解读明细 / 拍照识药记录明细页面不受影响
