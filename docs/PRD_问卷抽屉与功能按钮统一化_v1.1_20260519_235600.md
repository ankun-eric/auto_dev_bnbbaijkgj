## 1. 需求概述

### 1.1 背景与目的

bini-health 项目当前同时存在两套"问卷类功能按钮"的实现：

- **老健康自查**：底部抽屉一屏多题 → 提交后生成图文摘要卡 + AI 流式回复，但模板写死、运营无法自助配置；
- **新版对话内问卷（INLINE_CHAT）**：模板可配置、支持联动，但展示形态局限在"对话气泡内逐题问答"，体验不如抽屉。

同时，「功能按钮管理」后台还存在以下痛点：

1. 每次问卷类按钮的"引导卡片"标题、引导文案、预计耗时、隐私声明、图标都需要走代码硬编码，运营改一句话就要发一次版；
2. 用户重复点击同一个按钮时，缺乏"进行中 / 已完成"状态反馈，容易在抽屉关闭后误以为没操作过；
3. 不同按钮要不要"先弹引导卡片再进抽屉"没有开关，强制走同一种交互路径。

**目的**：用一套**统一架构（`ai_function_type=questionnaire` + `questionnaire_template` + `function_buttons` 引导卡片增强字段）**收口"问卷类功能按钮"全部交互，并把"引导卡片图标/文案/开关、按钮灰态"全部下沉到后台可配置，让运营 3 分钟内完成新问卷按钮上线。

### 1.2 目标用户

| 角色 | 使用场景 |
|---|---|
| C 端最终用户（H5 / 小程序 / Flutter App） | 在 AI 对话页/AI 首页点击「健康自查」「症状评估」「饮食习惯调查」等问卷按钮，先看到引导卡片，再进抽屉一屏多题作答，提交后看到精致结果卡片 + AI 流式解读 |
| 运营人员 | 在管理后台「功能按钮管理」+「通用问卷模板管理」3 分钟内创建一个新的问卷按钮，自定义图标、引导文案、题目、联动、结果摘要 |
| 健康顾问 / 客服 | 通过用户问卷结果卡片快速了解用户主诉，进入 AI 对话或人工对话深入沟通 |

### 1.3 核心价值

- **用户侧**：交互回归"底部抽屉一屏多题"的沉浸式体验；按钮态化反馈让"做过没做过/做完没做完"一目了然；引导卡片帮用户先建立预期再作答，完成率提升。
- **运营侧**：问卷类按钮**完全后台化**——图标、文案、是否要卡片、题目、联动、结果摘要全部可视化配置，零代码上线新问卷。
- **研发侧**：两套问卷实现合并为一套架构，长期维护成本下降 50% 以上。

### 2.2 功能详细描述

#### F1 数据架构统一（问卷类按钮）

- 所有"问卷类功能按钮"统一 `ai_function_type = 'questionnaire'`，并通过 `questionnaire_template_id` 关联一个具体的问卷模板。
- 原 `health_self_check` 类型作为**已废弃别名**保留，启动期幂等迁移自动升级。
- 老接口 `/api/health-self-check/*` 保留**向后兼容**，内部转调新接口。

#### F2 通用问卷模板（已落地基础上沿用）

模板字段：

| 字段 | 含义 |
|---|---|
| `code` | 模板唯一编码 |
| `name` | 模板名称（如"健康自查问卷"） |
| `display_form_default` | 默认展示形态：DRAWER_SCROLL / DRAWER_STEPPED / INLINE_CHAT |
| `result_summary_template` | 结果摘要模板（支持变量替换，用于结果卡片） |
| `source` | 来源：`system_migrated` / `operator_created` |
| `questions` | 题目列表（外键） |

题目字段：

| 字段 | 含义 |
|---|---|
| `code` | 题目编码 |
| `title` | 题目标题 |
| `layout` | 布局：`icon_grid` / `tag_grid` / `tag_list` |
| `options` | 选项列表 |
| `option_filter_json` | 联动规则（如部位↔症状的级联过滤） |
| `is_visible_when` | 可见性规则（基于其他题目答案） |
| `required` | 是否必填 |

#### F3 抽屉展示形态（三选一）

- **`DRAWER_SCROLL`（默认推荐）**：底部抽屉，所有题目滚动一屏多题，底部固定「提交」按钮；
- **`DRAWER_STEPPED`**：底部抽屉，分步显示，每步 1 题，含进度条、上一步/下一步；
- **`INLINE_CHAT`**：对话气泡内逐题问答（兼容现有"对话内问卷"形态）。

每个按钮可独立选择 `questionnaire_display_form` 字段，默认走模板的 `display_form_default`。

#### F4 引导卡片可配置（全部走后台）

每个问卷类功能按钮在「功能按钮管理」中独立配置以下字段：

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `pre_card_enabled` | Boolean | **`true`（默认开）** | 是否先展示引导卡片 |
| `pre_card_icon` | String(500) | NULL | 图标内容（URL 或 Emoji） |
| `pre_card_icon_type` | Enum | `default` | `url` / `emoji` / `default`（默认问卷图标） |
| `pre_card_title` | String(60) | NULL | 卡片标题，如"健康自查" |
| `pre_card_desc` | String(200) | NULL | 引导文案，如"约 1 分钟，3 道题，帮我了解您的身体状况" |
| `pre_card_eta` | String(30) | NULL | 预计耗时，如"约 1 分钟" |
| `pre_card_privacy_note` | String(200) | NULL | 隐私声明，如"答案仅用于本次咨询，不会对外分享" |

**字段呈现规则**：

- `pre_card_enabled=false`：跳过卡片，点击按钮**直接进抽屉**；
- `pre_card_enabled=true`：先在对话流插入一张"引导卡片"消息气泡，用户点击卡片上的「开始」按钮才进抽屉；
- 任一文案字段为空时，前端使用对应**默认兜底文案**（在前端常量中保留一份兜底，确保即使运营漏填也不报错）。

#### F5 引导卡片图标三选一

后台表单「卡片图标」控件为单一控件，**Tab 切换三种来源**：

1. **「上传图片」Tab**：标准图片上传组件，上传后回填 URL，`pre_card_icon_type='url'`；
2. **「选择 Emoji」Tab**：内置常用 Emoji 选择面板（🩺 🧬 💊 🥗 😴 🚶 …），选中后 `pre_card_icon='🩺'`、`pre_card_icon_type='emoji'`；
3. **「使用默认」Tab**：留空，`pre_card_icon=NULL`、`pre_card_icon_type='default'`，前端使用默认问卷图标（项目内置 SVG）。

前端渲染时根据 `pre_card_icon_type` 分别走 `<img>` / 文本字符 / 默认 SVG。

#### F6 按钮三态机（核心新增）

每个问卷类按钮在端侧维护一个 `button_state` 状态，按当前会话上下文计算：

| 状态 | 触发条件 | 视觉表现 | 可点击 | 旁边「重做」入口 |
|---|---|---|---|---|
| `idle` | 当前会话从未点击该按钮，或上次为「重做」清空后 | 正常彩色 | ✅ | 不展示 |
| `in_progress` | 抽屉已弹起未提交（关闭抽屉但未提交、或正在作答） | 灰底 + 「进行中」字样 + 浅色图标 | ✅ 点击重新打开抽屉，**自动恢复上次作答进度** | 展示，点击=重置 |
| `completed` | 已成功提交本轮问卷 | 灰底 + ✓ 图标 + 「已完成」字样 | ❌ 不可点击 | **展示**（小号"重做"链接） |

状态机存储：

- **前端会话级缓存**：使用 `sessionStorage`（H5）/ 内存（小程序、Flutter）维护 `button_state_map[button_id]`；
- **服务端补充**：抽屉提交成功后，后端在 `chat_messages` 插入 `kind=questionnaire_result_card` 消息时，将该按钮置为 `completed` 通过 SSE 推送给前端；
- **跨会话规则**：切换会话（新建对话 / 切换咨询人）时清空所有按钮态回到 `idle`。

**「重做」交互**：

- `in_progress` 态点击「重做」：弹 `Dialog.confirm`「重新开始问卷？已填写的答案会被清空。」→ 确认 → 清空 `button_state` + 清空缓存答案 → 走正常入口（卡片或抽屉）；
- `completed` 态点击「重做」：弹 `Dialog.confirm`「重新填写问卷？上次结果不会被删除，会保留在对话流中。」→ 确认 → 清空 `button_state` 回到 `idle` → 走正常入口。

#### F7 抽屉提交与结果卡片

- 用户在抽屉点击「提交」→ 调用 `POST /api/questionnaire/submit`：
  - 入参：`button_id` / `template_id` / `answers`（题号→选项）/ `consultant_id`
  - 返回：`result_card_data`（按 `result_summary_template` 渲染好的图文结构）+ `ai_prompt_hint`（给 AI 的提示词）
- 前端关闭抽屉，对话流插入 `kind=questionnaire_result_card` 消息气泡（精致图文摘要）；
- 紧接着 AI 基于 `ai_prompt_hint` 进行**流式解读**（SSE 复用既有 `/start-stream` 模式），生成普通回复消息气泡；
- 按钮 `button_state` 切换为 `completed`。

#### F8 后台「功能按钮管理」表单增强

表单分组重排：

```
[基本信息] 名称 / 排序 / 是否启用
[按钮类型] page_navigate / ai_function（→ 子类型 questionnaire / image_capture / …）
[问卷配置] （仅 ai_function_type=questionnaire 显示）
   ├─ 关联问卷模板（下拉）
   └─ 问卷展示形态（DRAWER_SCROLL / DRAWER_STEPPED / INLINE_CHAT）
[引导卡片] （仅 ai_function 类显示）
   ├─ 是否先展示引导卡片  [开关，默认开]
   ├─ 卡片图标（Tab：上传图片 / 选择 Emoji / 使用默认）
   ├─ 卡片标题（输入框，60 字）
   ├─ 引导文案（TextArea，200 字）
   ├─ 预计耗时（输入框，30 字）
   └─ 隐私声明（TextArea，200 字）
[AI 开场白] （可选，沿用现有字段）
[页面跳转配置] （仅 page_navigate 显示）
```

校验规则：

- `pre_card_enabled=true` 时，标题与引导文案至少填一个（避免空白卡片）；其他字段允许为空走默认；
- `ai_function_type=questionnaire` 时，问卷模板与展示形态均必填。

#### F9 后台「问卷模板管理」增强

- 表单新增「结果摘要模板」TextArea（支持 Mustache 变量 `{{q1.answer_text}}` 等）；
- 列表新增「来源」列：`system_migrated` 显示蓝色徽章「系统迁移」、`operator_created` 显示绿色徽章「运营新建」；
- 系统迁移的模板**不可删除**，只能克隆后再修改。

#### F10 数据迁移（启动期幂等）

`prd_questionnaire_drawer_v1_migration.py` 在原有基础上**扩展**以下迁移：

1. 所有 `button_type/ai_function_type='health_self_check'` 的按钮 → `ai_function_type='questionnaire'`、`questionnaire_template_id` 指向系统迁移生成的 `health_self_check` 模板；
2. 所有问卷类按钮的 `pre_card_enabled` 默认置 `true`（与决策一致）；
3. 老健康自查按钮的"引导卡片字段"使用项目内已有默认文案回填（标题"健康自查"、引导文案"约 1 分钟，3 道题，帮您了解身体状况"、预计耗时"约 1 分钟"、隐私声明"答案仅用于本次咨询"、图标使用默认 SVG）；
4. 迁移幂等：再次启动时，已迁移的按钮不重复处理（通过 `migration_flag` 字段或基于内容判断）。

#### F11 多端适配

| 端 | 关键改造 |
|---|---|
| **管理后台**（admin-web，Next.js） | 表单分组重排、Emoji 选择器、图片上传、表单校验 |
| **H5**（h5-web，Next.js） | `QuestionnaireDrawer` 沿用；新增 `PreCardBubble` 引导卡片气泡组件；按钮渲染加 `ButtonStateBadge`（进行中/已完成）+ 「重做」浮层小链接 |
| **小程序**（原生） | 同 H5 三件套；wxml/wxss 同步样式 |
| **Flutter App** | 同 H5 三件套；`Drawer`/`BottomSheet` 沿用 Material 风格 |

#### F12 数据埋点

漏斗 6 节点（事件名规范）：

1. `questionnaire_button_exposed`（按钮曝光，含 `button_id` / `button_state`）
2. `questionnaire_precard_exposed`（引导卡片曝光，仅 `pre_card_enabled=true` 时）
3. `questionnaire_precard_confirmed`（卡片「开始」点击）
4. `questionnaire_drawer_exposed`（抽屉曝光）
5. `questionnaire_submitted`（提交成功，含答题用时）
6. `questionnaire_ai_replied`（AI 流式解读完成）

每个事件统一带 `button_id` / `template_id` / `consultant_id` / `session_id`。

## 4. 非功能性需求

### 4.1 性能要求

- 「功能按钮管理」列表分页加载，每页 20 条；
- `GET /api/questionnaire/buttons/{button_id}/render-meta` 接口响应 ≤ 300ms（含问卷模板 + 题目 + 联动规则）；
- 引导卡片首屏渲染 ≤ 50ms（前端缓存模板数据）；
- 抽屉提交端到端 ≤ 1.5s（不含 AI 流式解读）。

### 4.2 安全要求

- 问卷答案落库时仅与当前用户 + 当前咨询人 + 当前会话三元组绑定，不对外暴露；
- 隐私声明字段内容由运营在后台自行维护，前端展示需 100% 还原后台配置；
- 后台「功能按钮管理」与「问卷模板管理」仅 `admin` / `operator` 角色可见。

### 4.3 兼容性要求

- H5：最新 2 个版本主流移动浏览器（iOS Safari / Android Chrome）；
- 小程序：微信 8.0+；
- Flutter App：iOS 13+ / Android 7+；
- 老接口 `/api/health-self-check/*` 至少保留 6 个月向后兼容。

## 5. 业务规则与约束

1. **按钮态生命周期**：以"当前会话"为隔离边界，切换会话/咨询人即清空所有按钮态；
2. **重做不删旧结果**：completed 态重做后，旧的"问卷结果卡片"消息**仍保留**在对话流中，仅按钮态重置；
3. **引导卡片可关闭**：用户在引导卡片上滑掉/点击 X → 等同于"未确认开始"，按钮态保持 `idle`；
4. **必填校验**：抽屉内 `required=true` 的题目未答时提交按钮置灰；
5. **联动规则**：题目 A 的选项变化时，题目 B 的可见选项实时刷新（`option_filter_json`）；不可见题目不参与必填校验；
6. **运营无新建模板权限时**：仅可编辑已有模板的"结果摘要"和"题目文案"，不可改"题目结构 / 联动规则"（防止破坏迁移模板）；
7. **图标兜底**：`pre_card_icon_type=url` 但 URL 加载失败时，前端自动降级为 `default` SVG；`emoji` 类型在不支持 Emoji 的设备上同样降级。

## 6. 权限设计

| 角色 | 权限说明 |
|---|---|
| `super_admin` | 全部功能：可创建/编辑/删除所有按钮和模板，可强删迁移模板 |
| `admin` | 可创建/编辑所有按钮和模板，可克隆迁移模板，不可强删迁移模板 |
| `operator`（运营） | 可创建/编辑/删除自己创建的按钮和模板；可克隆迁移模板；不可编辑系统迁移模板的题目结构 |
| `customer_service`（客服） | 只读访问按钮和模板列表，不可编辑 |
| `end_user`（C 端用户） | 仅可见已启用的按钮，可点击发起问卷，可重做 |

## 7. 异常处理与边界情况

| 场景 | 处理方案 |
|---|---|
| 后台未配置引导卡片任意文案，但开关为开 | 使用前端默认兜底文案 |
| `pre_card_icon` URL 失效 | 自动降级为默认问卷 SVG 图标 |
| 抽屉提交时网络中断 | 前端 Toast「提交失败，请重试」+ 保留作答缓存，按钮态保持 `in_progress` |
| AI 流式解读中断 | 抽屉提交本身已成功，按钮态已切 `completed`；AI 解读失败的消息气泡显示「AI 回复失败，点击重试」 |
| 用户切换咨询人时抽屉正打开 | 弹 `Dialog.confirm`「切换将放弃当前作答，是否继续？」→ 确认才切人 |
| 老 `health_self_check` 按钮迁移后用户已有的"进行中"缓存 | 启动期前端比对 `button_id`，无效缓存自动清空 |
| `questionnaire_template_id` 指向已删除模板 | 按钮自动隐藏并在后台列表显示红色「模板已失效」徽章 |
| `pre_card_enabled=true` 但前端尚未拉到模板元数据 | 按钮显示 loading 态，禁用点击，500ms 后兜底回到 `idle` |
| 引导卡片在对话流停留过久（用户没点击「开始」） | 不做超时处理，用户可手动滑掉；下次点击按钮重新弹出新的卡片气泡 |
