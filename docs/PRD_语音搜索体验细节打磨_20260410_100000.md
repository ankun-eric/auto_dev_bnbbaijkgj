# 语音搜索体验细节打磨 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

bini-health 项目已在 H5、小程序、Flutter App 三端实现了语音搜索功能，整体流程可用。但在实际用户使用中发现了三个影响体验的细节问题：

1. **语音识别结果包含标点符号**：腾讯云 ASR 返回的文本中带有中英文标点（如句号、逗号），直接用于搜索关键词不合理
2. **"点我结束"按钮文字太小**：当前按钮直径 80px、文字 12px，老年用户看不清楚
3. **语音识别后麦克风图标消失**：用户完成一次语音搜索后，搜索框右侧的麦克风图标被隐藏，无法再次使用语音输入

本次优化聚焦于以上三个问题的修复，保持现有语音搜索风格不变，仅做体验细节打磨。

### 1.2 目标用户

- bini-health 平台所有使用搜索功能的用户
- 重点关注中老年用户群体（视力偏弱，对小字体、小按钮不友好）

### 1.3 核心价值

- 提升语音搜索的结果准确性（去除无用标点，搜索更精准）
- 提升老年用户的可操作性（按钮更大、文字更清晰）
- 修复功能缺陷（麦克风图标不消失，支持连续语音输入）

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 语音识别结果处理 | ASR 识别文本自动去除所有标点符号 | P0 | 中英文标点全部清除，仅保留纯文字 |
| F02 | 录音交互优化 | "点我结束"按钮放大 | P0 | 文字 12px→16px，按钮 80px→100px |
| F03 | 搜索框图标优化 | 搜索框有文字时同时显示麦克风和清除图标 | P0 | 支持用户随时再次语音输入 |

### 2.2 功能详细描述

#### F01：ASR 识别文本自动去除所有标点符号

**现状**：语音识别完成后，腾讯云 ASR 返回的文本原样填入搜索框，例如："高血压怎么办。" 或 "感冒,发烧"

**优化后**：识别结果在填入搜索框前，自动清除所有中英文标点符号

**清除范围**：

| 类别 | 需清除的字符 |
|------|------------|
| 中文标点 | 。、，、！、？、；、：、""、''、（）、【】、《》、——、……、、·、～ |
| 英文标点 | . , ! ? ; : " ' ( ) [ ] { } - _ / \ @ # $ % ^ & * + = ~ ` < > |

**业务规则**：
- 清除标点后如果文本为空，则不触发搜索，保持搜索框空白状态
- 清除标点后的文本前后空格也需要 trim
- 该处理逻辑在**前端**执行（减少后端改动），在 ASR 识别结果回调中即时处理

**三端统一**：
- H5：在语音识别成功回调中，对 ASR 返回文本执行标点清除
- 小程序：在微信同声传译插件回调中，对识别文本执行标点清除
- Flutter App：在后端 ASR 返回结果后，前端对文本执行标点清除

#### F02：录音中"点我结束"按钮放大

**现状**：
- 按钮直径：80px
- 文字大小：12px
- 老年用户反馈看不清楚

**优化后**：
- 按钮直径：**100px**（增大 25%）
- 文字大小：**16px**（增大 33%）
- 按钮的脉动呼吸动画效果保持不变，仅调整尺寸
- 声波竖条动画区域随按钮放大做等比例调整

**三端统一**：
- H5：修改 CSS 中按钮宽高为 100px、font-size 为 16px
- 小程序：修改 WXSS 中对应样式
- Flutter App：修改 Widget 中对应尺寸参数

#### F03：搜索框有文字时同时显示麦克风图标和清除图标

**现状**：
- 搜索框为空时：右侧显示麦克风图标 🎙
- 搜索框有文字时：右侧只显示清除(×)图标，麦克风消失
- 用户做完一次语音搜索后，想再说一次只能先清空搜索框

**优化后**：
- 搜索框为空时：右侧显示麦克风图标 🎙
- 搜索框有文字时：右侧**同时显示**麦克风图标 🎙 和清除(×)图标
  - 布局：`[搜索文字] [🎙] [×]`
  - 麦克风在左，清除在右
  - 两个图标之间有适当间距（8px）
- 用户点击麦克风：触发新一轮语音录音，录音识别成功后**替换**搜索框中的现有文字
- 用户点击清除(×)：清空搜索框文字，恢复为仅显示麦克风图标的初始状态

**三端统一**：
- H5：修改搜索框组件的条件渲染逻辑
- 小程序：修改搜索框 WXML 模板和对应 JS 逻辑
- Flutter App：修改搜索框 Widget 的子组件布局

## 4. 非功能性需求

### 4.1 性能要求

- 标点清除处理应在 **5ms 以内**完成（纯前端正则替换，不应有性能问题）
- 按钮放大不影响页面渲染性能

### 4.2 安全要求

- 无新增安全需求（不涉及后端接口变更）

### 4.3 兼容性要求

- H5：iOS Safari 12+、Android Chrome 70+、微信内置浏览器
- 小程序：微信 7.0+
- Flutter App：iOS 12+、Android 5.0+

## 5. 业务规则与约束

| 规则编号 | 描述 |
|---------|------|
| R01 | 标点清除适用于所有 ASR 识别来源（语音识别结果），不影响用户手动键盘输入的内容 |
| R02 | 标点清除后文本为空时，不触发搜索 |
| R03 | 麦克风图标在录音过程中应隐藏（录音弹窗覆盖了搜索框），录音结束后恢复显示 |
| R04 | 点击麦克风重新语音输入时，新识别结果替换搜索框现有文字（非追加） |
| R05 | 三端行为保持一致，不允许各端自行发挥差异化交互 |

## 6. 权限设计

| 角色 | 权限说明 |
|------|----------|
| 所有用户（含未登录） | 语音搜索功能对所有用户开放，无需登录即可使用 |

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|---------|
| ASR 返回文本全为标点（如 "。。。"） | 清除后为空字符串，不触发搜索，搜索框保持空白 |
| ASR 返回空文本 | 维持现有逻辑不变，提示"未识别到内容" |
| 用户快速连续点击麦克风 | 防抖处理，前一次录音未结束时忽略新的点击 |
| 搜索框文字过长导致图标区域被挤压 | 搜索框文字区域应设置 max-width，确保右侧 🎙 和 × 图标始终可见 |
| 浏览器不支持录音（如旧版 Safari） | 维持现有逻辑：不支持时隐藏麦克风图标 |

## 8. 补充说明

### 8.1 三端技术要点

| 端 | 语音识别方式 | 标点清除执行位置 |
|---|---|---|
| H5 | 浏览器 MediaRecorder + 后端腾讯云 ASR | ASR 结果回调函数中 |
| 小程序 | 微信同声传译插件（WechatSI） | 插件识别结果回调中 |
| Flutter App | record 插件录音 + 后端腾讯云 ASR | ASR 结果回调函数中 |

### 8.3 不在本次范围内的功能

- 语音识别准确率优化（依赖腾讯云 ASR 服务端能力，非前端可控）
- 实时转写（边录边出文字）
- 方言/多语言支持
- 最大录音时长限制
- 语音按钮文字提示

以上功能可作为后续迭代方向，本次不做。
