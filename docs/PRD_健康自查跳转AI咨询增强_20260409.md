# PRD：健康自查跳转 AI 健康咨询增强

**版本**：v1.0  
**日期**：2026-04-09  
**产品**：bini-health 健康管理平台

---

## 一、需求背景

健康自查功能（/symptom）流程为：选择不适部位 → 描述症状 → 补充信息（持续时间）→ 点击"AI智能分析"→ 跳转到 AI 健康咨询聊天页（/chat/{sessionId}）。

**现有问题：**
1. 聊天页面未读取 URL `msg` 参数，症状信息无法自动发送给 AI（Bug）
2. `sessionId` 由前端临时生成（`symptom-${Date.now()}`），非后端真实会话 ID，历史消息加载失败（Bug）
3. 跳转时缺少"为谁咨询"的家庭成员选择环节，AI 无法针对具体成员的健康档案给出精准建议

---

## 二、需求目标

1. 修复症状内容传递 Bug，确保跳转后症状信息正确自动发送给 AI
2. 跳转前弹出家庭成员选择弹窗，支持新建家庭成员并填写健康档案
3. 聊天页第一条消息展示为结构化症状自查卡片（绿色简洁风）
4. 聊天页顶部展示可收起的症状摘要条
5. 后端用完整健康档案数据增强 AI 上下文，AI 回复语气亲切并点名咨询对象
6. 聊天页输入框区域提供"切换咨询对象"小图标入口

---

## 三、功能详细设计

### 3.1 Bug 修复

**3.1.1 聊天页读取 msg 参数**
- 聊天页（/chat/{sessionId}）初始化时检测 URL 参数 `msg` 和 `type`
- 若 `type=symptom` 且 `msg` 非空，自动将 `msg` 内容作为用户第一条消息发送给 AI
- 不再依赖前端手动触发，消息在页面加载完成后自动发送

**3.1.2 后端真实创建 ChatSession**
- 健康自查"确认并开始分析"时，前端调用后端 API 创建真实的 ChatSession（关联 `family_member_id`）
- 后端返回真实 `session_id`，前端使用该 ID 跳转到聊天页
- 废弃原有前端临时生成 ID 的方式

---

### 3.2 家庭成员选择弹窗（健康自查触发）

**触发时机**：用户在健康自查最后一步点击"AI智能分析"

**弹窗交互流程：**

1. 弹出家庭成员选择弹窗，列表显示：
   - "为自己"（默认第一项）
   - 已有家庭成员列表（显示关系 + 昵称，如"爸爸·王大明"）
   - "添加家庭成员"按钮（列表末尾）

2. **选择已有成员（含"为自己"）**：
   - 下方展开该成员的健康档案信息（自动从 HealthProfile 回显）
   - 展示字段：性别、出生日期、身高（cm）、体重（kg）、既往病史（标签展示）、过敏史（标签展示）
   - 用户可修改，修改后保存到对应 HealthProfile 记录

3. **点击"添加家庭成员"**：
   - 下方显示空白新建表单，包含字段：
     - 关系类型（下拉选择：父亲、母亲、配偶、子女、兄弟姐妹、祖父母、外祖父母、其他）
     - 昵称（输入框，必填）
     - 出生日期（**日期选择器**，非手动输入）
     - 性别（单选：男 / 女）
     - 身高（cm，数字输入）
     - 体重（kg，数字输入）
     - 既往病史（常见病标签多选 + "其他"自由输入框）
     - 过敏史（常见过敏原标签多选 + "其他"自由输入框）
   - 提交后同时创建 `FamilyMember` 记录和对应的 `HealthProfile` 记录

4. 点击"确认并开始分析"：
   - 前端调用后端 API 创建真实 ChatSession（关联 `family_member_id`）
   - 跳转到聊天页，自动发送症状摘要消息

**既往病史标签（约 12 个）**：
高血压、糖尿病（1型）、糖尿病（2型）、冠心病、脑卒中、哮喘、慢性支气管炎、甲状腺疾病、肝病、肾病、痛风、骨质疏松 + "其他"输入框

**过敏史标签（约 10 个）**：
青霉素、磺胺类、头孢类、阿司匹林、碘造影剂、海鲜、花粉、尘螨、乳胶、坚果 + "其他"输入框

---

### 3.3 数据模型变更

**HealthProfile 表新增字段：**
- `family_member_id`（外键，关联 `family_members.id`，可为空；为空时表示主用户自身档案）

**说明**：
- 主用户的 HealthProfile：`family_member_id = NULL`，`user_id = 当前用户`
- 家庭成员的 HealthProfile：`family_member_id = 成员ID`，`user_id = 当前用户`（数据归属于创建人）

---

### 3.4 AI 聊天页增强

#### 3.4.1 症状自查卡片（第一条消息）

**展示形式**：
- 样式：浅绿色背景 + 圆角边框，视觉上区别于普通白色气泡
- 顶部标题栏：绿色医疗十字图标 + "健康自查摘要"文字
- 内容区显示四行关键信息：
  1. 部位（对应人体部位小图标 + 文字，如 🔵 头部）
  2. 主要症状标签（如 头痛 / 头晕）
  3. 持续时间（如 持续 2 天）
  4. 咨询对象（如 为爸爸·王大明）

**展开/收起交互（默认收起）**：
- 默认状态：卡片收起，只显示一行摘要（如"头部 · 头痛/头晕 · 为爸爸"）
- 点击后展开显示完整四行卡片
- 右侧有展开/收起箭头图标

#### 3.4.2 顶部症状摘要条

**展示逻辑**：
- 进入聊天页后，顶部显示完整摘要条（展开状态）
- 显示内容：本次自查的部位、症状、持续时间、咨询对象
- **AI 首次回复后自动收起为一行摘要**（如"头部 · 头痛/头晕 · 为爸爸"）
- 点击一行摘要可重新展开完整摘要条

#### 3.4.3 切换咨询对象入口（蚂蚁阿福风格）

**位置**：聊天页输入框区域，放置一个小图标（人物/切换图标）

**默认状态**：图标静默显示，不展示当前咨询对象文字（精简不占空间）

**点击后**：弹出家庭成员选择列表，包含：
- 当前用户（"为自己"）
- 已有家庭成员列表
- "新建家庭成员"入口

**切换后的处理**：
- 继续当前对话，不新建会话
- AI 收到系统提示："用户已将咨询对象切换为 XXX（附健康档案摘要）"
- AI 后续回复基于新的咨询对象的健康档案

---

### 3.5 AI 上下文增强（后端）

**混合模式**：
- 前端：症状卡片展示中包含健康档案摘要（用户可见的关键字段）
- 后端：在 AI system prompt 中注入完整健康档案数据（用户不可见），包括：
  - 咨询对象基本信息（姓名/昵称、关系、性别、年龄）
  - 身高、体重、BMI（自动计算）
  - 既往病史
  - 过敏史
  - 本次自查症状摘要（部位、症状列表、持续时间）

**AI 回复风格（第一条回复）**：
- 语气亲切自然，以"您好，我注意到这次是为您的爸爸咨询……"开头
- 档案信息穿插在分析过程中自然提及，不集中列出
- 不采用生硬的"已接收以下数据"式开头

---

### 3.6 其他咨询入口的家庭成员选择

**健康自查**：跳转前**必须**弹出家庭成员选择弹窗（强制）

**其他入口**（通用咨询、中医养生、用药参考）：
- 不强制弹窗
- 进入聊天页后，通过输入框区域的小图标随时切换咨询对象
- 默认为"自己"

## 五、非功能需求

| 项目 | 要求 |
|------|------|
| 视觉风格 | 绿色健康主题，简洁清爽，与现有 UI 保持一致 |
| 响应速度 | 弹窗打开 < 300ms，自动发送消息 < 500ms |
| 兼容性 | 支持 iOS / Android H5，与现有 App 内嵌一致 |
| 数据安全 | 健康档案数据不在 URL 中明文传输，仅传递 session_id 和 family_member_id |

## 六、开发涉及范围

### 前端
- `/symptom`：健康自查最后一步，新增家庭成员选择弹窗组件
- `/chat/{sessionId}`：
  - 读取 `msg`/`type` URL 参数并自动发送
  - 新增顶部摘要条组件（展开/收起）
  - 第一条消息改为症状自查卡片组件（展开/收起）
  - 输入框区域新增"切换咨询对象"小图标

### 后端
- `HealthProfile` 表新增 `family_member_id` 字段（数据库迁移）
- 新增/修改 API：
  - `POST /api/chat/sessions`：创建 ChatSession，支持传入 `family_member_id`
  - `POST /api/family/members/{id}/health-profile`：为家庭成员创建/更新 HealthProfile
  - 聊天 AI 接口：在 system prompt 中注入家庭成员健康档案数据
