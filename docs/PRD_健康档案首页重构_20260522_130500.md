# 健康档案首页重构 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前健康档案首页（`/health-profile`）采用 5 个横向 Tab 切换模式（今日数据/健康信息/用药计划/共管与提醒/健康事件），页面层级较多、跳转路径长，对 40~75 岁目标用户群体来说认知负担较大。本次重构目标是**将首页改为单页平铺折叠式布局**，减少 Tab 切换和页面跳转，让用户在一个页面内完成大部分日常查看操作。

### 1.2 目标用户

- **主要用户**：40~75 岁中老年人，关注自身及家人的日常健康数据
- **次要用户**：家庭成员中的年轻子女，代为管理长辈健康档案

### 1.3 核心价值

- **少跳转、少迷路**：一个页面看完所有日常健康信息，减少页面切换
- **信息层次清晰**：高频信息（今日数据、用药计划）平铺优先展示，低频信息（就医资料、设备管理）折叠或后置
- **操作简单直观**：展开/收起代替页面跳转，编辑弹层代替独立编辑页

### 2.2 功能详细描述

#### F1~F2 页面结构改造

**现有设计**：页面顶部有 5 个横向 Tab（今日数据/健康信息/用药计划/共管与提醒/健康事件），用户通过点击 Tab 切换不同板块内容。

**新设计**：删除所有 Tab 导航，改为单页自上而下平铺折叠式布局。页面板块排列顺序如下：

```
1. 吸顶标题栏 + 成员切换条
2. Hero 卡片（头像+姓名+年龄+健康信息标签+编辑按钮）
3. 家庭成员 + 我的设备（紧凑双卡片并排）
4. 预警提示条（有预警才显示，点击进入预警页）
5. 用药计划（默认折叠，展开最多 5 条 + 查看全部 + 新增按钮）
6. 就医资料（默认折叠，展开为紧凑 2×2 卡片网格 → 底部抽屉详情）
7. 今日健康数据（用药提醒通栏高亮第一位 + 体征 2×2 网格：血压/血糖/心率/睡眠）
```

**业务规则**：
- 所有板块在同一页面内垂直排列，无需 Tab 切换
- 页面进入时自动滚动到顶部
- 切换家庭成员时，所有板块数据联动刷新

#### F4~F7 Hero 卡片重构

**Hero 卡片**是页面最顶部的个人身份/健康信息卡片区域。

##### 基本信息展示
- 头像（关系字徽圆形底色图标）
- 姓名
- 年龄（根据生日计算）
- 关系标签
- 性别

##### 健康信息标签展示（F6）
- 标题使用"健康信息"（不使用"健康底档"）
- 直接展示标签胶囊，不需要"既往病史：""过敏史："等前缀分类名
- 全量展示，自动换行，有多少显示多少
- 标签颜色需在蓝色 Hero 背景上醒目，使用与背景形成对比的半透明彩色方案
- 健康信息包括：
  - **既往病史（慢病）**：如高血压、糖尿病、高血脂等
  - **过敏史**：药物过敏、食物过敏、其他过敏
  - **手术史**：手术名称
  - **家族病史**：关系+疾病名称
  - **个人习惯**：抽烟、饮酒、运动频率、饮食偏好

##### 空状态引导（F7）
- 当用户未填写任何健康信息时，标签区域显示虚线引导框
- 提示文案如："尚未填写健康信息，点击补充 ›"
- 点击引导区域直接打开编辑弹层

##### 编辑按钮
- Hero 区域右上角或右下角放置编辑按钮（✏ 图标）
- 点击打开编辑弹层（详见 F8）

### 3.2 各板块功能说明

#### 3.2.1 成员切换条

- 横向滚动，支持手势左右滑动
- 每个成员显示：关系字徽（圆形彩色底+关系字）+ 关系标签 + 姓名
- 字徽底色按 5 色循环（橙/蓝/绿/紫/粉）
- 选中态：整块橙底白字；未选中：浅蓝灰底深字
- 被守护成员显示天蓝色"守护中"角标
- 末尾"+"按钮：点击调起新增家庭成员弹窗
- 切换成员时全页数据联动刷新

#### 3.2.2 Hero 卡片

**有数据状态**：
- 上半部分：字徽头像（圆形）+ 姓名 + 年龄 + 关系标签 + 性别
- 下半部分：健康信息标签胶囊全量展示，自动换行
- 右侧：编辑按钮（✏ 图标），点击打开编辑弹层

**空状态（无健康信息）**：
- 上半部分不变
- 下半部分：虚线引导区域，显示"尚未填写健康信息，点击补充 ›"
- 点击引导区域直接打开编辑弹层

#### 3.2.3 家庭成员+我的设备双卡片

- Hero 正下方，两张白色卡片等宽水平并排
- 每张卡片一行式紧凑布局
- 显示对应模块的实际数量（家庭成员数/设备数）
- 点击跳转到对应管理页面

#### 3.2.4 预警提示条

- 条件渲染：仅 `todayMetrics` 中存在异常指标时显示
- 橙色/红色渐变背景通栏
- 点击跳转预警页面

#### 3.2.5 用药计划（折叠板块）

- 折叠态：标题栏一行显示
- 展开态：药品卡片列表（每条含药品名、剂量、时间安排、打卡状态）
- 超 5 条截断 + "查看全部"按钮
- 底部"+ 新增用药"按钮
- 数据源：`/api/health-profile-v3/{profileId}/medication-plan`

#### 3.2.6 就医资料（折叠板块）

- 折叠态：标题栏一行显示
- 展开态：2×2 紧凑卡片网格，每张卡片显示分类图标+名称+数量+箭头
- 点击卡片弹出底部抽屉，显示分类下的资料列表
- 抽屉内可查看资料详情、上传新资料
- 数据源：`/api/health-archive-v5/medical-records`

#### 3.2.7 今日健康数据

- 用药提醒通栏高亮：蓝色渐变+环形进度，点击跳转用药提醒页
- 体征 2×2 网格：血压、血糖、心率、睡眠
- 数据源：`/api/health-profile-v3/{profileId}/today-metrics`
- 异常值使用黄色左边线高亮提示

## 4. 非功能性需求

### 4.1 性能要求

- 页面首屏加载时间 ≤ 2 秒（4G 网络环境）
- 折叠/展开动画流畅，无卡顿（动画时长 200~300ms）
- 切换家庭成员时，旧数据立即清空，新数据加载完成前显示加载状态，避免数据残影

### 4.2 安全要求

- 健康数据仅授权用户可查看和编辑
- 编辑弹层提交前需验证数据有效性
- 就医资料上传需校验文件类型和大小限制

### 4.3 兼容性要求

- 适配 iOS / Android 主流浏览器（WebView）
- 适配屏幕宽度 320px ~ 428px
- 支持深色模式（如系统已支持）

## 5. 业务规则与约束

| 编号 | 规则 | 说明 |
|------|------|------|
| R1 | 成员切换联动 | 切换成员后，Hero 卡片、预警提示条、用药计划、就医资料、今日数据全部刷新为对应成员的数据 |
| R2 | 预警条件显示 | 仅在当前成员有至少一项异常指标时显示预警提示条 |
| R3 | 用药计划折叠记忆 | 用户展开/折叠操作不持久化，每次进入页面默认折叠 |
| R4 | 就医资料折叠记忆 | 同 R3，每次进入页面默认折叠 |
| R5 | 就医资料上传限制 | 单次最多 9 个文件；图片格式 ≤ 10MB/个；PDF 格式 ≤ 20MB/个 |
| R6 | 就医资料来源标记 | AI 体检、AI 药物识别自动生成的资料标记对应来源，手动上传标记为"手动" |
| R7 | 健康信息标签展示 | 所有已填写的健康信息均以标签形式在 Hero 区域展示，不截断 |
| R8 | 编辑弹层保存 | 保存后立即刷新 Hero 区域的标签展示 |
| R9 | 删除血氧 | 今日健康数据中不再展示血氧（spo2）指标 |
| R10 | 邀请守护隐藏 | 本人态下邀请守护区域不显示 |

## 6. 权限设计

| 角色 | 权限说明 |
|------|----------|
| 本人（is_self=true） | 可查看和编辑自己的所有健康信息、用药计划、就医资料；不显示邀请守护区域 |
| 已守护家人（guarded=true） | 可查看被守护成员的健康数据（只读或可编辑取决于守护权限设置） |
| 未守护家人 | 可查看基本信息，健康数据根据成员关联状态决定 |

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|---------|
| 无家庭成员数据 | 成员切换条只显示"本人"+ 添加按钮 |
| Hero 无健康信息 | 显示虚线引导区域 + "尚未填写健康信息，点击补充 ›" |
| 无用药计划 | 用药计划板块标题旁不显示数量角标，展开后显示空状态提示 + "新增用药"按钮 |
| 无就医资料 | 就医资料展开后 4 个分类卡片数量均显示 0，点击任意分类弹出抽屉显示空状态 + "上传"按钮 |
| 无今日体征数据 | 体征卡片显示"暂无数据"或"--"占位，保持 2×2 网格布局不变 |
| 无预警 | 预警提示条区域完全不渲染，不占用空间 |
| 网络异常 | 各板块独立容错，单个接口失败不影响其他板块展示；失败区域显示"加载失败，点击重试" |
| 用药计划恰好 5 条 | 全部显示，不显示"查看全部"按钮 |
| 用药计划超过 20 条 | 仍然只显示前 5 条 + "查看全部 (N)"，展开后全部显示 |
| 就医资料某分类为 0 | 该分类卡片正常显示，数量显示 0，点击弹出抽屉显示空状态 |
| 切换成员时网络慢 | 先清空旧数据避免残影，显示加载骨架屏或加载动画 |
