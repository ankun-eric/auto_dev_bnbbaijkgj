# 健康档案共管功能优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前"家庭健康档案共管"功能已实现基本的邀请关联、扫码授权、关联列表等核心流程，但在用户体验、功能完整性上存在以下不足：

1. 用户不了解关联家人的好处，缺乏操作动力
2. 首页缺少通用扫码入口，用户需要深入功能页面才能扫码
3. 邀请二维码图片未生成真实二维码，无法实际扫码使用
4. 被邀请方同意/拒绝授权后，双方均未收到系统消息通知
5. 管理后台缺少系统消息管理功能，管理员无法查询消息记录

本次优化旨在提升共管功能的引导体验、补齐通知闭环、增强管理后台能力。

### 1.2 目标用户

- **C端用户**：使用健康档案的普通用户及其家人
- **管理员**：使用管理后台的运营/管理人员

### 1.3 核心价值

- 降低用户理解成本，提升家庭关联转化率
- 完善扫码入口，提升操作便捷性
- 补齐授权结果通知闭环，提升用户信任感
- 为管理员提供系统消息的查询与管理能力

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 关联好处说明 | 在"邀请本人关联"按钮附近固定展示关联好处 | P0 | 提升用户关联动力 |
| F02 | 首页扫码入口 | 在首页右上角消息图标旁增加扫码图标 | P0 | 通用扫码入口，支持多种二维码类型 |
| F03 | 邀请二维码优化 | 生成真实二维码，标题改为"宾尼小康AI健康管家" | P0 | 修复二维码不可用问题 |

### 2.2 功能详细描述

---

#### F01：关联好处说明

**功能描述**：在"邀请本人关联"按钮的下方，固定展示3条关联好处说明，始终可见，不可折叠/收起。

**展示内容（优化后措辞）**：

| 图标 | 标题 | 描述 |
|------|------|------|
| 📊 | 数据共享 | 家人档案共同维护，健康数据一目了然 |
| ⚠️ | 异常提醒 | 健康数据异常时，家人实时收到提醒通知 |
| 💊 | 用药提醒 | 远程了解家人用药情况，贴心守护健康 |

**展示形式**：
- 采用卡片式设计，每条好处包含图标、标题、简短描述
- 3条好处水平排列或垂直列表排列（根据屏幕宽度自适应）
- 始终固定展示在"邀请本人关联"按钮下方区域

**业务规则**：
- 无论用户是否已有关联家人，好处说明始终展示
- 好处说明为纯静态展示内容，无交互行为

**涉及端**：Flutter App、H5、微信小程序

---

#### F02：首页扫码入口

**功能描述**：在首页右上角，消息图标旁边增加一个扫码图标，点击后直接调起手机摄像头进行扫码。

**交互流程**：

1. 用户点击首页右上角扫码图标
2. 系统请求摄像头权限（首次使用时）
3. 打开扫码界面，实时识别二维码
4. 识别成功后，根据二维码内容类型进行**路由分发**

**路由分发逻辑**：

扫码功能定位为**通用扫码入口**，未来可扩展多种二维码类型，需设计路由分发机制：

| 二维码类型 | 识别规则 | 跳转目标 |
|------------|----------|----------|
| 家庭共管邀请码 | URL 包含 `/family-auth` 路径或特定参数标识 | 跳转到授权确认页面 |
| 其他类型（未来扩展） | 根据 URL 域名/路径/参数匹配 | 跳转到对应功能页面 |
| 无法识别的二维码 | 不匹配任何已知类型 | 提示"无法识别该二维码" |

**各端处理**：

| 端 | 实现方式 |
|----|----------|
| Flutter App | 调用摄像头插件直接扫码 |
| 微信小程序 | 调用 `wx.scanCode` API |
| H5 | 受浏览器限制，可降级为上传二维码图片识别，或提示用户使用 App/小程序扫码 |

**业务规则**：
- 扫码图标放置在消息图标的左侧
- 扫码图标使用通用的扫码/相机样式图标
- 摄像头权限被拒绝时，提示用户前往系统设置开启权限

**涉及端**：Flutter App、H5、微信小程序

---

#### F03：邀请二维码优化

**功能描述**：点击"邀请本人关联"按钮后，生成一张包含**真实可扫描二维码**的邀请图片，标题为"宾尼小康AI健康管家"。

**图片内容布局**：

```
┌──────────────────────────┐
│                          │
│   宾尼小康AI健康管家       │  ← 标题
│                          │
│    ┌──────────────┐      │
│    │              │      │
│    │   二维码区域   │      │
│    │              │      │
│    └──────────────┘      │
│                          │
│   扫码关联家人健康档案      │  ← 副标题/说明
│                          │
└──────────────────────────┘
```

**二维码内容**：
- 二维码编码的 URL 需包含邀请码信息
- 微信环境扫码：优先跳转到小程序的授权确认页面
- 非微信环境扫码：降级跳转到 H5 的 `/family-auth` 授权确认页面

**长按保存**：
- 支持用户**长按图片保存到手机相册**，方便通过微信/其他方式发送给家人
- Flutter 端：支持长按保存到相册
- 小程序端：支持长按保存到相册
- H5 端：支持长按保存图片

**业务规则**：
- 每次点击"邀请本人关联"按钮，生成新的邀请码和对应二维码
- 二维码需设置合理的有效期（建议24小时），过期后提示用户重新生成
- 二维码图片需清晰，确保在各种屏幕尺寸下均可正常扫描

**涉及端**：Flutter App、H5、微信小程序、后端API

---

#### F04：授权结果系统消息通知

**功能描述**：被邀请方同意或拒绝健康档案共管邀请后，系统自动向双方发送系统消息通知。

**消息场景与文案**：

| 场景 | 接收方 | 消息内容 |
|------|--------|----------|
| 同意授权 | 邀请方 | 「{被邀请方姓名}已同意您的健康档案共管邀请，您现在可以查看对方的健康数据了」 |
| 同意授权 | 被邀请方 | 「您已同意{邀请方姓名}的健康档案共管邀请，对方现在可以查看您的健康数据」 |
| 拒绝授权 | 邀请方 | 「{被邀请方姓名}已拒绝您的健康档案共管邀请」 |
| 拒绝授权 | 被邀请方 | 「您已拒绝{邀请方姓名}的健康档案共管邀请」 |

**通知方式**：站内消息 + 推送通知

| 通知渠道 | 说明 |
|----------|------|
| 站内消息 | 消息列表中新增一条消息，首页消息图标显示红点/数字角标 |
| Flutter 推送 | 通过手机系统推送通知（FCM / APNs） |
| 小程序通知 | 通过微信服务通知（订阅消息）提醒用户 |

**消息点击行为（区分场景）**：

| 消息类型 | 点击后行为 |
|----------|-----------|
| 同意授权的消息 | 跳转到**家庭共管列表页**，查看已关联的家人 |
| 拒绝授权的消息 | 跳转到**重新邀请页面**，可一键重新发起邀请 |

**业务规则**：
- 授权操作完成后，系统实时生成消息并推送
- 消息支持已读/未读状态管理
- 首页消息图标的角标数字应实时更新（包含新增的系统消息）
- 如果用户姓名为空，使用手机号脱敏后的格式（如 138****1234）代替

**涉及端**：后端API、Flutter App、H5、微信小程序

---

#### F05：管理后台系统消息管理

**功能描述**：在管理后台新增"系统消息管理"菜单，管理员可查询所有系统消息记录。

**菜单位置**：管理后台左侧导航栏，建议放在"用户管理"或"系统管理"分类下。

**消息列表页**：

| 列表字段 | 说明 |
|----------|------|
| 消息ID | 系统自动生成的唯一标识 |
| 消息类型 | 如：共管邀请结果、用药提醒、异常提醒等 |
| 发送对象 | 收到消息的用户姓名/手机号 |
| 消息内容 | 具体的消息文案 |
| 发送时间 | 消息生成时间 |
| 阅读状态 | 已读 / 未读 |

**筛选与搜索功能**：

| 筛选条件 | 类型 | 说明 |
|----------|------|------|
| 消息类型 | 下拉选择 | 按消息类型筛选（如：共管邀请结果、系统通知等） |
| 时间范围 | 日期选择器 | 按消息发送时间范围筛选 |
| 用户搜索 | 文本输入框 | 按用户手机号或姓名模糊搜索 |
| 阅读状态 | 下拉选择 | 全部 / 已读 / 未读 |

**列表功能**：
- 支持分页展示，每页默认20条
- 支持按发送时间倒序排列（最新消息在前）
- 点击消息行可展开查看消息详情

**业务规则**：
- 当前版本仅支持查询功能，暂不提供手动发送系统消息的能力
- 消息数据来源为系统自动生成的所有通知消息

**涉及端**：管理后台（admin-web）、后端API

## 4. 非功能性需求

### 4.1 性能要求

- 二维码生成时间 < 1秒
- 扫码识别响应时间 < 2秒
- 系统消息推送延迟 < 5秒（授权操作完成到消息送达）
- 管理后台消息列表查询响应时间 < 2秒

### 4.2 安全要求

- 邀请二维码需设置有效期（建议24小时），防止被恶意利用
- 二维码中不应包含用户敏感信息（仅包含邀请码标识）
- 管理后台系统消息管理需要管理员权限才能访问

### 4.3 兼容性要求

- Flutter App：iOS 12+ / Android 6.0+
- 微信小程序：基础库 2.10.0+
- H5：主流浏览器（Chrome、Safari、微信内置浏览器）
- 管理后台：Chrome 80+

## 5. 业务规则与约束

| 编号 | 规则描述 |
|------|----------|
| BR01 | 邀请二维码有效期为24小时，过期后扫码提示"邀请已过期，请联系对方重新生成" |
| BR02 | 用户不能邀请自己（扫自己生成的二维码时提示"不能关联自己"） |
| BR03 | 已关联的家人再次扫码时，提示"您已与对方建立关联" |
| BR04 | 系统消息中的用户姓名为空时，使用脱敏手机号代替 |
| BR05 | 消息推送失败时（如用户未授权推送），不影响站内消息的正常记录和展示 |
| BR06 | 首页扫码入口需要用户登录后才可使用，未登录时点击引导登录 |

## 7. 异常处理与边界情况

| 异常场景 | 处理方式 |
|----------|----------|
| 摄像头权限被拒绝 | 弹窗提示"请在系统设置中开启摄像头权限"，提供跳转设置按钮 |
| 扫码识别失败 | 提示"识别失败，请对准二维码重试" |
| 扫描到不支持的二维码类型 | 提示"无法识别该二维码" |
| 二维码已过期 | 提示"邀请已过期，请联系对方重新生成邀请码" |
| 网络异常导致消息推送失败 | 站内消息正常记录，推送通知在网络恢复后补发或静默失败 |
| H5端不支持摄像头扫码 | 降级为上传图片识别，或提示用户使用App/小程序扫码 |
| 用户未设置姓名 | 系统消息中使用手机号脱敏格式（如 138****1234） |
| 保存图片到相册失败 | 提示"保存失败，请检查相册权限" |

## 8. 补充说明

### 8.1 数据模型变更

需要新增或修改以下数据结构：

- **系统消息表**（新增/扩展）：支持存储消息类型、发送对象、消息内容、发送时间、阅读状态、关联业务类型和业务ID等字段
- **邀请码表**（修改）：需确保邀请码能生成对应的二维码URL，并支持有效期管理

### 8.2 第三方依赖

- 二维码生成：后端使用 Python 二维码生成库（如 qrcode）
- Flutter 扫码：使用 mobile_scanner 或 qr_code_scanner 插件
- 微信小程序扫码：使用原生 `wx.scanCode` API
- 推送通知：Flutter 端使用 Firebase Cloud Messaging（FCM）/ APNs；小程序端使用微信订阅消息

### 8.3 后续扩展方向

- 首页扫码入口后续可支持扫描体检报告二维码、医院报告二维码等更多场景
- 管理后台系统消息管理后续可扩展"手动发送系统消息"能力（如运营活动通知、系统维护通知等）
- 系统消息类型可扩展为：用药提醒通知、健康异常通知、计划到期通知等
