# 语音搜索体验细节优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

bini-health 项目的语音搜索功能已在 H5 端完成了基础体验升级（单击录音、脉动呼吸按钮、声波动画、腾讯云 ASR 一句话识别、2 秒自动搜索倒计时），小程序端使用微信同声传译插件，Flutter 端使用 record 插件 + 后端 ASR。

在实际使用中，用户反馈了三个影响体验的细节问题：

1. **语音识别结果包含标点符号**：腾讯云 ASR 返回的文本带有句号、逗号等标点，作为搜索关键词使用时不合理。
2. **"点我结束"按钮文字太小**：当前仅 12px，老年用户看不清楚。
3. **语音识别后麦克风图标消失**：识别完成后搜索框内的麦克风入口消失，用户无法再次使用语音输入。

本次需求聚焦于以上三个体验细节打磨，保持现有交互风格不变，三端（H5 + 小程序 + Flutter App）统一优化、同步上线。

### 1.2 目标用户

- bini-health 平台所有用户，尤其是**中老年用户**（对字号和操作便捷性敏感）
- 使用语音搜索功能的用户

### 1.3 核心价值

- 提升语音搜索的使用流畅度和可用性
- 降低老年用户的使用门槛
- 确保语音搜索功能的可重复使用，不因一次使用后入口消失而中断

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 语音识别 | 标点符号自动清除 | P0 | 语音识别返回文本自动去除所有中英文标点 |
| F02 | 录音界面 | "点我结束"按钮放大 | P0 | 按钮 80px→100px，文字 12px→16px |
| F03 | 搜索框 | 麦克风图标常驻 | P0 | 搜索框有文字时，同时显示麦克风和清除图标 |

### 2.2 功能详细描述

#### F01：标点符号自动清除

**需求描述**：语音识别（ASR）返回的文本在填入搜索框之前，自动去除所有中文和英文标点符号，只保留纯文字内容。

**业务规则**：
- 需要清除的中文标点包括但不限于：`。，！？、；：""''【】（）《》——…·`
- 需要清除的英文标点包括但不限于：`. , ! ? ; : " ' ( ) [ ] { } - _ / \ @ # $ % ^ & * + = ~ |`
- 清除后如果结果为空字符串，则不触发搜索
- 清除操作在前端完成，不影响后端 ASR 接口返回内容

**影响范围**：
- **H5**：`h5-web/src/app/search/page.tsx` — 在 ASR 识别结果回调中添加标点清除逻辑
- **小程序**：`miniprogram/pages/search/index.js` — 在微信同声传译回调中添加标点清除逻辑
- **Flutter**：`flutter_app/lib/screens/search/search_screen.dart` — 在 ASR 结果处理中添加标点清除逻辑

---

#### F02："点我结束"按钮放大

**需求描述**：录音进行中的"点我结束~"按钮，按钮直径从 80px 放大到 100px，按钮内文字从 12px 放大到 16px，让老年用户更容易看清和点击。

**设计规格**：
| 属性 | 当前值 | 优化后 |
|------|--------|--------|
| 按钮直径 | 80px | 100px |
| 文字大小 | 12px | 16px |
| 按钮样式 | 保持不变（白色背景+绿色边框+脉动呼吸动画） | 同左，仅放大尺寸 |

**影响范围**：
- **H5**：`h5-web/src/app/search/page.tsx` — 修改录音按钮的 CSS 尺寸
- **小程序**：`miniprogram/pages/search/index.wxss` — 修改对应按钮样式
- **Flutter**：`flutter_app/lib/screens/search/search_screen.dart` — 修改对应按钮 Widget 尺寸

---

#### F03：麦克风图标常驻

**需求描述**：当搜索框中有文字内容时，搜索框右侧同时显示**麦克风图标**和**清除（×）图标**，用户可以随时点击麦克风重新进行语音输入，无需先清空搜索框。

**交互说明**：

| 搜索框状态 | 右侧图标 | 操作说明 |
|-----------|----------|---------|
| 空（无文字） | 仅显示麦克风 🎤 | 点击开始语音输入 |
| 有文字 | 麦克风 🎤 + 清除 × | 点击麦克风重新语音输入；点击×清除文字（清除后回到上一状态） |
| 录音中 | 显示录音动画界面（全屏遮罩） | 与当前行为一致 |

**图标布局**：
- 搜索框右侧区域，从左到右依次为：麦克风图标、清除图标
- 两个图标之间保持 8px 间距
- 图标大小与当前麦克风图标保持一致

**影响范围**：
- **H5**：`h5-web/src/app/search/page.tsx` — 修改搜索框右侧图标的条件渲染逻辑
- **小程序**：`miniprogram/pages/search/index.wxml` 和 `index.js` — 修改搜索框右侧图标显示逻辑
- **Flutter**：`flutter_app/lib/screens/search/search_screen.dart` — 修改搜索框 suffixIcon 组件

## 3. 页面/界面设计

### 3.1 页面结构与导航

本次优化不涉及新页面，仅修改现有搜索页面的组件细节。

### 3.2 各页面功能说明

#### 搜索页（已有）

**搜索框区域变更**：
- 原始状态：搜索框为空时右侧显示麦克风图标 → **不变**
- 有文字状态：原来仅显示清除×图标 → **改为同时显示麦克风🎤和清除×图标**

**录音浮层/遮罩变更**：
- "点我结束~"按钮尺寸从 80px 放大到 100px
- 按钮文字从 12px 放大到 16px
- 其余动画效果（脉动呼吸、声波竖条）保持不变

**语音识别结果处理变更**：
- ASR 返回文本经过标点清除后，再填入搜索框
- 清除后的文本如果非空，进入 2 秒自动搜索倒计时（与现有逻辑一致）

## 4. 非功能性需求

### 4.1 性能要求

- 标点清除为纯前端正则替换操作，耗时应 < 1ms，对用户无感知
- 图标渲染不增加额外网络请求

### 4.2 安全要求

- 无新增安全需求

### 4.3 兼容性要求

- H5：与现有浏览器兼容性要求一致（Chrome、Safari、微信内置浏览器）
- 小程序：微信基础库 2.x+
- Flutter：iOS 12+ / Android 5.0+

## 5. 业务规则与约束

| 规则 | 说明 |
|------|------|
| 标点清除范围 | 所有中文标点 + 所有英文标点，包括但不限于 `。，！？、；：""''【】（）《》…·. , ! ? ; : " ' ( ) [ ]` |
| 清除后空文本 | 不触发搜索，保持搜索框为空状态 |
| 按钮尺寸 | 三端统一：100px 按钮直径 + 16px 文字 |
| 图标排列 | 麦克风在左，清除×在右，间距 8px |

## 6. 权限设计

| 角色 | 权限说明 |
|------|----------|
| 所有用户 | 可使用语音搜索功能（无需登录） |

本次优化不涉及权限变更。

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|---------|
| ASR 返回结果全是标点符号 | 清除后为空字符串，不触发搜索，搜索框保持空状态 |
| ASR 识别失败 | 与现有逻辑一致，显示错误提示 |
| 搜索框有文字时点击麦克风 | 启动新一次语音识别，识别成功后新结果替换搜索框原有内容 |
| 浏览器不支持录音 | 与现有逻辑一致，隐藏麦克风图标 |

## 8. 补充说明

- 本次为体验细节优化，不涉及后端接口变更和数据库迁移
- 三端（H5、小程序、Flutter App）同步开发、同步上线
- 保持现有语音搜索的整体交互风格不变，仅做局部打磨
- 后端 ASR 接口和搜索管理后台（语音配置、推荐搜索词、屏蔽词等）均无需改动
