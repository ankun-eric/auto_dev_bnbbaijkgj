# 语音搜索麦克风图标消失 Bug 修复方案文档

## 1. Bug 发生背景

### 1.1 项目概述

bini-health 是一个健康管理平台，包含 H5 网页端、微信小程序端和 Flutter App 三端。项目首页搜索栏支持文字输入和语音输入两种搜索方式，语音输入通过搜索框右侧的麦克风图标（🎤）触发。

### 1.2 涉及功能模块

- **首页搜索栏**：文字输入 + 语音输入功能
- **语音搜索交互**：单击麦克风触发录音 → 脉动按钮录音中 → 识别完成 → 文字填入搜索框 → 自动搜索
- **搜索框图标逻辑**：控制麦克风图标和清除(×)图标的显示与隐藏

### 1.3 发现时间与发现方式

- **发现方式**：用户使用反馈
- **出现时间**：从语音功能上线起一直存在
- **相关历史**：之前 PRD 文档中有"F03：麦克风图标常驻"的优化需求，但该功能未被正确实现或存在回归

## 2. Bug 描述

### 2.1 错误现象

首页搜索栏的语音输入功能，在搜索框为空时麦克风图标正常显示在搜索栏内部右侧。但当用户点击麦克风完成一次语音识别后，识别出的文字被填入搜索框，此时搜索框右侧**只剩下清除(×)图标**，麦克风图标完全消失，用户无法再次通过语音发起搜索。

**核心问题**：搜索框的图标显示逻辑采用了"互斥"模式——当搜索框内有文字时，只显示清除(×)图标，隐藏了麦克风图标。应改为"共存"模式。

### 2.2 重现步骤

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 打开首页，观察搜索栏 | 搜索栏右侧显示麦克风图标 | ✅ 正常，麦克风图标可见 |
| 2 | 点击搜索栏右侧的麦克风图标 | 弹出录音浮层，进入录音状态 | ✅ 正常，录音浮层正常弹出 |
| 3 | 对麦克风说话，完成语音录入 | 语音识别正常完成，文字填入搜索框 | ✅ 正常，识别文字正确填入 |
| 4 | 观察搜索框右侧的图标 | 应同时显示麦克风图标和清除(×)图标 | ❌ **异常：只显示清除(×)图标，麦克风图标消失** |
| 5 | 尝试再次发起语音搜索 | 应能点击麦克风图标再次录音 | ❌ **异常：找不到麦克风入口，无法再次语音搜索** |
| 6 | 清空搜索框文字 | 搜索框恢复空状态，麦克风图标重新出现 | ✅ 麦克风图标恢复显示（需手动清空） |

### 2.3 影响范围

- **影响端**：H5 网页端、微信小程序端、Flutter App **三端均受影响**
- **影响用户**：所有使用语音搜索功能的用户
- **影响程度**：语音搜索功能的复用性完全丧失，每次语音搜索后必须先手动清空搜索框才能再次使用语音功能，严重影响用户体验

## 3. 预期正确效果

修复后，搜索框应实现以下行为：

1. **麦克风图标始终常驻**：无论搜索框是否有文字、无论是否刚完成语音识别，麦克风图标应**始终显示**在搜索输入栏内部的右侧
2. **图标共存布局**：当搜索框有文字时，麦克风图标和清除(×)图标**并排显示**，布局为：`[搜索文字...] [🎤] [×]`（麦克风在左，清除在右）
3. **随时可用**：用户在任何状态下都能点击麦克风图标发起新的语音搜索
4. **三端统一**：H5 网页端、微信小程序端、Flutter App 三端的行为和布局保持一致

## 4. 补充说明

- 语音搜索功能本身（录音、语音识别、自动搜索）工作正常，无需修改，**仅需修复图标显示逻辑**
- 之前已有 PRD 文档 `PRD_语音搜索体验细节优化` 中定义了"F03：麦克风图标常驻"的需求，本次修复应确保该需求被正确完整实现
- 修复需覆盖所有搜索框状态：空输入、有文字输入、语音识别刚完成、手动输入文字等场景
- 图标交互不受影响：麦克风点击仍触发语音录入，清除点击仍清空搜索框内容
