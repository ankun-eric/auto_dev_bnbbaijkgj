# 健康计划模块 Bug 修复方案文档

## 1. Bug 发生背景

### 1.1 项目概述

bini-health 是一款面向中老年用户的健康管理平台，包含用药提醒、健康打卡、自定义健康计划等核心功能，覆盖 H5、微信小程序、Flutter App 多端。

### 1.2 涉及功能模块

- **用药提醒模块**（H5 端：`/health-plan/medications/`）
- **健康打卡模块**（H5 端：`/health-plan/checkin/`）

### 1.3 发现方式

用户在 H5 端手动测试过程中发现，其他端尚未验证。

---

## 2. Bug 描述

### Bug 1：用药提醒——必填项缺少红色 `*` 标识

**错误现象：** 添加用药提醒页面中，「药品名称」和「用药时段」是必填字段，但界面上没有显示红色 `*` 必填标识，用户无法直观识别哪些字段必须填写。

**重现步骤：**

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 进入用药提醒页面，点击"添加用药提醒" | 打开添加表单 | 正常打开 |
| 2 | 观察「药品名称」和「用药时段」字段标签 | 字段标签前应有红色 `*` | 没有红色 `*` 标识 |

**预期正确效果：** 「药品名称」和「用药时段」字段标签前方显示红色 `*`，清晰标识为必填项。

**影响范围：** H5 端添加用药提醒页面。

---

### Bug 2：用药提醒——选多个时段只生成一条记录

**错误现象：** 添加用药提醒时，用户选择了多个用药时段（如早晨、中午、晚上），每个时段都配置了独立的提醒时间，但提交后只生成了一条提醒记录（仅保留第一个时段）。

**重现步骤：**

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 进入添加用药提醒页面 | 正常打开 | 正常 |
| 2 | 填写药品名称，选择 3 个用药时段（早晨 08:00、中午 12:00、晚上 18:00） | 可以多选并分别设置时间 | 可以选择（正常） |
| 3 | 点击保存提交 | 生成 3 条独立的用药提醒记录 | 只生成了 1 条记录（仅第一个时段） |

**预期正确效果：**

- 选了 N 个时段，后台应生成 N 条独立的用药提醒记录
- 每条记录对应一个时段和该时段的提醒时间（如"早晨 08:00"、"中午 12:00"、"晚上 18:00"）
- 每条记录独立打卡，在今日待办中分别显示，每条带有自己的提醒时间

**根因分析：** `handleSubmit` 方法中只取了 `selectedPeriods[0]`（第一个时段），后续选择的时段数据全部丢失。

**影响范围：** H5 端添加用药提醒逻辑，影响所有多时段用药的用户。

---

### Bug 3：用药提醒——"添加用药提醒"按钮文字与 `+` 图标未对齐

**错误现象：** 用药提醒列表页底部的"添加用药提醒"按钮，按钮上的 `+` 图标和"添加用药提醒"文字没有垂直居中对齐，文字偏下。

**重现步骤：**

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 进入用药提醒列表页面 | 页面正常显示 | 正常 |
| 2 | 观察底部"添加用药提醒"按钮 | `+` 图标和文字垂直居中对齐 | 文字偏下，未与 `+` 图标对齐 |

**预期正确效果：** `+` 图标和"添加用药提醒"文字在按钮内垂直居中对齐。

**影响范围：** H5 端用药提醒列表页。

---

### Bug 7：健康打卡——"添加打卡项"按钮布局未对齐（同 Bug 3）

**错误现象：** 健康打卡管理页面底部的「添加打卡项」按钮，与用药提醒的 Bug 3 完全相同——`+` 图标和文字没有垂直居中对齐，文字偏下。

**重现步骤：**

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 进入健康打卡管理页面 | 页面正常显示 | 正常 |
| 2 | 观察底部"添加打卡项"按钮 | `+` 图标和文字垂直居中对齐 | 文字偏下，未与 `+` 图标对齐 |

**预期正确效果：** `+` 图标和"添加打卡项"文字在按钮内垂直居中对齐。

**影响范围：** H5 端健康打卡管理页。

---

### Bug 9：健康打卡——编辑页面内容为空，未加载已有数据

**错误现象：** 在健康打卡列表中点击编辑某条打卡项，跳转到编辑页面后，表单所有字段完全空白，没有加载到该打卡项的已有数据（名称、提醒时间、重复频率等全部为空）。

**重现步骤：**

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 进入健康打卡管理页面 | 显示打卡项列表 | 正常 |
| 2 | 点击某条打卡项的"编辑"操作 | 跳转到编辑页面，表单预填充该打卡项的已有数据 | 编辑页面打开，但所有字段为空 |

**预期正确效果：** 编辑页面打开后，表单自动加载并显示该打卡项的所有已有数据（打卡名称、提醒时间、重复频率等），用户可以在此基础上修改。

**根因分析：** 编辑页面通过 URL 参数 `?id=xxx` 传递打卡项 ID，调用 `/api/health-plan/checkin-items/{id}` 接口获取数据，可能是接口返回数据格式不匹配或请求未成功发出。

**影响范围：** H5 端健康打卡编辑功能完全不可用。**严重程度：高**。

## 3. 修复方案汇总

| Bug 编号 | 修复方案 | 涉及文件/模块 |
|----------|---------|--------------|
| Bug 1 | 在「药品名称」和「用药时段」的 label 前添加红色 `*` 标识（`<span style="color:red">*</span>`） | H5 端用药提醒添加页面 |
| Bug 2 | 修改 `handleSubmit` 逻辑，遍历 `selectedPeriods` 数组中所有选中的时段，为每个时段创建一条独立的用药提醒记录并分别提交 | H5 端用药提醒添加页面 + 后端接口（如需支持批量创建） |
| Bug 3 | 修复按钮内 `+` 图标和文字的 CSS 布局，确保 `display: flex; align-items: center;` 垂直居中 | H5 端用药提醒列表页 |
| Bug 7 | 同 Bug 3 的修复方案，统一修复健康打卡页面底部按钮的布局对齐 | H5 端健康打卡管理页 |
| Bug 9 | 排查编辑页面的数据加载逻辑（URL 参数获取 → API 调用 → 数据回填），修复接口调用失败或数据未正确填充表单的问题 | H5 端健康打卡添加/编辑页面 |

---

## 4. 补充说明

- 以上 Bug 均在 **H5 端**发现，其他端（微信小程序、Flutter App）尚未验证，修复后建议同步排查
- 用户群体以 **中老年人** 为主，UI/交互需确保清晰、直观、易操作
- 所有 Bug 统一在一期修复完毕
