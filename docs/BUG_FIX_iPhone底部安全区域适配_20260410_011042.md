# H5 底部菜单 iPhone 安全区域适配 Bug 修复方案文档

## 1. Bug 发生背景

### 1.1 项目概述

本项目包含 H5 移动端 Web 应用（h5-web），使用 Next.js + antd-mobile 构建，面向用户提供移动端服务入口。底部设有 TabBar 导航菜单，用于在首页、功能页、我的等核心页面之间切换。

### 1.2 涉及功能模块

- **H5 底部 TabBar 导航栏**：位于 `h5-web/src/app/(tabs)/layout.tsx`，使用 antd-mobile 的 `TabBar` 组件，通过 `fixed bottom-0` 固定在屏幕底部。
- 底部共 **4 个菜单项**，数据从后端接口 `/api/h5/bottom-nav` 动态获取，带本地缓存策略。

### 1.3 发现时间与发现方式

- 用户在使用 iPhone 全面屏机型（有 Home Indicator 横线的型号，如 iPhone X 及以后的全面屏系列）访问 H5 页面时发现此问题。

## 2. Bug 描述

### 2.1 错误现象

在 iPhone 全面屏机型上（带 Home Indicator 横线的机型），H5 页面底部 TabBar 的 **中间 2 个菜单项的文字被手机底部的 Home Indicator 横线遮盖**，导致：
- 菜单文字被横线盖住一部分，**看不清楚**
- 影响用户正常识别和点击中间的菜单项
- 左右两侧的菜单项因位置偏上一些，受影响相对较小，但整体体验不佳

### 2.2 重现步骤

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 使用 iPhone 全面屏机型（如 iPhone X 及以后型号）打开 H5 页面 | 页面正常加载 | 页面正常加载 |
| 2 | 观察底部 TabBar 导航菜单 | 4 个菜单项完整显示，不被任何元素遮挡 | 中间 2 个菜单项的文字被 Home Indicator 横线部分遮盖，看不清 |
| 3 | 尝试点击被遮挡的菜单项 | 能正常识别并点击 | 文字不清晰，影响识别和操作体验 |

### 2.3 影响范围

- **受影响设备**：所有带 Home Indicator 的 iPhone 全面屏机型（iPhone X、XS、XR、11、12、13、14、15、16 系列等）
- **受影响页面**：H5 首页及所有使用底部 TabBar 的页面（即 `(tabs)` 路由组下的所有页面）
- **受影响功能**：底部导航菜单的可读性和可点击性
- **不受影响**：安卓手机、非全面屏 iPhone（如 iPhone 8 及更早）、没有底部 TabBar 的页面

## 3. 预期正确效果

修复后应达到以下效果：

1. **TabBar 整体上移**：底部导航菜单的 4 个菜单项全部位于 iPhone Home Indicator 横线的**上方**，图标和文字完整清晰可见，不被任何元素遮挡。

2. **横线下方白色背景填充**：Home Indicator 横线下方的区域（即 safe-area-inset-bottom 的空间）用与 TabBar 相同的**白色背景**填充，视觉上 TabBar 和底部区域形成一个整体，不会出现突兀的空白或颜色断层。

3. **兼容性无损**：
   - 在没有 Home Indicator 的设备上（安卓手机、老款 iPhone），显示效果和修复前完全一致，不会出现多余的空白
   - 在有 Home Indicator 的 iPhone 上，底部填充区域高度自适应（使用 `env(safe-area-inset-bottom)` CSS 环境变量）

4. **页面内容不被遮挡**：页面主体内容区域的底部 padding 也应相应调整，确保最底部的内容不会被上移后的 TabBar 遮挡。

## 4. 补充说明

### 4.1 根因分析

当前底部 TabBar 容器使用了 `fixed bottom-0` 样式直接贴底定位，没有为 iPhone 的底部安全区域（Safe Area Inset Bottom）预留空间。关键代码位于 `h5-web/src/app/(tabs)/layout.tsx` 第 133-135 行：

```html
<div className="fixed bottom-0 left-1/2 -translate-x-1/2 w-full bg-white border-t border-gray-100"
     style={{ maxWidth: 750, zIndex: 100 }}>
```

此外，项目的 HTML `<meta name="viewport">` 标签可能缺少 `viewport-fit=cover` 属性，导致 `env(safe-area-inset-bottom)` 无法生效。

### 4.2 修复方向

修复需要涉及以下几个关键点：

1. **添加 `viewport-fit=cover`**：在 H5 的 HTML head 中确保 viewport meta 标签包含 `viewport-fit=cover`，这是启用安全区域 CSS 环境变量的前提。

2. **TabBar 容器增加底部安全区域填充**：在 TabBar 的外层容器上增加 `padding-bottom: env(safe-area-inset-bottom)` 或等效实现，使菜单项上移到安全区域内，同时白色背景自然延伸到屏幕最底部。

3. **页面主体内容区域适配**：调整页面内容区域的底部间距，确保不被上移后的 TabBar 遮挡。

### 4.3 多端排查结论

用户最初担心小程序和 APP 端也可能存在同样的底部安全区域适配问题，经代码排查确认：

| 端 | 技术栈 | 适配状态 | 说明 |
|---|---|---|---|
| **H5 (h5-web)** | Next.js + antd-mobile | ❌ 未适配 | 需要修复（本文档重点） |
| **小程序 (miniprogram)** | 微信原生自定义 TabBar | ✅ 已适配 | 样式中已有 `height: calc(108rpx + env(safe-area-inset-bottom))` 和 `padding-bottom: env(safe-area-inset-bottom)` |
| **APP (flutter_app)** | Flutter `BottomNavigationBar` | ✅ 已适配 | Flutter `Scaffold` 默认自动处理底部安全区域（`MediaQuery.of(context).padding.bottom`） |

**结论：本次只需修复 H5 端，小程序和 APP 无需改动。**

### 4.4 涉及修改的文件

- `h5-web/src/app/(tabs)/layout.tsx` — 底部 TabBar 容器样式修改
- `h5-web/src/app/layout.tsx`（或对应的 HTML 模板） — viewport meta 标签添加 `viewport-fit=cover`
