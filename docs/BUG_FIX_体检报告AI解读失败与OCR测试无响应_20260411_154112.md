# 体检报告 AI 解读失败 & 后台 OCR 测试无响应 Bug 修复方案文档

## 1. Bug 发生背景

### 1.1 项目概述

本项目为「宾尼小康」AI 健康管家平台，包含 H5 网页端、Flutter App、微信小程序和管理后台等多端。核心功能包括体检报告上传与 AI 解读、OCR 识别、健康档案管理等。

### 1.2 涉及功能模块

- **体检报告模块**：用户上传体检报告图片 → OCR 识别文字 → AI 模型结构化解读 → 展示解读结果
- **管理后台 OCR 配置模块**：管理员配置 OCR 厂商参数，并提供"测试 OCR"和"测试完整流程"两个测试按钮

### 1.3 发现时间与发现方式

- 发现时间：2026年4月10-11日
- 发现方式：用户使用 H5 网页端体检报告功能时发现 AI 解读失败；管理员在后台 OCR 配置页面测试时发现 "OCR 测试"按钮无响应

---

## 2. Bug 描述

本次共发现 **2 个 Bug**，均统一在一期修复。

---

### Bug 1：H5 端体检报告 AI 解读失败

#### 2.1 错误现象

用户在 H5 网页端（手机浏览器）进入体检报告页面，上传体检报告图片后：
- 图片上传成功，成功跳转到结果页面
- 页面出现"AI 正在分析..."的进度动画（渐进式加载提示）
- 等待一段时间后，弹出 Toast 提示"分析失败，请重试"
- 无论上传拍照版还是电子版图片，都会失败

#### 2.2 重现步骤

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 打开 H5 网页端首页 | 正常显示首页 | 正常 |
| 2 | 点击"体检报告"入口 | 进入体检报告页面 | 正常 |
| 3 | 点击上传，选择一张体检报告图片 | 图片上传成功 | 正常（上传成功） |
| 4 | 系统自动跳转到结果页 `/checkup/result/{id}` | 显示"AI 正在分析..."动画 | 正常（显示分析动画） |
| 5 | 等待 AI 解读完成 | 显示结构化的体检报告解读结果（健康评分、分类指标等） | **异常：弹出"分析失败，请重试"** |

#### 2.3 影响范围

- **受影响端**：H5 网页端（其他端如 Flutter App、小程序若共用同一后端接口，也可能受影响）
- **受影响功能**：体检报告 AI 解读核心功能完全不可用
- **受影响用户**：所有使用体检报告功能的用户

#### 2.4 根因分析方向

根据代码分析，结果页面调用 `POST /api/report/analyze` 接口失败。该接口的调用链路为：

1. 从数据库取出报告的 OCR 文本（如果没有则重新 OCR）
2. 调用 `analyze_report_structured()` → 内部调用 `call_ai_model()` → 向 AI 模型服务发送 HTTP 请求
3. AI 模型返回 JSON → 解析并存储

**最可能的根因**（按概率排序）：

1. **AI 模型服务端异常**：API Key 额度耗尽、过期或服务不可用。`call_ai_model()` 调用失败时，会返回 `"AI服务调用失败: {错误信息}"` 字符串，然后 `analyze_report_structured()` 会尝试 `json.loads()` 解析该字符串，必然解析失败，最终返回一个降级结果（score=0，categories 为空）。但 `report.py` 第 291 行调用是在 try 块中，第 293 行 catch 后会 raise HTTPException 500，前端收到 500 就弹出"分析失败"。
2. **AI 模型响应超时**：当前 httpx 客户端超时设置为 60 秒，如果模型响应慢可能触发超时。
3. **AI 模型配置问题**：数据库中 `AIModelConfig` 表的 `is_active` 记录的 `base_url`、`model_name`、`api_key_encrypted` 可能有误。

**需要在服务器上确认的内容**：
- 查看后端日志中 `/api/report/analyze` 接口的具体报错信息
- 检查 `AIModelConfig` 表中当前激活配置的 API Key 和 base_url 是否有效
- 手动测试 AI 模型 API 是否可达

#### 2.5 修复方案

1. **排查 AI 模型配置**：检查数据库 `AIModelConfig` 表中激活的模型配置，验证 API Key 是否有效、额度是否充足
2. **增加错误日志**：在 `report.py` 的 `analyze_report` 接口中增加详细的异常日志记录（当前 `except Exception as e` 只返回了错误信息但未写日志），方便后续排查
3. **增加重试机制**：在 `call_ai_model()` 中加入自动重试（如网络波动导致的偶发失败）
4. **优化超时配置**：将 httpx 超时从固定 60 秒改为可配置，对体检报告解读这种内容较长的请求可适当延长
5. **前端优化**：在"分析失败"时显示更具体的错误原因（如"AI 服务暂时不可用"vs"网络超时"），帮助用户和管理员快速判断问题

### Bug 2：后台 OCR 配置页面"测试 OCR"按钮无响应

#### 2.1 错误现象

管理员在管理后台 → OCR 识别配置页面 → 百度云 Tab → 点击"测试 OCR"按钮：
- 选择图片后点击"开始识别"
- 按钮变为加载状态（转圈圈）
- 等待后，没有任何结果提示（不显示识别结果，也不显示错误信息）
- 按钮恢复可点击状态

而同一页面的"测试完整流程"按钮是正常工作的。

#### 2.2 重现步骤

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 登录管理后台 | 正常进入后台 | 正常 |
| 2 | 进入 OCR 识别配置页面 | 显示配置页面 | 正常 |
| 3 | 选择百度云 Tab | 显示百度云配置 | 正常 |
| 4 | 点击"测试 OCR"按钮 | 打开测试对话框 | 正常 |
| 5 | 选择一张图片，点击"开始识别" | 应显示 OCR 识别结果文本 | **异常：转圈圈后无提示，按钮恢复可点击** |

#### 2.3 影响范围

- **受影响端**：管理后台（admin-web）
- **受影响功能**：OCR 配置页面的"测试 OCR"功能
- **受影响用户**：管理员

#### 2.4 根因分析（已确认）

**前后端字段名不匹配**：

- **前端**（`admin-web/src/app/(admin)/ocr-config/page.tsx` 第 313 行）：
  ```javascript
  formData.append('file', testOcrFile);  // ← 字段名为 "file"（单数）
  ```

- **后端**（`backend/app/api/ocr.py` 第 338 行）：
  ```python
  files: List[UploadFile] = File(...)  # ← 期望字段名为 "files"（复数）
  ```

前端传了 `file`，后端期望 `files`，字段名不匹配。后端使用了 `File(...)` 标记 `files` 为必填参数，当收不到这个字段时，FastAPI 会返回 **422 Unprocessable Entity** 错误。

前端的 catch 块虽然有 `message.error()`，但由于 axios 可能将 422 错误以不同方式抛出，导致错误提示未能正确显示，用户看到的表现就是"转圈圈后没有反应"。

而"测试完整流程"按钮能正常工作，是因为它的前端代码（第 341-342 行）使用的是：
```javascript
formData.append('files', f.originFileObj);  // ← 字段名为 "files"（复数），与后端一致
```

#### 2.5 修复方案

**方案：修改前端字段名，与后端保持一致**

将前端 `handleTestOcr` 函数中的 `formData.append('file', testOcrFile)` 改为 `formData.append('files', testOcrFile)`，使前后端字段名统一为 `files`。

修改位置：`admin-web/src/app/(admin)/ocr-config/page.tsx` 第 313 行

修改内容：
```diff
- formData.append('file', testOcrFile);
+ formData.append('files', testOcrFile);
```

## 3. 预期正确效果

### Bug 1 修复后

用户在 H5 网页端上传体检报告图片后，系统应在合理时间内（通常 10-30 秒）返回完整的结构化解读结果，包括：
- 健康评分（0-100 分，带评级和评语）
- 检测项目统计（总项数、异常项数、优秀项数）
- 分类指标详细解读（按分类展示各项指标的值、参考范围、风险等级、详细建议等）

### Bug 2 修复后

管理员在后台 OCR 配置页面点击"测试 OCR"按钮后，选择图片并点击"开始识别"，应在几秒内返回 OCR 识别结果文本，并在对话框中展示。若识别失败，应显示明确的错误提示信息。

## 4. 补充说明

1. **Bug 1 与 Bug 2 是独立的两个问题**：Bug 2 是前后端字段名不匹配的代码问题（根因已确认），可以立即修复；Bug 1 是 AI 模型服务端的问题，需要先排查服务器日志确认具体报错，再针对性修复
2. **Bug 2 的修复不会影响 Bug 1**：两个问题涉及不同的接口和流程
3. **建议修复优先级**：Bug 1（高优先级，影响用户核心功能）> Bug 2（中优先级，仅影响管理员测试功能）
4. **后端日志建议**：当前 `report.py` 中 AI 解读失败时仅返回 HTTP 500 错误，建议补充 `logging.error()` 记录完整的异常堆栈信息，便于日后快速定位类似问题
