# AI健康咨询对话字体大小设置 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前「宾尼小康」App 首页右上角有一个"Aa"字体大小调节按钮，可切换首页的基础字体大小（标准/大/超大）。但实际使用中，首页大量元素使用了 Tailwind 固定字号类（如 `text-xs`、`text-sm`），真正受控的文字范围非常有限，用户切换后感知不明显。

经过分析，用户真正需要调大字体的场景是在**阅读 AI 健康咨询的对话内容**时——AI 回复的文本通常较长，中老年用户阅读起来可能比较吃力。因此，本需求将字体大小设置功能从首页迁移到 AI 健康咨询的对话页面中，专门控制对话区域的字体大小。

### 1.2 目标用户

- 中老年用户：视力下降，需要更大字体阅读 AI 回复内容
- 所有使用 AI 健康咨询功能的用户：根据个人偏好调节阅读舒适度

### 1.3 核心价值

- **精准匹配场景**：将字体调节放在真正需要的地方（AI 对话页面），而非首页
- **提升阅读体验**：让用户可以根据自身需求调节 AI 对话的字体大小
- **云端同步**：字体偏好跟随账号，换设备也能保持一致

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 字体设置入口 | AI 对话页面右上角新增"Aa"图标 | P0 | 点击弹出字体大小选择浮层 |
| F02 | 字体选择浮层 | 气泡浮层，展示三档字体 | P0 | 紧贴"Aa"图标下方的小气泡 |
| F03 | 字体切换 | 切换对话区域字体大小 | P0 | 整个对话区域（用户消息+AI回复）生效 |
| F04 | 切换反馈 | Toast 提示当前字体档位 | P0 | 如"已切换为大字体" |
| F05 | 云端同步 | 字体设置存储到服务器 | P0 | 跟随用户账号，跨设备同步 |
| F06 | 移除首页字体设置 | 删除首页"Aa"按钮 | P0 | 首页不再提供字体调节功能 |
| F07 | 移除"我的"字体设置 | 删除"我的"页面字体入口 | P0 | 清理旧的字体设置入口 |

### 2.2 功能详细描述

#### F01：AI 对话页面右上角新增"Aa"图标

- 在 AI 健康咨询的对话页面（`/chat/[sessionId]`）右上角，新增一个"Aa"图标按钮
- 图标样式与之前首页的"Aa"保持一致，大小适中不抢占空间
- 所有对话类型（健康问答、健康自查、中医养生、用药参考、客服）共享同一个字体设置

#### F02：字体选择气泡浮层

- 点击"Aa"图标后，在图标正下方弹出一个小气泡浮层（Popover 样式）
- 浮层内容为简洁文字列表，共三行：
  - **标准**（14px）
  - **大**（18px）
  - **超大**（22px）
- 当前选中的档位带**勾号**或**高亮背景色**标识
- 点击某一档位后立即切换，浮层自动关闭
- 点击浮层外部区域也可关闭浮层

#### F03：字体切换——控制整个对话区域

- 字体大小设置生效范围：**整个对话区域**，包括：
  - 用户自己发送的消息文字
  - AI 回复的消息文字
  - 免责提示文字（disclaimer）
- 不受影响的元素：
  - 顶部导航栏标题
  - 底部输入框
  - 系统提示文字（如时间戳、"正在输入..."等）
  - 知识卡片内的固定样式元素

#### F04：切换反馈 Toast

- 切换字体后，底部弹出轻提示 Toast，显示 1.5 秒后自动消失
- 文案示例：
  - 切换到标准：`已切换为标准字体`
  - 切换到大：`已切换为大字体`
  - 切换到超大：`已切换为超大字体`

#### F05：云端同步

- 用户的字体大小偏好存储在服务器端（数据库），与用户账号绑定
- 所有对话类型统一使用同一个字体设置，不分类型独立存储
- 用户在 A 设备设置了"超大"字体，在 B 设备登录同一账号后，对话字体也应为"超大"
- 未登录用户或新用户默认使用"标准"（14px）
- API 设计：
  - `GET /api/user/font-setting`：获取当前用户的字体设置
  - `PUT /api/user/font-setting`：更新字体设置（参数：`font_size_level`，值为 `standard` / `large` / `extra_large`）

#### F06：移除首页"Aa"字体设置按钮

- 删除首页右上角的"Aa"字体大小调节按钮
- 删除首页的字体大小底部弹窗组件
- 删除首页最外层容器的动态 fontSize 设置逻辑
- 首页所有文字恢复为各自的固定字号

#### F07：移除"我的"页面字体设置入口

- 删除"我的"页面中与字体大小设置相关的入口/菜单项
- 如果"我的"页面有跳转到字体设置页面的链接，一并删除

## 3. 页面/界面设计

### 3.1 页面结构与导航

本需求不涉及新页面，仅修改现有页面：

| 页面 | 改动说明 |
|------|----------|
| AI 对话页面（`/chat/[sessionId]`） | 右上角新增"Aa"图标 + 气泡浮层 |
| 首页（`/(tabs)/home`） | 移除"Aa"按钮及相关逻辑 |
| "我的"页面（`/(tabs)/profile`） | 移除字体设置入口 |

### 3.2 各页面功能说明

#### AI 对话页面——"Aa"图标与气泡浮层

**"Aa"图标**：
- 位置：对话页面顶部导航栏右侧
- 大小：约 24px，颜色与导航栏文字一致
- 交互：点击弹出/收起气泡浮层

**气泡浮层（Popover）**：
- 位置：紧贴"Aa"图标正下方，带小三角箭头指向图标
- 尺寸：宽度自适应（约 120px），高度随内容撑开
- 背景：白色，圆角 8px，带轻微阴影
- 内容：
  ```
  ┌─────────────┐
  │  标准    ✓  │   ← 当前选中项（高亮 + 勾号）
  │─────────────│
  │  大         │
  │─────────────│
  │  超大       │
  └─────────────┘
  ```
- 每行高度约 44px，方便点击
- 选中项文字加粗，右侧显示勾号，背景为浅蓝色或浅绿色高亮
- 分隔线为 1px 浅灰色

## 4. 非功能性需求

### 4.1 性能要求

- 字体切换后对话区域渲染延迟不超过 100ms
- 字体设置 API 响应时间不超过 500ms
- 页面加载时并行获取字体设置，不阻塞对话内容展示

### 4.2 安全要求

- 字体设置 API 需要用户登录认证（Bearer Token）
- 用户只能读写自己的字体设置

### 4.3 兼容性要求

- H5 端（主要）：适配 iOS Safari、Android Chrome、微信内置浏览器
- 小程序端：跟随 H5 端同步实现
- Flutter App 端：跟随 H5 端同步实现

## 5. 业务规则与约束

| 规则编号 | 规则描述 |
|----------|----------|
| BR01 | 字体大小三档：标准（14px）、大（18px）、超大（22px） |
| BR02 | 默认字体大小为"标准"（14px） |
| BR03 | 所有 AI 对话类型共享同一个字体设置，不区分对话类型 |
| BR04 | 字体设置存储在服务器端，跟随用户账号同步 |
| BR05 | 未登录用户使用默认"标准"字体，登录后加载用户偏好 |
| BR06 | 首页和"我的"页面不再提供字体调节功能 |
| BR07 | 首页原有的字体大小相关 system_config 配置项可保留但不再使用 |

## 6. 权限设计

| 角色 | 权限说明 |
|------|----------|
| 普通用户（已登录） | 可读取和修改自己的字体大小设置 |
| 未登录用户 | 使用默认字体（14px），无法修改（不调用 API） |
| 管理员 | 无需额外管理界面，字体设置为用户个人偏好 |

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 获取字体设置接口失败 | 使用默认"标准"字体，不影响正常使用 |
| 保存字体设置接口失败 | Toast 提示"保存失败，请稍后重试"，本地临时生效但下次进入恢复为上次成功保存的值 |
| 用户未登录 | 不显示"Aa"图标，或显示但点击后提示登录 |
| 从旧版本升级 | 首次进入对话页面，字体默认"标准"；旧版首页的字体设置数据不迁移 |
| 网络断开时切换字体 | 本地立即生效，网络恢复后自动同步到服务器（或下次打开时同步） |

## 8. 补充说明

### 8.1 需要删除/清理的内容

1. **首页（H5）**：
   - 右上角"Aa"字体调节按钮
   - 字体大小底部弹窗组件（FontSizeSheet 或类似组件）
   - 页面最外层容器的动态 `fontSize` 样式绑定
   - 相关的 `fontSizeLevel` 状态管理和 localStorage/API 调用

2. **"我的"页面（H5）**：
   - 字体大小设置相关的菜单项/入口

3. **小程序**：
   - 如果首页也有类似的字体调节功能，一并移除

4. **Flutter App**：
   - 如果首页也有类似的字体调节功能，一并移除

### 8.2 数据库设计建议

用户字体偏好可存储在现有的 `system_config` 表（以 `user_font_size_level_{user_id}` 为 key）或在 `User` 模型中新增 `chat_font_size` 字段（推荐后者，查询更高效）。

字段定义：
- 字段名：`chat_font_size`
- 类型：`String(20)`
- 默认值：`standard`
- 可选值：`standard` / `large` / `extra_large`

### 8.3 各端影响范围

| 端 | 改动内容 |
|----|----------|
| 后端（FastAPI） | 新增用户字体设置 API（GET + PUT），User 模型新增字段 |
| H5（Next.js） | 对话页面新增"Aa"入口和浮层；首页和"我的"移除旧入口 |
| 小程序 | 对话页面新增"Aa"入口和选择交互；首页移除旧入口（如有） |
| Flutter App | 对话页面新增"Aa"入口和选择交互；首页移除旧入口（如有） |
| 管理后台 | 无改动 |
