# 后台菜单管理 - 图标选择器扩充优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「首页配置 → 首页菜单管理」中的 Emoji 图标选择器已具备分类浏览（7 大分类）、中文搜索（260+ 标签映射）和智能推荐功能。但在实际使用中，管理员反馈：

1. **图标种类不够丰富**：当前仅有 7 个分类（医疗健康、运动健身、食物饮料、自然植物、活动物品、符号标志、动物），缺少科技、人物、交通、建筑、天气、办公、金融等常见场景的图标
2. **搜索覆盖不全**：部分常用 Emoji（如🤖机器人、🚗汽车、🏠房屋等）未被收录到搜索标签库中，输入对应关键词搜索不到结果
3. **分类浏览不够全面**：想找一些非健康领域的通用图标（如科技类、交通类）时无处可找

本次优化旨在：大幅扩充 Emoji 分类和中文搜索标签库，让管理员拥有更丰富、更全面的图标选择。

### 1.2 目标用户

- 管理后台运营人员（管理员角色）

### 1.3 核心价值

- 将图标分类从 7 个扩充到 **14 个**，新增 7 个高频使用场景分类
- 将可浏览/可搜索的 Emoji 总量从约 **260 个**扩充到约 **450+ 个**
- 中文搜索标签映射同步扩充，确保新增 Emoji 全部可通过中文关键词搜索到
- 关键词映射表同步补充，提升智能推荐命中率

## 3. 页面/界面设计

### 3.1 页面结构与导航

本次优化不新增页面，仅修改现有「首页菜单管理」页面（`admin-web/src/app/(admin)/home-menus/page.tsx`）中的数据常量。

### 3.2 各页面功能说明

#### 3.2.1 Emoji 选择面板 Modal（仅数据扩充，交互不变）

**不变的部分**：
- Modal 整体结构和布局保持不变（搜索栏 → 推荐区 → 分类标签栏 → Emoji 网格 → 已选择提示）
- Modal 尺寸、交互方式、选中效果等均保持不变
- 搜索逻辑、推荐逻辑保持不变

**变化的部分**：

1. **分类标签栏**
   - 从 7 个分类增加到 14 个分类
   - 由于分类增多，标签栏支持自动换行展示（`flexWrap: 'wrap'` 已支持）
   - 默认选中第一个分类（医疗健康），与当前行为一致

2. **Emoji 网格区域**
   - 切换到新增分类时，展示对应分类下的 Emoji
   - 每个新增分类展示 20~30 个 Emoji

3. **搜索结果**
   - 由于标签库扩充，相同关键词可能返回更多搜索结果
   - 搜索如"机器人"、"汽车"、"学校"等之前搜不到的关键词，现在可以返回结果

## 4. 数据模型设计

### 4.1 数据实体与字段

本次优化为**纯前端改造**，不涉及数据库字段变更。

`home_menu_items.icon_content` 字段（VARCHAR，支持 UTF-8 4 字节 Emoji 存储）保持不变。

### 4.2 实体关系

无变更。

## 5. 接口需求

### 5.1 接口清单

本次优化为**纯前端改造**，不涉及后端接口变更。

现有接口 `POST /api/admin/home-menus` 和 `PUT /api/admin/home-menus/{id}` 的 `icon_content` 字段仍然接收 Emoji 字符串，无需修改。

## 6. 非功能性需求

### 6.1 性能要求

- Emoji 标签库数据为前端静态常量，无额外网络请求
- 标签库扩充后搜索仍为内存过滤，响应时间 < 50ms
- 新增的分类数据量较小（约 150~180 个 Emoji），不会对页面加载产生可感知的性能影响

### 6.2 安全要求

- 无新增安全风险（纯前端改造）

### 6.3 兼容性要求

- 确保新增 Emoji 在 Chrome、Edge、Firefox 主流浏览器中正确渲染
- 部分较新的 Emoji（如🤖🛰️等）在 Windows 7 等老旧系统上可能显示为方块，属于系统限制，不做额外处理

## 7. 业务规则与约束

1. 已有菜单的 Emoji 图标保持不变，不受影响
2. 现有 7 个分类的 Emoji 列表不做删减，只做新增
3. 搜索逻辑保持现有实现不变（`includes` 模糊匹配 + 300ms 防抖）
4. 智能推荐逻辑保持不变（菜单名称包含关键词即匹配，优先展示第一个匹配组）
5. 不支持自定义图标上传
6. 不做已有菜单图标的批量刷新/替换

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 14 个分类标签栏超出一行宽度 | 自动换行显示（现有 `flexWrap: 'wrap'` 样式已支持） |
| 新增 Emoji 在老旧系统上显示为方块 | 属于系统限制，不做处理。建议使用 Windows 10+ / macOS 10.14+ |
| 已有菜单使用的 Emoji 图标不受影响 | 数据库已有数据完全兼容，无需迁移 |
