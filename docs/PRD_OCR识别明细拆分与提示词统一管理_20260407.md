# OCR识别明细拆分与提示词统一管理 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

bini-health 健康管理平台目前已拥有完善的 OCR 多厂商配置与智能识别系统（支持百度、腾讯、阿里三大 OCR 厂商），包含"体检报告识别"和"拍照识药"两大预设场景。

现有系统存在以下待优化点：

1. **识别记录混杂**：所有 OCR 识别记录（体检报告、拍照识药等）都混在同一个"识别记录"Tab 中，缺乏按业务场景的独立管理视图，不便于管理员针对性查看和分析
2. **业务字段缺失**：现有 `OcrCallRecord` 表仅记录通用的 OCR 调用信息（厂商、状态、原始文字、AI结果），缺少体检报告和药品识别的业务维度字段（如异常指标数、药品名称、分类等）
3. **提示词分散管理**：每个 OCR 识别场景模板（`OcrSceneTemplate`）各自维护独立的 `prompt_content` 字段，导致提示词分散在各场景中，管理不统一，无法复用已有的「AI配置中心 → 提示词配置」体系

本次需求通过拆分独立的业务明细表和菜单、统一提示词管理入口，解决上述三个问题。

### 1.2 目标用户

- **管理员**：在管理后台查看、筛选体检报告解读明细和拍照识药记录明细，统一管理 OCR 相关提示词

### 1.3 核心价值

- 按业务场景独立查看识别明细，提升管理效率
- 增加业务维度字段（异常指标、药品分类等），丰富数据洞察
- 统计卡片提供直观的业务数据概览
- 提示词统一管理，消除分散配置的维护成本

### 2.2 功能详细描述

#### F01-F04：体检报告解读明细

**菜单位置**：AI管理 → 体检报告解读明细（与 AI模型配置、OCR识别配置 等并列）

**列表页字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 解读时间 | DateTime | 记录创建时间 |
| 关联用户 | String | 用户昵称/手机号（支持搜索） |
| 报告类型 | String | 如：血常规、肝功能、综合体检等 |
| 异常指标数 | Integer | 本次报告中识别出的异常指标数量 |
| 解读摘要 | String | AI 解读的简短文字摘要 |
| 识别状态 | String | 正常 / 异常 |
| 使用厂商 | String | OCR 识别使用的厂商名称 |

**统计卡片（页面顶部）**：

| 指标 | 说明 |
|------|------|
| 总报告数 | 累计所有体检报告解读记录总数 |
| 今日新增 | 今天新产生的解读记录数 |
| 异常报告数 | 状态为"异常"（含异常指标）的报告总数 |
| 本月解读数 | 当月累计解读记录数 |

**筛选条件**：

| 筛选项 | 类型 | 说明 |
|--------|------|------|
| 时间范围 | DateRangePicker | 按解读时间筛选 |
| 用户搜索 | Input | 支持手机号或昵称模糊搜索 |
| 报告类型 | Select | 下拉选择报告类型（血常规、肝功能等） |
| 状态筛选 | Select | 正常 / 异常 |

**详情弹窗（Drawer）**：

点击列表中某条记录，从右侧弹出 Drawer，展示以下完整信息：

| 区域 | 内容 |
|------|------|
| 用户信息 | 用户昵称、手机号、头像 |
| 解读时间 | 完整的日期时间 |
| 原始图片 | 上传的体检报告原始图片（支持放大查看） |
| OCR原始文字 | OCR 提取出的原始文字内容 |
| AI结构化分析 | AI 结构化分析结果（JSON 格式化展示，含各项指标分类） |
| 异常指标列表 | 所有被标记为异常的指标，以表格/列表形式展示（指标名、实际值、参考范围、异常说明） |

**操作**：纯查看，不需要编辑或删除操作。

#### F09-F11：OCR提示词统一管理

**F09：合并到AI配置中心 → 提示词配置**

在现有的「AI配置中心 → 提示词配置」页面中，新增 OCR 识别场景对应的 Tab：

| 新增Tab | chat_type 值 | 说明 |
|---------|-------------|------|
| 体检报告识别 | `ocr_checkup_report` | 体检报告OCR识别后的AI结构化分析提示词 |
| 拍照识药 | `ocr_drug_identify` | 拍照识药OCR识别后的AI结构化分析提示词 |

与现有的健康咨询、健康自查、中医养生、用药参考、智能客服等 Tab 并列展示。

**F10：移除 OcrSceneTemplate 的 prompt_content 字段**

- `OcrSceneTemplate` 模型中完全移除 `prompt_content` 字段
- OCR 识别流程中，不再从 `OcrSceneTemplate` 读取提示词，改为通过 `chat_type` 映射从 `AiPromptConfig` 表读取
- 映射关系维护在系统内部：
  - `scene_name = "体检报告识别"` → `chat_type = "ocr_checkup_report"`
  - `scene_name = "拍照识药"` → `chat_type = "ocr_drug_identify"`
- OCR识别配置页面中场景模板的编辑表单不再包含提示词输入框

**F11：新增场景自动联动**

当管理员在「OCR识别配置」中创建新的 OCR 识别场景模板时：

1. 系统自动在 `ai_prompt_configs` 表中创建一条新记录：
   - `chat_type`：根据 `scene_name` 自动生成（如 `ocr_{scene_name_pinyin}`，或 `ocr_custom_{id}`）
   - `display_name`：使用 `scene_name` 值
   - `system_prompt`：留空
2. 创建成功后弹出提示信息："已自动创建对应提示词配置，请前往【AI配置中心 → 提示词配置】完善内容"

---

#### F12：移除原识别记录Tab

- 在 OCR 识别配置页面（`admin-web/src/app/(admin)/ocr-config/page.tsx`）中，移除原有的"识别记录管理"区块/Tab
- 所有识别记录的查看功能由两个新的明细菜单完全承接

---

#### F13：数据双写机制

当用户通过客户端（H5/小程序/Flutter）使用体检报告解读或拍照识药功能时：

1. 照常写入 `ocr_call_records` 表（保持原有逻辑不变）
2. **同时**写入对应的独立明细表：
   - 体检报告场景 → 写入 `checkup_report_details` 表
   - 拍照识药场景 → 写入 `drug_identify_details` 表
3. 明细表中的业务字段从 AI 结构化结果中提取

## 3. 页面/界面设计

### 3.2 各页面功能说明

#### 3.2.1 体检报告解读明细页面

## 6. 非功能性需求

### 6.1 性能要求

- 列表页查询响应时间 ≤ 500ms
- 统计卡片数据查询响应时间 ≤ 1s
- 双写操作不应显著增加业务接口的响应时间（建议异步写入明细表或使用事务提交）

### 6.2 安全要求

- 所有明细查询接口仅限管理员角色访问（`require_role("admin")`）
- 用户手机号在列表中脱敏显示（如 `138****8888`），详情弹窗中完整显示

### 6.3 兼容性要求

- 管理后台页面兼容 Chrome、Firefox、Edge 最新版本
- 双写机制需兼容已有的 OCR 调用记录写入逻辑，不影响原有功能

---

## 7. 业务规则与约束

1. **数据双写一致性**：业务调用时同时写入 `ocr_call_records` 和对应的明细表，使用数据库事务保证一致性。若明细表写入失败，不应阻断主流程（降级处理）
2. **报告类型推断**：`report_type` 从 AI 结构化分析结果中自动提取，无需用户手动选择
3. **异常判定**：`abnormal_count > 0` 时，`status` 自动标记为 `abnormal`
4. **药品分类推断**：`drug_category` 从 AI 结构化分析结果中自动提取
5. **提示词映射**：OCR 场景模板与 `AiPromptConfig` 的映射关系由系统内部维护，管理员无需手动关联
6. **自动联动提示**：创建 OCR 场景后弹出的提醒信息为前端 Notification 或 Message 组件，不阻断操作流程
7. **提示词为空处理**：如果关联的 `AiPromptConfig` 中 `system_prompt` 为空，OCR 识别时跳过 AI 分析步骤，仅返回 OCR 原始文字

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 明细表写入失败 | 记录错误日志，不阻断主流程（OCR调用记录正常写入），后续可通过补偿机制修复 |
| AI结构化结果中无法提取业务字段 | 对应字段设为 null，不影响记录创建 |
| 删除 OCR 场景模板时 | 不自动删除对应的 `AiPromptConfig` 记录（保留提示词数据） |
| 用户未登录使用OCR功能 | `user_id`、`user_phone`、`user_nickname` 为 null，明细记录正常创建 |
| 统计卡片数据量大 | 使用数据库聚合查询，避免全量数据加载 |
| 提示词配置中删除 OCR 对应条目 | OCR 识别时找不到提示词，跳过 AI 分析，仅返回 OCR 原始文字 |
