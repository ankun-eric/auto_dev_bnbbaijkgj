# 语音搜索体验优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前 H5 搜索页面的语音录入功能存在以下体验问题：

1. **录音触发方式不友好**：需要长按麦克风图标 300 毫秒才能开始录音，操作门槛高，用户不容易发现和使用
2. **录音结束方式不直观**：录音中只显示"松开结束"文字提示，用户操作不够明确
3. **提示文案生硬**：如"说话时间太短"、"语音识别失败，请重试"等错误提示缺乏引导性和亲和力

本次优化旨在降低语音搜索的使用门槛，提升录音过程的视觉反馈和交互友好度，让用户在使用语音搜索时感到温暖和自然。

### 1.2 目标用户

- 通过 H5 网页端（尤其是 iPhone 微信内置浏览器）使用搜索功能的用户
- 习惯语音输入的用户（如中老年用户、不方便打字的场景）

### 1.3 核心价值

- **降低使用门槛**：单击即可开始录音，无需长按
- **提升操作明确性**：用醒目的大按钮替代文字提示，用户一目了然
- **增强情感体验**：全流程采用温暖亲切的文案风格，提升用户好感度
- **丰富视觉效果**：脉动呼吸按钮 + 更丰富的声波动画，让录音过程更生动

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 录音触发 | 单击即开始录音 | P0 | 将长按 300ms 改为单击触发 |
| F02 | 录音结束 | 脉动呼吸大按钮结束录音 | P0 | 替代原"松开结束"文字提示 |
| F03 | 声波动画 | 7~9 根声波竖条 | P1 | 从 5 根增加，排列在计时器和结束按钮之间 |
| F04 | 提示文案 | 全流程温暖亲切风格文案 | P0 | 覆盖所有状态的提示文案 |

### 2.2 功能详细描述

#### F01：录音触发方式优化

**当前行为**：用户需长按麦克风图标 300 毫秒，长按计时器到期后才开始录音。

**目标行为**：用户**单击**麦克风图标，即弹出录音界面并立即开始录音。

**交互说明**：
- 移除 `LONG_PRESS_MS = 300` 的长按延迟逻辑
- 移除 `onTouchStart` / `onTouchEnd` / `onMouseDown` / `onMouseUp` / `onMouseLeave` 组合事件绑定
- 改为简单的 `onClick` 事件，点击后立即调用 `startRecording()`
- 录音开始后松手不再停止录音（原"松开结束"逻辑移除）

#### F02：脉动呼吸大按钮结束录音

**按钮样式**：
- **形状**：圆形大按钮，直径约 80px
- **颜色**：白色背景 + 绿色边框（`#52c41a`）
- **呼吸效果**：通过 CSS `box-shadow` 外发光实现脉动呼吸动画，`box-shadow` 颜色为 `rgba(82, 196, 26, 0.4)`，按约 1.5 秒一个周期缩放/扩散
- **按钮内容**：停止方块图标（■） + 文字"点我结束~"
  - 方块图标：8x8px 白色填充方块（或深色），居于文字上方
  - 文字：字号 12px，颜色与边框一致（`#52c41a`）
- **点击行为**：点击后停止录音，进入语音识别阶段

**按钮位置**：位于录音弹层的中下部区域，声波竖条下方。

#### F03：声波动画优化

**当前**：5 根竖条，位于计时器下方。

**目标**：
- **数量**：增加到 **7~9 根**（推荐 7 根）
- **排列位置**：在**计时器**和**结束按钮**之间，作为中间过渡层
  - 上方：录音计时器
  - 中间：声波竖条
  - 下方：脉动呼吸结束按钮
- **样式**：保持原有渐变色（`#52c41a` 到 `#13c2c2`），宽度和间距可微调以适配更多竖条

#### F04：全流程文案优化

将所有语音搜索相关提示替换为温暖亲切风格：

| 位置/状态 | 原文案 | 新文案 |
|-----------|--------|--------|
| 录音中提示 | 松开结束 | （移除，改为大按钮"■ 点我结束~"） |
| 录音时间太短 | 说话时间太短 | 说的太快了，再试一次吧~ |
| 识别接口失败 | 语音识别失败，请重试 | 没听清楚，再说一次好吗~ |
| 识别文字为空 | 未能识别语音内容 | 好像没听到声音哦，再试试~ |
| 麦克风权限被拒 | 麦克风权限被拒绝 | 需要你允许使用麦克风才能听到你说话哦~ |
| 无法启动录音 | 无法启动录音 | 录音启动失败了，再试一次吧~ |
| 正在识别 | 正在识别... | 我在认真听，马上就好~ |
| 自动搜索倒计时 | {N}秒后自动搜索 | 听到啦~ {N}秒后帮你搜索 |
| 取消自动搜索 | 点击取消 | 先不搜了 |
| 错误后重试 | 重试 | 再来一次~ |

## 3. 页面/界面设计

### 3.1 页面结构与导航

本次优化不涉及页面结构变更，仅修改搜索页（`/search`）内的语音录入弹层（Voice recording overlay）。

### 3.2 录音弹层布局设计

#### 录音中状态（overlayState === 'recording'）

从上到下的布局顺序：

```
┌─────────────────────────────────────┐
│                              [×]    │  ← 关闭按钮（保持不变）
│                                     │
│            {N}s / 15s               │  ← 录音计时器
│                                     │
│         ┃ ┃ ┃ ┃ ┃ ┃ ┃              │  ← 7~9 根声波竖条（中间过渡层）
│                                     │
│          ╭─────────╮                │
│          │    ■    │  ←─ 脉动呼吸   │  ← 白色圆形大按钮
│          │点我结束~ │     外发光     │     绿色边框 + 外发光阴影
│          ╰─────────╯                │
│                                     │
└─────────────────────────────────────┘
```

#### 识别中状态（overlayState === 'recognizing'）

保持现有布局，仅文案更新：
- Loading 动画保持不变
- 文案改为"我在认真听，马上就好~"

#### 错误状态（overlayState === 'error'）

保持现有布局，仅文案更新：
- 错误图标保持不变
- 错误消息改为对应的温暖亲切文案
- 重试按钮文案改为"再来一次~"

## 4. 非功能性需求

### 4.1 性能要求

- 脉动呼吸动画使用 CSS `animation` 实现，不使用 JavaScript 定时器，避免性能开销
- 声波竖条增加到 7~9 根后，`requestAnimationFrame` 回调中的频率数据分段计算需同步调整

### 4.2 安全要求

- 无新增安全要求，录音权限获取流程保持不变

### 4.3 兼容性要求

- **主要目标浏览器**：iPhone 微信内置浏览器（WeChat WebView）
- 需兼容 Safari、Chrome 等主流移动端浏览器
- 脉动呼吸动画需兼容 `-webkit-` 前缀

## 5. 业务规则与约束

1. **录音最长时间**：保持 15 秒不变
2. **录音最短时间**：保持 1 秒不变（低于 1 秒显示"说的太快了，再试一次吧~"）
3. **自动搜索倒计时**：保持 2 秒不变
4. **录音格式**：录音格式检测逻辑不变（注意：`mp4` 格式映射问题为另一个独立 Bug，不在本需求范围内）

## 6. 权限设计

| 角色 | 权限说明 |
|------|----------|
| 所有用户 | 均可使用语音搜索功能（无需登录） |

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户快速连续点击麦克风 | 如果已在录音/识别中，忽略重复点击（保持现有 `overlayState !== 'idle'` 判断） |
| 录音中点击关闭按钮 | 停止录音并关闭弹层，不进行识别（保持现有行为） |
| 录音中达到 15 秒上限 | 自动停止录音并进入识别（保持现有行为） |
| 浏览器不支持麦克风 | 麦克风按钮不显示（保持现有行为） |
| ASR 服务不可用 | 麦克风按钮不显示（保持现有行为） |

## 8. 补充说明

### 8.1 脉动呼吸动画参考 CSS

```css
@keyframes pulse-glow {
  0%, 100% {
    box-shadow: 0 0 0 0 rgba(82, 196, 26, 0.4);
    transform: scale(1);
  }
  50% {
    box-shadow: 0 0 20px 10px rgba(82, 196, 26, 0.2);
    transform: scale(1.05);
  }
}
```

- 动画周期：约 1.5 秒
- 应用于结束按钮的 `animation` 属性：`pulse-glow 1.5s ease-in-out infinite`

### 8.2 涉及修改的文件

| 文件 | 改动范围 |
|------|----------|
| `h5-web/src/app/search/page.tsx` | 录音触发逻辑、弹层 UI、文案、声波竖条数量、结束按钮 |

### 8.3 与现有 Bug 的关系

本需求与已知的"mp4 not in list"语音识别 Bug 是**独立**的两个任务：
- 本 PRD 聚焦于**前端交互与文案优化**
- mp4 格式映射问题需要**前端 + 后端双修**（前端 `mimeToFormat()` 中 `mp4 → m4a` + 后端 `_ASR_FORMAT_MAP` 增加 `"mp4": "m4a"`）
- 建议两个任务可以在同一次迭代中一并完成
