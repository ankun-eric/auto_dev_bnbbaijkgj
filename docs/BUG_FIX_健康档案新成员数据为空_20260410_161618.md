# 健康档案 — 新添加家庭成员档案数据为空 Bug 修复方案文档

## 1. Bug 发生背景

### 1.1 项目概述

**bini-health**（缤你健康）是一个多端健康管理平台，包含后端（FastAPI）、H5 网页端（Next.js）、微信小程序端和 Flutter App 端。系统的核心功能之一是**健康档案管理**，用户可以为自己和家庭成员建立健康档案，记录姓名、性别、出生日期、身高、体重、血型、慢性病史、过敏史、遗传病史等信息。

### 1.2 涉及功能模块

- **家庭成员管理**（`/api/family/members`）：添加、查询、更新、删除家庭成员
- **健康档案管理**（`/api/health/profile/member/{member_id}`）：按成员查看和编辑健康档案
- **健康档案页面**（前端 health-profile 页面）：展示成员列表 Tab 切换栏 + 健康档案详情内容

### 1.3 发现时间与发现方式

- 发现时间：一直存在
- 发现方式：用户操作发现，在健康档案页面添加新家庭成员后，点击该成员的 Tab 圆形图标，下方健康档案信息全部为空

## 2. Bug 描述

### 2.1 错误现象

在健康档案页面，用户通过"+"按钮添加了一个新的家庭成员（如"爸爸"），在添加弹窗中填写了完整的基本信息（姓名、性别、出生日期等），系统提示添加成功。但随后点击该新成员的 Tab 圆形图标时，下方的**健康档案信息**（姓名、性别、出生日期、身高、体重、血型、慢性病史、过敏史等）全部为空，显示"暂无档案数据"或所有字段处于未填写状态。

**"本人"的档案正常显示，只有新添加的家庭成员存在此问题。**

**多端均存在此 Bug**（H5 网页端、微信小程序端、Flutter App 端）。

### 2.2 重现步骤

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 进入首页 → 点击健康档案页面 | 正常显示页面，成员 Tab 栏展示"本人" | 正常 ✓ |
| 2 | 点击"+"按钮，打开添加家庭成员弹窗 | 弹窗弹出，显示关系选择列表 | 正常 ✓ |
| 3 | 选择一个关系（如"爸爸"），填写完整信息：姓名、性别、出生日期等 | 信息填写界面正常 | 正常 ✓ |
| 4 | 点击"确认添加" | 系统提示添加成功，新成员出现在 Tab 栏 | 正常 ✓ |
| 5 | 点击新成员的 Tab 圆形图标 | 下方显示该成员的健康档案信息（姓名、性别、出生日期等应为刚才填写的内容） | **异常 ✗**：档案内容全部为空 |

### 2.3 影响范围

- **受影响的端**：H5 网页端、微信小程序端、Flutter App 端（全端）
- **受影响的功能**：
  - 新添加家庭成员的健康档案查看功能完全不可用
  - 用户需要添加成员后再手动重新在档案中填写一遍信息，体验极差
  - 影响后续依赖家庭成员健康档案数据的功能（如健康报告、AI 健康建议等）
- **受影响的用户**：所有添加过家庭成员的用户

## 3. 预期正确效果

1. **添加家庭成员时**：用户在添加弹窗中填写的所有信息（姓名、性别、出生日期、身高、体重、既往病史、过敏史等），应**同时写入** `FamilyMember` 表和 `HealthProfile` 表
2. **切换到新成员 Tab 后**：应立即显示刚才填写的完整健康档案信息，无需用户重复填写
3. **数据一致性**：`FamilyMember` 和 `HealthProfile` 之间的姓名、性别、生日等重叠字段应保持同步

## 4. 补充说明

### 4.1 根因分析

通过代码审查，已定位到 Bug 的根本原因：

**添加成员的后端接口** `POST /api/family/members`（`backend/app/api/family.py`）在创建 `FamilyMember` 记录时，**没有同步创建对应的 `HealthProfile` 记录**。用户填写的姓名、性别、出生日期、身高、体重、既往病史、过敏史等信息只保存到了 `FamilyMember` 表中。

而**获取成员档案的接口** `GET /api/health/profile/member/{member_id}`（`backend/app/api/health_profile.py`）在查询不到 `HealthProfile` 时，虽然会自动创建一条新记录，但只设置了 `user_id` 和 `family_member_id`，**所有其他字段均为 None**，导致前端展示的档案内容全部为空。

**数据流断裂示意：**

```
添加成员时:
  前端填写信息 → POST /api/family/members → 只写入 FamilyMember 表 ✓
                                          → 未写入 HealthProfile 表 ✗

查看档案时:
  点击成员 Tab → GET /api/health/profile/member/{id} → HealthProfile 不存在
              → 自动创建空 HealthProfile → 前端展示全空 ✗
```

### 4.2 修复要点

1. **后端** `POST /api/family/members` 接口：在成功创建 `FamilyMember` 后，应立即创建对应的 `HealthProfile` 记录，并将用户填写的姓名、性别、出生日期、身高、体重、既往病史、过敏史等信息同步写入
2. **数据回填**：对已有的 `FamilyMember` 记录但缺少 `HealthProfile` 的历史数据，可在 `GET /api/health/profile/member/{member_id}` 自动创建 `HealthProfile` 时，从对应的 `FamilyMember` 记录中回填 name、gender、birthday、height、weight、medical_histories、allergies 等字段
3. **全端生效**：由于此 Bug 的根因在后端，修复后端后，所有前端（H5、小程序、App）将自动修复，无需逐端改动

### 4.3 涉及文件

| 文件路径 | 修改说明 |
|---------|---------|
| `backend/app/api/family.py` | `add_family_member` 函数：添加成员后同步创建 HealthProfile |
| `backend/app/api/health_profile.py` | `get_member_health_profile` 函数：自动创建 HealthProfile 时从 FamilyMember 回填数据 |
