# 宾尼小康 AI 健康管家 — COS 全局 500 错误修复 + 存储配置页面重构

## 1. Bug 发生背景

### 1.1 项目概述

宾尼小康 AI 健康管家（bini-health）是全栈健康管理平台，技术栈为 FastAPI + SQLAlchemy + MySQL（后端）、Next.js + Ant Design（管理后台与 H5 前端）、Flutter（移动端）、微信小程序。近期完成了腾讯云 COS 对象存储功能的全量接入。

### 1.2 涉及功能模块

- **COS 基础配置保存**（`PUT /api/admin/cos/config`）
- **COS 连接测试**（`POST /api/admin/cos/test`）
- **上传限制配置保存**（`PUT /api/admin/cos/upload-limits`）
- **文件上传路径分配**（`backend/app/utils/cos_helper.py`）
- **管理后台存储配置页面**（`admin-web/src/app/(admin)/cos-config/page.tsx`）

### 1.3 发现时间与发现方式

- **发现时间**：部署完腾讯云 COS 存储功能之后立即出现
- **发现方式**：用户在管理后台操作存储配置时发现

## 2. Bug 描述（全局 500 错误）

### 2.1 错误现象

1. 管理后台「存储配置」页面，修改配置内容后点击「保存」按钮，前端提示 `Request failed with status code 500`
2. 测试连接按钮和上传限制保存也均报 500 错误
3. 系统其他模块的保存操作也出现 500 错误（全局性问题）
4. GET 读取类接口正常工作（页面能正常展示数据）
5. 问题在部署完腾讯云 COS 存储功能之后立即出现

### 2.2 重现步骤

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 部署腾讯云 COS 存储功能到服务器 | 系统正常运行 | 部署后出现全局性异常 |
| 2 | 打开管理后台 → 存储配置页面 | 页面正常加载 | ✅ 页面正常加载，数据正常显示 |
| 3 | 修改任意字段，点击「保存」 | 提示"配置更新成功" | ❌ 提示 500 错误 |
| 4 | 点击「测试连接」 | 返回测试结果 | ❌ 500 错误 |
| 5 | 修改上传限制配置，点击保存 | 提示"上传限制已更新" | ❌ 500 错误 |

### 2.3 影响范围

- **直接影响**：管理后台所有涉及 COS 的写操作全部不可用
- **间接影响**：系统其他模块的保存操作也报 500 错误（session 污染）
- **不影响**：GET 读取类接口正常

### 2.4 根因分析

**最大可能原因：数据库 Schema 不同步**

本次 COS 功能部署为 `cos_configs` 表新增了 `cdn_domain`、`cdn_protocol`、`path_prefix`、`test_passed` 四个字段，但 SQLAlchemy 的 `create_all` **不会自动为已存在的表添加新列**。导致：

1. 所有涉及这些新字段的写操作抛出 `OperationalError`（Unknown column）
2. 数据库 session 进入错误状态，通过 session 污染波及其他模块的接口
3. 因此表现为"全局性 500 错误"

**佐证**：
- 时间线吻合：部署后立即出现
- GET 正常、写操作异常：说明数据库连接没问题，但写操作触碰了不存在的字段
- 全局波及：session 异常后影响所有后续写请求

## 3. 需求变更一：路径前缀拆分

### 3.1 变更说明

将原来的统一路径前缀（`path_prefix`）彻底移除，拆分为 3 个独立的分类路径前缀。

### 3.2 变更详情

| 维度 | 变更前 | 变更后 |
|------|--------|--------|
| **字段** | 1 个统一路径前缀 `path_prefix` | 3 个分类前缀：`image_prefix`、`video_prefix`、`file_prefix` |
| **前端展示** | 1 个输入框 | 3 个独立输入框（图片路径前缀、视频路径前缀、文件路径前缀） |
| **默认值** | 空字符串 | `images/`、`videos/`、`files/` |
| **是否可清空** | 可以 | **不可以**，必须有值（至少使用默认值） |
| **前端位置** | Region 下方、COS 启用开关上方 | 同原位置，1 个改为 3 个 |
| **path_prefix 字段** | 存在 | **彻底删除**（数据库模型和接口中均移除） |

### 3.3 上传分配逻辑

按文件 MIME 类型自动判断使用哪个前缀：

| 文件类型 | MIME 类型匹配 | 使用前缀 | 默认值 |
|----------|--------------|----------|--------|
| 图片 | `image/*` | `image_prefix` | `images/` |
| 视频 | `video/*` | `video_prefix` | `videos/` |
| 其他文件 | 除 image/video 外的所有类型 | `file_prefix` | `files/` |

### 3.4 校验规则

- 3 个路径前缀均为**必填项**，不允许为空
- 前端需在保存前校验，若用户清空了前缀值，应提示"路径前缀不能为空"或自动回填默认值
- 后端接口也需校验，若传入空值则拒绝并返回明确错误提示

## 4. 需求变更二：存储配置页面 Tab 拆分

### 4.1 变更说明

将当前存储配置页面中的"上传限制配置"和"数据迁移"两个功能区块从「存储配置」Tab 中拆出，各自成为独立的 Tab 页。

### 4.2 当前页面结构（变更前）

```
[存储配置]  [文件管理]
     │           │
     ├ COS基础配置 (Card)         ├ 文件统计 (Card)
     ├ CDN加速配置 (Card)
     ├ 上传限制配置 (Card)   ← 需拆出
     ├ 操作区 (Card)
     └ 数据迁移 (Card)       ← 需拆出
```

### 4.3 拆分后页面结构（变更后）

```
[存储配置]  [上传限制]  [数据迁移]  [文件管理]
     │           │           │           │
     ├ COS基础配置   ├ 上传限制配置   ├ 扫描/迁移操作   ├ 文件统计
     ├ CDN加速配置
     └ 操作区(保存/测试连接)
```

### 4.4 Tab 详细定义

| 序号 | Tab 名称 | 图标 | 包含内容 | 说明 |
|------|----------|------|----------|------|
| 1 | **存储配置** | 自动分配合适图标（如云存储/设置相关） | COS 基础配置 + CDN 加速配置 + 操作区（保存按钮、测试连接按钮） | 保留原有样式和交互 |
| 2 | **上传限制** | 自动分配合适图标（如上传/限制相关） | 上传限制配置表格（7 个业务模块的图片/视频大小限制、保存按钮） | 从原 Card 独立为 Tab 页，保持现有样式 |
| 3 | **数据迁移** | 自动分配合适图标（如迁移/同步相关） | 扫描待迁移文件、开始迁移、查看迁移进度、重试失败文件 | 从原 Card 独立为 Tab 页，保持现有样式 |
| 4 | **文件管理** | 保持不变 | 文件统计概览 | 保持不变 |

### 4.5 样式要求

- 各 Tab 页内容保持现有样式不变，仅从 Card 提升为独立 Tab 页
- Tab 导航栏保持与现有 `[存储配置] [文件管理]` 一致的风格

## 5. 修复方案（Bug 修复 + 两项需求变更统一为一期）

### 5.1 修复数据库 Schema 不同步

1. 连接生产数据库，为 `cos_configs` 表补齐缺失的新增字段
2. 或删除 `cos_configs` 表后重启后端容器，让 `create_all` 重新创建完整表结构
3. 在后端启动流程中增加 Schema 自动同步机制：对比 ORM 模型定义与实际表结构，自动 ALTER TABLE 补齐缺失字段

### 5.2 彻底删除 `path_prefix` 字段

| 改动点 | 文件 | 操作 |
|--------|------|------|
| 数据库模型 | `backend/app/models/models.py` | 从 `CosConfig` 模型中移除 `path_prefix` 字段 |
| Schema | `backend/app/schemas/cos.py` | 从请求/响应 Schema 中移除 `path_prefix` |
| 配置保存接口 | `backend/app/api/cos.py` | 接口中不再读写 `path_prefix` |
| 上传工具 | `backend/app/utils/cos_helper.py` | 移除 `path_prefix` 优先级逻辑，改为纯粹使用分类前缀 |
| 前端页面 | `admin-web/src/app/(admin)/cos-config/page.tsx` | 移除统一路径前缀输入框 |
| 数据库 | 生产环境 | `ALTER TABLE cos_configs DROP COLUMN path_prefix`（在补齐字段后执行） |

### 5.3 启用 3 个分类路径前缀

| 改动点 | 文件 | 操作 |
|--------|------|------|
| 数据库模型 | `backend/app/models/models.py` | 确认 `image_prefix`、`video_prefix`、`file_prefix` 字段已存在且默认值分别为 `images/`、`videos/`、`files/` |
| Schema | `backend/app/schemas/cos.py` | 确认三个前缀字段在请求/响应 Schema 中已包含 |
| 配置保存接口 | `backend/app/api/cos.py` | 确认保存/读取时正确处理三个前缀字段 |
| 上传工具 | `backend/app/utils/cos_helper.py` | 修改路径生成逻辑：按 MIME 类型自动分配前缀 |
| 前端页面 | `admin-web/src/app/(admin)/cos-config/page.tsx` | 在原统一路径前缀位置展示 3 个独立输入框，带必填校验 |
| 后端校验 | `backend/app/api/cos.py` | 保存时校验三个前缀不可为空 |

### 5.4 存储配置页面 Tab 拆分

| 改动点 | 文件 | 操作 |
|--------|------|------|
| 前端页面 | `admin-web/src/app/(admin)/cos-config/page.tsx` | 将 Tab 从 2 个扩展为 4 个：`存储配置` / `上传限制` / `数据迁移` / `文件管理` |
| 存储配置 Tab | 同上 | 只保留 COS 基础配置 + CDN 加速配置 + 操作区（保存/测试连接） |
| 上传限制 Tab | 同上 | 将上传限制配置 Card 内容移入独立 Tab 页 |
| 数据迁移 Tab | 同上 | 将数据迁移 Card 内容移入独立 Tab 页 |
| Tab 图标 | 同上 | 为每个 Tab 自动分配合适的 Ant Design 图标 |

### 5.5 增强异常处理

1. 为 COS 相关接口添加 try-except 包裹，捕获数据库异常并返回明确错误信息
2. 确保异常发生时正确回滚 session，避免污染后续请求

### 5.6 Docker 部署验证

1. 重新部署后端容器，确认数据库表结构与 ORM 模型完全一致
2. 验证所有 COS 相关接口（保存配置、测试连接、更新上传限制）均可正常调用
3. 验证系统其他模块的保存操作恢复正常
4. 验证上传文件时路径前缀按 MIME 类型正确分配
5. 验证 4 个 Tab 页均可正常切换和操作
