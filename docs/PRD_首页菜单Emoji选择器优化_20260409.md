# 首页菜单 Emoji 选择器优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前管理后台「首页配置 → 首页菜单管理」中，新增和编辑菜单记录时有一套 Emoji 图标推荐机制。该机制存在以下问题：

1. **关键词映射覆盖率极低**：当前仅有 11 组关键词（体检/运动/饮食/心理/睡眠/家庭/预约/报告/购物/专家/健康），几乎大部分菜单名称都无法匹配到合适的推荐，只能显示兜底的默认 Emoji（⭐ 📋 🔖 💡 🎯）
2. **选择方式单一**：当推荐不满意时，用户只能手动输入 Emoji 字符，操作不便且容易出错
3. **手动输入限制问题**：`maxLength={4}` 的限制可能导致某些复合 Emoji 被截断

本次优化旨在：大幅扩充关键词映射覆盖范围，并新增完整的 Emoji 选择面板（Modal），让管理员能够快速、准确地为菜单项选择合适的 Emoji 图标。

### 1.2 目标用户

- 管理后台运营人员（管理员角色）

### 1.3 核心价值

- 提升 Emoji 推荐的匹配准确率，从当前约 20% 提升至 90% 以上
- 提供完整的 Emoji 浏览和搜索能力，覆盖推荐未命中的场景
- 统一 Emoji 选择入口，去掉手动输入框，降低误操作风险

#### F02: Emoji 选择面板 (Modal)

**交互方式**：独立 Modal 对话框

**面板结构**：

```
┌──────────────────────────────────────────────┐
│  选择 Emoji 图标                        [×]  │
├──────────────────────────────────────────────┤
│  🔍 搜索 Emoji（输入中文关键词）             │
├──────────────────────────────────────────────┤
│  ✨ 推荐 (根据菜单名称"XXX"智能推荐)        │
│  [😴] [🌙] [🛌] [💤] [⭐] ...              │
├──────────────────────────────────────────────┤
│  分类: [医疗] [运动] [食物] [自然] [活动]    │
│        [物品] [符号]                         │
│                                              │
│  🏥 🩺 💊 💉 🧪 🔬 🩸 🫀 🫁 🦷           │
│  🦴 🩻 🧬 ⚕️ 🛡️ 🚑 🦽 😷 🤧 🌡️          │
│  ... (当前分类下的所有 Emoji)                │
│                                              │
│                                              │
│  已选择: [🌙]                                │
├──────────────────────────────────────────────┤
│                    [取消]   [确定]            │
└──────────────────────────────────────────────┘
```

**交互流程**：

1. 用户在菜单新增/编辑弹窗中，点击"选择 Emoji"按钮，打开 Emoji 选择面板 Modal
2. Modal 打开后，如果已输入菜单名称，顶部「推荐」区域展示智能推荐的 Emoji
3. 推荐区下方是分类标签栏，用户可点击切换不同分类
4. 用户点击某个 Emoji，该 Emoji 高亮选中（蓝色边框 + 浅蓝背景）；可以切换选择其他 Emoji
5. 底部显示"已选择: [Emoji]"提示当前选中的 Emoji
6. 点击「确定」按钮，选中的 Emoji 填入表单的 `icon_content` 字段，Modal 关闭
7. 点击「取消」或右上角 × 关闭 Modal，不做任何修改

**面板尺寸**：宽度 520px，高度自适应（最大 70vh，超出可滚动）

#### F03: 智能推荐区整合

**规则**：
- 如果当前菜单名称为空，推荐区显示"请先输入菜单名称以获取推荐"的提示文字
- 如果当前菜单名称有值，推荐区展示匹配到的 Emoji（最多 8 个）
- 如果未匹配到任何关键词，推荐区显示兜底 Emoji 并标注"通用推荐"
- 推荐区作为 Modal 内第一个区域，用户打开面板后可以直接点选推荐的 Emoji

#### F04: 去除手动输入框

**变更**：
- 原有的 Emoji 文本输入框（`Input` + `maxLength={4}`）替换为：
  - 一个 Emoji 预览区域（显示当前已选择的 Emoji，大号字体展示）
  - 一个"选择图标"按钮（点击打开 Emoji 选择面板 Modal）
  - 如果已选择 Emoji，预览区域旁边有"清除"按钮（可选移除已选的 Emoji）
- 表单校验：`icon_content` 在 `icon_type === 'emoji'` 时为必填项，提示"请选择一个 Emoji 图标"

#### F05: 中文关键词搜索

**实现方式**：前端维护一套 Emoji 中文标签库（纯前端数据，无需后端参与）

**标签库结构**：每个 Emoji 对应多个中文标签，例如：
- 🏥 → ["医院", "医疗", "看病", "就医"]
- 🍎 → ["苹果", "水果", "营养"]
- 🏃 → ["跑步", "运动", "健身", "锻炼"]

**搜索逻辑**：
- 用户在搜索框输入中文关键词
- 实时匹配所有标签中包含该关键词的 Emoji（模糊匹配，`includes`）
- 搜索结果替代分类列表展示
- 清空搜索框后恢复分类浏览模式
- 搜索有 300ms 防抖

**标签覆盖范围**：
- 精选与业务相关的 Emoji 分类（见下方 F02 分类说明），每个 Emoji 至少配 2~4 个中文标签
- 标签库估计覆盖 200~300 个 Emoji

## 3. 页面/界面设计

### 3.1 页面结构与导航

本次优化不新增页面，仅修改现有的「首页菜单管理」页面（`admin-web/src/app/(admin)/home-menus/page.tsx`）内的弹窗组件。

### 3.2 各页面功能说明

#### 3.2.1 菜单新增/编辑弹窗 —— Emoji 图标选择区域（改造）

**改造前**：
- 一个 Input 输入框（maxLength=4）用于手动输入 Emoji
- 输入框下方有一排推荐 Emoji 按钮（5 个）

**改造后**：
- Emoji 预览区域：居中显示已选择的 Emoji（48px 字体大小），未选择时显示虚线框占位
- "选择图标"按钮：Ant Design Button，点击打开 Emoji 选择面板 Modal
- "清除"链接按钮：已选择 Emoji 时显示，点击可清除

#### 3.2.2 Emoji 选择面板 Modal（新增）

**Modal 属性**：
- 标题："选择 Emoji 图标"
- 宽度：520px
- 关闭方式：右上角 ×、取消按钮、点击遮罩层
- 底部按钮：取消 + 确定（确定按钮在有选中 Emoji 时可用）

**内部区域**：

1. **搜索栏**
   - Ant Design `Input.Search` 组件
   - placeholder: "搜索 Emoji（输入中文关键词，如：医院、苹果）"
   - 300ms 防抖实时搜索

2. **推荐区（条件展示）**
   - 标题："✨ 为「{菜单名称}」推荐"（菜单名称为空时：标题为"✨ 推荐"，内容为提示文字）
   - 展示 5~8 个推荐 Emoji，网格排列
   - 与下方分类区域以浅灰色分割线隔开

3. **分类标签栏**
   - 横向排列的分类 Tab 按钮
   - 选中分类高亮（蓝色），未选中为灰色
   - 分类列表：

| 分类名 | 包含的 Emoji 范围 |
|--------|-------------------|
| 医疗健康 | 医院、药品、注射器、听诊器、显微镜、DNA、骨骼、X光、牙齿、肺、心脏、血液、温度计、轮椅、口罩、绷带、急救箱等 |
| 运动健身 | 跑步、举重、瑜伽、骑行、游泳、篮球、足球、乒乓球、奖牌、奖杯等 |
| 食物饮料 | 水果、蔬菜、肉类、饮品、烹饪、餐具等 |
| 自然植物 | 花、树、叶子、太阳、月亮、星星、彩虹、天气等 |
| 活动物品 | 日历、时钟、礼物、购物车、书本、灯泡、工具、放大镜、手机等 |
| 符号标志 | 打勾、叉号、心形、星星、数字、箭头、警告、禁止等 |
| 动物 | 常见动物表情（与健康/自然/可爱相关的选择） |

4. **Emoji 网格区域**
   - 网格布局：每行 8~10 个 Emoji
   - 每个 Emoji 为 36px × 36px 的可点击方块
   - 悬停效果：浅灰色背景 + 圆角
   - 选中效果：蓝色边框 + 浅蓝色背景
   - 区域可滚动，最大高度 300px

5. **已选择提示**
   - 固定在 Emoji 网格下方
   - 格式："已选择：🌙"（大号展示）
   - 未选择时："请点击上方 Emoji 进行选择"

### 4.2 实体关系

无变更。

## 5. 接口需求

### 5.1 接口清单

本次优化为**纯前端改造**，不涉及后端接口变更。

现有接口 `POST /api/admin/home-menus` 和 `PUT /api/admin/home-menus/{id}` 的 `icon_content` 字段仍然接收 Emoji 字符串，无需修改。

## 6. 非功能性需求

### 6.1 性能要求

- Emoji 标签库数据为前端静态数据，首次加载后无额外请求
- 搜索响应时间 < 50ms（前端内存过滤）
- Modal 打开/关闭动画流畅，无卡顿

### 6.2 安全要求

- 无新增安全风险（纯前端改造）

### 6.3 兼容性要求

- 确保 Emoji 在 Chrome、Edge、Firefox 主流浏览器中正确渲染
- 不同操作系统的 Emoji 展示风格可能不同（系统原生渲染），属于正常现象

## 7. 业务规则与约束

1. Emoji 图标选择方式统一为面板选择，不再支持手动文本输入
2. 推荐区始终显示在面板顶部，作为快捷入口
3. 搜索状态下，推荐区和分类标签栏隐藏，仅显示搜索结果
4. 当菜单名称变化时，推荐 Emoji 实时更新（通过 `Form.useWatch` 监听）
5. 编辑已有菜单时，面板打开后默认选中当前已保存的 Emoji

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 菜单名称为空时打开面板 | 推荐区显示"请先输入菜单名称以获取推荐"，分类浏览正常可用 |
| 搜索无结果 | 展示空状态："未找到匹配的 Emoji，请尝试其他关键词" |
| 未选择 Emoji 直接点确定 | 确定按钮置灰不可点击，需要先选择一个 Emoji |

| icon_type 切换为非 emoji 类型 | 面板入口隐藏，icon_content 清空 |

## 10. 补充说明

### 10.1 影响范围

- **改动文件**：仅 `admin-web/src/app/(admin)/home-menus/page.tsx`
- **不涉及**：后端、H5、小程序、Flutter、数据库
- **不影响**：现有菜单数据的展示和使用

### 10.2 关键词映射扩充策略

采用两者结合的方式：
1. 先由开发基于健康管理类 App 常见场景出一套通用的关键词映射（覆盖上述 40+ 组维度）
2. 用户后续可补充自己系统特有的菜单名称，开发再追加到映射表中

### 10.3 Emoji 标签库维护

- 标签库以 TypeScript 常量形式内嵌在前端代码中
- 后续如需动态管理，可考虑将标签库移至后台配置（本期暂不实现）
