# 语音搜索体验细节优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

bini-health 项目的语音搜索功能已在 H5 端实现了基础能力（单击录音、脉动呼吸按钮动画、腾讯云 ASR 识别、2秒自动搜索倒计时），小程序端使用微信同声传译插件，Flutter 端使用 record 插件 + 后端 ASR。

但在实际使用过程中发现以下体验问题：
1. 语音识别返回的文本包含标点符号（如句号、逗号），作为搜索关键词不合适
2. 录音中"点我结束~"按钮文字太小（仅 12px），老年用户看不清
3. 语音识别完成后，搜索框中的麦克风图标消失，用户无法再次使用语音输入

本次需求旨在对以上三个体验细节进行优化打磨，提升语音搜索的使用体验，特别是对老年用户群体的友好性。

### 1.2 目标用户

- **主要用户**：中老年健康管理人群（50岁以上），对手机操作不太熟练
- **次要用户**：所有使用语音搜索功能的 bini-health 用户

### 1.3 核心价值

- 消除语音识别结果中的无用标点，提高搜索精准度
- 增大按钮和文字尺寸，提升老年用户可操作性
- 保持麦克风入口始终可用，降低语音搜索的使用门槛

### 1.4 优化范围

**三端统一优化，同时开发、同步上线**：
- H5 移动端网页（Next.js）
- 微信小程序（原生）
- Flutter App（iOS / Android）

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 标点符号清除 | 语音识别结果自动去除所有标点 | P0 | 中文+英文标点全部清除，只保留纯文字 |
| F02 | 结束按钮优化 | 录音中"点我结束"按钮放大 | P0 | 文字 12px→16px，按钮 80px→100px |
| F03 | 麦克风图标常驻 | 搜索框有文字时同时显示麦克风和清除图标 | P0 | 用户随时可再次语音输入 |

### 2.2 功能详细描述

---

#### F01：语音识别结果标点符号清除

**需求说明**：

语音识别（ASR）返回的文本通常包含标点符号（如"高血压怎么治疗。"），这些标点在搜索场景中是无意义的，甚至可能影响搜索匹配结果。需要在识别结果填入搜索框之前，自动清除所有标点符号。

**清除规则**：

去掉所有中文和英文标点符号，只保留纯文字和数字。具体包括但不限于：

| 类型 | 需清除的字符 |
|------|-------------|
| 中文标点 | 。，、；：！？""''【】《》（）—…· |
| 英文标点 | . , ; : ! ? " ' [ ] { } ( ) - _ ... ~ @ # $ % ^ & * + = / \ \| < > |
| 空白处理 | 多个连续空格合并为一个空格，首尾空格去除 |

**处理时机**：

- 语音识别完成后、将文字填入搜索框之前
- 各端统一在客户端侧完成标点清除（无需后端处理）

**三端实现要点**：

| 端 | 处理位置 | 说明 |
|----|---------|------|
| H5 | ASR 识别回调中 | 在将识别文本设置到搜索框 state 之前执行清洗 |
| 小程序 | 微信同声传译插件回调中 | `translateVoice` 的 `success` 回调里清洗 |
| Flutter | ASR 识别结果解析后 | 在 `setState` 之前对识别文本执行正则替换 |

**验收标准**：

1. 语音说"高血压怎么治疗。" → 搜索框显示"高血压怎么治疗"（无句号）
2. 语音说"维生素C，维生素D" → 搜索框显示"维生素C 维生素D"（逗号变空格或直接去除）
3. 语音说"What is COVID-19?" → 搜索框显示"What is COVID19"（去除问号和连字符）
4. 不影响中文汉字、英文字母、阿拉伯数字的正常显示

---

#### F02：录音中"点我结束"按钮放大

**需求说明**：

当前录音状态下的"点我结束~"脉动呼吸按钮直径为 80px，文字大小为 12px，对老年用户来说太小，难以看清和点击。需要适当放大。

**优化参数**：

| 属性 | 优化前 | 优化后 |
|------|--------|--------|
| 按钮直径 | 80px | 100px |
| 文字大小 | 12px | 16px |
| 脉动动画范围 | 按 80px 基准 | 按 100px 基准同步放大 |

**三端实现要点**：

| 端 | 涉及组件 | 说明 |
|----|---------|------|
| H5 | 搜索页语音录音弹出层 | 修改 CSS 样式，按钮 width/height 和 font-size |
| 小程序 | 搜索页语音录音组件 | 修改 WXSS 样式 |
| Flutter | 搜索页语音录音 Widget | 修改 Container 的 width/height 和 Text 的 fontSize |

**注意事项**：

- 脉动呼吸动画（pulse-glow）的缩放范围需要跟随按钮尺寸调整，保持视觉比例协调
- 声波竖条动画不需要调整（保持 7 根竖条不变）
- 按钮内文案保持"点我结束~"不变

**验收标准**：

1. 三端录音状态下的结束按钮直径为 100px，文字大小为 16px
2. 脉动动画视觉效果正常，无截断或溢出
3. 老年用户能清楚看到按钮文字并准确点击

---

#### F03：语音识别后麦克风图标常驻

**需求说明**：

当前行为：语音识别成功后，识别文本填入搜索框，搜索框右侧仅显示清除(×)图标，麦克风图标消失。用户如果想再次使用语音输入，必须先手动清空搜索框。

期望行为：搜索框中有文字时，右侧**同时显示**麦克风图标和清除(×)图标，用户随时可以点击麦克风重新发起语音输入。

**交互规则**：

| 搜索框状态 | 右侧图标显示 |
|-----------|-------------|
| 空（无文字） | 仅显示麦克风图标 🎤 |
| 有文字（手动输入或语音识别后） | 同时显示麦克风图标 🎤 + 清除图标 × |
| 正在录音中 | 录音动画弹层（遮盖搜索框） |

**图标布局**：

- 搜索框右侧区域，从右到左依次排列：**清除(×)图标** → **麦克风图标**
- 两个图标之间保持适当间距（建议 8~12px）
- 清除(×)图标仅在搜索框有文字时显示
- 麦克风图标始终显示

**点击麦克风的行为**：

1. 如果搜索框中已有文字，点击麦克风后：
   - 清空当前搜索框文字
   - 立即启动新一轮录音
   - 进入录音状态（显示脉动按钮 + 声波动画）
2. 如果搜索框为空，点击麦克风后：
   - 直接启动录音（与当前行为一致）

**三端实现要点**：

| 端 | 涉及组件 | 说明 |
|----|---------|------|
| H5 | 搜索页搜索框组件 | 调整 Input 的 suffix/右侧区域，同时渲染两个图标 |
| 小程序 | 搜索页搜索框 | 调整 WXML 结构，右侧 view 包含两个 icon |
| Flutter | 搜索页 TextField | 调整 suffixIcon 为 Row 组件包含两个 IconButton |

**验收标准**：

1. 语音识别完成、文字已填入搜索框后，麦克风图标依然可见且可点击
2. 点击麦克风可直接发起新一轮语音录音
3. 清除(×)图标功能不受影响，点击后清空搜索框文字
4. 搜索框为空时不显示清除(×)图标，仅显示麦克风图标
5. 三端行为一致

## 4. 非功能性需求

### 4.1 性能要求

- 标点清除为纯字符串操作（正则替换），耗时 < 1ms，不影响用户感知
- 图标渲染不增加额外网络请求

### 4.2 安全要求

- 无新增安全风险（纯前端 UI 优化）

### 4.3 兼容性要求

- H5：兼容 iOS Safari 12+、Android Chrome 60+、微信内置浏览器
- 小程序：兼容微信基础库 2.10.0+
- Flutter：兼容 iOS 12+ / Android 5.0+
- 三端体验保持一致

## 5. 业务规则与约束

| 规则编号 | 描述 |
|---------|------|
| BR-01 | 标点清除仅在客户端执行，不修改后端 ASR 返回逻辑 |
| BR-02 | 标点清除不区分语言，中英文标点统一清除 |
| BR-03 | 清除标点后如果结果为空字符串，则不触发自动搜索 |
| BR-04 | 麦克风图标在搜索页始终可见（不受搜索框内容影响） |
| BR-05 | 三端统一开发，统一上线，体验保持一致 |

## 6. 权限设计

本次优化不涉及权限变更。语音搜索功能对所有用户开放（无需登录即可使用）。

| 角色 | 权限说明 |
|------|----------|
| 所有用户 | 可使用语音搜索，体验优化对所有用户生效 |

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|---------|
| ASR 返回纯标点文本（如"。。。"） | 清除后为空字符串，不触发自动搜索，搜索框保持空 |
| ASR 返回空字符串 | 提示"未识别到语音内容，请重新尝试"（现有逻辑不变） |
| 用户快速连续点击麦克风 | 防抖处理，录音未结束时忽略重复点击 |
| 搜索框有文字时点击麦克风 | 先清空搜索框内容，再启动录音 |
| 录音权限被拒绝 | 提示用户开启麦克风权限（现有逻辑不变） |
| 按钮放大后在超小屏幕上溢出 | 使用 min() 或 vw 单位确保不超出屏幕宽度 |

## 8. 补充说明

### 8.1 与现有功能的关联

- 本次优化不改变语音搜索的核心流程（录音 → ASR 识别 → 填入搜索框 → 自动搜索）
- 不改变 ASR 后端接口和配置
- 不改变搜索结果页的展示逻辑
- 保持现有的 2 秒自动搜索倒计时机制不变

### 8.2 技术栈确认

| 端 | 技术栈 | 语音方案 |
|----|--------|---------|
| H5 | Next.js + React + Ant Design Mobile | MediaRecorder + 后端腾讯云 ASR |
| 小程序 | 微信原生 | 微信同声传译插件（WechatSI） |
| Flutter | Flutter + record 插件 | record 录音 + 后端腾讯云 ASR |

### 8.3 标点清除正则参考

```
// 匹配所有中英文标点的正则表达式
/[\u3000-\u303F\uFF01-\uFF60\u2000-\u206F\u0021-\u002F\u003A-\u0040\u005B-\u0060\u007B-\u007E]/g
```

实际实现时可根据各端语言特性调整正则写法，确保覆盖完整的标点范围。
