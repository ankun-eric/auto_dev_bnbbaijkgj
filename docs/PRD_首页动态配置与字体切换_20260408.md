# 首页动态配置与字体切换 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

bini-health 健康管理平台目前有 H5 前端、微信小程序、Flutter App 三端。三端首页上的**搜索栏、Banner 轮播、宫格菜单**均为前端代码硬编码，每次修改都需要开发人员改代码、重新发版，运营效率极低。此外，Banner 轮播图当前不支持点击跳转，浪费了高价值的运营位资源。

同时，平台的目标用户群体以**中老年人**为主，现有界面字体偏小（菜单文字约 12-14px），对该群体不够友好。

本次需求旨在：
1. 将首页核心区域（搜索栏、菜单、Banner）改为**后台动态配置**，支持运营人员自主维护
2. 修复 Banner 不可点击的问题，支持配置跳转链接
3. 新增**字体大小切换功能**（3 档滑动调节），提升中老年用户的阅读体验

### 1.2 目标用户

| 角色 | 说明 |
|------|------|
| **运营人员** | 通过管理后台维护首页菜单、Banner、基础设置 |
| **终端用户（中老年人为主）** | 使用 H5/小程序/App 浏览首页，可调节字体大小 |
| **管理员** | 具有后台所有配置权限 |

### 1.3 核心价值

- **降低运营成本**：菜单、Banner 修改无需开发介入，运营人员可在后台自助完成
- **提升运营灵活性**：可快速上下线菜单项、调整排序、更换 Banner 推广内容
- **改善中老年用户体验**：字体大小切换功能让用户按自身需要调整字号
- **三端统一管理**：后台配置一次，H5、小程序、App 三端同步生效

### 2.2 功能详细描述

#### 2.2.1 首页基础设置（F01-F06）

**搜索框配置**
- **Placeholder 文字**：文本输入框，后台输入后三端统一展示。默认值为"搜索症状、药品、健康知识..."
- **显示/隐藏开关**：Switch 控件。关闭时，三端首页不显示搜索栏区域，页面内容自动上移填充空间

**宫格列数配置**
- 下拉选择器，可选值：3列、4列、5列
- 修改后三端实时生效（下次打开首页时刷新）
- 默认值：3列（与当前布局保持一致）

**字体切换后台配置**
- **功能开关**：Switch 控件。关闭时，三端不显示字体切换入口（包括个人中心设置项和首页快捷按钮）
- **默认档位**：下拉选择器，可选值：标准、大、超大。新用户首次打开应用时使用该档位。默认值：标准
- **档位参数**：每个档位对应一个基础字号（px），后台可微调
  - 标准：默认 14px
  - 大：默认 18px
  - 超大：默认 22px

#### 2.2.2 首页菜单管理（F07-F11）

**菜单项属性**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 菜单名称 | 字符串 | 是 | 菜单显示文字，最长 10 个字符 |
| 图标类型 | 枚举 | 是 | emoji / image |
| 图标内容 | 字符串 | 是 | Emoji 字符或图片 URL |
| 跳转类型 | 枚举 | 是 | internal（应用内）/ external（外部H5）/ miniprogram（小程序） |
| 跳转地址 | 字符串 | 是 | 应用内路径（如 /checkup）、外部 URL、或小程序页面路径 |
| 小程序 AppID | 字符串 | 否 | 跳转类型为 miniprogram 时必填 |
| 排序值 | 整数 | 是 | 数值越小越靠前，默认 0 |
| 是否显示 | 布尔 | 是 | 控制菜单项是否在前端展示，默认显示 |

**排序机制**
- 列表页支持**拖拽排序**：通过拖拽表格行来调整菜单顺序，拖拽完成后自动更新所有受影响项的排序值
- 同时支持**排序值字段**手动编辑：可以在编辑弹窗中直接修改排序值数字
- 前端展示时按排序值升序排列，排序值相同按创建时间升序

**图标配置**
- 图标类型为 **Emoji** 时：弹窗中提供 Emoji 输入框，可直接粘贴/输入 Emoji 字符
- 图标类型为 **图片** 时：弹窗中提供图片上传组件（支持 PNG/JPG/SVG，建议尺寸 96x96px，最大 500KB），上传后返回图片 URL

**跳转配置**
- 跳转类型为 **应用内页面（internal）** 时：输入框填写应用内路径，如 `/checkup`、`/drug`、`/chat`
- 跳转类型为 **外部H5链接（external）** 时：输入框填写完整 URL，如 `https://example.com/page`
- 跳转类型为 **微信小程序（miniprogram）** 时：填写目标小程序的 AppID 和页面路径

**布局说明**
- 三端首页去掉宫格区上方的"功能入口"标题文字
- 宫格列数由后台配置控制（3/4/5列）
- 菜单项数量超出一屏时全部展示，页面自动撑高，用户滚动查看
- 只展示"是否显示"为开启状态的菜单项

#### 2.2.3 首页 Banner 管理（F12-F16）

**Banner 属性**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Banner 图片 | 图片 URL | 是 | 上传 Banner 图片（建议比例 16:9 或 3:1，最大 2MB） |
| 跳转类型 | 枚举 | 是 | none（不跳转）/ internal（应用内）/ external（外部H5）/ miniprogram（小程序） |
| 跳转地址 | 字符串 | 否 | 跳转类型非 none 时必填 |
| 小程序 AppID | 字符串 | 否 | 跳转类型为 miniprogram 时必填 |
| 排序值 | 整数 | 是 | 数值越小越靠前，默认 0 |
| 是否显示 | 布尔 | 是 | 控制 Banner 是否在前端展示，默认显示 |

**排序机制**
- 与菜单管理一致：支持拖拽排序 + 排序值字段手动编辑

**跳转支持**
- **不跳转（none）**：纯展示 Banner，点击无反应（适用于品牌展示类 Banner）
- **应用内页面（internal）**：点击跳转到应用内指定路径
- **外部H5链接（external）**：点击通过 WebView / 浏览器打开外部链接
- **微信小程序（miniprogram）**：点击跳转到指定小程序的指定页面

**现有问题修复**
- 当前三端首页 Banner 均不可点击，本次需同步修复，使 Banner 支持根据后台配置的跳转类型和地址进行跳转

#### 2.2.4 字体大小切换（F17-F18）

**功能说明**
- 提供 3 档字体大小调节：**标准** → **大** → **超大**
- 采用**滑块（Slider）** 交互方式，用户拖动滑块在 3 个档位之间切换
- 切换后立即生效，无需重启应用
- 用户选择的字体档位本地持久化存储（localStorage / AsyncStorage / SharedPreferences），下次打开应用自动恢复

**入口位置**
- **入口一：个人中心/设置页** — 设置列表中增加"字体大小"项，点击进入字体调节页，包含滑块和实时预览
- **入口二：首页快捷按钮** — 首页右上角或顶部区域增加一个"Aa"图标按钮，点击弹出字体调节浮层（底部弹出 Sheet），包含滑块和预览文字

**生效范围**
- 仅影响首页和核心页面，包括：
  - 首页（搜索框、菜单文字、Banner 下方文字区域）
  - 文章详情页（标题、正文）
  - 个人中心页
  - AI 健康咨询页（对话气泡文字）
- 其他页面保持原有字体大小不变

**档位参数（默认值，可由后台调整）**

| 档位 | 基础字号 | 标题字号 | 菜单图标下方文字 |
|------|---------|---------|-----------------|
| 标准 | 14px | 18px | 12px |
| 大 | 18px | 22px | 16px |
| 超大 | 22px | 26px | 20px |

**后台管理**
- 功能开关：后台可控制字体切换功能的开启/关闭，关闭时三端隐藏所有入口
- 默认档位：后台设置新用户的默认档位
- 档位参数：后台可微调每个档位对应的字号数值

## 3. 页面/界面设计

### 3.1 页面结构与导航

**管理后台新增菜单结构**

```
首页配置（新增顶级菜单）
├── 首页基础设置
│   ├── 搜索框配置区（placeholder 文字 + 显示/隐藏开关）
│   ├── 宫格配置区（列数选择）
│   └── 字体切换配置区（功能开关 + 默认档位 + 档位参数）
├── 首页菜单管理
│   └── 菜单项列表（CRUD + 拖拽排序）
└── 首页 Banner 管理
    └── Banner 列表（CRUD + 拖拽排序）
```

### 3.2 各页面功能说明

#### 3.2.1 管理后台 — 首页基础设置页

**页面布局**：单页表单，分为 3 个卡片区域

**卡片一：搜索框配置**
- 标签："搜索框显示"，控件：Switch 开关
- 标签："提示文字"，控件：Input 文本输入框（placeholder: "请输入搜索框提示文字"）
- 底部"保存"按钮

**卡片二：宫格配置**
- 标签："菜单列数"，控件：Select 下拉选择（选项：3列 / 4列 / 5列）
- 底部"保存"按钮

**卡片三：字体切换配置**
- 标签："字体切换功能"，控件：Switch 开关
- 标签："新用户默认档位"，控件：Select 下拉选择（选项：标准 / 大 / 超大）
- 标签："标准档基础字号"，控件：InputNumber（单位 px，范围 12-16）
- 标签："大档基础字号"，控件：InputNumber（单位 px，范围 16-22）
- 标签："超大档基础字号"，控件：InputNumber（单位 px，范围 20-28）
- 底部"保存"按钮

#### 3.2.2 管理后台 — 首页菜单管理页

**页面布局**：顶部操作栏 + 可拖拽表格

**操作栏**
- 左侧："新增菜单"按钮（Primary）
- 右侧：无（菜单项通常不多，无需搜索）

**表格列**

| 列名 | 宽度 | 说明 |
|------|------|------|
| 拖拽手柄 | 40px | 拖拽排序操作区 |
| 排序 | 60px | 显示排序值 |
| 图标 | 60px | 展示 Emoji 或缩略图 |
| 菜单名称 | 120px | 文字 |
| 跳转类型 | 100px | Tag 标签（应用内/外部链接/小程序） |
| 跳转地址 | 200px | 路径或 URL，超长截断 |
| 显示状态 | 80px | Switch 开关，可直接切换 |
| 操作 | 120px | 编辑 / 删除 |

**新增/编辑弹窗（Modal）**
- 表单字段：菜单名称、图标类型（Radio: Emoji/图片）、图标内容（根据类型切换 Input/Upload）、跳转类型（Select）、跳转地址（Input）、小程序 AppID（条件显示）、排序值（InputNumber）、是否显示（Switch）
- 底部：取消 / 确定

#### 3.2.3 管理后台 — 首页 Banner 管理页

**页面布局**：顶部操作栏 + 可拖拽表格

**操作栏**
- 左侧："新增 Banner"按钮（Primary）

**表格列**

| 列名 | 宽度 | 说明 |
|------|------|------|
| 拖拽手柄 | 40px | 拖拽排序操作区 |
| 排序 | 60px | 显示排序值 |
| 预览图 | 160px | Banner 缩略图 |
| 跳转类型 | 100px | Tag 标签（不跳转/应用内/外部链接/小程序） |
| 跳转地址 | 200px | 路径或 URL |
| 显示状态 | 80px | Switch 开关 |
| 操作 | 120px | 编辑 / 删除 |

**新增/编辑弹窗（Modal）**
- 表单字段：Banner 图片（Upload，显示预览）、跳转类型（Select: 不跳转/应用内/外部链接/小程序）、跳转地址（条件显示 Input）、小程序 AppID（条件显示）、排序值（InputNumber）、是否显示（Switch）
- 底部：取消 / 确定

#### 3.2.4 用户端 — 首页改造

**H5 / 小程序 / App 首页改造点**

1. **搜索栏**：由后台配置控制显示/隐藏和 placeholder 文字
2. **Banner 轮播**：数据从 API 动态获取，支持点击跳转
3. **宫格菜单**：数据从 API 动态获取，列数由后台配置决定，去掉"功能入口"标题
4. **字体快捷入口**：首页右上角增加"Aa"按钮（字体切换功能开启时显示）

#### 3.2.5 用户端 — 字体调节页/浮层

**交互流程**

1. 用户点击"Aa"按钮（首页）或"字体大小"设置项（个人中心）
2. 弹出底部 Sheet / 跳转设置页
3. 显示一个 3 档滑块（Slider），当前档位高亮
4. 滑块下方显示预览文字，实时展示当前档位的字体效果
5. 用户拖动滑块，预览文字实时变化
6. 用户关闭浮层/返回，字体设置立即生效并持久化

## 4. 数据模型设计

#### 4.1.3 首页配置（复用 SystemConfig 表）

利用现有的 `system_configs` 键值对表存储以下配置项：

| config_key | 类型 | 默认值 | 说明 |
|------------|------|--------|------|
| home_search_visible | BOOLEAN | true | 搜索框是否显示 |
| home_search_placeholder | STRING | "搜索症状、药品、健康知识..." | 搜索框提示文字 |
| home_grid_columns | INT | 3 | 宫格列数 |
| home_font_switch_enabled | BOOLEAN | true | 字体切换功能是否开启 |
| home_font_default_level | STRING | "standard" | 新用户默认字体档位 |
| home_font_standard_size | INT | 14 | 标准档基础字号（px） |
| home_font_large_size | INT | 18 | 大档基础字号（px） |
| home_font_xlarge_size | INT | 22 | 超大档基础字号（px） |

### 4.2 实体关系

- `home_menu_items` 和 `home_banners` 是独立实体，无外键关联
- 首页配置存储在 `system_configs` 表中，通过 config_key 检索
- 用户字体偏好存储在客户端本地（不入库），通过 localStorage / AsyncStorage / SharedPreferences 持久化

## 5. 接口需求

### 5.1 接口清单

## 6. 非功能性需求

### 6.1 性能要求

- 用户端首页配置 API（A03、B06、C06）响应时间 < 200ms
- 管理端 CRUD 操作响应时间 < 500ms
- 首页配置数据建议在前端缓存 5 分钟，减少 API 调用频率
- 拖拽排序采用乐观更新，先更新 UI 再异步提交后端

### 6.2 安全要求

- 管理端 API（/api/admin/*）需要管理员身份认证
- 用户端 API（/api/home-config、/api/home-menus、/api/home-banners）无需登录即可访问
- Banner 和菜单图标上传需校验文件类型（仅允许 PNG/JPG/SVG/WebP）和大小限制

### 6.3 兼容性要求

- H5：Chrome 60+、Safari 12+、微信内置浏览器
- 小程序：微信基础库 2.10.0+
- Flutter App：iOS 12+、Android 5.0+
- 字体大小切换需确保三端的基础字号计算逻辑一致（H5 用 px/rem，小程序用 rpx 换算，Flutter 用 sp）

## 7. 业务规则与约束

1. **菜单项数量**：不设硬上限，但建议运营人员控制在 15 个以内，避免首页过长
2. **Banner 数量**：不设硬上限，但建议控制在 5 个以内，确保轮播体验流畅
3. **删除保护**：删除菜单项/Banner 时需二次确认弹窗（"确认删除？删除后不可恢复"）
4. **宫格列数变更**：变更列数后，已配置的菜单项自动按新列数重排，无需额外操作
5. **三端一致性**：后台配置一次，三端展示完全一致（文案、图标、排序、可见性）
6. **字体档位存储**：用户选择的字体档位只存在客户端本地，不同设备/端之间不同步（各端独立存储）
7. **初始数据**：系统部署后，自动将现有硬编码的 6 个菜单项（AI健康咨询、体检报告、健康自查、中医养生、用药参考、健康计划）作为默认数据写入 `home_menu_items` 表

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 所有菜单项均设为隐藏 | 前端不显示宫格区域，页面内容上移 |
| 所有 Banner 均设为隐藏 | 前端不显示轮播区域，页面内容上移 |
| Banner 图片加载失败 | 显示默认占位图，不影响轮播 |
| 菜单图标（图片类型）加载失败 | 显示默认占位图标 |
| 首页配置 API 请求失败 | 使用本地缓存的上次成功数据；如无缓存则使用硬编码的兜底默认值 |
| 跳转地址配置错误（404） | 前端 Toast 提示"页面不存在"，不崩溃 |
| 小程序跳转 AppID 无效 | 微信 SDK 会抛异常，前端 catch 后 Toast 提示"无法打开目标小程序" |
| 字体切换功能被后台关闭 | 已调整字体的用户保持当前字号不变（本地存储不清除），但入口隐藏，用户无法再次调节 |
| 上传图片超过大小限制 | 前端校验提示"图片大小不能超过 500KB / 2MB" |
| 拖拽排序网络请求失败 | 回滚 UI 到拖拽前状态，Toast 提示"排序保存失败，请重试" |
