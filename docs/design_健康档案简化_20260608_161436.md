# 健康档案完善判定逻辑简化 - 设计文档

> 生成时间: 2026-06-08 16:14:36
> 关联需求: health-profile-simplify-v1

## 1. 需求概述

简化健康档案"本人 Tab"完善度判定逻辑，下线 guide-status 接口，清理死代码。

### 1.1 现有问题

1. `GET /api/health/guide-status` 和 `GET /api/health-profile/self` 两个接口功能重叠
2. `guide-status` 接口维护 `guide_count` 做频控（最多弹 2 次），可由前端 localStorage 实现
3. `_compute_missing_fields_v2` 跨三表取并集做兜底，其中：
   - `users` 表无 `gender`/`birthday` 字段（死代码）
   - `family_members(is_self=True)` 兜底逻辑多余（已由 PUT 接口自动同步）

### 1.2 设计目标

1. 下线 `GET/POST /api/health/guide-status` 接口
2. 简化 `_compute_missing_fields_v2`：只查 `health_profiles` 表
3. 清理所有死代码
4. 前端频控改 localStorage，弹窗判断改调 `GET /api/health-profile/self`

## 3. 改动设计

### 3.1 后端改动

#### 3.1.1 简化 `_compute_missing_fields_v2`

**文件**: `backend/app/api/health_profile_self.py`

**改动前** (L78-135):
```python
async def _compute_missing_fields_v2(
    db: AsyncSession, user_id: int, profile: Optional[HealthProfile],
) -> List[str]:
    # 先查 health_profiles (name_ok, gender_ok, birthday_ok 三个 bool)
    # 若不全：查 family_members(is_self=True) 兜底
    # 若还不全：查 users 表死代码（L112-126，users 表无 gender/birthday）
    # 返回 missing 列表
```

**改动后**:
```python
async def _compute_missing_fields_v2(
    db: AsyncSession,
    user_id: int,
    profile: Optional[HealthProfile],
) -> List[str]:
    """简化版：只判断 health_profiles 表 family_member_id IS NULL 那条记录
    name/gender/birthday 三个字段是否齐全。"""
    missing: List[str] = []
    if profile is None:
        return ["name", "gender", "birthday"]
    if _is_name_empty(profile.name):
        missing.append("name")
    if not profile.gender or not str(profile.gender).strip():
        missing.append("gender")
    if profile.birthday is None:
        missing.append("birthday")
    return missing
```

**说明**: 简化后与 `_compute_missing_fields` 逻辑相同，保留 `_compute_missing_fields_v2` 作为 v2 入口以保持接口兼容（不改变调用方）。原函数中的三表并集逻辑全部删除：
- 删除查 `family_members(is_self=True)` 的兜底（L92-110）
- 删除查 `users` 表的死代码（L112-126）

### 3.2 前端改动

#### 3.2.1 H5 端健康引导页适配

**文件**: `h5-web/src/app/health-guide/page.tsx`

**改动内容**:
1. 删除 `handleFinish()` 中 POST /guide-status 调用（L191），改为调用 GET /health-profile/self 验证完善状态
2. 删除 `handleSkip()` 中 POST /guide-status 调用（L203），改为 localStorage 记录跳过次数
3. 频控逻辑：前端 localStorage 维护 `healthGuideSkipCount`，最多跳 2 次

**改动前** (L187-208):
```typescript
const handleFinish = async () => {
    setSubmitting(true);
    try {
      await saveCurrentStep();
      await api.post('/api/health/guide-status', { action: 'complete' });
      Toast.show({ content: '档案完善成功 ✓', duration: 2000 });
      setTimeout(() => router.replace('/ai-home'), 1500);
    } catch { ... }
};
const handleSkip = async () => {
    try {
      await api.post('/api/health/guide-status', { action: 'skip' });
    } catch { }
    router.replace('/ai-home');
};
```

**改动后**:
```typescript
const GUIDE_COUNT_KEY = 'healthGuideSkipCount';
const handleFinish = async () => {
    setSubmitting(true);
    try {
      await saveCurrentStep();
      localStorage.removeItem(GUIDE_COUNT_KEY); // 完善后清理频控
      Toast.show({ content: '档案完善成功 ✓', duration: 2000 });
      setTimeout(() => router.replace('/ai-home'), 1500);
    } catch { ... }
};
const handleSkip = async () => {
    const cnt = parseInt(localStorage.getItem(GUIDE_COUNT_KEY) || '0', 10);
    localStorage.setItem(GUIDE_COUNT_KEY, String(cnt + 1));
    router.replace('/ai-home');
};
```

#### 3.2.2 小程序端健康引导页适配

**文件**: `miniprogram/pages/health-guide/index.js`

**改动内容**:

```javascript
const GUIDE_COUNT_KEY = 'healthGuideSkipCount';
async finishGuide() {
    // ... 保存逻辑 ...
    try {
      await put('/api/health/profile', payload);
      wx.removeStorageSync(GUIDE_COUNT_KEY);  // 清理频控
      wx.showToast({ title: '档案创建成功', icon: 'success' });
      setTimeout(() => { wx.switchTab({ url: '/pages/home/index' }); }, 1500);
    } catch (e) { ... }

skipGuide() {
    const cnt = parseInt(wx.getStorageSync(GUIDE_COUNT_KEY) || '0', 10);
    wx.setStorageSync(GUIDE_COUNT_KEY, String(cnt + 1));
    wx.switchTab({ url: '/pages/home/index' });
}
```

#### 3.2.3 弹窗触发逻辑适配
