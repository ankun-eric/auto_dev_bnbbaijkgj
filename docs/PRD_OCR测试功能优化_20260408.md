# OCR测试功能优化 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

当前管理后台的 **OCR全局设置** 页面包含三个 Tab："场景模板管理"、"上传限制"、"OCR测试"。其中 "OCR测试" Tab 的功能（选择提供商、选择场景模板、仅OCR测试、完整测试含AI）与 **OCR识别配置** 页面已有的测试功能存在重复。

同时，OCR识别配置 页面的 "测试完整流程" 弹窗目前仅支持单图上传，无法满足多图批量测试的实际需求。

本次优化旨在：
1. 移除 OCR全局设置 中的重复 "OCR测试" Tab，简化页面结构
2. 增强 OCR识别配置 的 "测试完整流程" 弹窗，支持多图上传和批量结果展示

### 1.2 目标用户

管理后台管理员（Admin）

### 1.3 核心价值

- 消除功能重复，减少管理员的认知负担
- 增强测试弹窗的多图能力，让管理员在一次测试中验证多张图片的 OCR+AI 效果
- 上传张数与全局设置保持一致，避免配置不一致问题

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | OCR全局设置 | 移除"OCR测试"Tab | P0 | 仅保留"场景模板管理"和"上传限制"两个Tab |
| F02 | OCR识别配置 | "测试完整流程"弹窗多图上传 | P0 | 单图改多图，使用picture-card模式 |
| F03 | OCR识别配置 | 多图上传张数联动全局设置 | P0 | 读取OCR全局设置中的"单次最多上传张数" |
| F04 | OCR识别配置 | 多图测试结果分图展示 | P0 | 逐图可展开/折叠，含成功/失败标记 |
| F05 | OCR识别配置 | 多图测试结果合并展示 | P0 | 底部显示合并OCR文本和合并AI结果 |
| F06 | OCR识别配置 | 弹窗宽度调整 | P1 | "测试完整流程"弹窗从700px加宽至850px |

### 2.2 功能详细描述

#### F01：移除 OCR全局设置 的 "OCR测试" Tab

**当前状态**：OCR全局设置页面（`ocr-global-config`）有三个 Tab —— 场景模板管理、上传限制、OCR测试

**变更内容**：
- 删除第三个 Tab "OCR测试" 及其所有相关代码（上传组件、提供商选择、场景选择、仅OCR测试按钮、完整测试按钮、结果展示区域）
- 保留 "场景模板管理" 和 "上传限制" 两个 Tab，不做任何其他调整

**影响范围**：仅前端 `admin-web/src/app/(admin)/ocr-global-config/page.tsx`

#### F02：OCR识别配置 "测试完整流程" 弹窗 — 多图上传

**当前状态**：OCR识别配置页面（`ocr-config`）的 "测试完整流程" 弹窗使用 `maxCount: 1` 的单图上传

**变更内容**：
- 将 Upload 组件的 `maxCount` 从 `1` 改为动态值（从 OCR全局设置的上传限制配置读取）
- 上传模式改为 `multiple` + `listType="picture-card"`，支持同时选择多张图片
- 点击 "开始测试" 后，将所有图片通过 `FormData` 一次性提交到 `POST /api/admin/ocr/test-full` 接口（该接口已支持多文件 `files: List[UploadFile]`）

**"测试OCR" 弹窗保持不变**：仍为单图上传，不做改动

#### F03：多图上传张数联动全局设置

**交互逻辑**：
- 弹窗打开时，调用 `GET /api/admin/ocr/upload-limits` 获取 `max_batch_count` 值
- Upload 组件的 `maxCount` 设置为该值
- 图片数量提示文字：`最多 {max_batch_count} 张`
- 当已选图片数达到上限时，隐藏上传按钮

#### F04：多图测试结果 — 分图展示

**展示方式**：
- 使用 Ant Design 的 `Collapse` 组件，每张图片的结果为一个折叠面板
- 面板标题：`图片 {序号}` + 成功/失败图标标记
  - 成功：绿色 `CheckCircleOutlined`
  - 失败：红色 `CloseCircleOutlined` + 错误信息
- 面板内容（展开后）：
  - OCR 文本（`<pre>` 格式展示）
  - AI 结构化结果（JSON 格式展示）
- 默认全部折叠，可手动展开查看

#### F05：多图测试结果 — 合并展示

**展示位置**：在逐图结果下方

**展示内容**：
- **合并 OCR 文本**：所有成功图片的 OCR 文本合并后展示（以 Divider 分隔标题）
- **合并 AI 结果**：所有成功图片经 AI 处理后的合并结果（JSON 格式展示）

**条件显示**：仅当有合并结果时才显示对应区域

#### F06：弹窗宽度调整

- "测试完整流程" 弹窗宽度从 `700px` 调整为 `850px`
- 弹窗内部内容区域有足够空间展示多图缩略图和结果

## 3. 页面/界面设计

### 3.1 页面结构与导航

无导航变更，仍通过管理后台左侧菜单进入：
- **AI管理 → OCR全局设置**：保留"场景模板管理"和"上传限制"两个 Tab
- **AI管理 → OCR识别配置**：保留原有结构，仅修改"测试完整流程"弹窗

### 3.2 各页面功能说明

#### 3.2.1 OCR全局设置页面（变更后）

| Tab | 内容 |
|-----|------|
| 场景模板管理 | 不变 — 卡片式场景列表，新增/编辑/删除 |
| 上传限制 | 不变 — max_batch_count + max_file_size_mb 表单 |

（原"OCR测试" Tab 已移除）

#### 3.2.2 OCR识别配置页面 — "测试完整流程" 弹窗（变更后）

**弹窗标题**：`测试完整流程 - {当前提供商名称}`

**弹窗宽度**：`850px`

**弹窗内容**（从上到下）：

1. **图片上传区域**
   - Upload 组件，`listType="picture-card"`，`multiple` 模式
   - 最多上传 `{max_batch_count}` 张（动态读取）
   - 提示文字：`最多 N 张`

2. **业务场景选择**
   - Select 下拉框，从 `GET /api/admin/ocr/scenes` 获取场景列表
   - 必选

3. **操作按钮**
   - "开始测试" 按钮（当未选图片或未选场景时 disabled）

4. **测试结果区域**（测试完成后显示）
   - 总体状态标签（成功/失败 + 提供商名称）
   - 逐图结果（Collapse 折叠面板）
   - 合并 OCR 文本（Divider + pre 区域）
   - 合并 AI 结果（Divider + pre 区域）

## 4. 数据模型设计

### 4.1 数据实体与字段

本次需求不涉及数据模型变更。所有相关模型和接口已存在：
- `OcrUploadConfig`：`max_batch_count`、`max_file_size_mb`
- `OcrSceneTemplate`：`id`、`scene_name`、`prompt_content`、`is_preset`

### 4.2 实体关系

无变更。

## 5. 接口需求

### 5.1 接口清单

本次需求不需要新增后端接口。仅需前端调整调用方式。

| 编号 | 方法 | 路径 | 说明 | 变更类型 |
|------|------|------|------|----------|
| API-01 | GET | /api/admin/ocr/upload-limits | 获取上传限制配置 | 前端新增调用（OCR识别配置页面） |
| API-02 | POST | /api/admin/ocr/test-full | 完整流程测试 | 前端改为多文件提交（接口已支持） |
| API-03 | GET | /api/admin/ocr/scenes | 获取场景列表 | 无变更（已有调用） |

**API-02 请求参数变更说明**：
- 原：`file: File`（单文件）+ `provider: string` + `scene_id: number`
- 新：`files: File[]`（多文件）+ `provider: string` + `scene_id: number`

**API-02 响应格式**（`OcrTestFullBatchResponse`，已支持）：
```json
{
  "success": true,
  "provider_name": "百度云",
  "results": [
    {
      "success": true,
      "provider_name": "百度云",
      "ocr_text": "识别文本...",
      "ai_result": { ... }
    }
  ],
  "merged_ocr_text": "合并后的OCR文本...",
  "merged_ai_result": { ... }
}
```

## 6. 非功能性需求

### 6.1 性能要求

- 多图测试时前端显示 loading 状态，避免用户重复点击
- 大量图片测试可能耗时较长，建议弹窗不可关闭（或关闭后不中断请求）

### 6.2 安全要求

- 沿用现有管理员认证机制，无新增安全需求

### 6.3 兼容性要求

- 管理后台仅支持桌面浏览器（Chrome、Edge、Firefox），无移动端适配需求

## 7. 业务规则与约束

| 编号 | 规则 |
|------|------|
| BR-01 | "测试完整流程" 弹窗的上传张数上限始终与 OCR全局设置 的 `max_batch_count` 一致 |
| BR-02 | "测试OCR" 弹窗保持单图上传，不做改动 |
| BR-03 | 测试时提供商固定跟随当前 Tab 所选的提供商，无需手动切换 |
| BR-04 | 场景模板为必选项，不选择不能发起完整测试 |

## 8. 权限设计

| 角色 | 权限说明 |
|------|----------|
| Admin | 拥有 OCR全局设置 和 OCR识别配置 的全部操作权限 |
| 普通用户 | 无权访问管理后台 |

无新增权限需求。

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 上传限制接口请求失败 | 使用默认值 10 张 |
| 多图测试中部分图片识别失败 | 失败的图片在逐图结果中标红，合并结果仅包含成功的图片 |
| 所有图片都识别失败 | 整体状态标记为失败，不显示合并结果区域 |
| 用户未选择图片就点击测试 | 按钮置灰（disabled），不可点击 |
| 用户未选择业务场景就点击测试 | 按钮置灰（disabled），不可点击 |

  - `admin-web/src/app/(admin)/ocr-config/page.tsx` — 升级测试完整流程弹窗
- 不涉及 H5、小程序、Flutter App、后端的任何改动
