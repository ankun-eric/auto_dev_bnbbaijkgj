# AI 健康咨询页面功能按钮与数字人对话模式 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

bini-health 项目的 AI 健康咨询聊天页面目前已具备「咨询对象切换」、文字/语音输入等基本交互能力。为进一步提升用户的健康咨询体验，需要在输入框上方新增一排可配置的功能按钮区域，提供拍照识药、上传报告、数字人对话等丰富的快捷操作入口。其中最核心的创新功能是**数字人对话模式**——用户点击"打电话"按钮后进入全屏数字人视频通话界面，通过语音与 AI 健康顾问实时对话，体验接近真实视频通话。

### 1.2 目标用户

- 使用 bini-health 应用进行健康咨询的普通用户（各年龄段，包括中老年用户）
- 后台管理员（负责配置功能按钮、数字人形象等）

### 1.3 核心价值

- **降低交互门槛**：语音对话模式让不擅长打字的用户（尤其中老年群体）也能方便使用 AI 健康咨询
- **提升咨询体验**：数字人形象增加拟人感和信任感，字幕同步保证信息可回看
- **丰富功能入口**：通过可配置的功能按钮，将拍照识药、报告解读、症状自查等高频操作前置展示，提升功能发现率和使用率
- **灵活可扩展**：后台完全自定义按钮配置，支持快速增减功能入口

### 1.4 多端覆盖

本需求在以下三端全部实现：

- **微信小程序**（原生技术）
- **H5 Web**（Next.js）
- **Flutter App**（iOS / Android）

#### F01：横向滚动功能按钮栏

**位置**：AI 健康咨询聊天页面，输入框正上方

**展示方式**：
- 始终可见的横向滚动条
- 每个按钮由**小图标 + 文字**组成
- 按钮多时可左右滑动浏览
- 按钮的顺序、内容由后台配置决定

**按钮样式**：
- 每个按钮为圆角矩形或胶囊形
- 图标在上/左，文字在下/右（具体视设计稿）
- 按钮间有适当间距，触摸区域不小于 44x44pt

**默认按钮列表**（按初始排序）：

| 序号 | 按钮名称 | 图标描述 | 功能触发 |
|------|----------|----------|----------|
| 1 | 打电话 | 电话图标 | 进入数字人对话模式 |
| 2 | 拍照识药 | 相机图标 | 弹出拍照/选图界面 |
| 3 | 上传报告 | 文档图标 | 弹出文件/相册选择 |
| 4 | 症状自查 | 听诊器图标 | 触发 AI 引导式问诊 |
| 5 | 用药提醒 | 药丸图标 | 触发 AI 用药提醒创建 |
| 6 | 健康日记 | 笔记图标 | 触发 AI 健康数据记录 |
| 7 | 预约挂号 | 医院图标 | 触发 AI 预约引导 |
| 8 | 急救指南 | 急救图标 | 触发 AI 急救知识展示 |

#### F02：后台按钮管理

**管理入口**：后台管理系统 → AI 咨询配置 → 功能按钮管理

**可配置字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 按钮名称 | 文本 | 是 | 显示在按钮上的文字，建议 2~4 字 |
| 按钮图标 | 图片上传/图标选择 | 是 | 支持上传自定义图标或从预设图标库选择 |
| 按钮类型 | 下拉选择 | 是 | 数字人通话 / 拍照上传 / 文件上传 / AI 对话触发 / 外部链接跳转 |
| 排序权重 | 数字 | 是 | 数值越小越靠前 |
| 启用状态 | 开关 | 是 | 启用/禁用 |
| 关联参数 | JSON | 否 | 根据按钮类型不同，配置不同参数 |

**关联参数示例**：

- **数字人通话类型**：`{ "digital_human_id": "dh_001" }`（绑定数字人形象 ID）
- **AI 对话触发类型**：`{ "prompt_template": "symptoms_check" }`（触发的 AI 引导模板）
- **外部链接类型**：`{ "url": "https://xxx.com/booking" }`

**管理功能**：
- 支持拖拽排序
- 支持批量启用/禁用
- 支持预览效果

#### F03：全屏数字人通话页面

**进入方式**：点击功能按钮栏中的"打电话"按钮

**页面布局（简洁模式）**：

```
┌─────────────────────────────┐
│                             │
│                             │
│      数字人 MP4 视频        │
│      （全屏背景播放）        │
│                             │
│                             │
│                             │
├─────────────────────────────┤
│  ┌───────────────────────┐  │
│  │ 对话气泡流（半透明）    │  │
│  │ 用户：我最近血压偏高... │  │
│  │ AI：建议您注意...       │  │
│  └───────────────────────┘  │
│                             │
│         [挂断按钮]           │
│                             │
└─────────────────────────────┘
```

**页面元素**：
1. **全屏数字人视频背景**：占满整个屏幕，循环播放数字人视频
2. **底部对话气泡流区域**：半透明背景，显示最近几轮对话内容，支持上滑查看更多历史对话
3. **挂断按钮**：页面底部居中，红色圆形挂断按钮，点击后结束通话并返回聊天页

**AI 角色身份**：固定为"您的专属健康顾问"，无论用户当前选择的咨询对象是谁

#### F04：数字人视频播放

**视频素材**：每套数字人形象包含两个 MP4 视频文件：
- **静默视频**：数字人处于安静等待状态的循环视频（如微微点头、眨眼等自然动作）
- **说话视频**：数字人处于说话状态的循环视频（嘴部有说话动作）

**切换逻辑**：
- 默认播放静默视频
- 当 AI 开始 TTS 语音播报时，切换到说话视频
- 当 AI 语音播报结束后，切换回静默视频
- 视频切换需平滑过渡，避免画面跳动

**技术要求**：
- 视频文件需做多码率适配（高/中/低画质）
- 视频预加载，进入页面时两个视频均需提前缓冲
- 视频循环播放，不出现黑屏或卡顿

#### F05：语音识别（STT）

**技术方案**：腾讯云实时语音识别（ASR）

**交互模式**：VAD（语音端点检测）模式
- 进入数字人通话后，麦克风自动开启，全程持续监听
- 使用 VAD 算法智能检测用户是否说完一句话
- 检测到用户说完后，将整句语音识别结果发送给 AI 处理
- 在用户说话过程中，实时显示识别中的文字（以淡色/动态效果显示在气泡流中）

**VAD 参数配置（混合模式）**：

VAD 相关参数采用混合配置方式——常用体验参数由管理员在后台可视化调整，底层技术参数写在后端配置文件中。

*后台管理界面可调参数*：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 静音检测时长 | 1.5 秒 | 用户停顿多久后判定为说完一句话，范围 0.5~5 秒 |
| 最大单次录音时长 | 60 秒 | 用户单次连续发言的最大时长，范围 10~120 秒 |
| 语音识别语言 | 中文普通话 | 支持选择：中文普通话 / 粤语 / 英语 等 |
| 灵敏度 | 中 | VAD 触发灵敏度：高（容易触发）/ 中 / 低（不容易触发） |

*后端配置文件参数（`.env` / `config.yaml`）*：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 采样率 | 16000 Hz | 音频采样率 |
| 音频编码格式 | PCM | 编码格式（PCM / SILK / OPUS） |
| 音频通道数 | 1（单声道） | 录音通道数 |
| 数据包大小 | 640 字节 | WebSocket 单次发送音频数据包大小 |
| 引擎模型类型 | 16k_zh | 腾讯云 ASR 引擎模型标识 |

**权限处理**：
- 进入通话前检测麦克风权限
- 未授权时弹出权限申请提示，说明用途
- 用户拒绝授权后提示"需要麦克风权限才能使用语音对话功能"，并提供返回按钮

---

#### F06：语音合成（TTS）

**技术方案**：腾讯云语音合成

**播放逻辑**：
- AI 返回文本回答后，调用 TTS 将文本转为语音
- 语音播放的同时，数字人视频切换到"说话"状态
- 语音播放结束后，数字人视频切换回"静默"状态
- TTS 播放期间，STT 暂停监听（避免 AI 的声音被识别为用户输入）

**音色选择**：
- 默认使用温和、专业的女声音色
- 后台可配置不同数字人形象的 TTS 音色参数

---

#### F07：对话气泡流（字幕区域）

**位置**：数字人通话页面底部

**样式**：
- 半透明黑色/深色背景区域，覆盖在视频之上
- 高度约占屏幕 1/4 ~ 1/3
- 显示最近 3~5 轮对话内容

**气泡样式**：
- 用户发言：右对齐，浅色气泡
- AI 回答：左对齐，白色/亮色气泡
- 每条消息显示角色标识（如"您"和"健康顾问"）

**交互**：
- 支持上滑查看更多历史对话
- 新消息到来时自动滚动到底部
- AI 回答的文字逐字出现（与 TTS 播放同步）

#### F08：多套数字人形象管理

**后台管理入口**：后台管理系统 → AI 咨询配置 → 数字人形象管理

**可配置字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 形象名称 | 文本 | 是 | 如"健康顾问小薇"、"营养师小林" |
| 静默视频 | 视频上传 | 是 | MP4 格式，循环播放用 |
| 说话视频 | 视频上传 | 是 | MP4 格式，AI 说话时播放 |
| TTS 音色 | 下拉/参数 | 否 | 腾讯云 TTS 音色 ID |
| 形象描述 | 文本 | 否 | 形象的简介描述 |
| 缩略图 | 图片上传 | 否 | 在管理列表中展示的小图 |
| 启用状态 | 开关 | 是 | 启用/禁用 |

**绑定方式**：管理员在配置"打电话"功能按钮时，通过关联参数选择绑定的数字人形象

#### F09：通话结束对话同步

**触发时机**：用户点击"挂断"按钮结束数字人通话后

**同步行为**：
- 将数字人通话中的所有对话内容（用户说的 + AI 回答的）自动同步到 AI 健康咨询的聊天记录中
- 同步后的消息在聊天记录中以特殊标记展示（如带有"语音通话"标签），以区分普通文字对话
- 用户可在聊天记录中随时回看这些通话内容

**同步格式**：
- 通话开始标记：`——— 语音通话开始 HH:MM ———`
- 每条对话保留原始内容
- 通话结束标记：`——— 语音通话结束 HH:MM，时长 X 分 X 秒 ———`

#### F10：拍照识药

**触发方式**：点击"拍照识药"按钮

**交互流程**：
1. 弹出选择面板：拍照 / 从相册选择
2. 用户拍照或选择图片后，图片以消息形式发送到对话中
3. 显示"AI 正在识别药品..."的加载状态
4. AI 调用图像识别能力，识别药品名称、厂商、规格等信息
5. AI 在对话中回复识别结果，并附带用药建议、注意事项、药物相互作用提醒等

**AI 回复内容**：
- 药品名称（通用名 + 商品名）
- 药品规格、厂商
- 主要适应症
- 用法用量建议
- 常见不良反应
- 药物相互作用提醒（如用户健康档案中有其他用药记录）

---

#### F11：上传报告

**触发方式**：点击"上传报告"按钮

**交互流程**：
1. 弹出选择面板：拍照 / 从相册选择 / 选择文件
2. 用户选择体检报告图片后，图片以消息形式发送到对话中
3. 显示"AI 正在解读报告..."的加载状态
4. AI 调用 OCR + 智能分析能力，识别并解读各项指标
5. AI 在对话中回复解读结果

**AI 回复内容**：
- 识别出的各项指标及其数值
- 标注异常指标（偏高/偏低）
- 对异常指标的通俗解释
- 健康建议与注意事项
- 建议进一步检查的项目（如有需要）

---

#### F12：症状自查

**触发方式**：点击"症状自查"按钮

**交互流程**：
1. AI 在对话中发送引导消息："请描述您目前最主要的不适症状"
2. 用户描述后，AI 继续追问关键信息：
   - 症状持续时间
   - 严重程度（轻/中/重）
   - 伴随症状
   - 既往病史
   - 是否正在用药
3. 收集足够信息后，AI 给出初步分析：
   - 可能的健康问题（列出 2~3 种可能性，注明概率）
   - 建议就诊科室
   - 是否需要紧急就医
   - 日常注意事项

**安全提示**：AI 回复末尾必须附带免责声明："以上分析仅供参考，不构成医疗诊断。如症状持续或加重，请及时就医。"

---

#### F13：用药提醒

**触发方式**：点击"用药提醒"按钮

**交互流程**：
1. AI 在对话中询问：
   - 药品名称
   - 每次剂量
   - 服药频次（每日几次）
   - 具体服药时间
   - 饭前/饭后/空腹
   - 持续天数（可选）
2. 收集完成后，AI 展示提醒计划摘要，请用户确认
3. 用户确认后，自动创建用药提醒计划
4. 到时间后推送提醒通知

---

#### F14：健康日记

**触发方式**：点击"健康日记"按钮

**交互流程**：
1. AI 在对话中引导用户选择要记录的数据类型（支持多选）：
   - 血压（收缩压/舒张压）
   - 血糖（空腹/餐后）
   - 体重
   - 心率
   - 体温
   - 其他自定义指标
2. 用户逐项输入数值
3. AI 记录数据后，给出简要反馈：
   - 当前数值是否在正常范围
   - 与近期数据的趋势对比（如"您的血压较上周有所下降"）
   - 针对性的健康建议

---

#### F15：预约挂号

**触发方式**：点击"预约挂号"按钮

**交互流程**：
1. AI 在对话中引导用户依次选择：
   - 症状/需求描述（AI 推荐科室）
   - 确认科室
   - 选择医生（展示可选医生列表）
   - 选择就诊时间（展示可选时段）
2. AI 展示预约信息摘要，请用户确认
3. 用户确认后生成预约记录
4. 发送预约成功通知，包含就诊时间、地点、注意事项

---

#### F16：急救指南

**触发方式**：点击"急救指南"按钮

**交互流程**：
1. AI 在对话中展示常见急救场景列表（卡片式展示）：
   - 心肺复苏（CPR）
   - 噎住/异物卡喉

## 4. 非功能性需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 功能按钮区加载 | 页面打开后 200ms 内渲染完成 |
| 数字人视频首帧 | 进入通话页后 2 秒内显示首帧画面 |
| 语音识别延迟 | 用户说完到文字显示 < 1 秒 |
| TTS 合成延迟 | AI 回复到语音开始播放 < 2 秒 |
| 视频切换延迟 | 静默/说话视频切换 < 200ms |
| 图片识别（拍照识药/上传报告） | 结果返回 < 5 秒 |

### 4.2 安全要求

- 麦克风权限需用户明确授权
- 语音数据传输使用 HTTPS/WSS 加密
- 通话对话内容仅存储在用户自己的聊天记录中
- 上传的图片/报告需做隐私信息脱敏处理
- 后台管理操作需有权限控制和操作日志

### 4.3 兼容性要求

| 端 | 兼容范围 |
|----|----------|
| 微信小程序 | 微信 7.0+，支持基础库 2.10.0+ |
| H5 Web | 现代浏览器（Chrome 80+，Safari 13+），移动端优先 |
| Flutter App | iOS 12+，Android 6.0+ |

### 4.4 网络与离线

- 数字人通话模式需要稳定网络连接
- 网络断开时自动暂停，显示"网络连接中…"提示，网络恢复后自动继续通话
- 功能按钮区的配置数据需做本地缓存，弱网环境下可展示按钮（但点击后提示网络不可用）

## 5. 业务规则与约束

| 规则编号 | 规则描述 |
|----------|----------|
| BR01 | 功能按钮的配置数据从后台 API 获取，前端按排序权重从小到大展示 |

| BR06 | 拍照识药和上传报告功能，单次上传图片不超过 5 张，每张不超过 10MB |

| BR08 | 通话中的对话同步到聊天记录后，消息类型标记为"voice_call"以便区分 |

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 功能按钮配置 API 请求失败 | 使用本地缓存数据展示按钮；如无缓存则隐藏按钮区域，不影响正常聊天 |
| 数字人通话中网络断开 | 暂停通话，页面显示"网络连接中…"动画提示，网络恢复后自动续连 |
| 数字人通话中 STT 识别失败 | 在气泡流中显示"语音识别失败，请再说一次"提示 |
| 数字人通话中 TTS 合成失败 | 跳过语音播报，仅显示文字气泡，数字人保持静默视频 |
| 数字人视频加载失败 | 显示默认的健康顾问头像静态图 + 文字提示"视频加载中"，功能不受影响 |
| 麦克风权限被拒绝 | 弹出提示引导用户开启权限，提供"返回"按钮退出通话页 |
| 拍照识药图片模糊/无法识别 | AI 回复"图片不够清晰，请重新拍摄，建议保持药品文字清晰可见" |
| 上传报告格式不支持 | 提示"暂不支持该格式，请上传 JPG/PNG 格式的图片" |
| 后台未配置任何功能按钮 | 前端不显示功能按钮区域，聊天页恢复原始布局 |
| 后台未配置数字人形象 | "打电话"按钮点击后提示"暂未开通此功能"，不进入通话页 |
