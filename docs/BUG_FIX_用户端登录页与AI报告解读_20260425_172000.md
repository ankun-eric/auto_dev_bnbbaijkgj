## 2. Bug 描述

### 2.1 错误现象

#### Bug 1：用户端登录页 LOGO 显示不出来

- **现象**：登录页 LOGO 区域**有占位框 / 裂图小图标**（不是完全空白），说明前端拿到了 `logo_url` 字段，但浏览器加载该 URL 时返回 404 / 无法访问。
- **历史背景**：之前 LOGO 是正常显示的。说明此前曾在 Admin 后台上传过 LOGO，数据库里仍保留旧记录的 URL，但实际图片文件已不存在 / 路径已变更。
- **接口链路**：H5 登录页 → `GET /api/settings/logo` → 拿到 `data.logo_url` → `<img src={logoUrl}>` → 加载 404。

#### Bug 2："未注册手机号验证后将自动创建会员"等辅助提示需要整块删除

- **现象**：登录页（手机号 + 验证码登录区下方）会显示一段辅助提示文本，包含两句逻辑：
  - 当"开放自助注册"时显示：**"未注册手机号验证后将自动创建会员"**
  - 当"不开放自助注册"时显示：**"当前未开放自助注册，仅支持已注册账号登录"**
- **业务方诉求**：**整块辅助提示都不要了**（不论开/关注册都不显示）。

#### Bug 3：H5 端「其他登录方式」区块整块删除（含微信入口）

- **现象**：H5 登录页底部存在一块「**其他登录方式**」区域，目前里面只挂着一个**微信**渠道按钮。
- **业务方诉求**：**整块"其他登录方式"区域都不要了**（含分割线、标题文字、微信图标按钮，全部移除）。
- **范围限定**：**仅 H5 端**进行该删除操作；小程序/APP 端的微信登录方式**不动**。

#### Bug 4：从"AI 报告解读"进入 AI 咨询页提示"AI 服务未配置，请联系管理员配置 AI 模型"

- **现象**：用户做完体检报告 OCR + AI 解读之后，点击进入"针对该报告的 AI 咨询"会话页，**首屏即弹出/展示**：
  > AI 服务未配置，请联系管理员配置 AI 模型。
- **关键反证**：用户**直接进入"AI 健康咨询"入口**对话**完全正常**，说明 AI 模型本身在 Admin 后台是配置好且可用的。
- **根因方向**：「AI 报告解读 → 咨询会话」的请求链路与「直接 AI 健康咨询」走的不是同一套配置读取分支，导致后端在 `ai_service` 中走到了 `config["base_url"] is None / config["model"] is None` 的兜底分支，吐出 "AI 服务未配置..." 这句话。
- **影响范围**：**所有报告解读后的对话**都报错；**H5 / 小程序 / APP 三端均能稳定复现**。

## 3. 根因分析与修复方案

### Bug 1：登录页 LOGO 裂图（404）

**根因**：

数据库 `system_settings.logo_url` 字段中保留了旧值，但磁盘上对应的图片文件已不存在（典型场景：容器重建后 `uploads/` 目录被覆盖、迁移环境时未带上文件、之前手工删过文件等）。前端拿到不可达 URL → 浏览器 404 → 显示裂图。

**修复方案**：

1. **后端 `GET /api/settings/logo`** 增加"文件存在性校验"：
   - 拼好绝对 URL 后，**先检查物理文件是否存在**（或先做一次 HEAD 探活）；不存在则在响应中将 `logo_url` 置为 `null`，并把数据库字段同步清空（自愈逻辑）。
2. **前端登录页（含 H5 / 小程序 / APP / Admin）**：
   - 当 `logo_url` 为空 / 加载失败（`<img onError>`）时，**回退显示内置默认 LOGO**（项目内静态资源），不再出现裂图占位。
3. **Admin 后台「品牌设置」**：
   - 增强提示，告知管理员当前 LOGO 是否真实可访问（探活成功/失败状态徽标）。

### Bug 2：删除登录页"未注册手机号..."等整块辅助提示

**根因**：

当前 H5 登录页 `h5-web/src/app/login/page.tsx` 中根据 `registerSettings.enable_self_registration` 渲染了两条辅助文案；小程序、APP 端有相同的提示组件。

**修复方案**：

- **H5 端 `login/page.tsx`**：删除"未注册手机号验证后将自动创建会员"和"当前未开放自助注册，仅支持已注册账号登录"**整块辅助提示 DOM**（含包裹容器、空行间距）。
- **小程序 `miniprogram/pages/login/index.wxml` + `index.js`**：同步删除对应文案块。
- **Flutter APP `flutter_app/lib/screens/login_screen.dart`**：同步删除对应 Widget。
- 仅删除 UI，**不动后端 `register-settings` 接口**（后台开关仍保留供其他业务使用）。

### Bug 3：H5「其他登录方式」整块删除（含微信）

**根因**：

H5 登录页底部存在一段"其他登录方式 + 微信图标"区域。

**修复方案**：

- **仅修改 H5 端 `h5-web/src/app/login/page.tsx`**：
  - 删除分割线 + "其他登录方式" 标题 + 微信图标按钮 + 相关 onClick 处理函数 + 不再使用的图标资源引用。
- **小程序、APP 端不动**（其微信登录入口保留）。

### Bug 4：AI 报告解读进入会话提示"AI 服务未配置"

**根因初判**（以最终代码为准，预计在以下 2 类原因中之一）：

1. **报告解读会话创建时未指定模型**：报告解读侧会话创建走的是 `report_interpret` / `checkup` 相关接口，给 AI 调用传入的 `model_id` / `prompt_template` 与"通用 AI 健康咨询"不一致，导致 `_get_active_model_config(db)` 在该上下文下取不到 `base_url` / `model`，进入兜底分支 → 输出"AI 服务未配置..."。
2. **报告解读 prompt 模板缺失/被禁用**：报告解读使用的是独立的 prompt template 配置，模板被禁用或被误删后，后端 fallback 链路触发"未配置"提示。

**修复方案**：

1. **统一模型读取入口**：让"报告解读会话"与"AI 健康咨询会话"使用**完全相同的 `_get_active_model_config` 调用路径**与同一份 active model 数据，不再从其他配置表读取。
2. **报告解读 prompt 模板兜底**：当报告解读专用 prompt 模板缺失时，**自动回退到默认 AI 健康咨询 prompt**，而非提示"未配置"。
3. **后端兜底文案精细化**：`ai_service.py` 中的"AI 服务未配置..."兜底分支增加日志埋点，记录调用方上下文（来源接口、session 类型、用户/咨询人 ID、传入的 model_id），便于以后快速定位类似问题。
4. **多端联动验证**：H5、小程序、APP 三端均跑通"上传报告 → 解读完成 → 进入 AI 咨询"全链路，确保不再出现兜底文案。
5. **回归用例**：补充自动化测试 `test_report_interpret_chat_uses_active_model.py`，覆盖"报告解读会话首条 AI 回复必须不是 'AI 服务未配置'"。
