# 咨询人信息收起/展开改造 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

目前在 **AI报告解读（体检报告）**、**用药参考（拍照识药）**、**健康自查** 三个功能的最后一步"选择咨询人"中，当用户选择了某位家庭成员后，系统会一次性展示该成员的全部健康档案信息（姓名、出生日期、性别、身高、体重、既往病史、过敏史、家族遗传病史等），信息量较大，页面较长。

本次改造目的是 **优化信息展示层次**，让用户聚焦核心信息，减少视觉负担，同时保持完整信息的可编辑能力。

### 1.2 目标用户

所有使用 AI 健康咨询功能的注册用户（包括为家庭成员咨询的用户）。

### 1.3 核心价值

- 减少选择咨询人后的信息视觉负担，提升页面简洁度
- 保留完整信息的查看和编辑能力，不损失功能
- 通过档案完善度提示，引导用户补全健康档案信息，提高 AI 分析的准确性

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 信息收起/展开 | 默认只展示姓名、出生日期、性别，其余信息收起 | P0 | 核心改造 |
| F02 | 展开按钮 | 卡片底部"更多信息"分隔线按钮，含完善度提示 | P0 | 核心交互 |
| F03 | 档案完善度提示 | 根据关键字段填写情况显示"已完善"或"待完善" | P0 | 引导用户完善信息 |
| F04 | 字段可编辑 | 所有字段（默认展示区和展开区）均可直接编辑 | P0 | 保持现有编辑能力 |
| F05 | 提交时保存 | 点击"开始咨询"时同步保存修改到健康档案 | P0 | 数据同步 |
| F06 | 提交校验 | 必填字段（姓名、性别、出生日期）校验 | P0 | 数据完整性 |
| F07 | 展开/收起动画 | 平滑的滑动过渡动画效果 | P1 | 体验优化 |

### 2.2 功能详细描述

#### F01：信息收起/展开

**改造范围**：AI报告解读（体检报告）、用药参考（拍照识药）、健康自查，三个模块同时改造，行为完全一致。

**默认展示区域（收起状态）**：
- 姓名
- 出生日期
- 性别

以上三个字段保持与展开后其他字段一样的表单/列表风格，支持直接编辑。

**展开区域（点击"更多信息"后显示）**：
- 身高
- 体重
- 既往病史
- 过敏史
- 家族遗传病史
- 其他现有字段（保持现有顺序不变）

**展开状态不记忆**：每次进入"选择咨询人"步骤，都默认收起，需要用户手动展开。切换不同咨询人时也恢复为收起状态。

#### F02：展开按钮

**位置**：卡片底部，横跨整个宽度。

**视觉样式**：简洁线条型 —— 一条细分隔线，中间嵌入文字和箭头。

**按钮状态变化**：

| 状态 | 按钮文字（示例） | 箭头 |
|------|-----------------|------|
| 收起 + 已完善 | `更多信息（已完善）` | ▼ |
| 收起 + 待完善 | `更多信息（待完善）` | ▼ |
| 展开 + 已完善 | `收起信息（已完善）` | ▲ |
| 展开 + 待完善 | `收起信息（待完善）` | ▲ |

**颜色规则**：
- "已完善" 文字使用 **绿色**
- "待完善" 文字使用 **橙色**

#### F03：档案完善度提示

**判断标准**：以下 **5 项字段全部填写** 才算"已完善"：
1. 姓名
2. 性别
3. 出生日期
4. 身高
5. 体重

任意一项为空则显示"待完善"。

**提示方式**：仅显示"已完善"或"待完善"两种状态文字，不显示具体几项未填。

**注意**：身高和体重是完善度判断的依据字段，但 **不是提交咨询的必填字段**（即身高体重为空时显示"待完善"，但不阻止用户提交咨询）。

#### F04：字段可编辑

- 默认展示区域的 3 个字段（姓名、出生日期、性别）和展开区域的所有字段，均可直接在当前页面编辑修改
- 修改后的数据将同步保存到该家庭成员的健康档案（在提交咨询时保存）
- 空字段显示占位提示（如"请选择性别"、"请输入出生日期"），用户自行填写即可，无需特殊高亮处理

#### F05：提交时保存

**保存时机**：用户点击"开始咨询"（提交整个表单）时，一并将修改的信息同步保存到家庭成员健康档案。

**保存提示**：
- 仅在校验通过并成功提交后，弹出 Toast 轻提示：`健康档案信息已同步更新`
- 如果校验不通过，只弹校验失败的提示，不弹保存提示

**切换咨询人时的处理**：如果用户选择了咨询人 A 并修改了信息，但未提交就切换到咨询人 B，对 A 的修改 **直接丢弃**，不做任何提示。

#### F06：提交校验

**必填字段**（阻止提交）：
1. 姓名
2. 性别
3. 出生日期

**非必填字段**（不阻止提交）：
- 身高、体重、既往病史、过敏史、家族遗传病史等

**校验失败时的提示方式**（延续现有方式）：
1. 在未填写的必填字段下方显示红色错误提示文字（如"请输入姓名"、"请选择性别"、"请选择出生日期"）
2. 同时弹出全局 Toast 轻提示：`请填写完整的必填信息`

**咨询人选择为强制**：必须选择一位咨询人才能继续，不可跳过。

#### F07：展开/收起动画

- 展开和收起时使用 **平滑的滑动过渡动画** 效果
- 展开：从上到下滑出，内容区域高度逐渐增加
- 收起：从下到上滑入，内容区域高度逐渐减小

## 3. 页面/界面设计

### 3.1 页面结构与导航

本次改造不涉及页面结构和导航变化，仅改造以下三个页面中"选择咨询人"步骤的信息展示区域：

1. **健康自查页** (`/symptom`)
2. **AI报告解读页** (`/checkup`)
3. **用药参考页** (`/drug`)

### 3.2 各页面功能说明

#### 选择咨询人后的信息展示区域（三个模块统一）

**收起状态（默认）**：

```
┌─────────────────────────────────────┐
│  姓名：[张三            ]  (可编辑) │
│  出生日期：[1990-01-15   ]  (可编辑) │
│  性别：[男 ▼]              (可编辑) │
│─────────────────────────────────────│
│   更多信息（待完善）▼               │
└─────────────────────────────────────┘
```

**展开状态**：

```
┌─────────────────────────────────────┐
│  姓名：[张三            ]  (可编辑) │
│  出生日期：[1990-01-15   ]  (可编辑) │
│  性别：[男 ▼]              (可编辑) │
│                                     │
│  身高(cm)：[175          ]  (可编辑) │
│  体重(kg)：[70           ]  (可编辑) │
│  既往病史：[高血压        ]  (可编辑) │
│  过敏史：[无             ]  (可编辑) │
│  家族遗传病史：[无        ]  (可编辑) │
│─────────────────────────────────────│
│   收起信息（已完善）▲               │
└─────────────────────────────────────┘
```

**"更多信息"按钮样式**：
- 横跨卡片整个宽度的细分隔线
- 文字和箭头居中嵌入分隔线中
- "已完善"使用绿色，"待完善"使用橙色
- 其余文字（"更多信息"/"收起信息"）和箭头使用常规灰色

## 4. 非功能性需求

### 4.1 性能要求

- 展开/收起动画应流畅无卡顿，动画时长建议 200~300ms
- 切换咨询人时的状态重置应即时完成，无明显延迟

### 4.2 安全要求

- 无额外安全要求，沿用现有的用户认证和数据权限机制

### 4.3 兼容性要求

- 本次改造需在以下端同步生效：
  - H5 Web 端（Next.js）
  - 微信小程序端
  - Flutter App 端（iOS / Android）
- 三端行为和视觉保持一致

## 5. 业务规则与约束

| 编号 | 规则 | 说明 |
|------|------|------|
| R01 | 默认收起 | 每次进入选择咨询人步骤，信息默认收起 |
| R02 | 切换咨询人重置 | 切换不同咨询人时，展开状态恢复为收起，对前一位咨询人的未保存修改直接丢弃 |
| R03 | 完善度判断 | 姓名 + 性别 + 出生日期 + 身高 + 体重，5项全填 = 已完善，否则 = 待完善 |
| R04 | 必填 ≠ 完善度 | 提交校验只看姓名、性别、出生日期 3 项必填；完善度判断看 5 项 |
| R05 | 提交时保存 | 信息修改不即时保存，仅在用户点击"开始咨询"提交时一并保存到健康档案 |
| R06 | 保存提示 | 校验通过且提交成功后弹 Toast：`健康档案信息已同步更新` |
| R07 | 字段顺序 | 展开区域字段保持现有顺序不变 |
| R08 | 三模块统一 | AI报告解读、用药参考、健康自查的收起/展开逻辑完全一致 |
| R09 | 收起状态提交 | 收起状态下点击提交，走正常的提交+保存流程，不关心展开区域 |

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 新添加的家庭成员（仅有姓名等基础信息） | 默认收起，底部显示"更多信息（待完善）▼"，用户可手动展开填写 |
| 默认展示区域的字段为空（如新成员没填性别） | 显示占位提示（如"请选择性别"），不做特殊高亮处理 |
| 用户修改信息后切换咨询人 | 直接丢弃修改，不弹确认提示 |
| 提交时必填字段（姓名/性别/出生日期）未填 | 字段下方显示红色错误文字 + Toast 提示"请填写完整的必填信息"，阻止提交 |
| 提交时身高体重未填 | 不阻止提交，但完善度显示"待完善" |
| 网络异常导致保存失败 | 沿用现有的网络异常处理逻辑 |
| 病史类字段的处理 | 沿用代码中现有的处理方式（后续开发时由开发者查看代码确认当前实现） |

## 8. 补充说明

- 本次改造涉及 **H5 Web、微信小程序、Flutter App** 三端的前端页面修改
- 后端 API 无需改动，沿用现有的家庭成员信息查询和更新接口
- 三个模块（AI报告解读、用药参考、健康自查）共用相同的咨询人选择逻辑，建议将收起/展开组件抽取为公共组件，一次开发三个模块同时生效
- 病史类字段（既往病史、过敏史、家族遗传病史）的"未填写"状态处理，开发时需查看代码中当前的实现方式并保持一致
