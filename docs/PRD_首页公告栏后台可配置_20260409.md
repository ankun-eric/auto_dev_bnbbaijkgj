# 首页公告栏后台可配置功能 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

首页顶部 NoticeBar 公告内容目前硬编码在前端代码中（`h5-web/src/app/(tabs)/home/page.tsx` 第 107 行），无法灵活修改，每次变更都需要开发人员介入修改代码并重新发布。

本需求将其改造为后台动态配置，支持多条轮播展示，让运营/管理人员可以随时在后台管理公告内容，无需改代码。

### 1.2 目标用户

- **后台管理员**：负责在管理后台新增、编辑、删除、排序公告内容
- **H5 前端用户**：在首页看到动态展示的公告轮播

### 1.3 核心价值

- 公告内容实时可配，运营灵活度大幅提升
- 支持多条公告按时间段和启用状态精细控制展示
- 点击公告可跳转站内页面，增强用户引导能力

## 2. 功能需求

### 2.1 功能清单总览

| 编号 | 功能模块 | 功能点 | 优先级 | 说明 |
|------|----------|--------|--------|------|
| F01 | 后台公告管理 | 新增公告 | 高 | 填写公告内容、跳转路径、时间段、排序权重 |
| F02 | 后台公告管理 | 编辑公告 | 高 | 修改已有公告的所有字段 |
| F03 | 后台公告管理 | 删除公告 | 高 | 永久删除公告记录 |
| F04 | 后台公告管理 | 启用/禁用公告 | 高 | 快捷开关，独立于时间段控制 |
| F05 | 后台公告管理 | 手动排序 | 中 | 可输入排序权重或拖拽调整顺序 |
| F06 | 前端首页展示 | 动态获取公告 | 高 | 从接口获取有效公告列表，缓存 30 分钟 |
| F07 | 前端首页展示 | 多条公告轮播 | 高 | 保持 NoticeBar 横向滚动样式，内容动态化 |
| F08 | 前端首页展示 | 点击跳转 | 中 | 点击有跳转路径的公告，跳转到对应站内页面 |

### 2.2 功能详细描述

#### F01-F03：公告增删改

- 管理员在"公告管理"菜单中可对公告进行增删改操作
- 新增/编辑公告时，弹出表单，包含以下字段：
  - 公告文字（必填，文本输入框）
  - 跳转路径（选填，文本输入框，如 `/health-profile`）
  - 生效开始时间（必填，日期时间选择器）
  - 生效结束时间（必填，日期时间选择器，须晚于开始时间）
  - 启用/禁用开关（必填，默认启用）
  - 排序权重（必填，整数，数值越小越靠前，默认 100）
- 删除操作需二次确认弹窗，防止误删

#### F04：启用/禁用

- 公告列表中每条公告有快捷启用/禁用开关（Toggle）
- 禁用后该公告不在前端展示，即使在生效时间段内
- 启用/禁用操作即时生效（下次缓存刷新后前端更新）

#### F05：手动排序

- 公告列表支持通过输入排序权重值来调整顺序
- 排序权重数值越小，在前端 NoticeBar 中展示越靠前
- 可选支持拖拽排序（拖拽后自动更新权重值）

#### F06：前端动态获取公告

- 前端首页加载时，调用接口 `GET /api/notices/active` 获取当前有效公告列表
- 接口返回已过滤（在生效时间段内 且 启用状态）的公告，按排序权重升序排列
- 前端对接口返回数据做本地缓存，缓存时长 **30 分钟**
- 缓存期间不重复请求接口；缓存过期后自动重新请求

#### F07：多条公告轮播展示

- 保持现有 NoticeBar 横向滚动样式不变
- 将原来硬编码的单条文字替换为从接口动态获取的多条公告内容
- 多条公告在 NoticeBar 中按顺序依次展示（轮播/滚动）
- 若接口返回为空（无有效公告），NoticeBar 不显示或显示默认文字（开发自行决定）

#### F08：点击跳转

- 若公告配置了跳转路径，点击该公告条目，跳转到对应站内页面
- 若未配置跳转路径，点击无任何反应
- 跳转方式为站内路由跳转（不打开外部浏览器）

## 3. 页面/界面设计

### 3.1 页面结构与导航

**后台管理系统新增菜单：**

```
后台管理系统

└── 公告管理（新增）
    └── 公告列表页
        └── 新增/编辑公告弹窗
```

**H5 前端首页（改造）：**

```
首页
└── 顶部 NoticeBar（原硬编码 → 改为动态数据）
```

### 3.2 各页面功能说明

#### 后台：公告管理列表页

| 区域 | 说明 |
|------|------|
| 页面标题 | 公告管理 |
| 新增按钮 | 点击打开新增公告弹窗 |
| 公告列表表格 | 展示所有公告，含：序号、公告文字（截断显示）、跳转路径、生效时间段、启用状态、排序权重、操作列 |
| 操作列 | 编辑、删除、启用/禁用开关 |
| 排序 | 可通过排序权重字段排序，或拖拽调整 |

#### 后台：新增/编辑公告弹窗

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 公告文字 | 文本输入框 | 是 | 展示在 NoticeBar 的内容 |
| 跳转路径 | 文本输入框 | 否 | 站内页面路径，如 `/health-profile` |
| 生效开始时间 | 日期时间选择器 | 是 | 公告开始展示时间 |
| 生效结束时间 | 日期时间选择器 | 是 | 公告停止展示时间，须晚于开始时间 |
| 启用开关 | Toggle | 是 | 默认启用 |
| 排序权重 | 数字输入框 | 是 | 默认 100，越小越靠前 |

#### H5 前端：首页 NoticeBar（改造）

- 样式：保持原有横向滚动 NoticeBar 不变
- 数据：从接口动态获取，本地缓存 30 分钟
- 交互：有跳转路径的公告点击后站内跳转

## 4. 数据模型设计

### 4.1 数据实体与字段

**表名：`notices`**

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | 主键，自增 | 公告唯一标识 |
| content | VARCHAR(500) | NOT NULL | 公告文字内容 |
| link_path | VARCHAR(200) | NULL | 站内跳转路径，如 `/health-profile` |
| start_time | DATETIME | NOT NULL | 生效开始时间 |
| end_time | DATETIME | NOT NULL | 生效结束时间 |
| is_enabled | TINYINT(1) | NOT NULL, DEFAULT 1 | 启用状态：1=启用，0=禁用 |
| sort_order | INT | NOT NULL, DEFAULT 100 | 排序权重，越小越靠前 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 最后更新时间 |

### 4.2 实体关系

公告（notices）为独立表，暂无外键关联其他实体。

## 5. 接口需求

### 5.1 接口清单

| 编号 | 方法 | 路径 | 说明 | 请求参数 | 返回数据 |
|------|------|------|------|----------|----------|
| API01 | GET | `/api/notices/active` | 前端获取当前有效公告列表 | 无 | 公告列表（已过滤+排序） |
| API02 | GET | `/api/admin/notices` | 后台获取全部公告列表 | 分页参数（可选） | 公告列表（含全部状态） |
| API03 | POST | `/api/admin/notices` | 新增公告 | 公告字段 JSON | 新增的公告对象 |
| API04 | PUT | `/api/admin/notices/{id}` | 编辑公告 | 公告字段 JSON | 更新后的公告对象 |
| API05 | DELETE | `/api/admin/notices/{id}` | 删除公告 | 路径参数 id | 操作结果 |
| API06 | PATCH | `/api/admin/notices/{id}/status` | 启用/禁用公告 | `{"is_enabled": 0/1}` | 更新后的公告对象 |
| API07 | PUT | `/api/admin/notices/sort` | 批量更新排序权重 | `[{"id": 1, "sort_order": 10}, ...]` | 操作结果 |

**API01 过滤规则（后端实现）：**
- `is_enabled = 1`
- `start_time <= 当前时间 <= end_time`
- 按 `sort_order` 升序排列

## 6. 非功能性需求

### 6.1 性能要求

- 前端公告数据本地缓存 **30 分钟**，缓存期间不重复请求接口
- 后端 `GET /api/notices/active` 接口响应时间 < 200ms

### 6.2 安全要求

- 后台管理接口（API02-API07）需鉴权，仅登录的管理员可访问
- 前端公告接口（API01）为公开接口，无需鉴权

### 6.3 兼容性要求

- H5 前端兼容现有项目支持的主流移动端浏览器
- 后台管理页面兼容现有管理系统支持的 PC 浏览器

---

## 7. 业务规则与约束

1. 公告的生效结束时间必须晚于生效开始时间，否则前端报错提示
2. 一条公告同时满足"在生效时间段内"且"启用状态为启用"才会在前端展示
3. 若当前无任何有效公告，前端 NoticeBar 不显示（或显示默认占位文字，由开发决定）
4. 公告条数无上限
5. 排序权重相同时，按创建时间升序排列作为次级排序

---

## 8. 权限设计

| 角色 | 权限说明 |
|------|----------|
| 普通后台管理员 | 可访问"公告管理"菜单，可进行新增、编辑、删除、启用/禁用、排序操作 |
| 超级管理员 | 同普通管理员，拥有全部操作权限 |
| H5 前端用户 | 只能查看首页展示的有效公告，无管理权限 |

---

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 接口请求失败（网络异常） | 前端 NoticeBar 不显示，不影响页面其他功能 |
| 缓存数据为空且接口返回空列表 | NoticeBar 不显示 |
| 公告文字过长 | NoticeBar 横向滚动展示，不截断 |
| 跳转路径填写错误（页面不存在） | 跳转后显示 404 页面，不做特殊处理 |
| 生效结束时间早于开始时间 | 后台表单校验报错，不允许提交 |
| 删除正在展示中的公告 | 允许删除，下次缓存刷新后前端自动更新 |
