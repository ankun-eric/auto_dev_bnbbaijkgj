# 比尼健康 — 语音搜索多端升级 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

比尼健康（bini-health）已上线统一搜索功能，其中语音搜索模块（F05）最初设计仅面向 H5 端，采用"点击 → 半屏弹窗 → 录音 → 识别 → 回填搜索框"的交互方式。但在实际测试中发现以下问题：

1. **H5 端语音搜索未真正接通**：页面上显示"语音搜索功能即将上线"的占位文案，实际并未对接麦克风录音和 ASR 识别流程，用户说话无法被识别
2. **浏览器麦克风权限未获取**：H5 端未通过 `navigator.mediaDevices.getUserMedia` 请求麦克风权限，导致无法录音
3. **仅支持 H5 端**：小程序和 Flutter App 端完全没有语音搜索能力

本次需求将语音搜索功能**全面升级**，覆盖 H5、微信小程序、Flutter App 三端，同时将交互体验升级为更接近微信语音消息的长按录音方式，提供更自然、更高效的语音搜索体验。

### 1.2 目标用户

- 比尼健康全平台用户（H5 网页端、微信小程序端、Flutter App 端）

### 1.3 核心价值

- **多端统一体验**：H5、小程序、Flutter App 三端同时上线语音搜索，用户在任何端都能使用
- **自然交互方式**：长按录音、松手识别，接近微信语音消息的操作习惯，零学习成本
- **智能搜索衔接**：识别结果自动填入搜索框 → 2 秒倒计时自动搜索，减少操作步骤
- **可靠的权限与容错处理**：完善的权限获取流程、录音时长限制、失败重试机制

### 2.2 功能详细描述

#### V01 — 长按麦克风按钮录音

**入口**：搜索输入框右侧的麦克风图标按钮（沿用已有设计）。

**交互方式**：
- 用户**长按**麦克风按钮开始录音
- 用户**松手**即停止录音，自动进入 ASR 识别流程
- 类似微信语音消息的操作体验，用户无需额外点击"停止"按钮

**各端实现方案**：

| 端 | 录音技术 | ASR 方案 |
|----|----------|----------|
| H5 网页端 | 浏览器原生 `MediaRecorder` API（`navigator.mediaDevices.getUserMedia`） | 调用后端腾讯云 ASR 接口（已有 `POST /api/search/asr/token`） |
| 微信小程序 | 微信原生 `wx.getRecorderManager()` | 微信同声传译插件 / 微信自带语音转文字能力 |
| Flutter App | `record` 插件（`package:record`） | 调用后端腾讯云 ASR 接口 |

---

#### V02 — 录音中视觉反馈

**交互方式**：用户长按麦克风按钮后，立即展示以下视觉效果：

1. **全屏半透明遮罩**：覆盖整个页面，深色半透明背景（opacity 约 0.6），聚焦用户注意力
2. **中央声波动画**：遮罩正中央显示大尺寸声波/脉冲动画，跟随用户说话的音量实时波动
3. **底部提示文字**：遮罩底部居中显示文字 **"松开结束"**
4. **录音时长显示**：声波动画上方显示当前已录音时长（如 "3s / 15s"）

**状态变化**：
- **长按中**：显示遮罩 + 声波动画 + "松开结束"
- **松手后（识别中）**：遮罩保留，声波动画替换为加载动画 + "正在识别..."
- **识别完成**：遮罩消失，文字填入搜索框

---

#### V03 — 录音时长限制

- 最长录音时间：**15 秒**
- 到达 15 秒时自动停止录音，无需用户操作
- 自动停止后进入 ASR 识别流程（与松手停止流程一致）
- 录音过程中通过时长显示（V02）让用户感知剩余时间
- 当剩余 3 秒时，时长数字变为红色提醒

---

#### V04 — ASR 语音转文字

**H5 端流程**：
1. 用户松手 / 自动停止后，将录音音频数据（Blob）发送到后端
2. 后端调用腾讯云一句话识别 ASR 接口
3. 返回识别文字结果

**小程序端流程**：
1. 录音结束后，使用微信自带的语音转文字能力进行识别
2. 识别结果返回前端

**Flutter App 端流程**：
1. 录音结束后，将音频文件上传至后端
2. 后端调用腾讯云 ASR 接口
3. 返回识别文字结果

---

#### V05 — 识别结果填入搜索框

- ASR 识别完成后，将识别出的文字**自动填入搜索输入框**
- 搜索框获取焦点，光标定位到文字末尾
- 用户可以直接修改识别结果（手动编辑搜索框中的文字）

---

#### V06 — 2 秒自动搜索倒计时

**核心逻辑**：
1. 识别结果填入搜索框后，立即启动 **2 秒倒计时**
2. 倒计时结束后，**自动触发搜索**，跳转到搜索结果页
3. 倒计时期间，用户可以：
   - **手动修改文字**：修改搜索框中的文字后，倒计时重置为 2 秒重新开始
   - **点击取消**：取消自动搜索，保留搜索框中的文字，用户手动决定何时搜索
   - **直接点击搜索按钮/回车**：立即搜索，不等倒计时

---

#### V07 — 倒计时视觉提示

- 识别结果填入搜索框后，搜索框下方出现一行小字提示：**"2 秒后自动搜索，点击可取消"**
- "点击可取消"为可点击文字（链接样式），点击后取消自动搜索
- 倒计时过程中数字实时更新（2→1→0 搜索）
- 自动搜索触发后或用户取消后，提示文字消失

---

#### V08 — 录音失败 / 识别失败处理

**错误场景与处理方式**：

| 错误场景 | 处理方式 |
|----------|----------|
| 录音启动失败（硬件/权限问题） | 遮罩内显示错误图标 + "录音启动失败" + "重试"按钮 |
| 未检测到语音（用户没说话） | 遮罩内显示提示图标 + "未检测到语音，请重试" + "重试"按钮 |
| ASR 识别失败（网络/服务异常） | 遮罩内显示错误图标 + "识别失败，请重试" + "重试"按钮 |
| ASR 识别超时（超过 5 秒无返回） | 遮罩内显示错误图标 + "识别超时，请重试" + "重试"按钮 |

**"重试"按钮行为**：点击后重新进入录音状态（需用户再次长按录音），同时遮罩右上角有"关闭"按钮可退出。

---

#### V09 — 麦克风权限请求

**各端处理方式**：

| 端 | 权限请求方式 | 被拒绝后处理 |
|----|------------|------------|
| H5 网页端 | 浏览器默认权限弹窗（`getUserMedia` 触发） | 显示 Toast "无法访问麦克风" |
| 微信小程序 | 微信系统级授权弹窗（`wx.authorize`） | 显示 Toast "无法访问麦克风" |
| Flutter App | 系统级权限弹窗（Android/iOS） | 显示 Toast "无法访问麦克风" |

三端均使用各自平台默认的权限弹窗，不做额外自定义引导。

---

#### V10 — 不支持语音时隐藏按钮

- **H5 端**：检测浏览器是否支持 `navigator.mediaDevices.getUserMedia`，不支持时隐藏麦克风按钮
- **小程序端**：微信基础库版本 1.6.0 以上均支持，低版本隐藏
- **Flutter App 端**：录音插件不可用时隐藏

## 3. 页面/界面设计

### 3.1 页面结构与导航

```
搜索页（/search 或各端对应页面）
  └─ 搜索输入框右侧麦克风按钮
       ├─ 长按 → 全屏半透明遮罩（录音中）
       │    ├─ 中央声波动画
```

## 5. 接口需求

### 5.1 接口清单

#### 5.1.1 新增接口

| 编号 | 方法 | 路径 | 说明 | 请求参数 | 返回数据 |
|------|------|------|------|----------|----------|
| C01 | POST | /api/search/asr/recognize | 语音识别（上传音频） | audio_file(multipart), format(wav/mp3/m4a), sample_rate(16000) | {text: "识别文字", confidence: 0.95} 或 {error: "错误信息"} |

#### 5.1.2 复用已有接口

| 编号 | 方法 | 路径 | 说明 | 变更 |
|------|------|------|------|------|
| A08 | POST | /api/search/asr/token | 获取 ASR 临时凭证 | 无变更，三端复用 |
| A01 | GET | /api/search | 统一搜索 | 无变更 |

### 5.2 接口详细说明

#### C01 — 语音识别接口

**请求**：
```
POST /api/search/asr/recognize
Content-Type: multipart/form-data

Fields:
  audio_file: 录音文件（支持 wav/mp3/m4a/webm 格式）
  format: 音频格式（wav/mp3/m4a/webm）
  sample_rate: 采样率（默认 16000）
```

**成功响应**：
```json
{
  "success": true,
  "data": {
    "text": "感冒了怎么办",
    "confidence": 0.95
  }
}
```

**失败响应**：
```json
{
  "success": false,
  "error": "ASR 服务未启用",
  "error_code": "ASR_DISABLED"
}
```

**错误码**：

| error_code | 说明 |
|------------|------|
| ASR_DISABLED | ASR 服务未启用（管理员关闭了开关） |
| AUDIO_EMPTY | 未检测到有效音频数据 |
| AUDIO_TOO_SHORT | 音频时长过短（< 0.5 秒） |
| AUDIO_TOO_LONG | 音频时长过长（> 15 秒） |
| RECOGNIZE_FAILED | 腾讯云 ASR 识别失败 |
| RECOGNIZE_TIMEOUT | 识别超时 |

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 录音启动延迟 | ≤ 300ms（从长按到开始录音） |
| 语音识别延迟 | ≤ 3s（从松手到文字回填搜索框） |
| 音频上传大小 | 单次录音最大 15 秒，约 240KB（16kHz, 16bit, mono） |
| 自动搜索倒计时精度 | 误差 ≤ 100ms |

### 6.2 安全要求

- ASR SecretKey 加密存储，客户端不接触密钥
- 音频文件仅用于识别，识别完成后不持久化存储（即用即删）
- 录音权限仅在用户主动触发时请求，不在页面加载时预请求
- 防止恶意高频语音识别请求（每用户每分钟不超过 10 次）

### 6.3 兼容性要求

| 端 | 最低要求 |
|----|----------|
| H5 网页端 | iOS Safari 14.5+、Android Chrome 70+、微信内置浏览器（需 HTTPS） |
| 微信小程序 | 微信基础库 1.6.0+（`wx.getRecorderManager` 最低要求） |
| Flutter App | Android 5.0+ / iOS 12.0+（`record` 插件最低要求） |

## 7. 业务规则与约束

| 编号 | 规则 | 说明 |
|------|------|------|
| VR01 | 长按触发阈值 | 长按 300ms 以上才开始录音，短按（< 300ms）不触发 |
| VR02 | 最短录音时长 | 录音时长 < 1 秒时提示"说话时间太短"，不发起识别请求 |
| VR03 | 最长录音时长 | 15 秒自动停止，进入识别流程 |
| VR04 | 自动搜索倒计时 | 识别结果填入搜索框后 2 秒自动搜索 |
| VR05 | 倒计时重置规则 | 用户修改搜索框文字 → 重置为 2 秒重新倒计时 |
| VR06 | 倒计时取消规则 | 用户点击"取消"提示文字 → 取消自动搜索，保留文字 |
| VR07 | 搜索来源标记 | 语音搜索触发的搜索请求，SearchLog.source = 'voice' |
| VR08 | ASR 未启用处理 | 管理后台 ASR 开关关闭时，三端隐藏麦克风按钮 |
| VR09 | 录音格式 | H5 端优先使用 webm/opus 格式；小程序使用 mp3；Flutter 使用 m4a |
| VR10 | 剩余时间警告 | 录音剩余 3 秒时，时长数字变红色提醒 |

## 9. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 浏览器不支持 MediaRecorder（H5） | 隐藏麦克风按钮，不影响文本搜索 |
| 用户拒绝麦克风权限 | Toast 提示"无法访问麦克风" |
| 录音时长 < 1 秒 | 提示"说话时间太短"，不发起识别 |
| 录音到 15 秒自动停止 | 自动进入识别流程，视觉过渡平滑 |
| ASR 服务未启用 | 隐藏麦克风按钮 |
| ASR 识别返回空文字 | 提示"未识别到语音，请重试" + 重试按钮 |
| ASR 识别超时（> 5 秒） | 提示"识别超时，请重试" + 重试按钮 |
| 网络断开 | 提示"网络异常，请检查网络后重试" + 重试按钮 |
| 用户录音过程中切到后台 | 自动停止录音，尝试识别已录音频 |
| 用户录音过程中来电（App） | 自动暂停，通话结束后需重新录音 |
| 2 秒倒计时中用户清空搜索框 | 取消自动搜索，回到正常搜索状态 |
| 识别结果含特殊字符 | 前端转义处理后填入搜索框 |

## 10. 补充说明

### 10.1 与已有统一搜索功能的关系

本次语音搜索升级是对已有统一搜索功能（PRD_统一搜索_20260408）中 **F05 语音搜索** 模块的全面升级和多端扩展：

| 维度 | 原 F05 方案 | 本次升级方案 |
|------|-------------|-------------|
| 覆盖端 | 仅 H5 | H5 + 小程序 + Flutter App |
| 交互方式 | 点击 → 半屏弹窗 → 录音 → 用户确认搜索 | 长按 → 全屏遮罩 → 松手识别 → 2 秒自动搜索 |
| 视觉反馈 | 半屏弹窗 + 波形 | 全屏半透明遮罩 + 中央声波动画 |
| 搜索触发 | 手动点击搜索 | 2 秒自动搜索（可取消/修改） |
| 错误处理 | Toast 提示 | 遮罩内显示错误信息 + 重试按钮 |
| 录音限制 | 超时 10 秒 | 最长 15 秒 |

### 10.2 三端开发优先级

三端（H5、小程序、Flutter App）**同时开发、同步上线**，无优先级差异。

### 10.3 后端复用说明

- ASR 配置管理（管理后台）：沿用已有的搜索管理 → 语音配置页面，无需修改
- ASR Token 接口：沿用已有 `POST /api/search/asr/token`
- 新增 `POST /api/search/asr/recognize` 接口供 H5 和 Flutter App 端直接上传音频识别
- 小程序端使用微信自带语音转文字能力，不经过后端 ASR
