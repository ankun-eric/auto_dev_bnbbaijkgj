# 首页搜索语音输入麦克风图标消失 Bug 修复方案文档

## 1. Bug 发生背景

### 1.1 项目概述

bini-health 是一个健康管理平台，包含 H5 网页端、微信小程序端和 Flutter App 端。首页提供搜索功能，搜索框内集成了语音输入能力，用户可通过点击麦克风图标发起语音识别，识别结果自动填入搜索框进行搜索。

### 1.2 涉及功能模块

- **首页搜索框组件**：包含文字输入、语音输入（麦克风图标）、清除文字（×图标）等交互元素
- **语音识别功能**：点击麦克风后弹出录音浮层（含脉动按钮和声波动画），完成语音采集与识别
- **搜索框图标显示逻辑**：控制麦克风图标和清除图标的显示/隐藏状态

### 1.3 发现时间与发现方式

- **发现方式**：用户使用过程中发现
- **存在时间**：从语音功能上线起一直存在此问题
- **备注**：之前 PRD 文档（PRD_语音搜索体验细节优化）中已定义过"F03：麦克风图标常驻"优化需求，但该功能未被正确实现或存在回归

## 2. Bug 描述

### 2.1 错误现象

首页搜索框的语音输入功能存在以下问题：

1. **搜索框为空时**，麦克风图标（🎤）正常显示在搜索框右侧，用户可以点击发起语音搜索
2. **点击麦克风完成语音识别后**，识别出的文字被正确填入搜索框，但此时搜索框右侧**只剩下清除(×)图标**，麦克风图标消失了
3. 用户如果想再次使用语音搜索，**必须先手动清除搜索框内的文字**，等搜索框恢复为空状态后，麦克风图标才会重新出现
4. 语音录音和识别功能本身工作正常，问题仅在于麦克风图标的显示逻辑

### 2.2 重现步骤

| 步骤 | 操作 | 预期结果 | 实际结果 |
|------|------|----------|----------|
| 1 | 打开首页，观察搜索框 | 搜索框右侧显示麦克风图标 | ✅ 正确，麦克风图标正常显示 |
| 2 | 点击麦克风图标，进行语音录入 | 弹出录音浮层，开始录音 | ✅ 正确，录音功能正常 |
| 3 | 说出搜索内容，等待识别完成 | 识别文字填入搜索框，搜索框右侧同时显示麦克风图标和清除图标 | ❌ 错误，搜索框右侧只显示清除(×)图标，麦克风图标消失 |
| 4 | 尝试再次发起语音搜索 | 可以点击麦克风图标继续语音搜索 | ❌ 错误，找不到麦克风图标，无法发起语音搜索 |
| 5 | 点击清除(×)清空搜索框 | 搜索框清空 | ✅ 正确 |
| 6 | 观察清空后的搜索框 | 麦克风图标重新显示 | ✅ 正确，麦克风图标恢复显示 |

### 2.3 影响范围

- **影响端**：全端影响——H5 网页端、微信小程序端、Flutter App 端均存在此问题
- **影响用户**：所有使用语音搜索功能的用户
- **影响场景**：用户需要连续多次使用语音搜索时体验受阻，必须每次手动清除文字后才能再次发起语音输入
- **功能影响**：语音搜索的连续使用体验严重受损，降低了语音搜索功能的可用性和便捷性

## 3. 预期正确效果

修复后应达到以下效果：

1. **麦克风图标始终常驻**：无论搜索框是否有文字、无论是否刚完成语音识别，麦克风图标（🎤）始终显示在搜索框输入区域的右侧
2. **图标共存布局**：当搜索框有文字时，搜索框右侧应同时显示**麦克风图标（左）和清除图标（右）**，麦克风图标位于输入栏内部的右侧
3. **功能可用性**：用户在任何时候都能直接点击麦克风图标发起新的语音搜索，无需先清除已有文字
4. **全端一致**：H5 网页端、微信小程序端、Flutter App 端三端行为保持一致

## 4. 补充说明

### 4.1 根因分析方向

当前搜索框的图标显示逻辑采用了"互斥"模式：
- 搜索框为空时 → 显示麦克风图标
- 搜索框有文字时 → 显示清除图标（麦克风图标被隐藏）

应修改为"共存"模式：
- 搜索框为空时 → 仅显示麦克风图标
- 搜索框有文字时 → 同时显示麦克风图标和清除图标

### 4.2 修复要点

1. **三端搜索框组件**均需修改图标显示逻辑，确保麦克风图标不受搜索框内容状态的影响
2. 麦克风图标和清除图标共存时的布局：麦克风在左，清除(×)在右（参照用户确认的方案）
3. 麦克风图标位于搜索输入栏内部的右侧位置
4. 修改后需在三端逐一验证，确保语音搜索全流程（点击麦克风 → 录音 → 识别 → 文字填入 → 再次点击麦克风）无回归问题

### 4.3 关联历史需求

- PRD_语音搜索体验细节优化中的 **F03：麦克风图标常驻** 需求与本 Bug 直接相关，本次修复即为该需求的正确落地实现
