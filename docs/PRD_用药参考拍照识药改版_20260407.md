# 用药参考 · 拍照识药改版 产品需求文档（PRD）

## 1. 需求概述

### 1.1 背景与目的

bini-health 健康管理系统当前"用药参考"功能入口中包含"搜索药品"子功能。根据业务调整，需要删除"搜索药品"功能，将"用药参考"的入口行为改为直接进入全新设计的"拍照识药"页面。用户通过拍照或从相册选择药品图片，由 AI 自动识别药品并提供详细的用药建议，支持对话式追问和多药品对比分析。

### 1.2 目标用户

- 普通用户（C端）：需要了解药品用法、用量、注意事项的患者或家属
- 管理员（B端）：需要查看和管理用户拍照识药记录及对话内容的运营人员

### 1.3 核心价值

1. **降低使用门槛**：从文字搜索转向拍照识别，操作更直观便捷
2. **智能化用药建议**：AI 自动识别药品并给出自然语言的专业用药建议
3. **多药品对比分析**：支持在对话中多次拍照，AI 自动进行药物相互作用分析
4. **历史可追溯**：完整的识别历史记录，支持随时回顾和继续咨询

### 1.4 适用端

所有端统一改版：

| 端 | 技术栈 |
|---|---|
| H5 移动端网页 | Next.js + React + Ant Design Mobile |
| 微信小程序 | 原生微信小程序 |
| Flutter App | Flutter |
| 管理后台 | Next.js + React + Ant Design |

### 2.2 功能详细描述

#### F01 / F02：入口调整

**操作**：在所有用户端（H5、微信小程序、Flutter App）首页的"功能入口"区域：
- 保留"用药参考"菜单入口（图标、文字不变）
- 删除"用药参考"下的"搜索药品"子功能
- 点击"用药参考"后直接跳转到全新设计的"拍照识药"页面
- 不做旧路径兼容跳转，直接删除搜索药品入口

#### F03：拍照/相册选择区域

**位置**：拍照识药主页面上方

**包含元素**：
1. **拍照按钮**：点击调起设备摄像头，拍照后自动上传并开始 AI 识别
2. **从相册选择按钮**：点击打开系统相册，选择图片后自动上传并开始 AI 识别
3. **上传进度指示**：图片上传和识别过程中显示 loading 状态

**交互说明**：
- 拍照/选图后，图片通过 `POST /api/ocr/recognize` 接口上传识别
- 识别成功后自动跳转到 AI 对话界面
- 识别失败时显示错误提示（见 F11）

#### F04：历史识别记录列表

**位置**：拍照识药主页面下方（紧接拍照区域）

**列表项内容**：
| 元素 | 说明 |
|------|------|
| 药品图片缩略图 | 用户上传的原始图片缩略图（第一张） |
| 药品名称 | AI 识别出的药品名称（多个药品时显示第一个） |
| 识别时间 | 格式：YYYY-MM-DD HH:mm |
| 状态标签 | "已识别"（绿色标签）/ "识别失败"（红色标签） |

**交互说明**：
- 列表按时间倒序排列（最新在前）
- 支持上下滚动浏览
- 点击某条记录进入该次识别的 AI 对话界面（F12）
- 一次对话 = 一条历史记录，一条记录可能包含多个药品

#### F05：空状态提示

当用户首次使用或无历史记录时：
- 拍照区域正常展示（相机图标 + "拍照识药" / "从相册选择"按钮）
- 下方历史记录区域显示空状态提示文案："暂无识别记录，拍张药品照片试试吧"
- 空状态可配合简洁的插图/图标增强视觉引导

#### F06 / F07：药品识别结果与 AI 用药建议

**触发**：用户拍照/选图后，OCR 识别成功

**展示方式**：识别完成后自动进入 AI 对话界面

**AI 首条消息包含**：
1. 药品名称
2. 详细药品信息（用法用量、注意事项、禁忌、不良反应等）
3. AI 用药建议（自然语言回复，类似医生建议口吻）

**示例**：
> 我已识别出您拍摄的药品为「**阿莫西林胶囊**」，以下是详细的用药参考：
>
> **药品名称**：阿莫西林胶囊
> **适应症**：用于敏感菌所致的呼吸道感染、泌尿生殖道感染等
> **用法用量**：成人一次 0.5g，每 6-8 小时 1 次
> **注意事项**：对青霉素过敏者禁用，服药期间避免饮酒
> **不良反应**：可能出现皮疹、腹泻等
>
> **用药建议**：建议按医嘱服用，饭后30分钟服用可减少胃肠道不适。如出现严重过敏反应请立即就医。如有任何疑问，请继续提问。

#### F08：文字追问

- 用户可在 AI 对话界面底部输入框发送文字消息
- 追问范围：仅限本次识别结果相关（如"这个药能和阿司匹林一起吃吗？"）
- 追问次数：不限制，用户可无限追问
- AI 回复时保持本次对话的药品上下文

#### F09 / F10：对话中再次拍照与多药品对比分析

- 在 AI 对话界面中提供"拍照"/"上传图片"按钮
- 用户可在对话中再次拍照或上传新药品图片
- AI 自动识别新药品后：
  - 在对话中回复新药品的详细信息和用药建议
  - **自动结合之前已识别的药品进行对比分析**（如"阿莫西林与布洛芬可以同时服用，但请间隔30分钟"）
  - 分析维度包括：药物相互作用、是否可同时服用、服用间隔建议等

#### F11：识别失败处理

当 OCR 识别或 AI 分析失败时：
- 显示错误提示文案："未能识别药品，建议在光线充足的环境下拍摄药品正面标签"
- 提供"重新拍照"按钮，点击后重新调起摄像头
- 识别失败的记录也保存到历史记录中，状态标签显示"识别失败"（红色）

#### F12：历史记录恢复

- 点击历史记录列表中的某条记录
- 进入该次识别的 AI 对话界面
- 恢复之前的完整对话上下文（所有消息、图片、AI 回复）
- 用户可继续追问或再次上传新药品图片

#### F13：管理后台 - 查看对话记录

- 在现有"拍照识药记录明细"页面的表格中，操作列增加"查看对话"按钮
- 点击后弹出 Ant Design Drawer 组件
- Drawer 中展示该条记录对应的完整对话记录：
  - 用户消息（文字 + 图片）
  - AI 回复消息
  - 消息时间戳
  - 对话气泡样式（类似聊天界面）

## 3. 页面/界面设计

### 3.1 页面结构与导航

```
首页
 └── 功能入口 → 用药参考（入口）
       └── 拍照识药主页面
             ├── 拍照/相册选择区域（上方）
             └── 历史识别记录列表（下方，可滚动）
                   └── 点击某条记录 → AI 对话页面
                         ├── 对话消息流（AI + 用户）
                         ├── 文字输入框
                         └── 拍照/上传图片按钮
```

### 3.2 各页面功能说明

#### 3.2.1 拍照识药主页面

**UI 风格**：沿用 bini-health 现有风格

**页面布局（自上而下）**：

1. **顶部导航栏**
   - 标题："拍照识药"
   - 返回按钮（返回首页）

2. **拍照操作区域**（页面上半部分）
   - 大尺寸相机图标居中
   - 两个操作按钮并排：
     - 「拍照识药」按钮（主按钮，主色调填充）
     - 「从相册选择」按钮（次按钮，边框样式）
   - 简短说明文字："拍摄药品包装，AI 帮您解读用药信息"

3. **历史记录区域**（页面下半部分）
   - 区域标题："识别记录"
   - 记录列表（Card 样式）：
     - 左侧：药品图片缩略图（圆角矩形，固定尺寸）
     - 中间：药品名称（粗体）+ 识别时间（灰色小字）
     - 右侧：状态标签（已识别/识别失败）
   - 空状态：居中显示空状态图标 + "暂无识别记录，拍张药品照片试试吧"

#### 3.2.2 AI 对话页面

**页面布局**：

1. **顶部导航栏**
   - 标题："用药咨询"
   - 返回按钮（返回拍照识药主页面）

2. **对话消息区域**（中间，可滚动）
   - AI 消息气泡（左侧，浅色背景）
   - 用户消息气泡（右侧，主色调背景）
   - 图片消息：显示用户上传的药品图片缩略图，可点击放大
   - 消息时间戳（每条消息下方，灰色小字）
   - AI 正在回复时显示 loading 动画

3. **底部输入区域**
   - 文字输入框
   - 发送按钮
   - 拍照按钮（调起摄像头）
   - 相册按钮（打开相册选择）

#### 3.2.3 管理后台 - 对话记录 Drawer

**触发方式**：点击"拍照识药记录明细"表格操作列的"查看对话"按钮

**Drawer 内容**：
- 宽度：600px
- 标题："对话记录 - {药品名称}"
- 对话气泡列表：
  - 用户消息（右侧气泡）：文字 + 图片
  - AI 消息（左侧气泡）：药品信息 + 用药建议
  - 每条消息显示发送时间
- 底部无输入框（管理员仅查看，不参与对话）

## 4. 数据模型设计

### 4.1 数据实体与字段

本次改版以前端重构为主，复用现有后端数据模型。涉及的核心数据实体如下：

#### 4.1.1 drug_identify_details（已有，药品识别明细表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, 自增 | 记录ID |
| user_id | Integer | FK → users.id | 用户ID |
| image_url | String | NOT NULL | 上传的药品图片URL |
| drug_name | String | NULL | AI 识别出的药品名称 |
| drug_category | String | NULL | 药品分类 |
| ai_result | Text | NULL | AI 识别分析结果（JSON） |
| status | String | NOT NULL | 识别状态：success/failed |
| ocr_text | Text | NULL | OCR 识别出的原始文字 |
| created_at | DateTime | NOT NULL | 创建时间 |
| session_id | Integer | FK → chat_sessions.id, NULL | **新增**：关联的对话会话ID |

#### 4.1.2 chat_sessions（已有，对话会话表）

复用现有 `ChatSession` 模型，通过 `chat_type = 'drug_query'` 关联。

#### 4.1.3 chat_messages（已有，对话消息表）

复用现有 `ChatMessage` 模型，消息中包含图片的通过 `image_urls` 字段存储。

### 4.2 实体关系

```
users (1) ──────── (N) drug_identify_details
users (1) ──────── (N) chat_sessions (chat_type = 'drug_query')
chat_sessions (1) ─ (N) chat_messages
drug_identify_details (N) ── (1) chat_sessions (通过 session_id 关联)
```

## 5. 接口需求

### 5.1 接口清单

本次改版主要复用现有接口，部分接口需要微调。

| 编号 | 方法 | 路径 | 说明 | 请求参数 | 返回数据 |
|------|------|------|------|----------|----------|
| API-01 | POST | `/api/ocr/recognize` | 药品图片识别（已有） | `file`: 图片文件, `scene`: "drug_identify" | `{ drug_name, drug_category, ai_result, image_url, status }` |
| API-02 | GET | `/api/drug-identify/history` | 获取用户识别历史 | `page`, `page_size` | `{ items: [{ id, image_url, drug_name, status, created_at, session_id }], total }` |
| API-03 | GET | `/api/chat-sessions` | 获取对话列表（已有） | `chat_type=drug_query` | `{ items: [ChatSession], total }` |
| API-04 | GET | `/api/chat-sessions/{id}` | 获取对话详情（已有） | - | `{ session, messages: [ChatMessage] }` |
| API-05 | POST | `/api/chat/send` | 发送对话消息（已有） | `session_id`, `content`, `image_urls` | `{ message, ai_response }` |
| API-06 | GET | `/api/admin/drug-details` | 管理员-识别明细列表（已有） | `page`, `page_size`, 过滤条件 | `{ items, total }` |
| API-07 | GET | `/api/admin/drug-details/{id}` | 管理员-识别明细详情（已有） | - | `{ detail }` |
| API-08 | GET | `/api/admin/drug-details/{id}/conversation` | 管理员-查看对话记录 | - | `{ messages: [{ role, content, image_urls, created_at }] }` |

**新增/调整接口说明**：

- **API-02**（新增）：为拍照识药主页面提供用户维度的识别历史列表，返回简要信息用于列表展示
- **API-08**（新增）：管理后台通过识别记录ID查询关联的对话记录，用于 Drawer 展示

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 图片上传 | 支持 JPEG/PNG 格式，单张不超过 10MB |
| OCR 识别响应 | ≤ 5 秒（含网络传输） |
| AI 用药建议生成 | ≤ 10 秒（含 AI 模型推理） |
| 历史记录加载 | 首屏 ≤ 1 秒，分页加载 ≤ 500ms |
| 对话消息发送 | ≤ 3 秒内收到 AI 回复 |

### 6.2 安全要求

- 用户上传的药品图片存储在 COS（已有），遵循现有存储安全策略
- AI 用药建议经过敏感词过滤（复用现有 `_filter_sensitive_words` 机制）
- AI 回复末尾附加免责提示（复用现有 `_append_disclaimer` 机制）
- 管理后台对话查看需要管理员权限验证

### 6.3 兼容性要求

| 端 | 兼容范围 |
|---|---|
| H5 | iOS Safari 12+, Android Chrome 80+, 微信内置浏览器 |
| 微信小程序 | 基础库 2.20.0+ |
| Flutter App | iOS 12.0+, Android 5.0+ |
| 管理后台 | Chrome 90+, Edge 90+, Firefox 90+ |

## 7. 业务规则与约束

| 编号 | 规则 | 说明 |
|------|------|------|
| BR-01 | 一次对话 = 一条历史记录 | 用户从主页发起拍照后创建一个对话会话，后续在该对话中的所有操作都归属同一条记录 |
| BR-02 | 追问范围限制 | 用户追问仅限本次识别的药品相关内容，AI 通过 system prompt 约束回答范围 |
| BR-03 | 多药品自动对比 | 同一对话中再次拍照新药品时，AI 自动关联之前的药品进行相互作用分析 |
| BR-04 | 追问无次数限制 | 用户可在同一对话中无限追问 |
| BR-05 | 识别失败记录保留 | 识别失败的记录也保存到历史记录中，状态标签为"识别失败" |
| BR-06 | 历史记录排序 | 按识别时间倒序排列，最新在前 |
| BR-07 | 旧路径不兼容 | 直接删除"搜索药品"入口，不做旧版本兼容跳转 |

## 9. 异常处理与边界情况

| 编号 | 场景 | 处理方式 |
|------|------|----------|
| E-01 | 图片上传失败（网络异常） | Toast 提示"图片上传失败，请检查网络后重试" |
| E-02 | OCR 识别失败 | 显示"未能识别药品，建议在光线充足的环境下拍摄药品正面标签"+ "重新拍照"按钮 |

## 10. 补充说明

### 10.1 后端复用策略

本次改版**以前端重构为主**，后端能力尽量复用现有系统：

| 能力 | 复用方式 |
|------|----------|
| 药品图片 OCR 识别 | 复用 `POST /api/ocr/recognize`（scene = drug_identify） |
| AI 对话 | 复用现有 ChatSession / ChatMessage 体系（chat_type = drug_query） |
| 文件上传 | 复用现有 COS 上传能力 |
| 敏感词过滤 | 复用 `_filter_sensitive_words()` |
| 免责提示 | 复用 `_append_disclaimer()`（drug_query 类型） |
| AI 模型调用 | 复用 `call_ai_model()` |
| 提示词配置 | 复用 AI 配置中心的 `ocr_drug_identify` 提示词 |

### 10.2 新增后端工作量估算

仅需新增 2 个接口：
1. `GET /api/drug-identify/history` — 用户端识别历史列表
2. `GET /api/admin/drug-details/{id}/conversation` — 管理端查看对话记录

以及 `drug_identify_details` 表新增 `session_id` 外键字段，关联 `chat_sessions` 表。

### 10.3 UI 风格参考

沿用 bini-health 现有设计风格：
- 主色调：系统现有主题色
- 卡片样式：圆角卡片，浅色阴影
- 按钮样式：圆角按钮，主色调填充 / 边框样式
- 文字层级：标题 16-18px 粗体，正文 14px，辅助文字 12px 灰色
- 图标风格：与现有图标一致

### 10.4 对话中 AI 行为规范

1. 首次识别成功后，AI 主动输出药品完整信息和用药建议
2. 用户追问时，AI 在当前药品上下文内回答
3. 用户在对话中上传新药品图片时，AI 自动：
   - 识别新药品并输出其信息
   - 与之前的药品进行相互作用分析
   - 给出联合用药建议
4. 所有 AI 回复末尾附加免责提示（由系统自动添加）
