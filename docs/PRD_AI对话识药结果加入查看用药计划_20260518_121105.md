# AI 对话识药结果 - 加入 / 查看用药计划 产品需求文档（PRD）

> 模块定位：在 AI 对话页面识别出药品后，咨询师可以直接为当前选中的"咨询人"快捷加入用药计划，或查看该咨询人已有的用药计划，所有交互通过抽屉完成，不再跳出新页面。

---

## 1. 需求概述

### 1.1 背景与目的

当前 bini-health 产品在 AI 对话页面具备"识药"能力（用户上传药品图片或输入药名后，AI 会识别并返回药品卡片）。但识药结果与"用药计划"模块之间缺少打通：

- 咨询师识别出药品后，无法在对话页直接为咨询人加入用药计划，必须离开对话页跳转到"用药计划"模块手动新增，操作链路长，咨询体验断裂。
- 咨询师也无法在对话页快捷查看咨询人当前已有的用药情况，不便结合既有用药给出咨询建议。

本需求的目的：

- 在 AI 对话页"识药卡片"上新增"加入用药计划"与"查看用药计划"两个入口；
- 所有交互在**抽屉**内完成（不打开新页面，不跳路由），保持对话流的连续性；
- 让识药结果与用药计划模块形成闭环。

### 1.2 目标用户

- **咨询师 / 健康管理师**（主要使用者）：在与咨询人进行 AI 对话过程中，识药后快速为其录入或查看用药计划。
- **本人态用户**（次要使用者）：在 AI 对话中识药后，可为自己加入用药计划。

### 1.3 核心价值

- **零跳转**：识药结果 → 加入/查看用药计划，全流程在抽屉内完成，不离开对话页。
- **零冗余**：药品名称从识药结果自动带入，减少咨询师手动输入。
- **状态可视**：识药卡片上的按钮能实时反映"是否已加入用药计划"，避免重复录入。

### 2.2 功能详细描述

#### F-01 / F-02 / F-03 识药卡片按钮区

**位置**：AI 对话页面，每一张"识药结果卡片"（含药品名称、图片、说明等）的底部操作区。

**两个按钮并列**：

| 按钮 | 文案 | 默认状态 |
|------|------|----------|
| 主按钮 | 「加入用药计划」 / 「已加入用药计划」 | 根据是否已加入动态切换 |
| 次按钮 | 「查看用药计划」 | 始终可点击 |

**视觉差异化（F-02）**：

- 「加入用药计划」：主色实心按钮（高强调），表示尚未加入，鼓励点击。
- 「已加入用药计划」：弱色 / 描边按钮 + 对勾图标（如灰色背景 + 主色描边 + ✓），表示已加入。
- 「已加入用药计划」**仍可点击**：点击后弹出确认对话框：
  - 标题：「该药品已加入用药计划」
  - 内容：「是否再次加入或修改用药计划？」
  - 按钮：「再次加入」「修改用药计划」「取消」
  - 点击「再次加入」：打开"加入用药计划"抽屉（药品名称仍自动带入，允许重复加入）。
  - 点击「修改用药计划」：打开"查看用药计划"抽屉，并默认展开匹配到的那条记录的明细 + 进入编辑态。
  - 点击「取消」：关闭确认对话框。

**咨询人空态保护（F-03）**：

- 若 AI 对话页顶部当前**没有选中咨询人**（咨询人为空 / 游客模式），「加入用药计划」按钮：
  - 显示为**置灰不可点击**状态。
  - 点击区域允许触发 Toast 提示：「请先选择咨询人」。
- 「查看用药计划」按钮在咨询人为空时同样置灰，Toast 提示同上（保持一致体验）。

#### F-04 / F-05 / F-06 / F-07 加入用药计划抽屉

**触发方式**：

- 点击识药卡片上的「加入用药计划」按钮（未加入状态）。
- 或在「已加入用药计划」的二次确认中选择「再次加入」。
- 或在"查看用药计划"抽屉空态点击「去新增」（关闭原抽屉后打开此抽屉）。

**抽屉标题文案规则**：

- 本人态（`isSelf=true`）：直接显示 **「加入用药计划」**（不带姓名前缀）。
- 非本人态：显示 **「为 {nickname} 加入用药计划」**，`nickname` 取咨询对象档案接口字段。

**抽屉内容（F-05 字段清单）**：

复用现有用药计划新增页 `/ai-home/medication-plans/new` 的**全部字段**：

| 字段 | 说明 | 在抽屉中的处理 |
|------|------|----------------|
| 药品名称 | 必填 | **自动填入识药结果中的药品名称**，仍可由用户修改 |
| 用量（数值 + 单位） | 必填 | 用户手填，单位下拉与现页面一致 |
| 用药频次 / 每日次数 | 必填 | 用户手填 |
| 用药时间点 | 选填 | 与现页面一致（早 / 中 / 晚 / 睡前等多选） |
| 用药周期（开始日期 ~ 结束日期 / 长期） | 必填 | 与现页面一致 |
| 用药指导 / 服用时机（饭前饭后等） | 选填 | 与现页面一致 |

> 字段类型、校验规则、单位选项均完全沿用现有"新增用药计划"页面的实现，不做任何缩水或扩展。

**底部操作（F-06）**：

- 「保存」主按钮、「取消」次按钮。
- 点击「保存」：
  - 抽屉内直接调用新增用药计划接口；
  - 保存成功 → 关闭抽屉 → 回到识药卡片 → 立即把按钮文案切换为「已加入用药计划」（同步触发 F-14 状态刷新）。
  - 期间按钮显示 Loading 防重复提交。

**保存失败处理（F-07）**：

- 抽屉**不关闭**。
- 抽屉顶部出现**红色 Toast**（或顶部条形错误提示），文案为后端返回的错误原因（无明确错误时使用兜底文案「保存失败，请重试」）。
- 用户可修改内容后再次点击「保存」重试。

#### F-08 ~ F-12 查看用药计划抽屉

**触发方式**：点击识药卡片上的「查看用药计划」按钮。

**抽屉标题文案**：「{nickname} 的用药计划」（本人态显示「我的用药计划」）。

**抽屉内容（F-08）**：

- 列表呈现**当前咨询人名下所有用药计划**，**完全沿用**目前用药计划页面的展示样式与字段（不做缩水、不做扩展）：
  - 药品名称、用量、用药周期、状态等字段按现页面规则展示。
- 排序规则：沿用现用药计划页面的排序规则。
- 条数上限：**全部展示**，超过抽屉高度时**抽屉内部滚动**。

**列表项点击展开明细（F-09）**：

- 点击列表项的主体区域 → **在当前抽屉内就地折叠/展开**该条记录的全部明细字段（不跳详情页，不打开新抽屉）。
- 展开后的明细内容包含该用药计划的所有字段（用法用量、频次、时间点、周期、服用时机、备注等）。
- 同一时刻只允许展开一条（点击另一条时自动收起前一条）；也可点击已展开项再次折叠。

**列表项「编辑」「删除」操作（F-10 / F-11）**：

每条记录的右侧操作区提供两个按钮：

- **「编辑」**：点击后该列表项**切换为表单态**（在抽屉内就地变为带输入控件的表单），字段与"加入用药计划"抽屉一致，底部出现「保存」「取消」。
  - 「保存」成功后切回展示态，并刷新本条数据；失败则保留表单态并 Toast 提示。
  - 「取消」直接切回展示态，丢弃修改。
- **「删除」**：点击后弹出**二次确认 Toast / 对话框**：
  - 文案：「确定删除该用药计划？删除后不可恢复。」
  - 按钮：「确定」「取消」
  - 「确定」：调用删除接口，成功后从列表中移除该条，并 Toast 提示「已删除」；失败 Toast 提示错误原因。

**空态处理（F-12）**：

- 当该咨询人名下**一条用药计划都没有**时，抽屉内容区展示：
  - 空状态插画（沿用项目现有空态插画风格）；
  - 文案：「该咨询人暂无用药计划」；
  - 一个主按钮：**「去新增」**。
- 点击「去新增」：**关闭当前"查看用药计划"抽屉，再打开"加入用药计划"抽屉**（两个独立抽屉切换，不打开新页面、不跳路由）。

#### F-13 / F-14 是否已加入的判定与状态刷新

**匹配范围（F-13）**：

- 范围：**当前 AI 对话页面选中的咨询人**名下、**有效期内**（即"未过期 / 未停用"）的用药计划。
- 字段：宽松匹配——**药品名 / 通用名任一字段包含识药关键字**即视为"已加入"。
- 当存在多条匹配时，识药卡片按钮状态显示为「已加入用药计划」即可（二次确认对话框中的「修改用药计划」操作默认展开**最近一条**）。

**状态刷新时机（F-14）**：

- **A. 列表 onShow 批量刷新**：AI 对话页 / 识药结果列表显示时，批量查询当前咨询人名下当前页所有识药结果的"是否已加入"状态。
- **B. 卡片单独刷新**：每张识药卡片被展开 / 第一次出现在视口时，再针对该药品单独刷新一次"是否已加入"状态。
- **C. 保存成功后刷新**：「加入用药计划」抽屉保存成功后，立即触发对应识药卡片的状态刷新，确保按钮文案立刻变成「已加入用药计划」。
- 切换咨询人后，所有识药卡片状态需重新批量刷新。

## 3. 页面 / 界面设计

### 3.1 页面结构与导航

本需求**不新增独立路由 / 页面**，所有交互在 AI 对话页内通过抽屉完成。

```
AI 对话页面
└── 识药卡片（每张卡片底部）
    ├── 按钮：加入用药计划 / 已加入用药计划
    │   ├── 点击「加入用药计划」 → 打开「加入用药计划」抽屉
    │   └── 点击「已加入用药计划」 → 弹出确认对话框（再次加入 / 修改用药计划 / 取消）
    │
    └── 按钮：查看用药计划 → 打开「查看用药计划」抽屉
        ├── 列表项点击 → 抽屉内就地展开明细
        ├── 列表项「编辑」 → 抽屉内切换为表单态
        ├── 列表项「删除」 → 二次确认 → 抽屉内移除
        └── 空态「去新增」 → 关闭本抽屉 → 打开「加入用药计划」抽屉
```

### 3.2 各页面 / 抽屉详细说明

#### 3.2.1 AI 对话页 - 识药卡片改造

- 卡片底部操作区从原来的"无操作按钮 / 单按钮"改造为**两个按钮并列**。
- 主按钮（加入用药计划）按状态切换样式，已加入时附带 ✓ 图标。
- 咨询人为空时，两个按钮均置灰，点击有 Toast 提示。

#### 3.2.2 「加入用药计划」抽屉

- 从屏幕**底部弹出**（移动端）/ **右侧弹出**（桌面端宽屏）。
- 高度：占屏幕约 75%~85%，顶部留出关闭区域。
- 顶部：标题 + 关闭按钮（×）。
- 内容区：表单字段（与现新增页一致，药品名称已自动填入）。
- 底部：「取消」「保存」两按钮，主按钮高强调。
- 保存失败时顶部出现红色 Toast 错误提示。

#### 3.2.3 「查看用药计划」抽屉

- 弹出方式同上。
- 顶部：标题「{nickname} 的用药计划」 + 关闭按钮（×）。
- 内容区：用药计划列表（沿用现页面样式）。
  - 列表项默认折叠态：显示药品名称等关键字段 + 右侧「编辑」「删除」按钮。
  - 点击列表项主体 → 就地展开明细。
  - 点击「编辑」 → 该项就地变为表单态。
  - 点击「删除」 → 弹二次确认。
- 列表过长时抽屉内部滚动。
- 空态：插画 + 文案 +「去新增」主按钮，点击关闭本抽屉并打开「加入用药计划」抽屉。

## 4. 非功能性需求

### 4.1 性能要求

- 抽屉打开动画流畅，整体响应时间 ≤ 300ms。
- 「是否已加入」批量查询接口需支持一次请求多个药品名匹配，避免 N+1 查询。
- 状态刷新使用本地缓存 + 增量更新，避免每次都全量请求。

### 4.2 安全要求

- 抽屉内的增 / 改 / 删操作均需校验当前登录用户对该咨询人是否有操作权限（沿用现有用药计划模块的权限控制）。
- 本人态用户只能操作自己的用药计划。

### 4.3 兼容性要求

- 支持现有 Web 端、H5、小程序等需要承载 AI 对话页的所有端。
- 抽屉组件需在移动端（小屏）与桌面端（大屏）均有良好表现：移动端底部弹出全宽，桌面端右侧弹出固定宽度。

---

## 5. 业务规则与约束

1. 「加入用药计划」「查看用药计划」的所有操作均以**当前 AI 对话页选中的咨询人**为操作对象。
2. 「是否已加入」的判定范围限定为**该咨询人名下、有效期内**的用药计划，过期 / 已停的不计入。
3. 药品名匹配规则：**药品名 / 通用名任一字段包含识药关键字**即匹配（宽松匹配）。
4. 抽屉内的"加入用药计划"成功后，必须立即同步触发当前识药卡片状态刷新，不依赖用户手动刷新页面。
5. 删除操作必须二次确认，不允许"一键删除"。
6. 本人态下，"加入用药计划"抽屉标题不带姓名前缀；非本人态使用 `nickname` 字段。
7. 一切流程**不打开新页面、不跳路由**，所有交互在 AI 对话页 + 抽屉内完成。

## 6. 权限设计

| 角色 | 权限说明 |
|------|----------|
| 咨询师 / 健康管理师 | 对当前选中咨询人的用药计划可加入、查看、编辑、删除（沿用现有用药计划模块的权限规则） |
| 本人态用户 | 对自己名下的用药计划可加入、查看、编辑、删除 |
| 未登录 / 未选中咨询人 | 「加入用药计划」「查看用药计划」按钮置灰不可点击，Toast 提示「请先选择咨询人」 |

## 7. 异常处理与边界情况

| 场景 | 处理方式 |
|------|----------|
| 未选中咨询人时点击按钮 | 按钮置灰；点击 Toast 提示「请先选择咨询人」 |
| 加入用药计划保存失败 | 抽屉不关闭，顶部红色 Toast 提示，可重试 |
| 编辑保存失败 | 列表项保持表单态，Toast 错误提示 |
| 删除失败 | Toast 提示错误原因，列表项保留 |
| 查询用药计划列表失败 | 抽屉内显示重试按钮 + 错误文案 |
| 该咨询人无任何用药计划 | 抽屉显示空态 +「去新增」按钮 |
| 识药结果中无药品名（空文案） | 「加入用药计划」按钮置灰，Toast 提示「未识别到有效药品名」 |
| 同一药品已存在多条用药计划 | 按钮显示「已加入用药计划」；二次确认对话框中「修改用药计划」默认展开最近一条 |
| 切换咨询人 | 所有识药卡片按钮状态需重新批量刷新 |
| 网络异常 | 状态刷新接口失败时按钮默认显示「加入用药计划」（未加入态），避免误导 |
| 抽屉内列表数据较多 | 抽屉内部滚动，不再分页跳转 |
| 用户在抽屉表单填写过程中关闭抽屉 | 直接丢弃未保存内容（不提供草稿保留，保持简单） |

## 8. 补充说明

1. 本需求**完全复用**现有「用药计划」模块的数据结构、接口与表单字段，不新增业务字段，不改变数据库结构（如需新增"是否已加入批量查询"接口，由开发侧决定）。
2. 本期"查看用药计划"抽屉**不展示**"药物标准用药信息"（即药品通用知识、适应症、禁忌等），保留后续迭代的空间，但本期 UI 不留占位。
3. 抽屉组件应作为可复用的通用组件实现，便于其他识药 / 知识库等场景复用。
4. 本需求作为**一期工程一次性全量开发完毕**，所有功能（F-01 ~ F-14）统一交付，不分阶段、不分批。
