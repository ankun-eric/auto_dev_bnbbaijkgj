## 二、知识库管理

**入口**：左侧菜单 **AI管理 → 知识库管理**  
**直达**：[知识库管理](https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/admin/knowledge)

### 2.1 知识库列表与搜索

1. 页面展示知识库名称、描述、状态、是否通用、条目数量、创建时间等。
2. 在顶部搜索框输入**知识库名称**关键词，点击 **「搜索」**；点击 **「重置」** 清空条件并刷新列表。
3. 列表支持分页与每页条数调整。

### 2.2 创建知识库

1. 点击 **「创建知识库」**。
2. 填写 **知识库名称**（必填）、**描述**（选填）。
3. 打开 **「设为通用知识库」** 开关，可将该库作为**全局/通用**知识库参与检索（界面显示「通用」标签）。
4. 点击 **「确定」** 保存。

### 2.3 编辑知识库

1. 在对应行点击 **「编辑」**。
2. 修改名称、描述、是否通用后保存。

### 2.4 启用 / 停用知识库

1. 点击行内 **「启用」** 或 **「禁用」**，切换知识库状态（启用后条目才可参与检索，具体还受条目自身状态影响）。

### 2.5 删除知识库

1. 点击 **「删除」**，在确认框中确认。
2. 删除为不可逆操作，请确认无业务依赖后再执行。

### 2.6 条目管理（QA 与文档段落）

1. 在列表中点击 **「条目管理」**，进入该知识库的条目列表。  
   **URL 形式**：`.../admin/knowledge/{知识库ID}`  
   例如：`https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/admin/knowledge/1`（将 `1` 换成实际 ID）。

2. **筛选与排序**：可按关键词、条目类型（Q&A / 文档）、状态（启用/禁用）、排序（如按命中次数）查询；支持 **「搜索」** 与 **「重置」**。

3. **新增条目**  
   - 点击 **「新增条目」**。  
   - **条目类型**：**Q&A**（问题 + 答案）或 **文档段落**（标题 + 正文）。  
   - **关键词/标签**：在输入框中输入标签，按回车添加多个关键词。  
   - **展示方式**：**直接展示** 或 **AI改写后展示**（由服务端与对话逻辑决定最终呈现）。  
   - 保存后条目默认为启用状态（可在列表中再切换）。

4. **编辑 / 启用 / 禁用 / 删除条目**：与知识库操作类似，在表格行内点击对应按钮。

5. **富文本与多媒体（说明）**  
   - 后台表单以文本为主，便于快速录入。  
   - 知识条目的 `content_json` 在系统中可承载结构化内容；H5/小程序/App 端对话中的**知识卡片**支持解析并展示 **文本、图片、视频、商品卡片** 等（例如 HTML、`blocks` 结构或接口返回的商品列表）。  
   - 若需大批量富文本或结构化数据，建议使用 **批量导入** 或对接 API，保证 JSON 与前台约定一致。

### 2.7 批量导入

1. 在条目管理页点击 **「批量导入」**。
2. 选择 **「上传文件」** 或 **「输入URL」**。  
   - **上传文件**：支持将 **JSON**（数组，或含 `entries` 字段的对象）等文件拖入；界面提示亦包含 xlsx、csv、md 等扩展名，请以实际解析成功为准（JSON 需合法可解析）。  
   - **输入URL**：填写可访问的文件地址后提交（将创建 URL 导入任务，具体以后端处理为准）。
3. 导入成功后列表自动刷新；失败时根据页面提示检查文件格式或网络。

### 2.8 数据统计入口

在 [知识库管理](https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/admin/knowledge) 页面右上角点击 **「数据统计」**，进入统计页（见第六章）。

## 三、检索策略配置

**入口**：左侧菜单 **AI管理 → 检索策略配置**  
**直达**：[检索策略配置](https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/admin/search-config)

### 3.1 全局默认策略

1. 在 **「全局默认策略」** 卡片中配置：
   - **精确匹配 / 语义匹配 / 关键词匹配**：分别用开关启用或禁用。
   - **三种匹配的优先级**：数字越小通常表示优先级越高（范围 1–10，与业务实现一致）。
   - **语义匹配相似度阈值**：**宽松 / 标准 / 严格**。
   - **最大命中展示数量**：对话中最多展示几条知识命中（1–5）。
2. 点击 **「保存全局策略」**。

### 3.2 知识库级别策略覆盖

1. 在 **「知识库级别策略」** 表格中查看各知识库当前是 **全局默认** 还是 **自定义**。
2. 点击某行的 **「编辑策略」**，在弹窗中填写与全局相同维度的选项，保存后该库使用独立策略。

### 3.3 场景与知识库绑定

1. 在 **「场景-知识库绑定」** 表格中，为每个对话场景（如健康问答、症状分析、中医辨证、药品查询、客服）选择要检索的一个或多个知识库（多选）。
2. 列表中的**第一个**知识库在保存时会作为**主库**（`is_primary`）标记，用于与策略配合时的主次关系。
3. 点击 **「保存场景绑定」**。

## 四、兜底策略配置

**入口**：左侧菜单 **AI管理 → 兜底策略配置**  
**直达**：[兜底策略配置](https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/admin/fallback-config)

当知识检索**没有命中**或业务进入兜底分支时，按**对话场景**执行以下策略之一。

### 4.1 策略类型（与界面对应）

| 界面选项 | 含义 |
|----------|------|
| **回退AI生成** | 由 AI 继续生成回答（无需额外表单配置）。 |
| **固定文案** | 展示管理员配置的固定回复文案（适用于统一提示、免责声明等）。 |
| **转人工客服** | 走人工客服流程（无需在表单中填写额外项，具体接入依赖客服系统）。 |
| **推荐相关内容** | 按配置条数推荐知识库中的相关内容（需填写 **推荐条目数**，一般 1–10）。 |

选择 **固定文案** 时，在 **「固定文案」** 文本框中填写兜底回复内容。  
配置完成后点击页面底部 **「保存兜底策略」**（将按场景逐条提交保存）。

> **说明**：若产品文档中提及「静默处理」等扩展策略，请以实际上线菜单为准；当前后台提供上述四种单选项策略。

## 五、存储配置（腾讯云 COS）

**入口**：左侧菜单 **系统管理 → 存储配置**  
**直达**：[存储配置](https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/admin/cos-config)

页面分为三个子页：**存储配置**、**文件管理**、**用量统计**。

### 5.1 COS 连接与路径前缀

1. 打开 **「存储配置」** 子页，确认顶部 **启用** 开关为开启状态（启用后使用 COS 存储）。
2. 填写 **SecretId**、**SecretKey**（若已配置过 SecretKey，可只填 SecretId 与其余项；需更新密钥时再输入新 SecretKey）。
3. 填写 **Bucket**、选择 **Region**（如北京、上海、广州等）。
4. 配置 **图片 / 视频 / 文件** 路径前缀（默认常见为 `images/`、`videos/`、`files/`）。
5. 点击 **「保存配置」**。
6. 点击 **「测试连接」**，根据页面下方绿色/红色提示框判断连通性是否成功。

### 5.2 文件管理

1. 切换到 **「文件管理」** 子页。
2. 可按 **模块**（图片/视频/文件）筛选，点击 **「查询」**。
3. 列表支持分页；对图片可 **预览**，对文件可 **删除**（删除前会二次确认）。

### 5.3 用量统计

1. 切换到 **「用量统计」** 子页。
2. 查看 **总存储用量（MB）**、**文件总数** 及 **按类型** 的数量与占用空间。

## 六、数据统计（知识库）

**入口**：在 [知识库管理](https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/admin/knowledge) 点击 **「数据统计」**  
**直达**：[知识库数据统计](https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/admin/knowledge/stats)

### 6.1 概览卡片

页面顶部展示：

- **总命中次数**
- **命中率**（百分比）
- **知识库总数**
- **条目总数**

### 6.2 标签页说明

| 标签 | 内容 |
|------|------|
| **命中排行** | 各知识条目的命中次数排行（含所属知识库）。 |
| **未命中问题** | 用户提问未命中知识库的记录：问题内容、场景、出现次数、首次记录时间。 |
| **命中率趋势** | 按日期的命中次数、未命中次数及推算的日命中率。 |
| **各知识库分布** | 各知识库命中次数及占总命中量的占比。 |

点击 **「返回」** 可回到知识库列表页。

## 七、终端用户侧：AI 对话与知识检索（H5 / 小程序 / App）

以下说明面向**最终用户**行为；管理员可用于验收与培训。

### 7.1 访问 H5

打开：[H5 前端](https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/)，进入 AI 对话相关页面（具体入口以产品导航为准）。对话页 URL 通常为：  
`https://newbb.bangbangvip.com/autodev/3b7b999d-e51c-4c0d-8f6e-baf90cd26857/chat/{会话ID}`

### 7.2 检索与展示逻辑（概要）

1. 用户发送文本消息后，服务端按配置执行 **精确匹配 → 语义匹配 → 关键词匹配** 等多级策略（以检索策略配置为准）。
2. 命中结果以 **知识卡片** 形式展示，可包含 **文本、图片、视频、商品** 等信息。
3. 用户可对单条知识命中进行 **点赞 / 点踩** 反馈；反馈会提交到后端用于体验优化与统计（需用户已登录且接口成功）。

### 7.3 小程序与 Flutter App

行为与 H5 一致：在 AI 对话中自动展示知识检索结果与知识卡片，并支持点赞/点踩（具体入口与登录方式以各端发布版本为准）。

## 八、管理员日常使用建议

1. **先配存储**：确保 COS 启用且连接测试通过，再上传富媒体资源，避免对话中图片/视频无法加载。  
2. **再建知识库与条目**：保证条目为 **启用**，关键词尽量覆盖用户口语说法。  
3. **配置检索策略与场景绑定**：让不同业务场景命中对应知识库。  
4. **配置兜底策略**：避免无命中时体验断裂。  
5. **定期查看数据统计**：根据「未命中问题」补充条目，根据「命中排行」优化高频内容。
