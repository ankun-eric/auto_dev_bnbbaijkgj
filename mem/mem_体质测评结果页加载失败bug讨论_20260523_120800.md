# 体质测评结果页加载失败 Bug 讨论记录

## 时间
2026-05-23 12:08

## 讨论主题
用户反馈体质测评做完题目后，在 AI 对话界面中点击结果卡片的"查看详情"按钮，跳转到结果详情页后显示"加载失败"。

## 关键信息

### Bug 现象
- 入口：AI 对话界面中的体质测评结果卡片 → 点击"查看详情"
- 表现：跳转到 `/tcm/result/{id}` 后，短暂加载 → Toast 提示"加载结果失败" → 页面空白
- 持续时间：从上线开始就一直存在
- 影响范围：所有用户的结果卡片均会复现

### 控制台报错
1. `GET /api/constitution/result/37 → 404 (Not Found)`
2. `Uncaught TypeError: Cannot create property '_hide' on number '14'`

### 根因分析
- AI 对话中卡片跳转使用 `answerId`（QuestionnaireAnswer 表 PK），值为 37
- 后端接口 `GET /api/constitution/result/{id}` 期望 `diagnosis_id`（TCMDiagnosis 表 PK）
- 两张表主键自增序列独立，37 在 TCMDiagnosis 表中不存在 → 404
- JS TypeError 是 404 触发的次生错误

### 关键代码位置
- 前端跳转逻辑：`h5-web/src/app/(ai-chat)/ai-home/page.tsx` 第 4914-4918 行
- 前端结果页：`h5-web/src/app/tcm/result/[id]/page.tsx` 第 304 行
- 后端接口：`backend/app/api/constitution.py` 第 250 行 `get_full_result()`

### 修复方案
1. 后端问卷提交接口新增返回 diagnosis_id 字段
2. 前端跳转逻辑改用 diagnosis_id
3. 新增 answer_id → diagnosis_id 反查接口
4. 结果接口双重 ID 查询兼容
5. 全面排查所有入口
6. 前端 404 容错提示优化

### 用户确认的修复优先级
- 优先级：高（需尽快修复，已影响用户正常使用）
- 修复范围：主修 AI 对话入口 + 全面排查所有入口
