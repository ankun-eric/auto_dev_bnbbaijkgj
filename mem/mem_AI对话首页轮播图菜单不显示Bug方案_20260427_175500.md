# 历史记忆：AI 对话首页轮播图与菜单不显示 Bug 修复方案沟通

## 时间
2026-04-27 17:55

## 用户诉求
用户反馈 AI 对话版首页的轮播图、六宫格功能卡片、底部快捷标签栏全部不显示，旧版菜单模式下正常。

## 沟通过程与关键决策

### Bug 现象确认
- 轮播图区域完全不显示（连占位都没有）
- 六宫格和底部快捷标签栏都不显示
- 后台已配置过数据
- 旧版菜单模式正常，切到 AI 对话版本后才出现

### 根因定位
前端 API 请求路径与后端路由不匹配：
1. 轮播图：前端请求 `/api/h5/home-banners`，后端路由为 `/api/home-banners`（多了 `h5/`）
2. 功能按钮：前端请求 `/api/function-buttons`，后端路由为 `/api/chat/function-buttons`（少了 `chat/`）

### 用户决策
- 数据来源：两种模式共用同一套后台配置数据（选 A）

## 产出文件
- Bug 修复方案文档：`docs/BUG_FIX_AI对话首页轮播图菜单不显示_20260427_175500.md`

## 修复要点
- 仅修改 1 个前端文件（ai-home 页面组件）中的 2 个 API 路径字符串
- 不涉及后端、数据库、Admin 后台的任何改动
