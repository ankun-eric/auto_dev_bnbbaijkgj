# H5 登录页 LOGO 无法显示 Bug 沟通记录

## 时间
2026-05-01 00:30

## 用户反馈
H5 移动端登录页面的 LOGO 看不到了

## 关键信息
- **涉及端**：H5 移动端网页
- **之前状态**：之前一直能看到 LOGO，但记不清什么时候开始看不到的
- **现象**：LOGO 位置显示系统默认占位符（绿色渐变圆形 + 🌿图标）
- **管理后台**：之前上传过自定义 LOGO，现在管理后台里也看不到了
- **近期操作**：近期做过系统重新部署或更新
- **用户期望**：重新在管理后台上传一张新的 LOGO 图片即可
- **重新上传结果**：上传操作提示"成功"，但页面上依然看不到 LOGO 图片
- **加载表现**：页面加载时有一瞬间空白，然后显示占位图

## 根因分析
文件写入与读取链路断裂：
1. 文件可能存储在 Docker 容器内部，容器重建后丢失
2. Nginx 反向代理未配置上传文件的静态资源访问路由
3. 后端生成的图片 URL 缺少部署环境路径前缀
