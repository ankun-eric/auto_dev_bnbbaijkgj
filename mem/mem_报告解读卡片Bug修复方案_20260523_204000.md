# 历史记忆：报告解读卡片 Bug 修复方案梳理

## 时间
2026-05-23 20:40

## 用户需求
用户发现报告解读功能存在 2 个 Bug：
1. 报告解读卡片只显示"开始"按钮，没有"相册""拍照"入口
2. 看不到"历史报告"按钮入口（后台已配置但前端没生效）

## 根因分析
- Bug 1：数据迁移将 `ai_function_type` 从 `report_interpret` 改为 `image_capture`，但前端 `resolveCardType` 函数未处理 `image_capture`，落入 default 返回 `navigate`，导致渲染为 NavigateCard
- Bug 2：`capture_purpose` 字段未透传到前端 `ChatCardButton`，`UploadCard` 入口按钮硬编码为 2 个

## 修复方案
5 个修复点，涉及文件：
- ChatCards.tsx：resolveCardType 增加 image_capture case、接口增加 capturePurpose、适配器透传、UploadCard 动态添加历史报告按钮
- ai-home/page.tsx：调用 backendButtonToCardButton 时传入 capture_purpose、onAction 增加 history 跳转

## 用户确认的方案细节
- "历史报告"按钮应出现在 AI 对话中报告解读卡片内，与"相册""拍照"并列
