# 多规格商品价格显示为0（含搜索结果页）— Bug 沟通记录

## 时间
2026-04-28 12:47

## 沟通主题
多规格商品在用户端列表页和搜索结果页价格显示为 ¥0 的 Bug 梳理

## 关键结论

### Bug 现象
- 多规格商品在 H5、Flutter APP、微信小程序三端的**服务列表页**和**搜索结果页**价格均显示 ¥0
- 管理后台列表显示正确（¥XX起），用户端详情页选规格后价格也正确
- 单规格商品不受影响

### 用户反馈确认
1. 全部端（H5 + APP + 小程序）均存在此问题
2. 显示格式为 ¥0（无小数点）
3. 详情页选规格后价格正确
4. 修复后希望和管理后台一样显示「¥XX起」，"起"字用小一号字体
5. 单规格商品显示正常
6. 搜索结果页也有同样问题（与列表页表现一致，都是 ¥0）
7. 优先级：紧急，影响转化
8. 验证范围：全部端

### 根因
后端商品列表 API 和搜索 API 直接返回商品主表 `sale_price` 字段（多规格商品为 0），未从 SKU 表计算最低价

### 修复方案
- 后端 API 增加 `min_price` 和 `has_multi_spec` 字段
- H5 + APP + 小程序三端的服务列表页和搜索结果页统一适配
