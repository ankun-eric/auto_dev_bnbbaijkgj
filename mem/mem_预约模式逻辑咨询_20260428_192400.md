# 预约模式逻辑咨询

- **时间**：2026-04-28 19:24
- **用户诉求**：询问管理后台【商品体系-商品管理】设置里的【预约模式】中，预约日期、预约时段的逻辑是什么
- **性质**：功能逻辑咨询（非 Bug、非新需求）

## 回复要点

1. 预约模式共 4 种：无需预约、预约日期、预约时段、自定义表单
2. **预约日期**模式：运营配置「提前可预约天数」和「单日最大预约人数」，用户选择某一天预约，按天限流
3. **预约时段**模式：运营配置若干时段（开始时间/结束时间/容量），用户选择具体时段预约，按时段限流
4. 两种模式均需配置「下单预约方式」（下单即预约 / 先下单后预约）
5. 后端对联动字段做严格校验：日期模式要求天数和人数必填且大于 0，时段模式要求至少 1 条时段且容量大于 0

## 涉及关键文件

- 后端模型：`backend/app/models/models.py` — Product 表 appointment_mode / advance_days / daily_quota / time_slots 字段
- 后端校验：`backend/app/schemas/products.py` — _validate_appointment_mode 函数
- 前端页面：`admin-web/src/app/(admin)/product-system/products/page.tsx` — 预约与核销 Tab 的表单联动
- 用户端下单：`h5-web/src/app/checkout/page.tsx` — 根据 appointment_mode 展示预约时间选择
