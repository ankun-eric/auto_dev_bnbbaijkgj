# 健康档案 Tab 数据过滤逻辑查询

## 时间
2026-05-23 20:45

## 用户问题
用户询问：健康档案里，在某个 Tab（成员 Tab）里，查看用药、家庭成员、我的设备等模块时，显示的是 Tab 对应人的数据还是所有人的数据。

## 查询结论

### 用药计划
- **按 Tab 选中的成员过滤**
- 通过 `consultantId` 参数（本人=0，家庭成员=member.id）传给后端 `/api/health-plan/medications/list?consultant_id=X`
- 后端精确过滤：0=本人(family_member_id IS NULL)，>0=指定成员
- 子页面 `/ai-home/medication-plans` 也携带 `member_id` 参数保持过滤

### 家庭成员
- **不受 Tab 影响，显示所有家庭成员**
- 跳转 `/family` 时不携带 member_id 参数
- 后端 `/api/family/members` 只按 current_user.id 查询所有成员
- 这是合理的（家庭成员管理入口）

### 我的设备
- **不受 Tab 影响，显示所有设备**
- 跳转 `/devices` 时不携带 member_id 参数
- 后端 `/api/devices/my` 只按 current_user.id 查询所有绑定设备
- 不按家庭成员维度过滤

## 涉及关键文件
- h5-web/src/app/health-profile/page.tsx（健康档案主页面，成员 Tab 切换逻辑）
- h5-web/src/app/(ai-chat)/ai-home/medication-plans/page.tsx（用药计划列表页）
- h5-web/src/app/family/page.tsx（家庭成员列表页）
- h5-web/src/app/devices/page.tsx（我的设备页）
- backend/app/api/health_plan_v2.py（用药计划后端接口，含 consultant_id 过滤）
- backend/app/api/family.py（家庭成员后端接口）
- backend/app/api/devices_v2.py（设备后端接口）
