# AI 回复标题标记与体检解读流式重复 Bug 修复方案

## 对话时间
2026-04-28 10:50

## 用户诉求
用户反馈菜单模式下两个 Bug：
1. AI 回复内容中 `#` 标题标记原样显示，需要转换为标题样式
2. 体检报告解读 AI 流式输出时出现文字重复乱码（如"左左肾肾结石结石"）

## 关键决策
- Bug 1：用户选择 A - 前端渲染时将标题标记转换为样式展示（非去掉、非后端限制）
- Bug 1 样式：B - 按 Markdown 标题层级做字号区分（`##` 16px、`###` 15px），同时加粗
- Bug 1 层级：A - 只支持 `##` 和 `###` 两级
- Bug 1 效果：A+B - 字号区分 + 加粗 + 标题行上方加额外间距
- Bug 2 场景：A - 只在体检报告解读（首次自动解读）时出现重复
- Bug 2 历史：C - 不确定从何时出现
- Bug 2 最终状态：B - 流式过程中重复，输出完成后文字恢复正常
- 紧急程度：A - 非常紧急
- 交付方式：A - 输出修复方案文档

## 根因分析
### Bug 1
`renderMarkdown` 函数中 HTML 实体转义（`>` → `&gt;`）在 Markdown 解析之前执行，导致后续生成的 `<h2>` 等标签中 `>` 被转义，浏览器无法识别。

### Bug 2
后端 `_heartbeat_wrapped_sse` 中每条 `message.delta` 事件后额外推送 `__compat__` 兼容事件，前端两条事件都命中 `type === 'delta'` 导致同一段文字累加两次。
