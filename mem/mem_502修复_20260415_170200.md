# 502 Bad Gateway 修复 (2026-04-15)

## 用户需求
修复 H5 用户端和管理后台出现 502 Bad Gateway nginx/1.29.8 错误。

## 根因分析
- Gateway Nginx 容器在项目容器重建后，丢失了与项目 Docker 网络 `6b099ed3-7175-4a78-91f4-44570c84ed27-network` 的连接
- Gateway 使用 `resolver 127.0.0.11`（Docker 内置 DNS）解析容器名进行反向代理
- 不在同一网络中时 DNS 解析失败，返回 502

## 修复方式
- `docker network connect 6b099ed3-7175-4a78-91f4-44570c84ed27-network gateway`

## 验证结果
- H5: 200 ✅
- Admin: 200 ✅
- API/health: 200 ✅

## 关键经验
- 项目使用 docker-compose 创建的网络，gateway 容器是独立运行的
- 每次项目容器重建时，网络可能被重建，gateway 需要重新连接
- 应考虑在部署脚本中自动执行 `docker network connect`

## 关键文件
- `docker-compose.prod.yml`: 项目容器编排
- `gateway-routes.conf`: gateway 反向代理路由配置
- `user_docs/502修复_20260415_170100_a3f2.md`: 用户手册
