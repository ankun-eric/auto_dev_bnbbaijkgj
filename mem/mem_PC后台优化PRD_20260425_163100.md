# 记忆：PC 管理后台优化 PRD 需求梳理

- 时间：2026-04-25
- 主题：PC 管理后台（admin + 商家 Web + 商家 H5）优化需求 PRD 梳理
- 类型：需求咨询会话

## 用户原始诉求

用户提出 3 块优化诉求：
1. admin 后台 + 商家 Web 后台增加"个人信息 + 修改密码"页面（在退出登录菜单上方）
2. 商家 H5 后台目前提示去 Web 端改密码，需要 H5 端直接支持改密码
3. 商家 Web 员工管理页：编辑权限改为只读、修复"权限不回显" Bug

后续追加：
- 员工管理新规则：商家老板/店长可创建员工，角色权限固定只读
- 商家 H5 也能创建/启用员工
- 修复 H5 员工管理"启用/停用无效" Bug
- admin"新建商家账号"页面字段精简（去掉用户端昵称、商家端昵称、同时开通商家身份）
- 老板可重置员工密码

## 关键决策结论

### 块 1：admin/商家 PC 个人信息 + 修改密码
- 入口：退出登录上方两个独立菜单
- 个人信息全部只读
- 改密规则：≥8 位+字母数字；改密后强制重登

### 块 2：admin 新建商家账号精简
- 字段仅保留：姓名、手机号、登录密码、所属商家
- 角色固定为老板，状态默认启用
- 列表列同步精简
- 移除"批量导入商家账号"功能
- 不预生成 C 端用户身份；首次 C 端登录时手机号+验证码自动注册

### 块 3：商家 H5 修改密码
- 入口：「我的」Tab 现有"修改密码"位置直接打开
- 改密后强制退出回登录页

### 块 4：员工管理（PC + H5）
- 编辑权限改"查看权限"，只读 + 隐藏保存按钮
- 修复"权限不回显"Bug
- **仅老板可创建员工**（店长不可创建，但保留启停权限）
- 老板可创建：店长/财务/店员（不能创建老板）
- 三端开放（admin + PC + H5）
- 门店归属：老板可多选商家下任意门店
- PC 表单字段：姓名、手机号、角色、所属门店、头像、入职日期、备注（无邮箱）
- H5 简化版：去掉头像、入职日期、备注
- 初始密码：默认手机号后 6 位，员工首次登录强制改密
- 创建后默认启用
- H5 启停 Bug：开关弹回 + 红色 toast，需排查（待联动后端日志）

### 块 5：老板重置员工密码
- 入口：员工列表行操作列新增"重置密码"按钮
- 仅老板可见可用
- 重置方式：弹窗选"默认密码"（手机号后 6 位）或"自定义密码"
- 不发短信，老板线下告知
- 操作前二次确认弹窗
- PC + H5 都开放

### 块 6：C 端用户身份
- 老板/员工被停用/删除不影响 C 端身份
- C 端入口走手机号+验证码，无该手机号自动注册新用户

## 现有 4 角色权限矩阵（不调整）
- 老板 boss：8 模块全开
- 店长 manager：与老板相同模块（8 个全开）
- 财务 finance：可见 工作台/记录/消息/我的/财务对账（不可见 核销/员工管理/门店设置）
- 店员 clerk：可见 工作台/核销/记录/消息/我的（不可见 财务对账/员工管理/门店设置）
