# 报告解读历史记录与就医资料自动写入 Bug 修复方案讨论

## 时间
2026-05-24

## 用户需求
用户反馈之前的修复没有生效，要求排查原因。

## 问题根因
1. 自动写入逻辑 `_auto_sync_report_history` 被放在 `report_interpret.py` 的 `_run_interpret_worker` 中（仅 `POST /api/report/interpret/start` 触发）
2. 用户实际走的是 `chat.py` → `_stream_report_interpret` → `report_interpret_engine.py` 路径（`POST /api/chat/sessions/{sid}/stream`）
3. chat 路径的 done 事件只做了消息持久化（ChatMessage），完全没有调用 sync 逻辑
4. 额外问题：chat 路径下不创建 CheckupReport，`_auto_sync_report_history` 依赖的 `session.report_id` 为空会直接 return

## 讨论结论
1. 历史报告按咨询人（family_member_id）筛选 — 现有代码已实现
2. 采用方案 A：chat 路径解读成功后，先自动创建 CheckupReport，再复用现有 `_auto_sync_report_history` 写入 ReportHistory + MedicalRecord

## 修复方案要点
- 在 `chat.py` 的 `_stream_report_interpret` done 事件中新增：创建 CheckupReport → 回写 session.report_id → 调用 sync
- 将 `_auto_sync_report_history` 从 `report_interpret.py` 提取为公共方法
- 仅当 `meta.card_type == "report_interpret"` 时触发（OCR 失败/兜底不触发）
- 前端无需改动
