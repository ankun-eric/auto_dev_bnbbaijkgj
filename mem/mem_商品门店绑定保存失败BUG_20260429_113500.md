# 商品门店绑定保存失败 Bug 讨论记忆

## 时间
2026-04-29 11:35

## 用户诉求
用户反馈管理后台编辑商品时，选择"适用门店"后保存报错（红色提示"保存失败"），商品基本信息能保存但门店绑定保存不上。

## 关键信息
- 触发场景：编辑已有商品 → 选择适用门店 → 保存
- 所有门店都会出现，选一个或多个都报错
- 不选门店时保存正常
- 商品基本信息保存成功，门店绑定未保存
- 该问题自功能上线以来一直存在

## 根因分析
1. 前端 handleSubmit 将商品保存和门店绑定拆成两个独立 HTTP 请求，缺乏原子性
2. 独立门店绑定接口 PUT /api/admin/products/{id}/stores 在同一次 flush 中执行 delete+insert，可能触发 UniqueConstraint 冲突
3. 前端 saveProductStores 函数吞掉了异常，导致弹窗仍然关闭

## 修复方案
将 store_ids 直接放入商品更新的主请求 payload 中，利用 admin_update_product 已有的 store_ids 处理逻辑，单请求单事务完成保存。同时优化后端 flush 顺序，修复前端错误处理。
