# 预约日期选择超出"提前可预约天数"限制 Bug 咨询

## 时间
2026-04-28 19:53

## 用户反馈
用户设置了"提前可预约天数"为 7 天，但在前端日期选择器中可以选择大于 7 天的日期，怀疑是 Bug。

## 代码排查结论
**确认是 Bug**。

### 根因
- H5 端 checkout 页面的 `DatePicker` 组件只设置了 `min={new Date()}`，没有设置 `max` 属性
- 小程序端 checkout 页面的 `picker` 组件只设置了 `start={{minDate}}`（今天），没有设置 `end` 属性
- 后端 API（`/api/products/{id}`）已经通过 `ProductDetailResponse`（继承 `ProductResponse`）返回了 `advance_days` 字段
- 但前端 H5 的 `ProductInfo` interface 中没有定义 `advance_days` 字段，也没有从 API 响应中读取和使用它
- 小程序端同样没有读取 `advance_days` 来计算 `end` 日期上限

### 涉及文件
- `h5-web/src/app/checkout/page.tsx` — DatePicker 缺少 max 属性
- `miniprogram/pages/checkout/index.js` — picker 缺少 end 属性
- `miniprogram/pages/checkout/index.wxml` — picker 缺少 end 绑定
- Flutter APP 端待确认是否也存在同样问题

### 修复方向
1. 前端从 API 响应中读取 `advance_days`
2. 计算 max 日期 = 今天 + advance_days 天
3. 将 max 日期设置到日期选择器的上限属性中

## 当前状态
正在与用户确认影响范围、处理方式和天数计算逻辑，尚未输出修复方案文档。
