# 守护我的人页面交互细化讨论 - 2026-05-23 22:00

## 讨论主题
「守护我的人」列表页的具体交互设计

## 讨论要点

### 1. 守护者卡片展示信息（同时选了 B+C）
- 头像 + 昵称
- 守护关系开始时间
- 守护者可以查看的权限范围（健康数据、用药提醒等）
- 最近一次查看我健康数据的时间

### 2. 解除守护
- 被守护人有权随时主动解除任何一个守护者，不需要对方同意

### 3. 邀请别人守护自己
- 用户提出在「守护我的人」页面应有"邀请别人守护自己"的功能
- 用户要求先查看小程序端是否已实现
- **调研结果**：小程序端目前只有"守护者邀请被守护人"的正向流程（family-invite 生成邀请码 → family-auth 接受邀请），**反方向**（被守护人邀请别人来守护自己）**尚未实现**
- 待用户确认反向邀请的具体实现方式

### 4. 待确认事项
- 反向邀请功能的实现方式（生成邀请码 / 搜索用户 / 仅引导文案）
- 守护者权限范围是否可由被守护人控制

## 当前需求汇总（10项）
1. ①用药计划列表页竞态 Bug 修复
2. ②设备成员维度展示（本人看全部、非本人看自己的）
3. ③「守护我的人」独立卡片入口
4. ④守护关系允许双重身份
5. ⑤卡片命名：「我守护的人」+「守护我的人」
6. ⑥布局：双卡片并排 + 设备独立入口条在提醒管理下方（方案A）
7. ⑦图标底色：橙/绿/紫，不用蓝色
8. ⑧卡片去掉副标题
9. ⑨「守护我的人」列表页卡片信息：头像+昵称+开始时间+权限范围+最近查看时间
10. ⑩被守护人可随时解除守护者（单方面解除，无需对方同意）
