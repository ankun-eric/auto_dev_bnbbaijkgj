# 记忆：适用门店独立管理需求 PRD 生成

## 时间
2026-04-29 12:08

## 用户原始需求
用户希望将商品编辑弹窗中的"适用门店"功能独立出来，作为商品体系下的独立子菜单，提供双向管理能力。

## 需求背景
此前用户反馈了"商品门店绑定保存失败"的 Bug（商品保存和门店绑定分两步请求，门店绑定步骤失败）。在分析 Bug 后，用户决定将门店绑定从商品编辑弹窗中完全独立出来作为新功能。

## 关键决策记录

### 页面结构
- 新增独立菜单"适用门店"，位于"商品管理"之后
- 双向管理：商品维度 Tab + 门店维度 Tab

### 商品维度 Tab
- 列表字段：商品图片缩略图 + 名称 + 分类 + 价格 + 状态 + 已绑定门店数
- 点击进入门店勾选页

### 门店维度 Tab
- 列表字段：门店名称 + 门店编号 + 营业状态 + 关联商品数
- 点击进入商品勾选页

### 勾选页筛选
- 搜索 + 区域筛选 + 已绑定/未绑定快捷筛选

### 批量操作
- 双向支持，目标端单选、来源端多选
- 底部浮出操作栏（批量绑定/解绑）
- 操作确认弹窗显示数量+目标名称

### 保存方式
- 即时保存 + 批量操作二次确认

### 分页
- 每页 20 条

### 权限
- 跟随商品管理权限

### 状态处理
- 下架商品：仍显示，灰色标明"已下架"，可编辑绑定
- 已关闭门店：仍显示，灰色标明"已关闭"，可编辑绑定
- 已有绑定关系中门店被关闭：保留绑定，标记"门店已关闭"提示

### 商品编辑弹窗改造
- 完全移除门店选择功能
- 改为只读显示门店数量 + "去管理"跳转链接

### 业务规则
- 绑定数量不限制（商品和门店均不限）
- 新建商品默认不绑定，创建后弹出引导
- 新增门店需手动绑定
- 无特殊商品类型
