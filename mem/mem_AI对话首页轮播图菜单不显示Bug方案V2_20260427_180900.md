# 记忆：AI对话首页轮播图菜单不显示 Bug 修复方案 V2

## 时间
2026-04-27 18:09

## 用户诉求
用户反馈 AI 对话版首页的轮播图、功能菜单、底部快捷标签栏全部不显示，要求对照设计稿截图进行修复，并先出修复方案文档。

## 用户确认的关键决策
1. 健康贴士就是轮播图 Banner，用蓝色卡片样式展示
2. 功能菜单使用后台首页菜单数据（home-menus），不用 function-buttons
3. 菜单项需要灰色说明文字（如"智能问诊"、"报告解读"），需要后端新增 description 字段
4. AI 对话版和旧版菜单模式共用同一套后台配置数据

## 根因
1. Banner API 路径不匹配：前端 `/api/h5/home-banners` vs 后端 `/api/home-banners`
2. 功能菜单数据源错误：应使用 `/api/home-menus` 而非 `/api/function-buttons`
3. 底部快捷标签 API 路径不匹配：前端 `/api/function-buttons` vs 后端 `/api/chat/function-buttons`

## 修复方案（共 7 项）
1. 修正轮播图 API 路径
2. 功能菜单改用 home-menus 数据源
3. 底部快捷标签改用正确路径
4. Banner 改为蓝色卡片样式
5. 新增空对话状态提示
6. 后端菜单新增 description 字段
7. 底部标签增加 emoji 图标
