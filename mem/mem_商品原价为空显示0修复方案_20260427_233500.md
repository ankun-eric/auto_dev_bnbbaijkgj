# 历史记忆：商品原价为空时显示 0 的 Bug 修复方案

## 对话时间
2026-04-27 23:35

## 用户需求
用户反馈商品管理中，原价不填时应该为空，但系统显示 0；要求前端不显示 0。

## 关键确认信息
- 问题出现在所有地方（管理后台 + 用户端详情 + 用户端列表）
- 期望效果：只在用户端不展示，管理后台编辑框保持空白即可
- 原价业务含义：选填的划线价，不填说明不需要展示划线价
- 管理后台表现：新建时输入框是空的，但保存后再编辑时变成了 0
- 历史数据：原价为 0 的都是没有填的，不存在真实原价为 0 的情况

## 根因定位
1. 后端模型 `Product.original_price` 设为 `nullable=False`
2. 后端 Schema `original_price: float` 为必填
3. Admin 前端提交时 `values.original_price ?? 0` 兜底为 0
4. Admin 前端回填时 `Number(raw.original_price ?? 0)` 转为 0

## 修复方案概要
- 后端：模型改 nullable=True，Schema 改 Optional[float] = None
- Admin 前端：提交不兜底为 0，回填保持 undefined
- H5 前端：类型定义改为 number | null（已有判断逻辑可自动修复）
- 数据库：历史数据 UPDATE products SET original_price = NULL WHERE original_price = 0
