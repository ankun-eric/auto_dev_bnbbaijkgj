# 预约模式切换导致价格库存归零 Bug 讨论记忆

## 时间
2026-04-28 18:49

## 用户诉求
用户反馈：商品体系-商品分类-商品信息中，选中【预约日期】预约模式后保存，再打开编辑发现价格和库存都变成了0。

## 关键信息
- 商品规格：统一规格
- 触发条件：编辑已有商品 → 切换到"预约与核销"Tab → 修改预约模式为"预约日期" → 保存 → 价格库存归零
- 复现率：100% 必现
- 用户还问了「下单预约方式」是否有关系，确认无关

## Bug 根因
Tab 切换使用了条件渲染（if/else），切走后基本信息区的 Form.Item 被 React 卸载，form.getFieldsValue() 取不到值返回 undefined，payload 构建时 undefined ?? 0 = 0，导致 0 被写入数据库。

## 修复方案
1. 方案一（根治）：将 Tab 条件渲染改为 CSS display 控制，所有 Tab 始终挂载
2. 方案二（兜底）：payload 构建时 fallback 到编辑前原始值而非 0
