# 订单详情预约信息不显示 Bug 讨论记录

## 时间
2026-04-29

## 用户诉求
Admin 管理后台「商品体系 → 订单明细」中，打开订单详情看不到预约信息。

## 根因
后端 API 正确返回了 `appointment_time`、`appointment_data`、`store_name` 字段，但 admin-web 前端的 `OrderItem` 接口、`UnifiedOrder` 接口和 `mapOrder` 函数均未映射这些字段，导致预约信息卡片展示代码中条件判断永远为 false。

## 用户确认的展示需求
- 预约日期
- 预约时段
- 预约状态
- 关联门店名称

## 处理结果
生成了 Bug 修复方案文档，用户将在后续会话中执行修复。

## 涉及文件
- `admin-web/src/app/(admin)/product-system/orders/page.tsx`（前端，需修复）
- `backend/app/schemas/unified_orders.py`（后端，无需修改，已正确返回）
